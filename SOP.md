# SOP：学术报告内容生产流程

> 从选题到一篇高质量学术报告的内容生产全流程。
> 后续可扩展为 GitHub Pages 论文阅读站（参考 embodied-ai-reading-station 架构）。

---

## 适用场景

- 课程报告（综述 / 论文阅读报告）
- 论文阅读站内容生产（md → html）
- 学术写作训练

---

## 流程总览

```
[1] 选题定结构 → [2] 初稿撰写 → [3] 深入研究扩充 → [4] 质量审查 → [5] 发布/归档
```

---

## Phase 1：选题与结构设计

**输入**：课程要求 / 论文 PDF / 研究方向
**输出**：确定的章节大纲

### 综述报告结构（9 章模板）

```
1. 引言（背景 + 市场数据 + 问题陈述）
2. 基本概念与体系架构
3. 核心技术方向 A
4. 核心技术方向 B
5. 核心技术方向 C
6. 应用场景（3-4 个典型场景，含数据支撑）
7. 安全与挑战
8. 平台框架与标准化
9. 发展趋势与展望
+ 参考文献（≥20 篇，含近 2 年顶会）
```

**选题原则**：

- 领域足够大，能撑起 9 章内容（不要选太窄的子问题）
- 有明确的技术分层（概念 → 技术 → 应用 → 趋势）
- 近 2 年有活跃研究产出（能找到足够参考文献）

### 论文阅读报告结构（6 章模板）

```
论文信息（标题 / 作者 / 机构 / 发表 / 开源链接）
1. 研究动机（背景 + 现有方案不足 + 关键观察）
2. 系统设计（架构 + 各模块详解 + 算法）
3. 实验评估（设置 + 主要结果 + 消融实验）
4. 对比分析（与相关工作的表格对比，≥3 张表）
5. 批判性思考（优势 + 局限 + 改进方向）
6. 总结与启发
+ 参考文献
```

**选论文原则**：

- 顶会/顶刊（CCF A/B）优先
- 有开源代码的优先（方便验证理解）
- 与综述主题强相关（能互相引用）

## Phase 2：初稿撰写

**文件命名**：`综述报告-<主题>.md` / `论文阅读报告-<论文名>.md`

### 写作方法

**综述报告**：

1. 先写引言——用 3 个量化数据点建立问题的紧迫感
2. 概念章节画清边界——定义核心术语，区分相近概念（如 MEC vs 雾计算 vs Cloudlet）
3. 技术章节按"问题 → 方法 → 代表工作 → 局限"展开
4. 应用章节每个场景含：需求特征 + 技术方案 + 实际案例 + 数据指标
5. 趋势章节每个方向含：驱动力 + 技术路径 + 代表工作 + 挑战

**论文阅读报告**：

1. 研究动机——先说清"为什么现有方案不行"，列出具体缺陷
2. 系统设计——从整体架构图入手，再逐模块拆解，含算法伪代码或公式
3. 实验评估——提取关键数据点，不要照搬全部表格，聚焦"比 baseline 好多少"
4. 对比分析——自己构造对比表（不是论文里的），维度包括：方法类型、通信开销、内存需求、适用场景
5. 批判性思考——不能只说好话，必须指出 ≥3 个局限和改进方向

### 写作规范

- 摘要：150-250 字，一段话概括全文贡献
- 关键词：5-8 个，用分号分隔
- 每段有主题句（第一句概括本段要点）
- 数据有出处（市场报告 / 论文实验数据 / 标准文档）
- 对比表格用 markdown table，表头加粗
- 参考文献格式：`[序号] 作者, "标题," 会议/期刊, 年份.`

## Phase 3：深入研究扩充

初稿完成后，逐章检查并扩充，确保每章有足够深度。

### 扩充策略

| 内容类型 | 扩充方法 | 目标 |
|----------|----------|------|
| 引言 | 补充市场数据、行业报告数字 | ≥3 个量化数据点 |
| 概念 | 加对比表格区分相近术语 | 读者能一眼看清区别 |
| 技术 | 补充算法流程、公式、复杂度分析 | 不只是文字描述 |
| 应用 | 加具体案例和性能数据 | 每个场景有实际数字 |
| 对比 | 自建多维度对比表 | ≥3 张表，≥4 个维度 |
| 趋势 | 引用最新工作（2024-2025） | 每个方向 ≥2 篇引用 |
| 参考文献 | 补充至目标数量 | 综述 ≥20，论文报告 ≥15 |

### 深入研究检查清单

- [ ] 每章 ≥ 3 段实质论述（不是一两句话）
- [ ] 引言含 ≥ 3 个量化数据点（市场规模/增长率/设备数量）
- [ ] 技术章节含算法描述或公式（不只是定性描述）
- [ ] 对比表格 ≥ 3 张，每张 ≥ 4 个对比维度
- [ ] 参考文献数量达标且格式统一
- [ ] 趋势/展望章节含 ≥ 4 个方向，每个有驱动力+路径+挑战
- [ ] 无空洞段落（删除"众所周知""不言而喻"等废话）
- [ ] 章节间有逻辑衔接（上一章结尾引出下一章）

## Phase 4：质量审查（10 轮迭代）

每轮聚焦一个维度，找出问题并修复。

| 轮次 | 审查维度 | 检查要点 |
|------|----------|----------|
| R1 | 结构完整性 | 章节是否齐全、层次是否清晰、是否有遗漏的重要话题 |
| R2 | 论证深度 | 每个论点是否有证据支撑、是否只停留在表面描述 |
| R3 | 数据准确性 | 引用的数字是否有出处、是否是最新数据、单位是否正确 |
| R4 | 对比表格质量 | 维度是否合理、数据是否准确、是否遗漏重要对比对象 |
| R5 | 逻辑连贯性 | 段落间是否有跳跃、因果关系是否成立、结论是否有前文支撑 |
| R6 | 批判性思考 | 是否只说好话、局限分析是否具体、改进方向是否可行 |
| R7 | 参考文献 | 格式是否统一、是否有遗漏的重要工作、年份分布是否合理 |
| R8 | 术语一致性 | 同一概念是否用了不同名称、缩写是否首次出现时有全称 |
| R9 | 可读性 | 句子是否过长、是否有歧义、段落是否过大需要拆分 |
| R10 | 原创性 | 是否有自己的见解、对比表格是否是自己构造的、批判是否有独立思考 |

### 每轮操作模式

```
1. 通读全文，列出该维度下的所有问题（≥5 个）
2. 按严重程度排序
3. 逐一修复
4. 修复后重新检查该维度是否通过
```

## Phase 5：发布与归档

### 当前归档方式

```
explorations/competitions/<课程名>/
├── 综述报告-<主题>.md          # 源文件（唯一真相）
├── 论文阅读报告-<论文名>.md    # 源文件
└── SOP-学术报告生产流程.md     # 本文件
```

### 扩展为论文阅读站（GitHub Pages）

目标架构（参考 embodied-ai-reading-station）：

```
reading-station/
├── .github/workflows/deploy.yml   # GitHub Actions 自动部署
├── papers/                         # 每篇论文一个目录
│   ├── jupiter/
│   │   ├── index.md               # 论文阅读笔记
│   │   └── meta.yaml              # 元数据
│   └── edge-computing-survey/
│       ├── index.md
│       └── meta.yaml
├── deck/                           # HTML 幻灯片（可选）
├── site/                           # 站点生成器
│   ├── scripts/build.mjs           # md → html 构建脚本
│   ├── src/template.html           # 页面模板
│   └── content/                    # 生成的 html
├── index.html                      # 首页（论文卡片列表）
├── reading-progress.md             # 阅读进度追踪
└── README.md
```

### 从当前产物迁移的步骤

1. 每篇 md 报告拆入 `papers/<slug>/index.md`
2. 提取元数据到 `meta.yaml`
3. 编写构建脚本：遍历 papers/ → 生成 HTML 卡片 + 详情页
4. 配置 GitHub Actions：push 触发 build → deploy 到 gh-pages
5. 持续添加新论文，每篇走 Phase 1-4 流程

### meta.yaml 模板

```yaml
title: "Jupiter: Fast and Resource-Efficient Collaborative Inference"
authors: ["Shengyuan Ye", "Bei Ouyang", "..."]
venue: "IEEE INFOCOM 2025"
year: 2025

github: "https://github.com/ysyisyourbrother/Jupiter"

```

## 内容质量标准

一篇合格的学术报告应满足：

| 维度 | 最低标准 | 优秀标准 |
|------|----------|----------|
| 篇幅 | 综述 ≥5000 字，论文报告 ≥3000 字 | 综述 ≥8000 字，论文报告 ≥5000 字 |
| 参考文献 | 综述 ≥15 篇，论文报告 ≥10 篇 | 综述 ≥25 篇，论文报告 ≥15 篇 |
| 对比表格 | ≥2 张 | ≥4 张，含自建维度 |
| 批判性分析 | 有局限性讨论 | 有具体改进方案 + 可行性分析 |
| 数据支撑 | 关键论点有引用 | 每个技术方向有量化对比 |
| 原创见解 | 有总结性观点 | 有独立构造的分析框架 |
