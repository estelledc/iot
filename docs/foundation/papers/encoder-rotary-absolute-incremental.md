# 绝对值与增量式旋转编码器在IoT中的对比
> **难度**：🟢 初级 | **领域**：位置反馈传感器 | **阅读时间**：约 18 分钟

## 引言

想象你家有一个旋转门锁，每次锁门你都要转到某个特定位置。一种方式是记住"从上电开始转了多少格"——这是增量式编码器的思路，断电就忘了。另一种方式是锁的每个角度都有唯一的标记，你任何时候看一眼就知道现在转到了哪——这是绝对值编码器的思路。两种方式都能知道角度，但哲学完全不同：一个是"记住我走了多远"，另一个是"我知道我在哪里"。

## 1 旋转编码器概述

### 1.1 什么是旋转编码器

旋转编码器将轴的旋转运动转换为电信号：

- 测量旋转角度（位置）
- 测量旋转速度
- 测量旋转方向
- 作为电机控制的反馈元件

### 1.2 核心分类

旋转编码器按照位置信息的保持方式分为两大类：

| 特性 | 增量式 | 绝对值式 |
|------|--------|---------|
| 位置信息 | 相对（从某个起点开始计数） | 绝对（每个角度有唯一编码） |
| 断电后 | 丢失位置 | 保持位置 |
| 上电后 | 需要回零/homing | 立即知道位置 |
| 输出信号 | 脉冲序列 | 编码值 |
| 成本 | 低 | 高 |

## 2 增量式编码器原理

### 2.1 工作机制

增量式编码器的码盘上有均匀分布的标记（槽或磁极），传感器检测这些标记通过时的脉冲：

```
      码盘截面：
      ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
      │█│ │█│ │█│ │█│ │█│    ← 不透光/磁极
      └─┘ └─┘ └─┘ └─┘ └─┘
      A通道: ┌┐ ┌┐ ┌┐ ┌┐ ┌┐
             └┘ └┘ └┘ └┘ └┘
      B通道:  ┌┐ ┌┐ ┌┐ ┌┐ ┌┐
              └┘ └┘ └┘ └┘ └┘   (偏移1/4周期)
```

### 2.2 正交编码（A/B相）

增量式编码器输出两个相位差 90 度的方波信号：

- A 相和 B 相
- 正转时 A 领先 B 90 度
- 反转时 B 领先 A 90 度

通过判断 A/B 的相位关系确定旋转方向。

### 2.3 Z相（索引脉冲）

很多增量式编码器还有第三个通道 Z（或称 I/Index）：

- 每转一圈输出一个脉冲
- 用作机械参考点
- 配合 A/B 计数可实现"半绝对"定位

### 2.4 计数原理

MCU 通过硬件定时器的编码器模式计数：

```
每转脉冲数 = PPR (Pulses Per Revolution)

A/B四倍频计数:
- 计数A上升沿和下降沿
- 计数B上升沿和下降沿
总计数分辨率 = 4 x PPR

例如: 1000 PPR编码器 -> 4000 计数/转 -> 0.09度
```

### 2.5 MCU接口

STM32 的定时器内置编码器模式：

```c
// STM32 HAL: 编码器模式配置
void encoder_init(void) {
    TIM_Encoder_InitTypeDef enc = {0};
    enc.EncoderMode = TIM_ENCODERMODE_TI12;  // A/B双沿计数

    htim3.Init.Prescaler = 0;
    htim3.Init.CounterMode = TIM_COUNTERMODE_UP;
    htim3.Init.Period = 0xFFFF;  // 16位计数器

    HAL_TIM_Encoder_Init(&htim3, &enc);
    HAL_TIM_Encoder_Start(&htim3, TIM_CHANNEL_ALL);
}

int32_t encoder_read(void) {
    return (int32_t)__HAL_TIM_GET_COUNTER(&htim3);
}
```

## 3 绝对值编码器原理

### 3.1 工作机制

绝对值编码器的码盘上每个角度位置都有唯一的编码，直接读出当前角度：

```
4位格雷码码盘示例:

角度    二进制    格雷码
0度     0000     0000
22.5度  0001     0001
45度    0010     0011
67.5度  0011     0010
90度    0100     0110
```

### 3.2 格雷码

为什么用格雷码而不用二进制码？

二进制码的问题：从 0111（7）变到 1000（8）时，4 个 bit 同时翻转，如果某个 bit 先变就出现瞬态错误值。

格雷码的优势：相邻两个码只有 1 个 bit 不同，不存在瞬态错误。

```c
// 二进制 -> 格雷码转换
uint8_t bin_to_gray(uint8_t bin) {
    return bin ^ (bin >> 1);
}

// 格雷码 -> 二进制转换
uint8_t gray_to_bin(uint8_t gray) {
    uint8_t bin = gray;
    bin ^= (bin >> 1);
    bin ^= (bin >> 2);
    bin ^= (bin >> 4);
    return bin;
}
```

### 3.3 单圈与多圈

**单圈绝对值编码器：**

- 在一圈（360度）内位置唯一
- 超过一圈后位置重复
- 知道"在圈内的哪个位置"，不知道"转了几圈"

**多圈绝对值编码器：**

- 内置齿轮计数器记录圈数
- 即使断电也能记住转了几圈（通常用 Wiegand 传感器自供电）
- 典型范围：12位圈数 = 4096 圈
- 成本显著高于单圈

## 4 光电式与磁电式

### 4.1 光电式编码器

结构：LED 光源 + 码盘（玻璃或金属）+ 光电检测器

优势：分辨率高（20bit 以上）、精度好、技术成熟
劣势：怕灰尘油污、怕振动（码盘碎裂）、LED 寿命有限

### 4.2 磁电式编码器

结构：磁化码盘（径向磁化的环形磁铁）+ Hall/AMR/GMR 传感器

优势：抗污染、抗振动冲击、体积小、成本低
劣势：分辨率低于光电式、受外部磁场干扰

### 4.3 选型对比

| 特性 | 光电式 | 磁电式 |
|------|--------|--------|
| 分辨率 | 高（可达23bit） | 中（可达19bit） |
| 抗污染 | 差 | 好 |
| 抗振动 | 一般 | 好 |
| 成本 | 中-高 | 低-中 |
| 功耗 | 较高（LED） | 低 |
| 温度范围 | -20~+85度C | -40~+125度C |

IoT 场景倾向于磁电式：抗恶劣环境、低功耗、小体积。

## 5 分辨率与接口

### 5.1 增量式编码器的分辨率

| PPR | 四倍频后计数/转 | 角度分辨力 |
|-----|----------------|-----------|
| 100 | 400 | 0.9度 |
| 360 | 1440 | 0.25度 |
| 1000 | 4000 | 0.09度 |
| 5000 | 20000 | 0.018度 |
| 10000 | 40000 | 0.009度 |

### 5.2 绝对值编码器的分辨率

| 位数 | 位置数 | 角度分辨力 |
|------|--------|-----------|
| 8 bit | 256 | 1.4度 |
| 12 bit | 4096 | 0.088度 |
| 16 bit | 65536 | 0.0055度 |
| 20 bit | 1048576 | 0.00034度 |

### 5.3 接口类型

**增量式：** A/B/Z 方波输出、RS-422 差分信号（抗干扰长距离）、开路集电极

**绝对值：**

| 接口 | 特点 | 线数 |
|------|------|------|
| 并行 | 每位一根线 | 多 |
| SSI | 同步串行 | 4 |
| BiSS | 双向同步串行 | 4 |
| SPI | 同步串行 | 4 |
| EnDat | Heidenhain 专有 | 4-6 |

IoT 常用 SPI 或 SSI 接口的绝对值编码器，线数少、MCU 容易对接。

### 5.4 绝对值编码器MCU接口示例

以 SPI 接口的 AS5047P 为例：

```c
uint16_t as5047p_read_angle(void) {
    uint16_t cmd = 0x3FFF | 0x4000;  // 读角度命令
    uint16_t raw = SPI_Transfer(cmd);
    uint16_t angle = raw & 0x3FFF;  // 14位角度值
    return angle;  // 0-16383 对应 0-360度
}

float angle_to_degrees(uint16_t raw) {
    return (float)raw * 360.0f / 16384.0f;
}
```

## 6 对比与选型

### 6.1 优劣势对比

| 维度 | 增量式 | 绝对值式 |
|------|--------|---------|
| 成本 | 低（几十元起） | 高（百元起） |
| 断电保持 | 否 | 是 |
| 上电即知位置 | 否（需回零） | 是 |
| 分辨率 | 中-高 | 高 |
| 接口简单性 | 简单（2-3根信号线） | 需要通信协议 |
| 无限旋转 | 天然支持 | 单圈需额外处理 |

### 6.2 选型决策流程

```
需要断电后保持位置吗？
  ├── 否 → 只需速度/相对位移？
  │         ├── 是 → 增量式
  │         └── 否 → 看成本
  │                   ├── 预算紧 → 增量式 + Z相
  │                   └── 预算宽 → 绝对值
  └── 是 → 绝对值（必须）
            ├── 转超过一圈？→ 多圈绝对值
            └── 不超过一圈？→ 单圈绝对值
```

## 7 IoT应用场景

### 7.1 增量式编码器的典型应用

**电机速度反馈：**

```
[增量编码器] ──脉冲──> [MCU定时器] ──计算频率──> [PID速度控制]
```

**流量计脉冲计数：** 涡轮流量计叶轮带磁铁，累计脉冲数 = 流量

**用户界面旋钮：** 无限旋转的旋钮（如汽车音量旋钮），只关心旋转了几格

### 7.2 绝对值编码器的典型应用

- 阀门位置反馈：必须知道开度百分比，断电后不能丢失位置
- 天线指向：上电后立即工作，不允许回零
- 机器人关节：安全关键场景，断电不能丢失

## 8 实战示例：智能阀门执行器

### 8.1 需求

- 工业阀门电动执行器，需知阀门开度（0-100%）
- 断电后仍需知道阀门位置
- 通过 Modbus 与 IoT 系统通信

### 8.2 方案

```
[绝对值编码器(12bit)] ──SPI──> [STM32] ──UART──> [RS-485 Modbus]
    |                            |
  安装在阀杆上               驱动电机 + 位置反馈
```

选型：AEAT-6012（12bit 磁电绝对值编码器），磁环安装在阀杆上。

### 8.3 软件

```c
float read_valve_position(void) {
    uint16_t raw = AEAT6012_ReadPosition();  // 0-4095
    float percent = (float)raw / 4095.0f * 100.0f;
    return percent;
}

// Modbus寄存器映射
// 寄存器0x0001: 阀门开度 (0-1000 = 0.0-100.0%)
// 寄存器0x0002: 编码器原始值 (0-4095)
// 寄存器0x0003: 状态 (bit0=运行 bit1=故障)
```

### 8.4 为什么选绝对值

断电场景：工厂停电后阀门可能因管道压力自行动作。恢复供电后，绝对值编码器立即知道当前阀位，增量式编码器则必须先做 homing（可能驱动阀门到极限位），这在某些工艺中是不允许的。

## 9 设计注意事项

- 机械安装：同轴度偏差引起周期性误差，弹性联轴器吸收偏差，磁电编码器磁铁间距严格按数据手册
- 电气接口：长线传输用 RS-422 差分信号，信号线双绞 + 屏蔽，编码器电源去耦
- 软件抗干扰：脉冲跳变滤波，连续多次采样一致才确认
- 速度限制：注意最大响应频率和MCU定时器输入频率上限

## 总结

增量式和绝对值编码器就像"计步器"和"GPS"的区别：计步器告诉你走了多少步，但从哪开始需要你自己记住；GPS 直接告诉你当前坐标，不需要起点信息。

增量式编码器简单、便宜、适合速度测量和相对位移，但断电丢失位置、需要回零操作是硬伤。绝对值编码器上电即知位置、断电不丢失，但成本高、接口复杂。IoT 场景中的选择原则很明确：需要断电保持位置就选绝对值，只需要速度反馈就选增量式。

磁电式编码器在 IoT 中越来越流行，因为它不怕脏、不怕振、体积小、功耗低，虽然分辨率不及光电式，但 12-14bit 的分辨率对大多数 IoT 应用已经足够。
