# 电子纸E-Paper显示在低功耗IoT中的应用
> **难度**：🟡 中级 | **领域**：低功耗显示技术 | **阅读时间**：约 20 分钟

## 引言

想象一块超市货架上的价格标签：它显示着数字，但你看不到任何电源线，电池似乎永远用不完。这不是魔法，而是电子纸的特性——它就像印在纸上的墨水，写上去就留在那里，不需要电来维持。只有在"翻页"的一瞬间才需要能量。这种双稳态特性使电子纸成为低功耗IoT显示的最佳选择之一。

## 1 电子纸技术原理

### 1.1 电泳显示(Electrophoretic Display)

主流电子纸采用电泳技术，核心结构：

- **微胶囊**：直径50-100微米，内含透明液体
- **带正电白色粒子**：二氧化钛
- **带负电黑色粒子**：碳黑或黑色染料
- **上下电极**：顶部透明ITO电极，底部像素电极

工作过程：顶部加正电压->白色粒子上移->显示白色；顶部加负电压->黑色粒子上移->显示黑色；撤去电压->粒子停留->图像保持。

### 1.2 双稳态特性

- 有电时：施加电场驱动粒子移动
- 断电后：粒子因范德华力和静电作用停留
- 维持显示不需要任何电能

就像可擦写白板：字不会自己消失，只有主动擦才会变。

### 1.3 与LCD/OLED的功耗本质区别

| 特性       | E-Paper        | LCD            | OLED           |
|-----------|----------------|----------------|----------------|
| 维持显示功耗 | 接近零          | 需要背光持续耗电  | 每像素持续耗电   |
| 刷新功耗    | 短时大电流       | 持续小电流       | 持续中等电流    |
| 阳光可读性  | 优秀            | 较差            | 一般           |
| 暗环境可读性 | 需要外部光源      | 有背光可读       | 自发光可读     |

## 2 电子纸显示类型

### 2.1 黑白/三色/灰度

| 类型    | 颜色数 | 刷新时间    | 价格   | 典型应用          |
|--------|-------|------------|--------|------------------|
| 黑白    | 2     | 2-6秒      | 低     | 价格标签、日程表    |
| 三色    | 3     | 6-15秒     | 中     | 促销标签、状态指示   |
| 灰度    | 4-16  | 4-8秒      | 中高   | 电子书、仪表盘     |

三色在黑白粒子外增加红色或黄色粒子，灰度通过调整驱动波形脉冲宽度实现。

## 3 IoT中的优势与局限

### 3.1 核心优势

- **零维持功耗**：显示内容不需要电力维持
- **阳光可读**：反射式显示，光线越强越清晰
- **宽视角**：接近180度
- **超低平均功耗**：更新间隔长时(分钟/小时级)平均功耗极低

### 3.2 关键局限

- 刷新速度慢(2-15秒)，不适合动态内容
- 更新次数有限(典型数百万次)
- 温度敏感，低温刷新更慢
- 无背光，暗环境需外部照明
- 全刷新过程有黑白闪烁

### 3.3 适用场景判断

| 场景              | 是否适合   | 理由                  |
|------------------|-----------|----------------------|
| 货架价格标签       | 适合      | 更新少，零功耗维持      |
| 室内温湿度计       | 适合      | 更新频率低，阳光可读    |
| 智能手表表盘       | 部分适合   | 交互时刷新太慢         |
| 实时数据仪表盘     | 不适合    | 需要高刷新率          |

## 5 SPI接口与Busy引脚

### 5.1 接口信号

| 引脚    | 方向       | 功能                  |
|--------|-----------|----------------------|
| CS     | MCU->EPD  | 片选，低有效           |
| SCLK   | MCU->EPD  | SPI时钟               |
| MOSI   | MCU->EPD  | SPI数据               |
| DC     | MCU->EPD  | 命令(0)/数据(1)选择    |
| BUSY   | EPD->MCU  | 忙信号，高=正在刷新     |
| RST    | MCU->EPD  | 硬件复位，低有效        |

### 5.2 Busy引脚的重要性

电子纸刷新是物理过程，需数秒完成。BUSY=高期间MCU不可发送命令，否则显示异常。

```c
void epd_wait_busy(void) {
    while (gpio_get_level(EPD_BUSY_PIN) == 1) {
        vTaskDelay(pdMS_TO_TICKS(10));
    }
}
```

## 6 显示刷新模式

### 6.1 全刷新(Full Refresh)

流程：全黑->全白->目标图像(3次扫描)。消除残影效果好，但有黑白闪烁，时间长2-15秒。建议每10-50次部分刷新后做一次全刷新。

### 6.2 部分刷新(Partial Refresh)

只刷新变化区域，不经过全黑全白。速度快0.3-2秒，无闪烁，但残影会逐渐累积。

| 特性   | 全刷新         | 部分刷新       |
|-------|---------------|---------------|
| 速度   | 2-15秒        | 0.3-2秒       |
| 闪烁   | 有            | 无            |
| 残影   | 清除          | 累积          |

```c
epd_set_partial_mode(x_start, x_end, y_start, y_end);
epd_write_framebuffer(partial_buf);
epd_refresh_partial();
epd_wait_busy();
```

## 7 驱动波形与LUT

### 7.1 波形概念

驱动波形是施加在像素电极上的电压序列，不同序列实现白转黑、黑转白或保持不变。

### 7.2 LUT(查找表)

LUT定义完整波形序列，写入控制器寄存器。标准LUT由面板厂商提供保证可靠；自定义LUT可优化速度但需验证。

| 参数     | 加速方案      | 标准方案      |
|---------|-------------|--------------|
| 刷新时间  | 更短(1-3秒)  | 正常(4-6秒)   |
| 残影     | 更严重       | 较轻          |
| 可靠性   | 需验证       | 厂商保证      |

## 8 功耗分析

### 8.1 各状态功耗

| 状态       | 电流        | 持续时间   | 功耗特征   |
|-----------|-----------|-----------|-----------|
| 深睡眠     | <5uA      | 持续       | 接近零     |
| 待机(保持) | <10uA     | 持续       | 接近零     |
| 刷新中     | 10-40mA   | 2-15秒    | 短时脉冲   |

### 8.2 平均功耗计算

关键公式：平均电流 = (刷新电流 * 刷新时间) / 更新间隔

示例：2.13寸电子纸每小时更新一次

- 刷新电流20mA，刷新时间5秒，间隔3600秒
- 平均电流 = (20mA * 5s) / 3600s = 28uA
- 加深睡眠5uA，总计约33uA

CR2450(620mAh)寿命：620/0.033 = 18788小时 = 约783天

## 9 IoT实例：智能价格标签

### 9.1 需求

2.13寸黑白电子纸，每小时更新价格，CR2450供电，目标寿命1年以上。

### 9.2 电池寿命计算

| 状态       | 电流    | 时间占比   | 平均贡献   |
|-----------|--------|----------|-----------|
| 深睡眠     | 3uA    | 99.86%   | 3.0uA     |
| 唤醒+通信  | 30mA   | 0.01%    | 3.0uA     |
| 电子纸刷新  | 20mA   | 0.04%    | 8.0uA     |
| **合计**   |        |          | **14.0uA**|

CR2450(620mAh)理论寿命约5年，实际考虑自放电和温度约2-3年。

### 9.3 软件架构

```c
void main_task(void) {
    epd_init();
    epd_full_refresh(splash_screen);

    while (1) {
        enter_deep_sleep(3600);  // 1小时
        price_data_t price = fetch_price_from_server();
        render_price_tag(&price, framebuffer);

        if (has_changed(framebuffer, prev_framebuffer)) {
            diff_update(framebuffer, prev_framebuffer);
            epd_partial_refresh(framebuffer);
            memcpy(prev_framebuffer, framebuffer, FB_SIZE);
        }

        if (++refresh_count >= 24) {  // 24小时全刷新
            epd_full_refresh(framebuffer);
            refresh_count = 0;
        }
    }
}
```

## 10 差分更新与帧缓冲

### 10.1 帧缓冲策略

MCU需维护帧缓冲，2.13寸(250x122)需3813字节。大面板(800x480)需48000字节，需外部SRAM或IT8951控制器。

### 10.2 差分部分更新

只传输变化像素，减少刷新面积和时间：

```c
void diff_update(const uint8_t *new_fb, const uint8_t *old_fb) {
    int x_min = EPD_WIDTH, x_max = 0;
    int y_min = EPD_HEIGHT, y_max = 0;

    for (int y = 0; y < EPD_HEIGHT; y++) {
        for (int x = 0; x < EPD_WIDTH; x += 8) {
            int idx = (y * EPD_WIDTH + x) / 8;
            if (new_fb[idx] != old_fb[idx]) {
                if (x < x_min) x_min = x;
                if (x > x_max) x_max = x;
                if (y < y_min) y_min = y;
                if (y > y_max) y_max = y;
            }
        }
    }

    if (x_max >= x_min) {
        epd_set_partial_window(x_min, x_max, y_min, y_max);
        epd_write_partial(new_fb);
        epd_refresh_partial();
        epd_wait_busy();
    }
}
```

## 11 集成设计要点

### 11.1 深睡眠与RTC唤醒

```c
epd_deep_sleep();            // 电子纸进入低功耗
rtc_set_alarm(3600);         // 1小时后唤醒
HAL_PWR_EnterSTOPMode(...);  // MCU深睡眠
// 唤醒后重新初始化
system_clock_reconfig();
epd_init();
```

### 11.2 更新策略

| 策略     | 触发方式          | 适用场景          |
|---------|-----------------|------------------|
| 定时刷新  | RTC定时唤醒       | 价格标签、日程表    |
| 事件驱动  | 无线唤醒/中断      | 传感器报警、通知    |
| 混合策略  | 定时 + 事件       | 大多数IoT场景      |

### 11.3 温度补偿

低温下刷新时间需延长，高温需限制刷新频率。部分控制器支持根据温度选择不同LUT。

### 11.4 天线与面板布局

无线模块天线应远离电子纸面板(间距>10mm)，电子纸面板是大面积导体，会影响天线辐射。

## 总结

电子纸的双稳态特性使其成为低更新频率IoT场景的最佳显示方案——刷新时短暂耗电，维持显示几乎零功耗。设计要点：合理选择全刷新/部分刷新策略平衡速度和残影，差分更新减少刷新面积和功耗，深睡眠+RTC唤醒实现极低平均功耗。价格标签等场景下CR2450可供电2-3年。但电子纸不适合高刷新率场景，温度敏感性也需在设计时考虑。
