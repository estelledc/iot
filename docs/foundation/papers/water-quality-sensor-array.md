# 水质多参数传感器阵列：pH / DO / 浊度集成

> **难度**：🟡 中级 | **领域**：水质监测、电化学传感、环境 IoT | **阅读时间**：约 18 分钟

## 日常类比

你去医院体检，医生不会只量体温就下结论——还要测血压、验血、查心电图。每项指标只反映身体一个侧面，综合所有指标才能判断健康状况。

水质监测也是一样：pH 告诉你水"酸不酸"，溶解氧（DO）告诉你水里"够不够气喘"，浊度告诉你水"浑不浑"，电导率告诉你水里"盐多不多"，ORP 告诉你水"氧化能力怎么样"。单独一个参数异常可能说明不了什么，但 pH 降低 + DO 下降 + 浊度升高同时出现，基本可以断定：排污来了。

多参数传感器阵列就是把这几根"体检针"集成到一根探杆上，一次性把水质的关键生命体征全量出来。

## 1. 水质核心参数总览

### 1.1 五大基础参数

| 参数 | 物理含义 | 典型范围 | 单位 | 异常信号 |
|------|---------|---------|------|---------|
| pH | 氢离子活度的负对数 | 0–14 | pH | < 6.5 酸性污染，> 8.5 碱性排放 |
| DO（溶解氧） | 水中溶解的氧气浓度 | 0–20 | mg/L | < 4 mg/L 水生生物缺氧 |
| 浊度 | 悬浮颗粒对光的散射程度 | 0–4000 | NTU | > 5 NTU 饮用水不达标 |
| 电导率（EC） | 水中离子传导电流的能力 | 0–200000 | µS/cm | 骤升 = 盐碱化或工业排放 |
| ORP（氧化还原电位） | 水体氧化/还原能力的度量 | -2000–+2000 | mV | 负值偏大 = 还原性污染 |

### 1.2 参数间的关联性

这些参数并非独立，存在因果和耦合关系：

- **pH 与 ORP**：pH 每升高 1，ORP 约下降 59 mV（Nernst 方程），必须联合解读
- **DO 与 ORP**：DO 高则 ORP 为正（氧化态），DO 耗尽则 ORP 急速转负
- **浊度与 DO**：高浊度阻挡阳光 → 抑制藻类光合 → 间接降低 DO
- **EC 与 pH**：某些酸性排放既降低 pH 又增加 EC

单参数监测容易误判，多参数联合才是可靠的水质评估方式。

## 2. pH 玻璃电极原理

### 2.1 电位法测 pH

pH 玻璃电极是最经典也最"娇贵"的水质传感器。核心是一层对 H⁺ 敏感的玻璃薄膜（厚度仅 0.1–0.3 mm）。

类比：玻璃膜是一扇"只认氢离子"的门。门外是待测水样，门内是已知 pH 的缓冲液（通常 pH 7.0）。两边的 H⁺ 在膜表面发生离子交换，膜两侧形成电位差，服从 Nernst 方程：

```
E = E₀ - (2.303 × R × T) / (n × F) × pH
25°C 时斜率 = 59.16 mV/pH
```

### 2.2 复合电极结构

现代 pH 传感器将指示电极和参比电极集成在同一个壳体内：

```
┌─────────────────────────────────┐
│  pH 指示电极（玻璃球泡）         │
│  ┌───────┐                      │
│  │玻璃球泡│── Ag/AgCl 内参比     │
│  └───────┘                      │
│         │                       │
│  ───────┼──── 参比电极          │
│         │  ┌──────────┐         │
│         └──│KCl 盐桥   │         │
│            │液接界(多孔)│         │
│            └──────────┘         │
│  引线: mV → 高阻抗放大           │
└─────────────────────────────────┘
```

### 2.3 高阻抗难题

pH 玻璃电极内阻极高——典型值 50–500 MΩ。

类比：用极细的吸管喝水——阻力极大，水流极慢。高阻抗信号源稍受干扰就会"失真"。

**问题链**：信号电流 pA 级 → 任何漏电流都引入误差 → 电缆分布电容使响应变慢 → 电磁干扰极易耦合 → 连接器受潮即漂移。

**解决之道**：在电极引出端立即用 CMOS 运放（输入偏置电流 < 1 pA）构成电压跟随器，将 MΩ 级信号源转换为 kΩ 级低阻抗输出。这步必须在探头内部完成。

## 3. 溶解氧（DO）传感器类型

### 3.1 原电池型（Galvanic）

无需外加电压，本身是自发电池。铅阳极 + 银阴极 + PTFE 透气膜 + KOH 电解液。

```
阴极: O₂ + 2H₂O + 4e⁻ → 4OH⁻
阳极: 2Pb → 2Pb²⁺ + 4e⁻
总:   O₂ + 2Pb + 2H₂O → 2Pb(OH)₂
```

上电即用，但阳极消耗导致寿命有限（6–24 月）。

### 3.2 极谱型（Polarographic / Clark 电极）

需外加 0.6–0.8 V 极化电压。金阴极 + Ag/AgCl 阳极。阳极不消耗，寿命受电解液限制。首次使用需 30–60 分钟极化时间。精度和稳定性优于原电池型。

### 3.3 荧光法（Optical / Luminescence）

新一代技术，彻底摆脱膜和电解液：

1. 蓝光 LED（~470 nm）照射荧光染料层
2. 染料发射红光（~620 nm）
3. 氧分子通过"动态猝灭"缩短荧光寿命
4. 测荧光寿命（相位差法）→ 反推 DO 浓度

类比：荧光染料像"跑步选手"——氧分子是"障碍物"。障碍越多跑完越快，测"完赛时间"就知道障碍有多少。

**优势**：无膜无电解液、不受流速影响、无氧消耗、维护周期 1–2 年。
**劣势**：成本高（传统电极 3–5 倍）、染料长期光衰（5–10 年寿命）、硫化氢会"毒害"染料层。

## 4. 浊度测量方法

### 4.1 散射法（Nephelometric）

ISO 7027 和 EPA 180.1 标准方法。860 nm LED 照射水样，90° 方向测散射光强度。

为什么选 90°？小颗粒散射在 90° 对颗粒大小最不敏感——浊度读数主要反映颗粒数量。

### 4.2 比率法（Ratio Turbidity）

多角度同时采集散射光，用比值消除颜色和颗粒大小干扰：

- **90° 散射光**：主信号（对小颗粒灵敏）
- **前向散射（30°）**：对大颗粒灵敏
- **后向散射（138°）**：颜色吸收修正

```
Turbidity = a × I₉₀ + b × I₃₀/I₉₀ + c × I₁₃₈/I₉₀
```

### 4.3 干扰因素

| 干扰源 | 影响机制 | 消除方法 |
|--------|---------|---------|
| 色度 | 吸收入射光降低散射信号 | 比率法 / 双波长法 |
| 气泡 | 界面散射虚增浊度 | 脱气 / 光学窗口设计 |
| 光窗污垢 | 降低透光量 | 机械自清洗 / 空气吹扫 |
| 温度 | 改变折射率 | 温度补偿系数 |

## 7. 信号调理挑战

### 7.1 高阻抗 pH 信号调理

这是多参数系统中最难的部分：

```
pH 电极 (100 MΩ 内阻)
  → 漏电流必须 < 0.1 pA（否则 0.1 pH 误差）
  → 需输入偏置电流 < 1 pA 的运放
  → PCB 保护环 (guard ring)
  → 连接器绝缘电阻 > 10 GΩ
  → 低噪声同轴电缆 (带屏蔽驱动)
```

| 运放型号 | 输入偏置电流 | 失调电压 | 单价 |
|---------|------------|---------|------|
| LMP91200 (TI) | < 1 pA | < 200 µV | ¥25 |
| ADA4530-1 (ADI) | 20 fA | 10 µV | ¥60 |
| OPA129 (TI) | 100 fA | 2 mV | ¥40 |

LMP91200 是 TI 专为 pH 电极设计的前端芯片，内置 ESD 保护和温度补偿。

### 7.2 多通道采样时序

多参数探杆用一颗多通道 ADC 顺序采样，高阻抗通道需更长的采样保持稳定时间：

```c
// 多通道 ADC 采样时序控制
typedef struct {
    uint8_t  channel;
    uint32_t settle_us;   // 通道切换后的稳定时间
    uint32_t samples;     // 过采样次数
} adc_channel_config_t;

static const adc_channel_config_t channels[] = {
    { ADC_CH_PH,   5000, 16 },  // pH: 5ms 稳定 + 16x 过采样
    { ADC_CH_ORP,  3000, 16 },  // ORP: 3ms
    { ADC_CH_DO,   1000,  8 },  // DO: 1ms (低阻抗)
    { ADC_CH_EC,   1000,  8 },  // EC: 1ms
    { ADC_CH_TEMP,  500,  4 },  // 温度: 0.5ms
};
#define NUM_CHANNELS  (sizeof(channels) / sizeof(channels[0]))

void acquire_all_channels(uint16_t *raw_buf) {
    for (int i = 0; i < NUM_CHANNELS; i++) {
        adc_select_channel(channels[i].channel);
        delay_us(channels[i].settle_us);  // 等 S/H 电容充电
        uint32_t sum = 0;
        for (int j = 0; j < channels[i].samples; j++) {
            sum += adc_read_raw();
        }
        raw_buf[i] = (uint16_t)(sum / channels[i].samples);
    }
}
```

### 7.3 地线与屏蔽策略

信号地、模拟地、数字地必须分区，单点连接。pH 输入引脚周围用等电位保护环包围消除漏电流路径。通信侧用磁隔离或光耦防止地环路。

## 8. 多传感器数据采集代码示例

基于 ESP32 + FreeRTOS 的多参数水质采集框架：

```c
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"
#include "esp_log.h"

// 校准参数
typedef struct {
    float ph_slope, ph_offset;
    float do_slope, do_temp_coeff;
    float turb_offset, ec_cell_const;
} calibration_t;

static calibration_t cal = {
    .ph_slope = 59.16f, .ph_offset = 0.0f,
    .do_slope = 1.0f,   .do_temp_coeff = 2.0f,
    .turb_offset = 0.0f, .ec_cell_const = 1.0f,
};

// Nernst 温度补偿
float ph_slope_at_temp(float temp_c) {
    return 59.16f * (temp_c + 273.15f) / 298.15f;
}

// 工程量输出结构
typedef struct {
    float ph, do_mgl, turb_ntu, ec_us, orp_mv, temp_c;
} water_quality_t;

void raw_to_engineering(const uint16_t *raw, water_quality_t *wq) {
    // 温度 (NTC 查表)
    float v_temp = (raw[4] / 4095.0f) * 3.3f;
    wq->temp_c = ntc_voltage_to_temp(v_temp);

    // pH (温度补偿)
    float mv_ph = ((raw[0] / 4095.0f) * 3.3f - 1.65f) * 2000.0f;
    wq->ph = 7.0f + (mv_ph - cal.ph_offset) / ph_slope_at_temp(wq->temp_c);

    // DO (温度 + 盐度补偿)
    float do_pct = ((raw[1] / 4095.0f) * 3.3f) * cal.do_slope * 100.0f / 3.3f;
    wq->do_mgl = do_pct / 100.0f * do_saturation_at_temp(wq->temp_c);

    // 浊度
    wq->turb_ntu = ((raw[2] / 4095.0f) * 3.3f / 3.3f) * 4000.0f - cal.turb_offset;

    // EC (温补到 25°C)
    float ec_raw = ((raw[3] / 4095.0f) * 3.3f / 3.3f) * 200000.0f * cal.ec_cell_const;
    wq->ec_us = ec_raw / (1.0f + 0.02f * (wq->temp_c - 25.0f));

    // ORP
    wq->orp_mv = ((raw[5] / 4095.0f) * 3.3f - 1.65f) * 2000.0f;
}

void water_quality_task(void *pvParameters) {
    uint16_t raw[6];
    water_quality_t wq;
    while (1) {
        acquire_all_channels(raw);
        raw_to_engineering(raw, &wq);
        ESP_LOGI("WQ", "pH=%.2f DO=%.1fmg/L Turb=%.0fNTU EC=%.0fµS/cm ORP=%.0fmV T=%.1f°C",
                 wq.ph, wq.do_mgl, wq.turb_ntu, wq.ec_us, wq.orp_mv, wq.temp_c);
        if (wq.do_mgl < 4.0f) ESP_LOGW("WQ", "DO 缺氧报警!");
        if (wq.ph < 6.5f || wq.ph > 8.5f) ESP_LOGW("WQ", "pH 超限报警!");
        vTaskDelay(pdMS_TO_TICKS(5000));
    }
}
```

## 9. IoT 应用场景

### 9.1 水产养殖

水质直接决定鱼虾生死——最"刚需"的场景。

- DO 最关键：凌晨藻类呼吸耗氧，DO 可能骤降至 2 mg/L
- pH 日波动可达 1–2 单位（藻类光合消耗 CO₂ → pH 升高）
- 浊度反映藻类密度，过高需控藻

典型部署：5 参数探杆 + LoRa + 太阳能供电，DO < 4 mg/L 自动启动增氧机，单点成本 ¥2k–5k。

### 9.2 污水处理

水质传感器用量最大的工业场景，各工艺段需要不同参数组合：

```
进水 → [格栅] → [沉砂] → [初沉] → [生化池] → [二沉] → [消毒] → 出水
 EC,pH    浊度     浊度   DO,pH,ORP   浊度    ORP,余氯
                         ↑核心监测点↑
```

生化池 DO 控制在 2–3 mg/L（过低硝化不充分，过高浪费曝气能耗），ORP < -100 mV 判断缺氧/厌氧段达标。

### 9.3 饮用水安全

从水源地到龙头全链路监测：水源地预警污染、水厂出厂达标（浊度 < 1 NTU、余氯 0.3–0.5 mg/L、pH 6.5–8.5）、管网末端防二次污染。标准依据：GB 5749-2022。

### 9.4 场景对比

| 维度 | 水产养殖 | 污水处理 | 饮用水 |
|------|---------|---------|--------|
| 核心参数 | DO > pH > 浊度 | DO > ORP > pH | 浊度 > 余氯 > pH |
| 精度要求 | 中 | 中高 | 高 |
| 环境恶劣度 | 高 | 极高 | 低 |
| 维护频率 | 1–2 周 | 1–4 周 | 1–3 月 |
| 单点成本 | ¥2k–5k | ¥10k–30k | ¥20k–50k |

## 10. 总结与展望

### 核心要点

1. **多参数联合是刚需**：参数间强耦合，单参数监测容易误判
2. **pH 高阻抗是最大设计挑战**：信号调理必须在探头端完成，保护环和低偏置运放不可省
3. **荧光法 DO 是趋势**：无膜无电解液、低维护、不受流速影响，成本正在下降
4. **浊度测量核心是消除干扰**：比率法、双波长法是当前最成熟方案
5. **多通道 ADC 时序需差异化配置**：高阻抗通道需要更长稳定时间

### 技术趋势

- **固态 pH（ISFET）**：内阻从 MΩ 降到 kΩ，解决高阻抗难题，但长期漂移仍是瓶颈
- **芯片级多参数集成**：ADuCM355 等平台将 pH/DO/EC 传感单元集成于单颗 CMOS 芯片
- **边缘 AI 水质诊断**：TinyML 在 MCU 上运行异常检测，减少通信量
- **自清洗 + 自校准**：机械刷 + 标准液储存舱，实现 6 个月无人值守
- **数字传感器即插即用**：RS-485 Modbus 内置校准参数，换探头无需重新标定

### 对 IoT 开发者的建议

- 入门推荐"国产 5 参数探杆 + ESP32 + LoRa"，成本 ¥1k 以内
- pH 信号调理直接用 LMP91200 等专用前端，不要从头搭
- DO 首选荧光法，长期维护成本更低
- 浊度传感器务必关注光学窗口自清洗方案
- 校准是长期运营的隐性成本——设计时就要考虑校准流程自动化
