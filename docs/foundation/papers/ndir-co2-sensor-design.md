# NDIR 红外 CO2 传感器光路设计与标定

> **难度**：🟡 中级 | **领域**：气体检测、红外光学、环境监测 | **阅读时间**：约 25 分钟

## 日常类比

你走进一间挤满人的会议室，即使背对着人群，也能感觉到"空气变闷了"——其实是 CO2 浓度升高了。那机器怎么"闻到"CO2 呢？

想象一束手电筒光穿过烟雾：烟雾越浓，你看到的光越暗。NDIR（Non-Dispersive Infrared，非色散红外）传感器的原理几乎一样——"手电筒"换成了红外光源，"烟雾"换成了 CO2 气体，"你的眼睛"换成了红外探测器。CO2 分子会"吃掉"特定波长的红外光（4.26 μm），吃掉越多，到达探测器的光越少，浓度就越高。

和电化学传感器不同，NDIR 不消耗任何化学试剂，不产生废气，寿命几乎只取决于光源和探测器——这就是它在室内空气质量监测中越来越主流的原因。

## 1. NDIR 基本原理

### 1.1 气体分子与红外吸收

CO2 是不对称线型三原子分子（O=C=O），具有多种振动模式。其中反对称伸缩振动（ν3）在 4.26 μm（2349 cm⁻¹）处有强烈的基频吸收带，这是 NDIR 检测 CO2 的物理基础。选择 4.26 μm 而非其他吸收峰的原因：

- 吸收截面大，灵敏度高
- 与水蒸气（2.7 μm、5.9 μm）和甲烷（3.3 μm）的吸收峰不重叠，选择性好
- 在此波段有成熟的红外光源和探测器可用

### 1.2 Beer-Lambert 定律

NDIR 的核心定量关系是 Beer-Lambert 定律，描述光通过吸收介质后的衰减：

$$I = I_0 \cdot e^{-\alpha \cdot c \cdot L}$$

其中 $I_0$ 为入射光强度，$I$ 为透射光强度，$\alpha$ 为吸收系数，$c$ 为气体浓度，$L$ 为光程长度。改写为浓度表达式：

$$c = -\frac{1}{\alpha \cdot L} \cdot \ln\left(\frac{I}{I_0}\right)$$

**关键推论**：

1. 光程 $L$ 越长，信号响应越大——高精度 NDIR 用多次反射折叠光路
2. $I/I_0$ 接近 1 时信噪比差——决定检测下限
3. $I/I_0$ 接近 0 时信号饱和——决定检测上限

### 1.3 为什么叫"非色散"

"色散"指用棱镜或光栅把光按波长分开。"非色散"意味着不用分光元件，而是用**窄带滤光片**直接选取目标波长。代价是只能检测一种气体，好处是结构简单、成本低、光通量高。

```
红外光源 → [4.26μm 滤光片] → 气室(CO2) → 探测器
                                    │
                             CO2 吸收 4.26μm 光
```

## 2. 光路设计

### 2.1 单通道结构

最简单的 NDIR 结构：一个红外光源 + 一个带滤光片的探测器。信号通道检测 4.26 μm 透射光，通过 Beer-Lambert 定律计算浓度。

单通道的问题：$I_0$ 不能直接测量。实际做法是先在零气（N2 或干净空气）中标定 $I_0$，后续用存储值计算浓度。但光源老化、温度漂移、气室污染都会导致实际 $I_0$ 变化，而存储值不变——这是单通道精度低、需要频繁标定的根本原因。

打个比方：你用一把尺子量东西，但尺子的刻度在慢慢"缩水"，你却还在用第一次标定的读数——量出来的结果自然越来越偏。双通道方案正是为了解决这个问题。

### 2.2 双通道结构（参考通道）

双通道在单通道基础上增加一个**参考通道**：滤光片中心波长选在 CO2 不吸收的波段（如 3.9 μm），与信号通道共享同一光源和气室。

```
红外光源 → 气室(L) → ┬─ [4.26μm] 信号通道
                      └─ [3.9μm]  参考通道
```

参考通道信号 $I_{ref}$ 只受光源强度变化、气室污染、温度影响，不受 CO2 浓度影响。用比值消除共模干扰：

$$\text{Ratio} = \frac{I_{sig}}{I_{ref}}, \quad c = f(\text{Ratio})$$

**双通道 vs 单通道对比**：

| 特性 | 单通道 | 双通道 |
|------|--------|--------|
| 光源漂移补偿 | 无 | 有 |
| 气室污染补偿 | 无 | 有 |
| 成本 | 低 | 中等 |
| 长期稳定性 | 差（需频繁标定） | 好（ABC 可免维护） |
| 典型精度 | ±50–100 ppm | ±30–50 ppm |
| 适用场景 | 低成本产品 | 室内空气质量监测 |

### 2.3 光程与灵敏度的权衡

根据 Beer-Lambert 定律，光程越长灵敏度越高，但受限于：物理尺寸（IoT 设备通常 20–50 mm）、光损耗、信噪比。工程中常用**多次反射结构**（Herriott 池、White 池）在有限空间内实现长光路——等效光程 200 mm+，物理尺寸仅 30 mm。Senseair S8 就采用专利反射结构，在约 40 mm 物理尺寸内实现等效 200 mm 光程。

## 3. 红外光源与探测器选型

### 3.1 红外光源

| 参数 | 白炽灯（微热板） | 激光器（QCL/ICL） |
|------|-------------------|-------------------|
| 光谱范围 | 宽带（1–20 μm） | 窄带（<0.01 μm） |
| 功耗 | 50–300 mW | 0.5–5 W |
| 寿命 | 5–15 年 | 10+ 年 |
| 成本 | $0.5–5 | $100–1000+ |
| 适用场景 | 消费级 NDIR | 高精度工业级 |

消费级 NDIR 几乎全部使用白炽灯式微热板光源（如 PerkinElmer IRL715、Hamamatsu EMIRS200）。工作方式是周期性通电（1–4 Hz），利用热惯性产生近似方波的红外辐射，探测器检测交流信号以消除直流偏移。白炽灯的优势是光谱覆盖面广（1–20 μm 全覆盖），同一个光源换不同滤光片就能检测不同气体——这也是 NDIR 平台化的基础。

### 3.2 红外探测器

| 参数 | 热电堆 | 热释电 |
|------|--------|--------|
| 原理 | 塞贝克效应（温差→电压） | 极化随温度变化→电荷 |
| 直流响应 | 有 | 无 |
| 斩波需求 | 不必须 | 必须（机械斩波器） |
| 成本 | $2–10 | $5–20 |

**消费级 NDIR 普遍选用热电堆**：无需机械斩波器，配合电调制光源即可工作。双通道方案使用双元热电堆（两个敏感元各贴不同滤光片），封装在同一壳体内，确保两个通道的光路和温度条件完全一致。

热电堆的输出电压通常在 mV 级，需要低噪声放大器（如 OPA333）和 24 位 ADC（如 ADS124S08）才能实现 1 ppm 分辨率。不过消费级模块（如 MH-Z19B）内部已集成信号调理，直接输出数字浓度值。

### 3.3 滤光片

窄带干涉滤光片是 NDIR 选择性的关键：信号通道中心波长 4.26 μm（半宽 0.08–0.18 μm），参考通道 3.90 μm（半宽 0.10–0.20 μm）。峰值透过率 >70%，带外抑制 OD3–OD5。滤光片温度系数约 0.01–0.03 nm/°C，高精度应用需补偿。

## 5. 温度与气压补偿

### 5.1 温度影响

温度通过三条路径影响 NDIR 读数：气体密度变化（$PV=nRT$，温度升→密度降→读数偏低）、滤光片波长漂移、探测器响应变化。综合修正公式（简化版）：

$$c_{corrected} = c_{raw} \cdot \left(1 + \beta_T \cdot (T - T_0)\right)$$

$\beta_T$ 为温度补偿系数（典型 0.002–0.005 /°C），$T_0$ 为标定温度（通常 25°C）。SCD30 内置温度传感器并自动补偿；MH-Z19B 也内置温度补偿。

### 5.2 气压影响

气压降低 → 气体密度降低 → 吸收减弱。高海拔地区必须气压补偿：

$$c_{corrected} = c_{raw} \cdot \frac{P}{P_0}$$

$P_0 = 101.325$ kPa。海拔 2000 m（气压约 80 kPa），不补偿会低估约 20%。大多数消费级 NDIR 不内置气压传感器，需外部 BMP280/BME280 在软件层补偿。SCD30 提供了气压补偿接口。

### 5.3 湿度影响

水蒸气在 2.7 μm 和 5.9 μm 有吸收峰，滤光片带外透过会导致少量交叉干扰。高湿度（RH > 80%）可能在气室壁凝结水膜，严重衰减信号。处理：窄带滤光片（半宽 < 0.12 μm）、疏水涂层气室、软件微调（影响量通常 < 5 ppm，多数场景可忽略）。

## 6. 标定方法

### 6.1 ABC 自标定

ABC（Automatic Baseline Correction）是最常用的免维护标定方法，基于经验假设：**室内 CO2 浓度每几天至少降到接近室外水平（400–420 ppm）**。算法在 7–14 天滑动窗口内取最低读数作为基线，自动修正零点。

```
读取浓度 → 记录到滑动窗口(7-14天) → 窗口最低值 < 基线-阈值？
    → 是：更新基线偏移，应用修正到后续读数
    → 否：等待下一个窗口
```

ABC 本质上是一种"自愈"机制——不需要人工干预，只要环境满足前提条件，传感器就能自己纠偏。但前提条件非常关键。

ABC 的**失效场景**：24 小时有人值守的办公室、温室（CO2 长期偏高）、密闭仓库。失效后果：基线被推高，读数系统性偏低。SCD30 称之为 ASC，原理相同；S8 默认周期 7.5 天；MH-Z19B 默认 24 小时（偏短，容易误判）。

### 6.2 手动零点标定

**室外空气标定法**（最实用）：将传感器放室外通风处，离地 2–3 m，等待 10–15 分钟，设定当前浓度为 400–420 ppm，发送零点标定命令。

**两点标定法**（高精度场景）：零点标定（N2 或室外空气）+ 跨度标定（已知浓度标准气如 2000 ppm），两点确定线性关系。

### 6.3 标定周期建议

| 应用场景 | 标定方法 | 建议周期 |
|----------|----------|----------|
| 普通办公室 | ABC 自动 | 无需手动 |
| 智能家居 | ABC 自动 | 每 6–12 个月检查 |
| 温室 / 农业温室 | 手动零点 | 每 3–6 个月 |
| 工业厂房 | 两点标定 | 每 3–6 个月 |
| 实验室 / 标准 | 两点标定 + 标准气比对 | 每 1–3 个月 |

## 7. 代码示例：通过 UART 读取 MH-Z19B

### 7.1 硬件连接

ESP32 GPIO16 (RX2) ← MH-Z19B TX，GPIO17 (TX2) → MH-Z19B RX，5V → Vin，GND → GND。

### 7.2 读取 CO2 浓度（Arduino / ESP32）

```cpp
#include <HardwareSerial.h>
HardwareSerial co2Serial(2);

void setup() {
    Serial.begin(115200);
    co2Serial.begin(9600, SERIAL_8N1, 16, 17);  // MH-Z19B 默认 9600
    Serial.println("MH-Z19B CO2 Sensor - Preheat 3 min");
}

uint8_t cmd_read[9] = {0xFF, 0x01, 0x86, 0x00, 0x00, 0x00, 0x00, 0x00, 0x79};

int readCO2() {
    co2Serial.write(cmd_read, 9);
    unsigned long t0 = millis();
    while (co2Serial.available() < 9) {
        if (millis() - t0 > 1000) return -1;  // 超时
        delay(10);
    }
    uint8_t buf[9];
    co2Serial.readBytes(buf, 9);
    // 校验和：取 byte[1..7] 之和，0xFF - sum + 1
    uint8_t ck = 0;
    for (int i = 1; i < 8; i++) ck += buf[i];
    ck = 0xFF - ck + 1;
    if (ck != buf[8]) { Serial.println("CRC err"); return -1; }
    return (buf[2] << 8) | buf[3];  // 高字节 + 低字节
}

void loop() {
    int co2 = readCO2();
    if (co2 >= 0) Serial.printf("CO2: %d ppm\n", co2);
    delay(5000);
}
```

### 7.3 Python 读取（树莓派 / Linux）

```python
import serial, time

class MHZ19B:
    """MH-Z19B CO2 传感器 UART 读取类"""
    CMD_READ   = b'\xFF\x01\x86\x00\x00\x00\x00\x00\x79'
    CMD_ABC_ON = b'\xFF\x01\x79\xA0\x00\x00\x00\x00\xE6'

    def __init__(self, port='/dev/serial0'):
        self.ser = serial.Serial(port, 9600, timeout=1.0)

    def _ck(self, d: bytes) -> int:
        return (0xFF - sum(d[1:8]) + 1) & 0xFF

    def read_co2(self) -> int:
        self.ser.write(self.CMD_READ)
        r = self.ser.read(9)
        if len(r) != 9 or self._ck(r) != r[8]:
            return -1
        return (r[2] << 8) | r[3]

    def set_abc(self, on: bool):
        cmd = self.CMD_ABC_ON if on else b'\xFF\x01\x79\x00\x00\x00\x00\x00\x86'
        self.ser.write(cmd); time.sleep(0.1); self.ser.read(9)

    def close(self):
        self.ser.close()

if __name__ == '__main__':
    s = MHZ19B()
    try:
        while True:
            v = s.read_co2()
            if v >= 0:
                tag = "Good" if v < 800 else ("Fair" if v < 1500 else "Poor")
                print(f"CO2: {v} ppm [{tag}]")
            time.sleep(5)
    except KeyboardInterrupt:
        s.close()
```

## 8. IoT 应用场景

### 8.1 HVAC 按需通风

传统 HVAC 按固定风量运行。NDIR CO2 传感器实现**按需通风（DCV）**：CO2 < 800 ppm → 低风量（节能）；800–1200 ppm → 中风量；> 1200 ppm → 全风量。大型办公楼采用 DCV 后通风能耗可降低 20–30%。Senseair S8 就是为这个场景设计的。

### 8.2 农业温室 CO2 施肥控制

光照充足时提高 CO2 到 800–1200 ppm 可增产 20–30%，但 > 2000 ppm 对作物有害。温室场景中 ABC 失效（CO2 长期偏高），必须手动标定；湿度极高需疏水涂层气室；温度变化大需温补传感器（SCD30 或 S8）。

实际部署时，通常将 NDIR 传感器与电磁阀和 CO2 钢瓶联动：传感器读数低于设定下限时打开电磁阀补充 CO2，高于上限时关闭并启动通风。这是一个典型的闭环控制场景，传感器精度直接决定施肥效果和安全性。

### 8.3 室内空气质量（IAQ）监测

| 标准 | CO2 限值 | 适用空间 |
|------|----------|----------|
| WELL 铂金级 | < 600 ppm | 办公室、教室 |
| WELL 银级 | < 800 ppm | 办公室、教室 |
| RESET | < 1000 ppm | 室内公共空间 |
| GB/T 18883 | 日均值 ≤ 1000 ppm | 室内空气质量 |

NDIR CO2 传感器配合 WiFi（ESP32）或 LoRa 可组成分布式 IAQ 监测网络，数据上云实时可视化，超标时推送预警。

### 8.4 典型 IoT 系统架构

```
[NDIR CO2 传感器] → UART/I2C → [MCU (ESP32)]
                                ├─ WiFi → MQTT → 云平台 → App/大屏
                                ├─ LoRa → 网关 → 云平台
                                └─ OLED + 蜂鸣器报警
```

功耗优化：NDIR 功耗较高（15–20 mA @ 5V），可用间歇模式（每 30 秒测量 5 秒）+ ESP32 Light Sleep（待机 0.8 mA），搭配太阳能板 + 锂电池实现户外长期部署。

值得注意的是，NDIR 传感器预热后首次读数才准确——MH-Z19B 需预热约 3 分钟，SCD30 约 2 分钟，S8 约 30 秒。间歇模式下每次唤醒后需等待预热完成再读取，否则读数偏低。这也是低功耗设计中需要权衡的关键参数：测量间隔越短，预热功耗占比越高。

## 9. 总结与展望

### 9.1 核心要点回顾

1. **NDIR 原理**：CO2 在 4.26 μm 有强吸收，Beer-Lambert 定律定量；"非色散"指用滤光片选波长
2. **光路设计**：双通道用参考通道消除光源漂移和气室污染，是消费级标配；多次反射折叠光路平衡小型化与灵敏度
3. **器件选型**：白炽灯 + 热电堆是消费级主流；双元热电堆配合窄带滤光片实现双通道
4. **环境补偿**：温度和气压影响显著（海拔 2000 m 不补偿低估 20%），必须硬件或软件补偿
5. **标定方法**：ABC 适用于定期通风的室内场景；温室等必须手动标定；高精度需两点标定

### 9.2 发展趋势

- **光声传感（PAS）**：CO2 吸收红外光后产生声波，无需长光路，体积更小（如 SCD4x），是 NDIR 有力竞争者
- **MEMS 红外光源**：替代传统白炽灯，功耗更低、调制频率更高、寿命更长
- **片上 NDIR**：光源、气室、探测器集成在单一 MEMS 芯片上，体积降至 mm 级，成本有望 $2 以下
- **AI 补偿算法**：机器学习替代线性补偿，同时处理温度、气压、湿度、老化等多维非线性影响，精度可提升 30–50%
- **多气体 NDIR**：同一光路布置多个不同波长通道，同时检测 CO2、CH4、CO 等

NDIR CO2 传感器已从工业仪器走向消费级 IoT，价格从几百美元降到几美元，精度从 ±100 ppm 提升到 ±30 ppm。随着双碳目标和室内健康意识的推动，NDIR 在智慧建筑、精准农业、环境监测领域的部署量将持续增长。
