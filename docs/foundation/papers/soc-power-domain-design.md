# SoC多电源域设计与电压调节策略
> **难度**：🔴 高级 | **领域**：SoC电源架构 | **阅读时间**：约 22 分钟

## 引言

想象一栋大楼：会议室灯火通明、空调全开；走廊只留应急灯；地下车库干脆断电。不同区域按需供电，整栋楼的总电费就能大幅降低。SoC 的多电源域设计也是同样的思路——CPU 全速跑时需要 1.1V 高压；睡眠时降到 0.7V 就够；不用外设直接断电。本文从"为什么需要多个电源域"开始，逐步讲清电压调节、电源门控、电平转换、常开域等核心概念，最后用 nRF52840 和 ESP32 的实例说明这些设计如何让 IoT 设备在纽扣电池上运行数年。

## 1. 为什么需要多个电源域

### 1.1 不同模块需要不同电压

SoC 内部模块的工作电压差异很大：

| 模块 | 典型电压 | 原因 |
|------|----------|------|
| CPU 核心 | 0.9-1.1V | 先进工艺，低电压低功耗 |
| SRAM | 1.0-1.2V | 保持数据需要最低电压 |
| I/O Pad | 1.8-3.3V | 匹配外部器件电平 |
| 模拟(ADC/DAC) | 2.5-3.3V | 保证精度和动态范围 |
| Flash | 2.5-3.3V | 编程需要高电压 |
| RTC | 1.0-1.2V | 时钟保持，极低功耗 |

如果所有模块共用一个电压，就必须取最高值（如 3.3V），CPU 核心也承受 3.3V，功耗会暴增。

### 1.2 独立开关控制与功耗关系

- 不用的模块可以断电，节省漏电流
- 睡眠模式下只保留 RTC 和唤醒逻辑

动态功耗公式：

```
P_dynamic = alpha * C * V^2 * f
alpha: 活动因子, C: 电容, V: 电压, f: 频率
```

电压降 30%，动态功耗降约 50%；频率降 50%，动态功耗降 50%。

## 2. 电源域概念

### 2.1 定义

电源域(Power Domain)是一组共享同一电源轨的逻辑单元集合，可以作为一个整体被开启或关闭。

```
SoC 电源域划分示例：

+-------------------+
|   VDD_CORE 域     |  CPU, Cache, Bus
|   (1.0V)          |
+-------------------+

+-------------------+
|   VDD_IO 域       |  GPIO, UART, SPI
|   (3.3V)          |
+-------------------+

+-------------------+
|   VDD_ANA 域      |  ADC, DAC, PLL
|   (2.5V)          |
+-------------------+

+-------------------+
|   VDD_RTC 域      |  RTC, Wakeup Ctrl
|   (1.0V, 常开)    |
+-------------------+
```

### 2.2 电源域层次

电源域可以嵌套：

```
顶层域: VDD_SYS
  +-- 子域: VDD_CORE (可独立关闭)
  |     +-- 子子域: VDD_CPU0 (可独立关闭)
  |     +-- 子子域: VDD_CPU1
  +-- 子域: VDD_PERIPH
        +-- 子子域: VDD_UART
        +-- 子子域: VDD_SPI
```

父域关闭时所有子域也被关闭，子域可独立关闭不影响其他子域。

## 3. 动态电压频率调节(DVFS)

### 3.1 原理

DVFS 根据工作负载动态调整电压和频率：

- 高负载：高电压 + 高频率 -> 高性能
- 低负载：低电压 + 低频率 -> 低功耗
- 空闲：最低电压保持运行或进入睡眠

### 3.2 操作点(OPP)

```
OPP 表示例 (Cortex-M4 @ 28nm):

| OPP   | 电压  | 频率    | 功耗    |
|-------|-------|---------|---------|
| OPP0  | 1.1V  | 80MHz   | 12mW    |
| OPP1  | 0.9V  | 40MHz   | 4mW     |
| OPP2  | 0.7V  | 20MHz   | 1.5mW   |
| OPP3  | 0.6V  | 10MHz   | 0.5mW   |
```

降压 30%，频率也需降低约 30%，功耗从 12mW 降到 4mW，降幅 67%。

### 3.3 DVFS 控制环路

```
        +----------+
        | 性能监控 | <-- 任务队列深度 / CPU利用率
        +----+-----+
             |
             v
        +----------+
        | DVFS策略  | <-- 选择目标OPP
        +----+-----+
             |
             v
        +----------+         +----------+
        | 电压调节 | ----->  | 频率调节 |
        | (LDO/DCDC)|        | (PLL/MUX)|
        +----------+         +----------+
```

先降频率再降电压，升压时先升电压再升频率。

IoT 设备负载变化剧烈（BLE 发包需 80MHz，空闲可降至 10MHz），DVFS 收益取决于工作占空比。

## 4. 电源门控(Power Gating)

### 4.1 原理

完全切断不使用模块的电源，消除漏电流：

```
VDD ----[ 睡眠晶体管 ]---- VDD_DOMAIN ---- [逻辑电路]

睡眠信号 = 1: 晶体管关断，VDD_DOMAIN = 0
睡眠信号 = 0: 晶体管导通，VDD_DOMAIN = VDD
```

睡眠晶体管尺寸需仔细设计：过小则 IR Drop 大影响时序，过大则面积开销大。

### 4.2 状态保持

电源门控后逻辑状态丢失，需要保持策略：

**保留单元(Retention Cell)：**

```
+---------------------------+
| 主锁存器 (可断电)          | <-- 正常工作使用
+---------------------------+
| 影子锁存器 (常开，备用电源) | <-- 断电前保存状态
+---------------------------+
```

断电前主锁存器数据复制到影子锁存器，上电后恢复。代价：面积增加约 50%。

**软件策略：** 断电前将关键寄存器值保存到常开 RAM，上电后恢复。不需要特殊单元，但恢复时间更长。

### 4.3 隔离单元(Isolation Cell)

电源域边界信号需要隔离：

```
域A (可能断电)  ---[隔离单元]---> 域B (始终开启)

断电时: 隔离单元输出固定值(0或1)，防止X值传播
上电后: 隔离单元透明，信号正常传递
```

域A 断电前必须先使能隔离。

### 4.4 上电冲击电流(Rush Current)

电源域上电时大量电容同时充电产生大电流：

```
I_rush = C_total * dV / dt

dt = 1us, C = 10nF: I_rush = 11A (不可接受)
dt = 100us, C = 10nF: I_rush = 110mA (可接受)
```

PMU 控制上电斜率(Ramp Rate)来限制冲击电流。

## 5. 电平转换器(Level Shifter)

### 5.1 为什么需要

信号跨越不同电压域时需要电平转换：

```
域A (1.0V)  ---[电平转换器]---> 域B (3.3V)
不转换: 1.0V 信号在 3.3V 域被误判为低电平
转换后: 1.0V -> 3.3V，电平正确
```

### 5.2 方向与实现

| 方向 | 典型场景 |
|------|----------|
| 低到高 | CPU(1.0V) -> GPIO(3.3V) |
| 高到低 | GPIO(3.3V) -> CPU(1.0V) |
| 双向 | I2C 总线 |

低到高转换器使用交叉耦合 PMOS 结构，不需要额外控制信号，延迟约 1-2 个门延迟。

## 6. 常开域(Always-On Domain)

### 6.1 包含哪些逻辑

常开域在 SoC 最深睡眠状态下仍然供电：

| 组件 | 功能 |
|------|------|
| RTC | 保持时间，定时唤醒 |
| 唤醒控制器 | 监测唤醒源 |
| PMU | 控制其他域上电/断电 |
| 备用寄存器 | 保存关键状态 |
| 低速振荡器 | 为常开域提供时钟 |

### 6.2 设计原则与功耗

- 面积最小化、漏电流最小化（使用高阈值 HVT 晶体管）
- 电压最低化（如 0.7V）、独立电源轨

常开域功耗决定 IoT 设备睡眠电流：

```
nRF52840 System OFF: 0.3uA (仅 RTC + GPIO 唤醒)
STM32L4 Standby:    0.04uA (仅 RTC + 唤醒)
ESP32 Deep Sleep:   10uA (RTC + ULP 协处理器)
```

## 7. 电源管理单元(PMU)

### 7.1 功能

PMU 是 SoC 电源域管理的总控：控制各域上电/断电时序、管理电压调节器、监测唤醒源、管理上电斜率、与软件协同。

### 7.2 状态机与上电时序

```
              +----------+
    RESET --->|  OFF     |
              +----+-----+
                   | POR
                   v
              +----------+
              |  ACTIVE  | <--- 唤醒事件
              +----+-----+
                   | 睡眠请求
                   v
              +----------+
              |  SLEEP   | ---> 唤醒 ---> ACTIVE
              +----+-----+
                   | 深度睡眠
                   v
              +----------+
              | DEEP SLEEP| ---> 唤醒 ---> 恢复时序 ---> ACTIVE
              +----------+
```

上电时序（顺序不能随意更改）：

```
0us    [1] VDD_RTC 上电（常开域）
10us   [2] VDD_ANA 上电（模拟需要稳定时间）
50us   [3] VDD_CORE 上电（CPU 和 SRAM）
100us  [4] 释放复位，CPU 开始执行
200us  [5] VDD_IO 上电（外设就绪）
500us  [6] PLL 锁定，切换到高速时钟
```

## 8. 实例分析

### 8.1 nRF52840 电源域

| 电源状态 | 活跃域 | 电流 |
|----------|--------|------|
| System ON (Active) | 全部 | 约 5-10mA |
| System ON (Idle) | CPU 休眠，外设可选 | 约 1-2mA |
| System ON (RTC only) | RTC + RAM 保持 | 约 1.2uA |
| System OFF | RTC + GPIO 唤醒 | 约 0.3uA |

```
[VDD] (1.7-3.6V)
  +-- [DCDC] --> 1.3V --> CPU, RAM, Flash, 外设
  +-- [LDO] --> RTC 域 (常开)
  +-- [GPIO] --> 可配置唤醒引脚
```

RAM 保持选项：保持 1 块额外约 0.5uA，全部保持约 2uA，不保持则唤醒后需重新初始化。

### 8.2 ESP32 电源域

```
ESP32 电源域：
[电源域]     [内容]              [状态]
VDD_RTC      RTC, ULP协处理器     始终开启
VDD_DIG      CPU, WiFi/BT, 外设  可门控
VDD_ANA      ADC, WiFi射频       可门控
VDD_SDIO     Flash               可门控
```

ULP 协处理器位于 RTC 域，主 CPU 睡眠时仍可运行，可采样 ADC/读 I2C/监测 GPIO，功耗约 100-300uA。

ESP32 深度睡眠电流分解：总约 10uA（RTC 振荡器 1uA + RTC 存储 3uA + ULP 5uA + 漏电 1uA）。

## 9. 设计挑战

### 9.1 隔离单元插入与状态保持

- 需要自动识别跨域信号并插入隔离单元（工具：UPF/CPF）
- 手动遗漏会导致仿真无法发现的功能错误

状态保持策略选择：

| 策略 | 恢复速度 | 面积开销 | 适用场景 |
|------|----------|----------|----------|
| 保留单元 | 快(us级) | 大(50%) | 频繁开关的域 |
| 软件保存到RAM | 慢(ms级) | 小 | 偶尔开关的域 |
| 不保持 | 中等(重初始化) | 无 | 可重初始化的域 |

### 9.2 UPF 电源意图描述

```tcl
# UPF 示例 (简化)
create_power_domain PD_CORE
create_power_domain PD_RTC

create_supply_net VDD_CORE -domain PD_CORE
create_supply_net VDD_RTC  -domain PD_RTC

# 电源门控
create_power_switch PS_CORE \
  -domain PD_CORE \
  -input_supply_port {in VDD} \
  -output_supply_port {out VDD_CORE} \
  -control_port {sleep sleep_signal}

# 隔离
set_isolation ISO_CORE_OUT \
  -domain PD_CORE \
  -applies_to outputs \
  -clamp_value 0
```

## 10. 对IoT电池寿命的影响

### 10.1 电流分解

```
平均电流 = I_active * duty_active + I_sleep * duty_sleep

假设:
  I_active = 10mA,  duty = 0.1%
  I_sleep  = 2uA,   duty = 99.9%

平均电流 = 10mA * 0.001 + 2uA * 0.999 = 12uA
CR2032 (225mAh) 寿命 = 225mAh / 12uA ~ 780天
```

### 10.2 睡眠电流的关键性

睡眠电流从 2uA 恶化到 50uA：电池寿命从 780 天降到 156 天。睡眠电流每增加 1uA，电池寿命减少约 5%。

### 10.3 优化策略

1. 精确划分电源域：只有必须常开的逻辑放入常开域
2. 使用保留单元替代整域保持：减少漏电
3. DCDC 替代 LDO：效率从约 60% 提升到 85%+
4. 合理设置唤醒间隔：降低活跃占空比
5. 使用 ULP 协处理器：避免主 CPU 频繁唤醒

## 总结

SoC 多电源域设计的核心动机是不同模块需要不同电压和独立的开关控制。DVFS 在活跃时降压降频省功耗；电源门控在空闲时完全断电消除漏电；隔离单元和电平转换器保证域间信号正确传递；常开域只保留最小逻辑维持唤醒能力。nRF52840 通过精细的电源域划分实现 0.3uA 的深度睡眠电流，ESP32 的 ULP 协处理器则在 RTC 域提供低功耗感知能力。对 IoT 设备来说，电源域设计直接决定了电池寿命的天花板。
