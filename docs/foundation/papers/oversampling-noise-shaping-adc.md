# 过采样与噪声整形提高ADC有效分辨率

> **难度**：🔴 高级 | **领域**：高精度数据采集 | **阅读时间**：约 22 分钟

## 引言

想象你在嘈杂的菜市场用手机录音。如果只录一小段,噪音和说话声混在一起,很难听清。但如果你连续录很长一段,然后把多次重复的同一句话叠加平均,背景噪声会相互抵消,说话声却越来越清晰--这就是过采样提升分辨率的直觉。

ADC(模数转换器)把连续电压变成离散数字,不可避免地引入量化噪声。过采样以高于奈奎斯特速率的频率采样,让量化噪声能量分散到更宽的频带;再用数字滤波器砍掉带外噪声,等效信噪比提升,有效位数增加。Sigma-Delta架构更进一步,用反馈回路把噪声"推"到高频,带内噪声大幅降低--这就是噪声整形。

本文从量化噪声的白噪声模型出发,推导过采样SNR增益,再深入Sigma-Delta调制器的噪声整形原理,最后给出MCU上的工程实现。

## 1 基础概念回顾

### 1.1 奈奎斯特采样与量化噪声

对带宽为 f_b 的带限信号,采样率 f_s >= 2*f_b 时可无失真恢复。N位ADC的量化步长 LSB = V_ref / 2^N,量化误差 e(t) 落在 [-LSB/2, +LSB/2]。

假设量化误差在该区间均匀分布(白噪声模型),则:

- 方差: sigma_e^2 = LSB^2 / 12
- 功率谱密度: S_e(f) = LSB^2 / (12 * f_s)

白噪声假设在输入信号充分活跃(不断跨越多个量化台阶)时成立;对直流或极低频输入可能失效。

## 2 过采样原理

### 2.1 过采样率与噪声重分布

过采样率 OSR = f_s / (2 * f_b)。OSR = 1 即奈奎斯特采样;OSR = 4 表示采样率是奈奎斯特速率的4倍。

量化噪声总功率不变,但功率谱密度随采样率升高而降低:

```
S_e(f) = LSB^2 / (12 * f_s) = LSB^2 / (12 * 2 * f_b * OSR)
```

过采样时带内噪声功率仅为:

```
P_noise,in-band = S_e(f) * f_b = LSB^2 / (12 * 2 * OSR)
```

带内噪声降低了OSR倍。

### 2.2 SNR增益与有效位数提升

信号功率不变,带内噪声降低OSR倍,SNR改善:

```
SNR_gain = 10 * log10(OSR)  [dB]
```

每增加6 dB(即OSR翻4倍)等效增加1位:

```
额外位数 = (1/2) * log2(OSR)
```

| OSR | SNR增益(dB) | 额外位数 |
|-----|-------------|----------|
| 4   | 6.0         | 1        |
| 16  | 12.0        | 2        |
| 64  | 18.1        | 3        |
| 256 | 24.1        | 4        |

12位ADC通过OSR=256过采样可等效获得16位分辨率,但需256次采样,时间开销显著。

### 2.3 抽取滤波

过采样产生大量高速数据,需降回奈奎斯特速率:

1. **数字低通滤波**: 截止频率 f_b,滤除带外量化噪声
2. **抽取(Decimation)**: 每OSR个样本保留1个

最简单的滤波器是CIC(移动平均),对OSR个样本求和后右移:

```
y[n] = (1/OSR) * sum(x[n-k], k=0..OSR-1)
```

CIC频率响应呈sinc形状,旁瓣衰减约-13 dB。高精度场景需级联多阶CIC或后接FIR补偿滤波器。

## 3 Sigma-Delta调制与噪声整形

### 3.1 从过采样到噪声整形

单纯过采样每倍频程(OSR翻倍)只增加0.5位,收益递减严重。Sigma-Delta架构加入噪声整形,把量化噪声从低频推向高频,带内噪声进一步大幅降低。

类比: 过采样像把灰尘均匀扫到更大面积上;噪声整形像加了一台吹风机,主动把灰尘往角落(高频)吹,中间(低频信号带)更干净。

### 3.2 一阶Sigma-Delta调制器

```
         +---+     +---+     +---+
x[n] -->| + |-->| z^-1 |-->| + |--> y[n]  (1位量化输出)
         +---+     +---+     +---+
           ^                   |
           |      +---+        |
           +------| - |<-------+
                  +---+
       (积分器)     (1位DAC反馈)
```

线性化分析中,量化器等效为加性噪声源 e[n],z变换推导得:

```
Y(z) = X(z) * z^-1 + E(z) * (1 - z^-1)
```

- 信号传输函数(STF): z^-1 (纯延迟,增益=1)
- 噪声传输函数(NTF): 1 - z^-1 (高通)

NTF在低频增益趋零,高频增益趋2,量化噪声被整形到高频。

### 3.3 一阶噪声整形的频域分析

```
|NTF(f)|^2 = 4 * sin^2(pi*f/f_s)
```

带内噪声(低频 sin(x) ≈ x):

```
P_noise,in-band = (pi^2 / 3) * (1 / OSR^3) * sigma_e^2
```

对比纯过采样的 sigma_e^2 / OSR,一阶整形按 1/OSR^3 衰减,快得多:

```
SNR_1st = 6.02*N + 1.76 - 5.17 + 30*log10(OSR)  [dB]
```

每倍频程SNR改善9 dB,等效增加1.5位。

| OSR | 一阶整形额外位数 | 纯过采样额外位数 |
|-----|------------------|------------------|
| 64  | 4.5              | 3                |
| 256 | 6                | 4                |
| 1024| 7.5              | 5                |

### 3.4 二阶Sigma-Delta调制器

级联两个积分器,NTF = (1 - z^-1)^2:

```
P_noise,in-band = (pi^4 / 5) * (1 / OSR^5) * sigma_e^2
SNR_2nd = 6.02*N + 1.76 - 12.9 + 50*log10(OSR)  [dB]
```

每倍频程改善15 dB,等效增加2.5位。OSR=64时可达约12位(1位量化器)。

### 3.5 高阶调制器与稳定性

三阶NTF = (1 - z^-1)^3,每倍频程改善21 dB,等效3.5位。但高阶单环调制器面临严重稳定性问题:

- **根因**: 高阶NTF高频增益随阶数急剧增大(二阶4,三阶8,四阶16...)
- **后果**: 量化器输入发散,积分器饱和,输出陷入极限环振荡
- **Lee准则**: NTF极点应全部在单位圆内;高阶难以同时满足整形和稳定性

工程应对:
1. **NTF零点优化**: 将零点从z=1分散到信号带内
2. **极点配置**: 加入极点(反馈系数),牺牲少量整形换取稳定
3. **限幅器**: 积分器输出超限时截断
4. **输入幅度限制**: 限制输入不超过满幅的-6 dB至-3 dB

经验: 单环不超过2阶;3阶需仔细设计;4阶及以上几乎不可用。

### 3.6 MASH架构

MASH(Multi-stAge noise SHaping)用多个低阶稳定调制器级联实现高阶整形,绕过单环稳定性问题。

**2-1 MASH(三阶)**:

```
第一级: 二阶调制器 -> Y1, 量化噪声E1
第二级: 一阶调制器, 输入=E1 -> Y2, 量化噪声E2
数字组合: Y = Y1*H1(z) - Y2*H2(z) = X(z)*z^-3 + E2(z)*(1-z^-1)^3
```

最终只剩E2被三阶整形,E1被完全消除。优点是每级都稳定;代价是对模拟失配敏感(E1泄漏),且电路复杂度增加。

## 4 数字抽取滤波器

### 4.1 Sinc滤波器(CIC)

Sigma-Delta输出是高速1位数据流,需滤除带外整形噪声、降速、转多位。最常用的是CIC(Cascaded Integrator-Comb)滤波器。

k阶sinc传递函数与频率响应:

```
H(z) = [(1 - z^(-R*M)) / (1 - z^-1)]^k
|H(f)| = |sin(pi*R*M*f/f_s) / sin(pi*f/f_s)|^k
```

R为抽取比,M为微分延迟(通常=1),k为阶数。

**阶数选择**: sinc阶数 = 调制器阶数 + 1(一阶调制器用sinc2,二阶用sinc3)。

### 4.2 CIC的硬件高效实现

CIC可用纯整数运算(无乘法器)实现:

```c
// CIC积分级(高速f_s)
for (n = 0; n < N_total; n++) {
    int0 += input[n];
    int1 += int0;      // k=2示例
}

// CIC梳状级(抽取后速率f_s/R)
for (m = 0; m < N_total/R; m++) {
    out = int_prev1[m] - int_prev1[m - M];
}
```

通过Noble恒等式可交换抽取和梳状级顺序,所有运算在低速率执行。sinc通带下降可后接5~15阶FIR补偿滤波器修正。

## 5 MCU上的工程实现

### 5.1 软件过采样: 12位ADC到16位

```c
#include <stdint.h>

#define OSR_4X4     256   // 4^4 = 256, 增加4位
#define RIGHT_SHIFT 4     // log4(256) = 4

uint16_t oversample_adc12_to_16(void)
{
    uint32_t accumulator = 0;

    for (uint16_t i = 0; i < OSR_4X4; i++) {
        ADC1->CR1 |= ADC_CR1_ADON;
        while (!(ADC1->SR & ADC_SR_EOC));
        accumulator += ADC1->DR;
    }

    // 右移4位(非8位): 累加256个12位值,有效位数 = 12+4 = 16
    // 数值范围 = 4095*256/16 = 65520 ≈ 2^16
    return (uint16_t)(accumulator >> RIGHT_SHIFT);
}
```

注意: 256次采样总耗时约256us(1us/转换);输入须稳定(变化<1 LSB);累加器用uint32_t(256*4095 < 2^20,安全)。

### 5.2 STM32硬件过采样

STM32G4/L4/H7内置硬件过采样控制器:

```c
void adc_hardware_oversample_init(void)
{
    ADC1->CFGR2 &= ~ADC_CFGR2_OVSR_Msk;
    // 过采样率16x: OVSR=3, 对应2^(3+1)=16
    ADC1->CFGR2 |= (3U << ADC_CFGR2_OVSR_Pos);
    // 右移2位
    ADC1->CFGR2 &= ~ADC_CFGR2_OVSS_Msk;
    ADC1->CFGR2 |= (2U << ADC_CFGR2_OVSS_Pos);
    // 使能过采样
    ADC1->CFGR2 |= ADC_CFGR2_OVSE;
}
```

优势: 零CPU开销,硬件自动累加移位,可配合DMA。局限: OSR最大256x(额外4位),无法实现噪声整形。

### 5.3 Sigma-Delta软件抽取

通用MCU没有模拟Sigma-Delta调制器,但可实现数字抽取(接收外部调制器输出):

```c
// 外部1位Sigma-Delta调制器(如AD7401) -> MCU软件sinc3抽取
#define DECIMATION_RATIO 256

int32_t sinc3_integrator[3] = {0};
int32_t sinc3_comb[3] = {0};

int32_t sinc3_filter(int32_t one_bit_input)
{
    // 积分级(每采样调用一次)
    sinc3_integrator[0] += one_bit_input;
    sinc3_integrator[1] += sinc3_integrator[0];
    sinc3_integrator[2] += sinc3_integrator[1];

    static uint16_t count = 0;
    if (++count >= DECIMATION_RATIO) {
        count = 0;
        // 梳状级(差分)
        int32_t out = sinc3_integrator[2]
                    - 3 * sinc3_comb[0]
                    + 3 * sinc3_comb[1]
                    - sinc3_comb[2];
        sinc3_comb[2] = sinc3_comb[1];
        sinc3_comb[1] = sinc3_comb[0];
        sinc3_comb[0] = sinc3_integrator[2];
        return out;
    }
    return 0;
}
```

更高效实现用SPI+DMA接收1位数据流,定时器中断中执行CIC抽取。

## 6 局限性与收益递减

### 6.1 过采样的代价

| 维度 | 代价 |
|------|------|
| 时间 | OSR=256需256次转换,带宽降低256倍 |
| 功耗 | 更高采样率 = 更高模拟功耗 |
| 带宽 | 有效带宽 = f_s/(2*OSR),严重收窄 |
| 噪声模型 | 白噪声假设对直流/低频输入失效 |

12位ADC过采样到16位,带宽仅剩 f_s/512。若原始1 MSPS,等效16位带宽仅约2 kHz--只适合低速信号。

### 6.2 噪声整形的固有局限

- **总噪声功率不变**: 带内降低 = 带外增加,能量守恒
- **带外噪声很大**: 需高阶数字滤波器充分抑制,否则泄漏回带内
- **调制器自身噪声**: 积分器热噪声、时钟抖动、参考噪声降低实际性能
- **非线性**: 1位量化器本质非线性,大信号时谐波失真增加

### 6.3 过采样vs硬件高分辨率ADC

| 特性 | 12位ADC过采样到16位 | 16/24位硬件ADC |
|------|----------------------|----------------|
| 成本 | 几乎为零(用现有MCU) | 增加BOM成本 |
| 带宽 | 极低(Hz~kHz级) | 可达数十kHz~MHz |
| 精度 | 受白噪声模型限制 | INL/DNL由设计保证 |
| 功耗 | 较高(持续高速采样) | 低速高精度ADC功耗低 |
| 校准 | 需软件偏移/增益校准 | 片上自动校准 |

**选择建议**: 需1~2位额外精度、低带宽(温度/压力) -> 过采样;16位以上、带宽>10 kHz -> 专用Sigma-Delta ADC;24位精密测量 -> 专用芯片(如HX711、ADS124S08)。

## 7 应用案例

**温度传感器**: NTC+12位ADC,0.01度分辨率需额外约2位,OSR=16,采样率1 kHz有效带宽31 Hz,远高于温度变化速率。

**称重传感器**: 需24位精度,12位ADC过采样需OSR=2^24(不可行),必须用专用24位Sigma-Delta ADC(如HX711)。

**电池电压**: 12位ADC监测0~5V,5V/4096=1.22 mV刚好不够,OSR=4得13位(0.61 mV),代价仅4次采样。

## 总结

过采样和噪声整形是提升ADC有效分辨率的两大互补技术:

1. **过采样**: 利用白色量化噪声频谱,提高采样率分散噪声,每4倍OSR增1位,通用MCU即可实现。

2. **噪声整形(Sigma-Delta)**: 反馈回路将量化噪声推向高频,一阶每倍频程1.5位,二阶2.5位,远优于纯过采样0.5位。代价是模拟复杂度和稳定性约束。

3. **工程边界**: 过采样适合1~4位额外精度、kHz以下带宽;更高需求用专用Sigma-Delta ADC。MASH架构绕过单环稳定性问题,但对失配敏感。

4. **实现路径**: MCU硬件过采样最便捷;外部调制器+软件CIC抽取适合隔离采集;专用集成ADC仍是高精度场景最优解。

核心权衡始终是精度、带宽、成本三角--没有免费午餐,只有合适的选择。
