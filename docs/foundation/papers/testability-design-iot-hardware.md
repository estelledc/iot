# IoT硬件可测试性设计DFT与测试点布局
> **难度**：🟡 中级 | **领域**：测试工程 | **阅读时间**：约 20 分钟

## 引言

想象你买了一辆新车，仪表盘上什么指示灯都没有——没有油量、没有水温、没有故障灯。车能开，但你完全不知道内部状态是否正常，直到某天突然抛锚。硬件可测试性设计(Design For Test, DFT)做的事就是把"仪表盘"加到PCB上——让生产线上和现场维修时，都能快速定位问题。

DFT不是可选项，而是量产产品的必备属性。没有测试点的板子，就像没有仪表盘的车：问题一定会发生，但你无从得知。

## 1. DFT的重要性

### 1.1 为什么需要DFT

制造缺陷是不可避免的：

- 贴片偏移导致虚焊或短路
- 元器件来料不良
- 回流焊温度异常导致冷焊
- PCB制造缺陷(断线、短路)

没有DFT，这些问题只能在最终功能测试时发现——定位困难，返工成本高。

### 1.2 DFT的投资回报

| 项目 | 无DFT | 有DFT | 说明 |
|------|-------|-------|------|
| 故障定位时间 | 30-120分钟 | 5-15分钟 | 直接测量关键信号 |
| 返工成功率 | 60-70% | 90-95% | 精确定位故障器件 |
| 场地故障率 | 高 | 低 | 出厂前筛除隐患 |
| 测试开发成本 | 低 | 中(需设计测试点) | 前期投入换来后期节省 |

## 2. 测试接入点

### 2.1 电源轨测试点

每路电源必须有测试点，用于验证：

- 电压是否在规格范围内
- 纹波是否超标
- 上电时序是否正确

```
VCC_3V3 ----[LDO]---- VCC_1V8 ----[MCU]
   |                    |
  TP1                  TP2        (测试点)

测量内容: TP1 = 3.3V +/-5%, TP2 = 1.8V +/-3%
```

### 2.2 通信总线测试点

I2C/SPI/UART等总线必须引出测试点：

| 总线 | 必测信号 | 测试内容 |
|------|----------|----------|
| I2C | SDA, SCL | 信号完整性、上拉电压、通信内容 |
| SPI | MOSI, MISO, SCK, CS | 时钟频率、数据正确性 |
| UART | TX, RX | 波特率、数据帧 |

### 2.3 模拟信号测试点

传感器输出、ADC输入等模拟信号：

- 信号范围是否正确
- 噪声水平是否可接受
- 偏置是否正确

### 2.4 数字信号测试点

关键GPIO、中断信号、状态指示：

- 电平是否正确
- 时序是否满足
- 中断响应是否正常

## 3. 测试点设计规范

### 3.1 焊盘尺寸

测试探针需要足够的接触面积：

| 参数 | 最小值 | 推荐值 | 说明 |
|------|--------|--------|------|
| 焊盘直径 | 0.8mm | 1.0mm | 探针最小接触面 |
| 焊盘间距 | 0.8mm | 1.27mm | 匹配标准探针间距 |
| 焊盘至器件距离 | 0.5mm | 1.0mm | 避免探针碰到器件 |

### 3.2 可达性

- 测试点必须在一侧可达(通常是PCB背面)
- 不能被器件遮挡
- 不能被外壳遮挡
- 同一测试面上高度一致

### 3.3 丝印标注

每个测试点必须有清晰的丝印标注：

```
TP1  -> VCC_3V3
TP2  -> VCC_1V8
TP3  -> I2C_SDA
TP4  -> I2C_SCL
TP5  -> GND
```

没有标注的测试点等于不存在——产线操作员不知道测什么。

### 3.4 盘内过孔(Via-in-Pad)

空间紧张时可用盘内过孔节省面积：

- 过孔在测试焊盘中心
- 过孔必须电镀填平，否则探针可能卡住
- 适合0.5mm以下间距BGA的扇出测试点

## 4. 边界扫描JTAG

### 4.1 IEEE 1149.1标准

边界扫描通过在IC周边插入边界扫描单元(BSC)：

```
     [BSC]--[BSC]--[BSC]
       |      |      |
    Pin1    Pin2    Pin3
       |      |      |
     [BSC]--[BSC]--[BSC]
          TDI -> -> TDO
```

每个BSC可以：
- 捕获引脚状态(Sample)
- 驱动引脚输出(Extest)
- 测试IC之间的互连

### 4.2 JTAG在IoT中的用途

| 用途 | 说明 |
|------|------|
| 互连测试 | 检测BGA焊点开路/短路 |
| Flash编程 | 通过JTAG烧录固件 |
| 在线调试 | 断点、单步、查看变量 |

### 4.3 JTAG测试点

最低需要4个信号：TCK, TMS, TDI, TDO，加上VCC和GND，共6个测试点。可以用标准ARM 10-pin或20-pin连接器。

## 5. 在线测试ICT

### 5.1 针床夹具

ICT使用定制针床(Bed-of-Nails)夹具：

- 每个测试点对应一根探针
- 探针网格通常为100mil(2.54mm)间距
- 夹具定位孔至少2个(对角放置)

### 5.2 ICT能测什么

| 测试类型 | 原理 | 检测缺陷 |
|----------|------|----------|
| 开路测试 | 向网络施加电流，检测电压降 | 焊点开路 |
| 短路测试 | 检测网络间绝缘电阻 | 网络短路 |
| 器件值测试 | 向器件施加激励测响应 | 器件错料/不良 |
| 二极管测试 | 正向导通压降 | 极性反/开路 |

### 5.3 ICT对PCB的要求

- 测试点在标准网格上
- 测试点在PCB同一面
- 每个网络至少有一个测试点
- 测试点不能在器件下方

## 6. 功能测试

### 6.1 功能测试流程

```
上电 -> 固件加载 -> 外设测试 -> 通信测试 -> 校准 -> 通过/失败
```

### 6.2 固件加载

量产时固件加载方式：

| 方式 | 速度 | 适用场景 |
|------|------|----------|
| JTAG/SWD | 快(30秒) | 有调试接口的板子 |
| UART Bootloader | 中(1-2分钟) | 串口烧录 |
| SWD单线 | 中 | 引脚少的产品 |
| 预烧录 | 最快 | 从原厂买预烧录MCU |

### 6.3 通过/失败判据

必须定义明确的量化判据：

- 电源电压：3.3V +/- 5%，即3.135V - 3.465V
- BLE RSSI：-30dBm至-10dBm(1米距离)
- 传感器读数：25 +/- 2 (在标准条件下)

## 7. IoT量产测试策略

### 7.1 分层测试

| 层级 | 测试内容 | 设备 | 节拍 |
|------|----------|------|------|
| 裸板ICT | 开短路、器件值 | ICT针床 | 10-20s |
| 固件烧录 | 烧录+校验 | 烧录器 | 30-60s |
| 功能测试 | 上电自检、外设、通信 | 定制夹具 | 30-60s |
| 射频测试 | 发射功率、灵敏度 | 综测仪 | 15-30s |
| 校准 | 传感器校准、频率校准 | 定制夹具 | 10-20s |
| 贴标 | 打印SN标签 | 贴标机 | 5s |

### 7.2 射频测试

无线IoT设备必须进行射频测试：

- 发射功率：是否在法规范围内
- 频率误差：晶振偏差是否可接受
- 调制质量：EVM(Error Vector Magnitude)是否合格
- 接收灵敏度：能否在指定距离可靠通信

## 8. 测试夹具设计

### 8.1 顶针(Pogo Pin)

```
    [弹簧触头]
     ||||||||  <- 针头(尖/平/冠)
     |      |
     | 弹簧 |  <- 内部弹簧
     |      |
     ||||||||  <- 针管(焊在夹具PCB上)
```

选型要点：

- 行程：3-5mm为佳，太小接触不可靠
- 针头形状：尖头刺破氧化层，平头磨损小
- 电流容量：电源测试点需要大电流探针

### 8.2 定位特征

- PCB上至少2个定位孔(对角)
- 定位孔直径3mm或4mm
- 夹具上使用定位销对应
- 防呆设计：不对称的定位孔防止反插

### 8.3 自动化 vs 手动

| 项目 | 手动夹具 | 自动化夹具 |
|------|----------|------------|
| 投资 | 低($1K-5K) | 高($10K-50K) |
| 节拍 | 慢(2-5min/板) | 快(30s/板) |
| 一致性 | 依赖操作员 | 高 |
| 适用 | 小批量 | 大批量 |

## 9. 固件自测试

### 9.1 上电自检(POST)

固件启动时自动检测关键外设：

```c
// POST: 上电自检示例
int post_check(void) {
    int errors = 0;

    // 检查RAM
    if (ram_test() != 0) errors |= ERR_RAM;

    // 检查Flash CRC
    if (flash_crc_verify() != 0) errors |= ERR_FLASH;

    // 检查传感器通信
    if (sensor_read_id() != EXPECTED_ID) errors |= ERR_SENSOR;

    // 检查无线模块
    if (radio_init() != 0) errors |= ERR_RADIO;

    return errors;
}
```

### 9.2 传感器合理性检查

```c
// 温湿度传感器合理性检查
bool sensor_sanity_check(void) {
    float temp = read_temperature();
    float humi = read_humidity();

    // 温度范围检查: -40 ~ 85
    if (temp < -40.0f || temp > 85.0f) return false;

    // 湿度范围检查: 0 ~ 100
    if (humi < 0.0f || humi > 100.0f) return false;

    // 露点检查: 温度不应低于露点
    // (此检查可捕获传感器互换问题)
    return true;
}
```

### 9.3 内存测试

简单但有效的RAM测试——行走1模式：

```c
// 行走1测试: 每位依次写入1再读回
int ram_walking_one_test(volatile uint32_t *base, uint32_t size) {
    uint32_t pattern = 1;
    for (uint32_t i = 0; i < 32; i++) {
        *base = pattern;
        if (*base != pattern) return -1;
        pattern <<= 1;
    }
    return 0;
}
```

## 10. 实战案例：IoT传感器节点测试方案

### 10.1 测试流程

```
Step 1: ICT(裸板)
  -> 检测开短路、关键器件值
  -> 通过: 进入Step 2; 失败: 报废或返修

Step 2: 烧录固件
  -> 通过SWD接口烧录
  -> 校验CRC通过: 进入Step 3

Step 3: 功能测试
  -> 上电自检POST
  -> I2C传感器通信
  -> BLE广播检测
  -> LED/按键功能
  -> 通过: 进入Step 4

Step 4: 射频校准
  -> 发射功率校准
  -> 频率偏差校准
  -> 校准值写入NVS
  -> 通过: 进入Step 5

Step 5: 贴标
  -> 打印SN + MAC + 校准日期
  -> 贴标完成: 入库
```

### 10.2 各步骤测试点需求

| 步骤 | 测试点/接口 | 数量 |
|------|-------------|------|
| ICT | 电源轨 + 关键信号 | 15-20个 |
| 烧录 | SWDIO, SWCLK, GND, VCC | 4个 |
| 功能 | UART TX/RX(日志), 按键, LED | 4个 |
| 射频 | 天线(辐射测试) | 1个 |
| 校准 | 同功能测试 | - |

## 11. 常见DFT错误

### 11.1 关键网络无测试点

错误：电源使能信号、复位信号、中断信号没有测试点。这些信号一旦出问题，整板不工作，但无法测量。

正确：每个电源使能、复位、中断、状态信号都留测试点。

### 11.2 测试焊盘太小

错误：测试点焊盘0.5mm，探针无法稳定接触，测试结果时好时坏。

正确：测试点焊盘至少1.0mm。

### 11.3 没有烧录接口

错误：量产时才发现没有烧录接口，只能先焊MCU、烧录、再焊其他器件——流程倒退。

正确：设计之初就确定烧录方式，预留接口。

### 11.4 埋藏连接不可测

错误：BGA扇出只在内层走线，没有引到测试点。BGA焊点问题无法通过ICT检测。

正确：每个BGA网络至少有一个测试点在外层。

## 总结

DFT是连接设计与制造的桥梁，核心原则是"可观测、可控制"：

1. 可观测：每个关键信号都有办法测量——测试点、LED、串口日志
2. 可控制：关键功能可以通过测试命令触发——进入测试模式、手动控制GPIO
3. 分层测试：ICT抓制造缺陷，功能测试抓设计问题，射频测试抓合规
4. 固件自检：POST在每次上电时验证核心功能，是最后一道防线

设计阶段每多花1小时规划测试点，量产阶段就能节省数十小时的故障定位时间。
