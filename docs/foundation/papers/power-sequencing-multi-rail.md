# 多路电源上电时序设计与保护
> **难度**：🟡 中级 | **领域**：电源系统设计 | **阅读时间**：约 18 分钟

## 引言

想象一个酒店, 客房部、餐饮部、安保部各有各的工作流程. 如果安保系统还没启动就先开了客房门禁, 就可能出现安全漏洞. 多路电源系统也一样: 如果I/O电压先于内核电压上电, 芯片可能进入闩锁状态, 甚至烧毁. 本文讲解多路电源系统的上电时序设计方法、保护电路, 以及在IoT网关中的实际设计案例.

## 1 为什么上电时序很重要

### 1.1 闩锁效应(Latch-up)

CMOS集成电路内部寄生有双极型晶体管结构(PNPN). 如果I/O引脚的电压先于内核电压上电, 寄生晶体管可能被触发导通, 形成低阻抗通路, 产生大电流:

```
V_DD_io ---+
           |  (寄生PNPN结构)
    I/O ---|--- 寄生PNP --- 寄生NPN --- GND
           |
           +--- 大电流通路 --> 芯片烧毁
```

闩锁一旦触发, 只能通过断电恢复. 这是不可逆损伤的典型原因.

### 1.2 FPGA和SoC的时序要求

FPGA和SoC通常有多路电源输入, 厂商规定了严格的上电顺序:

典型顺序(以Xilinx Zynq为例):
1. VCCINT (1.0V, 内核) -- 最先上电
2. VCCAUX (1.8V, 辅助) -- 第二
3. VCCBRAM (1.0V, Block RAM) -- 与内核同步
4. VCCO (3.3V/1.8V, I/O) -- 最后上电

违反顺序可能导致:
- 配置加载失败
- I/O引脚在内核就绪前输出不确定电平
- 内部状态机卡死

### 1.3 I/O电压先于内核电压的问题

当I/O电压先于内核电压建立时:

- 芯片内部的ESD保护二极管正向导通, 电流从I/O电源流入内核
- 内核电路在未就绪状态下被驱动, 逻辑状态不确定
- 外部器件可能收到错误的总线信号, 导致系统级故障
- 多次累积可能逐渐损伤芯片

## 2 时序控制方法

### 2.1 RC延迟法

最简单的时序控制: 利用RC网络的充电延迟来延后使能信号.

```
V_IN ---[R]---+--- EN引脚(下一级DC-DC)
               |
              [C]
               |
              GND

延迟时间 t = -R*C * ln(1 - V_EN_th / V_IN)
```

示例: V_IN=5V, EN阈值1.2V, R=100k欧姆, C=1uF:

```
t = -100e3 * 1e-6 * ln(1 - 1.2/5.0)
  = 0.1 * 0.274
  = 27.4ms
```

优点: 成本极低, 一个电阻一个电容
缺点:
- 延迟时间受元件容差影响(电解电容+/-20%)
- 温度漂移大
- 只能做粗略延时, 无法验证前级是否真正就绪
- 掉电时可能放电不充分, 重复上电时序错乱

### 2.2 使能引脚级联法

利用前级DC-DC的Power-Good信号驱动后级的使能引脚:

```
5V输入 --> [DC-DC 3.3V] --PG--> EN --> [DC-DC 1.8V] --PG--> EN --> [LDO 2.8V]
```

```c
// 使能级联的软件模拟(实际由硬件自动完成)
void power_sequencing_cascade(void) {
    // 第1级: 3.3V数字电源
    enable_3v3();
    wait_for_power_good(POWER_3V3);  // 等待3.3V稳定

    // 第2级: 1.8V内核电源(3.3V的PG信号使能)
    enable_1v8();
    wait_for_power_good(POWER_1V8);  // 等待1.8V稳定

    // 第3级: 2.8V模拟电源(1.8V的PG信号使能)
    enable_2v8();
    wait_for_power_good(POWER_2V8);  // 等待2.8V稳定
}
```

优点:
- 确保前级稳定后才启动后级(不是简单延时)
- 成本低(无需额外IC)
- 时序可靠

缺点:
- PG信号的阈值和延迟因DC-DC型号而异
- 级联级数多时累积延迟大
- 掉电时序可能不受控(PG信号释放速度不同)

### 2.3 专用时序控制器

对于复杂系统(4路以上电源, 严格时序要求), 使用专用时序IC:

| 型号 | 厂商 | 通道数 | 接口 | 特点 |
|------|------|--------|------|------|
| LM3880 | TI | 3 | 无(固定顺序) | 简单, 三个标志输出 |
| TPS3880 | TI | 3 | 无 | 精确延时, 看门狗 |
| ADM1186 | ADI | 4 | I2C | 可编程, 电压监测 |
| LTC2928 | ADI | 4 | 电阻配置 | 电压监测, 时序控制 |
| MAX16046 | Maxim | 12 | I2C/SMBus | 复杂系统, 闪存配置 |

专用时序控制器提供:
- 精确的可编程延迟(1ms到1000ms)
- 电压阈值监测(确认每路电源真的达到目标)
- 上电和掉电双向时序控制
- 故障管理和报警输出

## 3 Power-Good信号的使用

### 3.1 PG信号特性

DC-DC转换器的Power-Good(PG)信号指示输出电压是否在规定范围内:

- PG为高: 输出电压在目标值的+/-5%到+/-10%以内(具体阈值见数据手册)
- PG为低: 输出电压异常(过高或过低)
- 开漏输出: 需要外部上拉电阻(通常10k到100k欧姆)

### 3.2 PG信号的时序

```
输出电压
  +--+
  |  |
  |  |  稳定输出
  |  +---------------------------
  |
--+--+---------------------------+----> 时间
     ^                          ^
   软启动完成                输出掉电
     |                          |
PG   +--------------------------+
     低   高(有效)             低(失效)
```

从输出电压稳定到PG有效, 通常有0.5到5ms的内部延迟(防抖), 防止电压短暂波动导致PG反复跳变.

### 3.3 PG作为时序使能信号

将前级的PG连接到后级的EN引脚:

```
[DC-DC 3.3V].PG ---[100k上拉到3.3V]--- [DC-DC 1.8V].EN
```

注意事项:
- PG是开漏输出, 必须上拉
- 上拉电压不能超过后级EN引脚的最大电压
- 如果前级PG释放太快(掉电时), 可能导致后级还没关断, 时序失控

## 4 软启动与浪涌电流限制

DC-DC启动时, 输出电容从0V充电, 瞬间电流 I_inrush = C_out * dV/dt 可能很大. 如100uF电容在1ms内充到3.3V, 浪涌电流达0.33A, 叠加正常启动电流可能导致输入电压跌落或前级过流保护.

DC-DC内部通常集成软启动功能, 控制输出电压上升速率, 时间1到10ms. 有些DC-DC通过外部CSS电容可调: t_ss = CSS * 0.8V / I_SS (I_SS为内部恒流源, 5到10uA).

对于无内置软启动的LDO, 可用RC电路延迟EN引脚, 但这只延迟启动时刻, 不控制输出上升斜率.

## 5 热插拔与浪涌电流限制

IoT设备中常见热插拔场景: USB接口、可更换电池、模块化扩展板.

专用热插拔IC(如TPS2041, MAX4370, LTC4231)在插入瞬间限制浪涌电流, 然后正常导通. 内部通过MOSFET和电流检测电阻实现限流.

简单方案使用NTC热敏电阻: 冷态阻值大(限流), 通电后发热阻值降低. 缺点是频繁热插拔时NTC未冷却, 失去限流效果.

## 6 反向极性保护

### 6.1 二极管保护

最简单的方法: 串联一个二极管.

```
V_IN ---|>|--- V_PROTECTED  (正接时导通)
V_IN ---|<|--- V_PROTECTED  (反接时截止, 保护后级)
```

缺点: 正向压降0.3到0.7V, 在低压/大电流系统中功耗显著.

### 6.2 P-MOSFET保护

利用P-MOSFET的体二极管方向实现低损耗反向保护:

```
         P-MOSFET
V_IN ----[S   D]---- V_PROTECTED
         |   |
    [G]--+   (体二极管方向: S->D)
    |
    +---[R_pulldown 100k]--- GND

工作原理:
1. 正接时: 电流通过体二极管先导通, V_PROTECTED上升
   V_GS = 0 - V_IN = -V_IN, MOSFET导通, 压降极低
2. 反接时: V_GS = 0, MOSFET截止, 电路断开
```

关键参数选择:
- V_GS(th) < V_IN_min (确保在最低输入电压下也能导通)
- |V_DS(max)| > V_IN_max
- R_DS(on) 尽量小(降低压降和功耗)

### 6.3 理想二极管

理想二极管IC内部集成MOSFET和控制电路, 模拟理想二极管特性:

- 正向压降仅10到30mV(远低于二极管的0.3到0.7V)
- 反向截止速度快(微秒级)
- 支持或门(OR-ing)并联冗余电源

常用IC: MAX40200, LTC4357, LM74700.

## 7 过压保护

**TVS管**: 瞬态电压抑制管在电压超过击穿值时迅速导通泄放能量, 响应时间亚纳秒级. 选型关键: V_RWM(反向工作电压)大于正常工作电压, V_BR(击穿电压)为V_RWM的1.1到1.2倍, P_PP(峰值功耗)需大于预期浪涌能量.

**齐纳钳位**: 齐纳二极管将输出钳位在齐纳电压, 但功耗能力有限, 仅适合小功率信号线保护.

**过压保护IC**: 专用OVP IC(如TPS2595, MAX4840, NCP346)在检测到过压时快速断开MOSFET. 响应时间微秒级, 比TVS慢, 但可自恢复和编程阈值.

## 8 欠压锁定(UVLO)

### 8.1 UVLO的作用

欠压锁定(Under-Voltage Lockout)确保输入电压低于阈值时DC-DC不工作:

- 防止低电压下DC-DC输出不稳定(可能输出错误电压)
- 防止低电压下MOSFET驱动不足(导通电阻大, 过热)
- 确保上电时电压足够才开始工作

### 8.2 UVLO阈值设置

DC-DC的UVLO通常通过分压电阻设置:

```
V_IN ---[R1]---+--- EN引脚
               |
              [R2]
               |
              GND

V_EN_rise = V_IN * R2 / (R1 + R2)
V_IN_rise = V_EN_rise * (R1 + R2) / R2
```

示例: V_EN阈值1.2V, 要求V_IN高于4.0V才开始工作:

```
R2 / (R1 + R2) = 1.2 / 4.0 = 0.3
取 R2 = 30k, R1 = 70k
```

### 8.3 UVLO迟滞

为防止输入电压在阈值附近波动导致反复启停, UVLO通常有迟滞:

```
上升阈值: 4.0V (开始工作)
下降阈值: 3.6V (停止工作)
迟滞: 0.4V
```

DC-DC内部通常有固定迟滞(100到400mV), 通过EN引脚外加分压电阻可以调整.

## 9 上电复位(POR)电路

上电复位(Power-On Reset)确保MCU在电源稳定前保持复位状态, 防止电压不足时执行代码(可能误写Flash)或寄存器状态不确定.

简单方案用RC电路, 但电压缓慢上升时可能无法产生有效复位脉冲. 可靠方案用专用复位IC(如CAT811, MCP100), 提供精确阈值(2.93V, 3.08V等)和固定脉冲宽度(140到560ms).

多路电源系统中, 所有电源稳定后才能释放复位:

```
3.3V_PG ---+
1.8V_PG ---+--- [AND门] --- [复位IC EN] --- RESET# ---> MCU
2.8V_PG ---+
```

只有所有PG信号都有效时才释放MCU复位, 确保MCU开始执行代码时所有供电域已稳定.

## 10 实际设计案例: IoT网关多路电源

### 10.1 系统需求

一个IoT网关的电源架构:

| 电源轨 | 电压 | 电流 | 用途 | 时序优先级 |
|--------|------|------|------|-----------|
| 5V_USB | 5.0V | 2A | USB输入 | N/A(外部供电) |
| 3.3V_DIG | 3.3V | 800mA | MCU, 数字外设 | 1(最先) |
| 1.8V_CORE | 1.8V | 500mA | MCU内核, DDR | 2 |
| 2.8V_ANA | 2.8V | 200mA | 传感器模拟前端 | 3(最后) |

### 10.2 电源树设计

```
5V_USB ----+--- [TPS563200 3.3V] ----+--- 3.3V_DIG
            |                         |
            |                    [PG]----+
            |                         |   |
            |      [TLV62569 1.8V] <--+   |
            |           |                 |
            |      [PG]----+              |
            |               |              |
            |    [TLV70033 2.8V LDO] <----+
            |           |
            |      2.8V_ANA
            |
            +--- [TVS 5.5V] --- GND
```

### 10.3 时序控制方案

采用使能级联法:

1. 5V_USB上电, UVLO确保电压>4.5V后TPS563200开始工作
2. TPS563200输出3.3V, PG信号有效后使能TLV62569
3. TLV62569输出1.8V, PG信号有效后使能TLV70033
4. TLV70033输出2.8V
5. 3.3V和1.8V的PG信号AND后驱动复位IC, 释放MCU复位

### 10.4 保护电路

- 输入: TVS管(5.5V) + 保险丝(3A) + P-MOSFET反向保护
- 3.3V: 过压保护IC(TPS2595, 阈值3.8V)
- 每路DC-DC: 输入输出电容紧邻放置
- 复位: CAT811(阈值2.93V), 确保3.3V稳定后MCU才启动

### 10.5 时序验证用示波器

上电时序需要用示波器验证:

- 4个通道分别接5V, 3.3V, 1.8V, 2.8V
- 触发源选5V上升沿
- 时间刻度设为5ms/div
- 确认: 每路电源在前级稳定后才开始上升, 无反向馈电

验证清单:
- [ ] 3.3V在5V稳定后开始上升
- [ ] 1.8V在3.3V_PG有效后开始上升
- [ ] 2.8V在1.8V_PG有效后开始上升
- [ ] 所有电源上升斜率<50mV/us(无振铃)
- [ ] 掉电时高电压先降, 低电压后降(或同时)
- [ ] 快速重复上掉电10次, 时序一致

## 11 常见时序设计错误

### 11.1 错误: 用同一个EN信号同时使能所有电源

```
EN ---+--- [3.3V DC-DC].EN
      +--- [1.8V DC-DC].EN    (错误: 同时启动, 无时序)
      +--- [2.8V LDO].EN
```

问题: 不同DC-DC的软启动时间不同, 输出电容不同, 实际上升时刻不确定. 碰巧可能I/O先于内核上电.

### 11.2 错误: 忽略掉电时序

上电时序设计完善, 但掉电时各路电源按各自电容大小随机下降. 可能出现:
- 3.3V(I/O)电容大, 掉电慢
- 1.8V(内核)电容小, 掉电快
- 结果: I/O电压还在, 内核已掉电, 闩锁风险

解决: 确保I/O电容小于内核电容, 或使用时序控制器同时管理上电和掉电.

### 11.3 错误: PG信号上拉到错误的电压

PG开漏输出需要上拉电阻, 上拉电压选择:
- 正确: 上拉到PG输出侧的电源(即后级EN引脚能接受的电压)
- 错误: 上拉到前级输出(可能超过后级EN引脚耐压)

### 11.4 错误: UVLO阈值设置过高

如果UVLO设在4.5V, 但USB线缆压降导致实际输入只有4.3V, 系统永远无法启动. 建议:
- UVLO上升阈值设为最低工作电压以下0.5V
- 考虑线缆压降、连接器接触电阻
- 留足UVLO迟滞(>200mV)

### 11.5 错误: 软启动电容选择不当

CSS太大导致软启动时间过长(>100ms), 系统启动慢; CSS太小导致浪涌电流过大, 前级过流保护. 建议:
- 计算浪涌电流: I_inrush = C_out * V_out / t_ss
- 确保I_inrush < 前级过流阈值的70%
- 软启动时间通常2到10ms

## 总结

多路电源上电时序设计的核心原则:

1. **先内核后I/O**: 防止闩锁, 确保芯片内部控制电路先就绪
2. **用PG而非延时**: Power-Good信号确认前级真正稳定, 比固定延时可靠
3. **上下电都要管**: 掉电时序同样重要, 不能只管上电
4. **保护要独立**: 反向保护、过压保护、UVLO是独立于时序的安全防线
5. **用示波器验证**: 纸面设计必须通过实际测量确认

对于简单系统(2到3路电源), 使能级联法足够. 4路以上或时序要求严格(如FPGA), 使用专用时序控制器. 无论哪种方案, 最终都要用示波器验证上电和掉电波形.
