# 传感器老化漂移补偿与在线校准策略

> **难度**：🔴 高级 | **领域**：传感器可靠性 | **关键词**：漂移, 在线校准, 灵敏度, 寿命, 冗余 | **阅读时间**：约 18 分钟

## 日常类比

弹簧秤用久了弹性变差，同样 1 kg 指针偏少——传感器也会“疲劳”：电化学电极消耗、微机电系统（MEMS）蠕变、发光二极管（LED）光衰。老化漂移缓慢且常不可逆；不补偿时，两年后的读数可能系统性偏了你还当它准[1][2]。

## 摘要

归纳机理与漂移类型、离线/在线补偿、冗余与寿命预测思路。寿命“约 2–3 年”、漂移率等为类型级经验量级，具体器件以数据手册与加速寿命试验为准[3][6]。

## 1. 老化机理

| 类型 | 机理线索 | 影响 |
|------|----------|------|
| 电化学气体 | 电解液/电极消耗、干涸 | 灵敏度下降 |
| MEMS | 蠕变、疲劳、粘连、颗粒 | 零偏/灵敏度变 |
| 光学 | 光源衰减、窗污染 | 示值漂移 |
| 热敏 | 封装应力、湿气 | 曲线缓变 |

## 2. 漂移类型

| 类型 | 表现 |
|------|------|
| 零点漂移 | 空输入输出爬移 |
| 灵敏度漂移 | 斜率变 |
| 非线性恶化 | 高阶项变大 |
| 突发阶跃 | 冲击/中毒后跳变（近故障） |

## 3. 校准与在线补偿

离线：送检或现场标准气/标准源。在线：基线跟踪、双传感器差分、温度联动模型、周期性自动调零（需确认“零点环境”可信）。机器学习可拟合慢漂移，但需防把真实缓变工况学成“新正常”[4][5][7]。

| 方法 | 条件 | 风险 |
|------|------|------|
| 定期离线校准 | 有标准 | 成本高 |
| 自动调零 | 可知清洁基线 | 污染基线则越补越错 |
| 模型+温湿度补偿 | 有表征数据 | 过拟合 |
| 冗余投票 | 多传感器 | 共模老化 |

## 4. 维护与寿命

用灵敏度趋势或残差超限触发维护；加速寿命与现场统计结合做更换策略。虚拟传感器（用相关通道估计）可作软备份，但不能替代安全关键硬传感[6][8]。

| 策略 | 目的 |
|------|------|
| 趋势告警 | 计划维护 |
| 冗余切换 | 降级运行 |
| 寿命模型 | 备件与巡检 |

## 5. 局限、挑战与可改进方向

### 1. 把冲击阶跃当慢漂移滤掉

**局限**：中毒/撞击后应用慢滤波器会长期偏差。
**改进**：与故障检测隔离（FDI）联动，阶跃走故障通道[8]。

### 2. 无可信零点却自动调零

**局限**：在污染环境“调零”固化错误。
**改进**：零点条件检测；否则禁止自动调零[4]。

### 3. 单传感器机器学习硬扛

**局限**：标签少、概念漂移，现场难维护。
**改进**：先物理特征与温度补偿；ML 作辅助并监控残差[5][7]。

### 4. 忽略共模老化

**局限**：同批次双传感器一起漂，投票失效。
**改进**：异构冗余或定期外标[2][6]。

## 总结

老化补偿是“可维护性设计”：机理决定观测特征，校准策略决定能否纠偏，FDI 决定何时该换而非死补。长期 IoT 传感没有“一次校准管终身”。
