# UART/RS-232/RS-485串口通信对比

> **难度**：🟢 初级 | **领域**：串行通信接口 | **阅读时间**：约 18 分钟

## 引言

想象你在一个办公室里和同事交流。UART 像是你们两人之间的默契约定：谁先开口、说多快、一句话多长，全靠事先商量好，没有任何额外的信号线来协调。RS-232 是把这种交流延伸到走廊对面的办公室，声音要大一些(电压摆幅大)才能远距离传达，但只能一对一说话。RS-485 则像是装了对讲系统的楼层广播：一个人按住对讲键说话，其他人都能听到，而且因为用了差分信号(两根线一起传)，即使楼里有噪声也能听清楚。

在物联网和嵌入式开发中，"串口"这个词经常出现，但它其实包含了三个不同层次的概念：UART 是通信协议，RS-232 和 RS-485 是电气标准。很多初学者把它们混为一谈，导致选型错误、通信不稳定。本文将逐一拆解，帮你建立清晰的知识框架。

## 1. UART 基础

### 1.1 什么是 UART

UART (Universal Asynchronous Receiver/Transmitter) 是一种异步串行通信协议。它定义了数据如何一位一位地传输，但不规定电平高低、线缆类型等物理细节。

核心特点：

- **异步**：没有单独的时钟线，收发双方靠事先约定的波特率来同步
- **全双工**：发送(TX)和接收(RX)独立，可以同时进行
- **点对点**：一根 TX 连一根 RX，天然只支持两个设备之间通信

### 1.2 帧格式

一个完整的 UART 数据帧：

```
空闲(高) -> [起始位0] [数据位5-9bit, LSB first] [校验位0/1] [停止位1/1.5/2]

起始位: 1 bit, 固定低电平, 下降沿告诉接收方"数据来了"
数据位: 常用 8 bit, 低位在前
校验位: 可选(无/奇/偶), 工业场景常用偶校验
停止位: 固定高电平, 1 bit 最常见
```

### 1.3 波特率

波特率是每秒传输的比特数，收发双方必须一致，否则数据乱码。

| 常用波特率 | 适用场景 |
|-----------|---------|
| 9600 | 低速设备(GPS模块等) |
| 19200 | 中速传感器 |
| 38400 | 调试输出 |
| 57600 | 蓝牙模块 |
| 115200 | 最常用的调试和通信速率 |

波特率容差约 2-3%，采样点在每 bit 中间位置，偏差不超过半 bit 时间就不会出错。

### 1.4 UART 的局限

1. **没有电气标准**：TTL 电平(0-3.3V/5V)不适合远距离传输
2. **没有寻址机制**：只能一对一，无法区分多个设备
3. **没有总线仲裁**：多设备同时发送会冲突
4. **通信距离短**：超过 1-2 米就不可靠

正因为这些局限，才需要 RS-232 和 RS-485 来"加持"UART。

## 2. RS-232 电气标准

### 2.1 概述

RS-232 是 EIA 于 1962 年发布的串行通信电气标准，定义了电压范围、连接器、信号线等物理层细节。

核心特点：

- **负逻辑**：-3V 到 -15V = 逻辑 1(Mark)，+3V 到 +15V = 逻辑 0(Space)
- **点对点**：只支持两个设备
- **全双工**：收发独立
- **通信距离**：约 15 米

### 2.2 电压电平

```
RS-232:  +3V ~ +15V = 逻辑0    |    TTL:  3.3V/5V = 逻辑1
         -3V ~ -15V = 逻辑1    |           0V    = 逻辑0
```

RS-232 用负逻辑且摆幅大(最大 30V)，比 TTL 更抗干扰。但 MCU 引脚不能直连，必须经电平转换芯片。

### 2.3 DB9 连接器

最少只需 3 根线即可通信：

| 引脚 | 名称 | 说明 |
|------|------|------|
| 2 | RXD | 接收数据 |
| 3 | TXD | 发送数据 |
| 5 | GND | 信号地 |

两台 DTE 直连时需要交叉接线(Null Modem)：A 的 TXD 接 B 的 RXD，A 的 RXD 接 B 的 TXD。

### 2.4 RS-232 的局限

| 局限 | 原因 |
|------|------|
| 距离短(约15m) | 单端信号，长距离衰减严重 |
| 只能点对点 | 没有总线机制 |
| 抗干扰有限 | 单端信号参考地线，共模噪声直接影响 |
| 电压高不安全 | +/-15V 对低压 MCU 有风险 |

## 3. RS-485 电气标准

### 3.1 概述

RS-485 是为解决 RS-232 距离和节点数限制而设计的差分总线标准。

核心特点：

- **差分信号**：A/B 两根线的电压差表示逻辑值，抗共模噪声
- **多点通信**：一条总线可挂 32 个单位负载(1/4 负载芯片可达 128 个)
- **半双工或全双工**：取决于两线还是四线
- **通信距离远**：最远 1200 米

### 3.2 差分信号原理

```
逻辑 1: A - B < -200mV  (A 低, B 高)
逻辑 0: A - B > +200mV  (A 高, B 低)

接收端只看差值, 不关心绝对电压
共模噪声同时耦合到 A/B, 差值不变 -> 抗干扰
```

外部干扰通常同时耦合到两根线上(共模)，接收端做减法后噪声被抵消。就像两个人同时听到背景噪音，但比较他们说话的差别就能提取信息。

### 3.3 主要参数

| 参数 | 典型值 |
|------|--------|
| 共模电压范围 | -7V 到 +12V |
| 差分输出电压 | 最小 1.5V(54 欧姆负载) |
| 接收灵敏度 | +/- 200mV |
| 最大节点数 | 32(1/4 负载芯片 128, 1/8 负载芯片 256) |
| 最大速率 | 10 Mbps(短距离) |
| 最大距离 | 1200 m(低速) |

### 3.4 RS-422 简介

RS-422 是 RS-485 的"前身"，同样差分信号，但关键区别：RS-422 总线上只有 1 个驱动器(一发多收，10 个接收器)，全双工 4 线；RS-485 支持多驱动器多接收器，更通用。新设计除非有明确单向广播需求，一般直接选 RS-485。

## 4. 三者关系与层级

### 4.1 协议层 vs 物理层

UART 是协议层，RS-232/RS-485 是物理层(电气标准)，是上下层关系而非互斥：

```
应用层: 用户数据
   |
UART 协议层: 帧格式、波特率
   |
物理层: RS-232 / RS-485 / TTL
   |
传输介质: 线缆 / PCB走线
```

数据流：应用数据 -> UART 加帧 -> 电平转换芯片 -> 线缆传输。所以"RS-485 通信"的数据帧格式本质上还是 UART 那一套，只是物理层用了差分信号。

### 4.2 对比总览

| 特性 | UART(TTL) | RS-232 | RS-485 |
|------|-----------|--------|--------|
| 信号类型 | 单端 | 单端 | 差分 |
| 逻辑电平 | 0/3.3V 或 0/5V | +/-3V 到 +/-15V | 差分 +/-200mV |
| 拓扑 | 点对点 | 点对点 | 多点总线 |
| 节点数 | 2 | 2 | 32(可扩至256) |
| 最大距离 | 1-2 m | 约 15 m | 约 1200 m |
| 双工方式 | 全双工 | 全双工 | 半双工/全双工 |
| 抗干扰 | 差 | 一般 | 强 |
| 典型用途 | 芯片间通信 | PC与设备通信 | 工业总线、IoT |

## 5. 电平转换芯片

### 5.1 MAX232 (TTL <-> RS-232)

MAX232 内部集成电荷泵，4 个外接电容即可将 5V TTL 转为 +/-10V RS-232 电平。3.3V 版本选 MAX3232 或 SP3232。

### 5.2 MAX485 / SP3485 (TTL <-> RS-485)

RS-485 收发器引脚功能：DI(数据输入, 接 MCU TX)、RO(数据输出, 接 MCU RX)、DE(发送使能, 高有效)、RE(接收使能, 低有效)、A/B(差分总线端)。

| 芯片 | 速率 | 供电 | 节点负载 | 备注 |
|------|------|------|---------|------|
| MAX485 | 2.5 Mbps | 5V | 1 单位负载 | 最常用 |
| MAX3485 | 3.3V 版 | 3.3V | 1 单位负载 | 3.3V 系统 |
| SP3485 | 10 Mbps | 3.3V | 1 单位负载 | 高速常用 |
| MAX487 | 0.25 Mbps | 5V | 1/4 单位负载 | 可挂 128 节点 |

## 6. RS-485 半双工与全双工

### 6.1 两线半双工(最常用)

所有设备共享 A/B 两根线，同一时刻只能一个设备发送。核心挑战是方向控制：发送前切换为发送模式，发完立即切回接收模式，否则会"堵住"总线。

### 6.2 四线全双工

使用独立的发送对和接收对，主机发对所有从机收，一个从机发对主机收。需要双倍线缆，工业中较少见。

### 6.3 方向控制(DE/RE 引脚)

DE 和 RE 通常连在一起由一个 GPIO 控制，这是最容易出问题的地方：

```c
// STM32 RS-485 方向控制
#define RS485_DE_HIGH()  HAL_GPIO_WritePin(DE_PORT, DE_PIN, GPIO_PIN_SET)
#define RS485_DE_LOW()   HAL_GPIO_WritePin(DE_PORT, DE_PIN, GPIO_PIN_RESET)

void RS485_SendData(uint8_t *data, uint16_t len)
{
    RS485_DE_HIGH();            // 切换为发送模式
    HAL_UART_Transmit(&huart1, data, len, 100);
    // 关键: 等待最后一个字节完全发出
    while (__HAL_UART_GET_FLAG(&huart1, UART_FLAG_TC) == RESET);
    RS485_DE_LOW();             // 切回接收模式
}
```

常见错误：用 `TXE`(发送寄存器空)代替 `TC`(发送完成)。TXE 只表示数据从 DR 移到了移位寄存器，字节还没完全发出，提前切换方向会截断最后一字节。

## 7. 终端电阻与偏置电阻

### 7.1 终端电阻

总线两端各加 120 欧姆终端电阻，匹配传输线特性阻抗，防止信号反射。万用表测 A-B 约 60 欧姆(两个 120R 并联)。

注意事项：只在两端加，中间节点不加；短距离低速可不用但加上更稳妥；终端电阻增加直流负载。

### 7.2 偏置电阻

总线无人发送时 A/B 悬空，噪声可能导致接收器随机跳变。偏置电阻给总线确定状态：

```
VCC -[560R]- A -[120R终端]- B -[560R]- GND
A ≈ 2.65V, B ≈ 2.35V, 差值 ≈ 300mV > 200mV 阈值
空闲时保持逻辑 1
```

偏置电阻只在主机端加一处即可。

## 8. 总线拓扑

### 8.1 菊花链(推荐)

所有节点依次串联在主干线上，分支(stub)尽量短(< 0.3m)：

```
主机 ─── 从机1 ─── 从机2 ─── 从机3
 T       stub       stub       T

T = 终端电阻, 长分支会造成信号反射
```

### 8.2 星型(不推荐)

所有节点连到中心分线点，每个分支都是反射源，很难用终端电阻匹配。若必须用，控制分支短且一致。

### 8.3 拓扑经验法则

1. 主干走直线，不走环形
2. 分支长度不超过总线 10%
3. 终端电阻只在两端
4. 用 120 欧姆特性阻抗双绞线
5. 超过 1200 米需中继器

## 9. 距离与波特率的关系

RS-485 距离和波特率此消彼长，经验公式：距离(m) x 波特率(bps) <= 10^8

| 距离 | 推荐波特率 | 备注 |
|------|-----------|------|
| < 10 m | 最高 10 Mbps | 短距离高速 |
| 10-100 m | 9600-115200 bps | 典型工厂车间 |
| 100-300 m | 9600-38400 bps | 建筑内跨层 |
| 300-1200 m | 1200-9600 bps | 建筑间或厂区 |

波特率选择应留余量。理论上 100m 支持 1Mbps，建议只用 115200 bps。

## 10. STM32 UART/USART 配置

### 10.1 UART 与 USART 的区别

- **UART**：仅异步通信
- **USART**：支持同步+异步(多一个 SCLK 时钟引脚)

RS-485 通信用 UART 模式即可。

### 10.2 HAL 库配置

```c
UART_HandleTypeDef huart1;

void MX_USART1_UART_Init(void)
{
    huart1.Instance = USART1;
    huart1.Init.BaudRate = 115200;
    huart1.Init.WordLength = UART_WORDLENGTH_8B;
    huart1.Init.StopBits = UART_STOPBITS_1;
    huart1.Init.Parity = UART_PARITY_NONE;
    huart1.Init.Mode = UART_MODE_TX_RX;
    huart1.Init.HWFlowCtl = UART_HWCONTROL_NONE;
    huart1.Init.OverSampling = UART_OVERSAMPLING_16;
    HAL_UART_Init(&huart1);
}
```

| 参数 | 常用选择 | 说明 |
|------|---------|------|
| WordLength | 8B | 9B 用于带地址唤醒模式 |
| StopBits | 1 | 对端较慢时用 2 |
| Parity | None | 8N1 是最常见配置 |
| HWFlowCtl | None | RS-485 不用硬件流控 |

### 10.3 RS-485 收发完整示例

```c
// 发送(中断方式)
void RS485_Send(uint8_t *buf, uint16_t len)
{
    RS485_DE_HIGH();
    HAL_UART_Transmit_IT(&huart1, buf, len);
}

void HAL_UART_TxCpltCallback(UART_HandleTypeDef *huart)
{
    if (huart->Instance == USART1) RS485_DE_LOW();
}

// 接收(中断方式)
void RS485_StartReceive(void)
{
    HAL_UART_Receive_IT(&huart1, rx_buf, RX_BUF_SIZE);
}

void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart)
{
    if (huart->Instance == USART1) {
        ProcessData(rx_buf, RX_BUF_SIZE);
        HAL_UART_Receive_IT(&huart1, rx_buf, RX_BUF_SIZE);
    }
}
```

## 11. 工业 IoT 多节点通信实例

### 11.1 场景与架构

一条生产线监控 8 个温湿度传感器、4 个压力传感器和 2 个 PLC，通过 RS-485 连到边缘网关(STM32F407)，再经以太网上报云平台。

```
云平台 --以太网-- 边缘网关(STM32F407)
                      |  RS-485, 9600bps, 8N1
                      +-- 温湿度 x8 (0x01-0x08)
                      +-- 压力 x4   (0x11-0x14)
                      +-- PLC x2    (0x21-0x22)
```

### 11.2 通信协议

自定义 Modbus-RTU 风格协议：

```
帧格式: | 地址(1B) | 功能码(1B) | 数据长度(1B) | 数据(NB) | CRC16(2B) |
功能码: 0x03=读数据, 0x83=读响应, 0x06=写数据, 0x86=写确认
轮询: 温湿度每5秒, 压力每2秒, PLC事件驱动
```

### 11.3 时序分析

9600bps 下每字节约 1.04ms。查询帧 5 字节约 5.2ms，响应帧 9 字节约 9.4ms，一个传感器约 20ms，8 个轮询约 160ms，远小于 5 秒周期。总线负载率约 3.2%，留有充足余量。

### 11.4 实际踩坑提醒

1. **地址冲突**：新传感器默认地址可能是 0x01，上线前必须逐一配置
2. **总线死锁**：某节点 DE 卡在高电平会瘫痪整条总线，加超时检测和硬件看门狗
3. **地线连接**：节点间地电位差超 -7V~+12V 会损坏收发器，远距离必须连 GND
4. **线缆选择**：用屏蔽双绞线，屏蔽层单点接地

## 12. 总结

### 12.1 选型决策树

```
需要串口通信?
  +-- PCB板内两芯片间? -> UART(TTL), 直连TX/RX
  +-- 连PC或传统设备?  -> RS-232(短距离, 点对点)
  +-- 多节点或远距离?  -> RS-485
        +-- 节点>2 或 距离>15m? -> RS-485, 毫不犹豫
```

### 12.2 核心要点速查

| 要点 | 说明 |
|------|------|
| UART 是协议, RS-232/RS-485 是电气标准 | 不在同一层, 不是替代关系 |
| RS-232 用单端大摆幅抗噪 | 适合短距离点对点 |
| RS-485 用差分信号抗噪 | 适合长距离多节点 |
| RS-485 半双工需方向控制 | DE/RE 引脚, 用 TC 标志不是 TXE |
| 终端电阻只在两端加 | 120 欧姆, 中间节点不加 |
| 偏置电阻防止总线悬空 | 空闲时保持确定状态 |
| 距离 x 波特率 <= 10^8 | 经验公式, 留余量 |
| 菊花链拓扑最优 | 避免长分支和星型 |

### 12.3 从这里出发

- Modbus-RTU 协议：基于 RS-485 的应用层协议，工业标准
- CAN 总线：更高级的差分总线，自带仲裁和错误处理
- RS-485 隔离设计：光耦/磁隔离，提高系统可靠性
