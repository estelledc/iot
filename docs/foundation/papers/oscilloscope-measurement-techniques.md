# 示波器测量技巧：触发/解码/电源纹波
> **难度**：🟢 初级 | **领域**：测试测量工具 | **阅读时间**：约 18 分钟

## 引言

示波器就像高速摄影机：肉眼无法看清子弹穿过苹果的瞬间，但高速摄影机可以逐帧记录；同理，电路中微秒级的信号跳变肉眼不可见，示波器却能捕捉每一刻的电压变化。对于 IoT 开发者来说，示波器是排查通信时序、电源纹波、信号完整性问题的核心工具。本文从探头选择到触发设置，从协议解码到 FFT 分析，系统梳理示波器的实用测量技巧。

## 1 示波器基础

### 1.1 电压-时间显示原理

示波器的核心功能是将电压随时间的变化绘制为波形图：

- 横轴代表时间，纵轴代表电压
- 屏幕上每一格 (division) 对应设定的时间/电压刻度
- 通过调节时基 (Time/Div) 和垂直灵敏度 (Volts/Div) 改变显示范围
- 数字示波器通过 ADC 将模拟信号转换为数字样本存储并显示

### 1.2 带宽与采样率

带宽和采样率决定示波器能"看到"什么：

| 参数 | 含义 | 选择建议 |
|------|------|----------|
| 带宽 | 能准确测量的最高频率 (-3dB 点) | 至少为被测信号最高频率的 3-5 倍 |
| 采样率 | 每秒采集的样本数 | >= 5x 带宽 (Nyquist 的工程余量) |
| 存储深度 | 单次采集能保存的样本数 | 越大越好，影响长时间捕获能力 |

实际选择举例：测量 32kHz 晶振至少 100MHz 带宽，调试 I2C (100kHz) 则 70MHz 足够。

### 1.3 通道与耦合

- 通道数：2 通道满足大多数 IoT 调试，4 通道适合同时观察多路信号
- 耦合方式：DC 耦合显示信号全貌，AC 耦合滤除直流分量（电源纹波测量必用）
- 输入阻抗：通常 1M欧 并联约 15pF，与探头匹配使用

## 2 探头选择与使用

### 2.1 探头类型对比

| 探头类型 | 分压比 | 带宽 | 适用场景 | 价格 |
|----------|--------|------|----------|------|
| 无源 10:1 | 10:1 | 100-300MHz | 通用测量，大多数日常使用 | 低 |
| 无源 1:1 | 1:1 | 约 6MHz | 低电压小信号，低频精确测量 | 低 |
| 有源探头 | 1:1 | >1GHz | 高速数字信号，低电容负载 | 高 |
| 电流探头 | -- | 数十 MHz | 非侵入式电流测量 | 中-高 |

### 2.2 10:1 无源探头详解

10:1 探头是最常用的选择：

- 信号被衰减为原来的 1/10，示波器显示值需 x10 还原
- 优点：带宽高 (典型 200MHz)，对电路负载小 (10M欧 输入阻抗)
- 缺点：小信号 (mV 级) 可能被噪声淹没
- 使用时务必在示波器中设置正确的探头比

### 2.3 1:1 无源探头

1:1 探头适合低频小信号测量：

- 无衰减，信号直接输入示波器，适合 mV 级传感器信号
- 带宽仅约 6MHz，不适用于高速数字信号
- 输入电容大 (约 50-100pF)，对高频电路负载明显

### 2.4 电流探头

电流探头通过感应磁场测量电流，不需要断开电路：

- 交流电流探头：基于变压器原理，只能测交流
- 交直流电流探头：基于霍尔效应，可测交流和直流
- 适合测量电源上电冲击、电机启动电流等瞬态大电流
- 使用前需要消磁 (degauss) 和清零

## 3 触发模式

触发决定了示波器何时开始采集和显示波形。没有合适的触发，波形在屏幕上不停滚动，无法稳定观察。

### 3.1 边沿触发 (Edge Trigger)

最常见的触发方式：

- 在信号越过设定电平时触发，可选上升沿、下降沿或双边沿
- 适用场景：时钟信号、PWM 波形、通用数字信号
- 设置要点：触发电平设在信号幅度的 50% 处最稳定

### 3.2 脉冲宽度触发 (Pulse Width Trigger)

捕捉特定宽度的脉冲：

- 触发条件：脉冲宽度大于/小于/等于设定值
- 典型应用：捕获偶发的毛刺 (glitch)
- IoT 实例：捕捉看门狗复位脉冲、检测异常窄的中断信号

### 3.3 协议触发

在特定协议事件上触发：

- I2C：触发于特定地址、特定数据值
- SPI：触发于特定 MOSI/MISO 数据
- UART：触发于特定帧内容
- 优势：直接定位感兴趣的通信帧，无需在长波形中手动搜索

### 3.4 其他触发模式

| 触发类型 | 功能 | 典型用途 |
|----------|------|----------|
| 视频触发 | 同步到 NTSC/PAL 行/场 | 视频信号调试 |
| 欠幅触发 | 信号幅度低于阈值 | 检测信号跌落 |
| 超时触发 | 信号长时间未跳变 | 检测通信超时、总线挂起 |
| 逻辑触发 | 多通道逻辑组合 | 复杂时序条件 |

## 4 测量功能

### 4.1 自动测量

示波器内置多种自动测量参数：

| 测量项 | 含义 | IoT 调试场景 |
|--------|------|--------------|
| 频率 | 信号周期倒数 | 检查时钟频率 |
| 周期 | 一个完整波形的时间 | 验证 PWM 频率 |
| 上升时间 | 10% 到 90% 的时间 | 评估信号速度 |
| Vpp | 峰峰值电压 | 检查信号幅度 |
| Vrms | 有效值电压 | 评估信号能量 |
| 占空比 | 高电平占比 | 验证 PWM 输出 |

### 4.2 光标测量

手动定位测量点：

- 时间光标：测量两点间的时间差，如传播延迟
- 电压光标：测量两点间的电压差，如纹波幅度
- 追踪光标：在波形上沿曲线移动，同时显示时间和电压

### 4.3 数学运算

示波器的数学通道可以实现：

- CH1 - CH2：差分测量（无需差分探头即可测差分信号）
- CH1 x CH2：功率测量（电压通道 x 电流通道）
- FFT：频谱分析（详见第 7 节）
- 积分/微分：信号处理与滤波

## 5 电源纹波测量

电源纹波是叠加在直流输出上的交流分量，直接影响 IoT 设备的模拟性能和射频指标。

### 5.1 测量设置

纹波测量的正确设置至关重要：

1. 耦合方式：AC 耦合，滤除直流偏移，只显示纹波
2. 带宽限制：开启 20MHz 带宽限制，抑制高频噪声
3. 探头接地：使用弹簧接地夹而非鳄鱼夹长地线
4. 垂直灵敏度：调至 mV 级别，让纹波占屏幕 3-5 格

### 5.2 接地方式对比

| 接地方式 | 电感 | 效果 | 推荐度 |
|----------|------|------|--------|
| 弹簧接地夹 | 极低 | 干净的纹波波形 | 推荐 |
| 鳄鱼夹长地线 | 高 | 引入振铃和噪声 | 不推荐 |
| 探头帽直接接触 | 最低 | 最佳但操作不便 | 精密测量 |

弹簧接地夹的做法：取下探头的鳄鱼夹帽，套上弹簧接地套，弹簧另一端接触电路板的地焊盘，接地路径从数厘米缩短到数毫米。

### 5.3 纹波指标参考

| 电源类型 | 典型纹波 | 说明 |
|----------|----------|------|
| LDO 输出 | <10mVpp | 低噪声 |
| Buck 变换器 | 20-100mVpp | 与开关频率相关 |
| Boost 变换器 | 50-200mVpp | 通常比 Buck 大 |
| 电池供电 | <5mVpp | 几乎无纹波 |

纹波超标时的排查方向：输出电容 ESR 过大、布局走线引入寄生电感、反馈回路不稳定。

## 6 协议解码

### 6.1 I2C 解码

I2C 解码将模拟波形翻译为可读的协议数据：

- 设置：指定 SCL 和 SDA 对应的通道
- 显示：地址 (7 位/10 位)、读/写位、数据字节、ACK/NACK
- 优势：直接看到哪个设备发送了什么数据，无需手动数时钟周期

```
I2C 解码示例显示:
START -> Addr:0x50(W) ACK -> Data:0x00 ACK ->
Data:0x3F ACK -> STOP
```

### 6.2 SPI 解码

SPI 解码配置要点：

- 指定 SCLK、MOSI、MISO、CS 通道
- 设置时钟极性 (CPOL) 和相位 (CPHA)
- 设置 MSB/LSB 优先和 CS 有效电平
- 解码后直接显示每帧的数据值和帧间隔

### 6.3 UART 解码

UART 解码设置：

- 指定 TX/RX 通道，设置波特率、数据位、校验位、停止位
- 解码后显示 ASCII 字符或十六进制值
- 适合排查波特率不匹配、帧错误等问题

### 6.4 协议与模拟关联

解码的最大价值在于关联协议事件与模拟行为：

- 当 I2C 通信失败时，检查 SDA/SCL 的电平是否达标
- 当 SPI 数据出错时，检查时序是否满足建立/保持时间
- 当 UART 丢字节时，检查是否有噪声干扰

## 7 FFT 频谱分析

### 7.1 FFT 基本原理

快速傅里叶变换 (FFT) 将时域信号转换为频域频谱：

- 横轴从时间变为频率，纵轴从电压变为功率/幅度
- 可以直观看到信号包含哪些频率分量
- 分辨率 = 采样率 / 采样点数，点数越多频率分辨率越高

### 7.2 FFT 测量设置

| 参数 | 设置建议 | 原因 |
|------|----------|------|
| 窗函数 | Hanning (通用) | 减少频谱泄漏 |
| | Rectangular (瞬态) | 保持幅度精度 |
| | Flat Top (校准) | 幅度测量最准 |
| 采样率 | 足够高 | 提高 FFT 频率上限 |
| 存储深度 | 足够长 | 提高 FFT 频率分辨率 |

### 7.3 FFT 典型应用

- 识别噪声源：开关电源的噪声频率与开关频率对应
- 检测谐波失真：正弦波是否含有高次谐波
- 分析干扰频率：准确定位干扰信号的频率

## 8 实测案例

### 8.1 测量 PWM 占空比

```text
步骤:
1. 信号接入 CH1，设置 DC 耦合
2. 边沿触发，上升沿，电平设为信号幅度 50%
3. 时基调至显示 2-3 个完整周期
4. 开启自动测量: 频率 + 占空比
5. 调节 PWM 参数，观察占空比实时变化
```

### 8.2 调试 I2C 通信时序

```text
步骤:
1. CH1 接 SCL，CH2 接 SDA
2. 协议触发: I2C 地址 0x50 写操作
3. 开启 I2C 解码
4. 观察 ACK/NACK: NACK 说明从设备未响应
5. 检查时序: SDA 是否在 SCL 高电平期间变化
6. 测量信号上升时间: 上拉电阻过大导致上升缓慢
```

### 8.3 检查电源上电时序

```text
步骤:
1. CH1 接 3.3V 电源，CH2 接 1.8V 电源
2. 触发: CH1 上升沿，电平 1.65V (50%)
3. 时基: 1ms/Div，观察完整的上电过程
4. 测量: 两路电源上升的时间差
5. 验证: 上电顺序是否符合芯片手册要求
```

### 8.4 测量传播延迟

```text
步骤:
1. CH1 接输入信号，CH2 接输出信号
2. 边沿触发 CH1，放大时间轴到 ns 级
3. 使用光标测量两路信号过阈值的时差
4. 多次测量取平均值以消除抖动
```

## 9 探头补偿

### 9.1 为什么需要补偿

10:1 探头内部有可调电容，与示波器输入电容形成分压网络：

- 电容匹配时：方波显示平坦，频率响应均衡
- 电容过大（过补偿）：方波前有过冲，前沿出现尖峰
- 电容过小（欠补偿）：方波前沿变圆，高频衰减

### 9.2 补偿步骤

```text
1. 连接探头到示波器自带的校准方波输出 (通常 1kHz)
2. 用绝缘小螺丝刀调节探头上的补偿电容
3. 观察方波顶部: 调至最平坦为止
4. 每次更换示波器或探头后都应重新补偿
5. 多通道使用时，每个探头都需单独补偿
```

## 10 常见测量误区

### 10.1 接地引线电感导致振铃

- 错误认知：探头接地鳄鱼夹和弹簧夹效果一样
- 正确理解：长接地引线的电感在高频下与探头电容形成 LC 谐振，产生虚假的振铃
- 解决办法：使用弹簧接地夹，尽量缩短接地路径

### 10.2 带宽限制隐藏快沿

- 错误认知：100MHz 示波器看 50MHz 信号足够了
- 正确理解：50MHz 方波含有大量高次谐波，100MHz 带宽只能保留基波和 2 次谐波
- 解决办法：测量数字信号时，带宽至少为信号基频的 5 倍

### 10.3 采样率不足导致混叠

- 错误认知：采样率只要满足 Nyquist (2x) 就行
- 正确理解：Nyquist 是理论下限，实际需要 5-10 倍才能准确重建波形
- 解决办法：确认采样率 >= 5x 被测信号最高频率

### 10.4 探头负载影响电路

- 错误认知：10:1 探头 10M欧 输入阻抗不会影响电路
- 正确理解：探头的约 15pF 电容在 1MHz 时阻抗仅约 10k欧
- 解决办法：高频测量使用低电容有源探头

## 11 IoT 调试技巧

### 11.1 差分测量

利用数学通道实现差分测量：

- 设置 CH1 - CH2，等效为差分探头
- 适用于测量没有公共地的差分信号
- 注意：两个通道的偏移和增益需要精确匹配

### 11.2 分段存储 (Segmented Memory)

捕捉罕见事件：

- 设置触发条件捕获感兴趣的事件
- 示波器只保存触发点附近的波形，跳过中间无用的部分
- 适合捕获偶发的通信错误、间歇性毛刺

### 11.3 余辉显示 (Persistence)

分析信号抖动和稳定性：

- 旧波形逐渐变暗，新波形保持高亮
- 直观显示信号边沿的抖动范围
- 适合评估时钟抖动、数据眼图质量

### 11.4 滚动模式 (Roll Mode)

观察慢速变化：

- 波形从右向左滚动，类似心电图
- 适合监测电池电压变化、温度传感器信号
- 不需要触发，实时连续显示

## 总结

示波器是 IoT 硬件调试的核心工具，掌握以下要点可以显著提高调试效率：

1. 选对探头：10:1 通用，1:1 测小信号，电流探头测电流
2. 设好触发：稳定触发是有效观察的前提，善用协议触发定位问题
3. 正确测纹波：AC 耦合 + 带宽限制 + 弹簧接地
4. 善用解码：协议解码将波形翻译为可读数据，关联协议与模拟
5. 避免误区：注意接地电感、带宽限制、采样率、探头负载
6. 进阶技巧：差分测量、分段存储、余辉显示解决特定问题
