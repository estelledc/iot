# 舵机控制协议PWM/总线式在IoT中的对比

> **难度**：🟢 初级 | **领域**：舵机控制 | **阅读时间**：约 18 分钟

## 引言

想象一个木偶戏台：传统的木偶每根线只能拉一个关节，你想让木偶挥手，就得一个人拽左手线、另一个人拽右手线；而现代的电子木偶，所有关节共享一条数据线，导演只需喊"3号关节转30度"，3号就自己转到位，还能回报"我转好了"。前者就是 PWM 舵机——每个舵机独占一根信号线，主人只管发令不管收报；后者就是总线舵机——多个舵机串联在一根总线上，既能听令也能回话。

在 IoT 场景中，舵机是最常见的执行器之一：从智能家居的窗帘开合，到农业机器人的关节运动，再到 PTZ 摄像头的云台转动。选择 PWM 还是总线协议，直接影响布线复杂度、反馈能力和系统可靠性。本文从舵机原理出发，系统对比两种协议。

## 1. 舵机基本原理

### 1.1 舵机组成

一个典型舵机由四部分组成：

| 组件 | 功能 |
|------|------|
| 直流电机 | 产生旋转动力 |
| 减速齿轮组 | 降速增扭，将高速低扭转为低速高扭 |
| 位置反馈电位器 | 与输出轴同轴，实时反映当前角度 |
| 控制电路 | 比较目标角度与实际角度，驱动电机闭环调节 |

这四部分构成一个闭环位置控制系统：控制电路读取电位器电压得到当前角度，与目标角度做差，差值驱动电机转动，直到差值为零。就像空调温控——设定26度，传感器测到24度就制热，到了26度就停机。

### 1.2 闭环位置控制流程

```
目标角度 --(+)--> 误差 --> PID/PI --> 电机驱动 --> 减速齿轮 --> 输出轴
              ^                                                     |
              |              当前角度 <--- 电位器 <-----------------+
              |_____________________________________________________|
                        (反馈回路)
```

关键参数：

- **死区(Dead Band)**：误差小于此值时电机不动作，防止抖动，典型值 5-10us
- **响应速度**：从收到指令到到达目标角度的时间，与电机力矩和齿轮比有关
- **稳态精度**：最终停留在目标角度的偏差，取决于电位器分辨率和齿轮间隙

## 2. 传统PWM舵机协议

### 2.1 协议规范

PWM 舵机是最经典、最广泛使用的舵机控制方式：

- **刷新频率**：50Hz(周期 20ms)为主流标准，部分数码舵机支持 333Hz
- **脉冲宽度范围**：1ms ~ 2ms 对应 0 ~ 180 度
- **中位**：1.5ms 脉冲宽度对应 90 度(中位)

```
         |--1.5ms--|
20ms周期 |_________|
|        |         |                                  |
| 高电平 |  低电平  |          低电平延续               |
|________|_________|__________________________________|

0度:     |--1ms--|___________________________________|
90度:    |--1.5ms--|_________________________________|
180度:   |---2ms---|________________________________|
```

实际使用中，不同品牌舵机的脉冲范围可能扩展到 0.5ms ~ 2.5ms 以获得更大的转动范围，但超出 1-2ms 范围可能损坏舵机齿轮，需查阅具体规格书。

### 2.2 MCU端的PWM生成

以 STM32 为例，使用定时器输出比较模式生成 PWM：

```c
// STM32 HAL: 定时器配置为50Hz PWM
// 时钟84MHz, 预分频84 -> 1MHz计数频率
// 自动重载值 = 20000 -> 50Hz (1us分辨率)
void servo_pwm_init(TIM_HandleTypeDef *htim) {
    htim->Instance = TIM2;
    htim->Init.Prescaler = 83;       // 84MHz/84 = 1MHz
    htim->Init.CounterMode = TIM_COUNTERMODE_UP;
    htim->Init.Period = 19999;       // 1MHz/20000 = 50Hz
    htim->Init.AutoReloadPreload = TIM_AUTORELOAD_PRELOAD_ENABLE;
    HAL_TIM_PWM_Init(htim);
}

// 设置舵机角度 (0-180度 -> 1000-2000us脉冲)
void servo_set_angle(TIM_HandleTypeDef *htim, uint32_t channel, uint8_t angle) {
    // angle: 0-180
    // pulse_us = 1000 + angle * 1000/180
    uint32_t pulse = 1000 + (uint32_t)angle * 1000 / 180;
    __HAL_TIM_SET_COMPARE(htim, channel, pulse);
}
```

### 2.3 PWM舵机的局限

| 局限 | 说明 |
|------|------|
| 无反馈 | 主机只能发指令，无法获知舵机实际位置、是否到达、是否堵转 |
| 无法串联 | 每个舵机需要独立信号线，N个舵机需要N个PWM通道 |
| 线缆多 | 多舵机系统布线复杂，如6自由度机械臂需6根信号线 |
| 信息单向 | 只能从主机到舵机，舵机状态只能通过外部传感器间接获取 |
| 精度有限 | 1us分辨率下，角度分辨率约0.18度，受定时器时钟约束 |

## 3. 数字总线舵机

### 3.1 总线舵机概述

总线舵机通过串行通信协议替代 PWM，在一条总线上串联多个舵机，每个舵机有唯一 ID 地址。主流协议包括：

| 协议 | 厂商 | 物理层 | 波特率 | 代表型号 |
|------|------|--------|--------|----------|
| Dynamixel Protocol | ROBOTIS | TTL / RS485 | 9600~4.5Mbps | XM430, AX12 |
| SCS/STS Protocol | 飞特(Feetech) | TTL | 1Mbps | SCS0009, STS3215 |
| LewanSoul Protocol | 乐幻智能 | TTL | 1Mbps | LX-16A, LX-224 |

### 3.2 总线协议帧结构

以飞特 SCS 协议为例，帧结构如下：

```
| 0x55 0x55 | ID | Length | Cmd | Param1 | ... | ParamN | CheckSum |
|-----------|----|--------|-----|--------|-----|--------|----------|
| 帧头(2B)  | 1B | 1B     | 1B | N字节参数                    | 1B校验和  |
```

- **帧头**：固定 0x55 0x55，用于同步
- **ID**：舵机地址，0~253(0xFE为广播地址)
- **Length**：从 Cmd 到 CheckSum 的字节数
- **Cmd**：指令码(读/写/注册等)
- **Param**：指令参数(如目标位置、速度)
- **CheckSum**：`~(ID + Length + Cmd + Param1 + ... + ParamN) & 0xFF`

### 3.3 总线舵机通信示例

```c
// 飞特SCS协议: 写入舵机目标位置
// UART半双工模式, 发送后切换为接收等待应答
uint8_t servo_write_pos(UART_HandleTypeDef *huart, uint8_t id, uint16_t pos) {
    uint8_t buf[8];
    buf[0] = 0x55;                // 帧头
    buf[1] = 0x55;                // 帧头
    buf[2] = id;                  // 舵机ID
    buf[3] = 5;                   // 长度
    buf[4] = 0x03;                // 指令: 写
    buf[5] = 0x2A;                // 寄存器地址: 目标位置(低字节)
    buf[6] = pos & 0xFF;          // 位置低字节
    buf[7] = (pos >> 8) & 0xFF;   // 位置高字节

    // 计算校验和
    uint8_t checksum = ~(buf[2] + buf[3] + buf[4] + buf[5] + buf[6] + buf[7]) & 0xFF;

    // 发送指令帧(含校验和)
    HAL_HalfDuplex_EnableTransmitter(huart);
    HAL_UART_Transmit(huart, buf, 8, 100);
    HAL_UART_Transmit(huart, &checksum, 1, 100);

    // 切换接收, 读取舵机应答
    HAL_HalfDuplex_EnableReceiver(huart);
    uint8_t resp[6];
    HAL_UART_Receive(huart, resp, 6, 100);
    return (resp[4] == 0x00) ? 0 : 1;  // 0=成功
}
```

### 3.4 总线舵机优势

- **位置/速度/力矩反馈**：可读取当前角度、当前速度、当前负载电流，实现堵转检测
- **菊花链连接**：多个舵机串联在一根总线上，大幅减少布线
- **ID 寻址**：最多 253 个舵机共享一条总线
- **可编程 PID**：部分型号允许调整内部 PID 参数，优化动态响应
- **寄存器映射**：所有功能(位置/速度/加速度/力矩限制)映射为寄存器地址，读写统一

## 4. 协议对比

### 4.1 三种物理层对比

| 特性 | PWM | TTL串行总线 | RS485串行总线 |
|------|-----|-------------|---------------|
| 信号线数 | 每舵机1根 | 1根数据线(半双工) | 2根差分线(半双工) |
| 最大舵机数 | 受PWM通道限制(通常4-16) | 253(ID寻址) | 253(ID寻址) |
| 反馈能力 | 无 | 位置/速度/力矩/温度 | 位置/速度/力矩/温度 |
| 通信距离 | <5m(信号衰减) | <3m(TTL电平) | <10m(差分信号) |
| 抗干扰 | 一般 | 较弱(单端信号) | 强(差分信号) |
| 布线复杂度 | 高(N根信号线) | 低(1根总线) | 低(1对差分线) |
| 成本 | 最低(无协议芯片) | 中(内置协议IC) | 较高(RS485收发器) |
| 典型应用 | 简单云台/闸门 | 教育机器人/机械臂 | 工业机器人/人形机器人 |

### 4.2 选型决策流程

```
你需要舵机反馈(位置/力矩)吗？
|-- 否 --> 舵机数量 <= 4？
|           |-- 是 --> PWM舵机(最简单最便宜)
|           └── 否 --> 考虑总线舵机(减少布线)
└── 是 --> 通信距离 > 3m 或 工业环境？
            |-- 是 --> RS485总线舵机(Dynamixel)
            └── 否 --> TTL总线舵机(飞特/乐幻)
```

## 5. 舵机规格解读

### 5.1 关键参数

| 参数 | 单位 | 说明 |
|------|------|------|
| 扭矩 | kg.cm | 力臂1cm处能提起的重量，如4.8V下3.2kg.cm |
| 速度 | sec/60deg | 转动60度所需时间，如0.12s/60deg |
| 工作电压 | V | 常见4.8V/6V/7.4V/12V |
| 堵转电流 | A | 输出轴卡死时的电流，可达1-3A，远大于空载电流 |
| 齿轮材质 | - | 塑料/铜/合金钢/钛合金 |
| 死区 | us | 最小响应脉冲宽度差 |

### 5.2 齿轮材质选择

| 材质 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 塑料 | 低成本、低噪音 | 易磨损、不耐冲击 | 轻载、低频使用 |
| 铜 | 中等强度、自润滑 | 较重 | 中等负载 |
| 合金钢 | 高强度、耐冲击 | 成本高、噪音略大 | 大力矩、频繁动作 |
| 钛合金 | 最高强度、轻量 | 最贵 | 航模、竞赛机器人 |

注意：扭矩和速度与供电电压直接相关，同一舵机 6V 下的扭矩和速度通常高于 4.8V。规格书一般给出多个电压下的数据。

## 6. MCU接口设计

### 6.1 PWM接口

PWM 舵机接口只需 MCU 的定时器输出比较通道：

```
MCU GPIO/TIMx_CHy ──────> 舵机信号线(橙/黄)
                         舵机电源线(红) ──> 5-6V独立电源
                         舵机地线(棕/黑) ──> GND(与MCU共地)
```

要点：

- 使用定时器的 PWM 输出模式，分辨率至少 1us
- STM32 的 TIM 输出比较可同时驱动多路 PWM
- ESP32 的 LEDC 外设也可生成舵机 PWM

### 6.2 总线接口

总线舵机使用 UART 半双工模式，单根数据线分时收发：

```
MCU UART_TX ──┐
              ├──[1/2收发方向控制]──> 总线数据线 ──> 舵机1 ──> 舵机2 ──> ...
MCU UART_RX ──┘                          GND ─────────> GND ───> GND ──>
```

半双工方向切换的实现：

```c
// UART半双工模式初始化(STM32)
void servo_bus_init(UART_HandleTypeDef *huart) {
    huart->Instance = USART2;
    huart->Init.BaudRate = 1000000;  // 1Mbps, 飞特/乐幻默认
    huart->Init.WordLength = UART_WORDLENGTH_8B;
    huart->Init.StopBits = UART_STOPBITS_1;
    huart->Init.Parity = UART_PARITY_NONE;
    huart->Init.Mode = UART_MODE_TX_RX;
    HAL_HalfDuplex_Init(huart);
}
```

如果 MCU 不支持硬件半双工，可用 TX 输出 + RX 输入外接一个三态缓冲器(如 74HC126)实现方向控制。

## 7. IoT应用实例

### 7.1 机械臂：总线舵机多关节控制

6自由度机械臂使用总线舵机的典型方案：

- **肩关节 x2**：大扭矩舵机(如 STS3215，20kg.cm)
- **肘关节 x1**：中扭矩舵机(如 SCS0009，4.5kg.cm)
- **腕关节 x2**：小扭矩舵机(如 SCS0009)
- **夹爪 x1**：小扭矩舵机

优势：6个舵机仅1根数据线 + 1根电源线，走线沿臂布置；可读取各关节角度做运动学解算反馈。

### 7.2 智能门锁：PWM舵机简单旋转

智能门锁只需控制锁舌旋转0-90度：

- **舵机选择**：9g微型PWM舵机(SG90)
- **控制**：一个 GPIO 输出 PWM，开锁=90度，关锁=0度
- **注意**：门锁需要断电保持位置，选带自锁功能的舵机或机械自锁结构

### 7.3 PTZ云台摄像头：双PWM舵机

水平/垂直两个方向各一个 PWM 舵机：

- **水平(Pan)**：180度或360度连续旋转舵机
- **垂直(Tilt)**：90度普通舵机
- **控制**：2路 PWM，MCU 根据图像识别结果调整角度
- **扩展**：若需回传当前角度，换用总线舵机 + 位置反馈

## 8. 电源设计要点

### 8.1 电流尖峰问题

舵机启动和堵转时会产生远超额定电流的尖峰：

| 状态 | 典型电流(9g舵机) | 典型电流(20kg舵机) |
|------|-------------------|---------------------|
| 空载保持 | 5-10mA | 10-30mA |
| 正常转动 | 100-200mA | 200-500mA |
| 启动/堵转 | 500mA-1A | 1-3A |

多舵机同时启动的尖峰电流可轻松超过电源供电能力，导致电压骤降，MCU 复位。

### 8.2 独立电源轨

```
主电源(5V/2A+) ──┬──> 舵机电源轨(5-6V)
                  |     |
                  |   [大容量电容 1000uF+]
                  |     |
                  |   舵机1  舵机2  舵机3 ...
                  |
                  ├──> LDO(3.3V) ──> MCU电源轨(3.3V)
                  |
                  └──> 共地(GND必须连在一起)
```

核心原则：

- 舵机电源与 MCU 电源隔离，避免舵机电流尖峰拉低 MCU 供电
- 在舵机电源端并联大容量电解电容(1000uF以上)，吸收电流尖峰
- 地线必须连在一起，否则信号参考不一致
- 若空间允许，舵机电源用独立 DC-DC 而非与 MCU 共用 LDO

### 8.3 大容量电容选型

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 容值 | 470-2200uF | 视舵机数量和堵转电流而定 |
| 耐压 | 大于电源电压1.5倍 | 5V电源选10V或16V耐压 |
| ESR | 低ESR类型 | 铝电解或铝聚合物电容 |
| 位置 | 尽量靠近舵机电源输入 | 减小走线电感 |

## 9. IoT舵机选型指南

### 9.1 选型维度

| 维度 | 关键问题 | 建议 |
|------|----------|------|
| 扭矩 | 负载多重？力臂多长？ | 选额定扭矩1.5-2倍余量 |
| 尺寸 | 安装空间多大？ | 微型9g / 标准23g / 大型55g+ |
| 协议 | 需要反馈吗？舵机几个？ | 无反馈/少舵机=PWM，需反馈/多舵机=总线 |
| 供电 | 电池还是市电？电压多高？ | 电池供电关注空载电流和堵转功耗 |
| 价格 | BOM预算多少？ | PWM 5-30元，总线 50-300元 |
| 精度 | 定位精度要求多高？ | PWM ~1度，总线 0.1-0.3度 |

### 9.2 常见IoT场景推荐

| 场景 | 推荐类型 | 理由 |
|------|----------|------|
| 智能窗帘 | PWM舵机(SG90/MG996R) | 单轴旋转，无需反馈，低成本 |
| 智能门锁 | PWM舵机(SG90) | 小角度旋转，低频使用 |
| PTZ摄像头 | 2xPWM或2x总线 | 水平+垂直，总线可回传角度 |
| 机械臂 | 总线舵机(飞特/乐幻) | 多关节串联，需位置反馈 |
| 人形机器人 | RS485总线舵机(Dynamixel) | 高精度、多舵机、工业级 |
| 农业灌溉阀 | PWM舵机+减速齿轮 | 大扭矩、慢速、无需反馈 |
| 教育机器人 | TTL总线舵机(乐幻LX-16A) | 低成本、有反馈、易教学 |

## 总结

PWM 舵机和总线舵机各有适用场景，核心取舍在于：简单性 vs 功能性。

- **PWM 舵机**：协议极简，一根线一个脉冲就能控制，MCU 端只需定时器资源，成本最低。适合舵机数量少、不需要反馈的简单场景(门锁、窗帘、简单云台)。
- **总线舵机**：通过串行协议实现双向通信，支持位置/速度/力矩反馈，菊花链布线大幅简化多舵机系统。适合机械臂、人形机器人等需要精确控制和状态反馈的复杂场景。
- **电源设计**是舵机系统最容易出问题的环节，独立电源轨和大容量电容是必须的，否则舵机堵转时 MCU 复位是常见故障。
- 选型时从扭矩和协议两个维度切入：先确定负载需求(扭矩/速度)，再根据反馈需求和舵机数量选择协议类型。
