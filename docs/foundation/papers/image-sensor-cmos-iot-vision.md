# CMOS图像传感器在IoT视觉应用中的选型
> **难度**：🟡 中级 | **领域**：视觉传感器 | **阅读时间**：约 20 分钟

## 引言

想象一下,你站在一个大型体育馆的看台上,每个座位上坐着一个人。每个人的任务是:当光线照到他时,举起手中的牌子显示光线有多强。如果所有人同时举牌,你就能一次性看到整幅"画面"。这就是CMOS图像传感器的工作方式——每个像素就是一个独立的"座位上的人",各自把光信号转换为电信号,然后一起读出来,构成一幅图像。

CMOS图像传感器凭借低功耗、低成本和高度集成的优势,已成为IoT视觉应用的主流选择。从智能门铃到工业检测,从野生动物监测到农业感知,CMOS传感器让"会看"的IoT设备成为可能。但不同应用场景对传感器的要求差异巨大,选型时需要在分辨率、帧率、功耗和成本之间做出权衡。

## 1. CMOS图像传感器基础

### 1.1 工作原理

```
CMOS主动像素(APS)结构:
         VDD
          |
        [复位管] --- RST信号
          |
    光电二极管 --+-- [源极跟随器] -- [行选择管] -- 列输出线

工作流程:
1. 复位: RST清空光电二极管
2. 曝光: 光子产生电子-空穴对,电荷积累
3. 读出: 行选择打开,电压信号输出到列线
4. ADC: 列级ADC转数字信号
```

### 1.2 卷帘快门与全局快门

```
卷帘快门(Rolling Shutter): 逐行曝光,逐行读出
行1: |  曝光  | 读出 |
行2:   |  曝光  | 读出 |  <- 每行起始时间不同
问题: 运动物体产生"果冻效应"(倾斜/拉伸)
优点: 电路简单,功耗低,大部分IoT应用够用

全局快门(Global Shutter): 所有像素同时曝光
行1: |  曝光  | 读出 |
行2: |  曝光  | 读出 |  <- 所有行同时曝光
优点: 无果冻效应  缺点: 需存储节点,面积大成本高
```

| 对比维度 | 卷帘快门 | 全局快门 |
|----------|---------|---------|
| 果冻效应 | 有 | 无 |
| 功耗 | 低 | 较高 |
| 成本 | 低 | 高 |
| IoT适用性 | 大多数场景 | 工业检测/高速运动 |

## 2. 关键规格参数

| 参数 | 含义 | IoT关注点 |
|------|------|----------|
| 分辨率 | 像素总数(宽x高) | 不是越高越好,够用即可 |
| 像素尺寸 | 单个像素边长(um) | 越大进光越多,低光性能越好 |
| 帧率 | 每秒输出帧数(fps) | IoT常只需1-5fps |
| 量子效率(QE) | 光子转化为电子的比例 | 影响低光灵敏度 |
| 动态范围 | 最亮与最暗可同时记录的比 | 户外场景要求高 |
| 功耗 | 工作状态耗电 | 决定电池寿命 |
| 接口 | 数据输出方式 | 决定MCU/AP选型 |

像素尺寸选择原则: IoT设备通常不需要高分辨率,优先选大像素。大像素 + 低分辨率 = 低光好 + 功耗低 + 成本低。

## 3. 接口类型

### 3.1 MIPI CSI-2

```
+----------+                    +-----------+
| CMOS     | --- Data Lane 0 --|           |
| Sensor   | --- Data Lane 1 --| AP/SoC    |
|          | --- CLK Lane ----| (CSI-2 RX)|
+----------+                    +-----------+

高带宽: 1 Lane约1Gbps,4 Lane可达4Gbps
适合: 高分辨率(1080p+)、高帧率(30fps+)
典型: ESP32-S3, i.MX RT, RK3568
```

### 3.2 DVP并行接口

```
+----------+                       +-----------+
| CMOS     | -- PCLK ------------|           |
| Sensor   | -- VSYNC -----------| MCU/AP    |
|          | -- HSYNC -----------| (DCMI)    |
|          | -- D0-D7(8bit) -----|           |
+----------+                       +-----------+

带宽: PCLK * 数据位宽(如24MHz * 8bit = 24MB/s)
适合: VGA及以下, STM32F4/H7, ESP32(I2S模式)
优点: 接口简单  缺点: 信号线多,PCB布线空间大
```

### 3.3 SPI接口(低分辨率)

适合QVGA及以下,任意MCU都能驱动,最少3根信号线,帧率低分辨率受限。

### 3.4 接口选型决策

| 应用处理器 | 推荐接口 | 最大分辨率(30fps) |
|-----------|---------|------------------|
| Cortex-M0/M3 | SPI | QVGA |
| STM32F4/H7 | DVP | VGA |
| ESP32 | DVP | VGA |
| ESP32-S3 | MIPI/DVP | 1080p |
| i.MX RT1170 | MIPI CSI-2 | 1080p+ |
| RK3568 | MIPI CSI-2 | 4K |

## 4. 分辨率与应用匹配

```
QVGA (320x240): 人形检测够用,功耗最低,代表: HM01B0
VGA (640x480):  人脸检测/物体识别够用,功耗低,代表: OV7670
720p (1280x720): 人脸识别/文字识别,功耗中等,代表: OV5640
1080p (1920x1080): 细节检测/工业检测,功耗较高,代表: OV2640
```

## 5. 低光性能

### 5.1 低光性能提升路径

```
1. 增大像素尺寸 -> 每像素收集更多光子
2. 背照式(BSI)技术 -> 从背面进光,QE从约50%提升到约70%
   FSI: 金属布线在光电二极管上方(遮挡)
   BSI: 光电二极管直接受光,金属布线在下方
3. HDR模式 -> 短曝光捕获亮区+长曝光捕获暗区,动态范围120dB+
   代价: 运动物体可能有鬼影
```

### 5.2 夜间IoT场景策略

```c
typedef struct {
    float current_lux;
    int base_frame_rate;
    int night_frame_rate;
    int max_exposure_ms;
} low_light_config_t;

void adjust_camera_for_low_light(const low_light_config_t *cfg) {
    if (cfg->current_lux < 1.0f) {
        // 极低光(月光级): 延长曝光,降低帧率,最大增益
        set_exposure(cfg->max_exposure_ms);
        set_frame_rate(cfg->night_frame_rate);
        set_gain(MAX_ANALOG_GAIN);
    } else if (cfg->current_lux < 10.0f) {
        // 低光(室内暗光): 中等曝光和帧率
        set_exposure(cfg->max_exposure_ms / 2);
        set_frame_rate(3);
        set_gain(MEDIUM_GAIN);
    } else {
        // 正常光照: 自动曝光
        set_exposure(AUTO_EXPOSURE);
        set_frame_rate(cfg->base_frame_rate);
        set_gain(LOW_GAIN);
    }
}
```

## 6. 功耗优化

### 6.1 三种功耗模式

```
1. 持续工作: 始终开启,50-200mW,适合有常供电的场景
2. 快照模式: 上电->捕获->下电,平均<5mW(1fps),适合事件触发
3. 始终在线: 极低帧率持续工作,10-100uW,适合持续监测
```

### 6.2 快照模式与ROI

```c
void snapshot_capture(void) {
    sensor_power_on();
    delay_ms(5);              // 等待稳定
    sensor_set_resolution(VGA);
    sensor_trigger_capture();
    wait_frame_ready();
    read_frame_buffer(image_data);
    sensor_power_off();       // 下电节省功耗
    process_image(image_data);
}
// OV2640: 工作约120mW,待机约5mW
// 1fps平均: 120*0.1 + 5*0.9 = 16.5mW
// 0.1fps平均: 120*0.01 + 5*0.99 = 6.15mW
```

ROI(感兴趣区域): 只读出局部区域。VGA全帧614KB,ROI(320x240)仅154KB,节省75%数据量。

## 7. IoT常用CMOS传感器

| 型号 | 分辨率 | 接口 | 像素尺寸 | 特色 | 参考价格 |
|------|--------|------|---------|------|---------|
| OV2640 | 2MP | DVP | 1.4um | JPEG输出,廉价 | 1-2 USD |
| OV5640 | 5MP | DVP/MIPI | 1.4um | 自动聚焦,MIPI | 3-5 USD |
| OV7670 | VGA | DVP | 3.6um | 大像素,需MCU缓存 | 0.5-1 USD |
| GC0328 | VGA | DVP | 3.6um | 国产低价 | 0.5 USD |
| HM01B0 | QVGA | SPI/DVP | 3.6um | 超低功耗1uW/帧 | 1-2 USD |
| HM0360 | VGA | MIPI/DVP | 2.2um | 低功耗,始终在线 | 2-3 USD |

**HM01B0**: 超低功耗(1uW/帧),内置运动检测,检测到运动后中断唤醒MCU。工作流: HM01B0始终在线 -> 检测运动 -> 唤醒MCU -> 启动高分辨率传感器。

**OV5640**: 5MP自动聚焦,内置ISP,DVP+MIPI双接口,YUV/JPEG输出。适合智能门铃、安防摄像头。

## 8. 镜头与ISP

### 8.1 镜头关键参数

```
焦距选择:
- 短焦(2-4mm): 广角FOV大,适合监控大范围
- 标准焦(6-8mm): 正常视角
- 长焦(12mm+): 窄角,远处细节

FOV = 2 * arctan(sensor_width / (2 * focal_length))
VGA 1/4寸(宽3.6mm): 2.5mm->72度, 4mm->48度, 8mm->25度

安装接口: M12(工业标准,可换) / 针孔(隐蔽) / 集成镜头(免调焦)
```

### 8.2 ISP(图像信号处理器)

```
ISP流水线:
RAW -> 坏点校正 -> 黑电平 -> 镜头阴影 -> 白平衡
  -> 去马赛克 -> 降噪 -> 边缘增强 -> 色彩校正
  -> 伽马校正 -> YUV/RGB输出

关键功能: AE(自动曝光), AWB(自动白平衡), NR(降噪)
```

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 传感器内置 | MCU无需处理 | 功能有限 | 简单应用 |
| AP内置 | 功能丰富 | 需AP级处理器 | 高端应用 |
| 软件ISP | 灵活可定制 | 消耗MCU算力 | 学习/原型 |

## 9. 实际应用案例

### 9.1 智能门铃

```
+--------+     MIPI CSI-2     +------------+
| OV5640 | ----------------- | ESP32-S3   |
| 5MP+AF |   (2 Lane)        | + 8MB PSRAM|
+--------+                   +------+-----+
                                    |
                         WiFi / PIR / 按键 / 音频

1. PIR检测人体 -> 唤醒ESP32-S3
2. 启动OV5640 -> 自动聚焦
3. 捕获JPEG -> TFLite Micro人形检测
4. WiFi推流到手机APP
5. 超时 -> 传感器下电 -> 系统休眠
```

### 9.2 野生动物相机(双传感器)

```
+--------+  SPI  +----------+  DVP  +--------+
| HM01B0 | ----> | STM32L4  | ----> | OV2640 |
| 始终在线|       | 低功耗MCU|       | 2MP    |
+--------+       +----------+       +--------+

1. HM01B0始终在线,1fps,约1uW
2. 检测到运动 -> 中断唤醒MCU
3. MCU启动OV2640 -> 捕获2MP JPEG -> 存SD卡
4. OV2640下电,MCU休眠

功耗: 休眠3uW, 工作140mW*2s
每小时触发2次: 平均0.16mW, 2000mAh电池可用>1年
```

### 9.3 工业检测

```
+----------+  MIPI CSI-2  +------------+
| 全局快门  | ----------> | i.MX RT    |
| AR0234   |              | 1170 + NPU |
| 2MP 60fps|              +------------+

全局快门: 产线零件高速运动,无果冻效应,图像几何精度高
流程: 光电传感器触发 -> 全局快门同步曝光 -> NPU检测 -> OK/NG(< 10ms)
```

## 10. 事件相机简介

```
传统帧式: 固定帧率输出全帧,冗余数据大,运动模糊
事件相机(DVS): 每像素独立工作,仅输出亮度变化事件(x,y,t,polarity)
  微秒延迟, 120dB+动态范围, 几乎无冗余数据

帧式 vs 事件式:
| 维度 | 帧式 | 事件式 |
|------|------|--------|
| 输出 | 固定帧全帧 | 事件驱动仅变化 |
| 延迟 | 1/帧率 | 微秒级 |
| 动态范围 | ~60dB | >120dB |
| 算法 | 成熟 | 发展中 |
| 成本 | 低 | 高 |

IoT选择: 需完整画面(人脸识别)->帧式; 只需检测运动->事件相机(功耗低)
目前IoT主流仍是帧式相机(生态成熟,成本低)
```

## 总结

CMOS图像传感器是IoT视觉系统的核心,选型时需在分辨率、帧率、功耗和接口间权衡。对大多数IoT应用,QVGA到VGA已够,大像素比高分辨率更有价值(低光性能更好)。DVP适合MCU,MIPI CSI-2适合AP。功耗优化是关键: 快照模式和ROI可大幅降低平均功耗。HM01B0的超低功耗(1uW/帧)使其成为始终在线场景的理想选择,OV5640的自动聚焦和MIPI接口适合高质量图像应用。事件相机代表新方向,但帧式相机仍因生态成熟和低成本占IoT主导地位。
