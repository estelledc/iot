# 高速信号走线规则：差分对/等长/阻抗匹配

> **难度**：🔴 高级 | **领域**：高速PCB设计 | **阅读时间**：约 22 分钟

## 引言

想象一条城市道路：限速 30 公里时，路面有点坑洼也无所谓，车照样开。但如果这是一条高速公路，时速 120 公里，路面上一个拳头大的坑就可能导致失控翻车。PCB 走线也一样——信号变化慢的时候，铜线就是铜线；但当信号的上升沿快到纳秒级别，走线就不再是简单的导线，而变成了一条"传输线"，它有了阻抗、有了反射、有了时延差。

这篇文章要解决的核心问题：当走线变成传输线之后，你怎么布线才能让信号完整地到达接收端？三大块：差分对走线规则（USB/HDMI/Ethernet 都在用）、等长匹配规则（DDR/SDRAM 的命脉）、阻抗匹配与端接策略（信号完整性的根基）。

## 1. 什么时候走线变成传输线

### 1.1 经验法则：上升时间 vs 走线长度

判断走线是否需要当作传输线处理，业界最常用的经验法则：

```
当走线长度 > 信号上升时间所对应的传播长度的 1/6 时，
就需要按传输线来分析。

临界长度 L_crit = Tr / (6 * Tpd)

其中:
  Tr  = 信号上升时间 (ns)
  Tpd = 传播延迟, 典型 FR4 上约 0.006 ns/mm (6 ps/mm)
      即信号速度约 166 mm/ns (光速的约 55%)
```

举例：上升时间 1 ns 的信号，临界长度 = 1 / (6 * 0.006) = 27.8 mm。现代 MCU 的 GPIO 上升时间常在 0.5-2 ns，几乎任何超过几厘米的走线都需要注意信号完整性。

### 1.2 常见信号的临界长度

| 信号类型 | 典型上升时间 | 临界长度 | 需要传输线分析 |
|----------|-------------|----------|---------------|
| UART (115200) | ~500 ns | ~14 m | 否 |
| I2C (400 kHz) | ~100 ns | ~2.8 m | 通常否 |
| SPI (50 MHz) | ~5 ns | ~140 mm | 视走线长度 |
| USB 2.0 | ~0.5 ns | ~14 mm | 是 |
| DDR3-1600 | ~0.3 ns | ~8 mm | 是 |
| HDMI (3.4 Gbps/lane) | ~0.1 ns | ~2.8 mm | 是 |
| MIPI D-PHY | ~0.05 ns | ~1.4 mm | 是 |

结论：IoT 高速接口 (USB, SDIO, QSPI, MIPI) 几乎都超过了临界长度，必须按传输线规则布线。

## 2. 差分对走线规则

### 2.1 差分信号原理

差分信号用两根走线传输一个信号：一根传正相 (D+)，一根传反相 (D-)。接收端只关心差值：V_diff = V(D+) - V(D-)。

日常类比：两个人在嘈杂工地上对话。一个人喊话（单端信号），噪音会干扰理解。但如果两个人同时喊，一个说原话一个说反话，你把两个人的话相减，噪音就被抵消了——因为噪音对两个人的影响一样（共模噪声）。

关键特性：
- 共模抑制：外部干扰同时作用于两根线，差分后抵消
- EMI 抑制：两根线电流方向相反，磁场相互抵消
- 不依赖地参考：差分信号自己就是参考，减少地弹影响

### 2.2 间距与耦合

```
           间距 S (Spacing)
          <-------->
  ─────── ┐          ┌───────
  D+      │   W(线宽) │      D+
          │          │
  ─────── ┘          └───────
  D-                         D-

  紧耦合: S = 1~2W     松耦合: S > 3W
```

| 特性 | 紧耦合 (S = 1~2W) | 松耦合 (S > 3W) |
|------|-------------------|-----------------|
| 共模抑制 | 更好 | 稍差 |
| 阻抗对间距敏感度 | 高 | 低 |
| 布线难度 | 高（过孔区域难保持间距） | 低 |
| 适用场景 | USB, HDMI, PCIe | DDR 时钟对 |

实际建议：大多数 IoT 设计用紧耦合即可，过孔换层区域允许短暂松耦合（颈部放松），长度不超过 50 mil。

### 2.3 对称性与常见差分对规范

对称性是差分对布线最核心的要求——不对称会引入共模噪声和时序偏移。规则：两根线同时拐弯且半径一致、经过的过孔数量相同、与参考平面距离相同、沿途串扰环境相同。

| 接口 | 速率 | 差分阻抗 | 线宽/间距 (4层板) | 最大线长差 |
|------|------|---------|------------------|-----------|
| USB 2.0 | 480 Mbps | 90 Ohm | 5/5 mil | 150 mil |
| USB 3.0 | 5 Gbps | 90 Ohm | 4/5 mil | 5 mil |
| HDMI 2.0 | 6 Gbps/lane | 100 Ohm | 4/5 mil | 50 mil |
| Ethernet 1G | 1 Gbps | 100 Ohm | 5/5 mil | 50 mil |
| MIPI D-PHY | 2.5 Gbps | 100 Ohm | 3.5/5 mil | 10 mil |

注意：USB 用 90 Ohm 差分阻抗 (45 Ohm 单端)，HDMI/Ethernet 用 100 Ohm (50 Ohm 单端)，不要搞混。

## 3. 单端阻抗控制

### 3.1 微带线与带状线

```
微带线 (Microstrip):            带状线 (Stripline):

  ┌──────────────────┐           ┌──────────────────┐
  │    走线 W/H      │           │  上参考平面 GND   │
  │  ────────         │           ├──────────────────┤
  │  介电层 H         │           │    介电层 H1      │
  ├──────────────────┤           │  ──────── 走线    │
  │  参考平面 GND     │           │    �电层 H2      │
  └──────────────────┘           ├──────────────────┤
                                  │  下参考平面 GND   │
  Z0 与 W, H, Er 有关            └──────────────────┘
```

近似公式：微带线 Z0 = 87/sqrt(Er+1.41) * ln(5.98H/(0.8W+T))；带状线 Z0 = 60/sqrt(Er) * ln(4H/(0.67*(0.8W+T)))。其中 Er = 介电常数，W = 线宽，H = 到参考面距离，T = 铜厚。

### 3.2 关键参数与板厂配合

| 参数 | 对阻抗的影响 | 典型控制精度 |
|------|-------------|-------------|
| 线宽 W | W 增大, 阻抗降低 | +/- 0.5 mil |
| 介质厚度 H | H 增大, 阻抗升高 | +/- 10% |
| 介电常数 Er | Er 增大, 阻抗降低 | +/- 0.1 |
| 焊盘宽度 | 焊盘宽于走线, 阻抗突降 | 设计优化 |

阻抗控制标准流程：(1) 设计中指定目标阻抗 (2) Gerber 标注阻抗要求 (3) 板厂根据叠层参数反算线宽并出具阻抗叠层表 (4) 按板厂线宽更新设计 (5) 板厂生产时在测试条实测并出具报告。换板厂必须重新对阻抗，四层板精度通常 +/- 10%。

## 4. 端接策略

### 4.1 反射与端接原理

信号到达走线末端时，如果末端阻抗不等于走线阻抗，就会产生反射。反射系数 rho = (ZL - Z0) / (ZL + Z0)。ZL = Z0 时 rho = 0 无反射；ZL = 无穷 (开路) 时 rho = 1 全反射；ZL = 0 (短路) 时 rho = -1 负全反射。未端接的信号会来回振荡形成振铃。

### 4.2 四种端接方式

```
1. 串联端接: 源端串 Rs = Z0
   优点: 功耗低, 只需一个电阻
   适用: 点对点, 负载端高阻抗 (大多数 MCU 外设)

2. 并联端接: 末端并 Rt = Z0 到 GND
   优点: 全线无反射
   缺点: 持续直流功耗 (50 Ohm 到 3.3V = 66 mW)
   适用: 总线拓扑, 高速差分对

3. 戴维南端接: 末端 R1 到 VCC, R2 到 GND, R1//R2 = Z0
   优点: 可设定静态电平
   缺点: 功耗最大
   适用: 需要特定静态电平的总线 (如 PCI)

4. AC 端接: 末端串 C(100-200pF) 再并 Rt = Z0 到 GND
   优点: 无直流功耗
   缺点: 只对高频信号有效
   适用: 时钟信号, 交流耦合差分对
```

### 4.3 端接选型指南

| 场景 | 推荐端接 | 原因 |
|------|---------|------|
| MCU 到单个外设 (SPI, UART) | 串联端接 | 简单低功耗 |
| DDR 数据线 | 串联端接 (靠近 MCU) | 控制器内部有 ODT |
| DDR 地址/命令线 | 并联端接到 VTT | 多负载总线 |
| USB 差分对 | 不额外端接 | 收发器内部已匹配 |
| HDMI 差分对 | AC 耦合 + 接收端并联 | 协议要求 |
| 时钟信号 | 串联端接 | 抑制过冲 |

## 5. 等长匹配规则

### 5.1 为什么需要等长

信号在走线上的传播速度有限。同一组的信号线长度不同，信号到达接收端的时间就不一致——即时序偏移 (skew)。在 FR4 上，1 mm 走线长度差约产生 6 ps 时延差。DDR3-1600 的建立时间窗口约 375 ps，对应最大允许长度差约 62 mm，但实际设计需留裕量。

```
等长 (时序对齐):             不等长 (时序偏移):
CLK  ─────────────────       CLK  ─────────────────
D0   ─────────────────       D0   ─────────────────
D1   ─────────────────       D1   ────────────┐
D2   ─────────────────       D2   ────────────────────┐
所有信号同时到达               D1/D2 到达晚, 采样窗口缩小
```

### 5.2 DDR 等长匹配要求

| 接口 | 信号组 | 组内匹配 | 组间匹配 |
|------|--------|---------|---------|
| DDR3-1600 | DQ+DQS | +/- 5 mil | DQS 组间 +/- 25 mil |
| DDR3-1600 | ADDR/CMD | +/- 25 mil | 与 CLK +/- 25 mil |
| DDR4-2400 | DQ+DQS | +/- 3 mil | DQS 组间 +/- 10 mil |
| LPDDR4X | DQ+DQS | +/- 2 mil | 组间 +/- 5 mil |

注意：1 mil = 0.0254 mm，DDR4 的 +/- 3 mil 约 +/- 0.076 mm，要求非常精细的蛇形走线。

### 5.3 蛇形走线调长

短的线需要用蛇形走线 (Serpentine Routing) "拉长"：

```
  ──────┐     ┌──────
        │     │
        ├─────┤
        │     │
        ├─────┤
        │     │
  ──────┘     └──────

  振幅 A: 3~5 倍线宽    间距 G: >= 3 倍线宽
```

注意事项：(1) 间距 G < 3W 时蛇形段间串扰严重恶化 (2) 振幅 A > 5W 时拐弯阻抗不连续增大 (3) 蛇形走线放中段不放接收端 (4) 调长均匀分布而非集中一处 (5) 同组蛇形走线放相同区域保持串扰一致。

蛇形间距对阻抗的影响：

| 蛇形间距 G | 阻抗偏移 | 影响 |
|-----------|---------|------|
| G = 1W | -15% ~ -20% | 严重反射 |
| G = 2W | -8% ~ -12% | 明显反射 |
| G = 3W | -3% ~ -5% | 轻微 |
| G >= 5W | < 1% | 可忽略 |

## 6. 过孔换层与残桩效应

### 6.1 过孔寄生参数与残桩效应

每个过孔有寄生电感 (0.5-1.0 nH) 和寄生电容 (0.3-0.8 pF)。当过孔从顶层连接到某个内层但继续延伸到底层时，未使用的部分就是"残桩" (stub)，它像天线一样在谐振频率处产生强烈反射：

```
残桩谐振频率: f_stub = c / (4 * L_stub * sqrt(Er))

过孔截面:
  顶层走线 ──┬── 焊盘
             │
  内层1      │ ← 残桩 (无连接)
             │
  内层2 ─────┤ ← 目标层
             │
  内层3      │ ← 残桩
             │
  底层 ──────┴──
```

解决方法：

| 方法 | 原理 | 成本 | 适用场景 |
|------|------|------|---------|
| 背钻 | 钻掉残桩部分 | 中等 (加价 10-20%) | 5 Gbps 以上 |
| 盲孔 | 只钻到目标层 | 高 | HDI 板 |
| 不换层 | 避免过孔 | 零 | 优先考虑 |

### 6.2 差分对过孔与参考平面切换

差分对换层时两个过孔必须对称放置，过孔间距 <= 50 mil，每对信号过孔配 1-2 个地过孔。

走线换层导致参考平面改变时，回流路径被打断。必须在信号过孔旁 30 mil 内加地过孔，连接新旧参考平面。严禁高速信号跨越被分割的地平面。

```
正确做法: 信号过孔旁加回流地过孔

  信号过孔 S    地过孔 G
    G   S   S   G
    |   |   |   |
    GND1 ─── GND2 (地过孔桥接)
```

## 7. 眼图基础

### 7.1 什么是眼图

眼图是把大量比特周期的信号波形叠加在一起显示的图形，直观反映信号质量：

```
    1 ───┐     ┌───
         │  眼  │
         │  睛  │
    0 ───┘     └───
         │←Tb→│
```

关键指标：眼高 (1/0 电平开度)、眼宽 (稳定采样时间窗口)、抖动 (边沿时间不确定性)。

### 7.2 从眼图诊断问题

| 眼图特征 | 可能原因 | 解决方向 |
|---------|---------|---------|
| 眼高不足 (上下闭合) | 阻抗不匹配 / 端接错误 | 检查阻抗, 调整端接 |
| 眼宽不足 (左右闭合) | 抖动过大 / 串扰 | 增加间距, 检查等长 |
| 边沿台阶 | 残桩反射 / 阻抗突变 | 背钻, 优化过孔 |
| 上升/下降不对称 | 差分对不对称 | 检查差分对称性 |

## 8. IoT 高速接口实战

### 8.1 常用高速接口

| 接口 | 速率 | 类型 | 典型应用 | PCB 关注点 |
|------|------|------|---------|-----------|
| USB 2.0 | 480 Mbps | 差分 | 数据传输/固件升级 | 90 Ohm 差分阻抗, ESD 保护 |
| SDIO 3.0 | 208 MHz | 单端 | Wi-Fi 模块 | 串联端接, 串扰控制 |
| QSPI | 133 MHz | 单端 | 外部 Flash | 等长 +/- 100 mil |
| MIPI CSI-2 | 2.5 Gbps/lane | 差分 | 摄像头 | 100 Ohm 差分, 走线最短 |
| RGMII | 125 MHz | 单端 | 千兆以太网 | 等长 +/- 50 mil |

### 8.2 ESP32-S3 USB 2.0 布线实例

ESP32-S3 内置 USB 2.0 OTG，是 IoT 设备中最常见的高速接口。布线要点：

1. D+/D- 阻抗 90 Ohm +/- 10%，线宽/间距参考板厂叠层表 (典型 5/5 mil)
2. 走线长度匹配 +/- 150 mil，总长度 < 150 mm
3. ESD 保护器件靠近连接器放置，走线先过 ESD 再走线
4. 过孔尽量不超过 2 对，远离高速时钟和开关电源

走线路径：ESP32-S3 -> D+/D- 直连 -> ESD (靠近连接器) -> 阻抗控制走线 -> USB 连接器

### 8.3 MIPI 摄像头接口布线

MIPI D-PHY 布线要求最严格：差分阻抗 100 Ohm +/- 10%，Lane 内等长 +/- 5 mil，Lane 间等长 +/- 50 mil，走线 < 100 mm，下方参考面必须完整 (禁止跨分割)，远离 DC-DC 和晶振。典型 4-lane MIPI CSI-2 需要 5 对差分线 (1 对 CLK + 4 对 DATA)。

## 9. EDA 工具实操要点

### 9.1 KiCad 8+ 高速走线

| 功能 | 操作 | 适用场景 |
|------|------|---------|
| 差分对走线 | Route -> Differential Pair | USB/HDMI/MIPI |
| 差分对定义 | Board Setup -> Differential Pairs | 预定义 D+/D- |
| 长度调优 | Route -> Tune Length | 等长调优 |
| 差分对长度调优 | Route -> Tune Skew | 差分对内等长 |
| 阻抗计算 | Board Setup -> Impedance Calculator | 叠层阻抗 |

调长步骤：先完成所有布线 -> 找组内最长线 -> 选中需调长的线 -> 设置目标长度 -> KiCad 自动添加蛇形走线 -> 调整参数 (间距 >= 3W，振幅 3-5W)。

### 9.2 Altium Designer 高速走线

| 功能 | 操作 | 说明 |
|------|------|------|
| 差分对走线 | Place -> Differential Pair Routing | 实时差分走线 |
| 等长调优 | Tools -> Interactive Length Tuning | 蛇形/锯齿/圆弧 |
| 阻抗约束 | Design -> Rules -> Impedance | 自动计算线宽 |
| 组内等长 | Design -> Rules -> Matched Lengths | 约束组内等长 |
| SI 仿真 | Simulate -> Signal Integrity | 内置 SI 仿真器 |

### 9.3 通用布线检查清单

- 差分对阻抗是否匹配 (参考板厂阻抗表)
- 差分对内长度差是否满足要求
- 等长匹配组内所有线是否满足容差
- 蛇形走线间距是否 >= 3W
- 每个信号过孔旁是否有回流地过孔
- 高速信号是否跨越了平面分割
- ESD 保护器件是否靠近连接器
- 参考平面是否完整 (无割裂)
- 高速信号是否远离开关电源和晶振

## 总结

高速信号走线的三大支柱——差分对、等长、阻抗匹配——可以用一句话概括：让信号以正确的形态、正确的时间、正确的幅度到达接收端。

| 问题 | 本质 | 核心解法 |
|------|------|---------|
| 信号变形 | 阻抗不匹配 -> 反射 | 端接 + 阻抗控制 |
| 时序偏移 | 走线不等长 -> 到达时间不同 | 等长匹配 + 蛇形调优 |
| 共模噪声 | 差分对不对称 -> 差模转共模 | 差分对称布线 |
| 串扰 | 走线间距太小 -> 互感互容 | 3W 间距 + 正交走线 |
| 过孔反射 | 寄生参数 + 残桩 | 背钻 + 盲孔 + 回流地过孔 |

对 IoT 开发者而言，最常遇到的高速接口是 USB 2.0、QSPI 和 MIPI。建议从 USB 2.0 的差分对布线练手——容差最宽 (等长 +/- 150 mil)，阻抗要求也不苛刻 (90 Ohm +/- 10%)。熟练之后再挑战 DDR 和 MIPI 的严格约束。

最后记住：高速走线的规则不是"遵守了更好"，而是"违反了一定出问题"。临界长度以下的走线可以放松，但一旦超过，每一条规则都必须严格遵守。
