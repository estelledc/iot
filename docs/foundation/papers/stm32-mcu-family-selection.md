# STM32 MCU家族选型指南与外设对比

> **难度**：🟢 入门 | **领域**：MCU 选型 | **关键词**：STM32, 产品线, 外设, 长供 | **阅读时间**：约 16 分钟

## 日常类比

车系从代步到性能车共享品牌零件体系。意法半导体 STM32 用统一生态覆盖入门到高性能、低功耗与无线，选型先看“路况”（功耗、算力、外设、供货）再选车型[1][2]。

## 摘要

解读命名与系列定位（F/G/L/U/H/W 等演进以官网现行产品树为准），给出物联网（IoT）决策表。价格与供货随市场波动，本文不绑定报价[2]。

## 1. 如何读型号

典型字段编码系列、核、外设套件、Flash/RAM、封装与温度等级——以数据手册命名规则章节为准[1]。

| 需求倾向 | 系列方向（示例性） |
|----------|-------------------|
| 低成本控制 | 入门 F0/C0/G0 类 |
| 主流外设平衡 | F4/G4 等 |
| 低功耗电池 | L4/U5 等 |
| 高性能/AI | H7 等 |
| 无线一体 | WB/WL 等 |

## 2. 选型维度

| 维度 | 问题 |
|------|------|
| CPU/DSP | 是否需要浮点/DSP 指令 |
| 存储 | Flash/RAM 是否够协议栈+OTA |
| 模拟 | ADC 位数、运放、比较器 |
| 连接 | USB/CAN/Ethernet/外部总线 |
| 功耗 | 深睡电流与唤醒 |
| 封装 | 引脚与 PCB 工艺 |
| 寿命 | 工业长供 vs 消费 |

## 3. IoT 决策启发式

| 场景 | 建议 |
|------|------|
| 传感器节点 | 低功耗系列 + 足够 SRAM |
| 网关 | 更高主频/以太网或外挂模组 |
| 马达控制 | 先进定时器与模拟丰富系列 |
| 蓝牙/Sub-GHz | 无线系列或主机+模组 |

先做内存与功耗预算，再对比评估板生态与 Cube 包成熟度[3]。

## 4. 局限、挑战与可改进方向

### 1. 系列过多选择瘫痪

**局限**：相近型号难分。
**改进**：用约束表淘汰；默认团队熟悉系列[2]。

### 2. 外设“有”但引脚冲突

**局限**：复用导致功能不可同用。
**改进**：早期引脚分配与复用表检查[1]。

### 3. 供应链变更

**局限**：停产/缺货冲击。
**改进**：双型号引脚兼容设计；关注 PCN[4]。

### 4. 无线认证绑定

**局限**：自研射频认证重。
**改进**：模组方案或已认证 SIP[5]。

## 总结

STM32 选型是约束满足问题：算力、内存、模拟、功耗、封装与供货。用预算驱动，用评估板验证，为引脚与第二来源留后路。
