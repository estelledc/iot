# STM32 MCU家族选型指南与外设对比

> **难度**: 🟢 初级 | **领域**: MCU选型 | **阅读时间**: 约 20 分钟

## 引言

选MCU就像选车 -- 菜鸟骑共享单车 (F0) 就够, 老司机跑长途要SUV (F4), 极客玩混动省油 (L4), 物联网人装无线电收发 (WB). 选错型号, 轻轻则外设不够用要外挂芯片, 重则功耗爆表电池一周一换. 本文把STM32整个家族铺开, 从命名规则到外设对比到选型决策, 帮你一次选对.

STM32是STMicroelectronics (意法半导体) 基于ARM Cortex-M内核的32位MCU产品线, 目前全球MCU市场份额前三. 其产品线覆盖从不到1美元的入门级到数十美元的双核高性能, 是IoT项目最主流的平台之一.

## 1. STM32命名规则解读

看懂型号名, 就能直接判断内核、系列、引脚数和封装.

### 1.1 完整命名结构

```
STM32 F 4 1 1 R C T 6
  |    | | | | | | | |
  |    | | | | | | | +-- 工作温度: 6=-40~85度C, 7=-40~105度C
  |    | | | | | | +---- 封装: T=LQFP, H=BGA, U=QFN/DFN, I=BGA
  |    | | | | | +------ 闪存容量: C=256KB, E=512KB, G=1MB, I=2MB
  |    | | | | +-------- 引脚数: T=36, C=48, R=64, V=100, Z=144, I=176
  |    | | | +---------- 子系列特征: 1=基础, 5=DSM, 7=高级定时器, 9=双核
  |    | | +------------ 内核/性能: 0=入门, 1=基础, 2=高级, 3=带DSP/FPU
  |    | +-------------- 产品线: F=主流, L=低功耗, G=入门+模拟, H=高性能
  |    +---------------- 内核: 32=ARM Cortex 32位
  +--------------------- 厂商: ST
```

### 1.2 产品线首字母含义

| 首字母 | 系列 | 定位 | 典型内核 |
|--------|------|------|----------|
| F | 主流 (Mainstream) | 通用控制, 性价比之王 | Cortex-M0/M3/M4/M7 |
| L | 低功耗 (Low Power) | 电池供电, 超低待机电流 | Cortex-M0+/M4/M33 |
| G | 入门模拟 (Value+Analog) | 精密模拟, 替代F1/F3 | Cortex-M4/M33 |
| H | 高性能 (High Performance) | 双核, 大内存, 图形 | Cortex-M7, M7+M4 |
| U | 超低功耗 (Ultra Low Power) | L系列继任, 安全特性 | Cortex-M33 |
| W | 无线 (Wireless) | 蓝牙5/BLE, 802.15.4 | Cortex-M4/M0+ |
| MP | 微处理器 (Microprocessor) | 运Linux, 高端应用 | Cortex-A7+M4 |

## 2. 各系列定位与内核映射

### 2.1 主流F系列

F系列是STM32最早也最广的产品线, 覆盖Cortex-M0到M7全内核.

| 系列 | 内核 | 主频 | Flash上限 | RAM上限 | 定位 |
|------|------|------|-----------|---------|------|
| F0 | Cortex-M0 | 48 MHz | 256 KB | 32 KB | 入门, 替代8位MCU |
| F1 | Cortex-M3 | 72 MHz | 1 MB | 96 KB | 经典通用, 淘宝开发板之王 |
| F2 | Cortex-M3 | 120 MHz | 1 MB | 128 KB | F1的高频版, 较少用 |
| F3 | Cortex-M4 | 72 MHz | 512 KB | 80 KB | 带DSP/FPU, 模拟增强 |
| F4 | Cortex-M4 | 168/180 MHz | 2 MB | 256 KB | 高性能通用, 图形/DSP |
| F7 | Cortex-M7 | 216 MHz | 2 MB | 512 KB | F4升级, 双精度FPU |

### 2.2 低功耗L系列

| 系列 | 内核 | 主频 | Flash上限 | RAM上限 | 最低待机电流 |
|------|------|------|-----------|---------|-------------|
| L0 | Cortex-M0+ | 32 MHz | 192 KB | 20 KB | 0.27 uA |
| L1 | Cortex-M3 | 32 MHz | 512 KB | 80 KB | 0.35 uA |
| L4 | Cortex-M4 | 80 MHz | 1 MB | 128 KB | 0.33 uA (Stop2) |
| L4+ | Cortex-M4 | 120 MHz | 1 MB | 640 KB | 0.28 uA |
| L5 | Cortex-M33 | 110 MHz | 512 KB | 256 KB | 0.33 uA (带TrustZone) |

### 2.3 新世代G/H/U/W系列

| 系列 | 内核 | 主频 | 闪存上限 | 核心卖点 |
|------|------|------|----------|----------|
| G0 | Cortex-M0+ | 64 MHz | 512 KB | F0/G0替代F1入门, 性价比极高 |
| G4 | Cortex-M4 | 170 MHz | 512 KB | 精密模拟: 4x12bit ADC, 运放, 比较器 |
| H7 | Cortex-M7/M4双核 | 480/240 MHz | 2 MB | 性能天花板, 可跑GUI |
| U5 | Cortex-M33 | 160 MHz | 2 MB | L系列继任, 安全+低功耗兼备 |
| WB | Cortex-M4+M0+ | 64/32 MHz | 1 MB | BLE 5.0 + 802.15.4 无线 |
| WL | Cortex-M4/M0+ | 48 MHz | 256 KB | Sub-GHz LoRa/Sigfox |

## 3. 外设对比

### 3.1 ADC对比

ADC是IoT传感器接口的核心, 分辨率和采样率直接影响测量精度.

| 系列 | ADC位数 | 最多ADC数 | 最高采样率 | 特殊功能 |
|------|---------|----------|-----------|----------|
| F0 | 12 bit | 1 | 1 Msps | 基础 |
| F1 | 12 bit | 3 | 1 Msps | 双ADC交替/同步 |
| F3 | 12 bit | 4 | 5 Msps | 快速采样, 可编程增益 |
| F4 | 12 bit | 3 | 2.4 Msps | DMA双缓冲 |
| G4 | 12 bit | 5 | 4 Msps | 分辨率增强16bit, 内部运放/比较器 |
| H7 | 16/12 bit | 3 | 3.6 Msps | 硬件过采样, 16bit真分辨率 |
| L4 | 12 bit | 3 | 5 Msps | 低功耗模式自动关断 |

### 3.2 定时器对比

定时器数量决定你同时能驱动几路PWM、做几路编码器计数.

| 系列 | 高级定时器 | 通用定时器 | 基本定时器 | 低功耗定时器 | 总计 |
|------|-----------|-----------|-----------|-------------|------|
| F1 | 1 | 8 | 2 | 0 | 11 |
| F4 | 2 | 10 | 2 | 0 | 14 |
| G4 | 2 | 9 | 2 | 1 | 14 (含HRTIM高分辨率) |
| H7 | 2 | 12 | 2 | 1 | 17 |
| L4 | 1 | 7 | 2 | 1 | 11 |

### 3.3 通信接口对比

IoT项目常用SPI/I2C/UART连传感器, CAN连总线, USB连PC.

| 系列 | UART | SPI | I2C | CAN | USB OTG | 以太网 | SDIO |
|------|------|-----|-----|-----|---------|--------|------|
| F103 | 3-5 | 2-3 | 2 | 1 | 无 | 无 | 无 |
| F407 | 4-6 | 3 | 3 | 2 | 1 FS | 1 | 1 |
| G474 | 3 | 3 | 2 | 1 | 无 | 无 | 无 |
| H743 | 6-8 | 4-6 | 4 | 2 | 1 HS/1 FS | 1 | 2 |
| L476 | 3-5 | 3 | 3 | 1 | 无 | 无 | 1 |
| WB55 | 2 | 2 | 2 | 1 | 1 FS | 无 | 无 |

### 3.4 特殊外设速查

| 外设 | 典型系列 | 用途 |
|------|---------|------|
| DAC | F3/F4/H7/G4 | 模拟输出, 波形生成 |
| 运放 (OPAMP) | G4/H7 | 信号调理, 省外部运放 |
| 比较器 (COMP) | G4/H7/L5/U5 | 过流保护, 零交叉检测 |
| 加密引擎 (CRYP) | F4/F7/H7/U5/WB | AES/DES硬件加速 |
| 随机数 (RNG) | F4/F7/H7/L4+/U5/WB | 安全随机数生成 |
| LCD控制器 | F429/F769/H7 | 直接驱动TFT, 无需外部IC |
| HRTIM | G4 | 184ps分辨率, 数字电源/电机 |
| SAI | F4/F7/H7 | I2S增强, 多路音频 |
| DFSDM | F4/F7/H7 | 数字滤波, Sigma-Delta调制 |

## 4. 封装与引脚选择

### 4.1 常见封装类型

| 封装代码 | 封装形式 | 引脚间距 | 适合场景 |
|---------|---------|---------|---------|
| T | LQFP | 0.5mm | 最常见, 易手工焊接 |
| U | QFN/DFN | 0.5mm | 小体积, 需热风焊 |
| H | BGA | 0.5/0.65mm | 高引脚密度, 需回流焊 |
| I | TFBGA | 0.5mm | 超高密度, 需回流焊 |
| K | LQFP | 0.8mm | 宽间距, 更易焊接 |

### 4.2 引脚数与外设可用性

同一系列不同引脚数, 外设数量不同. 以F407为例:

| 封装引脚 | GPIO数 | 可用ADC通道 | 可用定时器 | 通信接口 |
|---------|--------|------------|-----------|---------|
| V (100) | 82 | 16 | 14 | 全部可用 |
| Z (144) | 114 | 24 | 14 | 全部可用 |
| I (176) | 140 | 24 | 14 | 全部可用 |

经验: IoT项目100引脚 (V封装) 通常够用, 除非需要并行LCD或大量GPIO.

## 5. 价格梯队与供货

### 5.1 单价参考 (1K片, USD)

| 梯队 | 系列 | 典型单价 | 说明 |
|------|------|---------|------|
| 超低价 | F0/G0 | 0.5-1.5 | 量产IoT节点首选 |
| 入门 | F1/L0 | 1.5-3.0 | 经典, 生态最丰富 |
| 中端 | F4/L4/G4 | 3.0-8.0 | 性能/功耗/模拟平衡 |
| 高端 | F7/H7 | 8.0-20+ | 图形/双核/大内存 |
| 无线 | WB/WL | 3.0-6.0 | 内置射频, 省BOM |

### 5.2 供货稳定性建议

- F1/F4: 量最大, 现货最稳, 但ST近年推G系列替代
- G0/G4: ST重点推的新系列, 长期供货有保障
- L4/U5: 低功耗方向, ST战略投入, 供货好
- H7: 高端系列, 非消费级项目放心用
- F0/F2/F3: 逐步退市 (NRND), 新项目不建议选

## 6. 开发生态系统

### 6.1 ST官方工具链

| 工具 | 用途 | 亮点 |
|------|------|------|
| STM32CubeMX | 图形化配置/代码生成 | 引脚冲突检查, 功耗估算, 一键生成初始化代码 |
| STM32CubeIDE | 集成开发环境 | 基于Eclipse, 内置CubeMX, 免费 |
| STM32CubeProgrammer | 烧录工具 | 支持ST-Link/SWD/JTAG/USB DFU |
| STM32CubeMonitor | 运行时监控 | 实时变量监控, 无需串口打印 |
| STM32CubeCLT | 命令行工具 | CI/CD集成, 自动化构建 |

### 6.2 软件抽象层

| 层级 | 说明 | 何时用 |
|------|------|--------|
| HAL (Hardware Abstraction Layer) | 高级抽象, 可移植 | 初学者, 快速原型, 通用项目 |
| LL (Low Layer) | 寄存器级, 几乎零开销 | 对性能/代码体积敏感, 熟练开发者 |
| 寄存器直接操作 | 直接读写寄存器 | 极致优化, 调试硬件问题 |

代码体积对比 (相同功能, F4点亮LED):

| 方式 | Flash占用 | 可读性 | 移植性 |
|------|----------|--------|--------|
| HAL | ~8 KB | 高 | 高 |
| LL | ~1 KB | 中 | 中 |
| 寄存器 | ~0.2 KB | 低 | 无 |

### 6.3 第三方生态

- **IDE**: Keil MDK (收费, 行业标准), IAR (收费, 优化强), PlatformIO (开源, VSCode插件)
- **RTOS**: FreeRTOS (CubeMX内置), Zephyr (IoT专用, 支持WB), ThreadX (Azure RTOS, ST认证)
- **调试器**: ST-Link V2/V3 (官方), J-Link (Segger, 全内核), DAPLink (开源)
- **开发板**: Nucleo (官方, 板载ST-Link), Discovery (官方, 带传感器), 各类淘宝最小系统板

## 7. IoT场景选型决策

### 7.1 决策树

```
IoT项目选型
|
+-- 需要无线通信?
|   +-- 是 --> BLE 5.0? --> WB55 (双核M4+M0+, 协议栈跑M0+)
|   |        +-- Sub-GHz? --> WL55 (LoRaWAN, 长距离低功耗)
|   +-- 否 --> 继续判断
|
+-- 电池供电?
|   +-- 是 --> 需要高性能? --> L4+ (120MHz, 低功耗)
|   |        +-- 否 --> L0/L4 (低功耗, 传感器采集)
|   |        +-- 需要安全? --> U5 (TrustZone, 安全启动)
|   +-- 否 --> 继续判断
|
+-- 需要精密模拟? (电机/电源/传感器调理)
|   +-- 是 --> G4 (运放+比较器+HRTIM, 一颗顶多颗)
|   +-- 否 --> 继续判断
|
+-- 需要图形/GUI?
|   +-- 是 --> H7 (480MHz, LCD控制器, 大RAM)
|   +-- 否 --> 继续判断
|
+-- 性能需求?
    +-- 入门 (< 50MHz) --> G0 (新一代F0替代)
    +-- 中端 (100-200MHz) --> F4 (生态最成熟)
    +-- 高端 (> 200MHz) --> H7 单核/双核
```

### 7.2 典型IoT项目推荐

| 项目类型 | 推荐型号 | 理由 |
|---------|---------|------|
| 温湿度传感器节点 | STM32L052 | 超低功耗, I2C连SHT30, 电池可用1年+ |
| 智能门锁 | STM32WB55 | BLE手机通信 + 低功耗待机 |
| 电机驱动 (BLDC) | STM32G474 | 运放+比较器+HRTIM, 省外围电路 |
| 网关/边缘计算 | STM32H743 | 大RAM, 以太网, 可跑轻量ML推理 |
| LoRa远程抄表 | STM32WL55 | Sub-GHz + LoRaWAN协议栈片上 |
| 可穿戴设备 | STM32U575 | 超低功耗 + TrustZone安全 + 小封装 |
| 工业数据采集 | STM32F407 | 生态成熟, 3个ADC, 大量参考设计 |
| 简单开关/LED控制 | STM32G070 | 不到1美元, 淘宝开发板多 |

### 7.3 选型检查清单

选型号前, 按以下清单逐项确认:

1. **GPIO数量**: 传感器/执行器/通信各需几个引脚, 预留20%余量
2. **Flash需求**: 代码体积 + RTOS + 协议栈 (BLE约150KB) + OTA备份区
3. **RAM需求**: 全局变量 + 协议栈栈 + DMA缓冲 + 留30%余量
4. **ADC通道数**: 模拟传感器数量, 是否需要同步采样 (双ADC)
5. **通信接口**: SPI (高速传感器) / I2C (低速传感器) / UART (调试/AT指令) / CAN (工业总线)
6. **功耗预算**: 工作电流 x 工作占比 + 待机电流 x 待机占比, 对比电池容量
7. **封装与焊接**: QFN需热风, BGA需回流, 打样阶段优先LQFP
8. **供货与价格**: 查Mouser/DigiKey库存, 淘宝现货, 避免NRND系列
9. **开发板可用性**: Nucleo/Discovery板是否存在, 加速前期验证
10. **软件支持**: CubeMX是否有配置, HAL驱动是否完善

## 8. 常见选型误区

### 8.1 误区与纠正

| 误区 | 正确理解 |
|------|---------|
| "F103是万能的" | F103无FPU, 做浮点运算极慢; 无USB OTG; Flash架构旧, 速度受限 |
| "主频越高越好" | 高主频 = 高功耗; 很多传感器采集场景48MHz绰绰有余 |
| "外设多就选高端" | H7的外设多但价格10x; G4的模拟外设比H7更实用 (对电机/电源场景) |
| "L系列性能太弱" | L4可达80MHz + FPU, 跑FreeRTOS+BLE协议栈没问题 |
| "WB只能做BLE" | WB55同时支持BLE + 802.15.4, 可做Zigbee/Thread/自定义协议 |
| "先选型号再评估" | 应先列需求表 (引脚/功耗/外设/价格), 再反向匹配型号 |

### 8.2 F1 vs G0 vs G4 -- 三代入门之争

这是最常见的纠结点, 直接对比:

| 维度 | F103 | G070 | G474 |
|------|------|------|------|
| 内核 | Cortex-M3 72MHz | Cortex-M0+ 64MHz | Cortex-M4 170MHz |
| FPU | 无 | 无 | 有 |
| ADC | 12bit 1Msps x1 | 12bit 1.8Msps x1 | 12bit 4Msps x4 |
| 运放/比较器 | 无 | 无 | 4运放 + 3比较器 |
| 价格 (1K) | ~$1.8 | ~$0.8 | ~$3.5 |
| 生态成熟度 | 极高 | 中 | 中 |
| 推荐场景 | 旧项目维护, 学习 | 新项目低成本入门 | 电机/电源/精密模拟 |

新项目建议: 无脑选G0入门, 有模拟需求选G4, F1仅用于维护存量.

## 9. 从选型到验证的快速路径

选定型号后, 不要直接画PCB, 按以下路径降低返工风险:

1. **买Nucleo板**: ST官方, 板载ST-Link, 免焊接
2. **CubeMX配置外设**: 先在CubeMX里把所有外设配好, 确认无引脚冲突
3. **跑BSP示例**: ST每个系列都有BSP (Board Support Package) 示例, 先跑通
4. **功耗测量**: 用万用表/功耗分析仪测各模式电流, 对比数据手册
5. **原型验证**: 面包板/洞洞板验证关键外设 (传感器/通信/存储)
6. **画PCB**: 确认引脚/封装/外设全部OK后, 才开始PCB设计

## 总结

STM32家族庞大但有序, 理解命名规则后就能从型号读出70%的信息. 选型的核心逻辑:

- **先定需求, 再找芯片** -- 引脚/功耗/外设/价格四维约束, 反向匹配
- **跟着ST的战略方向走** -- G/U/W是未来, F0/F2/F3正在退场
- **生态比规格更重要** -- F4之所以经典, 不是因为规格最强, 而是参考设计/社区/第三方库最丰富
- **IoT选型的特殊性** -- 低功耗 (L/U) 和无线 (W) 是IoT与通用控制的最大分水岭
- **验证先于量产** -- Nucleo板 + CubeMX + BSP示例, 三件套快速验证选型可行性

最后记住: 没有最好的MCU, 只有最适合当前项目的MCU. 花半小时做需求表和选型决策, 能省后面几十小时的返工.
