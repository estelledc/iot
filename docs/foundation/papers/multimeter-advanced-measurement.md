# 万用表高级测量：真有效值/温度/频率
> **难度**：🟢 初级 | **领域**：基础测量工具 | **阅读时间**：约 18 分钟

## 引言

万用表就像电工的"听诊器"：医生用听诊器初步判断心肺状态，工程师用万用表快速了解电路的电压、电流、电阻。大多数初学者只用万用表量电压和通断，却忽略了真有效值、电容、频率、温度等高级功能。对于 IoT 开发者来说，一台功能齐全的万用表可以解决大量调试问题，无需动用示波器或专业设备。本文系统介绍万用表的高级测量功能及其在 IoT 调试中的实际应用。

## 1 超越基础 V/I/R

### 1.1 基础功能回顾

万用表的基础测量功能：

- 直流电压 (DCV)：测量电池、电源输出
- 交流电压 (ACV)：测量市电、变压器输出
- 直流电流 (DCA)：测量电路功耗
- 电阻 (R)：测量元件阻值、检查连接
- 通断测试：快速检查线路连通性

### 1.2 常被忽略的高级功能

| 功能 | 初学者使用率 | 实际价值 |
|------|-------------|----------|
| 真有效值 (True RMS) | 低 | 非正弦波必须 |
| 电容测量 | 低 | 验证电容值、排查故障 |
| 频率/占空比 | 极低 | 验证 PWM、检查时钟 |
| 温度测量 | 极低 | 验证传感器、检查散热 |
| 二极管测试 | 中 | LED/二极管识别 |
| Min/Max/Hold | 低 | 捕获瞬态值 |
| 相对值 (REL) | 极低 | 消除引线电阻 |

本文逐一介绍这些高级功能，配合 IoT 实例说明使用场景。

## 2 真有效值 (True RMS) 测量

### 2.1 平均响应 vs 真有效值

万用表测量交流信号有两种方式：

- 平均响应 (Average Responding)：测量平均值后乘以波形系数，换算为有效值
  - 仅对纯正弦波准确
  - 测量非正弦波时误差可达 10-40%
- 真有效值 (True RMS)：直接计算热等效值
  - 对任何波形都准确
  - 公式：Vrms = sqrt(1/T * integral(v^2 dt))

### 2.2 非正弦波为何常见

IoT 开发中非正弦波随处可见：

- PWM 信号：方波，占空比变化时有效值随之变化
- 开关电源电流：脉冲状，远非正弦
- LED 驱动电流：恒流斩波，波形严重畸变
- 电机驱动电流：含大量谐波

### 2.3 测量误差对比

| 波形类型 | 平均响应误差 | True RMS 读数 |
|----------|-------------|---------------|
| 正弦波 | 0% | 准确 |
| 方波 (50%占空比) | +11% | 准确 |
| PWM (25%占空比) | 约 +50% | 准确 |
| 三角波 | -4% | 准确 |
| 开关电源输入电流 | 5-40% | 准确 |

### 2.4 选择建议

- 如果只测正弦波（市电），普通平均响应表即可
- 如果涉及 PWM、开关电源、电机驱动，必须用 True RMS 表
- IoT 开发者强烈建议选择 True RMS 万用表
- True RMS 表通常在表盘上标注 "True RMS" 字样

## 3 电容测量

### 3.1 测量原理

万用表测电容的常见方法：

- 充放电计时法：对电容充放电，测量 RC 时间常数
  - 精度一般 (5-10%)
  - 适合 nF 到数百 uF 范围
- 交流阻抗法：施加交流信号测量阻抗
  - 精度较高
  - 部分高端台式表使用

### 3.2 实际应用场景

- 验证新购电容值：批次抽检，确认标称值
- 排查电解电容失效：容量显著下降表明电容干涸
- 区分电容规格：区分 100pF 和 1nF 贴片电容
- 检查 MLCC 退化：直流偏置下实际容量可能大幅降低

### 3.3 测量注意事项

- 测量前务必对电容放电，否则可能损坏万用表
- 从电路板上取下电容后测量（在路测量受并联元件影响）
- 大容量电解电容需要较长时间稳定读数
- 注意极性：电解电容有正负极，反向测量值不准

## 4 频率与占空比测量

### 4.1 内置计数器功能

许多万用表内置频率计数器：

- 测量范围：通常 1Hz 到数 MHz
- 精度：取决于时基，通常 0.01-0.1%
- 输入要求：信号幅度需在万用表规格内
- 灵敏度：低频时要求信号幅度较高

### 4.2 验证 PWM 输出

IoT 开发中 PWM 信号非常常见：

- 测量 PWM 频率：确认定时器配置正确
- 测量占空比：验证 LED 亮度控制、电机速度控制
- 调试步骤：先测频率对不对，再测占空比对不对
- 优势：比示波器更快，无需设置触发

### 4.3 检查振荡器频率

验证时钟是否正常工作：

- 测量晶振输出频率：确认 MCU 是否在正确频率运行
- 检查 RTC 时钟：32.768kHz 是否准确
- 注意：万用表输入电容可能影响晶振电路，测完后确认是否恢复正常

### 4.4 测量注意事项

- 信号幅度需满足万用表最低灵敏度要求
- 频率测量模式下输入阻抗可能不同
- 高频信号测量时注意探头电容的影响
- 占空比测量需要信号频率在规格范围内

## 5 温度测量

### 5.1 热电偶输入

许多万用表支持 K 型热电偶温度测量：

- K 型热电偶：最常见，测量范围 -200 到 +1370 摄氏度
- 万用表内置冷端补偿 (CJC)
- 精度：通常 +/- 1-2 摄氏度
- 响应速度：取决于热电偶探头类型

### 5.2 IoT 调试应用

- 验证温度传感器读数：比对万用表测量值和传感器上报值
- 检查元件温度：MCU、LDO、功率 MOSFET 是否过热
- 评估散热设计：满载运行后测量关键元件温升
- 电池温度监测：充电时电池温度不应超过 45 摄氏度

### 5.3 测量技巧

- 热电偶探头需紧密接触被测表面
- 使用导热硅脂改善热接触
- 等待热平衡后读数（通常需 10-30 秒）
- 注意冷端补偿：热电偶连接器处的温度影响测量精度

## 6 二极管与通断测试

### 6.1 二极管测试模式

二极管测试模式提供正向压降测量：

- 万用表输出约 1mA 测试电流
- 显示二极管正向压降 (Vf)
- 硅二极管：约 0.6-0.7V
- 肖特基二极管：约 0.2-0.4V
- LED：红约 1.8V，蓝/白约 2.8-3.3V

### 6.2 实际应用

- 识别不明二极管：通过 Vf 判断类型
- 检查 LED 极性：红表笔接阳极时点亮
- 排查二极管损坏：开路（显示 OL）或短路（接近 0V）
- 验证 TVS 管：正常 Vf 约 0.6V，击穿需用更高电压测试

### 6.3 通断测试

通断测试是日常最频繁使用的功能：

- 电阻低于阈值（通常 30-50 欧）时蜂鸣器响
- 快速检查：线路是否连通、焊点是否虚焊、开关是否正常
- 注意：被测电路必须断电，否则可能误判或损坏万用表

## 7 Min/Max/Hold 功能

### 7.1 最小/最大值记录

捕获测量过程中的极值：

- Min：记录最小读数，适合检测电压跌落
- Max：记录最大读数，适合检测电压尖峰
- 工作原理：以一定速率采样并比较，记录极值
- 采样速率：通常约 1 次/秒，无法捕获 ms 级瞬态

### 7.2 IoT 调试实例

- 测量电池电压在负载瞬间的跌落：Min 功能
- 测量充电时电池最高电压：Max 功能
- 记录电源输出在 24 小时内的波动范围
- 检测 I2C 总线 SCL 信号是否有异常高电压

### 7.3 Hold 功能

保持当前读数：

- Data Hold：按下后冻结显示，方便记录
- Peak Hold：捕获瞬态峰值（部分高端表支持）
- 使用场景：探头在难以观察的位置时，先测量再查看

## 8 相对值测量 (REL)

### 8.1 功能说明

相对值测量可以归零当前读数：

- 按下 REL 按钮，当前读数作为基准值
- 后续显示 = 实际读数 - 基准值
- 常见用途：消除测试引线的电阻

### 8.2 小电阻精确测量

测量低阻值元件时的标准流程：

1. 将两根测试引线短接
2. 按 REL 归零，消除引线电阻
3. 测量目标电阻
4. 显示值已自动扣除引线电阻

### 8.3 IoT 应用实例

- 精确测量 I2C 上拉电阻（通常 4.7k欧）
- 测量电流采样电阻（m欧 级，引线电阻不可忽略）
- 比较两个相近阻值的差异
- 校准零点偏移：如测量很小的电压差

## 9 IoT 调试实战

### 9.1 测量电池负载电压

```text
场景: CR2032 纽扣电池供电的 BLE 传感器
1. 空载测量: 3.0V (标称 3V，正常)
2. 万用表串联测量工作电流:
   - 睡眠: 3uA (用 uA 档)
   - 采样: 2mA (用 mA 档)
3. 广播时测量电池端电压: 2.85V
4. 评估: 压降 0.15V，电池内阻 = 0.15V/8mA = 18.75欧
5. 结论: 电池内阻正常，可继续使用
```

### 9.2 验证稳压器输出

```text
场景: 检查 3.3V LDO 输出是否正常
1. 测量输入电压: 5.0V (在规格范围内)
2. 测量输出电压: 3.28V (标称 3.3V，偏差 0.6%)
3. 测量输出纹波 (ACV 档): 5mV (正常)
4. 加载后测输出: 3.25V (负载调整率 0.9%)
5. 结论: LDO 工作正常
```

### 9.3 检查 I2C 上拉电阻

```text
场景: I2C 通信不稳定，怀疑上拉电阻问题
1. 断电，测量 SCL 上拉: 4.7k欧 (正确)
2. 测量 SDA 上拉: 4.7k欧 (正确)
3. 测量 SCL 对地电阻: 3.2k欧 (异常偏低)
4. 原因: 某从设备 SCL 引脚对地泄漏
5. 修复: 更换故障从设备后通信恢复
```

### 9.4 测量睡眠电流

```text
场景: 验证 MCU 深睡眠模式电流
1. 万用表置 uA 档，串联在电源回路中
2. MCU 进入深睡眠模式
3. 读数: 3.5uA (规格 < 5uA，合格)
4. 逐个关闭外设，排查哪个外设漏电
5. 发现: 未关闭的 ADC 增加了 1.2uA
6. 优化后: 2.3uA
```

## 10 测量安全

### 10.1 CAT 等级

万用表的安全等级分类：

| 等级 | 适用场景 | 电压 |
|------|----------|------|
| CAT I | 电子电路、信号测量 | 低 |
| CAT II | 墙壁插座、家电 | 300V/600V |
| CAT III | 配电盘、固定安装设备 | 600V/1000V |
| CAT IV | 户外进线、电表 | 600V/1000V |

IoT 开发者使用 CAT II 即可，但选择 CAT III 更安全。

### 10.2 安全注意事项

- 绝不在带电电路上测量电阻（可能烧毁万用表）
- 测量电流前确认已串联在回路中（误并联会短路）
- 测量高压时使用带护套的探头
- 定期检查保险丝是否完好
- 测量未知信号时，从最大量程开始

### 10.3 保险丝保护

万用表电流档有保险丝保护：

- mA 档：通常 200mA/500mA 快熔保险丝
- A 档：通常 10A/20A 大电流保险丝
- 过载后保险丝熔断，需更换同规格保险丝
- 不要用普通保险丝替代，响应时间不匹配

## 11 万用表选型

### 11.1 IoT 开发所需功能

| 功能 | 必要性 | 说明 |
|------|--------|------|
| True RMS | 必须 | PWM/开关电源测量 |
| uA 电流档 | 必须 | 睡眠电流测量 |
| 电容测量 | 推荐 | 电容验证与排查 |
| 频率/占空比 | 推荐 | PWM 和时钟验证 |
| 温度 (K型) | 推荐 | 传感器校验 |
| 自动量程 | 推荐 | 减少操作失误 |
| 背光显示 | 可选 | 弱光环境 |
| 蓝牙传输 | 可选 | 数据记录 |

### 11.2 推荐型号

| 定位 | 型号 | 价格区间 | 特点 |
|------|------|----------|------|
| 高端专业 | Fluke 87V | 约 3000 元 | True RMS, 精度高, 耐用 |
| 中端实用 | UNI-T UT61E | 约 300 元 | True RMS, 性价比高 |
| 入门预算 | ANENG AN8008 | 约 80 元 | True RMS, 功能全 |
| 台式高精度 | Keysight 34461A | 约 10000 元 | 6.5 位, 数据记录 |

### 11.3 选购避坑

- 不选无 True RMS 的便宜表（IoT 测量不准）
- 不选无 uA 档的表（无法测睡眠电流）
- 不选无保险丝保护的表（安全风险）
- 不盲目追求高位数（3.5 位足够日常使用）
- 优先选知名品牌（安全性和精度有保障）

## 总结

万用表是 IoT 开发者的基础工具，掌握高级功能可以解决大量问题：

1. True RMS：非正弦波测量的必须功能，PWM/开关电源场景误差大
2. 电容测量：验证电容值、排查失效电容
3. 频率/占空比：快速验证 PWM 和时钟信号
4. 温度测量：校验传感器、检查散热
5. Min/Max/Hold：捕获极值和瞬态
6. REL 测量：消除引线电阻，精确测小阻值
7. 安全第一：遵守 CAT 等级，正确使用保险丝
