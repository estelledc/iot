# FMCW雷达液位传感器在工业IoT中的应用
> **难度**：🔴 高级 | **领域**：雷达液位测量 | **阅读时间**：约 22 分钟

## 引言

想象你站在一个很深的井边，想知道水面有多深。最原始的办法是扔一块石头下去然后计时——听到水花声的时间越长，水面越深。FMCW雷达做的事一模一样，只不过它扔出去的不是石头而是电磁波，听到回声不是用耳朵而是用天线，而且它不是简单计时，而是通过发射"变调"的电磁波（频率随时间线性变化），比较发射和接收的频率差来精确算出距离。这个频率差叫做"拍频"，它就是FMCW雷达的核心信号。

## 1 FMCW雷达液位测量原理

### 1.1 FMCW基本概念

FMCW（Frequency Modulated Continuous Wave，调频连续波）雷达：

- 发射一个频率随时间线性变化的连续波信号（称为chirp）
- 信号碰到液面后反射回来被接收
- 发射信号和接收信号之间存在时间延迟
- 时间延迟导致频率差异

与脉冲雷达不同，FMCW始终在发射信号，不存在"盲区"问题，平均功率低但精度高。

### 1.2 chirp信号

FMCW发射的chirp信号特征：

```
频率
  ^
  |          /
  |         /  斜率 S = BW / T
  |        /
  |       /
  |      /
  |     /
  |    /
  +──────────────> 时间
  |    T = chirp周期
  |
  f_start      f_stop
  |<--- BW --->|
```

关键参数：

- 起始频率 f_start
- 停止频率 f_stop
- 带宽 BW = f_stop - f_start
- chirp 周期 T
- 调频斜率 S = BW / T

### 1.3 拍频与距离的关系

发射信号与接收信号混频后产生拍频（beat frequency）：

```
fb = 2 * R * S / c = 2 * R * BW / (c * T)
```

其中：

- fb：拍频（Hz）
- R：雷达天线到液面的距离（m）
- S：调频斜率（Hz/s）
- c：光速（3 x 10^8 m/s）
- BW：chirp 带宽（Hz）
- T：chirp 周期（s）

从公式可以看出：距离越远，拍频越高。对拍频做 FFT 就能得到距离谱，谱峰对应液面位置。

### 1.4 测距分辨力

距离分辨力由带宽决定：

```
delta_R = c / (2 * BW)
```

实际例子：

- 6GHz 雷达（带宽约 500MHz）：分辨力约 0.3m
- 80GHz 雷达（带宽约 6GHz）：分辨力约 0.025m（25mm）

带宽越大，分辨力越高，这就是 80GHz 雷达在液位测量中越来越流行的原因。

## 2 FMCW信号处理

### 2.1 混频与拍频提取

FMCW接收机的核心操作是混频——将发射信号和接收信号相乘，低通滤波后得到差频信号（即拍频）。

```
[发射信号] ──┐
              ├──[混频器]──[低通滤波]──[ADC]──[FFT]
[接收信号] ──┘
```

混频的物理意义：将两个频率相近的信号相乘，产生差频和和频，低通滤波保留差频（拍频），滤除和频。

### 2.2 FFT距离谱与峰值检测

对采样后的拍频信号做 FFT，得到距离谱：横轴为频率（对应距离），纵轴为幅度（对应回波强度）。最大峰对应液面回波，其他峰为罐壁、搅拌器等杂波。

```c
// 简化的FMCW信号处理
float process_fmcw(uint16_t *adc_buf, int N) {
    // 1. 加窗减少频谱泄漏
    apply_hanning_window(adc_buf, N);

    // 2. FFT
    float complex *spectrum = fft(adc_buf, N);

    // 3. 取幅度谱
    float *magnitude = compute_magnitude(spectrum, N);

    // 4. 找峰值（液面回波）
    int peak_idx = find_peak(magnitude, N);

    // 5. 频率->距离
    float fb = peak_idx * fs / N;       // 拍频
    float R = fb * c * T / (2 * BW);   // 距离

    return R;
}
```

### 2.3 零中频处理

实际 FMCW 雷达芯片内部已集成混频和 ADC，输出就是数字化拍频信号。MCU 只需做 FFT 和峰值搜索，降低了系统复杂度。

## 3 雷达频段与天线

### 3.1 频段选择

| 频段 | 带宽 | 波长 | 波束角 | 适用场景 |
|------|------|------|--------|---------|
| 6GHz | ~500MHz | 50mm | 宽（约30度） | 固体料位，大储罐 |
| 26GHz | ~1GHz | 12mm | 中（约15度） | 通用液位 |
| 80GHz | ~6GHz | 3.75mm | 窄（约4度） | 小储罐，狭小安装空间 |

频段越高：波长越短波束越窄，带宽越大分辨力越高，但成本也越高。

### 3.2 天线类型

**喇叭天线（Horn Antenna）：**

- 传统设计，波束可控
- 金属喇叭口耐高温高压
- 适合炼油、化工等苛刻环境
- 体积较大，需要较大安装口径

**平面阵列天线（Planar Array）：**

- PCB 上的微带贴片阵列
- 体积紧凑，齐平安装
- 适合空间受限场景
- 雨雪冷凝可能影响性能

**杆式天线（Rod Antenna）：**

- 旁通管（bypass）内安装
- 适用于安装空间极小的场景
- 波束在旁通管内导引

### 3.3 波束角与罐体

波束角决定了雷达"照亮"的区域大小：

- 波束角越小，越不容易打到罐壁
- 小罐体需要窄波束（80GHz）
- 大罐体可以用宽波束（6GHz/26GHz）
- 罐内障碍物（搅拌器、加热盘管）需要避开波束

## 4 测量挑战

### 4.1 多径反射

罐内电磁波在罐壁之间来回弹跳，产生虚假回波：

- 一次反射：真实液面回波
- 二次反射：经过罐壁反弹的回波，距离偏大
- 三次及以上：更弱的虚假回波

处理方法：

- 虚假回波映射：空罐时记录虚假回波位置，测量时减去
- 阈值检测：只保留超过特定幅度的回波
- 回波追踪：跟踪上次测量位置附近的主回波

### 4.2 搅拌器和挡板的干扰

搅拌器叶片旋转时周期性进入波束，产生间歇性强回波，液面回波可能被遮挡。解决方案：回波追踪选最远有效回波，将搅拌器回波记入虚假回波表，安装时让波束避开搅拌器。

### 4.3 泡沫衰减与近区盲区

液面泡沫会吸收和散射电磁波，使真实液面回波变弱甚至消失。80GHz 波长短对泡沫更敏感，26GHz 在泡沫场景下反而更稳定。

近区盲区：紧邻天线的一小段距离内无法测量（发射信号直接耦合到接收端），典型盲区 0.1-0.5m。安装时保证最高液面不超过盲区下沿。

## 5 雷达芯片方案

### 5.1 芯片选型对比

| 芯片 | 频段 | 通道数 | 集成度 | 适用场景 |
|------|------|--------|--------|---------|
| BGT60LTR11 | 60GHz | 1Tx1Rx | 高 | 存在检测、短距 |
| Acconeer A121 | 60GHz | 1Tx1Rx | 高 | IoT 级液位 |
| TI IWR1443 | 77GHz | 3Tx4Rx | 中 | 高精度工业液位 |
| TI IWR6123 | 60GHz | 1Tx2Rx | 高 | IoT 液位 |

### 5.2 芯片特点

- BGT60LTR11：Infineon 60GHz FMCW 收发一体，内置运动检测算法
- A121：Acconeer 60GHz 脉冲相干雷达，mm级精度，mW级功耗，适合电池供电
- IWR1443/6843：TI 77/79GHz FMCW 前端，4接收通道，内置硬件FFT加速器
- IWR6123：TI 单芯片60GHz方案，集成MCU和硬件加速器，更适合IoT级液位应用

## 6 实战示例：化工储罐液位监测

### 6.1 需求与方案

- 储罐高度 15m，液体为有机溶剂，精度要求 1mm
- 环境：有蒸汽、泡沫、搅拌器
- 输出：4-20mA + HART + IoT 网关

```
[80GHz雷达传感器] ── [4-20mA + HART] ── [IoT网关] ── [云平台]
        |
   安装在罐顶法兰
   波束角 4度
   盲区 0.3m
```

### 6.2 安装与调试

- 法兰口径至少 50mm（80GHz 窄波束优势）
- 波束中心对准罐体中心
- 搅拌轴不在波束范围内
- 天线口加 PTFE 隔离罩防冷凝
- 调试：空罐扫描 -> 注液验证 -> 多点校验 -> 设报警阈值

### 6.3 IoT 数据流

```python
import time

def read_radar_level():
    """从 Modbus 寄存器读取液位"""
    level_m = modbus_read(0x0001) / 1000.0
    return level_m

def upload_to_cloud(level):
    """上传到云平台"""
    payload = {
        "tank_id": "TANK-001",
        "level_m": level,
        "timestamp": int(time.time()),
        "alarm": level > 14.0 or level < 0.5
    }
    mqtt_publish("tank/level", payload)

# 主循环：每 30 秒测量上传一次
while True:
    level = read_radar_level()
    upload_to_cloud(level)
    time.sleep(30)
```

## 7 安装注意事项

### 7.1 安装位置

- 避免正对罐壁：波束边缘打到罐壁产生虚假回波
- 远离进料口：液体流入的扰动产生回波
- 远离搅拌器：搅拌叶片周期性遮挡波束
- 避开加热盘管：金属管道是强反射体

### 7.2 法兰、接管与冷凝

- 接管长度不应超过推荐值（避免波束在接管内反射）
- 接管内壁光滑无焊缝，天线口伸出接管底部至少 10mm
- 天线面冷凝水膜影响性能，用 PTFE 隔离罩或压缩空气吹扫
- 定期检查清洁

## 8 FMCW与导波雷达对比

| 特性 | FMCW雷达 | 导波雷达（GWR） |
|------|---------|---------------|
| 接触方式 | 非接触 | 接触（探杆/缆） |
| 安装 | 罐顶法兰 | 罐顶法兰 + 探杆入罐 |
| 泡沫影响 | 可能受影响 | 探杆穿过泡沫直达液面 |
| 维护 | 免维护 | 探杆可能挂料需清理 |
| 成本 | 中-高 | 中 |
| 精度 | 极高（80GHz 1mm） | 高（3mm） |

选择原则：泡沫严重优先 GWR（探杆直接测液面），有搅拌器或内部障碍物优先 FMCW（非接触选窄波束避开障碍），卫生要求高优先 FMCW（无湿件），预算有限 GWR 通常更便宜。

## 9 IoT集成架构

### 9.1 协议栈

```
[雷达传感器] -> [Modbus RTU / HART] -> [IoT网关] -> [MQTT] -> [云平台]
```

常见工业雷达传感器输出协议：

- 4-20mA：模拟量，简单但信息少
- HART：4-20mA 基础上叠加数字信号
- Modbus RTU：RS-485，多设备总线
- Modbus TCP：以太网
- 无线HART / ISA100：工业无线

### 9.2 云端功能

- 实时液位显示和历史曲线
- 高/低液位报警推送
- 液位趋势预测（消耗速率、补液时间预估）
- 多罐统一管理仪表板
- 审计日志和合规记录

## 10 未来趋势

- 芯片化：从分立模块走向单芯片方案，降低成本和功耗
- AI 信号处理：用机器学习区分真实液面回波和虚假回波
- 多传感器融合：雷达 + 超声波 + 温度联合判断
- 5G 直连：传感器直接上 5G 网络，无需网关
- 边缘计算：雷达芯片内置 AI 推理，只上传结果

## 总结

FMCW雷达液位传感器是目前工业液位测量精度最高、可靠性最好的方案之一。它通过发射频率线性变化的电磁波、接收液面回波、提取拍频来计算距离，整个过程不需要接触液体，不受温度、压力、密度变化影响，也不怕蒸汽和粉尘。

80GHz 雷达凭借窄波束和超大带宽，在小口径安装和高精度测量上有独特优势，正在成为新建项目的首选。但在泡沫严重和有大量内部障碍的场景下，导波雷达仍然是更可靠的选择。

IoT 集成方面，雷达传感器通常通过 Modbus/HART 连接网关，再经 MQTT 上云。随着雷达芯片化和 AI 信号处理的发展，未来雷达液位传感器的成本和功耗会持续下降，应用范围将从工业扩展到民用和消费级 IoT。
