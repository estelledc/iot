# 比较器电路在阈值检测与报警中的应用

> **难度**：🟢 初级 | **领域**：模拟比较电路 | **阅读时间**：约 18 分钟

## 引言

想象你家门口装了一个门铃:来人按一下就响,不按就不响。它不管来的是谁,只关心"按了"还是"没按"这两种状态。比较器就是电子电路里的"门铃":它比较两个电压的大小,输出"高"或"低"两种状态。不需要知道具体差多少,只需要知道谁大谁小。

在IoT系统中,比较器是最实用的模拟模块之一。电池欠压报警、温度超限关断、光线阈值检测、振动唤醒——这些"超过/没超过"的判断场景,用比较器比用ADC更简单、更快、更省电。本文从比较器原理、迟滞设计、窗口比较到实际应用,帮你掌握阈值检测电路的设计方法。

## 1 比较器的功能与原理

### 1.1 比较器是什么

比较器是一种模拟电路,比较两个输入电压的大小:

```
如果 V+ > V-, 则 Vout = Voh (输出高电平)
如果 V+ < V-, 则 Vol = Vol (输出低电平)
```

本质上是一个1位ADC:只判断"谁大谁小",不做精确测量。

### 1.2 电路符号与行为

```
          Vcc
           |
    V+ --[+]      [输出级]
         [CMP]----+---- Vout
    V- --[-]      |
                   |
          GND (或开路)
```

传输特性:

```
Vout
Voh |           ___________
    |          /
    |         /
Vol |________/
    ---------------------- V+ - V-
          0
```

理想比较器在V+=V-时瞬间翻转,实际比较器有一个极窄的线性区。

## 2 比较器与运算放大器的区别

### 2.1 关键差异

| 特性 | 比较器 | 运算放大器 |
|------|--------|-----------|
| 工作模式 | 开环(无负反馈) | 闭环(负反馈) |
| 压摆率 | 快(专为开关设计) | 慢(需从饱和恢复) |
| 输出级 | 开漏/推挽,接近轨到轨 | 推挽,线性区输出 |
| 补偿 | 无内部补偿 | 有内部补偿电容 |
| 输出状态 | 只有高低两种 | 可输出中间电压 |
| 响应时间 | ns-us级 | us-ms级(从饱和恢复慢) |

### 2.2 能否用运放当比较器

可以但不推荐:
- 运放有内部补偿电容,从饱和状态恢复很慢(可能us到ms级)
- 运放输入级可能有过压保护二极管,差分输入电压受限
- 运放输出不保证轨到轨开关速度

专用比较器没有这些限制,响应速度远超运放。

### 2.3 输出级类型

**推挽(Push-Pull)输出**: 可直接驱动下一级,输出高和低都由内部管子驱动。

**开漏(Open-Drain)输出**: 只能拉低,需要外部上拉电阻才能输出高电平。

开漏的优势:
- 可做线或(Wired-OR):多个开漏输出连在一起,任一拉低则总输出为低
- 可做电平转换:上拉电阻接不同电压,输出高电平由上拉电压决定
- 适合多个报警源共用一条中断线

## 3 比较器关键规格

### 3.1 传播延迟(Propagation Delay)

输入跨越阈值到输出翻转的时间,通常ns到us级。高速比较器(如TLV3201)可达40ns,低功耗比较器(如MAX9117)约us级。

### 3.2 输入偏移电压(Input Offset Voltage)

V+与V-相等时输出实际翻转点的偏差:

| 等级 | 典型值 | 应用 |
|------|--------|------|
| 通用 | 1-10 mV | 电池报警、开关检测 |
| 精密 | 0.1-1 mV | 传感器阈值检测 |
| 超精密 | < 0.1 mV | 校准级比较 |

偏移电压等效于阈值误差,精密应用需校准。

### 3.3 输入偏置电流(Input Bias Current)

流过比较器输入端的电流:

- CMOS输入: 1-100 pA(适合高阻抗源)
- 双极型输入: 10-500 nA(速度快但电流大)

高阻抗信号源(如光敏电阻分压)必须选CMOS输入比较器。

### 3.4 供电范围与静态电流

| 类型 | 供电范围 | 静态电流 |
|------|---------|---------|
| 通用 | 2.7-36V | 0.5-2 mA |
| 低功耗 | 1.8-5.5V | 5-50 uA |
| 纳功耗 | 1.0-5.5V | 0.3-1 uA |

IoT电池供电优选纳功耗比较器,如MAX9117(500nA)。

## 4 迟滞比较器(施密特触发器)

### 4.1 为什么需要迟滞

当输入信号缓慢经过阈值时,噪声会导致输出在高低之间快速振荡:

```
无迟滞:
Vin  ---~~/~~~------  (噪声叠加在阈值附近)
Vout --^^^^^^^^^^----  (快速振荡,输出不稳定)
```

迟滞通过引入两个不同的翻转阈值(上阈值和下阈值)消除这种振荡:

```
有迟滞:
         上阈值(Vth+)
            |
Vin  ---~~/~~~------  上升时在上阈值翻转
            |
         下阈值(Vth-)

Vout ---________------  (干净的一次翻转)
```

### 4.2 正反馈电阻网络

迟滞通过从输出到正输入端的正反馈电阻实现:

```
              R1
Vref ---+---[====]---+--- V+ (比较器正输入)
        |            |
       R2          R3 (反馈电阻,从输出接回)
        |            |
       GND         Vout
```

当输出为高时,正反馈使V+略高(上阈值);当输出为低时,正反馈使V+略低(下阈值)。

### 4.3 阈值计算

设Vref为参考电压,R1、R2为分压电阻,R3为反馈电阻:

```
上阈值(输出为高,Vout=Voh):
Vth+ = Vref + (Voh - Vref) x R1//(R2) / (R1//R2 + R3)

下阈值(输出为低,Vol=0V,开漏时):
Vth- = Vref x R1//R3 / (R1//R3 + R2)
```

简化计算(当R3远大于R1、R2时):

```
Vth+ = Vref + Voh x (R1//R2) / (R1//R2 + R3)
Vth- = Vref - Vref x (R1//R2) / (R1//R2 + R3)
迟滞宽度 = Vth+ - Vth-
```

### 4.4 迟滞量选择

迟滞量应大于输入信号在阈值附近的峰峰噪声:

| 信号噪声 | 推荐迟滞 |
|---------|---------|
| < 5mV | 10-20 mV |
| 5-20mV | 30-50 mV |
| 20-50mV | 50-100 mV |
| > 50mV | 100-200 mV |

迟滞过小无法消除振荡,过大则降低检测精度。

## 5 窗口比较器

### 5.1 工作原理

窗口比较器用两个比较器检测信号是否在某个电压范围内:

```
            Vref_H ---[+]CMP1---+
                                |--- Vout (窗口内=高)
            Vref_L ---[-]CMP2---+
                  |
                Vin
```

- Vin > Vref_H: CMP1输出高,CMP2输出低,窗口外
- Vin < Vref_L: CMP1输出低,CMP2输出高,窗口外
- Vref_L < Vin < Vref_H: 两比较器输出均为低,窗口内

### 5.2 应用场景

- 温度范围检测: 10C-40C正常,超出报警
- 电压窗口: 3.0V-4.2V正常,超出异常
- 传感器信号验证: 信号应在合理范围内,超出则传感器故障

### 5.3 开漏输出实现线或

两个开漏比较器输出连在一起,上拉电阻接Vcc:

```
Vref_H ---[+]CMP1---|---[Rpullup]--- Vcc
                     |
Vref_L ---[-]CMP2---|--- Vout
                     |
                    GND(任一拉低)
```

任一比较器触发(信号超出窗口),输出拉低,产生报警。

## 6 主流比较器芯片

### 6.1 LM393(TI/NSC)

经典双路开集比较器:
- 供电: 2-36V(单/双电源)
- 偏移电压: 2mV(典型)
- 传播延迟: 1.3us
- 输出: 开集(需上拉电阻)
- 价格: ~$0.1

适用: 通用阈值检测、电池欠压报警、过温关断。便宜量大,几乎万能。

### 6.2 TLV3201(TI)

高速推挽比较器:
- 供电: 2.7-5.5V
- 传播延迟: 40ns
- 偏移电压: 1mV
- 输出: 推挽,轨到轨
- 价格: ~$0.8

适用: 高速过零检测、PWM峰值检测、需要快速响应的场合。

### 6.3 MAX9117(Maxim/ADI)

纳功耗推挽比较器:
- 供电: 1.0-5.5V
- 供电电流: 500nA(0.5uA)
- 传播延迟: 12us
- 偏移电压: 0.5mV
- 内置1.2V基准
- 价格: ~$1.0

适用: IoT电池监测、低功耗唤醒、便携设备。500nA几乎不影响电池寿命。

### 6.4 选型对比表

| 参数 | LM393 | TLV3201 | MAX9117 |
|------|-------|---------|---------|
| 通道数 | 2 | 1 | 1 |
| 供电范围 | 2-36V | 2.7-5.5V | 1.0-5.5V |
| 传播延迟 | 1.3 us | 40 ns | 12 us |
| 偏移电压 | 2 mV | 1 mV | 0.5 mV |
| 静态电流 | 0.5 mA | 3.5 mA | 0.5 uA |
| 输出类型 | 开集 | 推挽 | 推挽 |
| 内置基准 | 无 | 无 | 1.2V |
| 参考价格 | ~$0.1 | ~$0.8 | ~$1.0 |

## 7 开漏输出的应用技巧

### 7.1 上拉电阻选择

上拉电阻太小:功耗大(Vcc/R),漏电流大;上拉电阻太大:上升沿慢(R x C寄生电容),响应慢。

经验值:
- 高速应用: 1-10k欧
- 一般应用: 10-100k欧
- 超低功耗: 100k-1M欧(牺牲速度)

### 7.2 线或(Wired-OR)多报警源

多个开漏比较器输出连在一起,任一触发即拉低:

```
过压比较器 ---|
欠压比较器 ---+---[Rpullup]--- Vcc --- MCU中断
过温比较器 ---|
```

一条中断线监测多个报警条件,节省MCU GPIO。

### 7.3 电平转换

开漏输出上拉到不同电压,实现电平转换:

```
比较器(3.3V供电) --- 开漏输出 ---[Rpullup]--- 5V

输出高电平=5V(由上拉决定)
输出低电平=0V
```

3.3V系统的比较器输出可驱动5V逻辑,无需额外电平转换芯片。

## 8 实际IoT应用

### 8.1 电池欠压报警

需求: 单节锂电池,电压低于3.0V时触发MCU中断,执行优雅关机。

方案: MAX9117(内置1.2V基准) + 电阻分压:

```
Vbat ---[R1=150k]---+---[R2=100k]--- GND
                      |
                   V+ (MAX9117)
                   V- (内置1.2V基准)
                      |
                   Vout ---> MCU EXTI

Vth = 1.2V x (150k+100k)/100k = 3.0V
```

当Vbat降到3.0V时,分压点降到1.2V,比较器翻转,MCU中断执行关机。

加迟滞:在V+和Vout之间接R3=10M欧,迟滞约50mV,防止3.0V附近振荡。

### 8.2 过温关断

需求: 芯片温度超过85C时关断功率级。

方案: NTC热敏电阻 + LM393:

```
Vcc ---[NTC]---+---[R_fixed]--- GND
                |
             V+ (LM393)
             V- (Vref = 阈值对应电压)
                |
             Vout ---> 功率使能
```

NTC阻值随温度升高而降低,分压点电压上升,超过Vref时输出翻转,关断功率级。

### 8.3 光线阈值检测(白天/黑夜)

需求: 检测环境光是否低于阈值(黑夜),触发照明控制。

方案: 光敏电阻 + LM393:

```
Vcc ---[光敏电阻]---+---[R2]--- GND
                      |
                   V+ (LM393)
                   V- (Vref = 阈值)
```

黑夜时光敏电阻阻值大,分压点高,比较器输出高;白天阻值小,分压点低,输出低。

### 8.4 振动唤醒

需求: 设备休眠时,检测到振动则唤醒MCU。

方案: 压电振动传感器 + 比较器:

```
压电传感器 ---[整流滤波]--- V+ (比较器)
                              V- (Vref = 阈值)
                              Vout ---> MCU EXTI (唤醒)
```

振动产生交流信号,整流滤波后与阈值比较,超过则唤醒。

## 9 中断驱动设计

### 9.1 比较器输出接MCU中断

比较器输出连接到MCU外部中断引脚(EXTI):

```
比较器 Vout --- MCU EXTI0 (下降沿触发)
```

MCU可进入低功耗休眠,比较器翻转时EXTI唤醒MCU,响应时间us级。

### 9.2 从休眠唤醒的流程

```
1. MCU配置EXTI,进入STOP模式
2. 比较器翻转,EXTI触发
3. MCU从STOP模式唤醒
4. 读取比较器状态,确认报警条件
5. 执行相应动作(关断、报警、切换模式)
6. 清除中断标志,重新进入STOP模式
```

### 9.3 比较器功耗优化

纳功耗比较器(如MAX9117的500nA)可始终供电,不影响电池寿命。普通比较器(0.5mA)则需要:
- 在MCU休眠时断电(增加唤醒延迟)
- 用占空比方式间歇供电(可能漏检瞬态事件)
- 改用纳功耗比较器(最简单方案)

## 10 阈值参考电压产生

### 10.1 电阻分压

最简单的方式,用两个电阻从Vcc分压:

```
Vcc ---[R1]---+---[R2]--- GND
              |
           Vref = Vcc x R2 / (R1+R2)
```

优点: 简单、低成本。

缺点: 随Vcc变化而变化;电阻容差影响精度;不适合精密阈值。

### 10.2 电压基准IC

用精密基准源(如LM4040)提供稳定阈值:

```
Vcc ---[Rbias]---+--- Vref
                  |
                [LM4040]
                  |
                 GND
```

优点: 阈值精确稳定,不受Vcc波动影响。

缺点: 增加BOM成本,LM4040需要60uA以上最小电流。

### 10.3 选择建议

| 场景 | 推荐 |
|------|------|
| 阈值精度要求不高(>5%) | 电阻分压 |
| 电池欠压检测(3-5%精度) | 1%电阻分压 |
| 精密阈值(<1%) | 基准IC |
| 阈值需可调 | 数字电位器或DAC |

## 总结

比较器是IoT系统中最实用的模拟模块之一,核心要点:

1. **比较器不是运放**: 专为开环开关设计,速度快,不用负反馈。不要用运放替代比较器。
2. **迟滞不可省**: 信号缓慢经过阈值时,无迟滞必振荡。正反馈电阻网络实现施密特触发器。
3. **开漏输出灵活多用**: 线或多报警源、电平转换、上拉电阻决定高电平——一个开漏输出顶三个功能。
4. **纳功耗比较器适合IoT**: MAX9117的500nA可始终供电,不需要MCU控制通断。
5. **中断驱动省功耗**: 比较器输出接MCU EXTI,MCU可深度休眠,事件驱动唤醒。
6. **阈值参考要稳定**: 电池监测用基准IC,简单场景用电阻分压,精度和成本按需权衡。

记住开头的类比:比较器就是电路的"门铃",只回答"过了还是没过",简单但极其实用。
