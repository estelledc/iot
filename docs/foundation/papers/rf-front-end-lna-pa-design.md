# 射频前端LNA/PA设计基础与指标
> **难度**：🔴 高级 | **领域**：射频前端设计 | **阅读时间**：约 22 分钟

## 引言

想象一个助听器系统。外面的声音(微弱的射频信号)先经过一个高灵敏度麦克风(LNA，低噪声放大器)把声音放大但不引入杂音；播放时，扬声器(PA，功率放大器)把电信号放大到足以让人听清的音量。如果麦克风本身沙沙响(噪声系数高)，后面再怎么放大也只会放大噪音；如果扬声器失真(PA线性度差)，原本清晰的声音也会变得刺耳。射频前端的设计就是在这两端之间做极致的权衡：接收端追求最低噪声，发射端追求最高效率，同时两者还要通过一个快速切换的开关(SPDT)共享同一天线。对IoT设备来说，前端芯片可能只占BOM的5%，却决定了通信距离和可靠性的50%以上。

## 1. 射频前端架构

### 1.1 收发链路

```
接收: 天线 --> [BPF] --> [LNA] --> [混频器] --> [IF滤波] --> [ADC] --> 数字基带
发射: 数字基带 --> [DAC] --> [LPF] --> [调制器] --> [PA] --> [BPF] --> 天线

时分双工(TDD): 天线 --[SPDT开关]--> RX或TX
频分双工(FDD): 天线 --[双工器]--> RX和TX同时工作
```

### 1.2 IoT前端的特点

| 维度 | 手机 | IoT |
|------|------|-----|
| 频段数 | 多频段(5-40) | 通常1-2个频段 |
| PA功率 | +23~+28dBm | 0~+20dBm |
| LNA要求 | 极低NF(<1dB) | 中等NF(1.5-3dB) |
| 成本 | $3-8 | $0.3-2 |
| 功耗预算 | 宽裕(大电池) | 极紧张(小电池) |

## 2. LNA关键参数

### 2.1 噪声系数(Noise Figure, NF)

噪声系数衡量LNA本身引入的噪声：NF(dB) = 10 * log10(SNR_in / SNR_out)。实际LNA的NF约0.5~3dB。

NF对接收灵敏度的影响：

```
灵敏度(dBm) = -174 + NF + 10*log10(BW) + SNR_min

例: BLE (BW=1MHz, SNR_min=10dB)
NF=2dB: 灵敏度 = -102 dBm
NF=3dB: 灵敏度 = -101 dBm
1dB NF恶化 ≈ 10%距离缩短
```

### 2.2 增益、IIP3与P1dB

| 参数 | 含义 | IoT典型值 |
|------|------|-----------|
| 增益 | 信号放大倍数 | 15-25dB |
| IIP3 | 三阶交调点，衡量抗干扰线性度 | -5~+10dBm |
| P1dB | 1dB压缩点，增益开始下降 | -20~-5dBm |

经验关系：P1dB < IIP3 - 9~10dB。

### 2.3 关键参数权衡

| 优化方向 | 措施 | 代价 |
|----------|------|------|
| 降低NF | 增大偏置电流 | 功耗增加 |
| 提高IIP3 | 增大偏置/反馈 | NF恶化或增益降低 |
| 降低功耗 | 减小偏置电流 | NF恶化、IIP3降低 |

## 3. LNA设计考量

### 3.1 噪声匹配 vs 功率匹配

LNA输入匹配面临两个不同目标：

- **噪声匹配**：使NF最小，对应最优源阻抗Zopt
- **功率匹配**：使反射最小(S11最小)，对应共轭匹配Zin*

Zopt != Zin*(一般情况下)。噪声匹配时S11可能只有-5~-10dB；功率匹配时NF比最优点差0.5-1dB。IoT LNA通常在两者之间折中。

### 3.2 稳定性

LNA可能因寄生反馈而振荡。稳定条件：K > 1 且 |Delta| < 1。提高稳定性的方法：源极/发射极电感负反馈(最常用)、栅极串联小电阻、输出端并联稳定电阻。

### 3.3 IoT LNA的典型实现

| 方案 | NF | 增益 | 功耗 | 适用 |
|------|-----|------|------|------|
| SoC内置LNA | 3-6dB | 15-20dB | 2-5mA | 大多数场景 |
| 外置LNA(分立) | 0.8-1.5dB | 18-22dB | 3-8mA | 远距离BLE/LoRa |
| FEM集成LNA | 1.5-2.5dB | 12-16dB | 见FEM | WiFi/LTE模组 |

## 4. PA关键参数

### 4.1 输出功率与效率

PA能提供的最大不失真输出功率：

| 标准技术 | 典型输出功率 | 说明 |
|----------|-------------|------|
| BLE | 0~+4dBm | Class 2 |
| WiFi | +15~+20dBm | 消费级 |
| LoRa | +14~+20dBm | SX1276 |
| LTE | +23dBm | Power Class 3 |

功率附加效率(PAE) = (Pout - Pin) / Pdc * 100%。PAE越高，电池续航越长。Class A约25-35%，Class AB约35-50%，Class E约60-80%。

### 4.2 线性度

- AM-AM失真：输出幅度不再与输入幅度成线性关系
- AM-PM失真：输出相位随输入幅度变化
- ACPR(邻道功率比)：WiFi要求<-20~-30dBc，LTE要求<-30~-45dBc

### 4.3 谐波抑制

PA非线性会产生谐波(2f, 3f...)。法规要求谐波发射<-30dBm或相对载波-40dBc以上。Sub-GHz(如868MHz)的2次谐波(1.736GHz)可能落入移动通信频段，需特别注意。

## 5. PA工作类别

### 5.1 各类别对比

| 类别 | 导通角 | 效率 | 线性度 | 适用信号 |
|------|--------|------|--------|----------|
| A | 360度 | 25-35% | 最好 | 任意调制 |
| AB | 180-360度 | 35-50% | 较好 | WiFi/LTE |
| B | 180度 | 50-60% | 中等 | FM/PM |
| C | <180度 | 60-70% | 差 | 恒包络 |
| E | 开关 | 60-80% | 差(需滤波) | BLE/LoRa |
| F | 开关 | 70-85% | 差(需滤波) | 恒包络 |

### 5.2 IoT常用类别

**Class AB**：WiFi、LTE等非恒包络调制信号必须使用。回退功率时PAE急剧下降(10dB回退->PAE降至5-10%)。

**Class E**：BLE(GFSK)、LoRa(CSS)等恒包络信号适用。理论效率100%，实际60-80%，输出含大量谐波需低通滤波器。

### 5.3 功率回退与效率

```
输出功率回退  PAE(典型AB类)
0 dB          40%
3 dB          30%
6 dB          20%
10 dB         10%
```

WiFi IoT设备近距离通信功耗不比远距离低多少——PA大部分时间工作在低效区。解决方案：动态PA偏置控制。

## 7. TX/RX开关(SPDT)

### 7.1 功能与指标

```
        +----> RX (LNA)
天线 ---+
        +----> TX (PA)
```

| 指标 | 典型值 | 说明 |
|------|--------|------|
| 插入损耗 | 0.3-0.8dB | 信号通过开关的衰减 |
| 隔离度 | 20-40dB | TX到RX端的泄漏 |
| 切换速度 | 0.5-5us | TX/RX切换时间 |

IoT常用SPDT：HMC253(0.4dB插损)、SKY13350(0.3dB插损)，或集成在FEM内。

## 8. FEM集成方案

### 8.1 为什么用FEM

```
分立方案: PA + LNA + SPDT + 匹配元件 = 约9-13颗元件
FEM方案:  FEM + 匹配元件 = 约3-5颗元件
优势: 面积减少50-70%, 一致性更好
```

### 8.2 常见FEM产品

| 型号 | 厂商 | 频段 | 封装 |
|------|------|------|------|
| SKY85331-11 | Skyworks | 2.4GHz | 3x5mm |
| QPF4528 | Qorvo | 5GHz | 3x5mm |
| KCT8225M | 康希通信 | 2.4GHz | 4x4mm |

### 8.3 FEM设计注意事项

- 严格遵循参考设计：匹配网络经厂商优化，不要自行修改
- 电源去耦：多级去耦(10uF + 100nF + 10pF)
- 控制时序：TX_EN/RX_EN必须与SoC收发时序对齐
- 热设计：+20dBm输出耗散约200mW，需确保PCB散热

## 9. 级联噪声系数计算

### 9.1 Friis公式

```
F_total = F1 + (F2-1)/G1 + (F3-1)/(G1*G2) + ...

关键结论: 第一级噪声系数对总噪声系数影响最大
第一级增益越高，后级噪声贡献越小
```

### 9.2 计算示例

```
系统: 滤波器(IL=1.5dB) + LNA(NF=1.5dB, G=18dB) + 混频器(NF=8dB, G=5dB)

F_total = 1.41 + (1.41-1)/0.71 + (6.31-1)/(0.71*63.1)
        = 1.41 + 0.577 + 0.119 = 2.106
NF_total = 3.24 dB
```

混频器NF虽高达8dB，但LNA的18dB增益将其贡献压制到仅0.119——这就是LNA必须放在最前面的原因。

### 9.3 滤波器在LNA前还是后

```
方案A: 天线 --> [滤波器] --> LNA  | NF=3.24dB, LNA受保护
方案B: 天线 --> LNA --> [滤波器]  | NF=1.78dB, LNA无保护

IoT选择: 通常方案A，LNA过载后果远比NF增加1.5dB严重
```

## 10. 含FEM的链路预算

### 10.1 接收链路预算

```
BLE远距离通信，使用外置FEM:
  滤波器(IL=1dB) + FEM LNA(NF=2dB, G=13dB) + SoC LNA(NF=5dB)
  F_total = 2.13 --> NF = 3.28 dB
  灵敏度 = -174 + 60 + 10 + 3.28 = -100.72 dBm
  vs 无FEM: NF约6dB, 灵敏度-98dBm --> FEM改善约2.7dB
```

### 10.2 发射链路预算

```
WiFi IoT设备:
  SoC输出+15dBm, FEM PA增益+20dB(线性)?
  实际: FEM PA P1dB=+22dBm, 输入+15dBm远超P1dB, PA严重压缩

正确设计: SoC输出+5dBm, PA增益=17dB, 输出=+22dBm(在P1dB内)
注意: 需在SoC和FEM间加衰减器或降低SoC输出
```

## 11. PA热设计

### 11.1 热耗散计算

```
PA热耗散 = Pdc - Pout
PA输出+20dBm(100mW), PAE=40%: Pdc=250mW, 热耗散=150mW
PAE=20%: Pdc=500mW, 热耗散=400mW
```

### 11.2 散热设计

```
150mW --> PCB铜箔散热即可
400mW --> 散热过孔 + 大面积地平面
>500mW --> 可能需要散热片

PCB散热: PA下方铺满地铜 + 多打散热过孔(0.3mm/1mm间距)
         + 增加地平面面积(至少2倍芯片面积)
```

温度升高使PA增益下降(约-0.02dB/度)，PAE降低形成正反馈。IoT壳体封闭，温升可达20-40度，必须降额设计。

## 12. 法规输出功率限制

### 12.1 主要标准

| 地区 | 频段 | 最大EIRP | 标准 |
|------|------|----------|------|
| 中国 | 2.4GHz ISM | +20dBm | GB/T 22450.1 |
| 欧盟 | 2.4GHz ISM | +20dBm EIRP | ETSI EN 300 328 |
| 美国 | 2.4GHz ISM | +30dBm | FCC Part 15 |
| 中国 | Sub-GHz 470MHz | +17dBm | LPWAN频段 |
| 欧盟 | 868MHz | +14dBm | EN 300 220 |
| 全球 | LTE | +23dBm | 3GPP Power Class 3 |

### 12.2 EIRP vs 传导功率

```
EIRP = 传导功率 + 天线增益 - 馈线损耗

例: 传导+18dBm, 天线2dBi, 馈线0.5dB --> EIRP=+19.5dBm (合规)
换5dBi天线 --> EIRP=+22.5dBm (超标!) 需降传导功率至+15dBm
```

### 12.3 发射杂散限制

杂散发射限值通常为-30dBm到-54dBm，谐波需输出滤波器抑制。WiFi 2.4GHz的3次谐波(7.2GHz)和5GHz WiFi共存需特别注意。

## 总结

1. 射频前端由LNA(接收)和PA(发射)两个核心模块组成，通过SPDT开关共享天线
2. LNA最关键的指标是噪声系数(NF)，每增加1dB约等于通信距离缩短10%
3. PA的核心权衡是效率(PAE)与线性度(ACPR)的矛盾，IoT中Class AB和Class E最常用
4. Friis公式说明第一级(LNA)的NF和增益对整个接收链路噪声性能起决定性作用
5. FEM将PA+LNA+开关集成，是IoT前端的主流方案，需严格参考厂商设计
6. 外置PA可将发射功率从+4dBm提升到+20dBm，但需注意输入功率不能超过P1dB
7. PA热耗散随效率降低而急剧增大，封闭壳体IoT设备必须做降额设计和PCB散热
8. 法规限制的是EIRP而非传导功率，天线增益越高，允许的最大传导功率越低
