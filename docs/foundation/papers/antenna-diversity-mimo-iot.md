# 天线分集与MIMO在IoT设备中的应用
> **难度**：🔴 高级 | **领域**：多天线技术 | **阅读时间**：约 22 分钟

## 引言

想象你在嘈杂的餐厅里听朋友说话。只靠一只耳朵，旁边桌的谈话会不断干扰你；但如果用两只耳朵，大脑就能从两个略有差异的声音信号中"拼凑"出更清晰的语音。天线分集(antenna diversity)的原理与此类似——用两根或多根天线接收同一个信号，利用它们之间衰落的不相关性，大幅降低信号深衰落导致的丢包。MIMO更进一步：不仅用多根天线来抗衰落，还用它们同时传不同数据，就像高速公路从单车道扩建为多车道，吞吐量直接翻倍。但多车道也有代价——需要更多路权(频谱)、更复杂的信号处理、以及车道之间不能互相干扰(天线隔离)。对IoT设备来说，壳体小、电池小、成本敏感，多天线究竟是雪中送炭还是画蛇添足？本文从多径衰落出发，逐步拆解分集、MIMO的原理与IoT实践。

## 1. 多径衰落问题

### 1.1 什么是多径

无线电波在空间中传播时，遇到建筑物、地面、金属物体等会产生反射、散射和衍射。接收端收到的信号是直射径和若干反射径的叠加：

```
发射端 --> 直射径 --> 接收端
  |                         ^
  +--> 反射1(建筑) ---------+
  +--> 反射2(地面) ---------+
```

各径到达的相位不同，叠加后可能增强(同相)也可能抵消(反相)。当接收端移动哪怕半个波长(2.4GHz约6.25cm)，信号强度就可能从极强变到极弱——这就是多径衰落(multipath fading)，也叫瑞利衰落(Rayleigh fading)。

### 1.2 衰落对IoT的影响

- 短距离也不安全：室内10m通信，信号可能因某次反射叠加而深衰落15-20dB
- 静态设备更脆弱：固定安装的IoT传感器可能长期卡在衰落点
- 频率选择性衰落：宽带信号(如WiFi 20MHz)不同子载波衰落程度不同

### 1.3 衰落的量化

| 参数 | 典型值 | 含义 |
|------|--------|------|
| 衰落深度 | 20-30dB | 信号从均值跌落的最大幅度 |
| 相干带宽 | 1/时延扩展 | 频率间隔小于此值的两个信号衰落高度相关 |
| 相干距离 | 约半个波长 | 空间间隔小于此值的两个天线衰落高度相关 |

## 2. 天线分集技术

### 2.1 分集的基本思想

如果两根天线的衰落不相关，那么两根天线同时处于深衰落的概率远小于单根天线深衰落的概率。假设单天线深衰落概率为p，两天线同时深衰落概率为p^2——这就是分集增益的数学基础。

### 2.2 空间分集

最直观的分集方式：拉开天线间距，使各天线接收到的多径分量不同。

- 间距要求：通常需要大于半个波长(2.4GHz约6.25cm)，理想为1-2个波长
- IoT难点：设备尺寸常小于半个波长，物理上无法拉开足够距离
- 降低间距要求的方法：使用正交极化或不同辐射方向图的天线

### 2.3 极化分集

利用电磁波极化方向的不相关性：

- 水平极化与垂直极化的衰落几乎不相关
- 优势：两根天线可以紧挨着甚至共址，不需要空间间距
- 代价：极化不匹配会带来3dB极化失配损耗，但分集增益通常大于此损耗

### 2.4 方向图分集

不同辐射方向图的天线对多径的响应不同。实际方案：PIFA与单极子天线组合，或两副不同长度的PIFA，衰落相关性自然降低。

### 2.5 分集合并方式

| 合并方式 | 原理 | 分集增益 | 复杂度 |
|----------|------|----------|--------|
| 选择合并(SC) | 选信号最强的一路 | 最低 | 最低 |
| 等增益合并(EGC) | 各路同相后相加 | 中等 | 中等 |
| 最大比合并(MRC) | 各路按信噪比加权相加 | 最高 | 最高 |

WiFi和BLE通常采用选择合并——硬件简单，只需一个射频开关(SPDT)在两副天线间切换。

## 3. WiFi中的天线分集与MIMO

### 3.1 WiFi 4引入MIMO

802.11n是WiFi首次支持MIMO的标准：最大4x4 MIMO，支持空间复用和STBC，消费级路由器普遍2x2 MIMO，双流300Mbps。

### 3.2 2x2 MIMO工作模式

```
信道条件差 --> 分集模式(STBC): 1路数据流，可靠性提升
信道条件中 --> 空间复用: 2路数据流，速率约1.5-1.8倍
信道条件好 --> 波束成形: 2根天线相位调整，速率接近2倍
```

### 3.3 WiFi 5/6的MIMO演进

| 特性 | WiFi 4 | WiFi 5 | WiFi 6 |
|------|--------|--------|--------|
| 最大空间流 | 4 | 4 | 8 |
| MU-MIMO | 不支持 | 下行 | 上下行 |
| 波束成形 | 可选 | 标准 | 标准 |
| 典型IoT使用 | 1x1或2x2 | 1x1 | 1x1或2x2 |

## 4. MIMO基础原理

### 4.1 空间复用 vs 分集增益

**空间复用**：在相同频率、相同时间，通过不同天线发送不同数据流。条件是信道矩阵H接近满秩(多径丰富)，吞吐量随空间流数线性增长。

**分集增益**：多根天线传输相同数据(如STBC/Alamouti编码)，信道衰落不相关即可，误码率曲线斜率增大，深衰落概率指数级下降。

```
空间复用: 吞吐量优先，用多车道并行运货
分集增益: 可靠性优先，用多辆车运同一批货(不怕一辆车出事)
```

### 4.2 信道矩阵与秩

MIMO系统用信道矩阵H描述：y = H * x + n，其中H为Nr x Nt矩阵。信道矩阵的秩(rank)决定可支持的空间流数，秩等于min(Nr, Nt)时为满秩。多径丰富的环境中更可能满秩。

### 4.3 MIMO容量

```
C = min(Nt, Nr) * B * log2(1 + SNR)

MIMO容量与min(Nt, Nr)成正比，而非与SNR成正比。
增加天线数的回报可能比增加功率更高。
```

## 5. MIMO在IoT中的应用场景

### 5.1 WiFi/LTE IoT设备

| 场景 | 天线配置 | MIMO模式 | 典型设备 |
|------|----------|----------|----------|
| 传感器节点 | 1x1 | 无 | 温湿度传感器 |
| 智能家居网关 | 2x2 | 空间复用 | WiFi 6路由器 |
| 摄像头 | 2x2 | 空间复用 | AI摄像头 |

大多数WiFi IoT终端仍为1x1：成本(多一路射频增加BOM)、功耗(两路接收功耗翻倍)、体积(两根天线+隔离间距)。

### 5.2 LTE Cat.4的MIMO

LTE Cat.4强制要求终端支持2x2 MIMO下行，峰值150Mbps依赖空间复用。Cat.1只需1根接收天线，这也是Cat.1成本低的原因之一。

### 5.3 NB-IoT / LoRa / BLE为什么不用MIMO

| 技术 | 不用MIMO的原因 |
|------|----------------|
| NB-IoT | 窄带(180kHz)，增加天线不改善覆盖 |
| LoRa | 扩频增益已抗衰落，不需要分集 |
| BLE | 功耗和成本极致压缩，两路射频不可接受 |

## 6. 天线隔离度要求

### 6.1 隔离度定义与等级

天线隔离度指一根天线的发射信号泄漏到另一根天线的衰减量(S21参数)：

- 隔离度 > 20dB：MIMO性能良好
- 隔离度 15-20dB：性能轻微退化
- 隔离度 < 15dB：MIMO增益显著下降

### 6.2 提高隔离度的方法

```
方法1: 拉开间距 -- 间距>0.5波长约15-20dB，>1波长约20-25dB
方法2: 正交极化 -- 水平/垂直极化贡献约20-25dB，不需要额外空间
方法3: 解耦结构 -- 加入寄生元件或中和线，小间距下>20dB
方法4: 地槽/EBG -- 阻断表面波耦合，占用PCB面积
```

### 6.3 相关性与隔离度的关系

包络相关系数(ECC)衡量两天线信号衰落的相似程度：ECC < 0.3为良好，< 0.1接近理想。

| 隔离度 | ECC估计 | 分集效果 |
|--------|---------|----------|
| >25dB | <0.05 | 优秀 |
| 20-25dB | 0.05-0.15 | 良好 |
| 15-20dB | 0.15-0.3 | 可用 |
| <10dB | >0.5 | 无效 |

## 7. 紧凑MIMO天线设计

### 7.1 IoT设备的尺寸约束

| 设备 | 尺寸 | 尺寸/波长(2.4GHz) |
|------|------|-------------------|
| 智能插座 | 5x5cm | 0.4 |
| 温控器 | 8x8cm | 0.64 |
| 摄像头 | 10x10cm | 0.8 |

5cm见方的设备放两根2.4GHz天线，间距不到半个波长，隔离度天然不足。

### 7.2 小尺寸MIMO天线方案

**正交极化PIFA**：两副PIFA正交放置，极化正交提供隔离度，尺寸约8x8mm x 2。

**IFA + 单极子组合**：IFA辐射以水平极化为主，单极子以垂直极化为主，紧邻但隔离度可达20dB以上。

**自解耦天线**：两天线间加入"中和线"(neutralization line)，反相抵消空间耦合，0.1波长间距下实现>15dB隔离度。

## 8. 波束成形概念

### 8.1 波束成形原理

通过调整多根天线发射信号的幅度和相位，使信号在特定方向上相长干涉、其他方向相消干涉。选择各天线的加权系数，使目标方向叠加增强。

### 8.2 波束成形 vs 分集

| 维度 | 波束成形 | 分集 |
|------|----------|------|
| 目标 | 提高方向性增益 | 提高抗衰落能力 |
| 信道信息 | 需要CSI | 不需要CSI |
| IoT适用性 | 网关/AP端 | 终端端 |

IoT终端侧极少使用波束成形：天线数少(2-4根)、功耗敏感。

## 9. 实际实现案例

### 9.1 ESP32双天线分集

```
GPIO14 --> 控制SPDT射频开关
  +-- 状态0: 连接天线A (主天线)
  +-- 状态1: 连接天线B (分集天线)

工作流程: 接收前导码时快速切换天线，
比较RSSI后选更强的那副接收数据帧
```

硬件设计要点：SPDT插损<0.5dB(HMC253/SKY13350)；两副天线建议不同类型(极化分集)；射频走线等长对称。

### 9.2 LTE模组双天线布局

LTE Cat.4模组(如EC20)必须2x2 MIMO下行：主天线与分集天线分置设备两端(对角线)；分集天线只接收不发射；两根同轴线等长；最小间距>0.5波长。

### 9.3 WiFi 6 IoT网关设计

```
  +----[ANT1: PIFA(水平极化)]----+
  |        WiFi 6 SoC            |
  +----[ANT2: PIFA(垂直极化)]----+
  天线间距约1.5波长, 隔离度>25dB, ECC<0.1
```

## 10. 分集增益量化

### 10.1 理论分集增益

选择合并：分集增益(dB) ≈ 10 * log10(N)，N=2时为3dB。MRC增益更高约额外0.5-1.5dB。

### 10.2 实际分集增益

实测参考(2.4GHz室内)：

| 场景 | 选择合并 | MRC |
|------|----------|-----|
| 办公室(多径丰富) | 5-8dB | 7-10dB |
| 仓库(多径中等) | 4-6dB | 6-8dB |
| 开阔室外(多径少) | 1-3dB | 2-4dB |

注意：实际增益高于理论3dB，因为衡量的是衰落余量的改善，深衰落场景下改善远超3dB。

## 11. 何时MIMO对IoT过度设计

### 11.1 MIMO不合适的场景

```
1. 低数据率传感器 -- 每天几KB，1x1绰绰有余
2. 极小尺寸设备 -- 放不下两根有效隔离的天线
3. 电池供电低占空比设备 -- MIMO额外功耗占比过大
4. 成本极致压缩产品 -- 两路射频链路增加2-5元
```

### 11.2 单天线优化作为替代

| 优化方向 | 可获增益 | 难度 |
|----------|----------|------|
| 天线效率优化(匹配调谐) | 1-3dB | 低 |
| 外接高增益天线 | 3-6dB | 低 |
| 天线位置选择(远离金属) | 1-3dB | 低 |
| 地平面优化 | 1-2dB | 中 |

对大多数IoT场景，优化单天线的性价比远高于强上MIMO。

### 11.3 决策流程

```
1. 需要吞吐量 > 50Mbps? 否 --> 单天线
2. 设备尺寸 > 1个波长? 否 --> 正交极化分集或单天线优化
3. 功耗预算允许两路射频? 否 --> 单天线优化
4. LTE Cat.4强制要求? 是 --> 必须2x2 MIMO下行
```

## 总结

1. 多径衰落是无线通信的固有挑战，天线分集利用多根天线衰落的不相关性降低深衰落概率
2. 分集技术分空间、极化和方向图三种，IoT设备因尺寸限制常选极化分集
3. MIMO包含空间复用(提速)和分集增益(提可靠性)两条路径，WiFi 4/5/6和LTE Cat.4是典型应用
4. 天线隔离度是MIMO设计关键，需>20dB；小尺寸设备需用正交极化或解耦结构弥补间距不足
5. 波束成形适合网关/AP端，终端侧因功耗和复杂度极少使用
6. 大量IoT场景不需要MIMO，单天线优化是更务实的选择
7. LTE Cat.4的2x2 MIMO下行是强制要求，设计时必须规划双天线布局
