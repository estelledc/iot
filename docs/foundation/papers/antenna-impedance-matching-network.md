# 天线阻抗匹配网络设计与Smith圆图
> **难度**：🔴 高级 | **领域**：射频匹配设计 | **阅读时间**：约 25 分钟

## 引言

阻抗匹配就像水管接头。如果粗管直接接细管，水流会在接口处产生湍流和反射;加一个渐变过渡段，水流就顺畅通过。射频电路也一样——信号源(通常50 ohm)和天线(可能是20-j40 ohm)之间如果阻抗不匹配，功率就会在接口处反射回去，天线辐射出去的能量就少了。匹配网络就是那个"渐变过渡段"，它把天线的阻抗变换成信号源期望的50 ohm，让功率最大化传输。

本文从复数阻抗基础讲起，系统介绍Smith圆图的使用、匹配网络拓扑、元件选择、PCB布局和调谐流程，为IoT射频设计提供完整的匹配设计参考。

## 1 为什么需要阻抗匹配

### 1.1 最大功率传输定理

当信号源内阻为Rs+jXs，负载阻抗为RL+jXL时，最大功率传输条件为：

```
RL = Rs  (电阻相等)
XL = -Xs (电抗相反, 即共轭匹配)
即 Z_L = Z_S* (负载等于源阻抗的共轭)
```

### 1.2 反射系数与失配损耗

阻抗不匹配时信号在接口处反射。反射系数Gamma：

```
Gamma = (Z_L - Z_0) / (Z_L + Z_0)
ML = -10 * log10(1 - |Gamma|^2)  [dB]
```

| |Gamma| | 失配损耗 | 传输功率比 |
|--------|---------|-----------|
| 0 | 0 dB | 100% |
| 0.316 (S11=-10dB) | 0.46 dB | 90% |
| 0.5 (S11=-6dB) | 1.25 dB | 75% |
| 1.0 | 无穷大 | 0% |

### 1.3 对IoT产品的实际影响

以BLE为例，PA输出0dBm，天线效率50%。匹配不良(S11=-6dB, 损耗1.25dB)时EIRP = -4.26 + G_ant dBm;匹配良好(S11=-20dB, 损耗0.04dB)时EIRP = -3.05 + G_ant dBm。差距1.21dB，对应通信距离差约15%——在边际链路下可能就是连得上和连不上的区别。

## 2 复数阻抗回顾

### 2.1 阻抗与导纳

```
Z = R + jX   (R:电阻, X:电抗; X>0感性, X<0容性)
Y = 1/Z = G + jB   (G:电导, B:电纳; B>0容性, B<0感性)
```

### 2.2 串联与并联转换

```
R_p = R_s * (1 + Q^2),  X_p = X_s * (1 + 1/Q^2)
其中 Q = |X_s| / R_s = |B_p| / G_p
```

## 3 Smith圆图基础

### 3.1 归一化阻抗

Smith圆图上所有值相对Z_0归一化：z = Z / Z_0 = r + jx。Z_0通常取50 ohm。

### 3.2 Smith圆图的构成

Smith圆图是反射系数Gamma平面上等r圆和等x圆的叠加：

**等电阻圆**：圆心( r/(r+1), 0 )，半径 1/(r+1)。r=0为最外面单位圆，r=1圆心(0.5,0)半径0.5。

**等电抗圆**：圆心( 1, 1/x )，半径 1/|x|。x=0为实轴。

### 3.3 Smith圆图上的关键区域

| 区域 | 阻抗特征 |
|------|---------|
| 实轴(x=0) | 纯电阻 |
| 上半圆(x>0) | 感性阻抗 |
| 下半圆(x<0) | 容性阻抗 |
| 圆心(1,0) | 完美匹配(Z=Z_0) |
| 左半圆(r<1) | 电阻小于Z_0 |
| 右半圆(r>1) | 电阻大于Z_0 |

### 3.4 Smith圆图上的操作

- **串联电感**：沿等r圆顺时针移动(x增大)
- **串联电容**：沿等r圆逆时针移动(x减小)
- **并联电感**：沿等g圆逆时针移动(b减小)
- **并联电容**：沿等g圆顺时针移动(b增大)

> 关键记忆：串联走等r圆，并联走等g圆。

## 4 从VNA测量读取天线阻抗

### 4.1 VNA测量设置

1. VNA校准(SOL：开路/短路/负载)
2. 校准面延伸到天线馈点
3. 单端口测量S11，读取阻抗值(R+jX格式)

Z与S11的转换：Z = Z_0 * (1 + S11) / (1 - S11)

### 4.2 测量注意事项

- 天线必须在最终使用环境中测量(有外壳、电池等)
- VNA功率设低(-20dBm)避免压缩非线性器件
- 测量线缆使用半刚性同轴线

### 4.3 典型天线阻抗值

| 天线类型 | 2.4GHz典型阻抗 | 868MHz典型阻抗 |
|---------|---------------|---------------|
| IFA(优化后) | 45-j5 ~ 55+j5 ohm | 40-j10 ~ 60+j10 ohm |
| PIFA(优化后) | 50-j3 ~ 50+j3 ohm | 50-j8 ~ 50+j8 ohm |
| 蛇形天线 | 10-j30 ~ 20-j20 ohm | 8-j40 ~ 15-j25 ohm |
| 芯片天线 | 30-j20 ~ 50+j10 ohm | 20-j30 ~ 40-j10 ohm |

PIFA阻抗接近50ohm，几乎不需匹配;蛇形天线阻抗偏差大，匹配必不可少。

## 5 匹配网络拓扑

### 5.1 L型网络

两个元件(一串一并)，最简单的匹配网络：

```
形式1: 源 --[Z1串]--+--[Z2并]-- 地   (天线接在节点)
形式2: 源 --+--[Z1串]-- 天线  (Z2并接地)
           |
          [Z2]
```

- 只有一个解，元件值由源和负载阻抗唯一确定
- Q由阻抗比值决定，无法自由选择
- 带宽 = f0 / Q，Q大则带宽窄

### 5.2 PI型网络

三个元件(两并一串)：

```
源 --+--[Z2串]--+-- 天线
     |          |
    [Z1并]     [Z3并]
     |          |
     地         地
```

- 有一个自由度，可调Q -> 可调带宽
- 低通特性，对二次谐波抑制-15 ~ -30dB
- IoT射频前端中最常用

### 5.3 T型网络

三个元件(两串一并)，高通特性，对低频干扰抑制好，主要用于PA输出匹配。

### 5.4 拓扑选择原则

| 需求 | 推荐拓扑 |
|------|---------|
| 简单、元件少 | L网络 |
| 需要谐波抑制 | PI网络(低通) |
| 需要宽带宽 | PI网络(低Q) |
| PA输出匹配 | T网络 |
| 多频匹配 | 级联多组L网络 |

## 6 元件选择

### 6.1 电感

射频匹配电感关键参数：

- **Q值**：>30 at 2.4GHz减少损耗;多层陶瓷电感(0402/0603)Q值20-50，IoT主流
- **自谐振频率(SRF)**：必须高于工作频率2-3倍
- **直流电阻(DCR)**：<0.5ohm

| 厂商 | 系列 | 封装 | Q(2.4GHz) | SRF |
|------|------|------|-----------|-----|
| Murata | LQW15 | 0402 | 30-50 | >6GHz |
| Coilcraft | 0402HP | 0402 | 25-40 | >5GHz |
| Johanson | L-07 | 0402 | 20-35 | >5GHz |

### 6.2 电容

- **介质**：NP0/C0G是唯一选择(X5R/X7R温漂和压漂大)
- **精度**：5%(J档)或1%(F档)
- **封装**：0402最常用，0603用于>100pF

### 6.3 不推荐的选择

- 铁氧体电感：SRF太低，不适用于>500MHz
- 空气心电感：体积太大，不适合小型PCB

## 7 Smith圆图上的匹配设计流程

### 7.1 L网络设计示例

目标：将Z_A = 25-j50 ohm匹配到50 ohm(2.4GHz)。

**步骤1**：归一化 z_A = 0.5 - j1.0，在Smith圆图标出(下半圆，容性)。

**步骤2**：选择先串后并的L网络。串联Z1沿等r圆(r=0.5)移到g=1圆;并联Z2沿g=1圆移到圆心。

**步骤3**：确定串联元件。从z_A沿r=0.5圆顺时针(加感)移到与g=1圆交点：

```
交点: z_mid = 0.5 + j0.5
需增加电抗: jX = j0.5 - (-j1.0) = j1.5
L = 1.5 * 50 / (2*pi*2.4e9) = 4.97nH
```

**步骤4**：确定并联元件。从z_mid沿g=1圆移到圆心(1+j0)：

```
并联电纳增量: +j1.0 (容性)
C = 1.0 / (50 * 2*pi*2.4e9) = 1.33pF
```

**步骤5**：用SimSmith验证：S11 at 2.4GHz < -30 dB。

### 7.2 PI网络设计示例

PI网络可先选Q再定元件。如需BW=200MHz at 2.44GHz，Q=12.2。从源端出发经三段轨迹(并联-串联-并联)到达天线阻抗，在Smith圆图上直观跟踪，用仿真验证微调。

## 8 带宽考量

### 8.1 匹配带宽与Q值

匹配带宽 = f0 / Q_total。Q_total越高带宽越窄。

- BLE需覆盖83.5MHz -> Q < 29
- 单频Sub-GHz(868MHz+/-5MHz) -> Q < 87
- 蛇形天线本身Q=20-50时，匹配网络Q必须低

### 8.2 宽带匹配技巧

1. 选PI网络(有Q自由度)而非L网络
2. 使用较低Q值电感
3. 带宽仍不够则级联多级L网络
4. 在天线端改善——PIFA带宽优于IFA

## 9 射频频率下元件的寄生效应

### 9.1 电感寄生

```
L_ideal --[R_s]--+-- C_par --> 地
R_s: 串联电阻(决定Q),  C_par: 并联电容(决定SRF)
```

2.4GHz下3.3nH电感：理想感抗j49.7 ohm，C_par约0.15pF使实际阻抗偏移3-5%。接近SRF时偏移急剧增大。

### 9.2 电容寄生

0402 NP0 1.0pF在2.4GHz：理想容抗-j66.3 ohm，L_par约0.5nH使偏移约5%。封装越小L_par越小 -> 0402优于0603。

### 9.3 焊盘寄生

每个焊盘引入约0.2-0.5pF对地电容和0.1-0.3nH串联电感。对策：焊盘尽量小;焊盘下方地平面挖去(relief);仿真纳入焊盘寄生。

## 10 匹配网络的PCB布局

### 10.1 布局原则

1. **紧邻放置**：匹配元件尽量靠近天线馈点
2. **串联在前**：先串后并，减少并联接地走线长度
3. **元件间距小**：<0.5mm，减少走线电感
4. **地平面完整**：并联接地过孔紧邻焊盘，过孔>=2

### 10.2 布局示例

```
              L1(串联)     C1(并联)
馈点 --[焊盘]-[焊盘]---[焊盘]---[走线]---> 射频IC
               |          |
              (走线)    [过孔x2]
               |          |
              (到L1)     GND
```

### 10.3 布局禁忌

- 匹配元件远离馈点(>2mm走线引入不可控寄生)
- 并联接地走线长且细(电感增大)
- 匹配区附近走过信号线(串扰)
- 匹配元件跨地平面狭缝(回流路径被截断)

## 11 迭代调谐流程

### 11.1 首板测量与初版匹配

1. 首板匹配位留0402焊盘，不焊元件(0 ohm串联，开路并联)
2. VNA测原始天线阻抗，根据测量值设计匹配网络
3. 焊接计算元件值，再测S11

### 11.2 微调

| 观察到的S11 | 可能原因 | 调整方向 |
|------------|---------|---------|
| 谐振频率偏低 | 匹配电感/电容偏大 | 减小L或C |
| S11最深处不够深 | Q值偏高 | 增大并联电阻(减小Q) |
| S11曲线不对称 | 天线本身非对称 | 调PI网络中间元件 |
| 宽带内S11有突起 | 寄生谐振 | 检查焊盘/走线 |

每次调一颗元件，测量后再决定下一步。典型迭代：有经验1-2次，一般2-4次。

## 12 多频天线匹配

### 12.1 双频匹配挑战

单组L/PI网络只能在一个频点完美匹配。双频天线需在两个频点同时满足S11 < -10dB。

### 12.2 双频匹配方案

**方案1：级联两组L网络**

```
源 --[L1低频匹配]--[L2高频匹配]-- 天线
```

低频L网络在低频点匹配，高频L网络在高频点匹配，需联合优化。

**方案2：单组PI网络折中**

PI带宽若足够宽可同时覆盖两个频段。适用于频比<1.5(如2.4/5GHz WiFi)。GPS+BLE频比1.52，勉强可行但折中大。

**方案3：射频开关+独立匹配**

```
源 --[SPDT]--+--[GPS匹配]--[GPS天线]
              +--[BLE匹配]--[BLE天线]
```

最灵活但增加开关损耗(0.5-1.5dB)和成本。

## 总结

阻抗匹配是射频设计中不可回避的核心环节。要点回顾：

1. **为什么匹配**：最大功率传输、最小反射、链路预算每0.1dB都珍贵
2. **Smith圆图**：可视化阻抗变换的利器，串联走等r圆、并联走等g圆
3. **网络拓扑**：L网络简单但Q锁定;PI网络有Q自由度且抑制谐波，IoT最常用
4. **元件选择**：高Q电感 + NP0电容，注意SRF和寄生
5. **PCB布局**：紧邻、短走线、完整地平面
6. **调谐流程**：测量 -> 计算 -> 焊接 -> 再测 -> 微调，预留焊盘是安全网
7. **多频匹配**：级联L网络或PI宽带折中，频比大时考虑独立天线+开关

匹配设计是理论与实测的结合。计算和仿真给出起点，VNA实测决定终点。不要期望一次就完美——预留调试裕量，把匹配网络当作你与不确定之间的缓冲区。
