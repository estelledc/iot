# 锂电池CC-CV充电曲线与充电IC设计
> **难度**：🟢 初级 | **领域**：电池充电设计 | **阅读时间**：约 18 分钟

## 引言

给锂电池充电就像往气球里灌水: 一开始水管压力大, 水流稳定(恒流); 气球快满时, 如果还用同样的压力灌, 气球会爆, 所以要慢慢减小水流, 直到刚好灌满(恒压). 锂电池的CC-CV充电法就是这样的两步走策略. 理解这个过程, 是设计任何锂电池供电IoT设备的基础. 本文从电池化学讲起, 到充电IC选型和PCB布局, 覆盖充电设计的完整链路.

## 1 锂电池化学基础

### 1.1 常见锂电化学体系

| 化学体系 | 正极材料 | 标称电压 | 满充电压 | 能量密度 | 特点 |
|---------|---------|---------|---------|---------|------|
| LiCoO2 | 钴酸锂 | 3.7V | 4.2V | 高 | 手机常用, 循环寿命一般 |
| LiFePO4 | 磷酸铁锂 | 3.2V | 3.65V | 中 | 安全性好, 循环寿命长 |
| LiPo | 聚合物锂电 | 3.7V | 4.2V | 高 | 可做成薄片, IoT常用 |
| NMC | 镍锰钴 | 3.6到3.7V | 4.2V | 高 | 电动车常用, 平衡性好 |

IoT设备最常用的是LiPo(聚合物锂电池), 因为可以做成各种形状, 适合空间受限的设计.

### 1.2 电池关键参数

- **标称容量**: 如1000mAh, 表示以1000mA放电1小时的能量
- **标称电压**: 平均工作电压, 不是充电目标
- **充电截止电压**: 满充时必须不超过此值, 典型4.2V
- **放电截止电压**: 不能低于此值, 否则损伤电池, 典型2.8到3.0V
- **最大充电电流**: 通常标为C率, 如1C表示以容量数值的电流充电
- **内阻**: 影响充电效率和发热, 典型30到200毫欧

## 2 CC-CV充电曲线详解

### 2.1 充电的四个阶段

完整的锂电池充电过程分为四个阶段:

```
电压
4.2V |          _________________________
     |         /                        |
     |        /  恒压阶段(CV)            |
     |       /                          |
3.0V |______/                           |--> 充电终止
     |  预充 |    恒流阶段(CC)           |
     +------+----------------------------+----> 时间
       阶段1   阶段2                  阶段3

电流
I_CC |      |
     |      |\
     |      | \
I_pre|__    |  \___________________________
     |  |   |                              |
     |  |   |                              |--> I_term
     +--+---+------------------------------+----> 时间
      预充  恒流(CC)        恒压(CV)       终止
```

**阶段1 - 预充电(Pre-charge)**:
- 触发条件: 电池电压低于3.0V(深度放电)
- 充电电流: 0.1C(如1000mAh电池用100mA)
- 目的: 以小电流唤醒电池, 防止大电流损伤已过放的电极

**阶段2 - 恒流充电(Constant Current, CC)**:
- 触发条件: 电池电压达到3.0V以上
- 充电电流: 0.5C到1C(如1000mAh电池用500到1000mA)
- 过程: 电流恒定, 电压逐渐上升
- 截止: 电压达到4.2V(或其他满充电压)

**阶段3 - 恒压充电(Constant Voltage, CV)**:
- 触发条件: 电压达到4.2V
- 充电电压: 恒定4.2V
- 过程: 电流逐渐减小(电池内阻上的压差减小)
- 截止: 电流降到截止电流I_term

**阶段4 - 充电终止**:
- 触发条件: 充电电流降到I_term(通常C/10到C/20)
- I_term = 1000mAh / 10 = 100mA (C/10)
- 电池已基本充满, 停止充电

### 2.2 充电率(C-rate)的理解

C率表示充电或放电的速率:

```
C率 = 电流 / 标称容量
```

| C率 | 1000mAh电池电流 | 充电时间(约) | 适用场景 |
|-----|-----------------|-------------|---------|
| 0.1C | 100mA | 12到14小时 | 预充/涓流 |
| 0.5C | 500mA | 2到3小时 | 常规充电 |
| 1C | 1000mA | 1到1.5小时 | 快充(非典型IoT) |
| 2C | 2000mA | 30到45分钟 | 特殊快充 |

IoT设备通常使用0.5C或更低的充电率, 因为:
- 电池容量小, 即使0.5C绝对电流也不大
- 散热条件差, 大电流发热
- 延长电池循环寿命

### 2.3 满充电压4.2V的意义

4.2V是钴酸锂/聚合物锂电池的标准满充电压. 这个值非常关键:

- 超过4.25V: 正极材料开始分解, 产气, 有安全风险
- 超过4.35V: 可能引发热失控, 起火
- 低于4.1V: 没有真正充满, 容量损失5到10%

精度要求: 满充电压精度需要 +/-50mV 以内, 好的充电IC控制在 +/-20mV.

## 3 充电安全保护

### 3.1 四重保护机制

**过压保护(OVP, Over Voltage Protection)**:
- 监测电池电压, 超过4.25到4.3V时切断充电
- 防止充电IC故障导致电池过充
- 通常由独立的保护IC实现(如DW01)

**过流保护(OCP, Over Current Protection)**:
- 监测充电电流, 超过设定值时限制或切断
- 防止短路或充电IC失控
- 可由充电IC内部或外部保险丝实现

**过温保护(OTP, Over Temperature Protection)**:
- 通过NTC监测电池温度
- 典型范围: 0到45摄氏度允许充电, 之外暂停
- 高温充电加速衰减, 低温充电导致锂析出(枝晶)

**安全定时器(Safety Timer)**:
- 限制总充电时间, 防止充电IC卡在某个状态
- 预充定时器: 通常30分钟
- CC/CV定时器: 通常5到8小时
- 超时则终止充电并报错

### 3.2 保护IC与充电IC的分工

```
USB输入 --> [充电IC] --> [保护IC] --> 电池
              |              |
         控制充电过程    独立安全防线
         (CC/CV/温度)   (OVP/OCP/短路)
```

保护IC是最后一道防线, 即使充电IC完全失效, 保护IC也能在危险时断开电池. 永远不要只用充电IC内部保护而省去独立保护IC.

## 4 常用充电IC选型

### 4.1 四款充电IC对比

**TP4056 (国产, Nanjing TopPower)**:
- 充电电流: 可编程, 最高1000mA
- 输入电压: 4.5到6.5V(USB)
- 充电方法: CC-CV
- 特色: 极低价格(约0.1元), 应用极广
- 不足: 无电源路径管理, 无I2C控制, 精度一般

**BQ24072 (TI)**:
- 充电电流: 可编程, 最高1500mA
- 输入电压: 4.2到16V
- 充电方法: CC-CV
- 特色: 电源路径管理(动态优先USB或电池供电)
- 不足: 封装较大, 价格较高

**MCP73831 (Microchip)**:
- 充电电流: 可编程, 最高500mA
- 输入电压: 3.75到6V
- 充电方法: CC-CV
- 特色: 小封装(SOT-23-5), 简单易用
- 不足: 电流小, 无电源路径管理

**BQ25180 (TI)**:
- 充电电流: 可编程, 最高1000mA
- 输入电压: 3.9到17V
- 充电方法: CC-CV
- 特色: I2C控制, 集成LDO和负载开关, 超低静态电流
- 不足: 软件配置复杂度

| 接口 | 电阻配置 | 电阻配置 | 电阻配置 | I2C |

### 4.2 选型决策树

```
需要I2C控制吗?
  是 --> BQ25180
  否 --> 充电电流需求?
          <= 500mA --> MCP73831(最简单)
          <= 1000mA --> 需要电源路径管理?
                         是 --> BQ24072
                         否 --> TP4056(最便宜)
```

## 5 USB输入与电源路径管理

### 5.1 USB供电的特殊性

USB端口供电有严格规范:

- USB 2.0: 5V, 最大500mA
- USB 3.0: 5V, 最大900mA
- USB BC1.2: 5V, 最大1500mA(专用充电口)

充电IC需要处理:
- 输入电压可能不稳(5V +/- 5%)
- 输入电流受限(不能超过USB规范)
- 热插拔时的瞬态(浪涌电流)

### 5.2 电源路径管理

没有电源路径管理时, 系统负载和电池充电共享输入电流:

```
USB 500mA --> [充电IC] --> 电池(400mA) + 系统(100mA)
                          如果系统需要300mA, 电池只能充200mA
```

有电源路径管理时, 系统优先获得供电, 电池用剩余电流充电:

```
USB 500mA --> [充电IC] --> 系统优先(最高500mA)
                          电池用剩余(500 - 系统电流)
```

这在IoT设备中很重要: 发射瞬间电流可能超过500mA, 如果没有电源路径管理, USB电压会被拉低导致复位.

### 5.3 动态电源路径(DPPM)

高级充电IC(如BQ24072)支持动态电源路径管理:

- 输入电流余量不足时, 自动降低充电电流
- 电池电压足够时, 系统可由电池补充供电
- USB掉电时, 无缝切换到电池供电

## 6 太阳能充电考量

### 6.1 太阳能板的特殊性

太阳能板是电流源而非电压源, 输出随光照变化:

- 阴天: 可能只有额定输出的10%
- 部分遮挡: 输出急剧下降
- 电压范围宽: 0到V_oc(开路电压)

### 6.2 太阳能充电方案

**直接充电(低成本方案)**:
- 太阳能板 --> 二极管 --> 电池
- 问题: 无法控制充电电压和电流, 有过充风险
- 仅适用于小功率(< 100mW)且带保护IC的方案

**MPPT充电(高效方案)**:
- 使用MPPT(最大功率点跟踪)充电IC, 如BQ25504, SPV1040
- 动态调整负载使太阳能板工作在最大功率点
- 适合户外IoT传感器节点

**折中方案**:
- 普通线性充电IC + 限流电阻
- 成本低, 效率一般, 适合低功率应用

## 7 NTC温度监测

### 7.1 NTC热敏电阻原理

NTC(Negative Temperature Coefficient)电阻随温度升高而减小:

```
R_ntc = R_25 * exp(B * (1/T - 1/T_25))
```

典型参数:
- R_25 = 10k欧姆(25摄氏度时阻值)
- B值 = 3380到3950K(材料常数)

### 7.2 充电IC中的NTC使用

充电IC通过分压网络监测NTC电压:

```
V_CC
 |
[R_fixed]  (通常10k欧姆)
 |
 +--- V_ntc ---> 充电IC的NTC引脚
 |
[NTC]       (10k @ 25摄氏度)
 |
GND
```

充电IC内部预设温度阈值:
- T1 (冷): 0摄氏度, 低于此暂停充电
- T2 (凉): 10摄氏度, 预充电电流减小
- T3 (温): 45摄氏度, 高于此暂停充电
- T4 (热): 60摄氏度, 高于此完全关断

### 7.3 NTC的安装

NTC必须紧贴电池表面, 用导热胶固定. 如果NTC远离电池, 测量的是环境温度而非电池温度, 高电流充电时可能导致电池过热但NTC未检测到.

## 8 充电状态指示

### 8.1 STAT引脚功能

大多数充电IC提供STAT(status)引脚, 指示当前充电状态:

| 状态 | STAT引脚 | LED表现 |
|------|---------|--------|
| 正在充电 | 低电平 | LED亮(红) |
| 充电完成 | 高阻态 | LED亮(绿)或灭 |
| 充电错误 | 闪烁(1Hz) | LED闪烁 |
| 电池未接 | 闪烁(0.5Hz) | LED慢闪 |

### 8.2 典型LED指示电路

```
                     LED_RED        LED_GREEN
V_CC ---[1k]---|>|---+---[1k]---|>|--- V_CC
                    |                    |
                    +---- STAT引脚 ------+
```

STAT为低时红灯亮(充电中), 高阻时绿灯亮(充满), 闪烁时两灯交替.

### 8.3 MCU读取充电状态

对于有I2C的充电IC(如BQ25180), 可以通过寄存器读取详细状态:

```
// BQ25180 状态寄存器读取
uint8_t read_charge_status(void) {
    uint8_t val;
    i2c_read_byte(BQ25180_ADDR, 0x00, &val);
    // bit[2:0] = 充电状态
    // 000 = 禁止充电
    // 001 = 预充
    // 010 = 恒流充电
    // 011 = 恒压充电
    // 100 = 充电完成
    // 101 = 充电超时故障
    return val & 0x07;
}
```

## 9 PCB布局要点

### 9.1 热设计

充电IC在大电流充电时会发热. 关键布局原则:

- 充电IC底部thermal pad连接到大面积铜皮
- 铜皮不要被走线分割
- 过孔阵列将热量传导到内层或底层
- 充电IC远离温度敏感元件(NTC、电池)

### 9.2 输入输出电容

```
USB输入 ---[C_in]--- [充电IC] ---[C_bat]--- 电池
           4.7uF                 4.7到10uF
```

- C_in: 紧邻充电IC的VIN引脚放置, 距离<3mm
- C_bat: 紧邻充电IC的BAT引脚放置, 距离<3mm
- 电容的地端直接连接到地平面
- 缺少C_in时, USB热插拔瞬态可能损坏充电IC

### 9.3 检流电阻布局

对于外置检流电阻的充电IC:
- 检流电阻的Kelvin连接(4线制)走线要短且对称
- 电流路径和检测路径分开走线
- 检测走线直接从电阻焊盘引出, 不要从铜皮上引出

### 9.4 常见布局错误

- C_in和C_bat距离充电IC太远, 导致振荡
- 热焊盘没有打接地过孔, 散热不良
- 充电路径走线太细, 压降过大
- NTC走线与高电流走线平行走线, 串扰导致温度误判

## 10 IoT电池选型

### 10.1 容量、尺寸与循环寿命的权衡

| 优先级 | 典型场景 | 推荐电池 | 容量范围 |
|--------|---------|---------|---------|
| 尺寸最小 | 穿戴设备 | LiPo软包 | 50到200mAh |
| 均衡 | 传感器节点 | 18650圆柱 | 2000到3500mAh |
| 循环寿命 | 每日充放电 | LiFePO4 | 1000到3000mAh |
| 超长待机 | 一次性部署 | 锂亚硫酰氯 | 1000到12000mAh |

### 10.2 循环寿命考虑

不同化学体系的循环寿命差异大:

- LiCoO2/LiPo: 300到500次(80%容量)
- LiFePO4: 2000到5000次(80%容量)
- NMC: 500到1000次(80%容量)

对于每日充电的IoT设备, LiPo可能不到2年就需要更换电池. 如果设备寿命要求3年以上, 考虑LiFePO4或降低充电深度(只充到4.1V可显著延长寿命).

### 10.3 自放电与储存

长期储存建议:
- 储存电压: 3.7到3.8V(约50%SOC), 此状态下衰减最慢
- 储存温度: 15到25摄氏度
- 定期检查: 每3到6个月补电一次

## 11 快速充电概述

### 11.1 快充在IoT中不常见的原因

快充(1C以上)在消费电子中普及, 但IoT设备很少使用:

- 电池容量小, 常规0.5C已足够快
- 快充发热量大, IoT设备散热条件差
- 快充加速电池衰减
- 快充IC成本高, 需要更复杂的保护

### 11.2 了解快充协议

即使不用快充, 也需要了解其存在, 避免误触发:

- Qualcomm QC: 通过D+/D-线协商高压快充
- USB PD: 通过CC线协商电压和电流
- 如果充电IC不支持这些协议, USB充电器会默认5V/500mA输出

对于IoT设备, 通常直接用5V USB输入, 不需要支持任何快充协议.

## 总结

CC-CV是锂电池充电的基本方法, 理解其四个阶段(预充、恒流、恒压、终止)是充电设计的基础. 选型时:

- 极低成本: TP4056
- 简单可靠: MCP73831
- 需要电源路径管理: BQ24072
- 需要I2C控制和超低功耗: BQ25180

安全方面, 永远不要省去独立的保护IC. PCB布局要特别注意电容的放置位置和充电IC的散热. 对于长期运行的IoT设备, 电池化学体系和充电深度(4.1V vs 4.2V)的选择比充电IC本身更影响设备寿命.
