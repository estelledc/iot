# 蜂窝Cat.1/Cat.4模组硬件选型与集成
> **难度**: 中级 | **领域**: 蜂窝模组集成 | **阅读时间**: 约 20 分钟

## 引言

想象你的物联网设备需要联网上报数据。Wi-Fi只在室内有,蓝牙传不远,Sub-GHz要自己建基站。蜂窝网络就像现成的高速公路 -- 基站已经建好,插卡就能跑。但高速公路也分车道:Cat.4是四车道快车道(150Mbps下行),Cat.1是单车道慢车道(10Mbps下行),价格和功耗差好几倍。选哪个车道,取决于你的数据量和钱包。本文聚焦Cat.1和Cat.4模组的硬件选型与集成实践。

## 1 LTE Category概览

### 1.1 Cat等级与能力

LTE Category定义了终端设备的上下行峰值速率:

| Category | 下行峰值 | 上行峰值 | 空间复用 | 64QAM下行 | 典型应用 |
|----------|----------|----------|----------|-----------|----------|
| Cat.1 | 10Mbps | 5Mbps | 1层 | 不支持 | IoT中速 |
| Cat.1bis | 10Mbps | 5Mbps | 1层 | 支持 | Cat.1升级版 |
| Cat.4 | 150Mbps | 50Mbps | 2层 | 支持 | 视频/CPE |
| Cat.M1 | 1Mbps | 1Mbps | - | - | NB-IoT窄带 |

Cat.1和Cat.4的关键区别不只是速率,还有:
- **基带复杂度**: Cat.4需要2x2 MIMO和64QAM解码, 基带面积和功耗远大于Cat.1
- **射频带宽**: Cat.4需要20MHz带宽, Cat.1只需5MHz
- **成本**: Cat.4模组价格约Cat.1的2-3倍

### 1.2 Cat.1 vs Cat.1bis

Cat.1bis是3GPP Rel-14对Cat.1的增强:

- 增加了64QAM下行支持(速率不变但效率更高)
- 支持VoLTE(部分Cat.1不支持)
- 与Cat.1完全后向兼容
- 当前主流新品多已采用Cat.1bis(如EC800N)

### 1.3 定位分析

```
速率/功耗定位:

  高速高功耗          Cat.6/Cat.12 (手机)
      |                |
  中速中功耗          Cat.4 (CPE/视频)
      |                |
  低速低功耗          Cat.1/Cat.1bis (IoT中速)
      |                |
  极低功耗            Cat.M1/NB1 (IoT低频)
```

Cat.1的甜蜜点: 速率足够传图片/语音, 功耗远低于Cat.4, 成本接近NB-IoT的2-3倍但能力强得多。

## 3 硬件集成设计

### 3.1 接口选择: UART vs USB

蜂窝模组与主机MCU的通信接口:

| 接口 | 最高速率 | 适用场景 | 注意事项 |
|------|----------|----------|----------|
| UART | 921600bps(常用) | AT命令控制, <500KB/s数据 | 硬件流控(RTS/CTS)必须接 |
| USB 2.0 | 480Mbps | 大数据传输, Cat.4推荐 | 需MCU支持USB Host |
| USB CDC | 同USB | 虚拟串口, 简化驱动 | Linux下ttyUSB0 |

Cat.1选UART足够(10Mbps峰值, UART可到约5Mbps); Cat.4建议USB才能发挥速率。

### 3.2 电源设计

蜂窝模组的电源设计是最容易出问题的环节:

**电压范围**: VBAT通常3.3V-4.2V(Li-ion电池直接供电最理想)

**瞬态电流**: 发射时电流可达2A峰值(Cat.4), 1A峰值(Cat.1)

```
VBAT供电设计:

电源源 ---> LDO/DCDC(输出4.0V) ---> 100uF(钽电容) ---> VBAT
                                           |
                                      10uF(MLCC) x 2
                                           |
                                      100nF(MLCC) x 4
                                           |
                                       模组VBAT引脚
```

关键设计规则:
1. **LDO不适合大电流**: Cat.4发射2A时LDO压降和发热严重, 推荐DCDC
2. **VBAT电容必须够大**: 100uF钽电容+10uF MLCC, 容量不足导致发射时VBAT跌落, 模组复位
3. **电源走线宽**: VBAT走线至少2mm宽, 从电源到模组走线电阻<50mohm
4. **不要和MCU共用LDO**: MCU的数字噪声会耦合到模组射频

### 3.3 SIM卡设计

SIM卡接口信号:

```
MCU/模组          SIM卡座
  SIM_VDD -------> VCC (1.8V或3.0V, 模组自动选择)
  SIM_CLK -------> CLK
  SIM_DATA <-----> DATA (双向, 上拉4.7K到VDD)
  SIM_RST  -------> RST
  SIM_GND -------> GND
```

设计要点:
- SIM_DATA必须加上拉电阻(4.7K-10K), 否则初始化失败
- SIM卡座靠近模组, 走线<30mm
- ESD保护: 每条SIM线加TVS二极管(如PRTR5V0U2X)
- 插拔检测: 部分模组支持SIM卡热插拔检测引脚

### 3.4 天线接口

蜂窝天线要求:

| 项目 | 要求 |
|------|------|
| 频段 | 至少覆盖所用的LTE Band |
| 增益 | 0-3dBi (设备内嵌) |
| VSWR | < 2:1 |
| 效率 | > 30% |

常用LTE Band(中国):
- **Band 3**: 1800MHz (FDD, 移动/联通)
- **Band 38**: 2600MHz (TDD, 移动)
- **Band 39**: 1900MHz (TDD, 移动)
- **Band 40**: 2300MHz (TDD, 移动/联通)
- **Band 41**: 2500MHz (TDD, 电信/联通)

天线需覆盖至少3个Band才能保证全国漫游。主天线+分集天线(Cat.4 MIMO需要两根天线)。

## 4 PCB布局设计

### 4.1 天线净空区

蜂窝模组的射频对天线环境极其敏感:

```
+----------------------------------------+
|  模组区域      |  天线净空区            |
|  +--------+   |  (无铜/无走线/无元件)  |
|  | 模组   |   |                        |
|  | RF焊盘|-->|  天线或天线连接器       |
|  +--------+   |                        |
+----------------------------------------+
```

- 主天线净空区: 至少5mm宽, 从天线到最近金属件>10mm
- 分集天线(MIMO): 与主天线间距>lambda/2 (约60mm@2.6GHz)
- 天线下方不要有电池、LCD、金属外壳

### 4.2 地平面

蜂窝频段(1.8-2.6GHz)对地平面依赖度高:

1. 地平面越大越好, 最小要求50mm x 80mm
2. 模组下方必须有完整地平面
3. 地平面不要被走线截断
4. 顶层和底层地通过密集过孔连接

### 4.3 模组焊盘与走线

1. 模组焊盘按厂商推荐的Footprint, 不要自行修改
2. VBAT焊盘到电源的走线最短最宽
3. SIM卡走线差分等长(误差<1mm)
4. UART走线加串联电阻(22-33ohm)减缓边沿, 减少EMI
5. RF走线(模组到天线连接器)50欧姆阻抗控制

## 5 AT命令与数据连接

### 5.1 AT命令基础

蜂窝模组通过AT命令集控制, 核心命令:

```
AT+CPIN?              // 查询SIM卡状态: +CPIN: READY
AT+COPS?              // 查询注册网络: +COPS: 0,0,"CHINA MOBILE"
AT+CESQ              // 查询信号质量: +CESQ: 30,0,255,255,18,0
AT+CGDCONT=1,"IP","cmnet"  // 设置APN
AT+CGACT=1,1         // 激活PDP上下文(拨号上网)
AT+HTTPGET="http://..."    // HTTP GET请求(部分模组内置)
```

### 5.2 数据连接流程

```
1. 上电等待模组就绪   AT --> OK
2. 检查SIM卡          AT+CPIN? --> READY
3. 注册网络           AT+COPS? --> 运营商名
4. 设置APN            AT+CGDCONT=1,"IP","cmnet"
5. 激活连接           AT+CGACT=1,1 --> OK
6. 数据传输           Socket/HTTP/FTP...
7. 断开连接           AT+CGACT=0,1
```

### 5.3 常见问题排查

| 现象 | 可能原因 | 排查命令 |
|------|----------|----------|
| 无SIM卡 | SIM卡座虚焊/未插 | AT+CPIN? |
| 注册失败 | 信号弱/频段不匹配 | AT+COPS=? (搜网) |
| 拨号失败 | APN错误/欠费 | AT+CGDCONT? AT+CGACT? |
| 速率低 | 单天线/信号差 | AT+CESQ (看RSRP) |
| 频繁掉线 | VBAT跌落/天线差 | 示波器看VBAT |

## 6 VoLTE支持

### 6.1 VoLTE对硬件的要求

VoLTE(Voice over LTE)通过IMS网络传输语音:

- 模组必须支持IMS SIP协议栈
- 需要额外的音频编解码(AMR-WB/EVS)
- 部分Cat.1模组(如EC200S)不支持VoLTE -- 选型时需确认
- MIC和Speaker通过PCM/I2S连接到模组

```
MCU <--I2S/PCM--> 蜂窝模组 <--IMS--> LTE网络
                      |
MIC --> ADC --> 模组
模组 --> DAC --> Speaker
```

### 6.2 VoLTE AT命令

```
AT+CVHU=1             // 启用VoLTE
AT+CLCC               // 查询当前通话
ATD10086;             // 拨号
ATH                   // 挂断
AT+CLIP=1             // 来电显示
```

## 7 GNSS组合模组

### 7.1 组合方案

许多IoT设备同时需要蜂窝和定位, 组合模组省空间省成本:

| 方案 | 说明 | 示例 |
|------|------|------|
| 蜂窝+GNSS集成 | 同一颗芯片, 共享射频 | EC800N(GNSS版) |
| 蜂窝+GNSS双芯片 | 两个芯片共存 | SIM7600+MC20 |
| 蜂窝+GNSS+Wi-Fi | 三合一 | SIM7600C-H |

### 7.2 GNSS天线

GNSS天线(1.575GHz GPS L1)与蜂窝天线(1.8-2.6GHz)频段接近:

- 两根天线间距至少20mm, 避免互扰
- GNSS天线通常用陶瓷贴片天线, 右旋极化
- 蜂窝发射时GNSS可能被阻塞(约-5~-10dBm耦合), 协议栈需协调时隙

## 8 热管理

### 8.1 发热来源

蜂窝模组的主要发热源:

1. **PA(功放)**: 发射时效率约30-40%, 剩余功率变热
2. **基带处理器**: Cat.4基带复杂度高, 连续下载时持续发热
3. **DCDC**: 转换效率90-95%, 5%损耗变热

### 8.2 热设计

| 场景 | 策略 |
|------|------|
| Cat.1间歇传输 | 通常不需要额外散热, PCB铜皮足够 |
| Cat.4连续下载 | 需要导热焊盘连接到金属外壳或散热片 |
| 密封外壳 | 增加导热硅脂到外壳, 或限制连续传输时间 |

关键测量: 模组表面温度不能超过85度(工业级)或70度(商业级)。

## 9 认证要求

### 9.1 认证类型

蜂窝模组入网涉及多重认证:

| 认证 | 内容 | 周期 | 费用(参考) |
|------|------|------|-----------|
| CCC | 中国强制性产品认证 | 4-6周 | 2-5万 |
| NAL | 电信设备进网许可 | 6-8周 | 3-8万 |
| 运营商入库 | 移动/联通/电信入库测试 | 4-12周 | 5-15万 |
| GCF/PTCRB | 国际型号认证 | 8-12周 | 10-20万 |

### 9.2 使用认证模组vs裸芯片

- **认证模组**: 模组厂已完成CCC/NAL, 整机只需EMC认证, 周期短
- **裸芯片集成**: 整机需做全部射频+协议认证, 周期长、费用高, 但BOM成本低

对于年出货<10K的项目, 用认证模组(认证费摊薄后更划算); 对于>100K的项目, 可考虑裸芯片集成。

## 10 成本对比与2G/3G退网迁移

### 10.1 模组成本对比

| 方案 | 模组成本 | 数据流量费/月 | 典型应用 |
|------|----------|--------------|----------|
| NB-IoT | 15-25元 | 1-3元 | 智能水表/烟感 |
| Cat.1 | 30-50元 | 5-20元 | 共享设备/POS |
| Cat.4 | 60-100元 | 20-50元 | 视频监控/CPE |

### 10.2 2G/3G退网时间表

| 网络 | 中国移动 | 中国联通 | 中国电信 |
|------|----------|----------|----------|
| 2G GSM | 2025年逐步退 | 部分退网 | 已退(CDMA) |
| 3G TD-SCDMA | 已退 | - | - |
| 3G WCDMA | - | 2025年逐步退 | - |

### 10.3 迁移路径

```
2G设备迁移:
  |
  +-- 语音+低数据 --> Cat.1 + VoLTE (替代GSM语音)
  |     示例: POS机、对讲机 --> EC600N/A7670
  |
  +-- 纯低数据 --> NB-IoT (替代GPRS低频)
  |     示例: 水表、电表 --> BC260Y/BC95
  |
  +-- 中等数据 --> Cat.1 (替代3G数据)
        示例: 共享充电宝、广告屏 --> EC200S/EC800N
```

迁移注意事项:
1. SIM卡可能需要更换(部分USIM卡不兼容2G, 反之亦然)
2. AT命令集有差异, 需要适配(尤其是数据拨号和SMS)
3. 天线可能需要更换(2G用900/1800MHz, LTE用不同Band)
4. 电源设计需重新评估(LTE发射电流大于GSM)

## 总结

蜂窝Cat.1/Cat.4模组为物联网设备提供了"插卡即联网"的便捷方案。硬件集成的关键在于:

1. 电源设计是第一道关 -- VBAT跌落是蜂窝模组复位的头号原因,电容不能省
2. 接口选择要匹配速率 -- Cat.1用UART足够, Cat.4需要USB
3. SIM卡和天线是"看似简单实则容易出问题"的两环 -- ESD保护和天线净空不可省
4. 认证成本在选型时就需纳入 -- 认证模组vs裸芯片的决策影响整个项目周期
5. 2G退网是进行时 -- 新项目不要选GSM模组,存量项目需尽早规划Cat.1迁移
