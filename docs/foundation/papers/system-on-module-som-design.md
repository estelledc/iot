# 系统级模块SoM在IoT产品快速落地中的应用
> **难度**：🟢 初级 | **领域**：产品开发策略 | **阅读时间**：约 18 分钟

## 引言

想象你在组装电脑。如果每次都要自己焊 CPU、贴内存、做主板,恐怕一台电脑都做不出来。但如果有人已经把 CPU、内存、显卡焊在一块小板子上,你只需要做机箱和接口,效率就高多了。SoM(System on Module)就是这个思路:把复杂的计算核心做成标准模块,你只做应用相关的载板。

## 1 SoM 的概念

### 1.1 什么是 SoM

SoM 是一块小型 PCB 模块,集成了嵌入式系统的核心功能:

- 处理器(MCU 或 MPU)
- 内存(DRAM + Flash)
- 电源管理(PMIC)
- 无线连接(WiFi/BLE/蜂窝)
- 天线
- 高密度连接器(用于插到载板上)

你不需要设计这些复杂电路,只需要设计载板(Carrier Board),把传感器、接口、电源等应用相关的电路放上去。

### 1.2 SoM 的类比

可以把 SoM 想象成"电脑的主板",载板是"扩展坞":

- SoM: 提供计算能力和通信能力(通用)
- 载板: 提供行业特定接口(定制)

这就像台式机的标准 ATX 主板:主板是通用的,但机箱、风扇、接口面板根据需求定制。

### 1.3 SoM 与 SoC 的区别

| 术语 | 含义 | 例子 |
|------|------|------|
| SoC | 芯片级,单片集成处理器+外设 | STM32, ESP32 |
| SiP | 封装级,多芯片在同一个封装内 | nRF52840 + Flash |
| SoM | 模块级,完整系统在一块小PCB上 | CM4, Verdin iMX8 |

SoM 是比 SoC 更高一个层级的集成,它已经包含了 SoC 周围必须的配套器件。

## 2 SoM vs 全定制设计

### 2.1 SoM 方案的优势

1. **上市速度快**: 不需要设计复杂的处理器电路(DDR 布线、电源树),节省 2-4 个月
2. **NRE 成本低**: 不需要高端 PCB(4-8 层、阻抗控制),节省打样和工程费用
3. **射频已认证**: WiFi/BLE/蜂窝模块已经通过 FCC/CE 认证,载板只需做无意辐射测试
4. **设计已验证**: 模块厂商已经调试好处理器和内存的兼容性
5. **长期供货保证**: 模块厂商承诺 10-15 年供货,减少 EOL 风险

### 2.2 全定制方案的优势

1. **BOM 成本低**: 没有模块溢价,大批量时差异显著
2. **尺寸可控**: 没有模块+连接器的空间开销
3. **完全可控**: 可以优化每一个细节
4. **知识产权**: 核心设计在自己手中

### 2.3 选择决策

| 评估维度 | SoM 方案 | 全定制方案 |
|----------|----------|-----------|
| 年产量 | < 5K-50K | > 50K |
| 开发周期 | 3-6 个月 | 6-12 个月 |
| 硬件团队规模 | 1-2 人 | 3-5 人 |
| NRE 成本 | 低 | 高 |
| 单机 BOM | 较高 | 较低 |
| 认证难度 | 低 | 高 |

## 3 典型 SoM 的组成

### 3.1 模块上有什么

以 Raspberry Pi CM4 为例:

```
+----------------------------------+
|  BCM2711 (4x Cortex-A72)        |
|  [1GB/2GB/4GB/8GB LPDDR4]       |
|  [8GB/16GB/32GB eMMC]           |
|  [WiFi/BLE (可选)]               |
|  [PMIC: MXL7704]                |
|  [2x 100-pin 高密度连接器]       |
+----------------------------------+
  |  |  |  |  |  |  |  |  (引出GPIO/USB/CSI/DSI等)
```

### 3.2 载板上做什么

载板是应用相关的,通常很简单:

- 电源输入和保护(5V/12V/24V 转 SoM 需要的电压)
- 传感器接口(I2C/SPI/UART/ADC)
- 显示接口(HDMI/MIPI DSI/RGB)
- 通信接口(RS485/CAN/Ethernet)
- 工业数字量输入/输出
- 调试接口(UART/JTAG)
- 机械安装孔和外壳配合

### 3.3 载板 PCB 复杂度

- 2 层板即可满足大多数 IoT 载板需求
- 关键:连接器扇出可能需要 4 层
- 无需处理 DDR 高速信号(已在 SoM 上)
- 无需阻抗控制(除了 USB 和 Ethernet)

## 4 流行 IoT SoM 方案

### 4.1 方案对比

| SoM | 处理器 | OS | 连接 | 目标市场 |
|-----|--------|-----|------|----------|
| RPi CM4 | Cortex-A72 | Linux | WiFi/BLE(可选) | 通用/教育 |
| Particle SoM | nRF52840 | RTOS | BLE+Cellular | IoT/M2M |
| Toradex Verdin iMX8 | Cortex-A72+M4 | Linux | WiFi/BLE | 工业 |
| ESP32-S3 模块 | Xtensa LX7 | RTOS | WiFi/BLE | 消费级IoT |
| Variscite DART | iMX6/7/8 | Linux | WiFi/BLE | 工业/医疗 |

### 4.2 Raspberry Pi CM4

特点:

- 4 核 Cortex-A72,最高 1.5 GHz
- 1-8GB LPDDR4, 8-32GB eMMC
- 可选 WiFi/BLE
- 两个 100-pin 连接器
- 价格: $25-$75(取决于配置)
- 大量社区资源和文档

适合:需要 Linux 能力、界面显示、多媒体处理的 IoT 网关。

### 4.3 Particle SoM

特点:

- nRF52840 MCU + U-blox 蜂窝模组
- Device Cloud 平台(设备管理、OTA)
- 内置 SIM 卡(eSIM)
- 价格: $30-$60
- 非常适合蜂窝 IoT 应用

适合:资产追踪、远程监控、智能表计。

### 4.4 ESP32 模块

特点:

- ESP32-S3: 双核 240MHz, WiFi/BLE
- 模块化封装(.castellated pads)
- 极低成本: $2-$5
- 丰富的 GPIO 和外设
- 乐鑫和第三方提供大量模块变体

适合:成本敏感的消费级 IoT 产品。

## 5 SoM 的形态因子

### 5.1 SODIMM 型

- Toradex Apalis/Verdin, Variscite DART
- 插入 SODIMM 插座
- 引脚密度高,支持高速信号
- 更换方便(插拔即可)

### 5.2 板对板连接器型

- Raspberry Pi CM4(2x 100-pin Hirose)
- Particle SoM
- 需要匹配连接器在载板上
- 连接可靠,适合工业环境

### 5.3 Castellated Pads(半孔焊盘)

- ESP32 模块最常见
- 模块直接焊在载板上
- 占用面积最小
- 不可更换,维修困难

### 5.4 选择建议

| 形态 | 适合场景 | 维修性 | 面积效率 |
|------|----------|--------|----------|
| SODIMM | 工业设备,需更换模块 | 好 | 中 |
| 板对板 | 工业设备,高可靠性 | 中 | 中 |
| Castellated | 消费电子,成本敏感 | 差 | 高 |

## 6 SoM 选型标准

### 6.1 关键评估维度

1. **处理能力**: MCU(RTOS)还是 MPU(Linux)
2. **连接选项**: WiFi/BLE/蜂窝/LoRa/Ethernet
3. **工作温度**: 商业级(0-70C) vs 工业级(-40-85C)
4. **认证状态**: FCC/CE/IC/RCM 是否已完成
5. **长期供货**: 厂商是否承诺 10+ 年供货
6. **社区/支持**: 文档、示例代码、论坛活跃度
7. **价格**: 模块单价 + 载板成本 + 认证成本

### 6.2 容易忽视的评估项

- **功耗**: SoM 自身待机功耗,影响电池设计
- **启动时间**: Linux SoM 启动需 5-30 秒,MCU SoM 毫秒级
- **实时性**: Linux 不适合硬实时,需确认是否支持 PREEMPT_RT
- **散热**: 高性能 SoM 需要散热片甚至风扇

## 7 载板设计要点

### 7.1 遵循参考设计

SoM 厂商通常提供参考载板原理图和 PCB:

- 电源连接方式必须参考设计
- 连接器封装必须使用厂商提供的
- 高速信号走线遵循厂商指南

### 7.2 高速信号注意事项

即使 SoM 处理了 DDR,载板上仍可能有高速信号:

- USB: 差分对,90 欧姆阻抗,等长
- Ethernet: 差分对,100 欧姆阻抗
- MIPI CSI/DSI: 差分对,100 欧姆阻抗,等长
- HDMI: 差分对,100 欧姆阻抗

### 7.3 热设计

SoM 上的处理器可能产生 1-5W 热量:

- 确认 SoM 在载板上的安装方向(散热片朝上还是朝下)
- 载板上预留散热片安装空间
- 考虑外壳内部空气流通

## 8 实战案例: 工业IoT网关

### 8.1 需求

- Linux 操作系统(支持 Python/Node.js)
- 2 路 RS485(Modbus RTU)
- 1 路 CAN Bus
- 4 路数字量输入, 2 路继电器输出
- 可选 4G 蜂窝
- -40 到 70C 工作温度
- DIN 导轨安装

### 8.2 方案选择

选择 Toradex Verdin iMX8M-Plus SoM:

- Cortex-A53 + Cortex-M7(混合架构,可做实时任务)
- 工业级温度范围
- 预认证 WiFi/BLE
- 10 年供货承诺

### 8.3 载板设计

```
+------------------------------------------+
|  [电源: 24V 输入 -> 5V/3.3V]             |
|                                          |
|  [Verdin SoM 连接器]                     |
|     |                                    |
|     +--[RS485 x2]--[SP3485]             |
|     +--[CAN x1]---[SN65HVD230]          |
|     +--[DI x4]---[光耦隔离]              |
|     +--[DO x2]---[继电器驱动]            |
|     +--[4G 模块: SIM7600]               |
|     +--[USB 调试口]                      |
|     +--[Ethernet RJ45]                   |
|                                          |
|  [DIN 导轨安装孔]                        |
+------------------------------------------+
```

载板仅 4 层 PCB,设计周期约 4 周。

### 8.4 开发周期

| 阶段 | SoM 方案 | 全定制方案 |
|------|----------|-----------|
| 选型评估 | 2 周 | 4 周 |
| 载板/主板设计 | 4 周 | 12 周 |
| PCB 打样 | 2 周 | 3 周(高密度) |
| 软件开发 | 与硬件并行 | 硬件完成后开始 |
| 集成测试 | 2 周 | 4 周 |
| 认证 | 2 周(仅载板) | 8 周(整机) |
| **总计** | **14 周** | **35 周** |

## 9 成本分析

### 9.1 单机成本对比

以年产量 5000 台的工业网关为例:

| 成本项 | SoM 方案 | 全定制方案 |
|--------|----------|-----------|
| SoM/主板 | $60 | $25 |
| 载板/底板 | $15 | $0(集成在主板) |
| 连接器 | $5 | $0 |
| 总 BOM | $80 | $25 |
| NRE(研发) | $50K | $200K |
| 认证 | $20K | $80K |

### 9.2 盈亏平衡分析

```
SoM 方案总成本 = $80 * N + $70K
全定制总成本 = $25 * N + $280K

盈亏平衡: N = ($280K - $70K) / ($80 - $25) = 3818 台
```

年产量低于约 4000 台时,SoM 方案总成本更低。超过后,全定制的 BOM 优势开始体现。

## 10 认证优势

### 10.1 SoM 预认证

SoM 上的无线模块通常已通过:

- FCC Part 15(美国)
- CE/RED(欧洲)
- IC(加拿大)
- RCM(澳洲)

这意味着载板只需做:

- 无意辐射测试(Unintentional Radiator)
- 安全测试(如需要)
- EMC 测试(传导/辐射)

### 10.2 节省的时间和费用

| 认证项 | SoM 方案 | 全定制方案 |
|--------|----------|-----------|
| 有意辐射 | 已包含在SoM | 需单独做 |
| 无意辐射 | 需做 | 需做 |
| 认证周期 | 2-4 周 | 6-10 周 |
| 认证费用 | $15K-$30K | $40K-$80K |

## 11 从 SoM 过渡到全定制

### 11.1 过渡策略

1. **MVP 阶段**: 用 SoM 快速验证市场和产品概念
2. **增长阶段**: 产量上升,开始评估全定制的成本节省
3. **成熟阶段**: 产量超过盈亏平衡点,设计全定制方案
4. **保持接口兼容**: 全定制主板的连接器布局尽量与载板兼容

### 11.2 注意事项

- 过渡需要额外的研发投入和时间
- 确保产品形态已经稳定(接口、功能不再频繁变化)
- 全定制设计要预留兼容 SoM 引脚的可能性,以备紧急切换

## 总结

SoM 方案让 IoT 产品团队能以更低成本、更快速度将产品推向市场:

1. SoM 集成了复杂的计算核心,载板只需做应用接口
2. 上市速度可快 2-3 倍,NRE 成本降低 60-70%
3. 射频预认证大幅简化产品认证流程
4. 盈亏平衡点通常在 5K-50K 台,低于此范围 SoM 更划算
5. MVP 阶段用 SoM,量产后可考虑过渡到全定制

对于初创团队和小批量产品,SoM 是降低风险、加速落地的最佳策略。
