# IP防护等级与IoT设备外壳设计

> **难度**：🟢 入门 | **领域**：结构与防护 | **关键词**：IP67, 密封, 呼吸阀 | **阅读时间**：约 15 分钟

## 日常类比

手机防溅不等于潜水表：IP 码像“防尘+防水考卷”的分数，不是营销形容词。户外传感器壳体要在密封、散热、天线净空与电缆进线之间折中；封太死，昼夜温差会把水汽“泵”进腔体[1][3]。

## 摘要

解读 IEC 60529 IP 码、IoT 常见等级、材料与密封件、电缆入口与测试要点。达到某 IP 取决于整机（含接头），单靠灌胶或宣称材质不够[1][2]。

## 1. IP 码与选型

| 第一位（固体） | 含义概要 |
|----------------|----------|
| 5 | 防尘（允许有限进入） |
| 6 | 尘密 |

| 第二位（液体） | 含义概要 |
|----------------|----------|
| 4/5 | 溅水/喷水 |
| 7 | 短时浸水（条件见标准） |
| 8 | 持续浸水（由厂商与用户约定） |

| 场景 | 常见目标 |
|------|----------|
| 室内网关 | IP20–IP40 量级 |
| 户外壁挂 | IP65/IP66 常见 |
| 可能短时浸水 | IP67 并验证接头 |

## 2. 材料、密封与进线

塑料（PC/ABS 等）轻、射频友好但老化与蠕变需评估；压铸铝坚固、散热好，天线常需塑料窗。密封：O 形圈压缩量、平面度、紫外线与耐油；电缆用压盖/灌封接头；膜片呼吸阀平衡压差并挡液态水[3][4]。

| 项目 | 注意 |
|------|------|
| 天线 | 金属壳需透射频区域 |
| 指示灯/按键 | 膜片或磁耦合，避免直通孔 |
| 测试 | 按 IEC 程序；样机含量产密封工艺 |

## 3. 局限、挑战与可改进方向

### 1. IP68 口径不清

**局限**：未约定深度/时长的“IP68”无法复现。
**改进**：图纸与规格书写明试验条件[1]。

### 2. 压差进水

**局限**：日晒雨淋循环把潮气吸入。
**改进**：Gore 类通风窗；避免完全死腔无呼吸[3]。

### 3. 接头拖后腿

**局限**：壳体 IP67，廉价 RJ45 进水。
**改进**：防护等级匹配的连接器与压盖[5]。

### 4. 成本堆砌

**局限**：室内设备上 IP68 浪费 BOM。
**改进**：按场景选最低够用等级并做加速老化[4]。

## 总结

IP 是整机验证结果。先定环境与进线，再选材料与密封，最后按标准测——呼吸阀与接头往往比“加厚壳”更关键。
