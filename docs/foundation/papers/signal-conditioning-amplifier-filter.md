# 信号调理电路：放大/滤波/隔离基础

> **难度**：🟢 初级 | **领域**：模拟信号处理 | **阅读时间**：约 18 分钟

## 引言

想象你去医院体检：护士测到的体温是 36.5 度，但电子体温计内部的热电偶只输出零点几毫伏 -- 这就像用一根极细的吸管去喝一大桶水，口径对不上，根本喝不到。信号调理电路就是那个"漏斗"：把传感器输出的微弱、失真、带噪声的模拟信号，变换成模数转换器 (ADC) 能正确采样的干净信号。

在 IoT 系统中，传感器和 ADC 之间存在三类典型不匹配：

| 不匹配类型 | 传感器侧 | ADC 侧 | 调理手段 |
|-----------|---------|--------|---------|
| 幅度不匹配 | 毫伏/微伏级 | 0-Vref (如 0-3.3V) | 放大 / 衰减 |
| 极性不匹配 | 双极性 (正负) | 单极性 (0-Vref) | 电平移位 |
| 频率不匹配 | 含高频噪声/直流偏移 | 只需信号带宽 | 滤波 |

此外，工业现场还有共模电压、地环路、非线性等问题，需要隔离、缓冲、线性化等额外手段。本文逐一拆解这些基础模块。

## 1 为什么需要信号调理

### 1.1 传感器输出 vs ADC 输入

一个典型 IoT 采集链路：

```
传感器 -> 信号调理 -> ADC -> MCU
```

传感器种类繁多，输出形式各异：

- **热电偶**：K 型在 0-1000 度仅输出 0-41 mV
- **RTD (PT100)**：0 度时 100 欧姆，100 度时 138.5 欧姆 -- 电阻变化需激励电流转电压
- **压力传感器**：惠斯通电桥输出，满量程仅 10-100 mV
- **光电二极管**：输出电流，需跨阻放大转电压

而 ADC 的输入范围通常是 0-3.3V 或 0-5V，分辨率 12-24 位。如果不做调理，微弱信号会被 ADC 量化噪声淹没；双极性信号会被截断；高频噪声会折叠到基带。

### 1.2 不调理的后果

- **信噪比劣化**：1 mV 信号进 12 位 ADC (LSB = 0.8 mV @3.3V)，有效位数只剩 1-2 位
- **极性丢失**：负半周信号被 ADC 当零读，波形只剩一半
- **混叠**：高于奈奎斯特频率的噪声折叠进信号带，数字滤波也救不回来
- **地环路**：传感器远端接地与本地地之间有电位差，直接连 = 共模误差

## 2 放大：把微弱信号拉到 ADC 满量程

### 2.1 运放基础与增益公式

运算放大器 (op-amp) 是信号调理的核心器件。理想运放两条黄金规则：

1. 虚短：正负输入端电压相等 (V+ = V-)
2. 虚断：输入端不取电流

由此推导出两种基本增益配置：

**同相放大器**：输入信号接运放正端，Rf 和 Ri 分压反馈到负端。

增益 G = 1 + Rf/Ri。输入阻抗高，不加载信号源；增益 >= 1。

**反相放大器**：输入信号经 Ri 接运放负端，Rf 反馈从输出到负端。

增益 G = -Rf/Ri。增益可 < 1 (衰减)；输入阻抗 = Ri，需考虑源阻抗；输出反相。

### 2.2 增益选择原则

目标：让最大输入信号对应 ADC 满量程的 80-90%，留一点裕量。

G = (Vref * 0.9) / Vsig_max

举例：K 型热电偶满量程 41 mV，ADC Vref = 3.3V，G = (3.3 * 0.9) / 0.041 = 72.4。选 G = 70，用标准电阻比实现 (如 Rf = 69k, Ri = 1k -> G = 70)。

### 2.3 仪表放大器

对于惠斯通电桥等差分信号，普通运放共模抑制比 (CMRR) 不够。仪表放大器 (In-Amp) 三大优势：

- **高输入阻抗**：不吸取电桥电流
- **高 CMRR**：典型 80-100 dB，抑制共模干扰
- **增益由单一电阻设定**：G = 1 + 49.4k/Rg (AD620)

常见 In-Amp IC：

| 型号 | 增益范围 | CMRR | 供电 | 适用场景 |
|------|---------|------|------|---------|
| AD620 | 1-10000 | 100 dB | +/-2.3~+/-18V | 通用高精度 |
| INA333 | 1-1000 | 100 dB | 单 2.7-5.5V | 低功耗 IoT |
| AD8237 | 2-1000 | 80 dB | 单 2.7-5.5V | 零漂移 |

## 3 电平移位：双极性变单极性

### 3.1 问题场景

很多传感器输出双极性信号：加速度计 (+/-2g)、热电偶 (正/负温度差)、音频。但单电源 ADC 只接受 0-Vref 输入。

### 3.2 偏置电路

核心思路：加一个直流偏置 Vref/2，把信号"抬"到中点。

用电阻分压网络：Vout = Vin/2 + Vref/4 (需后续放大)。

更简洁的做法是用运放加法器：Vout = -(Rf/R1 * Vin + Rf/R2 * Vref/2)。选择 Rf/R1 = -1 (保持增益), Rf/R2 = 1 (加偏置)，即可把 +/-Vs 信号移到 Vref/2 +/- Vs。

### 3.3 注意事项

- 偏置电压必须稳定：用基准电压源分压，不要从电源直接取
- 偏置电阻匹配要 <0.1%，否则引入失调
- 移位后信号动态范围减半：原 +/-5V 变成 0.5-4.5V (仅 4V 范围)

## 4 滤波：只留下想要的频率

### 4.1 为什么滤波

信号调理中滤波的三类用途：

1. **抗混叠滤波** (Anti-Aliasing)：ADC 采样前必须限制信号带宽 < fs/2
2. **噪声抑制**：滤除 50/60 Hz 工频干扰、开关电源纹波
3. **直流阻塞**：去除传感器偏置，只保留交流分量

### 4.2 低通滤波器

最简单的一阶 RC 低通：

一阶 RC 低通：截止频率 fc = 1 / (2*pi*R*C)。R = 1.6k, C = 100nF -> fc = 1 kHz。在 fc 处衰减 -3 dB，每倍频程衰减 -20 dB。

一阶不够陡？用二阶 Sallen-Key 拓扑：运放做跟随器 (增益 = 1)，Q 值由 R1/R2 和 C1/C2 比值决定。Butterworth (最平坦) 二阶需 Q = 0.707，即 C1/C2 = 2 (R1 = R2 时)。

### 4.3 高通滤波器

一阶 RC 高通：截止频率 fc = 1 / (2*pi*R*C)，结构为 C 串联、R 并联接地。典型应用：心电 (ECG) 信号中去除基线漂移，fc = 0.5 Hz。

### 4.4 带通与带阻

- **带通** = 高通 + 低通级联：如语音 300-3400 Hz
- **带阻 (陷波)**：专门抑制 50/60 Hz 工频，用双 T 陷波电路或开关电容滤波器

### 4.5 滤波器选型指南

| 需求 | 推荐方案 | 常用 IC |
|------|---------|---------|
| 简单抗混叠 | 一阶 RC | 无源器件 |
| 陡滚降抗混叠 | 4-8 阶有源 | LTC1562, MAX274 |
| 精确截止频率 | 开关电容 | LTC1060, MF10 |
| 50/60 Hz 陷波 | 双 T + 运放 | 自建或 ADS129x 内置 |
| 数字后处理 | FIR/IIR 软件 | MCU/DSP 实现 |

## 5 隔离：切断地环路，保安全

### 5.1 为什么需要隔离

工业 IoT 中传感器与采集端可能相距几十米，两端的地电位差可达数伏到数十伏。不隔离的后果：

- 地环路电流在信号线上产生压降 -> 测量误差
- 共模电压超运放输入范围 -> 饱和甚至损坏
- 安全风险：高压侧故障可能传导到低压侧

### 5.2 光耦隔离

最传统的隔离方式：LED + 光敏管。

最传统的隔离方式：LED + 光敏管，两侧电气隔离。特点：

- 隔离电压：典型 2.5-5 kV
- 带宽：普通光耦 <100 kHz，高速光耦可达 10 MHz
- CTR (电流传输比)：50-300%，随温度和老化退化
- 适合数字信号隔离、低速模拟信号

### 5.3 数字隔离器

基于变压器或电容耦合的数字隔离器，是光耦的现代替代：

| 器件 | 隔离电压 | 速率 | 通道 | 优势 |
|------|---------|------|------|------|
| ADuM1200 | 2.5 kV | 1 Mbps | 2 | 低功耗 |
| ISO7721 | 5 kV | 100 Mbps | 2 | 高速 |
| Si8620 | 5 kV | 150 Mbps | 2 | 超低功耗 |

数字隔离器优势：无 CTR 退化、速率更高、功耗更低、寿命更长。

### 5.4 隔离放大器

需要隔离模拟信号时，用隔离放大器，内部通过调制/解调 + 变压器/光耦传输：

| 型号 | 隔离电压 | 增益 | 带宽 | 适用 |
|------|---------|------|------|------|
| AMC1100 | 5 kV | 固定 | 60 kHz | 电流采样 |
| AD202 | 2 kV | 1-100 | 2 kHz | 通用 |
| ISO124 | 1.5 kV | 1 | 50 kHz | 高精度 |

### 5.5 隔离方案选择

```
数字信号? --是--> 数字隔离器 (ISO77xx / ADuM)
    |
   否
    |
模拟信号带宽?
    |
  <10 kHz --> 隔离放大器 (AMC1100 / ISO124)
    |
  >10 kHz --> 调制到数字 -> 数字隔离 -> 解调
```

## 6 阻抗匹配与缓冲

### 6.1 为什么需要缓冲

高输出阻抗的传感器 (如光电二极管、压电传感器) 直接接 ADC 或低阻抗负载时，信号会被"拉低" -- 这就像细水管直接接大水桶，水压瞬间跌落。

运放电压跟随器 (增益 = 1) 是最常用的缓冲：

运放电压跟随器 (增益 = 1，输出直连负端反馈)：

- 输入阻抗：运放自身输入阻抗 (FET 输入 > 10^12 欧姆)
- 输出阻抗：接近 0 欧姆
- 作用：阻抗变换，不让后级"拉"前级

### 6.2 多路复用时的缓冲

ADC 前接多路开关 (MUX) 时，切换瞬间采样电容充电需要时间。源阻抗需满足 R_source < (t_acquire - t_switch) / (C_hold * ln(2^(N+1)))，其中 N 为 ADC 位数。12 位 ADC，t_acquire = 1 us，C_hold = 10 pF 时，R_source 需 < 4 k 欧姆。超过就加运放缓冲。

## 7 线性化：把弯曲拉直

### 7.1 非线性问题

许多传感器的输出与被测量不成正比：

- **热敏电阻 (NTC)**：R = R0 * exp(B * (1/T - 1/T0))，指数关系
- **热电偶**：电压-温度非线性，需查表或多项式拟合
- **湿度传感器**：电容与 RH 近似指数关系

### 7.2 硬件线性化方法

1. **对数放大器**：用运放 + 二极管/三极管实现 log 变换，抵消指数特性
2. **分段线性近似**：用多个电阻网络在不同区间切换增益
3. **反馈线性化**：把非线性元件放在运放反馈回路中

### 7.3 软件线性化 (推荐)

IoT 系统中更常用软件方法，在 MCU 中做后处理：

- **查表法**：预先存储 N 个 (输入, 输出) 点，线性插值
- **多项式拟合**：T = a0 + a1*V + a2*V^2 + a3*V^3 (热电偶标准多项式系数由 NIST 提供)
- **Steinhart-Hart 方程**：1/T = a + b*ln(R) + c*(ln(R))^3 (NTC 热敏电阻)

建议：优先用软件线性化，精度更高且无需额外硬件。只在 ADC 位数不够或动态范围太大时才考虑硬件方法。

## 8 激励电路：给无源传感器供电

### 8.1 电阻型传感器的激励

RTD 和应变片是纯电阻器件，本身不产生电信号，需要外部激励 (电流或电压) 将电阻变化转为电压变化。

**恒流源激励 (RTD 推荐)**：

```
I_excite = 1 mA (恒流)
V_rtd = I_excite * R_rtd
```

R 变化线性对应 V 变化，没有引线电阻引起的非线性。

三线制 RTD 用恒流源可消除引线电阻误差：

```
          R_lead1    R_rtd    R_lead2
I+ ----[===]----[RRRR]----[===]---- I-
                     |
                  V_sense (差分测 R_rtd 上压降)
```

**恒压源激励 (应变片/电桥推荐)**：

```
        R3       R1
Vex -[===]-+-[===]- GND
           |        |
          V-        V+
           |        |
Vex -[===]-+-[===]- GND
        R4       R2
```

电桥输出 Vout = Vex * (R1*R3 - R2*R4) / ((R1+R4)*(R2+R3))

### 8.2 激励电流/电压选择原则

| 传感器 | 激励方式 | 典型值 | 约束 |
|--------|---------|--------|------|
| PT100 RTD | 恒流 | 0.5-1 mA | <1 mA 避免自热 |
| PT1000 RTD | 恒流 | 0.1-0.5 mA | 自热更严重 |
| 应变电桥 | 恒压 | 5V 或 10V | 功耗 = V^2/R_bridge |
| 热敏电阻 | 恒压分压 | 3.3V | 串联电阻限流 |

关键：激励源必须稳定，其漂移直接 1:1 转化为测量误差。建议用基准电压源 (如 REF3030) 而非电源直接供电。

## 9 完整信号链路实例：热电偶温度采集

### 9.1 链路框图

K 型热电偶 -> 冷端补偿 (CJC) -> 仪表放大 (G=100) -> 低通滤波 (fc=10Hz) -> 16 位 ADC -> MCU

### 9.2 各级设计

**第一级：冷端补偿 (CJC)**

热电偶测量的是热端与冷端 (接线端子) 的温差。要知道绝对温度，必须测冷端温度并补偿。

方案：用集成 CJC 芯片 (如 AD8495 内置) 或用 NTC/硅二极管测冷端温度，软件查表补偿。

**第二级：仪表放大**

K 型热电偶灵敏度约 41 uV/ 度。假设量程 0-800 度 -> 最大 33 mV。

G = (3.3 * 0.9) / 0.033 = 90。选 INA333，Rg = 49.4k / (G-1) = 555 欧姆。

**第三级：低通滤波**

热电偶信号带宽很窄 (<1 Hz 慢变)，但环境有 50 Hz 工频和开关噪声。选 fc = 10 Hz 的二阶低通：

- R1 = R2 = 15.9 k
- C1 = 2 uF, C2 = 1 uF (Butterworth Q = 0.707)
- 实际用 C1 = 2.2 uF, C2 = 1 uF (标准值)

**第四级：ADC 选型**

| 需求 | 推荐 IC | 分辨率 | 接口 |
|------|---------|--------|------|
| 低功耗 IoT | ADS1115 | 16 位 | I2C |
| 高精度 | ADS124S08 | 24 位 | SPI |
| 内置 CJC | MAX31855 | 14 位 | SPI |
| 内置 PGA | AD7124 | 24 位 | SPI |

### 9.3 常见踩坑

- **冷端补偿忘做**：读数随环境温度漂移，以为是传感器坏了
- **滤波 fc 选太高**：50 Hz 噪声没滤干净，ADC 值跳动
- **增益太大**：正常温度时输出饱和，只能测低温段
- **热电偶引线用普通铜线**：在接线端子处产生额外热电势

## 总结

信号调理是传感器与数字世界之间的"翻译层"，核心任务就三件事：

1. **放大**：把微弱信号拉到 ADC 满量程，提高有效位数
2. **滤波**：只保留有用频段，拒绝噪声和混叠
3. **隔离**：切断地环路，保护设备和人身安全

在此基础上，电平移位解决极性不匹配、缓冲解决阻抗不匹配、激励电路给无源传感器供电、线性化修正非线性响应。

设计信号调理链路时的检查清单：

- [ ] 传感器输出范围是否已知 (最小/最大/典型)
- [ ] ADC 输入范围和分辨率是否确定
- [ ] 增益是否让满量程信号占 ADC 范围 80-90%
- [ ] 双极性信号是否已做电平移位
- [ ] 抗混叠滤波器截止频率 < fs/2
- [ ] 需要隔离吗? 隔离等级选多少?
- [ ] 传感器是否需要激励 (电流/电压)?
- [ ] 传感器是否有非线性需要补偿?
- [ ] 偏置和激励源的稳定性是否足够?
- [ ] 总链路功耗是否满足 IoT 功耗预算?

记住：信号调理做得好，后面的数字处理事半功倍；做得差，再强的算法也救不回被噪声淹没的信号。
