# Sub-GHz无线模组CC1310硬件设计
> **难度**: 中级 | **领域**: Sub-GHz射频设计 | **阅读时间**: 约 20 分钟

## 引言

想象你要在两栋楼之间喊话。用高频尖细的声音(2.4GHz)容易穿透窗户但传不远,而用低沉浑厚的嗓音(Sub-GHz)虽然音调不高,却能绕过障碍传到更远的地方。这就是Sub-GHz频段的核心优势 -- 牺牲带宽换取更远的传输距离和更强的穿透能力。CC1310是德州仪器(TI)推出的一款面向Sub-GHz应用的低功耗无线MCU,广泛用于智能表计、楼宇自动化和远距离传感器网络。

## 1 Sub-GHz频段的优势

### 1.1 传播特性对比

Sub-GHz频段(通常指低于1GHz的频段,如433MHz、868MHz、915MHz)与2.4GHz相比有三个关键物理优势:

1. **自由空间路径损耗更小**: 路径损耗与频率成正比,433MHz比2.4GHz低约15dB
2. **绕射能力更强**: 低频信号波长更长,更容易绕过建筑物和植被
3. **穿透损耗更低**: 在混凝土墙、玻璃等介质中衰减更小

| 参数 | 433MHz | 868MHz | 2.4GHz |
|------|--------|--------|--------|
| 波长 | 69cm | 34cm | 12.5cm |
| 自由空间损耗(1km) | 85dB | 91dB | 100dB |
| 混凝土墙穿透 | ~6dB | ~10dB | ~15dB |

### 1.2 频谱拥挤度

2.4GHz频段拥挤不堪 -- Wi-Fi、蓝牙、Zigbee都在抢同一块频谱。Sub-GHz频段则相对清静,尤其在433MHz的ISM频段,干扰源远少于2.4GHz。这意味着更低的丢包率和更稳定的链路。

## 2 CC1310芯片架构

### 2.1 内核与外设

CC1310是一颗"双核+射频"的三合一芯片:

- **主MCU内核**: ARM Cortex-M3, 48MHz, 128KB Flash, 20KB SRAM
- **Sensor Controller Engine**: 独立低功耗协处理器, 2KB RAM, 可在主MCU休眠时自主采样传感器
- **射频内核**: 专用Radio CPU, 运行TI-RTOS RF驱动, 支持Sub-GHz收发

```
+------------------+
|  ARM Cortex-M3   | <-- 主应用逻辑
|  48MHz / 128KB F |     128KB Flash / 20KB SRAM
+------------------+
| Sensor Controller | <-- 自主采样, uA级功耗
| 2KB SCR RAM      |
+------------------+
|   Radio Core     | <-- RF协议栈
|  + RF Front-End  |     匹配网络 + 天线
+------------------+
```

### 2.2 Sensor Controller Engine详解

Sensor Controller是CC1310的"夜间巡逻兵" -- 主MCU深度睡眠时,它仍能:

- 以微安级电流周期性读取ADC、GPIO、SPI传感器
- 满足阈值条件后唤醒主MCU
- 典型功耗: 1.2uA (ADC采样, 2Hz)

这使CC1310特别适合"睡多醒少"的传感器场景,如门窗磁感、温湿度周期上报。

### 2.3 支持的频段

CC1310覆盖的Sub-GHz ISM频段:

| 频段 | 频率范围 | 主要地区 | 信道数 |
|------|----------|----------|--------|
| 433MHz | 433.05-434.79MHz | 全球(尤其欧洲/中国) | 2 |
| 868MHz | 863-870MHz | 欧洲 | 6 |
| 915MHz | 902-928MHz | 北美 | 26 |

中国常用433MHz和470MHz(电力集抄专用频段)。

## 3 射频前端硬件设计

### 3.1 匹配网络与Balun

CC1310射频输出是差分信号,而大多数天线是单端(50欧姆),需要Balun(平衡-不平衡变换器)完成转换:

```
CC1310 RF_P ----+---- L1 ----+---- ANT (50 ohm)
                |            |
CC1310 RF_N ----+---- L2 ----+
                |            |
               C1           C2
                |            |
               GND          GND
```

设计要点:
- Balun可采用分立LC或集成器件(如Johanson 0402Balun)
- 匹配网络需在SmartRF Studio给出的参考值基础上,结合PCB实际做微调
- 元件选择: C0G/NP0型电容(温度稳定性好), 线艺1%电感

### 3.2 天线选型

| 天线类型 | 增益 | 尺寸 | 适用场景 |
|----------|------|------|----------|
| PCB走线(倒F) | -1~2dBi | 小 | 空间受限,内置天线 |
| 螺旋天线 | 0~3dBi | 中 | 手持设备 |
| 弹簧天线 | 2~5dBi | 中 | 智能表计 |
| 外置棒状 | 3~8dBi | 大 | 网关/集中器 |

433MHz波长69cm,四分之一波长约17cm。PCB走线天线在433MHz下尺寸偏大,通常推荐外置弹簧天线。

### 3.3 链路预算计算

链路预算(Link Budget)决定通信距离:

```
Link Budget = Ptx + Gtx - Lpath - Lobstacles + Grx - Lsystem

其中:
  Ptx      = 发射功率 (dBm), CC1310最大+14dBm
  Gtx/Grx  = 收发天线增益 (dBi)
  Lpath    = 自由空间路径损耗 (dB)
  Lobstacles = 穿透/绕射损耗 (dB)
  Lsystem  = 系统损耗(匹配+馈线) (dB)
```

示例(433MHz, 1km, 发+14dBm, 收-110dBm灵敏度):

| 项目 | 数值 |
|------|------|
| 发射功率 | +14dBm |
| 发天线增益 | +2dBi |
| 路径损耗(1km) | -85dB |
| 穿透损耗(1堵墙) | -6dB |
| 收天线增益 | +2dBi |
| 系统损耗 | -2dB |
| 接收信号 | -75dBm |
| 接收灵敏度 | -110dBm |
| 链路余量 | 35dB |

35dB余量意味着即使多穿2-3堵墙仍可通信,这就是Sub-GHz的距离优势。

## 4 时钟与电源设计

### 4.1 晶振要求

CC1310需要两个时钟源:

- **24MHz高频晶振**: 驱动主MCU和射频, 要求精度+/-40ppm(射频锁定需要)
- **32.768kHz低频晶振**: 驱动RTC和Sensor Controller, 可用内部RC替代但精度差

晶振选型注意:
- 负载电容CL必须与晶振规格匹配: C_load = (C1 * C2)/(C1 + C2) + C_stray
- PCB杂散电容C_stray通常2-5pF, 需实测确认
- 射频通信质量与晶振精度直接相关, 不要在这里省钱

### 4.2 电源设计

CC1310供电范围1.8V-3.8V, 典型3V纽扣电池或两节AA。电源设计要点:

1. VDD和VDDR(射频供电)需各加100nF+1uF去耦电容
2. 射频发射时瞬态电流可达20mA+, 电源内阻要低
3. 纽扣电池(CR2032)内阻高,发射时电压骤降可能导致复位 -- 加10uF钽电容支撑

```
Battery ---> 10uF(钽) ---> VDD
                          |
                      100nF + 1uF (每VDD引脚)
```

## 5 TI-RTOS与SmartRF Studio

### 5.1 TI-RTOS集成

CC1310软件架构基于TI-RTOS(现已开源为SimpleLink SDK):

- **RF Driver**: 抽象射频操作,支持EasyLink API和IEEE 802.15.4g
- **Power Manager**: 自动管理电源域,空闲时进入最低功耗模式
- **Sensor Controller Studio**: 图形化配置Sensor Controller程序,生成C代码嵌入主工程

典型的收发流程:

```c
// 发送
rfPacketTx txPacket;
txPacket.payloadLen = 10;
memcpy(txPacket.payload, data, 10);
RF_runCmd(rfHandle, (RF_Op*)&txPacket, RF_PriorityNormal, NULL, 0);

// 接收
rfPacketRx rxPacket;
RF_runCmd(rfHandle, (RF_Op*)&rxPacket, RF_PriorityNormal, callback, 0);
```

### 5.2 SmartRF Studio

SmartRF Studio是TI的射频参数配置工具:

- 可视化调节频率、功率、数据率、调制方式
- 实时查看寄存器值和链路质量(RSSI)
- 导出设置到代码中的`rfPatch`和`rfSettings`
- 必须先用SmartRF验证链路再写代码, 避免硬件问题被软件掩盖

## 6 与SX1276(LoRa)对比

CC1310和SX1276是Sub-GHz领域两个常见选择,设计理念截然不同:

| 对比项 | CC1310 | SX1276 (LoRa) |
|--------|--------|----------------|
| 调制方式 | FSK/OOK | CSS(啁啾扩频) |
| 最大距离 | ~1km(典型) | ~10km(典型) |
| 数据率 | 0.1-250kbps | 0.018-37.5kbps |
| 集成度 | MCU+RF一体 | 纯射频, 需外挂MCU |
| 功耗(RX) | ~5.9mA | ~10mA |
| 协议栈 | TI-RTOS/私有 | LoRaWAN |
| 灵敏度 | -110dBm(50kbps) | -137dBm(SF12) |
| 成本 | 中 | 中低 |

选择建议:
- 需要远距离(>2km)且能容忍低数据率: SX1276 + LoRaWAN
- 需要较高数据率、低功耗、MCU集成: CC1310
- 智能表计场景: CC1310更常见(数据率需求高、节点密度大)

## 7 PCB设计指南

### 7.1 射频走线

1. RF走线用50欧姆阻抗控制, 参考地紧邻信号线
2. 走线尽量短直, 避免直角拐弯(用45度或圆弧)
3. 匹配网络元件紧贴芯片RF引脚, 元件间距<1mm
4. 天线馈点下方挖空参考地(keepout), 避免镜像电流损耗

### 7.2 布局原则

```
+-----------------------------------+
|  MCU区域    |   RF匹配   | 天线   |
|  (数字噪声) |  (模拟)   | (辐射) |
|              |           |        |
|  晶振靠近芯片 |  地平面完整 | 净空区  |
+-----------------------------------+
```

- 数字电路和模拟电路分区, 不要交叉走线
- 晶振远离RF走线和数字信号
- 地平面尽量完整, 射频区下方不打过孔切割地
- 顶层和底层都铺地, 多打过孔连接两层地

### 7.3 常见PCB错误

1. 匹配网络离芯片太远 -- 寄生参数恶化匹配
2. 天线附近有金属外壳/电池 -- 偏谐失配
3. 地平面被信号走线截断 -- 阻抗不连续
4. 晶振走线旁有高速数字信号 -- 时钟抖动

## 8 法规符合性

### 8.1 主要标准

| 地区 | 频段 | 标准 | 发射限制 | 占空比 |
|------|------|------|----------|--------|
| 欧洲 | 868MHz | ETSI EN 300 220 | +14dBm | <10% |
| 北美 | 915MHz | FCC Part 15 | +30dBm(EIRP) | 无限制 |
| 中国 | 433MHz | GB/T 17618 | +10dBm | 待确认 |

### 8.2 认证流程

1. 预扫描: 用频谱仪检查杂散发射和带外辐射
2. 正式测试: 送认可实验室做完整EMC+RF测试
3. 调试整改: 不通过则调整匹配网络或加滤波器
4. 取证: 拿到CE/FCC/CCC证书后才能量产销售

注意: 即使使用认证过的模组, 整机仍需重新做EMC认证, 因为PCB和外壳会影响射频性能。

## 9 距离测试方法

理论计算只是起点,实际距离测试不可省略:

1. **空旷距离测试**: 在无遮挡环境测最大通信距离, 验证链路预算
2. **穿透测试**: 在不同墙体数量下测试, 记录丢包率随距离变化
3. **RSSI vs 距离曲线**: 每10米记录RSSI, 拟合路径损耗指数
4. **长期稳定性**: 72小时连续通信, 统计丢包率和重传率
5. **多节点干扰**: 多个CC1310同时发射, 观察同频干扰影响

测试工具: SmartRF Studio + CC1310 LaunchPad + 外置天线, 配合自制RSSI记录脚本。

## 总结

Sub-GHz无线通信在物联网远距离、低功耗场景中具有不可替代的优势。CC1310作为MCU+射频+Sensor Controller的三合一方案,大幅降低了硬件复杂度和BOM成本。设计关键在于:

1. 射频匹配网络和Balun是决定通信距离的第一要素
2. 晶振精度直接影响射频锁定和通信质量
3. PCB布局的好坏会"制造"或"消除"3-6dB的链路差异
4. Sensor Controller是实现微安级待机功耗的核心
5. 法规认证不是可选项, 必须从设计之初就纳入考虑
