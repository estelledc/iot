# IoT设备漏电流控制与PCB设计注意事项

> **难度**：🟡 中级 | **领域**：硬件低功耗设计 | **阅读时间**：约 18 分钟

## 引言

想象你的银行卡里有一笔定期存款,年利率 3%。但银行每月悄悄扣一笔"小额账户管理费",每次只有几分钱,一年下来竟吃掉一半利息。更气人的是,这笔费用没写在合同里,你得逐条翻账单才能发现。

IoT 设备的漏电流就是这笔"管理费"。MCU 标称深度睡眠电流 0.5 uA,实测却 15 uA——多出来的 14.5 uA 来自 GPIO 上拉、分压电路、LED、稳压器 Iq、ESD 二极管甚至 PCB 表面污染。CR2032 容量 225 mAh,0.5 uA 能跑 50 年,15 uA 就只剩不到 2 年。漏电流不控制,低功耗设计就是纸上谈兵。

本文系统梳理漏电流来源、nA级测量技巧、PCB 布局防护、超低功耗器件选型,最后用真实调试案例收尾,帮你把睡眠电流压到 sub-uA。

## 1. 漏电流的六大来源

### 1.1 GPIO 上拉/下拉电阻

MCU 数据手册"深度睡眠 0.5 uA"的前提是:所有 GPIO 配置为模拟输入或输出确定电平。实际电路中 GPIO 常外接上拉/下拉电阻,睡眠期间仍从电源抽取电流:

- I2C 总线 4.7k 上拉,两根线各 0.7 mA (3.3V / 4.7k),总计 1.4 mA——是 MCU 睡眠电流的近 3000 倍
- SPI 片选 10k 上拉,0.33 mA
- 未使用 GPIO 默认内部上拉 (约 40k),每脚 82 uA

**对策**: 睡眠时断开上拉供电,或将 GPIO 切换为模拟模式。

### 1.2 电阻分压网络

电池电压监测最常见的漏电大户。分压后接入 ADC:

```
VCC ── R1 (1M) ──┬── R2 (1M) ── GND
                  │
                ADC_IN
```

分压支路电流: I = 3.3V / 2M = 1.65 uA,对 sub-uA 睡眠已是数倍泄漏。

**对策**: MOSFET 在分压顶部做开关 (只在采样时导通);用 MCU 内部 VBAT 通道省掉外部分压。

### 1.3 LED 指示灯

LED 是"沉默的电流杀手"——"熄灭"状态也有漏电流:

| 状态 | 典型电流 | 说明 |
|------|---------|------|
| 正常点亮 | 2-20 mA | 正向导通 |
| 反向偏置 | 10-100 nA | ESD 二极管漏电 |
| GPIO 高阻 + LED 阳极接 VCC | 0.5-5 uA | 内部保护二极管漏电 |

MCU 睡眠后 GPIO 变高阻,LED 通过 ESD 二极管形成到 VDD/GND 的泄漏路径。

**对策**: 睡眠前 LED 驱动脚配置为推挽输出低,或 GPIO 控制 LED 的 VCC 端。

### 1.4 稳压器静态电流 (Iq)

| 器件 | Iq (典型) | 类型 |
|------|----------|------|
| TPS7A02 | 25 nA | 超低 Iq LDO |
| HT7333 | 2 uA | LDO |
| ME6211 | 2.5 uA | LDO |
| RT9013 | 50 uA | 普通 LDO |
| AMS1117 | 5 mA | 普通 LDO (禁用!) |
| TPS62740 | 360 nA | Buck (轻载) |

AMS1117 的 5 mA 直接吃掉 CR2032 全部预算;TPS7A02 的 25 nA 几乎可忽略。

**关键认知**: LDO 轻载效率 = Vout/Vin * Iload/(Iload+Iq),Iload 接近零时 Iq 决定效率。

### 1.5 ESD 保护二极管

ESD 二极管反向漏电流随温度指数增长:

```
25C:   < 1 uA (数据手册标称)
85C:   ~50 uA
125C:  ~500 uA
```

8 路 ESD 保护在 85C 时漏电可达 400 uA。对策:选低漏电器件 (如 TPD4E05U06, < 1 nA);睡眠时 I/O 设中间电平;减少不必要的外部 ESD。

### 1.6 PCB 表面污染

助焊剂残留、潮气在相邻走线间形成微弱导电路径:

```
清洁 PCB:      < 10 nA
助焊剂残留:    100 nA - 10 uA
严重污染+高湿: > 100 uA
```

高阻抗节点 (ADC 输入、分压中点) 附近尤为严重。对策:清洗、三防漆、guard ring。

## 2. nA级电流测量技术

### 2.1 测量方法对比

| 方法 | 量程 | 精度 | 优点 | 缺点 |
|------|------|------|------|------|
| DMM 串联 | 1 uA - 10 mA | 0.1% + 2d | 简单 | 最低只到 uA |
| SMU (Keithley 2450) | 1 nA - 1 A | 0.05% | 最精确 | 昂贵 |
| 采样电阻 + 运放 | 10 nA - 10 mA | 1-5% | 成本低 | 需自制电路 |
| 功耗分析仪 (PPK2) | 100 pA - 1 A | 0.1% | 动态范围大 | 需购买 |

### 2.2 采样电阻法实操

```
VCC ──[Rs]──┬── MCU VDD
             │
          运放 (同相放大) → ADC / 示波器

I = V_sense / Rs,  V_out = V_sense * (1 + Rf/Rg)
```

元件选型: Rs = 100-1000 (1 uA - 10 mA);运放 OPA333 (Ib = 200 pA) 或 AD8603 (Ib = 0.2 pA)。

实操要点:
1. Rs 压降会降低 MCU 供电,需根据量程切换阻值
2. 先短接 Rs 确认零点偏移 < 10 nA
3. 用电池供电,避免开关电源纹波耦合
4. Rs 下方不铺地铜,减少寄生电容噪声

### 2.3 Nordic PPK2 快速测量

PPK2 量程 100 pA - 1 A,采样率 100 kS/s,内置模拟电流源可替代电池。设为 Source 模式输出 3.3V,连接 DUT 端,观察睡眠平均电流。注意:长线缆增加寄生电容,测量瞬态功耗时用最短线缆。

## 3. 负载开关: 外设电源的门控

### 3.1 P-MOSFET 负载开关

```
VCC ──┬── P-MOS (SI2301) ──┬── 外设 VDD
      │     S    G    D     │
      │          │          │
      │        [R 100k]    │
      └──────── GPIO ──────┘

GPIO 高: MOSFET 截止,外设断电
GPIO 低: MOSFET 导通,外设供电
```

### 3.2 选型

| 器件 | Rdson | 关断漏电 | 最大电流 | 控制方式 |
|------|-------|---------|---------|---------|
| SI2301 (P-MOS) | 70 m | < 100 nA | 2.3 A | 低有效 |
| TPS22916 (集成) | 80 m | < 10 nA | 200 mA | 高有效 |
| AP22802 (集成) | 60 m | < 1 nA | 2 A | 高有效 |

选型要点:关断漏电必须低于睡眠预算 10%;大电流外设需低 Rdson;集成开关常带放电电阻和限流保护。

### 3.3 三个陷阱

1. **输出电容倒灌**: 关断后外设去耦电容通过 I/O 口倒灌回 MCU——选带放电功能的开关或加串联电阻
2. **体二极管导通**: P-MOSFET 寄生体二极管在 Vout > Vin 时倒灌——用背靠背 MOSFET 或专用开关
3. **浪涌电流**: 导通瞬间去耦电容充电可能触发限流——加缓启动 (slew rate control)

## 4. GPIO 睡眠配置: 模拟模式是最优解

### 4.1 GPIO 模式与漏电流对比

| GPIO 配置 | 漏电来源 | 典型漏电流 | 说明 |
|-----------|---------|-----------|------|
| 推挽输出 (确定电平) | 外接负载 | 取决于负载 | 无悬浮漏电 |
| 浮空输入 | ESD 二极管 | 100 nA - 50 uA | 最差,禁止! |
| 上拉/下拉输入 | 内部电阻 | 40-80 uA | 避免 |
| 模拟输入 | 无上拉/下拉 | < 5 nA | 最优 |

### 4.2 为什么模拟模式最省电

- 输入施密特触发器旁路 → 消除交叉导通电流
- 内部上拉/下拉断开
- ESD 二极管两端电压接近 VDD/2,均反偏,漏电最小

### 4.3 睡眠前 GPIO 配置代码

```c
// STM32L4 睡眠前 GPIO 配置
void configure_gpio_for_sleep(void) {
    GPIO_InitTypeDef GPIO_InitStruct = {0};
    GPIO_InitStruct.Mode = GPIO_MODE_ANALOG;
    GPIO_InitStruct.Pull = GPIO_NOPULL;

    // 未使用 GPIO 全部设为模拟模式
    GPIO_InitStruct.Pin = GPIO_PIN_0 | GPIO_PIN_1 | GPIO_PIN_2
                        | GPIO_PIN_3 | GPIO_PIN_4 | GPIO_PIN_5;
    HAL_GPIO_Init(GPIOA, &GPIO_InitStruct);

    // LED 引脚: 推挽输出低 (非浮空!)
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_0, GPIO_PIN_RESET);

    // I2C 引脚: 模拟模式 (前提是上拉由负载开关控制)
    GPIO_InitStruct.Pin = GPIO_PIN_6 | GPIO_PIN_7;
    HAL_GPIO_Init(GPIOB, &GPIO_InitStruct);
}
```

注意: 不要将已接外部上拉的 GPIO 设为推挽输出相反电平,这会造成短路。ESP32 用 `rtc_gpio_isolate()`,nRF52 需确认未用引脚非浮空输入。

## 5. PCB 布局注意事项

### 5.1 Guard Ring (保护环)

在敏感走线周围环绕等电位铜皮,使敏感走线与干扰源间无电位差,消除漏电流:

```
  ┌────────────────────┐
  │  GND / Shield      │
  │  ┌──────────────┐  │
  │  │  Guard Ring  │  │
  │  │  ┌────────┐  │  │
  │  │  │ Signal │  │  │
  │  │  └────────┘  │  │
  │  └──────────────┘  │
  └────────────────────┘
  Guard Ring 连接到与 Signal 同电位的低阻抗源
```

适用:ADC 输入、分压中点、电荷放大器输入等高阻抗节点。

### 5.2 三防漆涂覆 (Conformal Coating)

| 涂层类型 | 介电强度 | 耐温 | 修复性 | 典型应用 |
|---------|---------|------|-------|---------|
| 丙烯酸 (AR) | 1500 V/mil | 125C | 易 | 消费电子 |
| 硅树脂 (SR) | 1000 V/mil | 200C | 中 | 高温环境 |
| 聚氨酯 (UR) | 3500 V/mil | 125C | 难 | 工业控制 |

注意:涂覆前必须彻底清洗 PCB;连接器和天线区域需遮蔽;射频电路需考虑介电常数变化。

### 5.3 开槽隔离 (Slot Isolation)

在干扰源和高阻抗节点之间开槽,迫使表面漏电流绕行,路径增加 10x+,表面电阻相应增大 10x+:

```
  干扰源                敏感电路
  ┌──────────┐         ┌──────────┐
  │          │ ┌─────┐ │          │
  │          │ │SLOT │ │          │
  │          │ └─────┘ │          │
  └──────────┘         └──────────┘
  开槽宽度 0.5-1.0mm,贯穿 PCB
```

适用:Buck/Boost 与模拟采样隔离;继电器驱动与高阻抗检测隔离。

## 6. 超低功耗器件选型: LDO vs Buck

### 6.1 Iq 与效率

```
LDO 轻载效率 = (Vout/Vin) * Iload / (Iload + Iq)

Iload = 1 uA, Iq = 25 nA (TPS7A02):  eff = 81.2%
Iload = 1 uA, Iq = 5 mA (AMS1117):   eff = 0.017%
```

### 6.2 场景选择

| 场景 | 推荐 | 原因 |
|------|------|------|
| 电池供电,99% 睡眠 | 超低 Iq LDO | 睡眠时 Iq 主导,Buck 控制电路更耗电 |
| 电池供电,频繁高负载 | 低 Iq Buck | 压差大时 Buck 效率远高于 LDO |
| 稳压电源/USB | 普通 LDO | 压差小,效率可接受 |

### 6.3 推荐器件

超低 Iq LDO: TPS7A02 (25 nA, PSRR 85 dB), MAX38640 (330 nA, 500 mA), HT7333 (2 uA, 国产经典)。

低 Iq Buck: TPS62740 (360 nA, 效率 90%+), MAX38643 (400 nA, 2MHz), nPM1100 (500 nA, 专为 BLE 优化)。

```
选型决策树:
睡眠预算 < 1 uA? → 超低 Iq LDO (TPS7A02)
平均功耗 > 5 mW? → 低 Iq Buck (TPS62740)
否则 → 普通 LDO (HT7333)
```

## 7. 真实调试案例: 寻找隐藏的漏电路径

### 7.1 问题背景

某 LoRa 温湿度传感器,目标 CR2032 电池寿命 3 年 (平均电流 < 8.6 uA)。实测深度睡眠 47 uA,是目标的 5.5 倍。

### 7.2 逐步排查

**第一步: 隔离 MCU**——断开所有外设,实测 0.8 uA,MCU 正常,漏电来自外设。

**第二步: 逐个恢复外设**

```
SHT30:        +0.2 uA  (正常)
SX1276:       +0.1 uA  (正常)
LED:          +1.5 uA  (可疑!)
电池分压:     +1.65 uA (符合预期)
```

LED + 分压贡献 3.15 uA,但 47 - 4.25 = 42.75 uA 仍未找到!

**第三步: 检查 ESD**——8 路 TVS 阵列 (PRTR5V0U2X) 贡献约 2 uA,不是主因。

**第四步: 热成像排查**——LDO (ME6211) 表面温度略高。断开 LDO 输出测输入电流:45 uA! 标称 Iq = 2.5 uA,实测 18 倍。原因:LDO 输出走线经过 LoRa 模块下方,RF 地平面与 LDO 输出形成寄生电容,射频信号耦合到反馈环路导致 LDO 振荡。

**第五步: 修复**

1. LDO 输出走线与 LoRa 模块间加地隔离
2. LDO 输出加 1R + 100pF RC 滤波
3. 重新布局使 LDO 远离射频区

```
修复后实测:
  MCU:       0.8 uA    LED (模拟模式): +0.05 uA
  SHT30:     +0.2 uA   分压 (MOSFET):  +0 uA
  SX1276:    +0.1 uA   TVS (换型号):   +0.5 uA
  LDO (修复):+2.5 uA   ──────────────────────
                        总计: 4.15 uA
```

电池寿命预估约 6.2 年,远超目标。

### 7.3 案例教训

1. **LDO 振荡是隐形杀手**: 振荡时 Iq 暴增 20 倍,万用表只能测平均值
2. **逐个隔离是最高效方法**: 不要猜测,用二分法逐个断开外设
3. **RF 与模拟电路的布局隔离至关重要**: 一根走线位置就能决定系统功耗
4. **热成像是有力工具**: 能发现人手感知不到的异常发热点

## 8. Sub-uA 睡眠电流设计检查清单

### 8.1 器件选型

- [ ] MCU 选超低功耗系列 (STM32L/U, nRF52, MSP430)
- [ ] LDO 的 Iq < 100 nA (如 TPS7A02)
- [ ] 外设均有负载开关或可控电源
- [ ] ESD 器件漏电流 < 10 nA (per channel)
- [ ] 所有电阻分压有 MOSFET 开关

### 8.2 GPIO 配置

- [ ] 未使用 GPIO 设为模拟模式 (非浮空输入!)
- [ ] I2C/SPI 上拉由负载开关控制,睡眠时断开
- [ ] LED 驱动脚设为推挽输出低 (非浮空)
- [ ] 外部中断引脚上下拉状态与睡眠模式兼容
- [ ] 调试接口 (SWD/JTAG) 引脚上拉状态确认

### 8.3 PCB 设计

- [ ] 高阻抗节点有 guard ring
- [ ] LDO 远离 RF 走线,输出走线不穿过射频区
- [ ] 模拟地和数字地单点连接
- [ ] 生产时 PCB 清洗 + 三防漆

### 8.4 验证测试

- [ ] 室温睡眠电流 < 1 uA
- [ ] 高温 (85C) 睡眠电流 < 5 uA
- [ ] 所有 GPIO 逐一恢复浮空,确认无单脚贡献 > 100 nA
- [ ] 长时间 (24h+) 睡眠电流稳定
- [ ] 热成像扫描无异常发热点

## 总结

漏电流控制是 IoT 低功耗设计中最容易被忽视、也最影响电池寿命的环节。核心思路三条:

1. **断电优于省电**: 能用负载开关彻底断电的外设,就不依赖它的低功耗模式
2. **确定态优于悬浮态**: GPIO 睡眠配置核心——每个引脚必须在确定电平上,模拟模式是默认选择
3. **测量优于假设**: 数据手册标称值只是起点,实际 PCB 上的漏电路径需逐一排查

从 50 uA 压到 1 uA 以下,通常需要 3-5 轮迭代:先断开外设确认 MCU 基线,再逐个恢复排查增量,最后用热成像和分步测量定位隐藏路径。每次只改一个变量,做好记录——这是漏电流调试唯一可靠的方法。
