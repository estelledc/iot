# NB-IoT模组硬件集成与AT指令调试

> **难度**：🟡 中级 | **领域**：蜂窝IoT模组 | **阅读时间**：约 20 分钟

## 引言

想象你要在家里装一部固定电话:你得把电话线接好(硬件集成),然后拿起听筒听到拨号音(模组上电),拨号(发AT指令),才能和对方通话(数据传输)。NB-IoT模组就是这个"电话机"--它需要正确的供电和接线才能工作,需要通过AT指令"拨号"才能联网。和LoRa那种"写寄存器就能发"的方式不同,NB-IoT的协议栈完全在模组内部,MCU只通过UART发AT字符串来控制它。这种方式的代价是你看不见底层在干什么,好处是不用自己实现蜂窝协议栈。

## 1. NB-IoT技术定位

### 1.1 3GPP标准演进

NB-IoT(Narrow Band Internet of Things)是3GPP从Release 13开始定义的蜂窝物联网标准,专为低功耗广域(LPWA)场景设计:

| 特性 | Release 13 | Release 14 | Release 15+ |
|------|-----------|-----------|------------|
| 下行峰值速率 | 26.15 kbps | 127 kbps | 127 kbps |
| 上行峰值速率 | 62.5 kbps | 158 kbps | 158 kbps |
| 定位 | 无 | OTDOA | OTDOA增强 |
| 多播 | 不支持 | SC-PTM | SC-PTM |
| TDD | 不支持 | 支持 | 支持 |
| 连接态移动性 | 不支持 | 小区重选 | 小区切换 |

### 1.2 与其他LPWA技术对比

| 参数 | NB-IoT | LoRa | SigFox | eMTC |
|------|--------|------|--------|------|
| 频谱 | 授权频段 | 非授权 | 非授权 | 授权频段 |
| 带宽 | 180kHz | 125-500kHz | 100Hz | 1.4MHz |
| 覆盖增强 | 164dB MCL | 157dB | 160dB | 155dB MCL |
| 峰值速率 | ~100kbps | ~50kbps | 100bps | 1Mbps |
| 功耗(PSM) | 3-5uA | 0.2uA | 1uA | 3-5uA |
| 网络架构 | 运营商蜂窝 | 私有网关 | 运营商核心 | 运营商蜂窝 |

NB-IoT的优势:运营商网络覆盖(无需自建网关)、深度覆盖(地下/室内)、数据可靠性高。代价:需要SIM卡和资费、功耗比LoRa高。

## 3. 硬件集成设计

### 3.1 最小系统框图

```
MCU (UART) <--TX/RX--> NB-IoT模组 <--RF--> 天线
                         |
                    +----+----+
                    |         |
                 SIM卡座   电源管理
```

### 3.2 UART接口

NB-IoT模组与MCU通过UART通信,AT指令为文本协议:

| 参数 | 典型值 | 说明 |
|------|--------|------|
| 波特率 | 9600/115200 | BC26默认9600,可AT+IPR修改 |
| 数据位 | 8 | -- |
| 停止位 | 1 | -- |
| 校验 | 无 | -- |
| 流控 | 无/硬件 | 长数据传输建议启用RTS/CTS |

硬件设计注意:
- TX/RX走线加100欧姆串联电阻限流(热插拔保护)
- 波特率115200时线长不超过30cm(板内通信没问题)
- 建议预留RTS/CTS引脚,高速数据传输时需要流控

### 3.3 电源设计

NB-IoT模组的电源是集成中最容易被忽视的环节。发射瞬间电流可达500mA~700mA(23dBm),如果电源设计不当会导致模组复位:

```
电源要求:
- 电压: 3.3V ~ 4.2V (典型3.6V,锂电池直供最理想)
- 纹波: < 50mV 峰峰值
- 瞬态响应: 500mA/10us 压降 < 200mV
```

电源方案:

| 方案 | 输入 | 优势 | 劣势 |
|------|------|------|------|
| 锂电池直供 | 3.6-4.2V | 最简单,瞬态好 | 需充电管理 |
| LDO(LT3042) | 5V/3.8V | 纹波低 | 发热,效率低 |
| DCDC+LDO | 5V/12V | 兼顾效率和纹波 | BOM多 |

关键:VBAT引脚必须就近放置大容量电容(100uF钽电容+1uF陶瓷电容+100nF陶瓷电容),吸收发射瞬间的电流尖峰。

### 3.4 SIM卡座设计

SIM卡接口遵循ISO 7816标准:

```
模组 SIM_VCC ----[5.1K上拉]----> SIM_VCC
模组 SIM_DATA <-----------------> SIM_DATA
模组 SIM_CLK --------------------> SIM_CLK
模组 SIM_RST --------------------> SIM_RST
                      |
                  [5.1K上拉]
                      |
                    GND
```

设计要点:
- SIM_DATA和SIM_RST必须上拉4.7K~10K欧姆
- 走线越短越好,最好在模组正下方
- ESD保护(TVS管)不可省略
- 优先选Micro SIM或Nano SIM座,节约空间

### 3.5 天线接口

NB-IoT天线工作在900MHz附近(中国Band 8:880-915MHz上行,925-960MHz下行):

| 天线类型 | 增益 | 尺寸 | 适用场景 |
|----------|------|------|---------|
| FPC天线 | 0~2dBi | 40x10mm | 小型设备 |
| PCB天线 | -1~1dBi | 50x15mm | 成本敏感 |
| 外置棒状天线 | 2~5dBi | 80x10mm | 固定安装 |
| 弹簧天线 | 0~1dBi | 30x5mm | 便携设备 |

模组射频输出阻抗50欧姆,PCB走线需做50欧姆阻抗控制,射频走线下方不要铺地。

## 4. 上电时序

### 4.1 典型上电流程

```
时间轴:
0ms    VBAT上电稳定
       |
50ms   拉高PWRKEY(或由模组自动上电)
       |
       +---> 模组内部Bootloader启动
       |
500ms  UART就绪,可发送AT指令
       |
1~3s   AT指令响应正常
       |
5~15s  注册网络(AT+CGATT? 返回1)
       |
       +---> 可开始数据传输
```

### 4.2 PWRKEY控制

多数模组支持PWRKEY引脚开机:

```
开机: PWRKEY拉低 > 500ms 后释放
关机: PWRKEY拉低 > 3s 后释放
复位: PWRKEY拉低 > 15s 强制复位
```

MCU控制电路:

```
MCU_GPIO ---> N-MOSFET栅极
                |
MOSFET漏极 ---> 模组PWRKEY
                |
MOSFET源极 ---> GND
```

## 5. AT指令工作流

### 5.1 基础AT指令

```bash
# 1. 检查模组响应
AT
# 返回: OK

# 2. 查询SIM卡状态
AT+CPIN?
# 返回: +CPIN: READY

# 3. 查询信号质量
AT+CSQ
# 返回: +CSQ: 18,0   (18表示-72dBm,0表示未知误码率)
# CSQ值对照: 0=-113dBm, 2=-109dBm, 10=-93dBm, 20=-73dBm, 31=-51dBm

# 4. 查询注册状态
AT+CGATT?
# 返回: +CGATT: 1   (1=已附着,0=未附着)

# 5. 查询运营商
AT+COPS?
# 返回: +COPS: 0,0,"CHINA MOBILE"
```

### 5.2 网络附着流程

```bash
# 设置全功能模式
AT+CFUN=1

# 设置APN(中国移动OneNET)
AT+CGDCONT=1,"IP","CMNBIOT"

# 自动选择运营商
AT+COPS=0

# 等待附着(可能需要5~30秒)
AT+CGATT?
# 循环查询直到返回 +CGATT: 1
```

### 5.3 UDP数据传输

```bash
# 创建UDP Socket
AT+NSOCR="DGRAM",17,8888,1
# 返回: 0   (Socket ID=0)

# 发送UDP数据(Hex格式)
AT+NSOST=0,"180.101.147.115",8888,4,"48656C6C"
# 发送4字节 "Hell"

# 接收UDP数据(URC主动上报)
# +NSONMI:0,12   (Socket 0收到12字节)

# 读取接收数据
AT+NSORF=0,12
# 返回: 0,"180.101.147.115",8888,12,"48656C6C6F20576F726C6421",0

# 关闭Socket
AT+NSOCL=0
```

### 5.4 CoAP数据传输

部分模组(BC26)内置CoAP栈:

```bash
# 创建CoAP会话
AT+QCOAPCREATE=1

# 发送CoAP GET请求
AT+QCOAPGET=1,"coap://183.230.40.40:5683/sensor/temp"

# 发送CoAP POST请求
AT+QCOAPPOST=1,"coap://183.230.40.40:5683/sensor/data","48656C6C6F"

# 接收响应(URC上报)
# +QCOAPRECV:1,<msg_id>,<payload_len>,<payload>
```

## 6. PSM与eDRX低功耗模式

### 6.1 PSM(Power Saving Mode)

PSM是NB-IoT最核心的低功耗机制,模组在空闲一段时间后进入深度休眠,仅保留定时器:

```
连接态 ---> 空闲态 ---> PSM态
  |          |            |
  |     Active Timer   T3324定时器
  |     (T3324)        (T3412定时器)
  |          |            |
  |          v            v
  |    可接收寻呼    几乎断电(3~5uA)
  |          |            |
  +----------+------------+
              |
        T3412到期/MT数据触发
              |
          重新连接网络
```

PSM配置:

```bash
# 设置PSM: T3324=10s(Active Time), T3412=3600s(周期TAU)
AT+CPSMS=1,"","","00000100","00110001"
# T3324编码: 00000100 = 4*2 = 8秒
# T3412编码: 00110001 = 49*60 = 2940秒

# 查询PSM状态
AT+CPSMS?
```

T3324/T3412编码规则:

```
单位: bit7-6决定
00 = 10秒 * 值
01 = 1分钟 * 值
10 = 6分钟 * 值( deci-hours)
11 = 6分钟 * 值(同10)
```

### 6.2 eDRX(Extended Discontinuous Reception)

eDRX让模组周期性醒来监听寻呼,比PSM延迟低但功耗高:

```bash
# 启用eDRX,PTW=1.28s,eDRX=81.92s
AT+CEDRXS=1,4,"0101"
```

| 参数 | PSM | eDRX | 无低功耗 |
|------|-----|------|---------|
| 平均电流 | 3~5 uA | 50~200 uA | 3~10 mA |
| 下行延迟 | 分钟级 | 秒级 | 毫秒级 |
| 适用场景 | 定时上报 | 需要远程控制 | 实时性要求高 |

## 7. 信号质量与诊断

### 7.1 信号质量评估

```bash
# 基础信号强度
AT+CSQ
# +CSQ: <rssi>,<ber>

# 详细网络统计(BC26)
AT+NUESTATS
# +NUESTATS: SIGNAL,<rsrp>,<sinr>,<rsrq>,<cell_id>
# RSRP: -140 ~ -44 dBm, 优良>-90, 可用>-105, 差<-110

# 小区信息
AT+NCELLINFO
# 返回EARFCN、PCI、RSRP等

# 模组电量统计
AT+QBCEMCFG?
```

### 7.2 信号质量对照

| RSRP(dBm) | 信号等级 | 通信状态 |
|-----------|---------|---------|
| > -80 | 极好 | 数据稳定,速率高 |
| -80 ~ -90 | 好 | 数据稳定 |
| -90 ~ -100 | 一般 | 可通信,速率受限 |
| -100 ~ -110 | 差 | 可能丢包,速率低 |
| < -110 | 极差 | 可能无法附着 |

## 8. FOTA固件升级

### 8.1 FOTA流程

FOTA(Firmware Over-The-Air)通过蜂窝网络升级模组固件:

```
1. 检查当前版本
   AT+CGMR

2. 触发FOTA(BC26示例)
   AT+QFOTADL="http://fota.quectel.com/firmware/BC26/BC26MAR01A05M1_OCPU.bin"

3. 等待下载完成(URC上报)
   +QFOTADL: 0  (成功)

4. 模组自动重启
   RDY

5. 验证新版本
   AT+CGMR
```

### 8.2 FOTA注意事项

- FOTA过程中电流峰值可达700mA,电源设计必须满足
- 升级耗时2~5分钟,期间不可断电
- 建议在信号RSRP > -95dBm时执行
- 升级前确认有足够FLASH空间
- 保留回滚机制(双分区设计)

## 9. PCB天线设计要点(NB-IoT)

### 9.1 900MHz频段天线

NB-IoT工作在Band 8(900MHz),波长约333mm,四分之一波长约83mm。PCB天线设计:

**倒F天线(PIFA)参数(900MHz):**

```
辐射臂长度: 约40mm (取决于PCB介电常数)
短路针到馈点距离: 3~5mm
地平面尺寸: >= 60mm x 30mm
天线高度: 2~4mm (天线到地平面间距)
```

### 9.2 布局建议

- 天线放在PCB短边端部,远离金属外壳
- 天线下方和附近不要铺地(keepout区)
- 电池和LCD屏幕远离天线(至少15mm)
- 射频走线50欧姆阻抗,尽量短直
- 匹配网络预留PI型(3个0402焊盘)

## 10. 常见集成问题

### 10.1 问题排查表

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| AT无响应 | UART波特率不对/接线错 | 示波器看TX波形 |
| AT返回ERROR | SIM卡未插好/欠费 | AT+CPIN?检查 |
| 不附着网络 | 天线未接/频段不对 | AT+CSQ查信号 |
| 频繁掉网 | 电源纹波大/信号弱 | 示波器看VBAT |
| 发送失败 | APN错误/未建Socket | AT+CGDCONT?检查 |
| 模组重启 | 电源容量不足 | 看VBAT在TX时是否跌落 |
| PSM无法进入 | 网络不支持/定时器配置错 | AT+CPSMS?检查 |
| FOTA失败 | 信号差/空间不足 | RSRP > -95dBm |

### 10.2 功耗测量方法

```bash
# 1. 进入PSM后测量
AT+CFUN=0  # 最小功能模式
# 用万用表或功耗分析仪测量VBAT电流

# 2. 典型功耗曲线(BC26)
# 附网: ~50mA x 5s
# 数据发送: ~200mA x 2s (峰值500mA)
# 空闲: ~3mA
# PSM: ~5uA
# eDRX监听: ~30mA x 20ms per cycle
```

功耗估算示例(每小时发1次,PSM模式):

```
发送: 200mA x 2s = 400mAs
附网: 50mA x 5s = 250mAs
PSM: 5uA x 3593s = 18mAs
总计: 668mAs / 3600s = 0.186mAh/小时
日耗电: 0.186 x 24 = 4.46mAh
3000mAh电池理论寿命: 3000/4.46 = 672天
```

## 11. 功耗优化策略

### 11.1 软件优化

- 合理设置T3412(周期TAU),减少重连次数
- 合并上报数据,减少发送次数
- 使用CoAP而非HTTP(开销更小)
- 发送完立即进入PSM,缩短Active Time
- 关闭不必要的URC上报

### 11.2 硬件优化

- 选用低静态电流LDO(Iq < 1uA)
- VBAT端大电容(100uF)减少重连时压降
- 模组电源独立供电,不与其他大电流负载共享
- PSM期间关闭MCU外设,MCU也进入低功耗

## 总结

NB-IoT模组集成的核心是三件事:电源稳(UART通信才不会乱)、时序对(上电和PWRKEY要按规格来)、AT指令熟(配网传数全靠它)。和LoRa直接操作寄存器不同,NB-IoT的协议栈完全封闭在模组内部,MCU只能通过AT指令"遥控"。这种黑盒方式降低了开发难度,但也意味着调试手段有限--当网络侧出问题时,你只能看RSRP和附着状态,无法深入协议栈。

关键要点:
- 电源设计是NB-IoT集成的头号坑,发射瞬间500mA+必须有大电容兜住
- AT指令是MCU与模组的唯一接口,必须处理URC异步上报
- PSM是NB-IoT低功耗的核心,T3412决定休眠深度,T3324决定醒多久
- 信号质量看RSRP,低于-110dBm基本不可用
- FOTA升级必须确保电源稳定,断电可能变砖
