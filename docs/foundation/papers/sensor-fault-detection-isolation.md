# 传感器故障检测与隔离FDI方法
> **难度**：🔴 高级 | **领域**：故障诊断 | **阅读时间**：约 22 分钟

## 引言

想象一栋大楼里有三个烟感器。如果只有一个报警, 你会怀疑是误报; 如果三个同时报警, 你会立刻撤离。但如果其中一个坏了, 始终不报警, 你怎么知道它坏了? 传感器故障检测与隔离(FDI)就是解决这个问题的系统方法——先发现"有人生病了"(检测), 再确定"谁生病了"(隔离), 最后决定"怎么处理"(重构)。

## 1. FDI在IoT中的重要性

### 1.1 无人值守的挑战

IoT设备通常部署在无人值守的环境中:

- 偏远地区: 农业传感器、气象站, 人工巡检成本极高
- 危险环境: 化工厂、矿井, 人员无法随时进入
- 大规模部署: 智慧城市数千个节点, 不可能逐个检查

传感器故障如果不及时发现:

- 数据失真导致错误决策(如灌溉系统因故障温度传感器误开)
- 安全风险(如气体泄漏未被检测到)
- 数据污染扩散到云端, 影响模型训练

### 1.2 FDI的三个层次

1. **故障检测(Fault Detection)**: 系统是否存在异常?
2. **故障隔离(Fault Isolation)**: 哪个传感器出了问题?
3. **故障重构(Fault Reconfiguration)**: 系统如何继续运行?

检测是第一步, 隔离是核心难点, 重构是最终目标。

## 2. 传感器故障类型

### 2.1 偏差故障(Bias)

测量值系统性偏离真值一个常数:

```
z_fault(t) = z_true(t) + b
```

- 成因: 传感器零点漂移、校准失效
- 特征: 均值偏移, 方差不变
- 检测难度: 中等(缓慢变化的偏差容易被误认为正常变化)

### 2.2 漂移故障(Drift)

偏差随时间单调变化:

```
z_fault(t) = z_true(t) + d*t
```

- 成因: 传感器老化、电池电压下降
- 特征: 均值逐渐偏移
- 检测难度: 高(短期看像正常趋势, 长期才暴露)

### 2.3 卡死故障(Stuck-at)

输出固定在某个值不变:

```
z_fault(t) = c (常数)
```

- 成因: ADC卡死、通信中断、软件死锁
- 特征: 方差趋近零
- 检测难度: 低(最容易检测的故障类型)

### 2.4 噪声增大故障(Noise Increase)

测量噪声突然增大:

```
z_fault(t) = z_true(t) + v_large(t)
```

- 成因: 电磁干扰、电源纹波增大、连接松动
- 特征: 均值不变, 方差显著增大
- 检测难度: 中等(需统计检验)

### 2.5 完全失效(Complete Failure)

传感器无输出或输出随机值:

```
z_fault(t) = random(t) 或 NaN
```

- 成因: 硬件损坏、通信链路断开
- 特征: 与物理过程完全无关
- 检测难度: 低(最明显, 但需快速检测以切换备用)

### 2.6 故障类型总结

| 故障类型 | 均值变化 | 方差变化 | 检测难度 | 典型原因 |
|----------|----------|----------|----------|----------|
| 偏差 | 是 | 否 | 中 | 零点漂移 |
| 漂移 | 渐变 | 否 | 高 | 老化 |
| 卡死 | 是 | 减小 | 低 | ADC死锁 |
| 噪声增大 | 否 | 增大 | 中 | 电磁干扰 |
| 完全失效 | 是 | 变化大 | 低 | 硬件损坏 |

## 3. 基于模型的FDI

### 3.1 观测器与残差生成

基于模型的方法需要一个系统数学模型。通过比较模型预测与实际测量, 生成残差(Residual):

```
r(t) = z(t) - H * x_model(t)
```

正常情况下, 残差应该接近零(仅含测量噪声)。故障发生时, 残差会显著偏离零。

常用的观测器类型:

- **Luenberger观测器**: 简单, 适合线性系统
- **卡尔曼滤波器**: 自然产生新息作为残差
- **滑模观测器**: 对模型不确定性鲁棒, 适合非线性系统

### 3.2 阈值逻辑

残差生成后需要判断是否超出阈值:

**固定阈值法**:

```
故障标志 = |r(t)| > threshold
```

简单但需要手动设定阈值, 对工作点变化敏感。

**自适应阈值法**:

```
threshold(t) = k * sigma_r(t)
```

其中sigma_r(t)是残差的标准差, 根据当前工作条件动态调整。k通常取2-3(对应95%-99.7%置信区间)。

### 3.3 优缺点

优点:

- 物理意义明确, 残差直接反映模型-测量偏差
- 可同时估计故障幅值
- 适合有良好数学模型的系统

缺点:

- 依赖模型精度, 模型失配会导致误报
- 需要知道系统动力学方程
- 对于复杂系统, 建模困难

## 4. 基于信号的FDI

### 4.1 统计过程控制

不需要系统模型, 仅基于传感器输出信号进行统计检验:

**Shewhart控制图**:

```
UCL = mu + 3*sigma  (上控制限)
LCL = mu - 3*sigma  (下控制限)
故障 = 测量值超出[UCL, LCL]
```

适用于检测突发性故障(偏差、卡死), 对渐变故障(漂移)不敏感。

### 4.2 CUSUM(累积和)检验

CUSUM对微小但持续的漂移最敏感:

```
S_high(k) = max(0, S_high(k-1) + (z(k) - mu - delta/2))
S_low(k)  = max(0, S_low(k-1)  - (z(k) - mu + delta/2))
故障 = S_high(k) > h 或 S_low(k) > h
```

- delta: 要检测的最小偏移量
- h: 决策阈值, h越大, 误报越少但检测延迟越大

CUSUM的核心思想: 把微小偏差累加起来, 即使每次偏差很小, 累积后也能超过阈值。

### 4.3 EWMA(指数加权移动平均)

```
y(k) = alpha * z(k) + (1 - alpha) * y(k-1)
```

- alpha接近1: 对当前数据敏感, 检测快但误报多
- alpha接近0: 对历史数据平滑, 检测慢但误报少

EWMA相当于低通滤波, 适合检测均值漂移。与CUSUM相比, EWMA对历史数据的影响按指数衰减, 而CUSUM持续累加。

### 4.4 方法选择指南

| 方法 | 擅长检测 | 检测速度 | 计算量 | 实现难度 |
|------|----------|----------|--------|----------|
| Shewhart | 突发偏差 | 快 | 极低 | 极简单 |
| CUSUM | 渐变漂移 | 中 | 低 | 简单 |
| EWMA | 缓慢均值偏移 | 中 | 极低 | 极简单 |

## 5. 基于知识的FDI

### 5.1 规则库专家系统

将领域专家的经验编码为if-then规则:

```
IF temperature > 80 AND humidity < 20 AND sensor_id = T3
THEN fault_type = drift, confidence = 0.8

IF variance(z_T1) < 0.01 AND change_rate(z_T1) == 0
THEN fault_type = stuck_at, confidence = 0.95
```

优点: 直观可解释, 融合领域知识; 缺点: 规则库难维护, 不易覆盖所有故障模式。

### 5.2 神经网络分类器

用历史数据(含标签的正常/故障样本)训练分类器:

```
输入: 传感器时序特征(均值、方差、变化率、频谱特征)
输出: 故障类型分类(正常/偏差/漂移/卡死/噪声增大)
```

网络结构选择:

- MLP: 适合手工提取的特征向量
- 1D-CNN: 适合原始时序数据, 自动提取特征
- LSTM: 适合需要长期依赖的漂移检测

训练数据要求:

- 每种故障类型至少数百个样本
- 样本需覆盖不同工作条件(温度、湿度范围)
- 数据不平衡时需过采样或加权损失函数

### 5.3 混合方案

实际部署常采用混合方案:

1. 第一层: 统计检验(Shewhart/CUSUM)做快速初筛
2. 第二层: 规则库处理常见故障模式
3. 第三层: 神经网络处理复杂/未知模式

## 6. 硬件冗余

### 6.1 三模冗余(TMR)

最经典的硬件冗余方案——三个相同传感器测量同一物理量, 通过投票逻辑决定输出:

```
投票逻辑(多数表决):
z_out = median(z1, z2, z3)

故障检测:
if |z1 - z2| > threshold AND |z1 - z3| > threshold:
    sensor1 is faulty
elif |z2 - z1| > threshold AND |z2 - z3| > threshold:
    sensor2 is faulty
elif |z3 - z1| > threshold AND |z3 - z2| > threshold:
    sensor3 is faulty
```

TMR的优势:

- 原理简单, 实现容易
- 可检测和隔离任意单个故障
- 中值滤波天然抗野值

TMR的代价:

- 传感器成本x3
- 安装空间x3
- 功耗x3

### 6.2 N模冗余(NMR)

传感器数量N > 3时:

- 可容忍(N-1)/2个同时故障
- 可靠性更高但成本线性增长
- 适合安全关键应用(航空、核电)

## 7. 解析冗余

### 7.1 虚拟传感器

利用物理量之间的已知关系, 从其他传感器的测量推算出"虚拟"测量值:

```
示例: 气象站中的温度-湿度-气压关系
- 饱和蒸汽压 es = 6.1078 * exp(17.27 * T / (T + 237.3))
- 相对湿度 RH = e / es * 100%

如果温度传感器故障, 可以从湿球温度和气压反推:
T_virtual = f(T_wet, P)
```

解析冗余的优势: 不需要额外硬件, 利用已有信息; 劣势: 依赖物理关系的准确性, 适用范围有限。

### 7.2 相关变量冗余

不同传感器测量不同但相关的物理量时:

- 同一房间多个温度传感器, 读数应接近
- 压力和海拔有确定性关系
- 电流和功率在已知电压下相关

通过检查这些一致性关系, 可以在不增加硬件的情况下实现故障检测。

## 8. 故障隔离

### 8.1 结构化残差

设计多个残差, 每个残差对不同的故障敏感:

```
r1 = z1 - model1(x)    // 对传感器1故障敏感
r2 = z2 - model2(x)    // 对传感器2故障敏感
r1 AND r2 同时异常      // 执行器故障
```

### 8.2 诊断矩阵

将残差与故障的对应关系编码为诊断矩阵:

| 故障 | r1 | r2 | r3 | r4 |
|------|----|----|----|----|
| 传感器1偏差 | 1 | 0 | 0 | 1 |
| 传感器2漂移 | 0 | 1 | 1 | 0 |
| 传感器3卡死 | 1 | 1 | 0 | 0 |
| 执行器故障 | 1 | 1 | 1 | 1 |

(1=残差对故障敏感, 0=不敏感)

当实际残差模式为[1,0,0,1]时, 对照诊断矩阵即可定位为传感器1偏差。

### 8.3 隔离的必要条件

诊断矩阵中每行的模式必须不同, 否则无法区分对应故障。如果两行相同, 需要增加残差数量来提高可区分性。

## 9. 故障后重构

### 9.1 传感器排除

最直接的重构策略——将故障传感器从融合算法中移除:

```
if sensor_i is faulty:
    exclude sensor_i from measurement update
    increase Q to reflect reduced observability
```

排除后系统性能会下降, 但至少不会引入错误数据。

### 9.2 降级运行

故障传感器排除后, 系统进入降级模式:

| 正常模式 | 降级模式 |
|----------|----------|
| 3传感器TMR | 2传感器取平均 |
| GPS+IMU融合 | 仅IMU推算(短期) |
| 全状态估计 | 部分状态估计 |

降级模式需要明确定义:

- 可接受的性能下限
- 降级模式的持续时间限制
- 恢复条件(故障传感器是否可自动恢复)

### 9.3 虚拟传感器替代

当故障传感器被排除后, 用解析冗余生成的虚拟传感器替代:

```
z_virtual = f(z_remaining_sensors)
用z_virtual替代z_faulty参与融合
```

虚拟传感器的精度通常低于真实传感器, 但比完全丢失该测量维度要好。

## 10. 实践案例: 多传感器气象站

### 10.1 系统配置

气象站配备3个温度传感器:

- T1: PT100 RTD, 高精度(0.1度), 响应慢
- T2: NTC热敏电阻, 中精度(0.3度), 响应快
- T3: SHT31数字传感器, 中精度(0.3度), 含湿度

### 10.2 FDI策略

```c
// Fault detection: CUSUM for drift, Shewhart for bias/stuck
typedef struct {
    float cusum_high;
    float cusum_low;
    float mean;
    float sigma;
    float delta;   // minimum detectable shift
    float h;       // decision threshold
    int fault_flag;
} FDI_CUSUM;

void fdi_cusum_update(FDI_CUSUM* c, float z) {
    float diff = z - c->mean;
    c->cusum_high = fmaxf(0, c->cusum_high + diff - c->delta / 2.0f);
    c->cusum_low  = fmaxf(0, c->cusum_low  - diff + c->delta / 2.0f);

    if (c->cusum_high > c->h || c->cusum_low > c->h) {
        c->fault_flag = 1;
    }
}

// Fault isolation: pairwise comparison
int fdi_isolate(float t1, float t2, float t3, float threshold) {
    int d12 = fabsf(t1 - t2) > threshold;
    int d13 = fabsf(t1 - t3) > threshold;
    int d23 = fabsf(t2 - t3) > threshold;

    // T1 disagrees with both others -> T1 faulty
    if (d12 && d13 && !d23) return 1;
    // T2 disagrees with both others -> T2 faulty
    if (d12 && d23 && !d13) return 2;
    // T3 disagrees with both others -> T3 faulty
    if (d13 && d23 && !d12) return 3;
    // All disagree or all agree -> no isolation possible
    return 0;
}
```

### 10.3 故障处理流程

```
1. 每10秒执行一次FDI检查:
   a. Shewhart: |T_i - T_median| > 3*sigma?
   b. CUSUM: 累积和超过阈值?
   c. 一致性: 三个传感器两两差异是否在合理范围?

2. 检测到故障 -> 执行隔离:
   a. 三选二投票定位故障传感器
   b. 检查故障类型(卡死/漂移/偏差)

3. 隔离后重构:
   a. 排除故障传感器
   b. 剩余两个取平均
   c. 标记数据质量降级
```

## 11. 资源受限MCU的实现考量

### 11.1 内存占用

```
3传感器CUSUM FDI:
- 3个CUSUM结构体: 3 * 28字节 = 84字节
- 诊断矩阵: 3*3*1字节 = 9字节
- 历史缓冲(滑动窗口): 3 * 32 * 4字节 = 384字节
总计: ~480字节, Cortex-M0可承受
```

### 11.2 计算量控制

- CUSUM: 每个采样点2次加法+2次比较, 极轻量
- 统计量(均值/方差): 用Welford在线算法, 无需存储全部数据
- 神经网络: 在MCU上只做推理, 用量化INT8模型, 控制在10KB以内

### 11.3 实时性要求

- FDI检测周期: 通常1-10秒(远低于采样率)
- 故障隔离: 检测到后立即执行(毫秒级)
- 重构: 隔离后一个采样周期内完成

## 12. 误报率与检测延迟的权衡

### 12.1 ROC曲线概念

接收者操作特性(ROC)曲线描述检测性能:

- 横轴: 虚警率(P(报警|无故障)), 越低越好
- 纵轴: 检测率(P(报警|有故障)), 越高越好
- 理想点: (0, 1), 虚警为零且检测率为1
- 对角线: 随机猜测

### 12.2 阈值与权衡

阈值是调节误报率和检测延迟的核心旋钮:

- 阈值降低: 检测更快但虚警更多
- 阈值升高: 虚警更少但检测延迟增大

对于CUSUM, 阈值h的调节:

| h值 | 虚警率 | 检测延迟 | 适用场景 |
|-----|--------|----------|----------|
| 3sigma | 高 | 短(<5采样) | 安全优先 |
| 5sigma | 中 | 中(5-20采样) | 均衡 |
| 8sigma | 低 | 长(>20采样) | 避免频繁误操作 |

### 12.3 实际权衡策略

1. 根据故障后果严重性确定可接受的虚警率
2. 在此虚警率约束下, 选择使检测延迟最小的阈值
3. 在线自适应: 正常运行时用严格阈值, 检测到可疑信号后临时降低阈值
4. 多级检测: 初筛用宽松阈值(低漏检率), 确认用严格阈值(低虚警率)

## 总结

传感器FDI是IoT系统可靠运行的保障。核心框架:

1. **故障检测**: 模型法(残差)、信号法(CUSUM/EWMA)、知识法(规则/神经网络)
2. **故障隔离**: 结构化残差 + 诊断矩阵, 确保故障模式可区分
3. **故障重构**: 传感器排除 + 降级运行 + 虚拟传感器替代

实践建议:

- 从最简单的开始: Shewhart检测 + TMR投票隔离
- 渐变故障用CUSUM, 突发故障用Shewhart
- 资源受限时优先用统计方法, 避免神经网络
- 始终考虑误报率与检测延迟的权衡, 不要只追求其中一个
