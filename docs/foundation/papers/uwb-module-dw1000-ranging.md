# UWB模组DW1000测距定位硬件架构
> **难度**: 高级 | **领域**: UWB精密定位 | **阅读时间**: 约 22 分钟

## 引言

想象你站在一个漆黑的仓库里,想知道自己离出口有多远。你可以大声喊并计时回声 -- 但普通声音太慢太模糊。UWB(超宽带)就像是用一道闪电代替喊声:发出一个极短脉冲(纳秒级),精确记录飞行时间,再用光速换算距离。因为脉冲极短,时间分辨率可达皮秒级,定位精度因此达到厘米量级。DW1000是Decawave(现Qorvo)推出的经典UWB测距芯片,是工业室内定位的基石器件。

## 1 UWB技术概述

### 1.1 IEEE 802.15.4z标准

UWB的标准化经历了三个阶段:

- **802.15.4a**: 2007年,定义UWB物理层基础
- **802.15.4-2011**: 合并修订,增加HPRF(高脉冲重复频率)模式
- **802.15.4z-2020**: 增强安全测距(Secure Ranging), 是Apple U1和最新芯片的基础

4z标准的关键新增:
- **STS(Scrambled Timestamp Sequence)**: 加密时间戳序列, 防止中继攻击
- **HRP(High Rate Pulse)** 和 **LRP(Low Rate Pulse)** 两种模式
- 明确定义了测距帧(Ranging Frame)格式

### 1.2 UWB信号特征

UWB与传统窄带信号的本质区别:

| 特征 | UWB | 窄带(如蓝牙) |
|------|-----|-------------|
| 带宽 | >= 500MHz | ~1MHz |
| 脉冲宽度 | 纳秒级 | 微秒级 |
| 功率谱密度 | -41.3dBm/MHz | 集中在载波 |
| 时间分辨率 | ~65ps(DW1000) | 微秒级 |
| 定位精度 | 10cm级 | 米级 |
| 多径分辨 | 极强(脉冲短) | 弱 |

UWB功率极低(-41.3dBm/MHz),相当于"悄声说话但每个音节都精确",不会干扰其他无线电。

## 2 DW1000芯片架构

### 2.1 内部框图

```
+--------------------------------------------------+
|  DW1000                                           |
|  +----------+    +-----------+    +----------+   |
|  | SPI接口  |--->| 数字基带  |--->| 模拟前端 |   |
|  | (host)   |<---| (MAC/PHY)|<---| (TX/RX)  |   |
|  +----------+    +-----------+    +----------+   |
|       |               |                |          |
|  +----------+    +-----------+   +----------+    |
|  | 寄存器组 |    |  晶振     |   | 天线     |    |
|  | (ctrl)  |    | 38.4MHz   |   | (50ohm) |    |
|  +----------+    +-----------+   +----------+    |
+--------------------------------------------------+
```

关键模块:
- **模拟前端**: 发射PA、接收LNA、混频器、ADC/DAC
- **数字基带**: 脉冲成型、相关器、时间戳计数器(64bit, 15.65ps分辨率)
- **SPI接口**: 与外部MCU通信, 最高20MHz
- **晶振**: 38.4MHz, 是测距精度的基础

### 2.2 与DW3000对比

| 特性 | DW1000 | DW3000 |
|------|--------|--------|
| 标准 | 802.15.4-2011 | 802.15.4z |
| 信道 | 1-7 (3.5-6.5GHz) | 5,9 (6.5/8.0GHz) |
| STS安全测距 | 不支持 | 支持 |
| 功耗(TX) | ~110mW | ~70mW |
| SPI最高 | 20MHz | 36MHz |
| 封装 | QFN-48 | QFN-40 |
| Apple兼容 | 否 | 是(UWB芯片组兼容) |

DW3000是DW1000的升级版,功耗更低且支持4z安全测距,但DW1000因成熟度和成本优势仍有大量存量应用。

## 3 测距原理

### 3.1 双向测距(TWR)

TWR是最基本的UWB测距方法,分三步:

```
Tag                    Anchor
 |--- Poll ----------->|   t1: Tag发Poll, 记录发送时间
 |                     |
 |<--- Response -------|   t2: Anchor收到Poll, 延迟Treply1后回Response
 |                     |
 |--- Final ---------->|   t3: Tag收到Response, 延迟Treply2后发Final
 |                     |
     Anchor计算距离     |   t4: Anchor收到Final, 计算ToF
```

飞行时间(ToF)计算:

```
Tround1 = t3 - t1   (Tag侧总往返时间)
Tround2 = t4 - t2   (Anchor侧总往返时间)
Treply1 = t2 - t1_rx  (Anchor处理延迟)
Treply2 = t3 - t2_rx  (Tag处理延迟)

ToF = (Tround1 * Round2 - Treply1 * Treply2) / (Tround1 + Round2 + Treply1 + Treply2)
```

简化(对称Treply):

```
ToF = (Tround1 - Treply1 - Treply2) / 2
距离 = ToF * c   (c = 3e8 m/s)
```

1ns的ToF对应约30cm的距离,所以皮秒级时间精度才够。

### 3.2 到达时间差(TDoA)

TDoA需要多个时钟同步的Anchor:

```
         Tag (未知位置)
          |
    ------+------
    |     |     |
  A1     A2    A3   (时钟同步的Anchor)
```

1. Tag发射一个Blink脉冲
2. 多个Anchor各自记录到达时间 t1, t2, t3
3. 时间差: dt12 = t2 - t1, dt13 = t3 - t1
4. 每个时间差确定一条双曲线
5. 双曲线交点即为Tag位置

TDoA优势: Tag只需发射一次,功耗极低; 劣势: Anchor需要纳秒级时钟同步,硬件和部署复杂度高。

### 3.3 精度因素分析

| 因素 | 影响量 | 缓解方法 |
|------|--------|----------|
| 晶振频偏 | 10-20ppm -> 数厘米误差 | 天线延迟校准, 晶振温补 |
| 多径效应 | NLOS下误差>50cm | 信道选择, CIR分析检测NLOS |
| 天线延迟 | 未校准可致10cm+偏移 | 校准流程测固定偏移 |
| 温度漂移 | 晶振随温度变化 | 上电后预热, 周期重校 |
| 时钟漂移 | Treply越长误差越大 | 缩短Treply, 用DS-TWR(双边) |

## 4 信道与射频配置

### 4.1 UWB信道分配

| 信道 | 中心频率 | 带宽 | 可用地区 | 推荐用途 |
|------|----------|------|----------|----------|
| 1 | 3494.4MHz | 499.2MHz | 全球 | 远距离(绕射好) |
| 2 | 3993.6MHz | 499.2MHz | 全球 | 通用 |
| 3 | 4492.8MHz | 499.2MHz | 全球 | 通用 |
| 4 | 3993.6MHz | 1331.2MHz | 全球 | 超宽带 |
| 5 | 6489.6MHz | 499.2MHz | 全球 | 高频(衰减大) |
| 7 | 6489.6MHz | 1081.6MHz | 全球 | 高频超宽带 |

信道选择考量:
- 低信道(1-3)穿墙能力更好
- 高信道(5,7)与Wi-Fi 6GHz频段有潜在冲突
- 信道2和5最常用

### 4.2 前导码与SFD

DW1000每次发送帧包含:

1. **Preamble(前导码)**: 128/256/512/1024/2048个符号, 接收端用于同步和信道估计
2. **SFD(Start of Frame Delimiter)**: 标记有效数据开始, 有标准SFD和Decawave SFD两种
3. **PHR(Physical Header)**: 数据率和帧长度
4. **Data**: 有效载荷

前导码长度选择:
- 长前导(1024+): 远距离、高多径环境, 但耗时长
- 短前导(128/256): 近距离、低多径, 响应快
- 测距推荐: 128或256, 兼顾精度和速度

## 5 硬件设计要点

### 5.1 天线设计

UWB天线与窄带天线设计理念完全不同:

- 窄带天线: 只需在一个频率共振
- UWB天线: 需在500MHz+带宽内保持一致增益和相位

常见UWB天线类型:
1. **单极天线**: 最简单, 带宽足够, 但需要地平面
2. **偶极天线**: 不需要地平面, 适合标签
3. **贴片天线**: 方向性强, 适合固定Anchor

关键指标:
- 在信道频带内增益波动 < 3dB
- 群延迟(group delay)尽量平坦 -- 否则不同频率成分延迟不同, 脉冲变形
- 天线延迟(antenna delay)需精确校准, 通常10-500ps量级

### 5.2 晶振与定时精度

DW1000的38.4MHz晶振是测距精度的根基:

- **初始精度**: 推荐+/-10ppm或更好
- **温度稳定性**: TCXO(温补晶振)可到+/-0.5ppm, 普通XO约+/-20ppm
- **老化**: 每年1-3ppm, 长期运行需周期校准

```
晶振频偏对测距的影响:
  d = ToF * c
  误差 = d * (ppm偏差) * (Treply / Tround)

  示例: d=10m, 频偏=10ppm, Treply=1ms, Tround=2ms
  误差 = 10m * 10e-6 * (1ms/2ms) = 0.05mm (单次可忽略)
  但Treply增大或频偏更大时, 误差显著增长
```

### 5.3 SPI接口

DW1000通过SPI与主机MCU通信:

```
MCU                      DW1000
  MOSI -----------------> SPI_MOSI
  MISO <----------------- SPI_MISO
  SCK  -----------------> SPI_SCK
  CS   -----------------> SPI_CSN
  IRQ  <----------------- IRQ (测距完成/数据就绪)
```

- SPI模式0(CPOL=0, CPHA=0), 最高20MHz
- CS必须在帧开始时拉低, 帧结束时拉高
- IRQ是DW1000主动通知MCU的关键信号, 不可轮询替代
- 读取时间戳必须用快速SPI, 否则MCU响应延迟影响Treply

### 5.4 Anchor与Tag硬件差异

| 项目 | Tag(标签) | Anchor(锚点) |
|------|-----------|-------------|
| 角色 | 移动, 被定位 | 固定, 定位基准 |
| 供电 | 电池(纽扣/锂电) | 市电/POE |
| 功耗策略 | 深睡+周期唤醒 | 常收/常发 |
| 天线 | 全向偶极 | 墙壁贴装/吸顶 |
| 数量 | 多(几百) | 少(4-10/区域) |
| 时钟同步 | 不需要(TWR) | 需要(TDoA) |
| 校准 | 一次即可 | 部署后需位置校准 |

## 6 DW1000功耗模式

```
+------------------+
| TX (发送)        | ~110mW (最耗电)
+------------------+
| RX (接收)        | ~85mW
+------------------+
| IDLE (待机)      | ~12mW
+------------------+
| SLEEP (深度睡眠) | ~1uA
+------------------+
| OFF (关机)       | ~50nA
+------------------+
```

Tag节电策略:
1. 绝大部分时间在SLEEP
2. 周期唤醒 -> TX Poll -> RX Response -> TX Final -> SLEEP
3. 单次TWR测距约耗时1-5ms, 其余时间深度睡眠
4. 1Hz测距频率下, CR2032纽扣电池可续航数月

## 7 PCB设计指南

### 7.1 天线放置

UWB天线是PCB设计中最敏感的部分:

1. 天线区域下方和附近不要有地平面(keepout)
2. 天线远离金属结构件(电池、螺丝、LCD屏幕)
3. 馈电点阻抗匹配: 50欧姆微带线到天线馈点
4. Tag设备: 天线放在PCB短边端部, 远离主电路
5. Anchor设备: 天线可外置, 通过SMA/U.FL连接

### 7.2 地平面设计

```
+----------------------------------+
|  MCU + DW1000  |  天线keepout   |
|  区域           |  (无铜)        |
|  +-----------+  |                |
|  |地平面完整 |  |   天线          |
|  +-----------+  |                |
+----------------------------------+
```

- 数字电路区域: 地平面尽量完整
- 射频走线: 从DW1000到天线走50欧姆微带线, 参考地紧邻
- 过孔: 地平面换层处多打过孔(via fence), 间隔<lambda/20

### 7.3 布局规则

1. DW1000靠近天线, 射频走线最短
2. 38.4MHz晶振紧贴DW1000, 走线对称, 远离数字信号
3. SPI走线等长(误差<5mm), 加100ohm串联电阻减缓边沿
4. 电源去耦: 每个VDD引脚100nF, 芯片旁10uF
5. IRQ走线短直, 避免耦合噪声导致误中断

## 8 Apple U1/AirTag生态

### 8.1 苹果UWB战略

Apple从iPhone 11开始内置U1芯片(UWB), 开创了消费级UWB先河:

- **U1芯片**: 基于DW1000架构, 支持IEEE 802.15.4z
- **AirTag**: 内置UWB芯片, 与iPhone配合实现"精确查找"(Precision Finding)
- **NearbyInteraction框架**: iOS API, 允许App获取UWB测距和方向

### 8.2 对行业的影响

| 方面 | 影响 |
|------|------|
| 芯片生态 | 推动DW3000等4z兼容芯片开发 |
| 标准演进 | Apple深度参与802.15.4z标准制定 |
| 应用场景 | 从工业扩展到消费(找物、开锁、支付) |
| 频谱认可 | FCC/ETSI对UWB监管因消费级应用而明确 |
| 互操作 | Apple FiRa联盟推动跨厂商互操作 |

### 8.3 FiRa联盟

FiRa(Fine Ranging)联盟定义UWB互操作标准:
- 定义Anchor/Tag互操作协议
- 制定物理层和MAC层一致性测试
- 与Apple、Samsung、NXP、Qorvo等共同推动
- 目标: UWB设备即插即用, 类似蓝牙配对体验

## 9 天线延迟校准

天线延迟校准是UWB系统部署的关键步骤:

1. **原理**: 信号从DW1000芯片到天线辐射面有固定延迟(导线+天线), 这个延迟会被计入ToF, 必须扣除
2. **校准方法**:
   - 将两个DW1000放在已知距离d_ref(如2m)上
   - 测量TWR结果d_meas
   - 延迟 = (d_meas - d_ref) / 2 (两端各分担一半)
3. **写入寄存器**: 将校准值写入DW1000的天线延迟寄存器(0x00:0x0E)
4. **周期性**: 建议温度变化>10度或运行>1个月后重新校准

未校准的天线延迟可导致10-30cm的系统性偏移,是"测出来不准"的首要嫌疑。

## 总结

UWB以其皮秒级时间分辨率为室内定位提供了厘米级精度的可能。DW1000作为该领域的经典芯片,其硬件设计的关键在于:

1. 晶振精度和稳定性直接决定测距下限 -- 这是UWB系统中最不该省成本的器件
2. 天线设计和延迟校准消除了系统性偏差 -- 没有校准就没有精度
3. PCB布局中天线隔离和地平面完整性影响实际精度 -- 差的布局可浪费10cm+精度
4. TWR适合简单部署,TDoA适合大规模标签 -- 架构选择取决于应用规模
5. Apple生态推动了UWB从工业走向消费 -- DW3000和FiRa联盟是未来方向
