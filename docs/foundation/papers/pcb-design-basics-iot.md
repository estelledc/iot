# IoT设备PCB设计基础：布局布线规则
> **难度**：🟢 初级 | **领域**：PCB设计 | **阅读时间**：约 18 分钟

## 引言

想象你在装修一套新房子。你不会把冰箱和电视放在同一个角落，也不会让水管和电线交叉缠绕。你需要先规划哪个房间放什么家具，再决定水管和电线怎么走，最后检查有没有遗漏。PCB设计也是一样——元件是家具，走线是水电管线，地平面是地基，而设计规则检查就是竣工验收。

对于IoT设备来说，PCB设计还多了一层挑战：无线通信。天线附近的铜皮、元件的位置、甚至板子的形状，都会影响信号能不能顺利发出和接收。一块设计不当的PCB，可能让Wi-Fi信号衰减10dB，让BLE连接距离从50米缩到5米。

本文从PCB设计流程出发，逐步讲解布局原则、布线规则、接地策略、天线禁区，以及初学者最容易踩的坑，帮助你在做IoT硬件时少走弯路。

## 1 PCB设计流程总览

一块PCB从想法到实物，要经过以下五个阶段：

```
原理图(Schematic) -> 布局(Layout) -> 布线(Routing) -> DRC检查 -> Gerber输出
```

| 阶段 | 做什么 | 关键输出 |
|------|--------|----------|
| 原理图 | 确定元件及其电气连接 | 网表(Netlist) |
| 布局 | 在板子上放置元件 | 元件坐标 |
| 布线 | 用铜走线实现电气连接 | 走线几何 |
| DRC | 检查是否违反设计规则 | 错误报告 |
| Gerber | 生成制造文件 | 光绘文件 |

**每个阶段的时间分配大致为**：原理图20%，布局40%，布线30%，检查+输出10%。布局是最值得花时间的环节——好的布局让布线事半功倍，差的布局怎么布都别扭。

### 1.1 原理图到网表

原理图阶段的核心任务不是画好看的图，而是确保电气连接正确：

- 每个元件的封装(Footprint)必须与实物对应
- 网络标号(Net Label)命名要有意义，如`VCC_3V3`而非`N001`
- 电源和地要使用统一的符号，避免同名多义

生成网表后，EDA工具会将电气连接关系导入PCB编辑器，作为布局布线的约束基础。

### 1.2 从DRC到Gerber

DRC(Design Rule Check)不是走过场，它是最后的守门员：

- 间距检查：走线与走线、走线与焊盘之间的最小距离
- 连接检查：是否所有网络都已连通
- 制造检查：最小线宽、最小孔径是否满足工厂能力

DRC通过后，导出Gerber文件（包含各层的铜走线、阻焊、丝印信息）交给PCB打样厂。建议同时生成BOM表和坐标文件，方便SMT贴片。

## 2 元件布局原则

布局的本质是给元件安排座位。安排得好，走线短而直；安排得差，走线绕来绕去，引入噪声和延迟。

### 2.1 布局顺序：电源 -> 模拟 -> 数字

按照信号敏感度从高到低，布局遵循以下顺序：

1. **先放电源部分**：DC-DC转换器、LDO、电源滤波电容。电源是整个系统的根基，需要最短、最粗的走线。
2. **再放模拟敏感部分**：ADC输入、传感器接口、参考电压。模拟信号最怕噪声干扰，要远离开关噪声源。
3. **最后放数字部分**：MCU、Flash、数字接口。数字信号本身抗干扰能力强，但它是主要的噪声源，需要与模拟部分隔离。

```
+----------------------------------+
|  [电源区]    |    [模拟区]       |
|  DC-DC, LDO  |  ADC, Sensor     |
|--------------+-------------------|
|           [数字区]               |
|   MCU, Flash, UART/SPI          |
+----------------------------------+
```

### 2.2 功能分区与信号流向

好的布局让信号沿一个方向流动，避免来回折返：

- 将同一功能模块的元件放在一起（如Wi-Fi模块的芯片+匹配电路+天线馈点）
- 信号流向尽量从左到右或从上到下，保持一致性
- 连接器放在板边，MCU放在板中心附近

### 2.3 去耦电容的放置

去耦电容是元件的"随身充电宝"，位置不对等于没有：

| 规则 | 说明 |
|------|------|
| 就近放置 | 电容离IC电源引脚越近越好，目标<5mm |
| 先大后小 | 容值从大到小排列，大电容在外，小电容在内 |
| 多值搭配 | 0.1uF + 1uF + 10uF 组合覆盖不同频段 |
| 过孔位置 | 电容接地的过孔放在电容焊盘旁边，不要拉一段走线再打孔 |

```text
  IC电源引脚
     |
   [0.1uF] -- 紧贴引脚
     |
   [1uF]   -- 稍远
     |
   [10uF]  -- 更远
     |
  电源走线
```

去耦电容放得远的后果：电源引脚到电容之间的走线产生寄生电感，高频去耦效果大打折扣。一条5mm长的走线在100MHz时约有3nH电感，足以让0.1uF电容的阻抗从0.016欧增加到1.9欧——去耦效果几乎消失。

## 3 电源分配设计

电源分配网络(PDN)的目标是：在所有频率下，从IC看到的电源阻抗都低于目标值。

### 3.1 电源走线宽度计算

走线能过多少电流，取决于宽度和允许的温升。常用经验公式(IPC-2221)：

```
I = k * (dT^0.44) * (A^0.725)
```

其中：
- I：允许电流(A)
- dT：允许温升(通常取10度)
- A：走线截面积(mil^2) = 宽度(mil) * 铜厚(oz)
- k：内层取0.024，外层取0.048

**实用速查表(1oz铜厚，温升10度)**：

| 电流(A) | 外层最小线宽(mil) | 内层最小线宽(mil) |
|---------|-------------------|-------------------|
| 0.5     | 10                | 15                |
| 1.0     | 25                | 40                |
| 2.0     | 60                | 100               |
| 3.0     | 100               | 160               |

对于IoT设备，常见的电源走线建议：USB 5V至少40mil，3.3V主干至少25mil，分支信号电源10mil即可。

### 3.2 电源平面与分割

4层板中，通常将一层专做电源平面，不同的电源区域通过分割实现：

- 同一电源域保持连通，不同域之间留隔离带
- 分割线不要太窄也不要太宽，推荐20-40mil
- 避免信号走线跨过电源平面分割缝——跨缝的信号回流路径被切断，会产生严重EMI

2层板没有条件做完整电源平面，可以采用"电源走线主干道"策略：用较粗的走线(40-60mil)构成树状电源网络，分支从主干引出。

## 4 布线规则

布线是把原理图上的电气连接变成PCB上的铜走线。好的布线让信号干净到达，坏的布线让信号变形、延迟甚至出错。

### 4.1 避免直角走线

初学者最容易犯的错误就是走90度直角：

- 直角拐角处铜皮面积突变，产生额外的寄生电容
- 高速信号下直角会反射和辐射
- 更严重的是，制板工艺中直角拐角蚀刻不均匀

**正确做法**：使用45度斜角或圆弧拐角。大多数EDA工具可以设置自动圆角。

```text
错误:          正确:
  |              |
  |               \
  +---            |
                  +---
```

### 4.2 过孔的使用与最小化

过孔(Via)是连接不同层走线的"竖井"，每个过孔都引入：

- 寄生电感：约0.5-1nH（对高频信号影响显著）
- 寄生电容：约0.3-0.8pF
- 阻抗不连续：高速信号经过孔时反射

**使用原则**：

1. 能不走换层就不换层——同一根信号尽量在同一层布完
2. 电源和地过孔可以多打，信号过孔尽量少
3. 高速差分对换层时，两个信号的过孔要紧挨着放
4. BGA扇出区域是过孔密集区，注意不要堵死其他走线的通道

### 4.3 模拟与数字走线隔离

IoT设备通常是数模混合系统，隔离不好会出大问题：

- **空间隔离**：模拟走线和数字走线不要平行且相邻，尽量在不同区域布线
- **不跨分割**：模拟信号绝不能跨过地平面的分割缝
- **串扰估算**：两条平行走线的串扰与平行长度和距离相关。经验值：间距>=3倍线宽时，串扰可控制在5%以下

```text
+---------------------+---------------------+
|   模拟走线区域      |    数字走线区域      |
|   ADC, Sensor,     |    SPI, UART,       |
|   Reference        |    Clock            |
+---------------------+---------------------+
          |                    |
        AGND               DGND
          +--------+---------+
                   |
                 GND Plane
```

### 4.4 关键信号走线规则

| 信号类型 | 走线要求 | 典型应用 |
|----------|----------|----------|
| 高速时钟 | 最短路径，包地，避免过孔 | 32kHz晶振, RF参考时钟 |
| 差分对 | 等长等距，紧耦合 | USB, 天线馈线 |
| 电源走线 | 足粗越好，最短路径 | VCC, VDD |
| 复位信号 | 远离时钟和开关信号 | MCU NRST |
| 敏感模拟 | 远离数字，包地保护 | ADC输入, 传感器 |

## 5 地平面的重要性

地平面是PCB中最重要的概念之一，也是最容易被初学者忽略的。

### 5.1 地平面的作用

用地平面就像给房子打地基——看不见但最重要：

1. **回流路径**：每根信号走线都有回流，地平面提供最短回流路径，减小回路面积
2. **屏蔽**：完整的接地平面可以屏蔽层间干扰
3. **参考电平**：所有信号的参考都是地，地平面保证各处地电位一致
4. **散热**：大面积铜皮帮助散热

### 5.2 分割地还是统一地

这是一个经典的争议话题。经验法则：

- **低频(<1MHz)模拟+数字混合**：可以分割，但必须在一点连接(星形接地)
- **高频或射频**：统一地平面，绝对不要分割。分割会切断RF回流通路，导致辐射
- **IoT设备**：通常包含RF部分，建议统一地平面。模拟和数字的隔离靠布局分区实现，不靠分割地平面

### 5.3 2层板的地平面策略

2层板没有整层做地平面，但可以尽量弥补：

- 顶层元件面走信号，底层铺地铜皮(Copper Pour)
- 信号走线两侧尽量包地
- 关键信号(如RF走线)正下方保持完整地铜皮，不被走线打断
- 多打地过孔，连接顶层和底层的地铜皮，形成"围栏"效果

## 6 IoT无线天线禁区

IoT设备与普通电路板最大的区别就是有无线通信功能。天线附近的处理直接决定通信质量。

### 6.1 天线类型与禁区要求

| 天线类型 | 禁区要求 | 典型应用 |
|----------|----------|----------|
| PCB天线(倒F) | 天线周围无铜皮，保持净空 | BLE, Zigbee |
| 芯片天线 | 天线下方所有层无铜皮 | Wi-Fi, BLE |
| 外接天线(馈线连接) | 馈线50欧阻抗控制 | Sub-1GHz, LoRa |
| U型天线 | 天线臂下方无地铜皮 | 433MHz |

### 6.2 禁区设计规则

天线禁区(Antenna Keep-out Zone)的核心原则：**让天线"看到"自由空间**。

1. **无铜区**：天线正下方和周围所有层都不应有铜皮(包括地平面)
2. **无元件区**：天线附近不放任何元件(去耦电容也算)
3. **净空距离**：至少lambda/20，BLE(2.4GHz)约为6mm，Sub-1GHz约为15mm
4. **板边放置**：PCB天线最好放在板边，朝向自由空间
5. **馈线阻抗**：从芯片到天线的走线必须做50欧阻抗控制

```text
+----------------------------------+
|                    [净空区域]     |
|   MCU    RF芯片   ============> 天线
|                    [净空区域]     |
+----------------------------------+
                     ^馈线(50ohm)
```

### 6.3 阻抗匹配与馈线

RF馈线不是普通走线，需要做阻抗控制：

- 50欧微带线宽度取决于板厚、铜厚和介电常数
- 典型1.6mm板厚FR4上，50欧微带线宽约3mm
- 使用EDA工具的阻抗计算器或Saturn PCB Toolkit
- 馈线不要有过孔换层，不要有分支，不要直角拐弯

## 7 EDA工具选择

选对工具事半功倍。以下是IoT PCB设计常用EDA工具对比：

| 工具 | 授权 | 优势 | 不足 | 适合场景 |
|------|------|------|------|----------|
| KiCad | 开源免费 | 社区活跃，插件丰富，跨平台 | 复杂板支持较弱 | 学习、小项目 |
| Altium Designer | 商业(贵) | 功能强大，一体化设计 | 价格高，资源占用大 | 公司项目、多层板 |
| LCEDA(立创EDA) | 免费(在线) | 中文友好，元件库丰富，一键下单 | 依赖网络，高级功能有限 | 快速原型、学生项目 |

### 7.1 初学者推荐路径

1. 入门：LCEDA——中文界面，自带海量元件库，画完直接下单打样
2. 进阶：KiCad——学习专业设计流程，免费且无功能限制
3. 专业：Altium——如果公司提供License，这是行业标准

### 7.2 辅助工具

- **Saturn PCB Toolkit**：走线宽度、阻抗、过孔计算
- **Qucs/SimNEC**：RF匹配电路仿真
- **Gerber查看器**(GerbView)：导出后检查Gerber是否正确
- **3D查看器**：KiCad和Altium内置，检查元件干涉

## 8 初学者常见错误

这些错误几乎每个新手都会犯，提前了解可以省很多返工时间。

### 8.1 布局类错误

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| 去耦电容离IC太远 | 去耦失效，电源噪声大 | 电容紧贴电源引脚 |
| 元件太密集 | 无法布线，散热差 | 预留走线通道 |
| 连接器放板中央 | 走线绕远，线长增加 | 连接器放板边 |
| 没考虑元件高度 | 装壳装不下 | 建立完整3D模型 |

### 8.2 布线类错误

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| 走直角 | 信号反射，EMI | 45度或圆角 |
| 信号跨地缝 | 回流路径中断，严重EMI | 走线不跨分割 |
| 线宽太细 | 电源线过热甚至烧断 | 用电流-线宽表计算 |
| 差分不等长 | 差分信号偏斜，误码 | 差分对等长匹配 |

### 8.3 设计流程错误

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| 不跑DRC | 制造缺陷，短路断路 | 每次改完都跑DRC |
| 封装与实物不符 | 贴片无法焊接 | 下单前核对封装 |
| 没做阻抗控制 | RF信号反射，通信距离短 | RF走线用阻抗计算器 |
| 没留测试点 | 调试时无法探测信号 | 关键网络加测试焊盘 |

## 9 2层板与4层板选择

IoT设备的板层选择是成本与性能的权衡。

### 9.1 2层板

**优点**：成本低(约4层板的1/3)，交期快，适合简单设计

**缺点**：无完整地平面，信号质量差，EMI难控制

**适用场景**：
- 简单传感器节点(无RF或Sub-1GHz简单RF)
- 成本敏感的消费品
- 功能验证的早期原型

**2层板补偿策略**：
- 底层大面积铺地铜皮
- 关键信号走线旁包地
- 电源走线用较粗主线
- RF部分做局部"伪4层"(用走线构建参考面)

### 9.2 4层板

**典型层叠**：

| 层 | 用途 |
|----|------|
| Top | 元件+信号 |
| Inner1 | GND平面 |
| Inner2 | Power平面 |
| Bottom | 元件+信号 |

**优点**：完整地平面和电源平面，信号质量好，EMI易控制

**缺点**：成本高，交期略长

**适用场景**：
- 含Wi-Fi/BLE的IoT设备
- 数模混合系统
- 尺寸受限(小板更需要内层节省空间)

### 9.3 选择建议

```
是否有RF通信(Wi-Fi/BLE)?
  |
  +-- 是 --> 4层板(强烈建议)
  |
  +-- 否 --> 是否有高速数字信号(>50MHz)?
              |
              +-- 是 --> 4层板
              |
              +-- 否 --> 2层板(可接受)
```

简单来说：有射频就上4层，省那点钱不值得冒通信质量的风险。

## 10 设计审查清单

在交付Gerber之前，逐项检查以下清单：

### 10.1 电气检查

- [ ] DRC零错误零警告
- [ ] 所有网络100%连通
- [ ] 去耦电容位置符合就近原则
- [ ] 电源走线宽度满足电流要求
- [ ] 差分对等长误差在允许范围内
- [ ] RF馈线阻抗已控制(50欧)

### 10.2 布局检查

- [ ] 元件间距满足SMT贴片要求(>=0.2mm)
- [ ] 无器件干涉(3D检查)
- [ ] 连接器在板边，方向正确
- [ ] 天线禁区无铜皮无元件
- [ ] 敏感模拟电路远离数字开关噪声源
- [ ] IC下方地铜皮完整(散热+回流)

### 10.3 制造检查

- [ ] 最小线宽/间距满足工厂能力(通常6/6mil)
- [ ] 最小过孔内径满足工厂能力(通常0.3mm)
- [ ] 丝印不盖焊盘
- [ ] 板边到铜皮距离>=0.5mm
- [ ] 有定位标记(Fiducial)和工具孔

## 总结

PCB设计是一门实践性极强的工程技能，核心要点可以归纳为以下几条：

1. **布局决定布线**：花40%的时间在布局上，按电源->模拟->数字的顺序摆放
2. **去耦电容要近**：离IC电源引脚越近越好，远5mm效果减半
3. **走线不直角**：45度或圆角拐弯，避免信号反射和制造问题
4. **地平面是地基**：IoT设备用统一地平面，不分割；2层板用底面铺铜补偿
5. **天线要有净空**：天线周围无铜无元件，馈线做50欧阻抗控制
6. **有射频就4层**：Wi-Fi/BLE设备强烈建议4层板
7. **DRC不是走过场**：交付前逐项过清单，每一个勾都不要跳过

记住：PCB设计没有"差不多就行"。一个去耦电容放远5mm，可能让系统在特定条件下随机复位；一个地平面缺口，可能让产品过不了EMC认证。这些细节决定了产品是从实验室原型变成可量产的商品，还是永远停留在"在我电脑上是好的"阶段。
