# 超级电容器作为IoT能量缓冲的设计考量

> **难度**：🟡 中级 | **领域**：IoT储能设计 | **阅读时间**：约 20 分钟

## 引言

想象你家的水龙头出水量很小，只有一个细细的水流，但你偶尔需要一大桶水来冲马桶。你会怎么办？在龙头下面放一个大水缸，让细水慢慢攒满，需要时一下子倒出来。超级电容器在IoT系统里扮演的就是这个"水缸"的角色——能量收集源(太阳能、振动能)像细水龙头一样持续但微弱地供电，而LoRa或NB-IoT的发射瞬间需要几十到上百毫瓦的功率，远超收集源的即时输出能力。没有缓冲，发射时电压会瞬间跌落，MCU复位，数据丢失。

电池也能存能量，但电池更像一个"大桶"——容量大但出水慢，而且反复快速舀水会加速老化。超级电容器则是"宽口大缸"——容量不如桶，但出水极快，而且不怕反复舀。本文从超级电容器的物理原理出发，分析其在IoT能量缓冲场景下的选型计算、电路建模、混合架构设计，并给出基于BQ25570和LoRa的实际设计案例。

## 1. 超级电容器基础：三类原理与对比

### 1.1 双电层电容器 (EDLC)

EDLC是超级电容器中最常见的类型。充电时，电解液中的离子在电极表面吸附，形成两个紧密排列的电荷层(即"双电层")，每层厚度仅约1 nm。

- 储能机制：纯静电吸附，无化学反应
- 典型电极材料：活性炭(比表面积 1500-3000 m2/g)
- 电解液：水系(1 V/cell)或有机系(2.7-3.0 V/cell)
- 能量密度：5-10 Wh/kg，功率密度：10-50 kW/kg

因为不涉及化学反应，EDLC的循环寿命极长(50万次以上)，但单位体积存储的能量远低于电池。

### 1.2 赝电容器与混合型

赝电容器利用电极表面的快速可逆法拉第反应存储电荷，典型材料为RuO2或MnO2，能量密度10-30 Wh/kg但功率密度和循环寿命均低于EDLC，且材料成本高，IoT中应用较少。

混合型将EDLC的非极化电极与锂插层极化电极组合，能量密度15-30 Wh/kg，单cell电压3.8-4.0 V(与锂电兼容)，是增长中的方案。

### 1.3 三类汇总对比

| 特性 | EDLC | 赝电容器 | 混合型 |
|------|------|---------|--------|
| 储能机制 | 双电层(静电) | 法拉第(氧化还原) | 双电层+法拉第 |
| 能量密度 (Wh/kg) | 5-10 | 10-30 | 15-30 |
| 功率密度 (kW/kg) | 10-50 | 1-10 | 3-10 |
| 循环寿命 | 50万-100万 | 1万-10万 | 2万-5万 |
| 单cell电压 (V) | 2.7-3.0 | 1.0-1.2 | 3.8-4.0 |
| IoT适用性 | 最广泛 | 较少 | 增长中 |

## 2. 超级电容器 vs 电池：核心参数对比

| 参数 | 超级电容器(EDLC) | 锂离子电池 | 锂亚硫酰氯电池 |
|------|-----------------|-----------|--------------|
| 能量密度 | 5-10 Wh/kg | 150-265 Wh/kg | 300-500 Wh/kg |
| 功率密度 | 10-50 kW/kg | 0.5-2 kW/kg | <0.1 kW/kg |
| 循环寿命 | >50万次 | 300-3000次 | 一次性 |
| 充电时间 | 秒~分钟 | 小时级 | 不可充 |
| 工作温度 | -40~70 C | -20~60 C | -55~85 C |
| 自放电率 | 5-20%/月(高!) | 1-3%/月 | <1%/月 |
| 电压曲线 | 线性下降 | 平台电压 | 平台电压 |

关键差异在于电压曲线：电池放电时电压相对稳定，超级电容器则线性下降。超级电容器从Vmax到Vmin释放的能量为 `E = 0.5*C*(Vmax^2 - Vmin^2)`。若Vmin = 0.7*Vmax，可用能量仅51%。**设计含义：超级电容器供电必须有DC-DC变换器稳压。**

温度方面，超级电容器低温下ESR增大(-40 C时约2-3倍)，但不会像锂电那样容量断崖式下降或有热失控风险，这对户外IoT节点是重要优势。

## 3. 等效电路模型

### 3.1 三分支模型

最简模型是理想电容C串联ESR，但实际行为需用三分支模型描述：

```
         R1          R2          R3
Vin o---/\/\/---+---/\/\/---+---/\/\/---+
                |           |           |
               C1          C2          C3
                |           |           |
GND o-----------+-----------+-----------+
```

- 分支1 (R1+C1)：快速响应，时间常数约1秒，承担主要充放电电流
- 分支2 (R2+C2)：中期分支，时间常数10-100秒，电荷再分配
- 分支3 (R3+C3)：长期分支，时间常数>1000秒，深层电荷扩散

### 3.2 ESR与漏电流

ESR直接限制最大输出功率：`V_drop = I_burst * ESR`，`P_max = V^2 / (4*ESR)`。IoT发射脉冲100-500 mA范围内，ESR压降通常不是主要瓶颈。

漏电流才是超级电容器作为长期储能的劣势：

| 容值 | 典型漏电流 | 稳态经验公式 |
|------|----------|------------|
| 0.1-1 F | 1-20 uA | I_leak(uA) ≈ 1.5 + 0.004*C(F)*V(V) |
| 10-100 F | 20-500 uA | 同上 |

对于能量收集节点(收集功率可能仅几十uW)，漏电流可消耗相当比例的收集能量，必须计入系统预算。

## 4. IoT发射缓冲的容量计算

### 4.1 计算公式

发射所需能量 `E_tx = P_tx * T_tx`，考虑DC-DC效率eta后，电容器需释放 `E_cap = E_tx / eta`。从容值-电压关系解出：

```
C = 2 * P_tx * T_tx / (eta * (Vmax^2 - Vmin^2))
```

### 4.2 计算示例：LoRa发射缓冲

| 参数 | 值 | 说明 |
|------|---|------|
| P_tx | 120 mW | LoRa 20 dBm发射 |
| T_tx | 200 ms | SF7, 50字节负载 |
| Vmax | 5.0 V | 超级电容满电电压 |
| Vmin | 3.3 V | DC-DC最低输入电压 |
| eta | 0.90 | DC-DC变换器效率 |

```
C = 2 * 0.12W * 0.2s / (0.90 * (25.0 - 10.89))
  = 0.048 / 12.70 ≈ 3.78 mF
```

仅需约4 mF即可缓冲单次LoRa发射！但工程上还需1.5-2x余量(老化+温度)，加上连续重传需求。最终选型 10 mF / 5.5 V 可满足单次发射，30 mF可缓冲3次连续发射。

### 4.3 充电恢复时间

```
T_recharge = C * (Vmax - Vmin) / I_charge
```

BQ25570输出5 mA时，10 mF电容从3.3V恢复到5.0V需约340秒(5.7分钟)。若发送间隔5分钟，则需稍增大容值或降低Vmin——这是设计中常见的迭代过程。

## 5. 充放电曲线分析

### 5.1 基本曲线

恒流充电：`V(t) = V0 + I*t/C`(线性上升)
恒流放电：`V(t) = V0 - I*t/C`(线性下降)

超级电容器适合恒流充电(不像锂电需要CC-CV两阶段)，简化了充电电路设计。放电曲线线性下降是与电池最直观的区别。

### 5.2 脉冲负载下的稳态条件

IoT负载是脉冲式的。稳态下每个周期内收集能量等于消耗加漏电损失：

```
P_harvest * T_cycle = P_tx * T_tx + P_sleep * T_sleep + P_leak * T_cycle
```

若收集功率不足以覆盖消耗+漏电，电容器电压将持续下降直至Vmin，此时应跳过发送窗口等待恢复。

## 6. 串联均压设计

### 6.1 为什么要均压

单个EDLC cell额定电压仅2.7-3.0 V，IoT系统常需2-3个cell串联。各cell容值、ESR总有偏差，容值小的cell充电先到额定电压(过充损坏)，放电时电压先到零甚至反转(永久损坏)。

### 6.2 均压方法与IoT推荐

| 方法 | 优点 | 缺点 | IoT适用性 |
|------|------|------|----------|
| 电阻均压 | 极简无源 | 持续消耗能量 | 仅高电压/小容值 |
| 齐纳二极管 | 无源、速度快 | 齐纳管漏电 | 2-cell推荐(低漏齐纳) |
| 主动均衡IC | 精确低损耗 | 增加成本 | 3-cell推荐(ALD9100) |
| 单封装2-cell | 内部已均压 | 选型受限 | 最省心方案 |

**设计警告**：绝不能将多个超级电容器cell直接串联而不加均压措施，首次充电就可能因容值偏差导致某个cell过压损坏。

## 7. 超级电容+电池混合架构

### 7.1 为什么要混合

单独超级电容器漏电流大，长期待机能量流失严重；单独电池脉冲放电能力不足，循环寿命有限。混合架构让两者各司其职：电池提供长期基线能量，超级电容器提供脉冲峰值功率。

日常类比：电池是工资(稳定但金额有限)，超级电容器是信用卡(能刷大额但月底要还清)。

### 7.2 三种拓扑

**拓扑1：并联直连** — 电池与超级电容器直接并联，简单但超级电容器电压被电池钳位，无法利用全部能量。

**拓扑2：电池经DC-DC向超级电容器充电** — 超级电容器可在更宽电压范围工作，DC-DC仅在需要时开启减少静态损耗。

**拓扑3：超级电容器优先，电池后备** — 能量收集优先给超级电容器充电，超级电容器电压低于阈值时才启动电池补充。电池几乎不被脉冲电流冲击，寿命大幅延长。

### 7.3 容量分配经验法则

- 超级电容器：缓冲3-5次最大脉冲发射
- 电池：满足无能量收集时的预期续航时间
- 电池放电率：控制在C/20以下以大幅延长循环寿命

## 8. BQ25570能量收集IC集成

### 8.1 BQ25570核心特性

| 特性 | 规格 |
|------|------|
| 冷启动电压 | 330 mV(最低) |
| MPPT追踪 | 每16秒采样，动态调整工作点 |
| 充电管理 | 欠压锁定/过压保护/终止电压可设 |
| 稳压输出 | 可编程2.2/2.4/3.0/3.3 V |
| 静态电流 | <330 nA |

### 8.2 超级电容器充电管理

BQ25570对超级电容器非常友好：(1) 330 mV即可冷启动升压；(2) MPPT动态追踪最大收集功率；(3) Vbat_ov可设过压阈值防止过充；(4) Vbat_uv防止过放；(5) 稳压输出直接为MCU供电。

典型外围：超级电容器直接连VBAT引脚，无需额外充电电路。VBAT_OV通过电阻分压设为超级电容器额定电压的90-95%，VBAT_UV设为DC-DC最低输入电压。若同时接电池和超级电容器，电池通过VBRG引脚或二极管连接。

注意：BQ25570不提供cell级均压，使用2-cell串联时需外部均压电路。

## 9. 实战设计：LoRa发射缓冲系统

### 9.1 需求与功耗预算

| 阶段 | 功耗 | 持续时间 | 每次能量 |
|------|------|---------|---------|
| 睡眠 | 1.65 uW | ~300 s | 495 uJ |
| 传感器采样 | 1.65 mW | 50 ms | 82.5 uJ |
| LoRa发射(20dBm) | 132 mW | 200 ms | 26.4 mJ |
| LoRa接收 | 16.5 mW | 50 ms | 0.825 mJ |

每5分钟发送一次，每小时消耗约333.6 mJ，平均功耗约92.7 uW。室内光伏100 lux下仅30 uW，不足。调整发送间隔为15分钟后，平均功耗降至约32 uW，仍略低于收集功率，需超级电容器在光照不足时供电。

### 9.2 选型结果

```
C = 2 * 26.4e-3 / (0.90 * 14.11) ≈ 4.2 mF    // 单次发射
C_selected = 4.2 * 3(重传) * 1.5(老化) ≈ 19 mF
```

选型：22 mF / 5.5 V 双cell串联封装(如Panasonic EECS0HD103H)。

### 9.3 固件能量管理

```c
// 能量管理状态机
typedef enum {
    STATE_CHARGING,    // 超级电容电压 < V_ready
    STATE_READY,       // 超级电容电压 >= V_ready
    STATE_TRANSMIT,    // 正在发射
    STATE_RECOVERY     // 发射后等待恢复
} energy_state_t;

#define V_READY_MV  4500   // 足以进行一次发射
#define V_LOW_MV    3500   // 最低可发射电压
#define V_CRIT_MV   3000   // 临界电压, 仅睡眠

energy_state_t check_energy_state(void) {
    uint16_t vcap_mv = read_vcap_adc();
    if (vcap_mv < V_CRIT_MV) return STATE_CHARGING;
    if (vcap_mv >= V_READY_MV) return STATE_READY;
    if (vcap_mv >= V_LOW_MV) return STATE_RECOVERY;
    return STATE_CHARGING;
}
```

核心原则：**永远先检查能量再发射**，电压不足时跳过本次窗口。LoRa的ALOHA协议天然容忍丢包，偶尔跳过不影响系统可靠性。

### 9.4 性能估算

| 指标 | 值 |
|------|---|
| 超级电容器充满时间 | ~22分钟(从0到5V, 30uW) |
| 单次发射后电压跌落 | ~0.27 V (5.0V -> 4.73V) |
| 最长无光照续航 | ~2.5小时(每15分钟发1次) |
| 超级电容器寿命(25C) | >10年(每天充放~96次) |

## 10. 老化与可靠性

### 10.1 老化机制与寿命估算

超级电容器老化主要是电解液挥发(ESR增大、容值下降)和电极氧化，长期在额定电压附近工作加速老化。

寿命用电压-温度加速模型估算：

```
L(T,V) = L_ref * 2^((T_ref-T)/10) * 2^((V_ref-V)/(0.1*V_ref))
```

示例：某电容85C/2.7V下寿命2000小时，在60C/2.5V下使用：
`L = 2000 * 2^2.5 * 2^0.74 ≈ 18,900小时 ≈ 2.2年`

降额使用(电压降0.2V、温度降25C)将寿命从2000小时延长到2.2年。40C/2.4V下可达10年以上。

### 10.2 可靠性设计建议

| 措施 | 效果 | 代价 |
|------|------|------|
| 降额5-10%电压 | 寿命延长2-4x | 可用能量减少15-25% |
| 热管理(散热/避光) | 每降10C寿命翻倍 | 增加结构复杂度 |
| 避免过充/过放 | 防止电解液分解 | 需监测电路 |
| 容值降额选型 | 预留老化裕量 | 增加成本/体积 |

纯超级电容器方案BOM成本与纯电池相当，但寿命大幅延长。对于远程节点，10年免维护的经济效益远超初始BOM差异。

### 11.2 何时选超级电容器

| 场景 | 推荐 | 原因 |
|------|------|------|
| 能量收集供电 | 超级电容(必选) | 电池无法承受频繁浅充放 |
| 脉冲发射+电池 | 超级电容+电池 | 电池单独承受脉冲寿命骤减 |
| 超长待机(年) | 电池 | 超级电容漏电太严重 |
| 宽温度范围(-40C) | 超级电容(或混合) | 锂电低温性能差 |
| 万次以上充放循环 | 超级电容 | 电池仅几百-几千次 |

## 总结

超级电容器作为IoT能量缓冲的核心价值在于三个字：**快、久、宽**——功率输出快(kW/kg级)，循环寿命久(50万次以上)，温度范围宽(-40到70 C)。但短板同样明确：能量密度低、漏电流大、放电电压线性下降。

设计时的核心决策链：

1. 计算脉冲发射所需能量，确定超级电容器容值下限
2. 评估漏电流与收集功率的比例，判断是否需混合架构
3. 确定串联cell数和均压方案
4. 选择PMIC(如BQ25570)并设定充放电阈值
5. 在固件中实现"先检查能量再发射"的状态机
6. 降额使用(电压-5~10%、温度留余量)以保证10年寿命

在能量收集IoT系统中，超级电容器不是电池的替代品，而是电池的搭档。两者组合的混合架构，才能同时满足IoT节点对长待机(电池的低漏电)和高峰值功率(超级电容器的高功率密度)的双重需求。
