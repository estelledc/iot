# LoRa扩频因子SF优化与链路预算计算
> **难度**：🟡 中级 | **领域**：LoRa物理层 | **阅读时间**：约 20 分钟

## 引言

想象你在山谷里对远处的朋友喊话。如果你快速喊一句"吃饭了",声音传得近但信息量大;如果你把每个字拖得很长"吃——饭——了——",声音能传得更远,但说完一句话需要更久。LoRa的扩频因子(Spread Factor, SF)就是这个"拖长"的程度——SF越高,信号传得越远,但数据速率越低、传输时间越长。

如何选择最优的SF,使得通信既可靠又高效,是LoRa网络设计的核心问题。本文将从LoRa调制原理出发,深入解析SF的选择逻辑和链路预算计算方法。

## 1. LoRa调制基础

### 1.1 啁啾扩频(CSS)

LoRa使用Chirp Spread Spectrum(CSS)调制技术: 频率随时间线性变化的啁啾信号,数据通过啁啾的起始频率偏移来编码。

```
频率
  ^
  |    /|    /|    /|     <- 上啁啾(数据符号)
  |   / |   / |   / |
  |  /  |  /  |  /  |
  | /   | /   | /   |
  |/    |/    |/    |
  +-----|-----|-----|--> 时间
     T_sym  T_sym  T_sym
```

CSS的核心优势: 抗多径(不同路径反射信号在频域分散)、抗窄带干扰(能量分布在宽带上)、低功率谱密度(不易被检测)。

### 1.2 工作频段

LoRa通常工作在Sub-GHz ISM频段: EU868(863-870MHz)、US915(902-928MHz)、CN470(470-510MHz)、AS923(915-928MHz)。Sub-GHz比2.4GHz传播损耗更小,更适合长距离通信。

## 2. 扩频因子(SF)详解

### 2.1 SF的定义

SF决定每个符号(symbol)包含多少个chip: SF范围SF7到SF12,每个符号包含2^SF个chip,chip速率等于带宽。

```
SF7:  每符号 128 chips  -> 符号时间 = 128/125000 = 1.024 ms
SF8:  每符号 256 chips  -> 符号时间 = 2.048 ms
SF9:  每符号 512 chips  -> 符号时间 = 4.096 ms
SF10: 每符号 1024 chips -> 符号时间 = 8.192 ms
SF11: 每符号 2048 chips -> 符号时间 = 16.384 ms
SF12: 每符号 4096 chips -> 符号时间 = 32.768 ms
```

### 2.2 SF与灵敏度的关系

每增加一级SF,接收器灵敏度提升约2.5dB:

| SF | 接收灵敏度 (125kHz BW) | 相对SF7的增益 |
|----|----------------------|--------------|
| SF7 | -123 dBm | 基准 |
| SF8 | -126 dBm | +3 dB |
| SF9 | -129 dBm | +6 dB |
| SF10 | -132 dBm | +9 dB |
| SF11 | -134.5 dBm | +11.5 dB |
| SF12 | -137 dBm | +14 dB |

### 2.3 SF与数据速率

数据速率公式: `Rb = SF * BW / (2^SF) * CR`

以BW=125kHz, CR=4/5为例:

| SF | 比特速率(bps) | 相对SF7 |
|----|--------------|---------|
| SF7 | 5470 | 100% |
| SF8 | 3125 | 57% |
| SF9 | 1758 | 32% |
| SF10 | 977 | 18% |
| SF11 | 537 | 10% |
| SF12 | 293 | 5.4% |

SF7到SF12数据速率相差约18倍。

### 2.4 SF与空中时间

同样50字节有效载荷的空中时间:

```
SF7:  约 72 ms     |  SF10: 约 453 ms
SF8:  约 134 ms    |  SF11: 约 987 ms
SF9:  约 247 ms    |  SF12: 约 1810 ms (1.8秒)
```

空中时间直接影响功耗、信道占用率和碰撞概率。

## 3. 链路预算计算

### 3.1 链路预算公式

```
接收信号强度 = TX功率 + TX天线增益 + RX天线增益 - 路径损耗 - 额外损耗
通信条件: 接收信号强度 >= RX灵敏度 + 衰落余量
```

各参数: TX功率(+14到+20dBm), TX天线增益(IoT设备2-3dBi), RX天线增益(网关5-8dBi), 路径损耗(环境相关), 额外损耗(线缆/穿墙), 衰落余量(5-10dB)。

### 3.2 路径损耗模型

**自由空间(FSPL)**: `FSPL(dB) = 20*log10(d_km) + 20*log10(f_MHz) + 32.44`

**对数距离模型**(更贴近实际):
```
PL(d) = PL(d0) + 10*n*log10(d/d0)

典型路径损耗指数n:
- 自由空间: n = 2
- 郊区: n = 2.5-3
- 城市: n = 3-4
- 密集城市/室内: n = 4-6
```

### 3.3 链路预算计算实例

**场景**: 城市环境, LoRa节点到网关距离3km

```
参数:
- TX功率: +14 dBm, TX天线: +2 dBi, RX天线: +6 dBi
- 路径损耗 (城市3km, n=3.2, PL(1m)=40dB):
  PL = 40 + 32*log10(3000) = 40 + 32*3.477 = 151.3 dB
- 额外损耗: 5 dB, 衰落余量: 10 dB

接收信号: 14 + 2 + 6 - 151.3 - 5 = -134.3 dBm

各SF判断(需要 RSSI >= 灵敏度 + 余量10dB):
SF7(-123): -134.3 < -113 --> 不够
SF10(-132): -134.3 < -122 --> 不够
SF11(-134.5): -134.3 需要 >= -124.5 --> 不够
SF12(-137): -134.3 需要 >= -127 --> 不够

去掉单独余量,直接比灵敏度:
SF11: -134.3 > -134.5 --> 勉强可用(余量0.2dB)
SF12: -134.3 > -137 --> 可用(余量2.7dB)

结论: 需要SF12但余量很小,建议缩短距离或提升天线
```

### 3.4 反推最大距离

已知使用SF9(-129dBm灵敏度),求最大距离:

```
可用路径损耗 = 14 + 2 + 6 - (-129) - 5 - 10 = 136 dB
136 = 40 + 32*log10(d)
d = 10^(96/32) = 10^3 = 1000m = 1km

结论: SF9在城市环境(n=3.2)下最大可靠距离约1km
```

## 4. 带宽与编码率选择

### 4.1 带宽选项

| 带宽 | 灵敏度影响 | 数据速率 | 典型应用 |
|------|-----------|---------|---------|
| 125 kHz | 基准(最佳) | 基准 | EU868/CN470标准 |
| 250 kHz | -3 dB | 2x | 需要更快传输 |
| 500 kHz | -6 dB | 4x | US915上行 |

灵敏度公式: `灵敏度 = -174 + 10*log10(BW) + NF + SNR_required`

各SF所需SNR: SF7(-7.5dB), SF8(-10dB), SF9(-12.5dB), SF10(-15dB), SF11(-17.5dB), SF12(-20dB)。

### 4.2 编码率(CR)

| 编码率 | 有效比特:总比特 | 开销 | 适用场景 |
|--------|---------------|------|---------|
| CR 4/5 | 4:5 | 25% | 信号良好(默认) |
| CR 4/6 | 4:6 | 50% | 中等环境 |
| CR 4/7 | 4:7 | 75% | 较差环境 |
| CR 4/8 | 4:8 | 100% | 恶劣信道 |

编码率不影响灵敏度,但影响有效数据速率和空中时间。LoRaWAN默认CR 4/5。

## 5. SF优化策略

### 5.1 核心原则: 使用最低可靠SF

```
1%占空比限制下, 50字节载荷每小时可发消息数:
SF7: 50条 | SF8: 27条 | SF9: 14条
SF10: 8条 | SF11: 4条 | SF12: 2条
```

使用不必要的高SF会浪费信道容量、增加功耗、增加碰撞概率。

### 5.2 自适应数据速率(ADR)

LoRaWAN的ADR机制动态优化SF:

```
ADR流程:
1. 终端上报信号质量(SNR/RSSI)
2. 网络服务器统计最近20个包的SNR
3. 计算SNR余量 = 实测SNR - 所需SNR(当前SF)
4. 余量 > 阈值: 降低SF(提速)
5. 余量 < 0: 提高SF(保可靠性)
6. 通过下行MAC命令通知终端调整

示例: 当前SF10(所需SNR=-15dB), 实测SNR=-5dB
余量 = -5-(-15) = 10dB, 可降约4级 -> 切到SF7
```

ADR注意事项: 只适合静止/慢速设备; 快速移动应禁用ADR用固定高SF; ADR有滞后性,环境突变时可能丢包。

## 6. SF的准正交性

### 6.1 同信道并发

不同SF的信号可以在同一信道上同时传输而互不干扰:

```
信道1 (868.1 MHz):
  |-- SF7传输 --|
       |-- SF9传输 --|
  |---- SF12传输 ----|
三个传输同时进行,互不干扰(理论上)
```

实际中相邻SF有约16dB隔离,间隔越大隔离越好。实际容量提升约4-6倍。

### 6.2 容量规划

```
单信道利用所有SF的总容量:
SF7(50) + SF8(27) + SF9(14) + SF10(8) + SF11(4) + SF12(2) = 105 msg/h

8信道网关: 8 * 105 = 840 msg/h
考虑碰撞重传, 实际约 600-700 msg/h
```

## 7. 空中时间与占空比

### 7.1 EU868占空比规定

| 子频段 | 频率范围 | 占空比限制 |
|--------|---------|-----------|
| g1 | 868.0-868.6 MHz | 1% |
| g2 | 868.7-869.2 MHz | 0.1% |
| g3 | 869.4-869.65 MHz | 10% |

### 7.2 载荷优化

SF12时减小载荷效果显著: 10字节约1155ms, 50字节约1810ms。减少40字节节省约36%空中时间。建议使用紧凑二进制编码而非JSON文本。

## 8. 实际部署案例

### 8.1 场景: 城市水表监测

需求: 2000个水表, 每15分钟上报, 5km覆盖

分析: 单网关5km覆盖在城市环境中链路预算不足,经计算需要4个网关将最大距离缩短到1.5km。

ADR效果:
```
0-0.5km(40%设备): SF7/SF8 -> 高速低功耗
0.5-1km(30%设备): SF8/SF9
1-1.5km(30%设备): SF10/SF11/SF12

4网关 * 8信道, 可支撑约3000设备每15分钟上报
```

## 9. 计算工具

### 9.1 Python链路预算计算器

```python
import math

def lora_link_budget(tx_power, tx_ant, rx_ant, distance_m,
                     freq_mhz, n, extra_loss, margin, sf):
    sensitivity = {7:-123, 8:-126, 9:-129, 10:-132, 11:-134.5, 12:-137}
    # 对数距离路径损耗
    pl_ref = 20*math.log10(freq_mhz) + 20*math.log10(0.001) + 32.44
    path_loss = pl_ref + 10*n*math.log10(distance_m)
    rssi = tx_power + tx_ant + rx_ant - path_loss - extra_loss
    link_margin = rssi - sensitivity[sf] - margin
    return {'rssi': round(rssi,1), 'margin': round(link_margin,1),
            'ok': link_margin >= 0}

result = lora_link_budget(14, 2, 6, 2000, 868, 3.2, 3, 10, 10)
print(f"RSSI: {result['rssi']}dBm, 余量: {result['margin']}dB, "
      f"可通信: {'是' if result['ok'] else '否'}")
```

### 9.2 空中时间计算

```python
import math

def time_on_air(sf, bw_hz=125000, cr=1, payload=20, preamble=8):
    t_sym = (2**sf) / bw_hz * 1000  # ms
    t_pre = (preamble + 4.25) * t_sym
    de = 1 if sf >= 11 else 0
    num = 8*payload - 4*sf + 28 + 16
    den = 4*(sf - 2*de)
    n_pay = 8 + max(math.ceil(num/den)*(cr+4), 0)
    return round(t_pre + n_pay*t_sym, 2)

for sf in range(7, 13):
    toa = time_on_air(sf)
    print(f"SF{sf}: {toa}ms, 每小时最多{int(3600000/(toa*100))}条(1%占空比)")
```

## 总结

LoRa扩频因子的优化是多目标平衡: 距离(灵敏度) vs 速率(吞吐量) vs 功耗(空中时间) vs 容量(信道占用)。核心策略是"使用能保证可靠通信的最低SF"。

链路预算计算是SF选择的理论基础,将发射功率、天线增益、路径损耗等量化为明确数字。实际部署中ADR可自动优化,但规划阶段仍需手动计算确定网关数量和位置。
