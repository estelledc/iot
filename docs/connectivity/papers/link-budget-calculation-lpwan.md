# LPWAN链路预算计算与覆盖估算
> **难度**：🟡 中级 | **领域**：无线工程 | **阅读时间**：约 20 分钟

## 引言

想象你在一个嘈杂的体育场里,想和对面看台上的朋友说话。你需要考虑几个问题: 你能喊多大声(发射功率)?中间有多少噪音会淹没你的声音(路径损耗)?你的朋友听力有多好(接收灵敏度)?如果你喊的声音到达朋友耳边时比背景噪音大,他就能听到;如果被噪音淹没了,就听不到。

链路预算(Link Budget)就是对这个过程的精确计算。它把无线通信链路中所有的增益(让信号变强的因素)和损耗(让信号变弱的因素)全部列出来,做一笔"收支账",看看最终到达接收机的信号是否足够强。对于 LPWAN 来说,链路预算尤其重要,因为 LPWAN 追求的就是极远的通信距离和极深的室内穿透,每一个 dB 都关系到能不能覆盖到那只装在地下室里的水表。

## 1. 链路预算的基本概念

### 1.1 什么是链路预算

链路预算是对无线通信链路中所有增益和损耗的系统化计算。其核心公式:

```
链路余量 = 发射功率 + 发射天线增益 - 馈线损耗 - 路径损耗
           - 额外损耗 + 接收天线增益 - 接收灵敏度

如果链路余量 > 0: 通信可以建立
如果链路余量 < 0: 通信无法建立
如果链路余量 = 10dB: 有10dB的安全裕度
```

链路余量(Link Margin)是接收信号强度与接收灵敏度之间的差值。余量越大,通信越可靠。通常设计目标是保留 10-20dB 的余量,以应对信号的随机波动。

### 1.2 dB和dBm

在链路预算中,所有的计算都使用 dB(分贝)单位,因为 dB 可以将乘除法转换为加减法,简化计算。

dBm 是以1毫瓦为参考的功率单位: 0dBm = 1mW, 10dBm = 10mW, 20dBm = 100mW, 30dBm = 1W, -10dBm = 0.1mW。

dBi 是天线增益单位,以全向天线为参考: 0dBi 表示与全向天线相同,6dBi 表示比全向天线强4倍。

```
常用功率转换:
  1 mW  = 0 dBm
  10 mW = 10 dBm
  25 mW = 14 dBm
  100 mW = 20 dBm
  500 mW = 27 dBm
  1 W   = 30 dBm
```

### 1.3 链路预算的组成部分

一个完整的链路预算包含以下要素:

发射端: 发射功率(dBm)、发射天线增益(dBi)、馈线/连接器损耗(dB)。

传播路径: 自由空间路径损耗(dB)、阴影衰落余量(dB)、建筑穿透损耗(dB)、其他额外损耗(dB)。

接收端: 接收天线增益(dBi)、馈线/连接器损耗(dB)、接收灵敏度(dBm)。

## 2. 为什么链路预算对LPWAN至关重要

### 2.1 LPWAN的设计目标

LPWAN(Low-Power Wide-Area Network)的核心设计目标决定了链路预算的重要性:

长距离: 覆盖半径 5-30km,远超传统无线技术。在如此远的距离上,信号衰减巨大,需要精确计算才能确定是否可达。

深室内穿透: 信号需要穿透建筑物到达地下室的水表、气表。建筑穿透损耗可达 20-30dB,必须在链路预算中精确考虑。

电池供电: 终端设备通常使用电池,发射功率受限。不能简单地靠提高功率来增加覆盖,必须从系统层面优化每一个环节。

### 2.2 每个dB都很重要

在 LPWAN 系统中,链路预算通常处于极限状态。以 LoRa 为例,最大允许路径损耗可达 157dB,这意味着信号从发射到接收的过程中衰减了超过 10^15 倍(一千万亿倍)。在这种极端条件下,每多出 3dB 的损耗,覆盖半径就会缩短约 30%。

因此,LPWAN 系统的设计者必须仔细计算链路预算中的每一个分项,确保没有遗漏任何损耗来源,也不要高估任何增益。

## 3. 发射端参数

### 3.1 发射功率

发射功率是链路预算的起点。不同的 LPWAN 技术有不同的典型发射功率:

| 技术 | 典型发射功率 | 换算为mW | 限制因素 |
|------|-------------|---------|----------|
| LoRa(欧洲868MHz) | 14 dBm | 25 mW | ETSI ERP限制 |
| LoRa(美国915MHz) | 20-22 dBm | 100-158 mW | FCC EIRP限制 |
| Sigfox(欧洲) | 14 dBm | 25 mW | ETSI ERP限制 |
| NB-IoT | 23 dBm | 200 mW | 3GPP规范 |
| LTE-M | 23 dBm | 200 mW | 3GPP规范 |

发射功率不是越高越好。更高的功率意味着更大的电池消耗,而且必须遵守各地区的法规限制。

### 3.2 发射天线

终端设备的天线通常是内置的 PCB 天线或小型线圈天线,增益在 -3dBi 到 2dBi 之间。增益取决于天线设计、安装位置和外壳材料。

```
常见IoT终端天线增益:
  PCB蚀刻天线: 0 - 2 dBi
  芯片天线: -2 - 0 dBi
  弹簧天线: 1 - 3 dBi
  外接鞭状天线: 2 - 5 dBi
```

对于终端设备,通常假设天线增益为 0dBi,这是一个保守但合理的估计。

### 3.3 馈线和连接器损耗

终端设备天线通常直接焊接在PCB上,馈线损耗可忽略(约 0.1-0.5dB)。对于使用外置天线的设备,常见馈线损耗: RG-174 约 0.9dB/m, RG-58 约 0.6dB/m, LMR-195 约 0.4dB/m, LMR-400 约 0.15dB/m(均为 900MHz 参考值)。每个 SMA 连接器约引入 0.1-0.3dB 损耗。

### 3.4 EIRP计算

EIRP(等效全向辐射功率)是发射端的综合输出:

```
EIRP = 发射功率(dBm) + 天线增益(dBi) - 馈线损耗(dB)

示例(LoRa终端):
  发射功率: 14 dBm
  天线增益: 0 dBi
  馈线损耗: 0.5 dB
  EIRP = 14 + 0 - 0.5 = 13.5 dBm
```

EIRP 必须符合当地法规限制。例如欧洲 868MHz g1 子频段限制 EIRP 为 16.15dBm(14dBm ERP + 2.15dBi)。

## 4. 接收灵敏度

### 4.1 灵敏度的含义

接收灵敏度是接收机能正确解码信号的最低信号强度(通常定义为误码率低于某个阈值时的信号强度)。灵敏度越低(数值越小,负得越多),接收机越"灵敏",能从更微弱的信号中提取数据。

### 4.2 LPWAN的灵敏度优势

LPWAN 技术通过窄带宽和编码增益实现了极高的接收灵敏度,这是其实现长距离通信的关键:

| 技术 | 带宽 | 灵敏度 | 数据速率 |
|------|------|--------|----------|
| WiFi(802.11n) | 20 MHz | -85 dBm | 65 Mbps |
| BLE | 2 MHz | -97 dBm | 1 Mbps |
| Zigbee | 2 MHz | -100 dBm | 250 kbps |
| LoRa SF7/125kHz | 125 kHz | -123 dBm | 5.47 kbps |
| LoRa SF12/125kHz | 125 kHz | -137 dBm | 0.293 kbps |
| Sigfox | 0.1 kHz | -142 dBm | 0.1 kbps |
| NB-IoT(单tone) | 3.75 kHz | -141 dBm | 0.68 kbps |

### 4.3 灵敏度与数据速率的权衡

一个基本的物理规律: 带宽越窄,灵敏度越高,但数据速率越低。这是香农定理的直接推论。

LoRa 通过调整扩频因子(SF)在灵敏度和数据速率之间权衡:

```
LoRa灵敏度与扩频因子(125kHz带宽):
  SF7:  -123 dBm, 数据速率 5470 bps
  SF8:  -126 dBm, 数据速率 3125 bps
  SF9:  -129 dBm, 数据速率 1760 bps
  SF10: -132 dBm, 数据速率 980 bps
  SF11: -134.5 dBm, 数据速率 440 bps
  SF12: -137 dBm, 数据速率 293 bps
```

每增加一级 SF,灵敏度提升约 2.5-3dB(覆盖距离增加约 30%),但数据速率减半(传输时间翻倍,功耗翻倍)。

## 5. 最大允许路径损耗(MAPL)

### 5.1 MAPL的计算

最大允许路径损耗(Maximum Allowable Path Loss, MAPL)是链路预算的核心输出。它代表从发射机到接收机之间信号最多可以衰减多少,通信仍然能正常工作。

```
MAPL = EIRP(dBm) + 接收天线增益(dBi) - 接收灵敏度(dBm)

注意: 接收灵敏度是负数, 减去负数等于加上其绝对值
```

### 5.2 各LPWAN技术的MAPL对比

| 技术 | EIRP | RX天线增益 | RX灵敏度 | MAPL |
|------|------|-----------|---------|------|
| LoRa SF12(欧洲) | 14 dBm | 6 dBi | -137 dBm | 157 dB |
| LoRa SF12(美国) | 22 dBm | 6 dBi | -137 dBm | 165 dB |
| Sigfox(欧洲) | 14 dBm | 6 dBi | -142 dBm | 162 dB |
| NB-IoT | 23 dBm | 0 dBi | -141 dBm | 164 dB |
| WiFi | 20 dBm | 2 dBi | -85 dBm | 107 dB |
| BLE | 0 dBm | 0 dBi | -97 dBm | 97 dB |

LPWAN 的 MAPL 比 WiFi 高 50-58dB,这意味着 LPWAN 能承受的路径损耗是 WiFi 的10万到100万倍。这就是 LPWAN 能实现数公里覆盖的根本原因。

### 5.3 MAPL到距离的转换

有了 MAPL,就可以用传播模型将其转换为覆盖距离。以 Okumura-Hata 模型为例:

```
LoRa SF12, 欧洲868MHz, 城市环境:
  MAPL = 157 dB
  使用Hata城市公式(hb=30m, hm=1.5m, f=868MHz):
  157 = 126.40 + 35.23*log10(d)
  d = 10^((157-126.40)/35.23) = 10^0.869 = 7.4 km

  减去10dB阴影衰落余量:
  147 = 126.40 + 35.23*log10(d)
  d = 10^(0.585) = 3.85 km

郊区环境:
  MAPL_suburban = 157 + 9.85 = 166.85 dB (有效MAPL)
  166.85 = 126.40 + 35.23*log10(d)
  d = 10^(1.147) = 14.0 km

  减去8dB阴影衰落余量:
  d = 9.5 km
```

城市环境约 4km,郊区约 10km,这与 LoRaWAN 的实际覆盖经验基本一致。

## 6. 链路余量与衰落余量

### 6.1 为什么需要余量

链路预算计算出的 MAPL 是理论最大值。实际通信中,信号强度会因各种因素随机波动,如果按照理论极限设计,信号稍有波动就会中断。因此,必须在 MAPL 中扣除各种余量,用"有效 MAPL"来做覆盖规划。

### 6.2 阴影衰落余量

阴影衰落(Shadow Fading)是由建筑物、地形等大型障碍物引起的信号随机波动。在同一距离上的不同位置,接收信号强度可能相差 10-20dB。

阴影衰落通常用标准差为 sigma 的正态分布(对数正态分布)描述。要保证 P% 的位置能正常通信,需要预留的衰落余量为:

```
衰落余量与覆盖概率:
  50% 覆盖: 0 dB余量(中值)
  75% 覆盖: 0.67 * sigma
  90% 覆盖: 1.28 * sigma
  95% 覆盖: 1.65 * sigma
  99% 覆盖: 2.33 * sigma

典型sigma值:
  城市: 8-10 dB
  郊区: 6-8 dB
  农村: 4-6 dB

示例(城市, sigma=8dB, 95%覆盖):
  衰落余量 = 1.65 * 8 = 13.2 dB
```

### 6.3 其他余量

人体损耗: 如果 IoT 设备被人体携带,典型损耗 3-5dB;固定安装的传感器可忽略。

时间衰落余量: 无线信道随时间变化(植被生长/凋落、气象、建筑变化),为保证长期(10年)可靠性通常预留 5-10dB。

### 6.5 综合余量示例

```
综合链路余量预留:
  阴影衰落余量: 10 dB (城市, 90%覆盖)
  人体损耗: 0 dB (固定安装设备)
  时间衰落余量: 5 dB
  设备老化余量: 2 dB
  总余量: 17 dB

有效MAPL = MAPL - 总余量
         = 157 - 17
         = 140 dB
```

## 7. 上行与下行链路预算

### 7.1 非对称链路

LPWAN 系统的上行(终端到网关)和下行(网关到终端)链路预算通常是不对称的:

终端: 发射功率低(14-23dBm),天线增益低(0dBi),接收灵敏度一般。

网关/基站: 发射功率可以更高,天线增益更高(6-10dBi),接收灵敏度可以更好。

### 7.2 上行通常是瓶颈

在大多数 LPWAN 系统中,上行链路是覆盖的限制因素。原因是终端设备的发射功率和天线增益受限于体积、成本和法规。

```
上行链路预算(LoRa SF12):
  终端发射功率: 14 dBm
  终端天线增益: 0 dBi
  路径损耗: ?
  网关天线增益: 6 dBi
  网关灵敏度: -137 dBm
  上行MAPL = 14 + 0 + 137 + 6 = 157 dB

下行链路预算(LoRa SF12):
  网关发射功率: 27 dBm
  网关天线增益: 6 dBi
  路径损耗: ?
  终端天线增益: 0 dBi
  终端灵敏度: -137 dBm
  下行MAPL = 27 + 6 + 137 + 0 = 170 dB
```

上行 MAPL 为 157dB,下行 MAPL 为 170dB。下行比上行多 13dB 的余量。因此,上行是覆盖瓶颈: 网关能"听到"终端的距离,比终端能"听到"网关的距离要短。

### 7.3 设计原则

LPWAN 网络规划应基于上行链路预算。只要上行能通,下行通常也没问题。例外情况是: 某些国家对网关的下行发射功率有额外限制(如占空比限制),可能导致下行成为瓶颈。

## 8. 室内穿透损耗

### 8.1 建筑穿透的重要性

对于 LPWAN 来说,室内穿透是一个关键问题。大量的 IoT 传感器和计量设备安装在建筑物内部,甚至地下室。信号需要穿透建筑物的外墙、楼板等结构才能到达室外的网关。

建筑穿透损耗必须从 MAPL 中扣除,这会显著缩短覆盖距离。

### 8.2 典型穿透损耗值

| 建筑类型 | 外墙穿透损耗(dB) | 说明 |
|----------|----------------|------|
| 住宅(木结构) | 8-12 | 北美常见 |
| 住宅(砖混) | 12-18 | 欧洲/亚洲常见 |
| 商业建筑 | 15-25 | 钢筋混凝土结构 |
| 工业厂房 | 20-30 | 金属墙体/屋顶 |
| 地下室(一层) | 额外10-15 | 在建筑穿透基础上叠加 |
| 地下室(两层) | 额外20-25 | 深层地下室 |

### 8.3 对覆盖距离的影响与应对策略

以 LoRa SF12 为例: 室外有效MAPL 147dB时覆盖约4km;加上15dB住宅穿透损耗后缩小到约1.2km;再加地下室损耗(10dB)后只剩约0.5km。地下室覆盖距离只有室外的约八分之一。

应对策略包括: 提高扩频因子(SF7到SF12灵敏度增加14dB,但速率降低20倍)、增加网关密度、安装室内中继器、将天线引出建筑物外部、或对深层地下室改用有线(M-Bus/RS-485)或PLC技术。

## 9. LPWAN技术链路预算对比

### 9.1 完整链路预算对比表

以下是三种主流 LPWAN 技术的完整链路预算对比:

```
                        LoRa SF12    Sigfox     NB-IoT
                        (欧洲)      (欧洲)     (Band 8)
发射功率(dBm)           14          14         23
发射天线增益(dBi)        0           0          0
发射馈线损耗(dB)         0.5         0.5        0
EIRP(dBm)               13.5        13.5       23

接收天线增益(dBi)        6           6          18(基站)
接收馈线损耗(dB)         1           1          2
接收灵敏度(dBm)          -137        -142       -141

MAPL(dB)                 155.5       160.5      180

阴影衰落余量(dB)         10          10         10
建筑穿透损耗(dB)         15          15         15

有效MAPL(dB)             130.5       135.5      155

城市覆盖半径(km)         1.0         1.5        3.0
郊区覆盖半径(km)         3.0         5.0        8.0
```

注: NB-IoT 的 MAPL 较高主要因为基站天线增益大(18dBi)和终端发射功率高(23dBm)。

### 9.2 各技术的权衡与选择

LoRa MAPL 中等,优势在自建网络、无运营费、通过调整SF灵活权衡覆盖和速率。Sigfox MAPL 在免许可频段中最高,但数据速率极低(100bps,每天最多140条)。NB-IoT 整体MAPL最高,利用蜂窝基础设施,但有持续的SIM卡和流量费用。

选择建议: 室内深度覆盖且有蜂窝信号选 NB-IoT;农村或无蜂窝覆盖地区选 LoRa;数据量极小且超远距离选 Sigfox。

## 10. 实际案例: 地下室智能水表

### 10.1 问题描述

某水务公司计划为一栋10层住宅楼的地下一层水表间安装智能水表。最近的 LoRaWAN 网关安装在 2km 外一栋建筑的楼顶(高度30m)。问题: 水表的 LoRa 信号能否到达网关?

### 10.2 链路预算计算

```
发射端(水表):
  发射功率: 14 dBm
  天线增益: 0 dBi (内置PCB天线)
  馈线损耗: 0 dB (天线直接焊接)
  EIRP: 14 dBm

传播路径:
  路径损耗(Hata城市, 868MHz, 2km, hb=30m, hm=1.5m):
    PL = 126.40 + 35.23*log10(2) = 126.40 + 10.61 = 137.0 dB
  建筑外墙穿透: 15 dB (砖混结构)
  地下室额外损耗: 12 dB (一层地下室)
  阴影衰落余量: 8 dB

  总路径损耗: 137.0 + 15 + 12 + 8 = 172.0 dB

接收端(网关):
  天线增益: 6 dBi (玻璃钢全向天线)
  馈线损耗: 1 dB (LMR-400, 3m)
  灵敏度: -137 dBm (LoRa SF12/125kHz)

接收信号强度:
  RSSI = 14 + 0 - 172.0 + 6 - 1 = -153 dBm

与灵敏度比较:
  链路余量 = -137 - (-153) = -16 dB
```

### 10.3 结果分析

链路余量为 -16dB,这意味着接收信号强度比灵敏度还低 16dB。通信无法建立。

### 10.4 解决方案与决策

方案一: 在该楼或附近楼顶增设网关,距离缩短到200m,路径损耗减少约35dB,链路余量变为+19dB。方案二: 将水表天线引出到地面层窗外,消除地下室和大部分建筑穿透损耗(22dB),余量+6dB勉强可用。方案三: 建筑一层安装 LoRa 中继器。方案四: 改用 NB-IoT 水表(MCL 可达 164dB)。

最终选择方案一: 增设网关成本约2000-5000元,但同时覆盖周围500m内所有建筑,为未来扩展提供基础设施。这个案例说明链路预算能在部署前发现覆盖问题。

## 总结

链路预算是 LPWAN 无线系统设计的核心工具。它将发射功率、天线增益、路径损耗、穿透损耗、接收灵敏度等所有因素纳入一个统一的计算框架,回答"信号能不能到达目标设备"这个根本问题。对于 LPWAN 来说,由于工作在信号极限边缘,每一个 dB 的增益或损耗都可能决定一个部署方案的成败。掌握链路预算的计算方法,理解各分项的物理含义和典型数值,是 IoT 无线工程师的基本功。在实际项目中,链路预算计算应结合传播模型和现场测量一起使用,才能得到可靠的覆盖规划。
