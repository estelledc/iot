# NFC与Qi无线充电集成方案
> **难度**：🟡 中级 | **领域**：NFC能量传输 | **阅读时间**：约 20 分钟

## 引言

想象你有一个万能的电磁炉。普通电磁炉只能加热锅底, 但如果这个电磁炉既能加热做饭, 又能同时给旁边的小夜灯供电, 那就方便多了。NFC和Qi无线充电的集成方案就是这个思路: 两者都基于电磁感应(inductive coupling)原理, 只是工作频率不同。NFC在13.56MHz频率上传输数据, Qi在100-200kHz频率上传输电能。如果能让一个设备同时支持两种功能, 甚至共用一个线圈, 就能大幅简化硬件设计。

本文将从基础原理出发, 介绍NFC无线充电(WLC)规范、与Qi的对比、集成设计挑战, 以及在IoT场景中的实际应用。

## 1. 电磁感应耦合基础

### 1.1 共同的物理原理

NFC和Qi无线充电都依赖电磁感应耦合。发送端线圈中流过交变电流, 产生交变磁场。接收端线圈处于这个磁场中, 磁通量变化在接收线圈中感应出电动力, 从而实现无线能量传输。

两者的核心差异在于工作频率和设计目标:

| 参数 | NFC | Qi无线充电 |
|------|-----|-----------|
| 工作频率 | 13.56 MHz | 100-200 kHz |
| 主要目的 | 数据通信 | 电能传输 |
| 典型功率 | 数十mW(通信) | 5-15 W |
| 工作距离 | < 10 cm | < 4 cm |
| 线圈尺寸 | 较小(手机NFC天线) | 较大(充电底座) |

### 1.2 频率选择的工程权衡

为什么NFC用13.56MHz而Qi用100-200kHz? 这是一个经典的工程权衡问题。

高频(13.56MHz)的优势在于数据传输速率高, 天线尺寸可以做得更小。但高频下线圈的等效串联电阻(ESR)增大, 能量传输效率降低。低频(100-200kHz)正好相反: 数据速率低, 但能量传输效率高, 适合大功率充电场景。

```
频率与应用的关系:

100 kHz ----[Qi充电]---- 大功率, 高效率
  |
1 MHz
  |
13.56 MHz --[NFC通信]---- 高速数据, 小天线
  |                       [NFC WLC]---- 小功率充电
100 MHz
```

## 2. NFC无线充电(WLC)规范

### 2.1 NFC Forum WLC概述

NFC Forum在2020年发布了无线充电规范(Wireless Charging Specification, WLC)。这个规范的核心思想是: 让NFC设备(比如手机)不仅能读写NFC标签, 还能通过同一个13.56MHz NFC天线为小型设备充电。

WLC规范定义了两个角色:

- WLC Poller(充电端): 通常是手机, 提供能量
- WLC Listener(被充端): 小型设备, 接收能量

关键技术参数:

- 载波频率: 13.56 MHz(与NFC通信相同)
- 最大输出功率: 1 W
- 功率协商: 通过NFC通信协议动态调节
- 充电与通信: 在同一载波上交替进行

### 2.2 WLC工作流程

WLC充电过程分为几个阶段:

```
手机(Poller)                    小设备(Listener)
    |                                |
    |--- 1. NFC场检测 -------------->|
    |<-- 2. 设备识别+能力声明 -------|
    |--- 3. 功率协商 --------------->|
    |<-- 4. 确认功率等级 ------------|
    |--- 5. 增强RF场强度 ----------->|
    |    [能量传输开始]              |
    |<-- 6. 周期性状态报告 ----------|
    |    (电压/电流/温度)           |
    |--- 7. 动态功率调节 ----------->|
    |    ...                        |
    |<-- 8. 充电完成通知 ------------|
    |--- 9. 恢复正常NFC场 --------->|
```

在步骤5中, 手机增强NFC天线的RF场强度。普通NFC通信时场强较低(足够供电NFC标签芯片即可), 但WLC模式下场强显著提高以传输更多能量。

### 2.3 功率等级定义

WLC规范定义了多个功率等级:

| 功率等级 | 输出功率 | 适用场景 |
|---------|---------|---------|
| Class 0 | 250 mW | 传感器标签刷新 |
| Class 1 | 500 mW | 小型可穿戴设备 |
| Class 2 | 750 mW | 耳机充电盒 |
| Class 3 | 1000 mW | 智能手表/手写笔 |

## 3. WLC与Qi的对比分析

### 3.1 定位差异

WLC和Qi并非竞争关系, 而是互补关系。它们面向完全不同的充电场景:

Qi的定位是"充电底座给手机充电"。手机放在充电板上, 以5-15W的功率快速充电。用户需要一个专用的充电硬件。

WLC的定位是"手机给小配件充电"。手机变成了充电器, 小设备贴在手机背面即可充电。虽然功率只有1W, 但对于电池容量只有40-100mAh的小设备来说完全够用。

### 3.2 详细对比

| 特性 | Qi | NFC WLC |
|------|-----|---------|
| 充电方向 | 充电板 -> 手机 | 手机 -> 小设备 |
| 功率范围 | 5 - 15 W (EPP) | 最高 1 W |
| 载波频率 | 100-200 kHz | 13.56 MHz |
| 是否需要专用硬件 | 需要充电板 | 手机NFC即可 |
| 数据通信 | 有限(功率控制) | 完整NFC通信 |
| 目标设备 | 手机/平板 | 耳机/手表/传感器 |
| 标准组织 | WPC | NFC Forum |
| 效率 | 70-80% | 30-50% |

### 3.3 效率差异的原因

WLC的效率(30-50%)明显低于Qi(70-80%)。这并非设计缺陷, 而是物理限制:

第一, 13.56MHz的高频率导致线圈和周围金属部件中的涡流损耗更大。第二, 手机NFC天线并非为功率传输优化设计的, 线圈匝数和尺寸都受限。第三, WLC的目标功率本身就低, 在1W级别下固定损耗占比更高。

但这个效率对于目标场景是可接受的。给一个40mAh的耳机电池充满电, 即使效率只有40%, 也只消耗手机电池约100mAh, 对于3000-5000mAh的手机电池来说微不足道。

## 4. NFC能量收集(Energy Harvesting)

### 4.1 无源NFC标签的能量收集

在WLC规范之前, NFC标签就已经具备能量收集能力。普通NFC标签没有电池, 它们从读卡器的RF场中获取足够的能量来驱动自己的芯片, 完成数据读写操作。

NFC能量收集将这个概念进一步扩展: 不仅为标签芯片供电, 还为外部微控制器(MCU)和传感器供电。

### 4.2 NTAG I2C Plus方案

NXP的NTAG I2C Plus是一个典型的NFC能量收集方案:

```
+------------------+     I2C总线     +---------------+
|                  |<--------------->|               |
|  NTAG I2C Plus   |                 |   MCU + 传感器 |
|  (NFC标签芯片)    |--- VOUT ------->|  (如温度传感器) |
|                  |  (能量输出)      |               |
+------------------+                 +---------------+
        ^
        | NFC RF场
+------------------+
|  NFC读卡器/手机   |
+------------------+
```

工作流程:

1. 手机靠近标签, NFC场激活NTAG芯片
2. NTAG从NFC场中收集能量, 通过VOUT引脚输出
3. 收集的能量驱动外部MCU启动
4. MCU读取传感器数据(如温度)
5. MCU通过I2C将数据写入NTAG的SRAM
6. 手机通过NFC读取SRAM中的传感器数据

整个过程无需电池。手机一靠近, 传感器自动上电、采集、传输, 手机一离开, 系统断电。

### 4.3 能量收集的功率预算

NFC读卡器在标签位置提供的功率通常为:

```
典型场景功率预算:

NFC读卡器发射功率:    约200 mW (EIRP)
距离3cm处可用功率:     约30-50 mW
天线匹配损耗:          约5-10 mW
NTAG芯片自身消耗:      约5 mW
VOUT可输出功率:        约15-30 mW

15-30mW能做什么:
- 驱动低功耗MCU (如STM32L0): 约3 mW
- 读取温湿度传感器:            约1 mW
- 读取加速度计:                约2 mW
- 剩余用于数据传输和余量
```

## 5. 集成设计的关键挑战

### 5.1 天线设计挑战

在同一设备中集成NFC和Qi, 天线设计是最大的挑战。

NFC天线是一个在13.56MHz谐振的小型线圈, 通常3-5匝, 尺寸约30x40mm。Qi接收线圈在100-200kHz谐振, 通常10-20匝, 尺寸约50x50mm。两个线圈在物理空间上会产生电磁耦合, 互相影响对方的谐振频率和品质因子(Q值)。

解决方案有三种:

第一种是空间隔离: 将NFC天线和Qi线圈放在不同层, 中间用铁氧体(ferrite)隔离。这是目前手机中最常见的方案。

第二种是频率选择性设计: 在天线匹配网络中加入滤波器, 让每个天线只对自己的工作频率响应。

第三种是共用线圈: 用一个线圈同时工作在两个频率。这在理论上最优, 但实际设计难度极大, 目前还没有成熟的商用方案。

### 5.2 金属干扰问题

手机内部充满金属部件: 电池外壳、中框、散热片等。金属会影响NFC和Qi的性能:

对NFC: 金属会屏蔽13.56MHz磁场, 严重降低读写距离。解决方法是在NFC天线和金属之间放置铁氧体片, 引导磁力线绕过金属。

对Qi: 金属会产生涡流, 导致发热和效率下降。Qi线圈背面同样需要铁氧体屏蔽。

在集成设计中, NFC和Qi可以共用铁氧体层, 但需要仔细优化铁氧体的材料参数, 使其在两个频率上都有足够的磁导率。

### 5.3 热管理

无线充电过程中的能量损耗会转化为热量。Qi充电15W时, 即使效率80%, 也有3W变成热量。在小型设备中, 这些热量会导致温度快速升高。

WLC虽然功率低(1W), 但被充电设备通常也非常小(如耳机), 散热面积有限。0.5W的热耗散在耳机大小的设备上也可能导致明显温升。

热管理策略包括:

- 动态功率调节: 通过NFC通信实时报告温度, 超温时降低功率
- 占空比控制: 不连续充电, 交替充电和冷却
- 材料选择: 使用导热性好的基板材料

### 5.4 同时操作的时序管理

当设备同时支持NFC通信和WLC充电时, 需要在两种模式之间切换:

```
时间线:
|--NFC通信--|--WLC充电--|--NFC通信--|--WLC充电--|
|  100ms   |  900ms   |  100ms   |  900ms   |
           |<-- 1秒周期 -->|
```

通信和充电不能完全同时进行, 因为充电时的高场强会影响通信质量。实际方案是时分复用(TDM): 大部分时间用于充电, 周期性切换到通信模式交换状态信息。

## 6. 现代手机中的集成现状

### 6.1 典型手机天线堆叠

现代旗舰手机中, NFC和Qi的天线堆叠结构如下:

```
[手机背壳]
[NFC天线(FPC柔性线圈)]     <- 13.56 MHz
[铁氧体片(共用)]
[Qi接收线圈(多股绞合线)]    <- 100-200 kHz
[石墨散热片]
[电池]
```

NFC天线位于Qi线圈上方, 两者共用一层铁氧体。这种设计经过多年优化, 已经非常成熟。部分手机还支持反向Qi充电(Wireless PowerShare), 手机作为Qi发射端为其他手机充电。

### 6.2 芯片方案

目前主流的集成方案芯片:

| 芯片 | 厂商 | 功能 |
|------|------|------|
| ST25DV-W | ST | NFC动态标签 + 能量收集 + WLC |
| NTAG I2C Plus | NXP | NFC标签 + I2C接口 + 能量收集 |
| P9415 | IDT/Renesas | Qi 15W接收 + 发射 |
| nPM1300 | Nordic | 电源管理IC, 可配合NFC使用 |

ST25DV-W是目前最接近完整WLC方案的芯片, 它集成了NFC Forum Type 5标签功能和WLC能量收集输出。

## 7. 实际应用场景

### 7.1 手机充电耳机

这是WLC最直接的应用场景。用户将耳机充电盒放在手机背面, 通过NFC WLC充电。耳机充电盒电池通常为300-500mAh, 以1W功率充电约需30-60分钟。虽然不快, 但在外出忘带充电器时非常实用。

### 7.2 手机充电智能手表

智能手表电池通常为200-400mAh。通过NFC WLC, 手机可以为手表应急充电。即使只充10-15分钟, 也能续航几个小时。

### 7.3 IoT传感器标签

在工业场景中, NFC能量收集最有价值。温度记录标签贴在货物上, 工人用手持NFC读卡器一扫, 标签自动上电、读取传感器、回传数据。无需换电池, 无需布线, 维护成本极低。

典型应用包括冷链物流温度监控、设备状态巡检、医疗器械使用记录和食品新鲜度检测。

### 7.4 手写笔充电

触控笔(stylus)通常电池很小(约40-50mAh)。通过NFC WLC, 手写笔只需贴在手机或平板侧面几分钟即可充满。一些新型平板电脑已经开始采用这种方案, 取代磁吸充电接口。

## 8. 与其他小设备充电方案的对比

| 方案 | 功率 | 是否需要专用硬件 | 优势 | 劣势 |
|------|------|----------------|------|------|
| NFC WLC | 最高1W | 不需要, 手机即可 | 最便捷 | 功率低 |
| Qi 5W | 5W | 需要充电板 | 速度快 | 需额外设备 |
| USB-C | 5-100W | 需要线缆 | 最快 | 需要物理接口 |
| 太阳能 | 10-100mW | 需要光照 | 免费能源 | 依赖环境 |
| 纽扣电池 | N/A | 需要更换 | 简单 | 不环保 |

NFC WLC在"便捷性"维度上无可匹敌: 不需要任何额外硬件, 手机人人都有, 贴一下就充电。

## 10. 未来发展方向

### 10.1 更高功率的NFC WLC

NFC Forum正在研究将WLC功率提升到3-5W。这需要改进天线设计和热管理, 但技术上是可行的。更高功率将扩大适用设备范围, 比如为小型智能音箱或电子阅读器充电。

### 10.2 充电与配置一体化

NFC WLC的独特优势是充电和数据通信在同一个物理通道上。这意味着可以实现"一贴完成所有事": 手机贴近IoT设备, 同时完成充电、固件更新、配置参数写入、数据读取。

这种"充电+配置"(Charge and Configure)的模式对IoT设备部署特别有价值。想象一下, 工厂安装100个传感器节点, 工人只需用手机逐个贴一下, 就完成了上电、配网、参数设置的全部流程。

### 10.3 永续IoT传感器

如果NFC读卡器可以固定安装在特定位置(比如货架边缘), 每次传感器标签经过时都能获得一次能量补充和数据上传。这种模式下, 传感器可以真正实现"永不换电池"。

结合超低功耗MCU(如nRF52系列的System OFF模式, 电流小于1uA)和高效能量收集, 传感器在两次NFC激活之间靠微小的储能电容维持时钟, 在NFC场中快速唤醒、采集、传输、休眠。

## 总结

NFC与Qi无线充电的集成是一个"殊途同归"的故事。两者都基于电磁感应, 但服务于不同的功率和距离需求。NFC WLC将手机变成了小型设备的移动充电站, 虽然功率有限(1W), 但对于可穿戴设备和IoT传感器来说恰到好处。

核心要点回顾: NFC WLC使用13.56MHz载波, 最高1W功率, 手机作为充电端。NFC能量收集让无源传感器标签无需电池。集成设计的关键挑战是天线互耦、金属干扰和热管理。未来"充电+配置"一体化将大幅简化IoT设备部署。

对于IoT系统设计者来说, NFC WLC提供了一个独特的选项: 当设备功耗足够低、且需要近距离人机交互时, NFC WLC可以彻底消除充电接口和更换电池的麻烦。
