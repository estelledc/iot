# 私有 5G 网络在工业 IoT 中的部署

> 难度：🟡 中级 | 领域：蜂窝通信、工业物联网 | 更新：2025-06

---

## 一句话总结

私有 5G 网络让企业在自己的工厂、园区或矿山中部署专属的蜂窝网络，用 URLLC 实现毫秒级确定性延迟、用网络切片隔离不同业务流量、用 CBRS/n78 免授权或轻授权频谱降低成本——代价是比 WiFi 高出 3-10 倍的初始投资，但在可靠性和安全性上有质的飞跃。

---

## 从日常场景说起

你去过大型音乐节吗？几万人同时刷手机，4G/5G 信号差到几乎用不了。这是因为你在和几万人共享运营商的公共网络，带宽被稀释了。

现在想象你是一个工厂老板。你的 AGV（自动导引车）需要可靠的无线连接来接收实时调度指令，如果网络断了 1 秒钟，一辆载着 2 吨货物的 AGV 可能撞到人。你敢把这种关键业务放在运营商的公共 5G 网络上吗？

私有 5G 就是为这种场景诞生的：你在自己的工厂里建一套完整的 5G 网络——自己的基站、自己的核心网、自己的频谱（或至少是优先使用权）。就像大公司自建食堂而不是让员工去外面排队买饭——虽然建设成本高，但可控、可靠、不用和别人抢。

再进一步理解：如果说公共 5G 是"公交车"（便宜、共享、不保证座位），那私有 5G 就是"专车"（贵、独享、保证服务质量）。

## 1. 什么是私有 5G 网络

### 1.1 基本架构

一个私有 5G 网络包含以下核心组件：

```
┌─────────────────────────────────────────────────────┐
│                   企业园区/工厂                        │
│                                                     │
│  ┌──────┐  ┌──────┐  ┌──────┐                       │
│  │ gNB  │  │ gNB  │  │ gNB  │  ← 5G 基站 (小基站)    │
│  └──┬───┘  └──┬───┘  └──┬───┘                       │
│     │         │         │                           │
│     └────┬────┘────┬────┘                           │
│          │         │                                │
│     ┌────▼─────────▼────┐                           │
│     │    5G 核心网 (5GC)  │  ← 可以是边缘服务器        │
│     │  AMF / SMF / UPF   │                           │
│     └────────┬──────────┘                           │
│              │                                      │
│     ┌────────▼──────────┐                           │
│     │   企业应用服务器     │  ← MES, SCADA, AI 推理    │
│     └───────────────────┘                           │
└─────────────────────────────────────────────────────┘
         │ (可选) 回连公网
    ┌────▼────┐
    │ 运营商网络 │
    └─────────┘
```

关键区别于公网 5G：

- **gNB（基站）**：部署在企业园区内部，覆盖范围几百米到几公里
- **5GC（核心网）**：部署在企业本地（边缘），数据不出园区
- **频谱**：使用企业自有或轻授权频谱（如 CBRS），不依赖运营商
- **SIM 卡**：企业自己发放 eSIM/iSIM，自己管理设备身份

### 1.2 部署模式

根据企业的技术能力和预算，私有 5G 有三种部署模式：

| 模式 | 描述 | 频谱 | 核心网位置 | 典型客户 |
|------|------|------|----------|---------|
| 完全自建 | 企业自购设备、自建网络 | 自有/CBRS | 本地 | 大型制造业、矿业 |
| 运营商托管 | 运营商提供设备和运维 | 运营商授权频谱 | 本地或远程 | 中型企业 |
| 混合模式 | 企业自建 RAN，运营商提供核心网 | 共享 | 混合 | 对安全性要求高的企业 |

## 2. 频谱选择：CBRS、n77、n78

频谱是私有 5G 的"命门"——没有频谱就没有网络。全球不同地区的频谱政策差异很大。

### 2.1 全球频谱分配

| 地区 | 频段 | 带宽 | 授权方式 | 说明 |
|------|------|------|---------|------|
| 美国 | CBRS (n48) 3.55-3.7 GHz | 150 MHz | 三级共享 (PAL/GAA) | 最成熟的私有 5G 频谱 |
| 欧洲 | n78, 3.7-3.8 GHz | 100 MHz | 本地授权 (德/英/法) | 各国政策不同 |
| 日本 | n79, 4.6-4.9 GHz | 300 MHz | 本地 5G 牌照 | "ローカル5G" |
| 中国 | 5.925-6.125 GHz (讨论中) | 200 MHz | 工信部待定 | 尚未正式开放 |
| 韩国 | n78, 3.7-3.8 GHz | 100 MHz | 本地授权 | 2024 年开始发放 |

### 2.2 CBRS 三级频谱共享

美国的 CBRS（Citizens Broadband Radio Service）是全球最成功的私有 5G 频谱模型，值得深入了解：

```
CBRS 频谱分层结构 (3.55 - 3.7 GHz)：

优先级 高 ─►  Tier 1: 现任用户 (Incumbent)
               │  美国海军雷达、卫星地面站
               │  绝对优先，其他用户必须避让
               ▼
              Tier 2: PAL (Priority Access License)
               │  拍卖获得，$1-5/MHz/POP
               │  覆盖一个县 (county)，70 MHz 可拍
               ▼
优先级 低 ─►  Tier 3: GAA (General Authorized Access)
                  免费使用，80 MHz 可用
                  无干扰保护，但实际可用性很高

所有用户由 SAS (Spectrum Access System) 集中协调：
  ┌─────┐
  │ SAS │ ◄── 实时监测 Incumbent 活动
  └──┬──┘     动态分配频谱给 PAL/GAA
     │
  ┌──▼──┐  ┌──▼──┐  ┌──▼──┐
  │CBSD1│  │CBSD2│  │CBSD3│  ← CBRS 基站
  └─────┘  └─────┘  └─────┘
```

GAA 层级对私有 5G 特别有吸引力：免费、无需竞标、80 MHz 带宽足够部署一个完整的小型 5G 网络。实际测试中，GAA 在大多数地区的可用性超过 99%（因为海军雷达主要在海岸线附近活动）。

### 2.3 Sub-6G vs mmWave

| 参数 | Sub-6G (n78/CBRS) | mmWave (n257/n258) |
|------|-------------------|-------------------|
| 频率 | 3.5-3.8 GHz | 24-39 GHz |
| 覆盖 | 200-500 m (室内) | 50-150 m (室内) |
| 带宽 | 100-150 MHz | 400 MHz-1 GHz |
| 峰值速率 | 1-2 Gbps | 5-10 Gbps |
| 穿墙能力 | 中等 | 极差 |
| 设备成本 | 中 | 高 |
| 适用场景 | 工厂、园区、仓库 | 密集室内、体育场 |

对于工业 IoT，Sub-6G（特别是 n78/CBRS）是当前的主流选择。mmWave 的覆盖太小，部署成本过高，仅适合少数高密度场景。

## 3. SA vs NSA 组网

### 3.1 两种架构对比

| 特性 | SA (独立组网) | NSA (非独立组网) |
|------|-------------|----------------|
| 核心网 | 5GC (新) | 4G EPC (复用) |
| 基站 | 仅 gNB | gNB + eNB |
| 网络切片 | 支持 | 不支持 |
| URLLC | 支持 | 有限支持 |
| 部署复杂度 | 高 | 低 |
| 成本 | 高 | 低 |

**对于私有 5G，强烈建议 SA 组网**。原因：

1. NSA 需要依赖 4G 基站作为锚点，增加了不必要的复杂度
2. 网络切片是私有 5G 的核心价值之一，NSA 不支持
3. URLLC（超可靠低延迟）只有 SA 才能完整实现
4. 私有网络是从零开始建设，没有"复用现有 4G"的包袱

### 3.2 核心网部署选项

私有 5G 的核心网可以"小到一台服务器"：

```python
# 私有 5G 核心网规模估算
def estimate_core_resources(num_devices, throughput_gbps):
    """估算私有 5GC 所需的计算资源"""
    
    # AMF (接入和移动性管理): ~0.1 vCPU / 100 设备
    amf_vcpu = max(2, num_devices / 100 * 0.1)
    
    # SMF (会话管理): ~0.1 vCPU / 100 设备  
    smf_vcpu = max(2, num_devices / 100 * 0.1)
    
    # UPF (用户面): 取决于吞吐量, ~1 vCPU / Gbps
    upf_vcpu = max(4, throughput_gbps * 1)
    
    total_vcpu = amf_vcpu + smf_vcpu + upf_vcpu
    ram_gb = total_vcpu * 2  # 经验值: 每 vCPU 配 2GB RAM
    
    return {
        'total_vcpu': round(total_vcpu),
        'ram_gb': round(ram_gb),
        'note': '可部署在单台边缘服务器上' if total_vcpu < 32 else '建议多台服务器'
    }

# 示例: 500 个 IoT 设备, 2 Gbps 总吞吐
result = estimate_core_resources(500, 2)
print(f"vCPU: {result['total_vcpu']}, RAM: {result['ram_gb']} GB")
print(f"建议: {result['note']}")
# 输出: vCPU: 9, RAM: 18 GB — 一台普通服务器就够了
```

开源 5GC 选择：

| 项目 | 授权 | 成熟度 | 特点 |
|------|------|-------|------|
| Open5GS | AGPL-3.0 | 高 | 社区最活跃，功能最完整 |
| free5GC | Apache-2.0 | 中 | 台湾交通大学主导，商用友好 |
| OAI (OpenAirInterface) | OAI License | 中 | 含 RAN + Core，学术研究首选 |
| Aether (ONF) | Apache-2.0 | 中 | SD-Core + SD-RAN，企业级方案 |

## 4. URLLC：工业控制的无线化

### 4.1 什么是 URLLC

URLLC（Ultra-Reliable Low-Latency Communication）是 5G 三大应用场景之一，专为"出错就有严重后果"的场景设计：

- **Ultra-Reliable**：99.999% 的传输成功率（即百万个包最多丢 10 个）
- **Low-Latency**：端到端延迟 < 1 ms（用户面单向）

### 4.2 URLLC 实现机制

| 机制 | 作用 | 对延迟的贡献 |
|------|------|------------|
| Mini-slot 调度 | 用 2/4/7 个 OFDM 符号代替完整 slot (14 符号) | 减少调度延迟 50-85% |
| 免授权上行 (Grant-free) | 设备直接发送，不用等基站授权 | 减少 ~0.5 ms |
| HARQ 快速反馈 | 缩短重传等待时间 | 减少重传延迟 |
| 频率分集 | 在多个频率上冗余发送 | 提升可靠性 |
| 多 TRP (发射接收点) | 多个天线面板同时服务 | 提升覆盖可靠性 |

### 4.3 实际延迟预算

一个典型的工业控制场景的延迟分解：

```
PLC 控制指令 → AGV 执行（端到端延迟预算 10 ms）

应用处理:        1 ms
核心网 (UPF):    0.5 ms
传输网 (前传):   0.2 ms
RAN 调度等待:    0.5 ms  ← mini-slot 缩短
空口传输:        0.3 ms
UE 处理:         0.5 ms
────────────────────────
总计:            3.0 ms  ← 远低于 10 ms 预算

对比 WiFi 6:
CSMA/CA 退避:    2-10 ms (不确定)
重传:            5-20 ms (可能)
总计:            7-30 ms (不确定) ← 无法保证上界
```

关键区别：5G URLLC 提供的是**有界延迟**（bounded latency），而 WiFi 的 CSMA/CA 是概率性的——99% 的时候延迟还行，但 1% 的时候可能飙到几十 ms。对于工业控制，这 1% 的尾部延迟就是事故风险。

## 5. 网络切片用于 IoT

### 5.1 网络切片概念

网络切片允许在同一个物理 5G 网络上创建多个逻辑上隔离的虚拟网络，每个切片有独立的 QoS 配置：

```
物理 5G 网络
┌─────────────────────────────────────────────┐
│  ┌──────────────┐  切片 1: 工业控制           │
│  │ URLLC        │  延迟 < 5ms, 可靠性 99.99%  │
│  │ AGV, PLC     │  带宽: 50 Mbps (保证)        │
│  └──────────────┘                             │
│  ┌──────────────┐  切片 2: 视频监控            │
│  │ eMBB         │  带宽: 500 Mbps (尽力)       │
│  │ 摄像头 ×20   │  延迟不敏感                   │
│  └──────────────┘                             │
│  ┌──────────────┐  切片 3: 传感器             │
│  │ mMTC         │  连接数: 5000 设备            │
│  │ 温度/振动    │  低功耗优先                    │
│  └──────────────┘                             │
└─────────────────────────────────────────────┘
```

### 5.2 切片配置示例

在 Open5GS 中，网络切片通过 S-NSSAI（Single Network Slice Selection Assistance Information）标识：

```yaml
# Open5GS 网络切片配置示例 (open5gs.yaml)
amf:
  plmn_id:
    mcc: 999    # 私有网络 MCC
    mnc: 70     # 私有网络 MNC
  s_nssai:
    - sst: 1     # 切片 1: eMBB (视频监控)
      sd: 0x000001
    - sst: 2     # 切片 2: URLLC (工业控制)
      sd: 0x000002
    - sst: 3     # 切片 3: mMTC (大量传感器)
      sd: 0x000003

smf:
  session:
    - subnet: 10.45.0.0/16     # 切片 1 子网
      s_nssai: { sst: 1, sd: 0x000001 }
      qos:
        index: 9               # Non-GBR, 尽力而为
        arp: { priority: 10 }
    - subnet: 10.46.0.0/16     # 切片 2 子网
      s_nssai: { sst: 2, sd: 0x000002 }
      qos:
        index: 82              # GBR, 延迟 < 10ms
        arp: { priority: 1 }   # 最高优先级
    - subnet: 10.47.0.0/16     # 切片 3 子网  
      s_nssai: { sst: 3, sd: 0x000003 }
      qos:
        index: 79              # Non-GBR, 延迟容忍
        arp: { priority: 15 }  # 最低优先级
```

## 6. 成本分析：私有 5G vs WiFi 6

这是企业决策者最关心的问题。以一个 10,000 m² 的工厂为例：

### 6.1 初始部署成本

| 成本项 | 私有 5G (SA) | WiFi 6E |
|-------|-------------|---------|
| 基站/AP 数量 | 3-5 个小基站 | 15-25 个 AP |
| 基站/AP 单价 | $8,000-15,000 | $500-1,500 |
| 核心网/控制器 | $30,000-80,000 | $5,000-15,000 |
| 频谱授权 | $0 (CBRS GAA) - $50,000 (PAL) | $0 (免授权) |
| SIM/eSIM | $3-5/个 × 设备数 | $0 |
| 安装调试 | $20,000-50,000 | $5,000-15,000 |
| **总计 (500 设备)** | **$100,000-250,000** | **$20,000-60,000** |

### 6.2 运营成本 (年)

| 成本项 | 私有 5G | WiFi 6E |
|-------|---------|---------|
| 运维人员 | 需要专业工程师 ($50K+) | IT 人员即可 |
| 软件许可 | $10,000-30,000/年 | $3,000-8,000/年 |
| 电力 | 较高 (基站功耗大) | 较低 |
| **年运营成本** | **$60,000-100,000** | **$10,000-25,000** |

### 6.3 什么时候值得投资私有 5G？

| 条件 | WiFi 足够 | 需要私有 5G |
|------|----------|-----------|
| 延迟要求 | > 20 ms | < 10 ms (确定性) |
| 可靠性要求 | 99.9% | 99.999% |
| 移动设备 | 低速 (< 5 km/h) | 高速 AGV (20+ km/h) |
| 安全性 | 标准企业级 | 国防/关键基础设施级 |
| 设备密度 | < 500/AP | > 1000/小区 |
| 覆盖面积 | < 5,000 m² | > 10,000 m² |

## 7. 实践建议

### 7.1 初学者入门路径

1. **概念学习**：了解 5G 三大场景（eMBB/URLLC/mMTC）和网络切片的基本原理
2. **模拟实验**：用 Open5GS + srsRAN 搭建软件定义的私有 5G 测试环境（需要 SDR 硬件如 USRP B210，约 $2,000）
3. **频谱调研**：了解你所在地区的私有 5G 频谱政策（中国目前尚在讨论阶段）
4. **参观案例**：联系设备供应商（如华为/诺基亚/爱立信）参观已有的私有 5G 工厂示范点
5. **小规模试点**：先在一个车间或产线部署 1-2 个基站，验证覆盖和延迟
6. **对比评估**：同时搭建 WiFi 6E 做对比测试，用数据驱动最终决策

### 7.2 具体调优建议

- **覆盖规划**：工业环境金属反射严重，务必做现场无线勘测（site survey）。仿真工具（如 Ranplan、iBwave）可以辅助但不能替代实测
- **频谱选择**：CBRS GAA 在美国是性价比最高的选择。如果在中国，建议关注工信部 5.9 GHz/6 GHz 频段的政策动向
- **Open RAN 考量**：Open RAN 降低了设备锁定风险，但互操作性测试成本不低。中小企业建议用传统设备商的端到端方案
- **安全**：私有 5G 数据不出园区是核心卖点——确保 UPF（用户面功能）部署在本地，不要把用户面流量回传到运营商核心网
- **与 WiFi 共存**：私有 5G 不是替代 WiFi，而是补充。日常办公用 WiFi，关键业务用 5G。两个网络的管理系统建议统一
- **渐进式部署**：先从一个切片（如视频监控 eMBB）开始，验证效果后再扩展 URLLC 切片
