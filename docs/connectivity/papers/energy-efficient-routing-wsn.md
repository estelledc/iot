# 无线传感器网络能量高效路由协议
> **难度**: 高级 | **领域**: 路由协议 | **阅读时间**: 约 22 分钟

## 引言

想象一个快递配送网络。每个快递员都有有限的油量(电池), 如果所有包裹都走同一条最短路线, 这条路上的快递员很快就会把油耗光, 整条路就断了。聪明的做法是把包裹分散到不同路线上, 让每个快递员的工作量大致均衡, 这样整个网络能运转更长时间。

无线传感器网络(WSN)的路由问题本质上就是这个配送问题。每个传感器节点靠电池供电, 无线通信是最大的耗电来源。路由协议决定了数据经过哪些节点传到汇聚节点(sink), 直接影响哪些节点先耗尽电量、网络能运行多久。能量高效路由的目标不仅是节省总能量, 更重要的是均衡能量消耗, 避免局部节点过早死亡导致网络分裂。

## 1. 能量作为首要约束

### 1.1 WSN节点的能量预算

```
典型WSN节点能量分布:
  无线通信: 60-80% (发射 + 接收)
  数据处理: 10-20%
  传感采集: 5-10%
  待机/休眠: 5-10%

发射一个bit的能耗约等于执行1000条CPU指令
=> 路由协议的优化空间远大于算法优化
```

### 1.2 不均衡路由的后果

如果所有数据都走能耗最小的最短路径, 靠近sink的节点会承载所有流量。这些"热点"节点最先耗尽电量, 导致网络分裂: 即使远端节点还有大量剩余能量, 数据也无法到达sink。

```
不均衡路由示意:
  [远端节点群] ---> [中间节点] ---> [近sink节点] ---> [sink]
                                    ^^^^^^^^^^^
                                    承载所有流量
                                    最先死亡
                                    
  后果: 近sink节点死亡后, 整个网络与sink断开
```

## 2. 网络生命周期定义

### 2.1 多种定义方式

网络生命周期没有统一定义, 常见的有:

```
定义1: 第一个节点死亡的时刻
  最严格, 适合要求100%覆盖的应用(如安全监控)

定义2: 一定比例(如50%)节点死亡的时刻
  适合冗余部署的应用

定义3: 网络发生分裂(sink不可达)的时刻
  关注连通性而非个体节点

定义4: 覆盖率降至阈值以下的时刻
  关注感知质量
```

### 2.2 定义对路由设计的影响

选择不同的生命周期定义会导致完全不同的最优路由策略。以"第一个节点死亡"为目标时, 路由必须严格均衡所有节点的能耗; 以"连通性"为目标时, 可以允许非关键节点率先死亡而保护关键路径上的节点。

## 3. 扁平路由vs分层路由

### 3.1 扁平路由

所有节点地位平等, 任何节点都可参与路由转发:

```
扁平路由特点:
  - 无需组织结构, 部署简单
  - 所有节点执行相同的路由逻辑
  - 代表协议: SPIN, Directed Diffusion
  - 适合: 小规模网络, 节点同构
```

### 3.2 分层路由

节点组织成簇, 簇头负责聚合本簇数据并转发:

```
分层路由特点:
  - 簇内通信: 成员 -> 簇头 (短距离)
  - 簇间通信: 簇头 -> sink (长距离或多跳)
  - 数据聚合: 簇头合并多个成员的数据再转发
  - 代表协议: LEACH, HEED, PEGASIS
  - 适合: 大规模网络, 数据可聚合
```

### 3.3 对比

| 特征 | 扁平路由 | 分层路由 |
|------|---------|---------|
| 可扩展性 | 差(路由表膨胀) | 好(局部管理) |
| 数据聚合 | 不天然支持 | 天然支持 |
| 能量效率 | 中等 | 高(聚合减少传输) |
| 复杂度 | 低 | 中(需要簇维护) |
| 健壮性 | 高(无单点依赖) | 中(簇头是关键) |

## 4. 最小能量路由

### 4.1 贪心最短路径

最直觉的方法: 选择总传输能耗最小的路径, 即选择链路质量最好(路径损耗最小)的邻居转发:

```
最小能量路由:
  每条链路的代价 = 传输该链路所需的能量
  目标: min sum(E_link_i) 对路径上所有链路

  问题: 总是选择同一条最优路径
  结果: 最优路径上的节点被反复使用, 快速耗尽
```

### 4.2 为什么最小能量不等于最长生命

```
示例:
  路径A: 每包能耗 10uJ, 路径上节点初始能量 100mJ
  路径B: 每包能耗 15uJ, 路径上节点初始能量 200mJ

  纯最小能量: 总选路径A
  路径A节点寿命: 100mJ / 10uJ = 10000包后死亡
  
  如果50%走A, 50%走B:
  路径A: 50000包时能耗50mJ, 路径B: 50000包时能耗75mJ
  两条路径都还活着, 网络寿命翻倍
```

## 5. 最大生命周期路由

### 5.1 核心思想

分散流量以均衡能量消耗, 让剩余能量最少的节点尽量少承载流量:

```
最大生命周期路由:
  目标: max min(E_remain_i / E_consumed_rate_i)
  即: 最大化网络中最快耗尽的节点的预期寿命

  策略: 将流量路由到剩余能量多的节点
        避免反复使用同一批节点
```

### 5.2 与最小能量路由的对比

最小能量路由优化的是每个数据包的传输代价, 最大生命周期路由优化的是整个网络的运行时长。后者单包可能更耗能(走了更长的路), 但网络整体收集到的总数据量远多于前者。

## 6. SPIN协议

### 6.1 协议概述

SPIN(Sensor Protocols for Information via Negotiation)通过元数据协商避免冗余传输:

```
SPIN三步握手:
  1. ADV: 节点A有新数据, 广播元数据描述
  2. REQ: 节点B对数据感兴趣, 发送请求
  3. DATA: A将实际数据发送给B

  优势:
  - 避免泛洪(flooding)的冗余
  - 节点只接收自己需要的数据
  - 能量感知: 节点在能量低时可拒绝REQ
```

### 6.2 能量感知机制

SPIN的能量感知体现在: 节点在转发数据前检查自身剩余能量, 如果低于阈值则拒绝参与转发。这是一种简单但有效的自我保护机制。

## 7. Directed Diffusion

### 7.1 数据中心范式

Directed Diffusion是以数据为中心的路由, 不使用地址:

```
Directed Diffusion工作流程:
  1. Sink发布interest(对某类数据的需求)
     例: "温度>50, 区域=(10,20)-(30,40), 每5秒一次"
     
  2. Interest在网络中扩散, 建立梯度(gradient)
     每个节点记住interest来的方向
     
  3. 匹配的传感器沿梯度反向发送数据
     数据沿多条路径流向sink
     
  4. Sink选择最好的路径进行强化(reinforcement)
     增加好路径的数据率, 减少差路径的
```

### 7.2 能量效率特点

Directed Diffusion的能量效率来自两方面: 只传输sink感兴趣的数据(减少无用传输), 中间节点可以聚合来自多个源的数据(减少总传输量)。

## 8. 地理路由

### 8.1 贪心地理转发

利用节点位置信息进行路由, 每一跳选择地理上最接近目的地的邻居:

```
贪心地理路由(GPSR):
  当前节点位置: (x1, y1)
  目的节点位置: (xd, yd)
  
  对每个邻居Ni位于(xi, yi):
    计算 dist(Ni, D) = sqrt((xi-xd)^2 + (yi-yd)^2)
  
  选择: 距离D最近的邻居作为下一跳

  优势:
  - 不需要路由表, 扩展性好
  - 每跳距离短, 发射功率低
  - 决策简单, 计算开销小
```

### 8.2 空洞问题

当贪心路由遇到"空洞"(没有比当前节点更接近目的的邻居)时会失败。解决方案是切换到周边路由(perimeter routing), 沿空洞边缘绕行:

```
空洞场景:
  源S -> A -> B -> (空洞区域, 无节点) -> D
  
  B的所有邻居都比B离D更远
  贪心路由在B处卡住
  
  GPSR解决: 切换到周边模式, B沿空洞边界转发
  直到找到比B更接近D的节点, 恢复贪心模式
```

## 9. 多路径路由

### 9.1 基本思想

发现多条从源到sink的路径, 将流量分散到各条路径上:

```
多路径路由策略:
  发现K条不相交路径: P1, P2, ..., PK
  
  流量分配方式:
  1. 均匀分配: 每条路径 1/K 的流量
  2. 按能量分配: 剩余能量多的路径分更多流量
  3. 按质量分配: 链路质量好的路径分更多流量

  优势:
  - 均衡能量消耗
  - 容错: 一条路径失效其他路径仍可用
  
  代价:
  - 路径发现和维护的开销
  - 需要拓扑足够冗余以支持多条不相交路径
```

## 10. 能量收集对路由的影响

### 10.1 新的优化维度

当节点配备太阳能/振动/热能收集器时, 能量不再只是消耗而是可以补充:

```
传统路由优化:
  目标: 最小化能量消耗
  约束: 能量只减不增

能量收集路由优化:
  目标: 最大化数据吞吐量
  约束: 能量可补充, 但收集速率有限
  
  新策略: 优先使用正在收集能量的节点
          避免浪费已满的节点的收集能量(能量溢出)
          路由决策考虑当前能量 + 预期收集速率
```

### 10.2 动态路由调整

能量收集使得最优路由随时间变化(白天太阳能充足的节点多承担, 夜间切换到其他节点), 要求路由协议具有动态适应能力。

## 11. 数据聚合路由

### 11.1 聚合的价值

中间节点将来自多个源的数据合并后再转发, 显著减少传输次数:

```
无聚合: 10个源各发1个包 = 中间节点转发10个包
聚合(取平均): 10个源各发1个包 = 中间节点转发1个包(平均值)
能耗减少: 90%

常用聚合函数: 平均值, 最大值, 最小值, 计数
适用条件: 应用可以容忍聚合后的精度损失
```

### 11.2 聚合感知路由

路由设计应使来自不同源的数据尽早在中间节点汇合, 最大化聚合效果:

```
聚合感知路由:
  非聚合路由:
    S1 -> A -> B -> sink
    S2 -> C -> D -> sink
    总传输: 6次
    
  聚合路由:
    S1 -> A -> B -> sink
    S2 -> C -> B -> sink  (C转发到B而不是D)
    B聚合后发送1个包
    总传输: 5次(B只发1个聚合包)
```

树形拓扑天然适合聚合: 叶子节点是源, 中间节点聚合, 根节点是sink。

## 12. 实际案例: 森林环境监测

### 12.1 场景描述

200个电池供电的传感器部署在森林中, 测量温度和湿度, 汇聚节点在护林站。

### 12.2 最小能量路由的结果

```
最短路径路由:
  - 靠近护林站的节点承载所有流量
  - 这些节点3个月耗尽电量
  - 网络分裂: 远端节点数据无法到达
  - 远端节点剩余80%以上能量被浪费
```

### 12.3 最大生命周期路由的结果

```
能量均衡路由(地理路由 + 能量感知):
  每个传感器知道GPS位置
  转发选择: 距sink最近且剩余能量最多的邻居
  
  score = w1 * (1/dist_to_sink) + w2 * E_remain/E_max
  
  结果:
  - 网络运行18个月(最小能量方案的6倍)
  - 所有节点大约同时耗尽电量
  - 总采集数据量: 6倍于最小能量方案
  - 实现: 简单, 每个节点仅需位置和邻居能量信息
```

### 12.4 对比总结

| 指标 | 最小能量路由 | 最大生命周期路由 |
|------|------------|----------------|
| 网络寿命 | 3个月 | 18个月 |
| 总数据量 | 基准 | 6倍 |
| 节点死亡模式 | 近sink先死 | 大致同时 |
| 能量浪费 | 严重(远端节点) | 极少 |

## 总结

WSN能量高效路由的核心不是最小化单包能耗, 而是均衡整个网络的能量消耗以最大化网络生命周期。扁平路由如SPIN和Directed Diffusion适合小规模网络, 分层路由如LEACH通过聚合显著减少传输量。地理路由利用位置信息实现无路由表的高效转发, 多路径路由通过流量分散提升能量均衡和容错性。能量收集技术引入了新的优化维度, 使路由决策需要动态适应能量补充速率。
