# 设备到设备D2D通信在IoT中的应用
> **难度**: 高级 | **领域**: D2D通信 | **阅读时间**: 约 22 分钟

## 引言

想象住在一栋公寓楼里, 你想把一张照片发给隔壁邻居。正常的做法是先上传到云端服务器(几百公里外的数据中心), 服务器再推送给邻居。照片明明只需穿过一堵墙, 却跑了几百公里的来回。

D2D(Device-to-Device)通信就是让距离很近的两个设备直接对话, 不再绕路经过基站或接入点。在传统蜂窝网络中, 所有通信都必须走"设备-基站-设备"的路径, 即使两个设备相隔只有几米。D2D打破了这个限制, 允许近距离设备建立直接链路。

这带来三个核心好处: 更低的延迟(不绕远路), 更少的网络负载(基站不用中转本地流量), 以及基础设施不可用时仍能通信(如灾后救援)。对IoT来说, 工厂传感器可以直接把数据交给旁边的控制器, 不必经过远处的网关。

## 1. D2D通信基本概念

### 1.1 传统路径与D2D路径对比

在传统蜂窝架构中, 即使两个设备近在咫尺, 数据也必须经过基站:

```
传统路径:
  设备A ---(上行链路)---> 基站 ---(下行链路)---> 设备B
  延迟: 上行5ms + 核心网处理 + 下行5ms = 10-20ms
  能耗: 两次长距离传输

D2D路径:
  设备A ---(直接链路)---> 设备B
  延迟: < 1ms
  能耗: 一次短距离传输
```

### 1.2 核心特征

D2D通信有几个定义性特征。首先是近距离性, 通常D2D通信发生在几十米到几百米的范围内。其次是直接性, 数据不经过任何中间基础设施节点。第三是资源效率, 由于传输距离短, 所需发射功率远低于常规蜂窝通信。

从频谱角度看, D2D可使用授权频段(与蜂窝共享)或非授权频段(如ISM频段)。从管理角度看, D2D可以是运营商控制的(基站协调资源分配)或完全自主的(设备自行决定)。

### 1.3 与其他近距离技术的区别

蜂窝D2D与Wi-Fi Direct和蓝牙有本质区别。后两者工作在非授权频段, 干扰不可控。蜂窝D2D工作在授权频段内, 由基站统一管理资源:

| 特征 | 蜂窝D2D | Wi-Fi Direct | 蓝牙 |
|------|---------|-------------|------|
| 频段 | 授权频段 | 非授权5GHz | 非授权2.4GHz |
| 管理方式 | 基站协调 | 自主协商 | 自主配对 |
| 覆盖范围 | 几十到几百米 | 约50米 | 约10-30米 |
| QoS保障 | 有(资源预留) | 有限 | 有限 |
| 干扰管理 | 系统级优化 | 退避机制 | 跳频 |

## 2. 蜂窝框架中的D2D标准化

### 2.1 LTE ProSe(Proximity Services)

3GPP从Release 12开始标准化D2D通信, 称为ProSe。推动力来自公共安全需求: 消防员在建筑物内需要相互通信, 但蜂窝基站可能已损毁。

LTE ProSe定义了两个核心功能: 直接发现(Direct Discovery)让设备找到附近的ProSe设备, 直接通信(Direct Communication)建立数据传输链路。通信使用蜂窝上行频段中的专用资源, 称为sidelink。

### 2.2 5G NR Sidelink

5G NR在Release 16中大幅增强sidelink能力, 主要面向V2X但同样适用于IoT:

```
5G NR Sidelink 关键增强:
  - 物理层: OFDM, 灵活子载波间隔(15/30/60kHz)
  - 反馈: HARQ(混合自动重传请求)
  - 信道估计: CSI(信道状态信息)反馈
  - 功率控制: 开环+闭环
  - 频段: FR1(sub-6GHz)和FR2(毫米波)
```

### 2.3 资源分配模式

5G NR sidelink定义了两种资源分配模式:

Mode 1(网络调度): 基站为D2D分配具体时频资源。设备向基站发送调度请求, 基站根据全局信息做最优分配。干扰管理最好, 但需在基站覆盖内。

Mode 2(设备自主): 设备通过sensing机制检测哪些资源被占用, 然后选择空闲资源。不依赖基站, 适合覆盖外场景, 但可能产生资源冲突。

## 3. D2D设备发现机制

### 3.1 发现的重要性

D2D通信前, 设备首先需要发现附近可通信的对等设备。这个过程面临根本矛盾: 要发现其他设备就需要发送或监听信号, 而这会消耗能量。对电池供电的IoT设备, 发现开销可能成为能耗主要来源。

### 3.2 基于信标的发现

设备周期性广播包含自身标识的信标(beacon):

```
发现信标结构:
  +------------------+------------------+------------------+
  | 设备标识 (16bit) | 服务类型 (8bit)  | 发现序列号 (8bit) |
  +------------------+------------------+------------------+

  发送周期: 每 T_disc 秒广播一次
  功耗模型: P_disc = P_tx * t_beacon / T_disc

  示例:
  T_disc = 1s, t_beacon = 1ms, P_tx = 10mW
  P_disc = 10mW * 1ms / 1s = 0.01mW (平均功耗)
```

### 3.3 网络辅助发现

基站知道所有注册设备的位置信息, 可以直接告知设备哪些对等设备在附近。优势是无需持续广播或监听, 大幅降低发现能耗。代价是依赖基站覆盖, 基站不可用时无法工作。

### 3.4 主动与被动发现的权衡

主动发现(proactive)始终在发现状态, 随时知道附近有哪些设备, 通信时无延迟但持续消耗能量。被动发现(reactive)只在需要通信时启动发现, 能耗低但增加初始延迟。

对IoT场景的建议: 周期性数据采集适合被动发现(每次采集前发现), 紧急告警适合主动发现(需要立即通信)。

## 4. D2D通信模式详解

### 4.1 带内下行复用模式(In-band Underlay)

D2D复用蜂窝上行链路资源, D2D设备和蜂窝用户在相同时频资源上同时传输:

```
频谱资源示意:
  频率
  ^
  |  [蜂窝UE1] [D2D对]  [蜂窝UE2] [D2D对]
  |  [蜂窝UE3] [蜂窝UE4] [D2D对]  [蜂窝UE5]
  +-----------------------------------------> 时间

  优点: 不需要额外频谱, 频谱效率最高
  缺点: D2D和蜂窝之间存在互干扰
```

### 4.2 带内覆盖模式与带外模式

带内覆盖(In-band Overlay)从蜂窝频谱中划出专用D2D资源, 无跨模式干扰但减少蜂窝资源。带外(Out-of-band)使用ISM频段与蜂窝完全隔离, 不影响蜂窝但ISM拥挤不可控。

### 4.3 三种模式对比

| 指标 | 带内下行复用 | 带内覆盖 | 带外 |
|------|------------|---------|------|
| 频谱效率 | 最高 | 中等 | 取决于ISM |
| 干扰管理难度 | 高 | 低 | 低 |
| 实现复杂度 | 高 | 中 | 低 |
| 蜂窝影响 | 有(互干扰) | 有(资源减少) | 无 |
| QoS保障 | 可控 | 可控 | 不可控 |

## 5. 资源管理与干扰控制

### 5.1 干扰问题

D2D复用蜂窝频谱时产生两类干扰: D2D发射机对基站的干扰(D2D信号干扰基站接收蜂窝用户), 蜂窝用户对D2D接收机的干扰(蜂窝上行信号干扰D2D接收设备)。

### 5.2 模式选择策略

基站需要决定一对设备是D2D直连还是蜂窝中转:

```
模式选择判断:
  1. 测量设备间距离 d_D2D
  2. 测量设备到基站距离 d_BS
  3. 估算D2D直连信道质量 SINR_D2D
  4. 估算蜂窝中转信道质量 SINR_cellular

  判断:
  if d_D2D < D_threshold AND SINR_D2D > SINR_min:
      选择D2D模式
  else:
      选择蜂窝中转模式

  典型阈值:
  D_threshold = 50m(室内) / 200m(室外)
  SINR_min = 5dB
```

### 5.3 功率控制

D2D使用低功率是关键特征。由于传输距离短, 不需要高发射功率:

```
功率与距离对比:
  蜂窝上行: 设备到基站500m, 发射功率23dBm(200mW)
  D2D直连:  设备间距离20m,  发射功率0dBm(1mW)
  功率差异: 200倍

  路径损耗对比(自由空间, 900MHz):
  500m: PL = 85.5dB
  20m:  PL = 57.5dB
  差值: 28dB
```

基站可设定D2D设备的最大发射功率, 确保对蜂窝用户的干扰在可接受范围内。

## 6. D2D在IoT中的应用场景

### 6.1 本地数据交换

近距离传感器直接共享测量数据, 无需经过网关。例如温度传感器和湿度传感器交换数据进行本地环境判断, 只有异常结果才上报网关, 减少网络上行流量。

### 6.2 设备中继

处于网络边缘的IoT设备无法直接连接基站, 但附近有信号良好的设备。通过D2D, 边缘设备借助近邻设备中继数据:

```
两跳中继路径:
  边缘设备 ---(D2D)--> 中继设备 ---(蜂窝)--> 基站

  场景示例:
  地下室传感器 ---(D2D)--> 一楼设备 ---(NB-IoT)--> 基站

  好处: 扩展有效覆盖, 无需部署额外基站
```

### 6.3 本地数据聚合

密集部署中, 一个设备作为簇头通过D2D收集附近设备的数据, 聚合后统一上报。减少网络上行流量和基站处理负担。

### 6.4 应急通信

基础设施因自然灾害不可用时, D2D允许设备维持本地通信。这是3GPP推动ProSe标准化的核心动力, 对公共安全和灾后救援至关重要。

### 6.5 近距离服务

通过测量D2D信号强度或到达时间, 估算设备间距离和相对位置, 实现资产追踪和室内定位。

## 7. D2D中继扩展IoT覆盖

### 7.1 覆盖问题

对发射功率受限的IoT设备, 上行覆盖是瓶颈: 设备信号到不了基站, 即使基站下行信号能到达设备。

### 7.2 工作原理

```
覆盖扩展示意:

  基站有效覆盖范围
  +------------------------------------------+
  |                                          |
  |   [基站]                                 |
  |     |  蜂窝链路(良好)                     |
  |   [中继设备]                              |
  |     |                                    |
  +-----|--------------------------------------+
        |  D2D链路(短距离)
      [边缘设备]  <-- 在覆盖范围外

  等效路径: 边缘设备 -> 中继设备 -> 基站
```

### 7.3 适用场景

建筑阴影区域, 地下室或地下管道中的传感器, 农村蜂窝覆盖边缘, 以及高层建筑信号穿透困难的楼层。

## 8. D2D的能量效益

### 8.1 发射功率节省

```
能量对比:
  蜂窝传输:
    发射功率: 23dBm = 200mW
    数据包持续时间: 1ms
    能耗: 200mW * 1ms = 200uJ

  D2D传输(20m距离):
    发射功率: 0dBm = 1mW
    数据包持续时间: 1ms
    能耗: 1mW * 1ms = 1uJ

  能耗比: 200:1

  每天100个包的IoT设备:
    蜂窝: 100 * 200uJ = 20mJ/天
    D2D:   100 * 1uJ = 0.1mJ/天
```

### 8.2 链路预算优势

短距离传输意味着更低的路径损耗, 在相同接收灵敏度下D2D可用更低的发射功率达到同等通信质量。

### 8.3 开销考量

D2D并非没有额外能耗。设备发现(尤其主动发现的持续监听)、链路管理(建立、维护、释放)都有信令开销。D2D的能量效益只有在数据传输量足够大或频率足够高时才能抵消这些管理开销。

## 9. 安全与隐私挑战

### 9.1 安全威胁

D2D直接链路绕过了网络核心安全设施:

```
安全威胁模型:
  1. 身份伪造: 攻击者冒充合法D2D设备
  2. 窃听: 直接链路上的数据被截获
  3. 中间人攻击: 攻击者插入D2D链路
  4. 拒绝服务: 干扰D2D发现或通信过程
```

### 9.2 解决方案

3GPP为ProSe定义了安全框架。网络辅助场景下, 基站协助认证和密钥分发。网络不可用时, 使用预共享密钥或基于证书的认证。

### 9.3 隐私保护

D2D近距离特性暴露设备位置, 发现信标可能被用来追踪移动轨迹。解决方案包括使用临时标识和定期更换信标中的设备标识符。

## 10. 5G NR Sidelink在IoT中的应用

### 10.1 标准演进

```
5G NR Sidelink 演进:
  Release 16: V2X核心功能
    - 单播、组播、广播
    - Mode 1(网络调度) + Mode 2(自主选择)

  Release 17: 扩展应用
    - 公共安全增强
    - 功率节省机制
    - 中继增强

  Release 18: IoT增强
    - 轻量级sidelink(RedCap设备)
    - 工业IoT场景优化
    - 低功耗sidelink
```

### 10.2 工厂自动化

NR sidelink支持Mini-slot(2/4/7个OFDM符号), 最小延迟约0.125ms(120kHz SCS), 满足工业实时控制。智能建筑中灯光、空调、门禁设备可通过sidelink直接联动, 不依赖云端控制器。

## 11. 挑战与研究方向

D2D在IoT中面临四个核心挑战。第一是干扰管理: D2D设备密度增加时, D2D链路间也会互干扰, 复杂化资源管理。第二是发现开销: 始终监听耗能过多, 按需发现增加延迟, 需要在两者间找平衡。第三是移动性: 设备相对移动导致D2D链路中断, 需要快速链路切换。第四是标准化缺口: 现有标准主要面向手机和车载终端, 对低功耗IoT设备的特殊需求支持不足。

## 12. 实际案例: 智能工厂本地控制

### 12.1 场景描述

生产线机械臂需要根据附近传感器的实时数据调整抓取动作, 要求响应时间小于1ms。传感器测量工件精确位置, 机械臂据此调整角度。

### 12.2 蜂窝中转的局限

```
延迟分析:
  传感器 -> 基站上行: 5ms
  基站处理: 1ms
  基站 -> 控制器下行: 5ms
  总延迟: 约11ms

  问题: 生产线要求 < 1ms
  11ms延迟导致抓取精度下降, 废品率增加
```

### 12.3 D2D Sidelink方案

传感器和控制器距离仅5米, 使用5G NR sidelink直连:

```
D2D方案参数:
  设备间距离: 5m
  发射功率: -10dBm (0.1mW)
  路径损耗(5m, 3.5GHz): 约55dB
  接收信号功率: -10 - 55 = -65dBm
  接收灵敏度: -100dBm
  链路余量: 35dB (非常充裕)

  延迟: < 0.5ms (Mini-slot传输)
  可靠性: 99.999% (HARQ重传)

  能耗对比:
  蜂窝: 200mW * 1ms = 200uJ/包
  D2D:  0.1mW * 0.5ms = 0.05uJ/包
  能耗比: 4000:1
```

### 12.4 系统架构

```
基站角色: 管理者而非中转者
  - 为关键D2D对分配专用sidelink资源
  - 监控D2D链路质量
  - D2D链路故障时提供蜂窝备份路径

传感器 <----D2D sidelink----> 机械臂控制器
  |                              |
  +----蜂窝上行(非实时数据)-----> 基站 ----> 云端
```

### 12.5 实际效果

延迟从11ms降至0.5ms以下, 满足实时控制需求。D2D使用极低发射功率(-10dBm), 不干扰蜂窝用户。基站为每对关键设备预留专用sidelink资源, 不受网络拥塞影响。即使基站故障, 本地D2D链路仍可维持生产线运行, 实现了本地实时控制独立于蜂窝网络的目标。

## 总结

D2D通信让近距离设备直接对话, 实现更低延迟、更低能耗和更高本地可靠性。3GPP的ProSe和5G NR sidelink提供了标准化框架, 从公共安全扩展到V2X再到工业IoT。核心价值在于认识到不是所有通信都需要经过集中式基础设施, 当两个设备近在咫尺时直接对话才是最高效的选择。资源管理、设备发现和安全性方面仍有技术挑战, 但D2D作为蜂窝网络的补充将在IoT本地通信中发挥越来越重要的作用。
