# 私有LTE CBRS频段IoT部署方案
> **难度**: 高级 | **领域**: 私有网络 | **阅读时间**: 约 22 分钟

## 引言

想象一个巨型仓库里有2000台自动导引车(AGV)在穿梭搬运货物。如果用WiFi,AGV从一个接入点移动到另一个时会"断片"几百毫秒--对于高速移动的机器来说,这几百毫秒可能意味着碰撞事故。如果用公网4G/5G,又要担心外部网络拥堵影响生产。有没有一种方案,既有蜂窝网络的可靠性,又完全由企业自己控制?这就是私有LTE/5G网络。而美国的CBRS频段让这一切变得前所未有地容易--不需要花几十亿买频谱许可证。

本文将深入讲解私有LTE网络的架构、CBRS频段的共享机制、与WiFi的对比优势,以及在IoT场景中的实际部署方法。

## 1. 私有蜂窝网络概述

### 1.1 什么是私有LTE/5G

私有蜂窝网络是由企业或组织自行部署和运营的移动通信网络,不依赖公共运营商:

```
公网模式:
设备 -> 运营商基站 -> 运营商核心网 -> 互联网
(企业无控制权, 共享带宽, 依赖运营商SLA)

私有网络模式:
设备 -> 企业自有基站 -> 企业自有核心网 -> 企业内网/互联网
(企业完全控制, 专用带宽, 自主运维)
```

### 1.2 为什么需要私有网络

| 需求维度 | WiFi痛点 | 公网蜂窝痛点 | 私有LTE优势 |
|----------|----------|-------------|-------------|
| 覆盖范围 | 单AP约30-50m | 依赖运营商 | 单基站300-500m |
| 移动性 | 切换延迟100-500ms | 不可控 | 无缝切换<50ms |
| 容量 | 干扰严重 | 共享不保证 | 专用频谱可预测 |
| 安全性 | 开放频段 | 数据经第三方 | 物理隔离可选 |
| QoS | 尽力而为 | 不可定制 | 完全可定制 |
| 可靠性 | 约99% | 约99.5% | 可达99.999% |

### 1.3 频谱获取方式

部署私有蜂窝网络首先要解决频谱来源:

- **授权频段购买**: 参与拍卖(极贵,运营商专属)
- **授权频段租赁**: 从运营商租用(需谈判,灵活性有限)
- **共享频段**: CBRS(美国)/MulteFire(全球)--无需许可证
- **免授权频段**: 5GHz/6GHz上的LTE-U/LAA(干扰管理复杂)

## 2. CBRS频段详解

### 2.1 什么是CBRS

CBRS(Citizens Broadband Radio Service)是美国FCC在3550-3700MHz频段建立的创新频谱共享框架。这150MHz频谱设计为三层共享:

```
CBRS三层接入架构(优先级从高到低):

第1层: Incumbent Access(在位用户)
- 美国海军雷达、卫星地面站
- 最高优先级,必须被完全保护
- 不需要任何注册

第2层: Priority Access License(PAL)
- 通过拍卖获得的10MHz频道许可
- 按人口普查区划分,10年有效
- 受保护不被第3层干扰

第3层: General Authorized Access(GAA)
- 无需许可证,免费使用
- 最大EIRP 30dBm室内/47dBm室外
- 必须让位给第1层和第2层
- 企业私有网络最常用的方式
```

### 2.2 SAS: 频谱接入系统

CBRS的核心创新是SAS(Spectrum Access System),一个云端动态频谱管理系统:

```
SAS工作流程:
1. CBSD(基站)向SAS注册
   提供: 位置、天线高度、期望功率、频段请求
2. SAS查询Incumbent保护数据库
   确认该位置是否有雷达活动区域
3. SAS计算可用频谱
   考虑: 保护区、PAL持有者、其他GAA用户
4. SAS授权频谱使用
   返回: 允许频段、最大功率、有效期
5. 动态管理
   检测到Incumbent活动时,CBSD须300秒内释放
```

### 2.3 CBSD设备分类

| 参数 | Category A | Category B |
|------|-----------|-----------|
| 部署场景 | 室内 | 室外 |
| 最大EIRP | 30 dBm (1W) | 47 dBm (50W) |
| 天线高度 | 低于6m AGL | 无限制 |
| 覆盖范围 | ~100-300m | ~1-5km |
| 认证要求 | 标准FCC认证 | 需CPI专业安装 |
| 典型用途 | 工厂、仓库 | 园区、矿场、港口 |

## 3. 私有LTE网络架构

### 3.1 核心组件

```
私有LTE网络完整架构:

IoT设备/终端(Cat-M1/Cat-1/Cat-4模组)
         |
         v (Uu空口)
eNodeB(基站)
  - 射频单元(RRU): 信号发射接收
  - 基带处理(BBU): 编解码调度
  - 连接SAS获取频谱授权
         |
         v (S1接口)
EPC(演进分组核心网)
  +------+------+------+------+
  | MME  | S-GW | P-GW | HSS  |
  |信令  |本地GW|出口GW|用户DB|
  +------+------+------+------+
         |
         v
企业应用层(IoT平台/MES/WMS/视频监控)
```

### 3.2 EPC组件说明

| 组件 | 功能 | IoT场景考量 |
|------|------|-------------|
| MME | 移动性管理、认证 | IoT设备多为低移动性 |
| S-GW | 本地数据面锚点 | 数据不出企业网络 |
| P-GW | 策略执行、NAT | IoT可简化计费 |
| HSS | 用户签约数据库 | 存储所有设备SIM信息 |
| PCRF | QoS策略控制 | 为不同流量分级 |

### 3.3 部署模型

**模型一: 完全本地部署**
所有设备在企业机房: eNodeB + EPC + 应用服务器全本地。最低延迟、最高安全、完全自主。适合大型制造企业。

**模型二: 混合部署**
基站本地,核心网在云: eNodeB(本地) + EPC(公有云/边缘云)。降低运维负担,弹性扩展。适合多厂区统一管理。

**模型三: 网络即服务(NaaS)**
全托管: 运营商/集成商提供全套设备和运维,企业只提供场地。零运维快速部署。适合快速验证。

## 4. 私有LTE vs WiFi 6: IoT场景对比

### 4.1 关键性能实测对比

```
典型工厂环境实测数据:

指标              | 私有LTE (CBRS)  | WiFi 6 (802.11ax)
---------------------------------------------------------
覆盖半径          | 300-500m(室内)  | 30-50m(室内)
切换延迟          | <50ms           | 100-500ms
同时连接设备      | 1000+/基站      | 200-300/AP(理论)
上行可靠性        | 99.99%          | 99-99.5%
端到端延迟        | 10-30ms         | 20-100ms(波动大)
频谱干扰          | 几乎无(专用)   | 严重(共享频段)
移动速度支持      | >120km/h        | <30km/h(稳定)
```

### 4.2 IoT特定考量

**连接密度**: 智能工厂5000-10000传感器节点。WiFi密集时CSMA/CA竞争严重,LTE的OFDMA调度能有效管理。

**确定性延迟**: 工业自动化要求确定性。WiFi随机退避无法保证最大延迟上限,LTE调度器可预留资源。

**频段干扰**: 2.4/5GHz被无数设备共享。CBRS 3.5GHz由SAS协调干扰可控。

### 4.3 成本对比(10万平方米仓库)

```
WiFi 6方案:
- AP数量: ~300个(30m覆盖半径)
- 设备+交换机+布线+安装: ~$280,000
- 年运维: $40,000
- 5年TCO: ~$480,000

私有LTE (CBRS GAA)方案:
- 基站数量: ~8个(200m覆盖半径)
- 设备+EPC+SIM/模组+安装: ~$240,000
- 年运维+SAS: $30,000
- 5年TCO: ~$390,000

结论: 大面积场景私有LTE TCO更低且性能显著优于WiFi
```

## 6. 与私有5G的对比

### 6.1 技术演进路径

```
LTE (4G)                    5G NR
+-----------------------+   +-----------------------+
| 成熟稳定              |   | 更高性能              |
| 设备生态丰富          |   | 设备仍在发展中        |
| IoT模组$10-30        |   | IoT模组$30-100       |
| 足够多数IoT场景      |   | mmWave超高带宽        |
+-----------------------+   +-----------------------+

建议路径:
2024-2025: 私有LTE(CBRS GAA/PAL)
2025-2027: LTE+5G NSA混合(渐进升级)
2027+: 独立5G SA(设备成本下降后)
```

### 6.2 5G对IoT的额外价值

- **mMTC**: 单小区支持100万设备连接
- **URLLC**: 延迟低于1ms,可靠性99.99999%
- **网络切片**: 同一物理网络划分多个虚拟专网
- **TSN集成**: 与工业以太网时间敏感网络互通

## 7. 监管与合规

### 7.1 美国CBRS监管要求

```
关键合规要求:
1. 设备认证: 所有CBSD需FCC Part 96认证
2. SAS注册: 使用前必须向认证SAS注册
3. 专业安装: Category B需CPI安装
4. 功率限制: 严格遵守EIRP上限
5. 动态退让: 收到SAS指令300秒内执行
6. ESC配合: 沿海区域需配合环境感知能力
```

### 7.2 其他国家类似框架

| 国家/地区 | 频段 | 框架名称 | 状态 |
|-----------|------|----------|------|
| 美国 | 3.5GHz | CBRS | 商用中 |
| 英国 | 3.8-4.2GHz | Shared Access | 商用中 |
| 德国 | 3.7-3.8GHz | Local 5G License | 商用中 |
| 日本 | 4.6-4.9GHz | Local 5G | 商用中 |
| 中国 | 5.9GHz | 工业互联网专用 | 试点中 |

### 7.3 安全合规

私有网络安全优势: 物理隔离(数据不过公网)、自主SIM管理(认证可控)、LTE内置128bit AES加密、可满足HIPAA/GxP数据不出厂要求。

## 8. 实践案例: 智能仓库2000台AGV

### 8.1 项目背景

某电商巨头新建50万平方米超级仓库,部署2000台AGV全自动化拣选。初期WiFi方案POC暴露严重问题: AGV在AP切换时频繁停车,高峰网络拥堵导致调度延迟,每天因网络问题损失约3小时生产效率。

### 8.2 网络设计

```
覆盖规划:
- 仓库: 1000m x 500m, 层高15m
- CBSD Cat-A基站: 24个(间距~100m, 考虑货架遮挡)
- 频段: CBRS GAA, 40MHz(4个10MHz载波聚合)
- 容量: 每基站~85台AGV + ~200个传感器

核心网:
- 部署: 本地EPC(两台服务器主备)
- MME: 支持5000终端
- S-GW/P-GW: 吞吐2Gbps

终端:
- AGV模组: Quectel RM500Q (CBRS B48)
- 传感器网关: Sierra Wireless RV50X
```

### 8.3 QoS与切换优化

```python
# QoS策略配置
qos_profiles = {
    "agv_control": {
        "qci": 3,
        "gbr_ul": "2 Mbps",
        "gbr_dl": "5 Mbps",
        "max_delay": "20ms",
        "priority": 1,
        "preemption": True
    },
    "agv_video": {
        "qci": 7,
        "ambr_ul": "10 Mbps",
        "max_delay": "100ms",
        "priority": 3
    },

        "qci": 9,

```

切换优化措施: A3事件偏移3dB避免乒乓切换,Time-to-Trigger 160ms,结合AGV路径规划提前触发切换准备(已知路径切换成功率99.99%),相邻小区重叠>30%确保切换中有信号。

## 9. 部署最佳实践

### 9.1 规划阶段

- 射频(RF)勘测: 了解建筑材料、金属货架对信号影响
- 容量需求: 峰值并发设备数、每设备吞吐量
- 频谱选择: GAA(免费但不保证) vs PAL(付费有保障)
- 室外覆盖评估: Category B需CPI安装认证

### 9.2 部署阶段

- 从小规模POC开始验证覆盖和性能
- 使用射频规划工具(如iBwave)模拟
- 天线选型: 全向(一般覆盖) vs 定向(特定区域增强)
- SAS注册测试: 确保频谱授权正常

### 9.3 运维阶段

- 监控SAS连接(断开超时导致基站下线)
- 定期优化切换参数(基于实际KPI)
- 固件升级先在非生产基站测试
- 容灾演练: 模拟基站故障和核心网切换

## 总结

私有LTE/5G网络通过CBRS等共享频谱框架,为企业IoT部署提供了技术可行且经济合理的路径。相比WiFi,私有LTE在覆盖、移动性、可靠性和QoS方面有显著优势,特别适合工业自动化、仓储物流、港口矿场等对确定性连接有刚性需求的场景。CBRS三层共享机制和SAS动态管理大大降低了企业获取频谱的门槛。随着设备生态成熟和成本下降,私有蜂窝网络正从大企业专属工具变为更多组织可采用的基础设施。
