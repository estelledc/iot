# 低功耗广域网LPWAN技术综合综述
> **难度**: 初级 | **领域**: LPWAN | **阅读时间**: 约 18 分钟

## 引言

想象你要给远在几十公里外的农场大棚安装一个温度传感器。这个传感器需要满足三个条件: 电池能撑好几年(因为没人天天去换电池)、信号能穿墙越岭传到几十公里外(因为农场偏远)、不需要太贵(因为要装几千个)。普通WiFi传几十米就没信号了，手机4G虽然覆盖广但太耗电，蓝牙省电但传不远——似乎没有一种现有技术能同时满足这三个需求。

LPWAN(Low Power Wide Area Network, 低功耗广域网)就是为解决这个矛盾而生的技术家族。它用一个简单的交易实现了"鱼和熊掌兼得": 放弃高速数据传输(反正温度传感器每次只发几十个字节)，换来超低功耗和超远传输距离。本文将全面介绍LPWAN的核心技术、主流方案对比，以及如何为你的IoT项目选择最合适的LPWAN技术。

## 1. LPWAN是什么

### 1.1 核心特征

LPWAN不是某一种具体的技术，而是一类满足特定需求的无线通信技术的统称。它们共享三个核心特征:

低功耗(Low Power)意味着设备可以用一颗纽扣电池或AA电池工作数年。这是通过极低的占空比(设备大部分时间在睡眠)和简化的协议栈实现的。

广域覆盖(Wide Area)意味着通信距离可达5-50公里，远超WiFi(100米)和蓝牙(50米)。这是通过降低数据速率来提高接收灵敏度实现的——根据香农定理，带宽越窄、速率越低，同样功率下可以传得越远。

低数据速率(Low Data Rate)是前两者的代价。LPWAN的速率通常在几百bps到几十kbps之间，只适合传输传感器数据、状态信息等小数据量内容。

```
LPWAN 在无线技术图谱中的位置:

            高速率
              |
    WiFi      |     4G/5G
   (近/快)    |    (远/快)
              |
  近距离 -----+------ 远距离
              |
    BLE       |     LPWAN
   (近/慢)    |    (远/慢)
              |
            低速率

LPWAN 填补了"远距离 + 低速率"这个象限
```

### 1.2 市场驱动力

IoT的爆发增长是LPWAN技术兴起的根本驱动力。到2030年预计有250亿以上的IoT设备需要连接，其中大量属于"传感器型"设备: 每隔几分钟甚至几小时才发送一次数据，每次只有几十个字节，但数量庞大、分布广泛、无法频繁更换电池。

传统无线技术无法经济高效地服务这类设备。蜂窝网络(4G/5G)虽然覆盖广，但功耗高、模组成本贵、月租费累积可观。WiFi覆盖距离太短，不适合户外广域部署。LPWAN技术正是为这个"高容量、低成本、低功耗、广覆盖"的空白市场而设计的。

### 1.3 典型应用场景

LPWAN适用于满足以下条件的场景:

- 数据量小: 每次传输不超过几百字节
- 传输频率低: 每分钟到每天发送一次
- 延迟容忍: 不要求毫秒级实时响应
- 部署分散: 设备分布在广阔区域
- 维护困难: 无法频繁更换电池或上门维护

## 2. 主流LPWAN技术详解

### 2.1 LoRaWAN

LoRaWAN是目前部署最广泛的LPWAN技术之一，由LoRa Alliance管理。

"LoRa"是物理层调制技术(CSS啁啾扩频)的名称，由Semtech公司开发并持有专利。"LoRaWAN"是基于LoRa物理层的完整网络协议栈。这就像WiFi中802.11是物理层标准，而TCP/IP是上层协议。

```
LoRaWAN 协议栈:

+------------------+
| 应用层 (用户数据) |
+------------------+
| LoRaWAN MAC层     |
| (入网/确认/ADR)   |
+------------------+
| LoRa 物理层       |
| (CSS调制)         |
+------------------+
| 射频硬件          |
| (SX1276等芯片)    |
+------------------+
```

LoRaWAN的核心参数:

| 参数 | 值 |
|------|-----|
| 频段 | 868MHz(欧)/915MHz(美)/470MHz(中) |
| 频谱类型 | 免许可(ISM) |
| 调制方式 | CSS(啁啾扩频) |
| 最大速率 | 50kbps |
| 典型速率 | 0.3-11kbps |
| 覆盖距离 | 城区2-5km，郊区15km+ |
| 电池寿命 | 5-10年(典型) |
| 网络拓扑 | 星型 |
| 单基站容量 | 数千节点 |

LoRaWAN的独特优势在于它可以部署私有网络。企业或组织可以购买LoRa网关，搭建自己的LoRaWAN网络，无需依赖运营商，无需支付月租费。这对于农场、工厂、园区等场景非常有吸引力。

### 2.2 Sigfox

Sigfox是一家法国公司推出的LPWAN技术和全球网络服务，采用超窄带(Ultra Narrow Band, UNB)技术。

与LoRaWAN不同，Sigfox采用运营商模式——由Sigfox公司(及其合作伙伴)统一建设和运营网络，用户只需购买Sigfox模组并注册服务即可使用。

```
Sigfox 通信特征:

信号带宽对比:
  LoRa:   |====125kHz====|  (宽)
  Sigfox: |=100Hz=|         (极窄)

超窄带的好处:
  - 接收端噪声极低 -> 灵敏度极高 -> 传得更远
  - 频谱效率高 -> 同一频段可容纳更多设备

代价:
  - 数据速率极低: 仅100bps
  - 每条消息最大: 12字节(上行) / 8字节(下行)
  - 每天最多: 140条上行 / 4条下行
```

Sigfox的核心参数:

| 参数 | 值 |
|------|-----|
| 频段 | 868MHz(欧)/902MHz(美) |
| 频谱类型 | 免许可(ISM) |
| 调制方式 | DBPSK(超窄带) |
| 上行速率 | 100bps |
| 下行速率 | 600bps |
| 消息大小 | 上行12字节/下行8字节 |
| 覆盖距离 | 城区3-10km，郊区30-50km |
| 电池寿命 | 10-15年 |
| 网络拓扑 | 星型 |
| 商业模式 | 订阅制(按设备/年) |

Sigfox适合极简的"发了就走"场景: 烟雾报警器每天报一次平安、水表每小时报一次读数、物流追踪每次报一个GPS坐标。

### 2.3 NB-IoT

NB-IoT(Narrowband IoT)是3GPP在Release 13中标准化的蜂窝LPWAN技术，由电信运营商部署在其现有蜂窝网络基础设施上。

```
NB-IoT 部署模式:

模式1: 独立部署(Standalone)
  在GSM频段内部署独立的200kHz NB-IoT载波
  |--GSM--|--NB-IoT--|--GSM--|

模式2: 保护带部署(Guard-band)
  利用LTE频段边缘的保护带
  |--LTE频段--|NB|保护带|NB|--LTE频段--|

模式3: 带内部署(In-band)
  占用LTE频段内的一个PRB(180kHz)
  |---LTE频段---|NB|---LTE频段---|
```

NB-IoT的核心参数:

| 参数 | 值 |
|------|-----|
| 频段 | 运营商授权频段(多频段) |
| 频谱类型 | 授权频谱 |
| 带宽 | 200kHz |
| 峰值速率 | 上行63kbps/下行27kbps(单音) |
| 覆盖增强 | MCL 164dB(比GSM多20dB) |
| 电池寿命 | 10年+(PSM模式) |
| 网络拓扑 | 星型(蜂窝) |
| QoS保障 | 有(运营商级) |

NB-IoT的最大优势是运营商级的服务质量(QoS)保障和安全性。它使用授权频谱，不存在ISM频段的干扰问题; 它复用现有蜂窝基础设施，覆盖范围与手机信号基本一致; 它有运营商提供的SLA保障。

### 2.4 LTE-M (Cat-M1)

LTE-M(也称Cat-M1)同样来自3GPP，但定位与NB-IoT有所不同。如果说NB-IoT是为"静止的传感器"设计的，那么LTE-M更适合"移动的设备"。

| 参数 | 值 |
|------|-----|
| 频段 | 运营商授权频段 |
| 频谱类型 | 授权频谱 |
| 带宽 | 1.4MHz |
| 峰值速率 | 上行1Mbps/下行1Mbps |
| 移动性 | 支持小区切换 |
| 语音 | 支持VoLTE |
| 电池寿命 | 5-10年 |
| 定位 | 支持OTDOA |

LTE-M与NB-IoT的关键区别:

```
LTE-M vs NB-IoT 定位差异:

LTE-M 优势场景:
  - 需要移动性(车辆追踪、共享单车)
  - 需要语音(老人紧急呼叫设备)
  - 数据量较大(固件OTA升级)
  - 延迟敏感(报警器)

NB-IoT 优势场景:
  - 固定部署(水表、气表)
  - 数据量极小(每天几十字节)
  - 极端覆盖(地下室、井盖)
  - 海量连接(每平方公里5万设备)
```

## 3. 技术对比与选择

### 3.1 核心维度对比

| 维度 | LoRaWAN | Sigfox | NB-IoT | LTE-M |
|------|---------|--------|--------|-------|
| 频谱 | 免许可 | 免许可 | 授权 | 授权 |
| 最大速率 | 50kbps | 100bps | 63kbps | 1Mbps |
| 覆盖(城区) | 2-5km | 3-10km | 1-10km | 1-10km |
| 覆盖(郊区) | 15km+ | 30-50km | 15km+ | 15km+ |
| 电池寿命 | 5-10年 | 10-15年 | 10年+ | 5-10年 |
| 模组成本 | 3-5美元 | 2-3美元 | 5-8美元 | 8-12美元 |
| 运营费用 | 可自建(免费) | 订阅制 | 运营商月租 | 运营商月租 |
| QoS保障 | 无 | 无 | 有 | 有 |
| 移动性 | 有限 | 无 | 有限 | 好 |
| 延迟 | 秒级 | 秒-分钟级 | 秒级 | 亚秒级 |
| 双向通信 | 有限 | 极有限 | 好 | 好 |
| 私有部署 | 可以 | 不可以 | 需运营商 | 需运营商 |
| 生态成熟度 | 高 | 中 | 高 | 高 |

### 3.2 免许可vs授权频谱

频谱选择是理解LPWAN技术差异的关键:

免许可频谱(LoRaWAN, Sigfox)意味着任何人都可以在ISM频段上免费传输数据，但必须遵守监管机构的限制(如占空比、最大功率)。好处是部署灵活、成本低; 风险是同频段其他设备可能造成干扰，无法获得QoS保障。

授权频谱(NB-IoT, LTE-M)意味着运营商付费获得独占使用权，其上的设备通信不会被其他系统干扰。好处是可靠性高、有QoS保障; 代价是需要支付运营商服务费。

```
频谱监管限制举例(欧洲868MHz):

ISM免许可频段规则:
  - 最大发射功率: 25mW (14dBm)
  - 占空比限制: 1% (每小时最多发送36秒)
  - 无信道独占保护

影响:
  即使你的LoRa网关是空闲的，
  你的设备也不能连续发送超过36秒/小时。
  如果隔壁工厂也在用LoRa，可能互相干扰。
```

### 3.3 网络拓扑

LPWAN技术普遍采用星型拓扑(Star Topology):

```
星型拓扑 (LPWAN典型):

     [节点1]
        \
[节点2]--[网关/基站]--[节点3]
        /       |
     [节点4]  [节点5]

特点:
  - 所有节点直接与网关通信
  - 不存在节点间中继(无mesh)
  - 网关是单点(需要冗余设计)
  - 协议简单，节点功耗低
```

这与Zigbee/Thread等短距离技术的网状(Mesh)拓扑不同。星型拓扑的优势是协议简单、节点不需要转发其他节点的数据因此更省电; 劣势是覆盖完全取决于网关的位置和数量。

## 4. 按行业选择LPWAN

### 4.1 农业

农业场景的典型需求是广覆盖(农田面积大)、低成本(大量传感器)、可自建网络(偏远地区无蜂窝覆盖)。

推荐方案是LoRaWAN。原因是农村地区运营商蜂窝覆盖可能不完整，而LoRaWAN可以自建网关; 一个高处安装的网关可以覆盖10-15km半径的农田; 土壤湿度、温度等数据量极小(每次几十字节)，完全在LoRa的能力范围内。

```
农业LoRaWAN部署示例:

[土壤湿度传感器] x 200个
        |  LoRa 868MHz, SF10
        |  每30分钟发送一次, 每次20字节
        v
[LoRa网关] (安装在粮仓屋顶, 15m高)
        |  4G回程 或 卫星回程
        v
[网络服务器] -> [农场管理平台]

成本估算:
  传感器节点: 200 x 15美元 = 3000美元
  网关: 1 x 300美元 = 300美元
  网络服务器: 开源(免费)
  月运营费: 仅4G回程SIM卡费用
```

### 4.2 公用事业(水电气表)

智能计量的需求是高可靠性(涉及计费)、长期稳定(表计寿命10-15年)、深度覆盖(表计可能在地下室或铁箱中)。

推荐方案是NB-IoT。原因是运营商级QoS保障满足计费级可靠性要求; NB-IoT的164dB MCL提供了卓越的室内/地下穿透能力; 蜂窝网络覆盖城市区域，无需额外基础设施。

### 4.3 物流追踪

物流追踪需要全球覆盖(货物跨国运输)、低成本(每个包裹一个追踪器)。

推荐方案是Sigfox(全球覆盖)或LTE-M(区域覆盖)。Sigfox在全球70多个国家有网络覆盖，适合跨国物流; LTE-M支持移动性，适合需要实时跟踪的高价值货物。

### 4.4 车队管理

车队管理需要移动性支持(车辆在行驶)、较高数据速率(可能传输诊断数据)、语音能力(驾驶员通话)。

推荐方案是LTE-M。它是唯一同时支持移动性切换和VoLTE语音的LPWAN技术，1Mbps的速率足以支持OBD诊断数据和固件升级。

## 5. 未来演进

### 5.1 卫星LPWAN

LoRa和NB-IoT正在向卫星方向延伸，解决地面网络无法覆盖的海洋、沙漠、极地等区域:

```
卫星LPWAN生态:

LoRa卫星:
  - Lacuna Space: LEO卫星直接接收LoRa信号
  - 无需修改地面终端(标准LoRa模组即可)
  - 延迟: 分钟到小时级(等卫星过顶)

NB-IoT卫星:
  - 3GPP Release 17 NTN(非地面网络)
  - 标准NB-IoT芯片通过软件升级支持卫星
  - MediaTek等芯片厂商已推出方案

应用场景:
  - 远洋集装箱追踪
  - 偏远地区环境监测
  - 全球资产追踪(无地面网络依赖)
```

### 5.2 5G mMTC

3GPP在5G框架下定义了mMTC(大规模机器类通信)场景，作为NB-IoT和LTE-M的演进路径:

- NB-IoT和LTE-M已被正式纳入5G标准
- 将在5G核心网下继续演进
- 目标: 每平方公里100万设备连接
- 更低的功耗和更深的覆盖

### 5.3 技术融合趋势

未来的IoT设备可能集成多种LPWAN技术，根据场景自动切换:

```
多模LPWAN设备概念:

                +-----------+
  有蜂窝覆盖 -->| NB-IoT    | (首选: 可靠)
                +-----------+
  无蜂窝覆盖 -->| LoRaWAN   | (自建网络)
                +-----------+
  全球漫游   -->| Sigfox    | (全球覆盖)
                +-----------+
  无任何覆盖 -->| 卫星LoRa  | (最后手段)
                +-----------+
```

## 6. LPWAN技术选择决策指南

面对多种LPWAN技术，如何为你的项目做出正确选择? 下面是一个实用的决策框架:

```
决策树:

Q1: 设备需要移动吗(如车辆、穿戴)?
  |-- 是 --> LTE-M
  |-- 否 --> Q2

Q2: 需要运营商级可靠性(如计费、安全)?
  |-- 是 --> NB-IoT
  |-- 否 --> Q3

Q3: 部署区域有蜂窝覆盖吗?
  |-- 否 --> LoRaWAN(自建网关)
  |-- 是 --> Q4

Q4: 需要私有网络(数据不过运营商)?
  |-- 是 --> LoRaWAN
  |-- 否 --> Q5

Q5: 数据量极小且全球部署?
  |-- 是 --> Sigfox
  |-- 否 --> LoRaWAN 或 NB-IoT(按成本选)
```

实际决策时还需考虑:

当地运营商是否支持NB-IoT/LTE-M(不同国家覆盖不同)、设备生命周期内的总拥有成本(TCO)包括模组费用加网络服务费加维护成本、所在国家的ISM频段监管规则(占空比限制等)，以及未来的扩展需求(是否需要全球部署)。

## 总结

LPWAN技术家族为IoT世界解决了一个核心矛盾: 如何让数十亿简单传感器以极低成本、极低功耗实现广域连接。四种主流技术各有所长——LoRaWAN以灵活的私有部署见长，Sigfox以极致的简单和全球覆盖见长，NB-IoT以运营商级可靠性见长，LTE-M以移动性和较高带宽见长。

没有"最好"的LPWAN技术，只有"最适合"的选择。理解每种技术的核心特征和适用场景，结合项目的具体需求(覆盖、功耗、成本、可靠性、移动性)进行综合评估，才能做出正确的技术决策。

随着卫星LPWAN和5G mMTC的发展，LPWAN的覆盖将从地面延伸到全球，连接能力将从百万级提升到千万级。对于IoT从业者来说，掌握LPWAN技术是理解物联网底层连接的基本功。
