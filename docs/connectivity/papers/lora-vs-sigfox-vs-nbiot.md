# LoRa/Sigfox/NB-IoT LPWAN技术全面对比
> **难度**：🟢 初级 | **领域**：LPWAN选型 | **阅读时间**：约 18 分钟

## 引言

想象你要给三个住在偏远地区的朋友寄信。你有三种选择: 第一种是自己建一个邮局(LoRa) -- 前期投入大, 但之后寄信免费, 而且你完全控制运营; 第二种是用一家超低价快递公司(Sigfox) -- 每年交很少的费用, 但每天只能寄有限几封信, 对方几乎没法回信; 第三种是用中国移动/联通的短信业务(NB-IoT) -- 利用现有基站, 每月交话费, 速度更快, 双向通信方便。

这三种方式就是LPWAN(低功耗广域网)领域的三大主流技术。它们都为物联网设计 -- 追求长距离、低功耗、低成本, 但具体实现各有取舍。

## 1. LPWAN概述

### 1.1 什么是LPWAN

LPWAN(Low Power Wide Area Network)是专为物联网设计的无线通信技术类别, 核心目标: 长距离(数公里到数十公里)、低功耗(电池供电数年)、低数据率(每次几十字节)、低成本、大连接(单基站数千设备)。

### 1.2 在无线通信中的位置

```
          覆盖范围
            ^
     10km+ |  Sigfox   LoRa    NB-IoT
            |     *      *       *
      1km  |
     100m  |  BLE  WiFi  ZigBee
            |   *    *      *
      10m  |
            +--------------------------------> 数据速率
           10bps  1kbps  100kbps  10Mbps
```

LPWAN填补了"长距离+低速率"的生态位。

### 1.3 三大技术定位

| 维度 | LoRa/LoRaWAN | Sigfox | NB-IoT |
|------|-------------|--------|--------|
| 频谱 | 免许可频段 | 免许可频段 | 运营商许可频段 |
| 网络模式 | 私有或公共 | 仅运营商网络 | 仅运营商网络 |
| 标准组织 | LoRa Alliance | Sigfox公司 | 3GPP |
| 物理层 | CSS扩频 | 超窄带100Hz | OFDM |
| 通信方向 | 完整双向 | 极有限下行 | 完整双向 |
| 生态 | TTN/Helium/私有 | Sigfox网络 | 移动运营商 |

## 2. LoRa与LoRaWAN

### 2.1 技术架构

```
应用层     |  用户应用程序
-----------|------------------
LoRaWAN    |  MAC协议(入网/安全/ADR/Class A/B/C)
-----------|------------------
LoRa       |  CSS扩频调制(Semtech专利)
-----------|------------------
射频硬件   |  SX1276/SX1262收发器芯片
```

### 2.2 核心特点

**免许可频段**: 中国CN470(470-510MHz)、欧洲EU868、美国US915。无需付费但需遵守功率限制。

**可建私有网络**: 企业自己部署网关和服务器, 完全控制数据, 不依赖第三方。这是LoRa区别于其他两种技术的最大优势。

**自适应数据速率(ADR)**: 网络自动调整SF和功率。近距离用低SF(快速低功耗), 远距离用高SF(慢速长距离)。

### 2.3 设备类别

| 类别 | 下行方式 | 延迟 | 功耗 | 适用 |
|------|---------|------|------|------|
| Class A | 上行后开2个短窗口 | 秒-分钟 | 最低 | 传感器 |
| Class B | 周期性接收窗口 | 秒级 | 中等 | 执行器 |
| Class C | 持续接收 | 最低 | 最高 | 市电设备 |

## 3. Sigfox技术

### 3.1 超窄带原理

```
信号带宽对比:
LoRa:    125kHz
Sigfox:  100Hz  (比LoRa窄1000倍以上)
NB-IoT:  180kHz

100Hz极窄带宽 -> 接收噪声极低 -> 灵敏度极高 -> 超远距离
但代价是: 数据速率仅100bps
```

### 3.2 核心特点与限制

- **超长距离**: 农村30-50km, 城市3-10km
- **极简协议**: 上行每条最大12字节, 每天最多140条; 下行每天最多4条
- **无入网过程**: 烧录ID即可使用
- **仅运营商网络**: 不能自建, 覆盖取决于Sigfox在当地的部署
- **商业挑战**: 2022年财务困难被UnaBiz收购, 前景不确定

## 4. NB-IoT技术

### 4.1 技术架构

NB-IoT是3GPP Release 13标准, 三种部署模式:

- 独立部署: 使用空闲GSM频段
- 保护带部署: 使用LTE载波保护带
- 带内部署: 占用LTE载波中一个PRB(180kHz)

### 4.2 核心特点

- **利用现有基础设施**: 复用LTE基站, 软件升级即可
- **覆盖增强**: 比LTE多约20dB, 能穿透地下1-2层
- **较高数据速率**: 上行62.5kbps, 下行230kbps
- **完整双向**: 支持设备和网络双向发起通信
- **QoS保证**: 许可频段, 无干扰风险

### 4.3 节能机制

**PSM(省电模式)**: 无数据时深度休眠, 功耗接近关机, 由设备主动唤醒。

**eDRX(扩展非连续接收)**: 周期性监听下行(周期可达43分钟), 在省电和可达性之间取平衡。

## 5. 核心参数全面对比

### 5.1 覆盖范围

| 技术 | 城市 | 郊区 | 农村开阔地 |
|------|------|------|-----------|
| LoRa | 2-5km | 5-10km | 10-15km |
| Sigfox | 3-10km | 10-30km | 30-50km |
| NB-IoT | 1-5km | 5-8km | 取决于基站 |

### 5.2 数据速率

```
NB-IoT  |==========================================| 230kbps
LoRa    |============| 50kbps(SF7最高)
LoRa    |==| 0.3kbps(SF12最低)
Sigfox  |=| 0.1kbps
```

NB-IoT数据速率比LoRa/Sigfox高1-3个数量级。

### 5.3 功耗与电池寿命

| 技术 | 发射电流 | 接收电流 | 休眠电流 | 典型电池寿命 |
|------|---------|---------|---------|------------|
| LoRa | 20-40mA | 10-15mA | 1-2uA | 3-8年 |
| Sigfox | 10-50mA | - | 1-2uA | 5-10年 |
| NB-IoT | 120-220mA | 40-50mA | 3-5uA(PSM) | 2-5年 |

NB-IoT功耗较高因为: 需与基站同步, 射频频率更高, 协议栈更复杂。PSM/eDRX大幅改善平均功耗。

### 5.4 成本对比

| 成本项 | LoRa | Sigfox | NB-IoT |
|--------|------|--------|--------|
| 模块 | 3-8美元 | 2-5美元 | 5-10美元 |
| 基础设施 | 网关150-2000美元 | 无(运营商) | 无(运营商) |
| 通信费 | 0(私网) | 1-3美元/年/设备 | SIM+月费 |

## 6. 双向通信能力

这是三种技术差异最大的维度之一:

```
LoRaWAN:
- Class A: 每次上行后开2个下行窗口
- Class C: 几乎持续接收
- 能满足配置、控制等下行需求

Sigfox:
- 下行每天仅4条, 仅在设备请求时发送
- 近乎单向通信
- 无法实时推送命令

NB-IoT:
- 完整双向, 延迟秒级
- 支持IP协议栈(TCP/UDP/CoAP)
- 适合固件升级等大数据量下行
```

| 应用需求 | LoRa | Sigfox | NB-IoT |
|---------|------|--------|--------|
| 传感器上报 | 优秀 | 优秀 | 优秀 |
| 远程配置 | 良好 | 极差 | 优秀 |
| 固件升级 | 困难 | 不可能 | 良好 |
| 实时控制 | Class C可以 | 不可能 | 优秀 |

## 7. 延迟与部署模式

### 7.1 延迟对比

- LoRa Class A: 下行需等设备下次上行(分钟级)
- Sigfox: 下行延迟极高(分钟到小时)
- NB-IoT: 非PSM模式下上下行均为秒级

### 7.2 部署模式差异

```
LoRa - 灵活部署:
  A) 完全私有: 自建网关+服务器, 无月费
  B) 公共网络: TTN/Helium, 低费用
  C) 混合: 自有网关+公共网关

Sigfox - 纯运营商:
  购买模块 -> 注册设备 -> 配置回调URL -> 按年付费
  无法自建网络

NB-IoT - 电信运营商:
  购买模块+SIM -> 注册IoT平台 -> 对接API -> 按月付费
  利用已有蜂窝覆盖
```

## 8. 应用场景匹配

### 8.1 各技术最佳场景

**LoRa最适合**: 智慧农业(私网、长距离)、智慧工厂(数据主权)、建筑能耗监测(室内部署)、需要比Sigfox更多数据的场景。

**Sigfox最适合**: 极简环境监测(每小时报12字节)、极低功耗需求(10年电池)、无需下行控制的纯数据收集。

**NB-IoT最适合**: 智能表计(可靠上报+远程配置)、共享经济(实时定位控制)、需要OTA固件升级、有QoS要求的场景、中国市场(覆盖极广)。

### 8.2 快速决策树

```
需要自建私有网络?
  |-- 是 --> LoRa(唯一选择)
  |-- 否 --> 继续

需要双向实时控制?
  |-- 是 --> NB-IoT
  |-- 否 --> 继续

每日数据量 > 12字节x140次?
  |-- 是 --> LoRa 或 NB-IoT
  |-- 否 --> 继续

当地有Sigfox覆盖且成本极敏感?
  |-- 是 --> Sigfox
  |-- 否 --> LoRa 或 NB-IoT
```

## 9. 选型决策矩阵

| 决策因素 | LoRa优先 | Sigfox优先 | NB-IoT优先 |
|---------|---------|-----------|-----------|
| 数据主权 | 必须私有 | - | - |
| 数据量 | 中等 | 极少 | 较多 |
| 延迟 | 秒-分钟 | 不关心 | 需要秒级 |
| 设备寿命 | 3-8年 | 5-10年 | 2-5年 |
| 双向需求 | 一般 | 几乎不需要 | 强烈 |
| 网络控制 | 需要 | 不需要 | 不需要 |

### 9.1 总拥有成本示例

假设1000设备部署5年, 每小时上报一次:

```
LoRa(私有网络):
- 模块: 1000 x 5美元 = 5,000美元
- 网关(10个): 10 x 800美元 = 8,000美元
- 云服务器: 5年 x 200美元/月 = 12,000美元
- 安装维护: 5,000美元
- 合计: 约30,000美元

Sigfox:
- 模块: 1000 x 3美元 = 3,000美元
- 订阅: 1000 x 2美元/年 x 5年 = 10,000美元
- 合计: 约13,000美元(受限于覆盖和140条/天)

NB-IoT:
- 模块: 1000 x 8美元 = 8,000美元
- SIM+月费: 1000 x 2美元/月 x 60月 = 120,000美元
- 合计: 约128,000美元
- 注: 中国运营商物联网套餐可能远低于此估算
```

## 10. 市场趋势

### 10.1 各技术发展方向

**LoRa/LoRaWAN**: 增长最快, 尤其私有IoT领域。新功能持续加入(中继TS011、FUOTA、定位)。芯片持续迭代, 开源生态活跃。

**Sigfox**: 2022年被收购后重组, 技术仍在部分地区运营, 长期前景不确定, 部分用户迁移到LoRa或NB-IoT。

**NB-IoT**: 中国发展最强劲, 三大运营商全面部署。3GPP持续演进, 与5G融合为mMTC基础。

### 10.2 中国市场特殊性

- NB-IoT: 政策推动, 覆盖率极高, 成本低
- LoRa: CN470频段, 私有网络需求旺盛
- Sigfox: 覆盖有限, 市场份额极小

在中国做LPWAN选型, NB-IoT和LoRa是主要选择。

## 总结

三种技术各有核心优势:

- **LoRa**: 最灵活, 唯一支持私有网络, 生态开放。适合需要数据主权和网络控制的场景。
- **Sigfox**: 最简单最省电, 但受限于覆盖和商业前景不确定。
- **NB-IoT**: 数据速率最高, 双向最好, 利用蜂窝基础设施。适合需要QoS和双向控制的场景。

选型核心原则: 先明确需求(数据量、延迟、双向性、电池寿命), 再看当地基础设施, 最后算总拥有成本。
