# 全双工无线电在IoT中的自干扰消除
> **难度**: 高级 | **领域**: 无线通信 | **阅读时间**: 约 22 分钟

## 引言

想象你在打电话时,自己说话的声音直接灌进耳朵里,比对方的声音大十亿倍——你还能听到对方说什么吗?全双工无线电面临的正是这个挑战。我们的耳朵之所以能同时"收发"(听别人说话的同时自己也在说),是因为大脑能自动过滤自己的声音。全双工无线电要做的,就是让收发器学会这种"过滤自己声音"的能力。

当前几乎所有无线系统都采用半双工(收发交替)或频分双工(收发用不同频率)。如果能在同一频率上同时收发,频谱效率理论上翻倍。对频谱资源紧张的IoT应用,这意味着相同带宽内服务两倍的设备。

## 1. 什么是全双工无线电

### 1.1 双工方式对比

```
半双工(TDD):  [TX][RX][TX][RX]  同一频率交替收发
频分双工(FDD): f1=[TX], f2=[RX]  不同频率同时,需双倍频谱
全双工:        [TX+RX]          同频同时,理论效率翻倍
```

### 1.2 自干扰: 核心难题

发射信号与接收信号的功率差异惊人:

```
发射功率: +20dBm = 100mW (自己的信号)
接收信号: -90dBm = 0.000000001mW (远端设备信号)
功率差: 110dB = 一千亿倍

类比: 在喷气发动机旁边听一根针掉地上的声音
```

必须将自己的发射信号消除110dB以上,才能"听到"远端设备的微弱信号。长期以来这被认为不可能,直到2010年代研究突破使110dB以上的消除成为现实。

## 2. 自干扰消除: 三级架构

### 2.1 总体设计

自干扰消除(SIC)需要在三个域逐级进行:

```
[天线域] --> [模拟域] --> [数字域] --> 干净接收信号
 30-50dB     30-50dB     20-40dB
总消除量: 100-130dB (满足全双工需求)
```

每一级都是必要的,单独任何一级都无法达到所需的消除深度。

### 2.2 为什么需要三级

如果只有数字域消除: ADC输入端自干扰功率+20dBm,ADC满量程约0dBm,ADC被饱和,数字域无法处理。如果只有模拟域: 元器件精度有限,残余自干扰仍远强于目标信号。如果只有天线域: 设备尺寸限制物理隔离,反射散射路径无法完全隔离。三级联合才能达到所需消除深度。

## 3. 天线域自干扰消除

通过物理手段在天线层面隔离发射和接收:

```
方法1: 空间分离
  TX天线 <----距离d----> RX天线
  路径损耗提供自然隔离, 距离越远隔离越好

方法2: 极化隔离
  TX: 垂直极化 |   RX: 水平极化 --
  正交极化提供20-30dB隔离

方法3: 方向性隔离
  TX向前辐射 -->   RX <-- 向后接收
  利用天线方向性图的后瓣衰减
```

一种巧妙方法是对称天线布局: 两个TX天线到RX天线等距,发送反相信号,在RX位置相消为零;对远端设备则相长叠加,不损失发射性能。典型天线域消除量30-50dB。

## 4. 模拟域自干扰消除

在射频电路中创建发射信号的反相副本,从接收路径减去:

```
TX信号 --> [功放] --> TX天线
  |
  | (抽头)
  v
[延迟+反相] --> (+) <-- RX天线
                 |
            [到ADC/数字域]
```

### 4.1 自适应消除器

由于环境反射会变化,模拟消除器必须自适应。使用多抽头滤波器,每个抽头对应一条自干扰路径(直射+反射),持续调整幅度和相位以跟踪信道变化。

### 4.2 消除效果与带宽的关系

```
窄带(1MHz):    45-50dB
中等(20MHz):   35-40dB
宽带(80MHz):   25-35dB
IoT窄带(NB-IoT 180kHz): 更容易实现高消除量
```

带宽越宽,多径色散消除越困难。IoT系统通常窄带,这是一个天然优势。

### 4.3 实现难点

功放非线性产生谐波、宽带信号多径消除困难、温度漂移改变元器件参数、本振相位噪声无法被消除。精心设计的模拟SIC电路可实现30-50dB消除。

## 5. 数字域自干扰消除

在数字基带利用已知发射数据进行精细消除:

```python
import numpy as np

def digital_sic(rx_samples, tx_samples, filter_len=64):
    """
    数字域自干扰消除
    rx_samples: ADC采样(含残余自干扰)
    tx_samples: 已知发射数据
    """
    N = len(rx_samples)
    # 构建线性项特征矩阵
    X_lin = np.zeros((N, filter_len), dtype=complex)
    for i in range(filter_len):
        X_lin[i:, i] = tx_samples[:N-i]
    # 三阶非线性项(模拟功放失真)
    tx_cube = tx_samples * np.abs(tx_samples)**2
    X_nl = np.zeros((N, filter_len), dtype=complex)
    for i in range(filter_len):
        X_nl[i:, i] = tx_cube[:N-i]
    # 合并, 最小二乘估计, 消除
    X = np.concatenate([X_lin, X_nl], axis=1)
    h = np.linalg.lstsq(X, rx_samples, rcond=None)[0]
    return rx_samples - X @ h
```

### 5.2 非线性建模至关重要

功率放大器(PA)是最主要的非线性来源。PA的输入输出关系为: y(t) = a1*x(t) + a3*x(t)*|x(t)|^2 + a5*x(t)*|x(t)|^4 + ...,其中a3和a5项产生互调失真。

如果只用线性模型消除: 基波分量被消除,但三阶和五阶互调产物残留,限制消除深度在20-25dB。使用三阶非线性模型: 高阶项也被建模和消除,深度可达30-35dB。使用五阶模型: 消除深度35-40dB,但计算复杂度显著增加。

| 建模方式 | 消除量 | 计算复杂度 |
|---------|--------|-----------|
| 线性模型 | 20-25dB | 低 |
| 三阶非线性 | 30-35dB | 中 |
| 五阶非线性 | 35-40dB | 高 |
| 含IQ失衡补偿 | 38-42dB | 很高 |

对于IoT窄带系统(NB-IoT仅180kHz带宽),非线性建模相对容易,因为带宽窄意味着多径色散小,PA非线性模型更简单。这是IoT系统实现全双工的天然优势。

## 6. 全双工对IoT的优势

**频谱效率翻倍**: 拥挤ISM频段(2.4GHz, 868/915MHz)中,相同带宽服务更多设备。

**同时感知与传输**: 发送数据的同时监听信道,实时检测干扰并调整。认知无线电应用的理想选择。

**降低延迟**: 无TX/RX切换延迟,设备发送数据同时接收ACK。对URLLC IoT场景关键。

**中继效率倍增**:

```
半双工中继: 源 --> [接收][转发] --> 目的  (2个时隙)
全双工中继: 源 --> [接收+转发] --> 目的    (1个时隙)
延迟和吞吐量均提升一倍
```

## 7. 全双工中继在IoT网状网络中的应用

### 7.1 存储转发 vs 全双工中继

当前IoT网状网络使用存储转发,每跳需要两个时隙(先接收再转发)。全双工中继同时接收和转发,每跳仅一个时隙:

```
5跳路径延迟:
半双工: 每跳10ms(接收5ms+转发5ms), 5跳 = 50ms
全双工: 每跳约4ms, 5跳 = 20ms
延迟改善: 50-60%
```

### 7.2 适用场景

工厂车间多跳网络(实时控制回路)、矿井/隧道线性拓扑、大型建筑跨楼层中继。这些场景中中继节点是固定基础设施,尺寸和功耗限制较小。

## 8. RFID中的全双工

RFID读写器天然是全双工操作: 发射连续波(为无源标签供电+发送查询),同时接收标签的反向散射响应。

```
TX功率: +30dBm (1W)
标签反向散射: -70dBm
差距: 100dB

RFID读写器SIC技术:
- 环形器隔离: 20-25dB
- 泄漏消除电路: 20-30dB
- 数字域处理: 15-20dB

SIC性能 vs 读取距离:
基础SIC(60dB): 3m
改进SIC(80dB): 8m
先进SIC(100dB): 15m+
```

SIC每提升20dB读取距离约翻倍。RFID是全双工/SIC在IoT中最成熟的应用。

## 9. 当前技术限制

全双工对IoT不同角色的适用性差异显著:

**不适合微型传感器节点**: PCB太小难以实现天线隔离,数字SIC需持续计算消耗电池,精密模拟SIC电路昂贵。

**有条件适用较大设备**: 路由器/网关级别的设备有一定空间和供电。

**适合基础设施节点**: 中继/网关/基站有市电供电(功耗不是问题),物理尺寸不受限(可放大天线间距),数量少(成本可分摊),性能提升惠及所有下游设备。

### 9.1 成本与功耗的量化分析

```
全双工中继节点的额外成本(相比半双工):
- 第二根天线 + RF前端: 约200元
- 模拟SIC电路(多抽头消除器): 约150元
- 更大FPGA/DSP(数字SIC): 约100元
- 总额外成本: 约450元/节点

功耗增加:
- 额外RF前端: +2W
- 模拟SIC有源器件: +0.5W
- 数字SIC处理: +1W
- 总增加: 约3.5W (壁挂供电可接受)

对比: 微型传感器BOM成本<50元, 功耗<0.1W
=> 全双工对传感器完全不现实
=> 对基础设施节点(数百元成本)可接受
```

## 10. 研究进展与产业动态

```
2010: 莱斯大学提出天线消除技术
2013: 斯坦福首个实用全双工收音机
2014: 实现110dB总消除
2018: 集成芯片级SIC方案
2020+: IoT特定全双工设计研究
```

产业化: Kumu Networks推出商用SIC方案,高通和华为研究蜂窝全双工技术。3GPP Release 18+可能在基站侧引入全双工。设备侧全双工是更长期的目标。

## 11. 5G与全双工

基站更容易实现全双工: 空间充裕(天线间距大)、供电充足(复杂SIC可接受)。即使IoT设备不支持全双工,网络侧全双工仍有益——基站同时接收上行和发送下行,减少调度延迟,提升系统容量。IoT设备保持简单半双工设计即可间接受益。

### 11.1 灵活TDD/FDD

```
传统TDD: [DDDDSUDDDSU...]  固定上下行配比
全双工基站: [同时上下行]    无需预设配比

优势:
- 动态适应业务负载(IoT上行突发时即时响应)
- 无需等待上行时隙
- 频谱利用率最大化
```

### 11.2 对IoT的间接收益路径

IoT设备无需全双工能力,但可通过以下路径间接受益: 基站全双工使调度更灵活,IoT上行请求无需等待下行时隙; 基站同时收发减少排队延迟; 全双工中继节点(如有)降低多跳路径延迟; 系统总容量提升使更多IoT设备接入。

## 12. 实际应用案例: 工业IoT全双工中继

```
场景: 工厂车间网状网络, 传感器到网关5跳, 延迟要求<30ms

中继节点规格(壁挂式, 市电供电):
- TX/RX天线间距15cm + 交叉极化
- 天线域消除: 45dB
- 多抽头自适应RF消除器(模拟域): 35dB
- 三阶非线性数字SIC(数字域): 25dB
- 总消除: 105dB (需求100dB, 余量5dB)

性能:
半双工中继: 每跳10ms, 5跳=50ms (超出30ms要求)
全双工中继: 每跳4ms, 5跳=20ms  (满足要求)

成本对比:
5个全双工中继(800元/个) = 4000元
增加网关减少跳数: 5000元/个网关
全双工方案总体成本可接受

关键: 复杂度集中在中继基础设施, 终端传感器完全不变
```

## 总结

全双工无线电通过同频同时收发,理论频谱效率翻倍。核心挑战——自干扰消除需要天线域、模拟域、数字域三级联合处理,总消除量达100-130dB。

对IoT最直接的应用场景是基础设施节点(中继/网关)而非终端传感器,全双工中继可将多跳网络延迟减半。RFID领域已证明SIC的实际价值。随着5G在基站侧引入全双工,IoT设备将间接受益于网络容量提升,同时保持简单低功耗设计。
