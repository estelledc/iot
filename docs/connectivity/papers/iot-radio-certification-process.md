# IoT无线设备射频认证流程与测试
> **难度**: 中级 | **领域**: 合规认证 | **阅读时间**: 约 20 分钟

## 引言

想象你要在公路上开车。即使你的车性能再好,也必须先通过车检、拿到行驶证才能合法上路。无线设备的射频认证就是这个"行驶证"——你的IoT产品如果没有通过相应国家的射频认证,就不能在当地市场合法销售和使用。

射频认证的本质是确保设备不会对无线电频谱这一公共资源造成有害干扰。就像交通法规防止交通混乱一样,各国的无线电管理机构通过认证制度保证无线设备之间能够和谐共存。本文将详细介绍IoT设备从设计到取得认证的完整流程,覆盖全球主要市场的要求。

## 1 为什么射频认证是强制要求

### 1.1 法律基础

无线电频谱是有限的国家资源,受各国法律严格管制:

- 美国: 联邦通信法(Communications Act)授权FCC管理所有无线电发射
- 欧盟: 无线电设备指令(RED 2014/53/EU)要求所有无线设备合规
- 中国: 无线电管理条例规定未经核准的无线电发射设备不得销售使用
- 日本: 电波法(Radio Law)由总务省MIC执行

违规后果严重:

| 违规行为 | 美国(FCC) | 中国(SRRC) | 欧盟(CE) |
|----------|-----------|------------|----------|
| 未认证销售 | 罚款可达10万美元/天 | 没收设备+罚款 | 海关扣押+市场召回 |
| 超功率发射 | 设备查封+刑事责任 | 行政处罚 | 成员国各自执法 |

### 1.2 技术必要性

射频认证不仅是法律要求,也是工程质量的保证: 验证发射功率在安全限值内; 确认杂散辐射不干扰其他频段服务; 检验设备的频率稳定性; 评估电磁兼容性(EMC)。

## 2 全球主要认证体系

### 2.1 FCC认证(美国)

FCC(Federal Communications Commission)认证是覆盖面最广的认证之一:

```
FCC认证分类:
Part 15.247  -- 扩频和数字调制(WiFi/BLE等)
Part 15.249  -- 其他ISM设备
Part 22/24/27 -- 蜂窝设备(需运营商许可)

认证路径:
                有意辐射体?
                /        \
             是            否
            /                \
  Certification          DoC/SDoC
  (必须实验室测试)       (自我声明)
```

对于IoT设备,大多数属于Part 15的有意辐射体(Intentional Radiator),需要走Certification路径。

### 2.2 CE RED认证(欧盟)

欧盟无线电设备指令(Radio Equipment Directive)要求:

- 适用标准: EN 300 220(Sub-GHz), EN 300 328(2.4GHz), EN 301 893(5GHz)
- 必须由公告机构(Notified Body)评估或自我声明(取决于设备类型)
- 认证后获得CE标志,可在所有欧盟成员国销售

### 2.3 SRRC认证(中国)

SRRC(国家无线电监测中心检测中心)型号核准: 所有在中国销售的含无线电发射功能的设备都需要型号核准; 测试项目与FCC类似但具体限值有差异; 核准证书有效期通常为5年; 2018年后改革部分设备可走简化流程。

### 2.4 其他重要认证

| 国家/地区 | 认证机构 | 认证名称 | 特点 |
|-----------|----------|----------|------|
| 日本 | MIC/TELEC | 技术基准适合证明 | 功率限值非常严格 |
| 加拿大 | ISED(原IC) | RSS认证 | 与FCC高度互认 |
| 韩国 | KCC | KC认证 | 需韩国本地代理 |
| 澳大利亚 | ACMA | RCM | 接受部分国际报告 |
| 印度 | WPC | ETA认证 | 流程较长需本地测试 |

## 3 认证类型: 有意辐射体与无意辐射体

### 3.1 有意辐射体(Intentional Radiator)

设备的主要功能就是发射无线电信号: WiFi/BLE模组、LoRa/Zigbee/Z-Wave设备、蜂窝模组(4G/5G/NB-IoT)、所有LPWAN设备。这类设备必须走完整认证流程,需要实验室测试。

### 3.2 无意辐射体(Unintentional Radiator)

设备本身不是为了发射无线电,但工作时会产生电磁辐射: 开关电源、微处理器时钟、LED驱动器、高速数字信号线。这类设备通常只需做EMC测试,认证相对简单。

### 3.3 IoT设备的双重属性

一个典型IoT设备同时具有两种属性:

```
IoT温度传感器:
+-------------------------------------------+
| [MCU] --- [传感器] --- [电源管理]          | <- 无意辐射(EMC)
|   |                                       |
| [LoRa模组] --- [天线]                     | <- 有意辐射(RF认证)
+-------------------------------------------+

需要同时通过:
- RF认证(针对LoRa模组的发射特性)
- EMC测试(针对整机的电磁兼容性)
```

## 4 测试参数详解

### 4.1 传导功率(Conducted Power)

在天线端口测量的发射功率,不含天线增益:

```
测试配置:
[被测设备] --同轴线-- [衰减器] --同轴线-- [频谱分析仪]

典型限值:
- FCC Part 15.247 (2.4GHz扩频): 最大1W (30dBm) 传导
- FCC Part 15.249 (915MHz ISM): 最大-1.25dBm/3kHz带宽
- ETSI EN 300 220 (868MHz): 最大+14dBm ERP
- SRRC (470-510MHz LoRa): 最大+17dBm
```

### 4.2 杂散辐射(Spurious Emissions)

设备在工作频率之外产生的不需要的辐射:

| 频率范围 | FCC限值 | ETSI限值 | 测量方法 |
|----------|---------|----------|----------|
| 30MHz-1GHz | -36dBm | -36dBm | 峰值检波 |
| 1GHz-12.75GHz | -30dBm | -30dBm | 峰值检波 |
| 基波谐波 | 基波-20dB以下 | 基波-30dB以下 | 视标准 |

杂散辐射是最常见的测试失败原因之一。

### 4.3 占用带宽(Occupied Bandwidth)

信号实际占用的频谱宽度,定义为包含总发射功率99%的最小带宽。频谱分析仪的OBW功能可自动计算。

### 4.4 频率稳定性(Frequency Stability)

设备发射频率随温度和电压变化的偏移量: 测试温度范围-20到+50度C(消费级)或-40到+85度C(工业级); 测试电压范围标称电压的85%-115%; 典型要求偏移不超过正负20ppm。

```c
// 频率误差计算示例
double nominal_freq = 868.0e6;     // 标称868MHz
double measured_freq = 868.0023e6; // 实测值
double error_hz = measured_freq - nominal_freq;  // 2300 Hz
double error_ppm = error_hz / nominal_freq * 1e6; // 2.65 ppm
// FCC要求通常不超过正负20-50ppm, 2.65ppm远在合格范围内
```

## 5 测试实验室与设施

### 5.1 实验室认可体系

选择实验室需确认: 是否获得目标市场的认可(如FCC认可的测试实验室列表); 是否具备所需频段的测试能力; 是否能出具被认可的测试报告。认可机构包括NVLAP(美国)、CNAS(中国)、UKAS(英国),均要求ISO 17025认证。

### 5.2 测试设施要求

| 设施类型 | 用途 | 特点 |
|----------|------|------|
| 电波暗室(全电波) | 辐射发射测试 | 完全屏蔽+内壁吸波材料 |
| 半电波暗室 | EMC辐射测试 | 地面为反射面 |
| 屏蔽室 | 传导测试 | 防外部干扰,成本较低 |
| OATS(开阔场) | 辐射测试 | 成本低但受天气影响 |

## 6 模组认证策略

### 6.1 预认证模组(Pre-certified Module)

使用已通过认证的无线模组是IoT产品最快的上市路径:

```
预认证模组方案:
[你的产品] = [主控PCB] + [已认证WiFi模组]
                              -> 已有FCC/CE/SRRC证书
                              -> 你只需做EMC测试(更简单)

自研射频方案:
[你的产品] = [主控PCB + 自研射频电路 + 天线]
                              -> 需从头做完整RF认证
                              -> 周期长,成本高,风险大
```

### 6.2 模组认证的限制条件

使用预认证模组并非完全免除认证义务: 天线类型和增益不能超过模组认证时的参数; 集成方式需符合厂商集成指南; 某些国家仍要求整机做部分测试; 修改射频参数则认证失效。

### 6.3 常用预认证模组

| 厂商 | 模组型号 | 协议 | 已有认证 |
|------|----------|------|----------|
| Espressif | ESP32-WROOM | WiFi+BLE | FCC/CE/SRRC/MIC |
| Murata | CMWX1ZZABZ | LoRaWAN | FCC/CE/AS/MIC |
| Quectel | BG96 | NB-IoT/Cat-M | FCC/CE/SRRC多国 |
| RAK | RAK3172 | LoRaWAN | FCC/CE/SRRC |
| Nordic | nRF52840 DK | BLE 5.0 | FCC/CE |

## 7 认证流程、周期与成本

### 7.1 典型认证流程

```
步骤1: 预测试(Pre-scan)        -- 2-3天
  |  初步测试发现问题
  v
步骤2: 整改(如有问题)          -- 1-4周
  |  修改硬件/固件
  v
步骤3: 正式测试                 -- 3-5天
  |  出具正式测试报告
  v
步骤4: 提交申请                 -- 1天
  |  准备材料提交认证机构
  v
步骤5: 审核与发证               -- 2-8周(视国家)
  v
步骤6: 获得认证ID, 可合法销售
```

### 7.2 各市场周期与成本估算

| 认证 | 测试费(美元) | 审核费(美元) | 总周期 | 备注 |
|------|-------------|-------------|--------|------|
| FCC Part 15 | 3000-8000 | 1000-2000 | 4-8周 | 最通用 |
| CE RED | 3000-6000 | 1500-3000 | 6-10周 | 含公告机构费 |
| SRRC | 5000-15000 | 含在内 | 8-16周 | 需中国实验室 |
| MIC(日本) | 4000-8000 | 2000-3000 | 6-12周 | 限值严格 |
| IC(加拿大) | 2000-4000 | 500-1000 | 4-6周 | 可复用FCC报告 |

### 7.3 成本优化策略

- FCC和IC可以使用同一份测试报告(ISED接受FCC实验室数据)
- 部分认证支持系列化申请(同一设计不同外观)
- 使用预认证模组可将费用降低60-80%
- 预测试能提前发现问题,避免正式测试失败的重复费用

## 8 常见失败原因与应对

### 8.1 失败原因排名

```
杂散辐射超标    ||||||||||||||||  35%
谐波辐射超标    ||||||||||        22%
占用带宽超标    |||||||           15%
频率稳定性不达标 |||||            12%
功率超限        ||||              9%
标签/标识不合规  |||              7%
```

### 8.2 预防措施检查清单

```
设计阶段:
[ ] 射频部分与数字部分分区布局
[ ] 电源去耦电容充足且靠近IC
[ ] 天线馈点远离高速信号线
[ ] 预留谐波滤波器位置(即使初始不焊)
[ ] 接地平面完整无缝隙穿过射频区域

固件阶段:
[ ] 发射功率设置比限值低2-3dB余量
[ ] 频率校准程序正确
[ ] 占空比/驻留时间符合法规要求

预测试阶段:
[ ] 正式测试前做一次pre-scan
[ ] 验证所有工作模式(不仅默认模式)
[ ] 检查最高温度下的频率偏移
```

## 9 实践案例: LoRa设备通过FCC和CE认证

### 9.1 产品概述

一款基于Semtech SX1276的LoRaWAN环境传感器: 工作频段868MHz(EU)/915MHz(US); 发射功率+14dBm(EU)/+20dBm(US); 天线PCB印刷天线,增益2dBi; 目标市场美国和欧盟。

### 9.2 遇到的问题与解决

问题1: 915MHz三次谐波(2745MHz)落入WiFi频段,辐射超标。

```
解决方案: 天线馈点增加低通滤波器(截止频率1.2GHz)
- 3阶切比雪夫拓扑, 0402封装L和C
- 带内插损: <0.5 dB
- 2745MHz抑制: >30 dB
- 成本增加: 约0.03美元/台
```

问题2: CE测试中868MHz占用带宽超标(LoRa SF12 BW500kHz模式)。

```
解决方案: EU868频段带宽限制为125kHz
- 固件中对EU区域锁定带宽参数
- 500kHz模式仅在US915频段使用
```

### 9.3 最终测试结果

| 测试项目 | FCC限值 | CE限值 | 实测值 | 结果 |
|----------|---------|--------|--------|------|
| 传导功率 | +30dBm | +14dBm ERP | +13.8dBm | 通过 |
| 杂散(30M-1G) | -36dBm | -36dBm | -42dBm | 通过(余量6dB) |
| 杂散(1G-12.75G) | -30dBm | -30dBm | -38dBm | 通过(余量8dB) |
| 频率误差 | 正负20ppm | 正负20ppm | 正负3.2ppm | 通过 |

### 9.4 时间线总结

硬件冻结(第1周) -> 预测试(第2-3周) -> 发现问题改版(第4周) -> 验证通过(第5周) -> FCC正式测试(第6-7周) -> CE测试(第8-9周) -> 等待证书(第10-12周) -> 可上市(第13周)。总计约3个月含一次整改。

## 10 给初创团队的建议

- 预算中务必包含至少一次测试失败的整改费用
- 尽早做预测试(pre-scan),不要等PCB定型后才去实验室
- 优先考虑预认证模组,除非有充分理由自研射频
- 选择有IoT设备经验的认证实验室(能提供整改建议)
- FCC和IC同时申请可节省时间(报告互认)

文档准备清单: 电路原理图(射频部分)、PCB布局图(标注天线位置)、方框图(系统架构)、操作说明书、设备照片(含IC位置标注)、标签样本(含FCC ID/CE标志)、天线规格书。

## 总结

IoT设备的射频认证是产品上市前不可绕过的关键环节。虽然流程看似繁琐,但只要在设计阶段充分考虑合规要求,选择合适的认证策略(如使用预认证模组),大多数IoT产品都能在合理的时间和预算内完成认证。核心要点: 设计时预留余量(功率比限值低3dB以上)、预留滤波器位置、尽早做预测试、选择经验丰富的实验室。认证不是产品开发的终点障碍,而是质量保证的重要组成部分。
