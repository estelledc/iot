# 多频段天线在IoT多模设备中的设计
> **难度**: 高级 | **领域**: 天线设计 | **阅读时间**: 约 22 分钟

## 引言

想象你要在一个火柴盒大小的空间里,同时装进一根收音机天线(能收长波)、一根对讲机天线(能收短波)和一根WiFi天线——而且它们之间不能互相干扰。这就是IoT多模设备天线设计面临的核心挑战。

现代IoT设备越来越"贪心":一台资产追踪器可能同时需要LoRa(868MHz)进行远距离通信、BLE(2.4GHz)进行近场配置、NB-IoT(700-900MHz)进行蜂窝回传。每种通信技术工作在不同频率,理论上需要不同尺寸的天线。如何在几平方厘米的PCB上容纳所有天线并保证性能,是射频工程师面临的核心难题。

本文将从天线基础出发,介绍多频段天线的设计技术、仿真方法、工程权衡,以及一个实际的双频段天线设计案例。

## 1. IoT设备的频段需求

### 1.1 常见IoT通信频段

| 技术 | 频段 | 波长 | 典型天线尺寸(1/4波长) |
|------|------|------|---------------------|
| LoRa EU | 868 MHz | 34.6 cm | 8.6 cm |
| LoRa US | 915 MHz | 32.8 cm | 8.2 cm |
| NB-IoT Band 8 | 900 MHz | 33.3 cm | 8.3 cm |
| NB-IoT Band 3 | 1800 MHz | 16.7 cm | 4.2 cm |
| GPS L1 | 1575 MHz | 19.0 cm | 4.8 cm |
| BLE/WiFi | 2400 MHz | 12.5 cm | 3.1 cm |
| WiFi 5GHz | 5800 MHz | 5.2 cm | 1.3 cm |

### 1.2 尺寸矛盾

天线物理尺寸与工作波长直接相关。对于Sub-GHz频段(868/915MHz),四分之一波长约8-9cm——但很多IoT设备整个PCB可能只有40x40mm。天线必须"缩小化"设计,代价是带宽变窄、效率下降。

天线效率与电尺寸的关系由Chu-Harrington极限描述:

```
对于电小天线(尺寸 << 波长):
- 带宽 ∝ (a/lambda)^3  (a为天线包络球半径)
- 效率受辐射Q值限制: Q_min ≈ 1/(ka)^3 + 1/(ka)
  其中 k = 2*pi/lambda, a = 天线最大尺寸/2

实际含义:
- 868MHz, PCB 40mm: ka ≈ 0.37, 可用带宽约39MHz(勉强覆盖LoRa)
- 2.4GHz, PCB 40mm: ka ≈ 1.0, 带宽轻松覆盖整个ISM频段
```

## 2. 多频段天线技术

### 2.1 多谐振倒F天线(Multi-resonant IFA)

倒F天线(IFA)是IoT设备中最常用的天线形式。通过增加分支可引入多个谐振频率:

```
          分支1(868MHz谐振)
         /
馈电点---+---主辐射体---末端开路
         \
          分支2(2.4GHz谐振)
         |
        接地短路点
         |
    === 地平面 ===
```

每个分支长度大约为对应频率四分之一波长的等效长度(考虑基板介电常数后会缩短)。

### 2.2 双频PIFA

平面倒F天线(PIFA)是IFA的面状扩展,通过在辐射贴片上开槽来产生多个谐振:

```
俯视图(PCB顶层铜箔):
+------------------------------------------+
|   +---+  +--------+                     |
|   | 槽 |  | 辐射贴片 |  <- 2.4GHz主谐振   |
|   +---+  +--+-----+                     |
|             |   <- 槽引入868MHz谐振      |
|    馈电 --> * 短路 --> GND via            |
+------------------------------------------+
```

### 2.3 独立天线方案

当性能要求高且空间允许时,为每个频段设计独立天线:

```
PCB布局(50x30mm):
+--------------------------------------------------+
| [Sub-GHz天线区]        [2.4GHz天线区]             |
| (蜿蜒IFA)              (短IFA/chip ant)          |
|    +--------+    +--------+                      |
|    | LoRa   |    | nRF52  |                      |
|    +--------+    +--------+                      |
+--------------------------------------------------+
```

优点: 每个天线独立优化,性能最佳。缺点: 占用更多空间,天线间需要考虑隔离。

## 3. 设计工具与仿真

### 3.1 电磁仿真软件

| 工具 | 方法 | 特点 | 适用场景 |
|------|------|------|---------|
| HFSS (Ansys) | FEM | 精度高,3D建模 | 复杂3D结构 |
| CST Studio | FDTD/FIT | 宽带仿真快 | 时域宽带分析 |
| ADS (Keysight) | MoM/FEM | 电路+电磁联合 | 匹配网络设计 |
| openEMS | FDTD | 开源免费 | 学习和验证 |

### 3.2 仿真流程与关键指标

```python
# 天线设计仿真流程
class AntennaDesignFlow:
    def __init__(self, freq_bands, pcb_size):
        self.freq_bands = freq_bands  # [(868e6, 928e6), (2.4e9, 2.5e9)]
        self.pcb_size = pcb_size      # (40e-3, 40e-3) meters
    
    def step1_initial_geometry(self):
        """根据频率估算初始天线尺寸"""
        for f_low, f_high in self.freq_bands:
            wavelength = 3e8 / ((f_low + f_high) / 2)
            quarter_wave = wavelength / 4
            # FR4介电常数缩短效应(er=4.4)
            effective_length = quarter_wave / ((4.4 + 1) / 2)**0.5
            print(f"{f_low/1e6:.0f}MHz: 估算长度 {effective_length*1000:.1f}mm")
    
    def step2_simulate(self):
        """全波电磁仿真: S11/辐射方向图/效率"""
        pass
    
    def step3_optimize(self):
        """参数扫描: 各频段S11 < -10dB, 效率 > 50%"""
        pass
```

关键指标: S11 < -10dB(90%能量被辐射); 辐射效率Sub-GHz通常30-60%, 2.4GHz可达60-80%; 隔离度多天线间至少10-15dB。

## 4. 匹配网络设计

### 4.1 多频段匹配的挑战

单频段匹配只需2-3个元件。多频段匹配要求在每个目标频段都实现良好匹配,且元件数量尽可能少:

```
典型双频段匹配网络:

天线 ---+--- L1 ---+--- 射频芯片
        |          |
        C1        C2
        |          |
       GND        GND

L1: 两个频段处都接近50ohm的电感值
C1: 主要影响低频段(868MHz)
C2: 主要影响高频段(2.4GHz)
```

### 4.2 可调匹配网络

对于覆盖多个蜂窝频段的NB-IoT设备,可使用可调电容(Varactor)、RF开关切换不同匹配网络,或天线调谐器IC(如Qualcomm QAT3516)。

## 5. 地平面效应

### 5.1 地平面作为天线的一部分

在Sub-GHz频段,PCB地平面实际上是天线辐射体的一部分。地平面尺寸对性能影响极为显著:

| 地平面尺寸 | 868MHz效率 | 2.4GHz效率 | 谐振偏移 |
|-----------|-----------|-----------|---------|
| 20x20mm | 15-25% | 50-65% | 显著偏低 |
| 40x40mm | 30-45% | 60-75% | 中等偏低 |
| 60x60mm | 40-55% | 65-80% | 接近设计值 |
| 80x80mm | 45-60% | 70-80% | 基本吻合 |

设计建议: 天线布局在PCB一端,地平面在另一端; 天线正下方清除铜箔; 地平面边缘避免突然中断。

## 6. 天线间隔离

### 6.1 为什么需要隔离

当设备有多个射频前端同时工作时,一个射频的发射信号可能"泄漏"到另一个的接收端,造成接收灵敏度下降:

```
问题场景:
LoRa TX(14dBm @ 868MHz)泄漏到BLE RX(-96dBm灵敏度 @ 2.4GHz)
如果天线隔离度只有20dB: 泄漏功率 = -6dBm (远高于BLE底噪)
→ BLE接收机被阻塞
```

### 6.2 提高隔离度的方法

- 空间隔离: 天线间距增大(每增加1个波长约增加20dB)
- 频率隔离: 频率差距越大天然隔离越好(868MHz vs 2.4GHz约有15-20dB)
- 极化隔离: 两天线使用正交极化(理论增加20dB)
- 地平面隔离槽: 在天线间开缝隙阻断表面电流

## 7. 制造与工艺考虑

### 7.1 PCB工艺对天线的影响

天线性能对制造公差敏感,尤其在2.4GHz:

- 走线宽度公差 +/-0.05mm 在2.4GHz会导致数MHz频偏
- FR4的Er批次间可能3.8-4.8(标称4.4)
- 阻焊层覆盖天线走线会降低谐振频率

应对措施: 仿真目标比实际高2-3%; 预留匹配网络焊盘允许后期调谐; 天线区域标注公差要求和"不覆盖阻焊"。

### 7.2 芯片天线替代方案

| 芯片天线 | 频段 | 尺寸 | 效率 |
|---------|------|------|------|
| Johanson 2450AT | 2.4GHz | 3.2x1.6mm | 60-70% |
| Fractus Compact | 868MHz | 7x3mm | 20-35% |
| Molex 2068780001 | LTE多频 | 36x6mm | 30-50% |

芯片天线设计简单(按厂商参考设计放置),但效率比PCB天线低10-20%。

## 8. 实际案例: 40x40mm资产追踪器双频段天线

### 8.1 设计目标

- PCB: 40x40mm, 双层FR4, 1.0mm厚
- 频段: 868MHz(LoRa) + 2.4GHz(BLE)
- 目标S11: < -6dB; 效率: >25%(868MHz), >50%(2.4GHz)
- 天线区域不超过PCB面积30%

### 8.2 天线拓扑

选择多谐振IFA: 主辐射臂蜿蜒走线产生868MHz谐振,寄生分支产生2.4GHz谐振。

```
天线布局(PCB右上角):
+------------------------------+
|   [天线clearance区]          |  40mm
|  +--+  +--+  +--+           |  868MHz蜿蜒臂
|  |  |  |  |  |  |           |  (展开约42mm)
|  |  +--+  +--+  +--+        |
|  |                  |        |
|  +-[2.4G分支 15mm]  |        |
|  |                           |
|  *馈电   *短路               |
|  |       |                   |
|==|=======|===================|  <-- 地平面边界
|  [器件区: LoRa/BLE芯片]      |
+------------------------------+
```

### 8.3 仿真与实测结果

```
S11 vs 频率:
  0|
 -5|        *                          *
-10|      *   *                      *   *
-15|    *       **                 *       *
-20|  *           ****           *
-25+--+--+--+--+--+--+--+--+--+--+--+--+--> f(GHz)
   0.7 0.8 0.9 1.0  1.5  2.0 2.2 2.4 2.5
```

| 指标 | 仿真值 | 实测值 | 目标 | 达标 |
|------|--------|--------|------|------|
| 868MHz S11 | -12.3dB | -9.8dB | <-6dB | 是 |
| 868MHz效率 | 32% | 28% | >25% | 是 |
| 2.4GHz S11 | -18.5dB | -15.2dB | <-6dB | 是 |
| 2.4GHz效率 | 65% | 58% | >50% | 是 |

### 8.4 匹配网络

```
天线馈电点 --- 2.2nH --- +--- 射频开关 --- LoRa芯片(868MHz)
                         |           |
                        1.5pF       +--- BLE芯片(2.4GHz)
                         |
                        GND
```

由于LoRa和BLE不需要同时发射,使用射频开关共享天线,避免了隔离度问题。

### 8.5 制板验证与调谐流程

首批PCB回来后的典型调谐过程:

1. 用矢量网络分析仪(VNA)测量实际S11
2. 对比仿真: 通常实际谐振频率比仿真低3-5%
3. 调整匹配元件: 微调L和C值补偿偏差
4. 确认带宽覆盖目标频段
5. 进行辐射效率测量(需要暗室或Wheeler Cap法)
6. 装入外壳后重新测量,补偿外壳介电加载

### 8.6 外壳影响

产品外壳对天线性能影响显著:

- 塑料外壳(ABS, Er约2.5): 使谐振频率降低3-8%
- 金属外壳: 严重影响辐射,需要开天线窗口
- 橡胶密封圈: 局部介电加载,影响天线附近区域
- 人体接近: 手持设备需要考虑人手遮挡

设计时应在仿真中包含外壳模型,或在裸板调谐后对装壳偏移进行补偿。

## 9. 多天线布局最佳实践

在40-60mm级别的IoT PCB上布置多天线时:

- Sub-GHz天线放在PCB长边一端(利用地平面长度)
- 2.4GHz天线可以放在另一端或角落(对地平面依赖小)
- 天线下方(同层和对面层)清除地平面铜箔
- 馈电走线尽量短,做50欧姆阻抗控制
- 高频天线远离晶振、开关电源等干扰源
- 在PCB边缘给天线留出至少3mm"安静区"(无走线、无器件)

## 总结

多频段天线设计是IoT硬件中最具挑战性的环节之一。核心要点:

- Sub-GHz天线在小PCB上受Chu极限约束,效率和带宽难以兼得
- 多谐振IFA/PIFA通过分支结构在一个天线上实现多频段覆盖
- 地平面是Sub-GHz天线的关键组成部分,PCB尺寸直接影响性能
- 匹配网络需要在多个频段同时工作,设计复杂度显著增加
- 多天线系统需要考虑隔离度,避免射频间互相干扰
- 制造公差在2.4GHz以上影响显著,设计需留调谐余量
- 实际产品中天线性能总比仿真差,需要制板后实测调谐

天线设计没有万能公式,每个项目都需要根据尺寸约束、频段需求和性能目标做工程权衡。好的天线设计往往是"70分的折中",而非"某一项100分"。
