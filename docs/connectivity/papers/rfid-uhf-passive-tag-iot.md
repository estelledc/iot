# UHF RFID无源标签在IoT资产追踪中的应用
> **难度**：🟢 初级 | **领域**：RFID资产管理 | **阅读时间**：约 18 分钟

## 引言

去超市买东西, 收银员要逐个扫描商品条码。如果有一种技术, 能让你推着满满一购物车走过一个门, 所有商品瞬间被识别和计价, 那购物体验会完全不同。UHF RFID(超高频射频识别)就是这样一种技术: 它能在几秒钟内同时识别数百个物品, 不需要对准、不需要可视, 穿过纸箱和塑料包装也没问题。

本文将从无源标签的基本原理讲起, 逐步介绍UHF RFID在IoT资产追踪中的实际应用、系统集成和选型考量。

## 1. UHF RFID技术概述

### 1.1 什么是UHF RFID

UHF RFID是一种利用超高频(860-960MHz)无线电波进行物品识别的技术。系统由两个核心组件构成: 读写器(Reader)和标签(Tag)。

读写器发射无线电波, 标签接收到这些电波后, 利用其中的能量驱动自身芯片工作, 并将存储的数据通过反向散射(backscatter)方式返回给读写器。

关键特性:

- 工作频率: 860-960 MHz(各国/地区有细分频段)
- 无源标签: 不需要电池, 从读写器的RF场中获取能量
- 读取距离: 1-12米(取决于标签和读写器性能)
- 批量读取: 每秒可识别数百个标签
- 穿透能力: 可穿过纸箱、塑料、布料等非金属材料

### 1.2 与其他RFID频段的对比

RFID技术覆盖多个频段, 各有特点:

| 频段 | 频率 | 典型距离 | 批量读取 | 主要应用 |
|------|------|---------|---------|---------|
| LF(低频) | 125-134 kHz | < 10 cm | 否 | 动物标识, 门禁 |
| HF(高频) | 13.56 MHz | < 1 m | 有限 | 图书馆, 公交卡 |
| UHF(超高频) | 860-960 MHz | 1-12 m | 是 | 供应链, 零售 |

UHF的优势在于远距离和批量读取能力, 这使它成为大规模资产追踪的首选技术。

## 2. 无源标签工作原理

### 2.1 反向散射通信

无源标签的工作原理可以用"镜子反射阳光"来类比。读写器就像太阳, 持续发出RF能量(光线)。标签就像一面可以快速翻转的镜子: 镜面朝向读写器时反射强(代表"1"), 转开时反射弱(代表"0")。通过快速切换反射状态, 标签就"说话"了。

技术上, 标签通过改变天线的阻抗来调制反射信号。当天线阻抗匹配时, 吸收能量多、反射少; 当阻抗失配时, 反射增大。这种调制方式叫做反向散射(backscatter)调制。

```
读写器发射连续RF载波
         |
         v
+--------+--------+
|   标签天线接收    |
|                  |
|  芯片切换阻抗:    |
|  匹配 -> 低反射   |  "0"
|  失配 -> 高反射   |  "1"
|                  |
|  调制后的反射波   |
+--------+--------+
         |
         v
读写器接收并解码反射信号
```

### 2.2 能量收集与启动

标签没有电池, 那芯片怎么工作? 答案是能量收集。标签天线接收读写器的RF信号, 芯片内部的整流电路将RF能量转换为直流电, 给芯片供电。

标签芯片的典型功耗在10-30微瓦(uW)级别, 而读写器在标签位置提供的功率通常在微瓦到毫瓦级别, 足以驱动芯片。

标签的灵敏度(sensitivity)是一个关键参数, 表示标签能被激活的最小接收功率。现代UHF RFID标签的灵敏度通常在-18到-22dBm之间, 即约16到6微瓦。

### 2.3 标签的物理组成

一个UHF RFID标签由三个部分组成:

天线: 通常是印刷或蚀刻在基材上的金属图案。天线设计决定了标签的读取距离、方向性和频率特性。最常见的是偶极子(dipole)天线, 长度约为半波长(约16cm)。

芯片(IC): 存储EPC编码和用户数据, 执行通信协议逻辑, 包含整流电路和调制解调电路。芯片面积极小, 通常不到1平方毫米。

基材(Substrate): 承载天线和芯片的材料, 可以是PET薄膜、纸张、织物等。基材的选择影响标签的柔韧性、耐久性和成本。

这三部分组合后叫做"Inlay"(嵌体)。Inlay再封装成各种形态: 贴纸标签、硬质标签、耐高温标签、抗金属标签等。

## 3. EPC编码体系

### 3.1 什么是EPC

EPC(Electronic Product Code, 电子产品代码)是RFID标签中存储的唯一标识符。如果说条形码是商品的"身份证号", 那么EPC就是每一件商品个体的"身份证号"。条形码只能区分"这是可口可乐330ml", 而EPC可以区分"这是2024年3月15日生产的第10042瓶可口可乐330ml"。

### 3.2 96位EPC结构

标准的96位EPC编码结构如下:

```
|  Header  | Filter | Partition | Company Prefix | Item Ref | Serial |
|  8 bits  | 3 bits |  3 bits   |   20-40 bits   | 24-4 bits| 38 bits|
|----------|--------|-----------|----------------|----------|--------|
总计: 96 bits

示例(SGTIN-96编码):
Header:     0x30 (SGTIN-96格式)
Filter:     001  (贸易单品)
Partition:  101  (公司前缀6位)
Company:    0614141 (示例公司)
Item Ref:   012345
Serial:     1234567890
```

96位EPC可以表示约7.9 * 10^28个唯一标识, 足以给地球上每一粒沙子编号。

### 3.3 标签内存组织

EPC Gen2标签的内存分为四个Bank:

| Bank | 名称 | 内容 | 大小 |
|------|------|------|------|
| 0 | Reserved | 销毁密码 + 访问密码 | 64 bits |
| 1 | EPC | EPC编码 + CRC + PC | 96-496 bits |
| 2 | TID | 芯片厂商ID + 型号 | 32-96 bits |
| 3 | User | 用户自定义数据 | 0-512+ bits |

Bank 2(TID)是芯片出厂时写入的, 不可更改, 相当于芯片的"出生证明"。Bank 1(EPC)和Bank 3(User)通常可以读写, 用于写入业务数据。

## 4. 读取距离影响因素

### 4.1 链路预算分析

UHF RFID的读取距离由"链路预算"(Link Budget)决定。这是一个能量收支平衡问题: 读写器发出的RF能量经过空间传播衰减后, 到达标签时必须仍然大于标签的灵敏度阈值。

```
读取距离估算(Friis公式简化版):

d = (lambda / 4*pi) * sqrt(Pt * Gt * Gr / Pth)

其中:
  d     = 读取距离(米)
  lambda = 波长(约0.33m, 在915MHz)
  Pt    = 读写器发射功率(W)
  Gt    = 读写器天线增益
  Gr    = 标签天线增益
  Pth   = 标签灵敏度阈值(W)

示例计算:
  Pt = 1W (30dBm)
  Gt = 6dBi (约4倍)
  Gr = 2dBi (约1.6倍)
  Pth = -20dBm (10uW)

  d = (0.33 / 12.57) * sqrt(1 * 4 * 1.6 / 0.00001)
  d 约= 0.026 * 800 = 约21m (理论值)

实际值通常是理论值的30-50%, 即6-10m
```

### 4.2 实际环境对距离的影响

理论计算和实际读取距离之间总有差距, 主要原因包括:

多径效应: RF信号在地面、墙壁、金属表面反射, 产生多个传播路径。这些路径的信号可能相互增强或抵消, 导致距离忽远忽近。

金属干扰: 金属表面会反射UHF信号, 改变天线辐射方向图。标签贴在金属表面上时, 普通标签几乎无法工作(需要专用抗金属标签)。

液体衰减: 水对UHF频段有较强的吸收作用。含水量高的物品(饮料、生鲜)会显著降低标签读取距离。

标签方向: 线性极化标签在特定方向上性能最佳。如果标签朝向与读写器天线极化方向垂直, 读取距离会大幅下降。使用圆极化天线可以缓解此问题, 但会损失约3dB增益。

## 5. 批量读取与防碰撞

### 5.1 为什么需要防碰撞

当数百个标签同时处于读写器的覆盖范围内时, 如果所有标签同时响应, 信号会相互干扰(碰撞), 读写器无法正确解码任何一个标签的数据。防碰撞(anti-collision)协议解决了这个问题。

### 5.2 时隙ALOHA算法

EPC Gen2使用基于时隙ALOHA(Slotted ALOHA)的防碰撞算法:

```
读写器                         标签群(假设8个标签)
  |                              |
  |--- Query(Q=3, 8个时隙) ----->|
  |                              | 每个标签随机选择0-7中的一个时隙
  |                              |
  | 时隙0: [标签A响应]           | <- 成功读取
  | 时隙1: [标签B+D同时响应]     | <- 碰撞, 跳过
  | 时隙2: [无响应]              | <- 空时隙
  | 时隙3: [标签C响应]           | <- 成功读取
  | 时隙4: [标签E响应]           | <- 成功读取
  | 时隙5: [标签F+G+H同时响应]   | <- 碰撞, 跳过
  | 时隙6: [无响应]              | <- 空时隙
  | 时隙7: [无响应]              | <- 空时隙
  |                              |
  | 第一轮: 读取了3个, 碰撞2次    |
  |                              |
  |--- 第二轮Query(Q=2) -------->| 缩小Q, 只为剩余标签
  | ...继续直到所有标签被读取...   |
```

Q参数控制时隙数量(2^Q个时隙)。读写器根据碰撞率动态调整Q: 碰撞多则增大Q(更多时隙减少碰撞概率), 空时隙多则减小Q(加快速度)。

### 5.3 读取速率

在理想条件下, EPC Gen2的标签读取速率:

- 每个成功时隙约需2-3毫秒
- 考虑碰撞和空时隙的开销, 有效读取速率约200-1000标签/秒
- 实际环境中, 200-400标签/秒是比较可靠的数字

这个速度足以支持: 一辆满载货物的卡车通过仓库大门时, 在3-5秒内清点所有货物。

## 6. IoT资产追踪应用场景

### 6.1 仓储物流

仓库是UHF RFID最成熟的应用场景。典型部署:

入库: 货物通过安装有固定读写器的收货门, 自动识别所有箱子上的RFID标签, 与采购订单核对。

库存盘点: 工人持手持读写器在通道中行走, 数秒内扫描整排货架上的所有商品。传统人工盘点一个仓库可能需要一周, RFID盘点只需数小时。

出库: 货物通过发货门时自动核对装车清单, 防止漏装错装。

### 6.2 零售门店

零售行业正在大规模采用UHF RFID, 特别是服装零售:

库存准确性: RFID将库存准确率从约65%提升到95%以上。准确的库存数据意味着更少的缺货和更高的销售额。

防损: 标签在收银时消磁/注销, 未消磁的标签触发门禁报警。

全渠道零售: 线上下单、门店取货(BOPIS)依赖准确的门店库存数据, RFID使这成为可能。

### 6.3 制造业WIP追踪

在制造车间, RFID追踪在制品(Work-In-Progress)流经各个工序:

```
原材料 -> [工序1] -> [工序2] -> [工序3] -> 成品
  ^         ^          ^          ^         ^
  |         |          |          |         |
 RFID门   RFID门     RFID门    RFID门    RFID门
  |         |          |          |         |
  +-------- IoT平台: 实时WIP看板 ----------+
```

每个工序入口安装RFID读写器, 产品经过时自动记录。MES(制造执行系统)实时知道每个产品处于哪个工序, 从而优化排产、发现瓶颈、追溯质量问题。

### 6.4 医疗资产管理

医院中高价值设备(输液泵、轮椅、监护仪)数量庞大且经常移动。护士找设备平均花费30分钟/班次。RFID系统在病区出入口和关键位置安装读写器, 实时追踪设备位置, 显著减少寻找时间。

## 7. 固定读写器与手持读写器

### 7.1 固定读写器

固定读写器安装在固定位置(如门口、传送带旁), 持续监控经过的标签。

优势: 自动化程度高, 无需人工操作; 可24/7不间断工作; 适合流动物品的自动识别(如传送带、出入口)。

劣势: 安装成本高; 读取区域固定, 覆盖范围有限; 需要布线供电和联网。

典型部署: 仓库收发货口, 生产线工序间, 零售店出入口。

### 7.2 手持读写器

手持读写器是便携式设备, 由人员携带使用。

优势: 灵活性高, 可到达任何区域; 初始投资较低; 适合不定期的盘点任务。

劣势: 依赖人工操作; 读取速度受操作者影响; 不适合7x24小时持续监控。

典型使用: 库存盘点, 资产审计, 货物查找。

## 8. 与IoT平台的集成

### 8.1 数据流架构

RFID读写器产生的原始数据需要经过处理才能变成有价值的业务信息:

```
RFID读写器 -> RFID中间件 -> IoT平台 -> 业务应用
               |               |
          过滤/去重/聚合    事件处理
          LLRP协议         MQTT/HTTP
```

RFID中间件负责: 过滤重复读取(同一标签每秒可能被读取多次); 聚合事件(多个读取合并为"进入/离开"事件); 协议转换(LLRP -> MQTT/HTTP)。

### 8.2 数字孪生更新

在IoT平台中, 每个被追踪的实物资产都有一个数字孪生(Digital Twin)。RFID事件实时更新数字孪生的状态:

```json
{
  "assetId": "SGTIN:0614141.012345.1234567890",
  "location": "warehouse-A-zone-3-rack-12",
  "lastSeen": "2024-03-15T14:23:45Z",
  "status": "in-stock",
  "events": [
    {"type": "received", "time": "2024-03-15T10:00:00Z"},
    {"type": "putaway", "time": "2024-03-15T10:30:00Z"},
    {"type": "inventory", "time": "2024-03-15T14:23:45Z"}
  ]
}
```

## 9. 标签选型指南

### 9.1 不同环境的标签类型

不同应用场景需要不同类型的标签:

| 应用环境 | 标签类型 | 特点 | 价格范围 |
|---------|---------|------|---------|
| 纸箱/包装 | 通用贴纸标签 | 薄, 便宜, 可打印 | $0.05-0.10 |
| 金属表面 | 抗金属标签 | 有间隔层, 厚 | $0.50-3.00 |
| 液体容器 | 耐液体标签 | 特殊天线设计 | $0.30-1.00 |
| 高温环境 | 耐高温标签 | 陶瓷/特种材料 | $1.00-5.00 |
| 服装 | 织物标签/吊牌 | 柔软, 可洗涤 | $0.05-0.15 |
| 小型物品 | 微型标签 | 尺寸小, 距离短 | $0.20-1.00 |

### 9.2 选型决策流程

选择标签时需要考虑的关键问题:

第一, 贴附表面是什么材质? 金属和液体需要特殊标签。第二, 需要多远的读取距离? 距离要求越高, 天线越大, 成本越高。第三, 环境条件如何? 温度、湿度、化学品暴露都影响标签寿命。第四, 数据需求是什么? 只需EPC还是需要额外用户数据? 第五, 批量多大? 量大可以降低单价。

### 10.2 投资回报

RFID系统的ROI主要来自:

人工节省: 自动化盘点取代人工, 一个中型仓库每年可节省2-4个人工盘点岗位。库存准确性提升: 减少缺货损失和过量库存, 零售行业数据显示可提升销售额2-5%。损耗减少: 防损系统降低盗窃和管理损耗, 典型改善0.5-1%。运营效率: 加快收发货速度, 减少发货错误。

## 11. 局限性与注意事项

UHF RFID并非万能, 了解其局限性同样重要:

金属屏蔽: 完全封闭的金属容器内的标签无法被读取。信号无法穿透连续金属表面。

液体衰减: 大量液体(如整箱饮料)会严重吸收UHF信号, 需要调整天线位置或使用更高功率。

读取区域控制: 有时"读得太远"也是问题。比如在仓库走道中, 读写器可能误读相邻区域的标签。需要通过天线方向图设计和功率控制来限定读取区域。

标签方向: 线性极化标签有方向性。如果标签方向随机(如杂乱堆放的包裹), 需要使用圆极化天线, 并接受约3dB的增益损失。

隐私问题: RFID标签可以被任何兼容读写器读取, 存在隐私和安全风险。Gen2v2协议引入了加密和认证机制来缓解此问题。

## 总结

UHF RFID无源标签技术以其远距离读取、批量识别和无需电池的特性, 成为IoT资产追踪领域的核心技术。从仓储物流到零售门店, 从制造车间到医院病房, RFID正在推动实物资产管理从"人工 + 条码"向"自动化 + 数字孪生"转型。

选型时的核心考量: 应用环境决定标签类型, 业务规模决定系统架构, 集成能力决定数据价值。成本方面, 标签单价已经低到可以大规模部署, 系统的投资回报主要来自人工节省和库存准确性提升。

理解了这些基础, 你就有了评估"我的场景是否适合RFID"的判断框架。
