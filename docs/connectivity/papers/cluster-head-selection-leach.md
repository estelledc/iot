# 簇头选择LEACH协议与改进算法
> **难度**: 高级 | **领域**: 路由协议 | **阅读时间**: 约 22 分钟

## 引言

想象一个班级要向校长汇报情况。如果每个同学都直接跑去校长室汇报, 校长室会被挤爆, 同学们也浪费大量时间排队。更好的做法是把全班分成几个小组, 每组选一个组长。组长收集本组同学的信息, 汇总后代表小组去找校长汇报。这样校长只需要接待几个组长, 效率大幅提高。

分层路由的思想与此完全一致。在无线传感器网络(WSN)中, 将节点组织成若干个簇(cluster), 每个簇选出一个簇头(cluster head, CH)。普通成员节点只需将数据发给簇头(短距离), 簇头负责数据聚合后发给汇聚节点sink(长距离)。LEACH(Low-Energy Adaptive Clustering Hierarchy)是第一个也是最具影响力的分层路由协议, 提出了轮转式簇头选举的核心思想, 至今仍是WSN路由研究的基准参考。

## 1. 分层路由的动机

### 1.1 扁平路由的问题

在扁平路由中, 所有节点地位平等, 每个节点都参与数据转发:

```
扁平路由的问题:
  [S1] -> [A] -> [B] -> [C] -> [sink]
  [S2] -> [A] -> [B] -> [C] -> [sink]
  [S3] -> [D] -> [B] -> [C] -> [sink]

  节点B和C转发所有数据, 能耗远高于边缘节点
  结果: B和C最先死亡, 网络分裂
```

### 1.2 分层路由的优势

```
分层路由:
  簇1: [S1, S2, S3] -> [CH1] --\
  簇2: [S4, S5, S6] -> [CH2] ---}-> [sink]
  簇3: [S7, S8, S9] -> [CH3] --/

  优势:
  1. 簇内短距离通信: 成员节点发射功率低
  2. 数据聚合: CH合并多个成员的数据再发送
     例: 10个温度值聚合为1个平均值, 传输量减90%
  3. TDMA调度: 成员在分配的时隙发送, 其余时间休眠
  4. 负载集中在CH: 通过轮转CH分散负载
```

## 2. LEACH协议概述

### 2.1 轮(Round)的概念

LEACH将网络运行划分为多个轮(round), 每轮包含两个阶段:

```
一轮(round)的结构:
  |<--- 建立阶段(setup) --->|<-------- 稳态阶段(steady-state) -------->|
  
  建立阶段:
    1. 簇头选举
    2. 簇形成(非CH节点加入最近的CH)
    3. CH创建TDMA调度表
    
  稳态阶段:
    1. 成员按TDMA时隙发送数据给CH
    2. CH接收、聚合数据
    3. CH将聚合数据发送给sink
    4. 成员在非自己时隙时休眠节能
    
  稳态阶段远长于建立阶段, 减少选举开销
```

### 2.2 设计目标

LEACH的核心设计目标是通过簇头轮转均衡能量消耗。每一轮选出不同的簇头, 使得簇头的高能耗负担在所有节点间轮流承担, 避免任何节点因为一直当簇头而过早耗尽电量。

## 3. LEACH簇头选举机制

### 3.1 概率选举算法

每一轮开始时, 每个节点独立决定自己是否成为簇头:

```
簇头选举算法:
  每个节点n生成随机数 rand (0到1之间)
  
  如果 rand < T(n), 则n成为本轮簇头
  
  阈值函数:
  T(n) = p / (1 - p * (r mod (1/p)))   如果n属于G
  T(n) = 0                              如果n不属于G
  
  p = 期望的簇头比例 (通常 5%)
  r = 当前轮次
  G = 最近 1/p 轮内未当过簇头的节点集合
```

### 3.2 阈值函数的含义

```
直觉理解T(n):
  假设p = 0.05 (5%节点当簇头), 1/p = 20轮一个周期
  
  第1轮: T(n) = 0.05/(1-0.05*0) = 0.0526, 约5.26%概率当选
  第5轮: T(n) = 0.05/(1-0.05*5) = 0.0667, 约6.67%概率
  第19轮: T(n) = 0.05/(1-0.05*19) = 1.0, 100%当选
  
  效果: 越久没当过簇头的节点, 当选概率越高
  保证: 每个节点在1/p轮内恰好当一次簇头
```

### 3.3 选举过程

```
选举后的簇形成:
  1. 当选CH广播ADV消息(包含CH标识)
  2. 非CH节点收到所有CH的ADV
  3. 非CH节点选择信号最强的CH(即最近的)
  4. 非CH节点向选定的CH发送JOIN消息
  5. CH收集所有JOIN, 创建TDMA调度表
  6. CH广播TDMA调度给所有成员
  7. 进入稳态阶段, 成员按时隙发送数据
```

## 4. LEACH运行细节

### 4.1 稳态阶段的TDMA调度

```
TDMA调度示例(簇内5个成员):
  时隙1: 成员A发送 -> CH接收
  时隙2: 成员B发送 -> CH接收
  时隙3: 成员C发送 -> CH接收
  时隙4: 成员D发送 -> CH接收
  时隙5: 成员E发送 -> CH接收
  时隙6: CH聚合并发送给sink
  
  节能机制:
  - 成员A在时隙2-6休眠, 只在时隙1唤醒
  - 占空比: 1/6 = 16.7%
  - 能耗降低约83%
```

### 4.2 数据聚合

CH收到所有成员的数据后, 执行聚合操作:

```
聚合示例:
  成员A报告: 温度 = 25.1
  成员B报告: 温度 = 25.3
  成员C报告: 温度 = 24.9
  成员D报告: 温度 = 25.0
  成员E报告: 温度 = 25.2
  
  聚合(平均): 温度 = 25.1
  
  无聚合: CH需发送5个数据包给sink
  有聚合: CH只发送1个数据包给sink
  传输量减少: 80%
```

### 4.3 CH到sink的通信

LEACH原始版本中, CH直接以单跳方式与sink通信。这意味着距离sink较远的CH需要更高的发射功率, 能耗不均衡。

## 5. LEACH的优势

### 5.1 核心优势总结

```
1. 简单分布式: 无需集中控制, 每个节点独立决策
2. 能量均衡: CH轮转机制分散高能耗角色
3. 数据聚合: 减少总传输量
4. TDMA调度: 有序的簇内通信, 成员可休眠
5. 开创性: 启发了数百种改进协议
```

### 5.2 与泛洪和直传的对比

| 指标 | 泛洪 | 直传(直接到sink) | LEACH |
|------|------|-----------------|-------|
| 簇内能耗 | 高(全网广播) | 高(每节点远传) | 低(短距到CH) |
| 总传输量 | 极高(冗余) | 中(无聚合) | 低(聚合) |
| 网络寿命 | 最短 | 中 | 最长 |
| 复杂度 | 最低 | 最低 | 中 |

## 6. LEACH的局限性

### 6.1 随机选举的问题

```
问题1: 可能选中低能量节点
  节点A: 剩余能量 10% -> 被选为CH -> 很快死亡
  节点B: 剩余能量 90% -> 未被选中 -> 能量浪费
  
问题2: 可能选中位置不佳的节点
  CH选在簇的边缘 -> 部分成员距CH很远
  成员到CH的能耗增大

问题3: CH数量不稳定
  期望5%=5个CH, 但随机选举可能得到2个或8个
  太少: 簇过大, CH负担重
  太多: 聚合效果差, 建立开销大
```

### 6.2 单跳到sink的限制

CH直接向sink发送数据, 距离远的CH需要高发射功率。这限制了LEACH的网络规模: 当网络面积增大时, 远端CH的能耗急剧增加。

### 6.3 不考虑剩余能量

原始LEACH的选举阈值T(n)仅基于轮次, 不考虑节点的剩余能量。一个几乎没电的节点和满电的节点有相同的当选概率。

## 7. LEACH-C: 集中式改进

### 7.1 核心改进

LEACH-C由基站(sink)集中执行簇头选举和簇划分:

```
LEACH-C流程:
  1. 所有节点向基站报告位置和剩余能量
  2. 基站排除能量低于平均值的节点
  3. 基站用模拟退火算法选择最优CH位置
  4. 基站将CH分配和簇划分广播给所有节点
  
  优势:
  - 保证CH数量恰好等于期望值
  - CH分布均匀, 覆盖合理
  - 排除低能量节点, 避免选中将死节点
  
  代价:
  - 需要全局信息(每个节点的位置和能量)
  - 通信开销: 每轮所有节点上报 + 基站广播
```

## 8. SEP: 稳定选举协议

### 8.1 异构网络的挑战

LEACH假设所有节点初始能量相同。但实际部署中, 节点可能有不同的初始能量(不同型号电池, 不同部署时间):

```
SEP设计:
  两类节点:
  - 高级节点(advanced): 初始能量 E0 * (1 + alpha)
  - 普通节点(normal): 初始能量 E0
  
  选举概率:
  - 高级节点: p_adv = p * (1 + alpha) / (1 + alpha * m)
  - 普通节点: p_nrm = p / (1 + alpha * m)
  
  m = 高级节点占比
  alpha = 能量倍数
  
  效果: 能量多的节点更频繁地当簇头
  合理: 有更多能量就多承担责任
```

## 9. DEEC: 分布式能量高效聚类

### 9.1 多级能量异构

DEEC将SEP扩展到任意级别的能量异构:

```
DEEC选举概率:
  p_i = p * E_remain(i) / E_avg

  E_remain(i) = 节点i的剩余能量
  E_avg = 网络平均剩余能量(可估算)

  效果:
  - 剩余能量高的节点: p_i > p, 更可能当CH
  - 剩余能量低的节点: p_i < p, 更少当CH
  - 随着运行, 高能量节点先多当CH, 能量逐渐均衡
  - 动态适应: 不需要预先知道异构程度
```

### 9.2 与LEACH的对比

LEACH的选举概率固定为p, 不考虑能量状态。DEEC的选举概率随剩余能量动态调整, 在网络运行过程中自动均衡能量消耗。

## 10. HEED: 混合能量高效分布式

### 10.1 双参数选举

HEED使用两个参数进行簇头选举:

```
HEED选举:
  主参数: 剩余能量
    CH_prob = C_prob * E_remain / E_max
    C_prob: 初始CH概率
    能量越高, 当选概率越大
    
  副参数: 通信代价
    簇内通信成本最低的节点优先
    考虑: 到邻居的平均距离、邻居密度
    
  迭代过程:
    1. 每个节点计算CH_prob
    2. 如果CH_prob >= 1, 确定成为CH
    3. 否则以CH_prob概率暂定为CH
    4. 检查邻居中是否有更好的CH
    5. 重复迭代直到所有节点确定角色
```

### 10.2 收敛保证

```
HEED收敛性:
  每次迭代: CH_prob翻倍
  初始CH_prob最小值: 10^(-4)
  
  最多迭代次数: log2(1/10^(-4)) = 约14次
  
  保证:
  1. 有限次迭代内所有节点确定角色
  2. CH分布均匀(考虑了通信代价)
  3. 不依赖全局信息(分布式执行)
```

## 11. 多跳CH到sink

### 11.1 解决单跳限制

LEACH原始版本CH直接向sink发送, 限制了网络规模。改进方案让CH之间形成多跳骨干网:

```
多跳路由:
  远端CH -> 中间CH -> 近sink的CH -> sink
  
  典型实现:
  PEGASIS(链式): 所有节点形成链, 轮流当链头
  TEEN(阈值): 只有数据超过阈值时CH才转发
  
  优势: 远端CH不需要高功率直传sink
  代价: CH间路由增加延迟和复杂度
```

### 11.2 PEGASIS链式方法

```
PEGASIS:
  所有节点形成一条链(贪心最近邻)
  每轮选一个链头负责发送给sink, 数据沿链聚合传输
  vs LEACH: 无需簇形成, 每个节点只和最近邻通信, 更均衡
```

## 12. 实际案例: 精准农业传感器网络

### 12.1 场景描述

500个传感器部署在农田中(50x50网格), 测量土壤湿度、温度和养分。汇聚节点在田地边缘。

### 12.2 不同协议的性能对比

```
LEACH:
  随机CH选择, 有时集中在一个区域
  25%能量浪费在不合理的簇结构上
  网络寿命: 8个月(第一个节点死亡)

LEACH-C:
  sink优化CH位置, 分布均匀
  无能量浪费在不合理簇结构上
  网络寿命: 12个月(提升50%)

HEED:
  分布式执行, 无需全局信息
  剩余能量 + 通信代价双参数
  网络寿命: 11个月(提升38%)
```

### 12.3 结论分析

```
三种协议对比:
  | 协议    | 寿命   | 需要全局信息 | 复杂度 |
  |---------|--------|-------------|--------|
  | LEACH   | 8个月  | 不需要      | 最低   |
  | LEACH-C | 12个月 | 需要        | 中     |
  | HEED    | 11个月 | 不需要      | 中     |
  
  最实用: HEED
  原因: 接近LEACH-C的性能, 但不需要全局信息
  适合: 大规模农业部署中无法集中收集所有节点信息的场景
```

最实用的选择是HEED: 接近LEACH-C的性能但不需要全局信息, 适合大规模部署。

## 总结

LEACH开创了WSN分层路由范式: 簇头轮转均衡能耗, 数据聚合减少传输量, TDMA调度让成员休眠节能。后续改进中LEACH-C通过集中优化改善CH分布, SEP和DEEC通过能量感知适应异构网络, HEED用双参数迭代在分布式条件下实现接近最优性能。这些协议共同构成了WSN路由研究的基石。
