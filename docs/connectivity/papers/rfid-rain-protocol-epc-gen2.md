# RAIN RFID EPC Gen2协议与读写器设计
> **难度**：🟡 中级 | **领域**：RFID协议 | **阅读时间**：约 20 分钟

## 引言

如果把UHF RFID系统比作一个大型点名系统, 那EPC Gen2协议就是这套点名系统的"规则手册"。教室里有几百个学生(标签), 老师(读写器)需要在几秒钟内点完所有人的名。不能让所有人同时喊"到", 否则谁也听不清。Gen2协议定义了一套精巧的规则: 谁先答、碰撞了怎么办、怎么加密防偷听, 以及老师的"扩音器"(读写器硬件)怎么设计。

本文将深入介绍RAIN RFID联盟、EPC Gen2协议的各个层面, 以及读写器的硬件架构和设计挑战。

## 1. RAIN RFID联盟与标准体系

### 1.1 什么是RAIN RFID

RAIN是"RAdio frequency IdentificatioN"的缩写, 是一个推广UHF RFID技术的全球行业联盟。RAIN之于UHF RFID, 就像Wi-Fi Alliance之于IEEE 802.11: 它不制定底层标准, 而是推动标准的商业化落地、产品互操作性认证和市场推广。

RAIN RFID联盟成立于2014年, 成员包括芯片厂商(Impinj, NXP, ST等)、读写器厂商(Zebra, Alien等)、系统集成商和终端用户。

### 1.2 标准层级

RAIN RFID的技术标准有一个层级结构:

```
ISO 18000-63 (国际标准)
    |
    v
EPCglobal Class-1 Gen2 (行业标准)
    |
    v
RAIN RFID认证 (互操作性测试)
```

ISO 18000-63是国际标准化组织的正式标准, EPCglobal(现属GS1)的Class-1 Generation-2是广泛使用的行业标准名称。两者在技术内容上基本等同, Gen2是日常使用更多的名称。

### 1.3 标准版本演进

| 版本 | 发布年份 | 关键特性 |
|------|---------|---------|
| Gen2 v1.0 | 2004 | 基础协议, 防碰撞, EPC读写 |
| Gen2 v1.2 | 2008 | 改进密集读写器模式 |
| Gen2 v2.0 | 2013 | 加密认证, 隐私保护 |
| Gen2 v2.1 | 2018 | 增强安全特性 |

## 2. 物理层规范

### 2.1 读写器到标签(R->T)通信

读写器向标签发送命令时使用ASK(幅移键控)调制, 搭配PIE(Pulse Interval Encoding)编码。ASK调制通过改变载波的幅度来传递信息。PIE编码用脉冲间隔的长短来区分"0"和"1": data-0持续一个Tari时间单位(6.25-25us), data-1持续1.5到2倍Tari。调制深度通常为80-100%(深ASK), 使标签更容易检测信号边沿。

### 2.2 标签到读写器(T->R)通信

标签通过反向散射方式向读写器发送数据, 有两种主要编码:

FM0编码: 每个比特周期边界都有电平翻转, "0"在中间再翻转一次, "1"不翻转。频谱集中, 解码简单, 但抗干扰较弱。

Miller编码: 在子载波上调制, 有Miller-2/4/8三种模式(数字表示每个数据比特对应的子载波周期数)。数字越大, 数据速率越低, 但抗干扰能力越强。Miller-8最适合密集读写器(DRM)环境。

### 2.3 数据速率

| 链路方向 | 编码方式 | 数据速率范围 |
|---------|---------|-------------|
| R->T | PIE | 26.7 - 128 kbps |
| T->R | FM0 | 40 - 640 kbps |
| T->R | Miller-2 | 20 - 320 kbps |
| T->R | Miller-4 | 10 - 160 kbps |
| T->R | Miller-8 | 5 - 80 kbps |

在典型应用中, R->T使用40-80kbps, T->R使用FM0在160-320kbps, 实现较好的速率和可靠性平衡。

### 2.4 全球频率分配

UHF RFID的工作频段各国不同:

| 地区 | 频段 | 特点 |
|------|------|------|
| 北美 | 902-928 MHz | 50个跳频信道, FCC Part 15 |
| 欧洲 | 865.6-867.6 MHz | 4个信道, ETSI EN 302 208 |
| 中国 | 920-925 MHz | 信道分配类似北美 |
| 日本 | 916.8-923.4 MHz | 混合跳频/LBT |

读写器需要适配不同地区的频率规划和功率限制。北美允许更高的发射功率(36dBm EIRP), 欧洲限制更严格(33dBm ERP)。

## 3. 标签内存架构

### 3.1 四个内存Bank详解

Gen2标签的内存分为四个Bank, 每个Bank有独立的访问控制:

```
Bank 0 - Reserved (保留区)
+------------------+------------------+
| Kill Password    | Access Password  |
|    32 bits       |    32 bits       |
+------------------+------------------+
  销毁标签用         写保护/锁定用

Bank 1 - EPC (电子产品代码)
+------+-----+---------------------------+
| CRC  | PC  |         EPC               |
|16bits|16bits|    96-496 bits            |
+------+-----+---------------------------+
 校验码 协议控制  唯一标识符

Bank 2 - TID (标签标识)
+------------------+------------------+
| MDID + Designer  | Tag Model Number |
|    (厂商ID)      |    (型号)        |
+------------------+------------------+
  出厂写入, 只读

Bank 3 - User (用户区)
+------------------------------------------+
|          Application Data                |
|          0 - 512+ bits                   |
|    (用户自定义, 如批次号/生产日期等)       |
+------------------------------------------+
```

### 3.2 访问控制与锁定

每个Bank可以单独设置访问权限:

| 锁定状态 | 无密码时 | 有密码时 |
|---------|---------|---------|
| Unlocked | 可读写 | 可读写 |
| Perma-locked(可读写) | 永远可读写 | 永远可读写 |
| Locked | 不可访问 | 输入密码后可读写 |
| Perma-locked(不可访问) | 永远不可访问 | 永远不可访问 |

Perma-lock(永久锁定)是不可逆操作, 一旦设置就无法更改。这在安全性要求高的场景(如防伪)中很有用。

## 4. 盘存协议(Inventory Protocol)

### 4.1 Query-ACK机制

Gen2协议使用Query-ACK机制进行标签盘存。整个流程如下:

```
读写器                              标签
  |                                  |
  |--- Query(Q=3) ----------------->|
  |    (开始盘存, 8个时隙)           |
  |                                  | 标签随机选取slot(0-7)
  |                                  | slot=0的标签回复RN16
  |<-- RN16(16位随机数) ------------|
  |                                  |
  |--- ACK(RN16) ------------------>|
  |    (确认收到, 回传相同RN16)       |
  |                                  |
  |<-- EPC + PC + CRC16 ------------|
  |    (标签发送自己的完整EPC)        |
  |                                  |
  |--- QueryRep -------------------->|
  |    (下一个时隙)                   |
  |                                  | slot计数器减1
  |                                  | 新的slot=0标签回复
  |    ...重复直到所有时隙完成...     |
```

### 4.2 Q值的自适应调整

Q参数决定了时隙数量(2^Q个)。Q的选择直接影响盘存效率:

Q太小: 时隙少, 碰撞频繁, 很多标签需要多轮才能被读到。Q太大: 时隙多, 大量空时隙浪费时间。最优的Q值应该使时隙数量接近标签数量。

读写器通过观察碰撞和空时隙的比例来动态调整Q:

```
自适应Q算法:

如果碰撞率 > 阈值:
    Q = Q + 1  (增加时隙数)
如果空时隙率 > 阈值:
    Q = Q - 1  (减少时隙数)
否则:
    Q保持不变

理论最优: 当标签数 N 约= 2^Q 时, 信道利用率约36.8%
```

36.8%的利用率是时隙ALOHA协议的理论上限(即1/e)。实际读写器通过更复杂的算法可以接近但不会超过这个值。

### 4.3 Session与Inventoried Flag

Gen2定义了4个Session(S0-S3)和Inventoried Flag(A/B状态), 解决多读写器环境下的协同问题:

每个标签对每个Session维护一个A/B状态位。读写器在Query命令中指定目标Session和目标Flag(A或B)。只有处于目标状态的标签才会参与盘存。被成功读取后, 标签翻转该Session的状态(A->B或B->A)。

```
标签的Session状态:

Session:  S0  S1  S2  S3
状态:      A   B   A   A

读写器1使用S0, 目标A:  -> 标签参与
读写器2使用S1, 目标A:  -> 标签不参与(S1=B)
读写器3使用S2, 目标A:  -> 标签参与
```

S0的状态持续时间最短(无限制, 立即翻转), S1-S3持续时间较长(500ms到数秒)。这意味着: S0适合需要反复快速盘存的场景; S1-S3适合多读写器环境, 避免重复读取。

### 4.4 Select命令与过滤

在盘存之前, 读写器可以用Select命令对标签进行筛选:

```
Select命令参数:
  Target:  S0的Flag / S1的Flag / S2的Flag / S3的Flag / SL标志
  Action:  匹配时设A还是B
  MemBank: 匹配哪个内存Bank(EPC/TID/User)
  Pointer: 从哪个bit开始匹配
  Length:  匹配多少bits
  Mask:    匹配的bit模式
```

例如: "只盘存EPC以'3034'开头的标签", 或"只盘存TID中厂商ID为NXP的标签"。Select大幅提高了定向盘存的效率, 避免读取不相关的标签。

## 5. 读写器硬件架构

### 5.1 总体架构

UHF RFID读写器是一个复杂的射频收发系统。与蓝牙或Wi-Fi不同, RFID读写器在发射信号的同时接收标签的微弱反向散射信号, 这对硬件设计提出了极高的要求。

```
读写器硬件框图:

                    +----------------+
                    |  数字基带处理  |
                    | (FPGA/DSP)    |
                    +---+------+----+
                        |      |
                   TX数据  RX数据
                        |      |
              +---------+      +---------+
              |                          |
        +-----+-----+            +------+------+
        |  TX链路   |            |   RX链路    |
        | DAC->调制 |            | 混频->ADC  |
        | ->PA     |            | ->滤波     |
        +-----+----+            +------+-----+
              |                        |
              +---+  环形器/耦合器 +---+
                  |  (隔离TX/RX) |
                  +------+-------+
                         |
                    +----+----+
                    |  天线   |
                    +---------+
```

### 5.2 发射与接收链路

发射(TX)链路由三个核心组件构成: 频率合成器产生精确载波, 需支持跳频和低相位噪声; 调制器对载波进行ASK调制; 功率放大器(PA)将信号放大到30-33dBm(1-2W)。

接收(RX)链路负责从极其微弱的反向散射信号中提取数据: 低噪声放大器(LNA)放大信号, 混频器下变频到基带(通常使用零中频架构), ADC将模拟信号数字化送入基带处理器。

### 5.3 TX-RX隔离: 最关键的挑战

RFID读写器最大的设计挑战是TX-RX隔离。发射功率约30dBm(1W), 接收的反向散射信号仅约-70dBm(0.1nW), 两者差了100dB。隔离通过多级实现: 环形器提供20-25dB, TX泄漏消除电路20-30dB, 数字自干扰消除20-30dB, 合计60-85dB, 剩余泄漏由接收机动态范围吸收。

## 6. 密集读写器模式(DRM)

### 6.1 问题定义

当多个读写器在同一区域工作时, 它们之间会产生干扰:

读写器间干扰: 一个读写器的发射信号直接干扰另一个读写器的接收。标签响应干扰: 标签对一个读写器的响应被另一个读写器误读。频谱拥挤: 有限的频谱被多个读写器共用。

### 6.2 DRM解决方案

Gen2协议和各地区法规提供了多种DRM解决方案: 频率规划方面, 北美使用跳频扩频(FHSS)50个信道每次驻留最多400ms, 欧洲使用先听后发(LBT)。编码方面, 在DRM环境中使用Miller-4或Miller-8, 其反向散射频谱更窄, 减少相邻信道干扰。此外, 在可控环境中还可通过中央控制器协调多个读写器的发射时序。

## 7. 安全特性

### 7.1 基础Gen2安全机制

初代Gen2协议的安全机制相对简单:

Kill命令: 发送32位销毁密码, 永久禁用标签。用于零售场景中消费者离店后保护隐私。一旦Kill, 标签不可恢复。

Access密码: 32位密码保护Bank 0和Bank 3的写操作。只有提供正确密码才能修改EPC或用户数据。

Cover-Coding: 在空中传输敏感数据时, 将数据与随机数异或(XOR)。这不是真正的加密, 但可以防止简单的窃听。

### 7.2 Gen2v2安全增强

Gen2v2(ISO 18000-63 Amendment)引入了真正的密码学安全特性:

AES-128加密套件: 标签和读写器之间可以建立AES-128加密通道, 保护数据传输的机密性和完整性。

标签认证(Tag Authentication): 读写器可以验证标签的真实性。读写器发送一个随机挑战(challenge), 标签用内部密钥计算MAC(消息认证码)并返回。读写器验证MAC, 确认标签是真品。

```
标签认证流程:

读写器                            标签
  |                                |
  |--- Challenge(随机数) -------->|
  |                                | MAC = AES(Key, Challenge)
  |<-- Response(MAC) -------------|
  |                                |
  | 验证: MAC' = AES(Key, Challenge)
  | if MAC == MAC': 真品
  | else: 伪造
```

双向认证(Mutual Authentication): 不仅读写器验证标签, 标签也验证读写器。防止恶意读写器从标签中提取数据。

Untraceable命令: 指令标签隐藏部分或全部EPC, 只对授权读写器可见。这对隐私保护很重要: 消费者离开商店后, 标签对普通读写器不可见, 但商店的授权读写器仍能读取。

### 7.3 防伪应用

Gen2v2的认证功能使RFID成为强大的防伪工具。每个标签内置唯一加密密钥, 验证时执行加密运算返回结果, 即使拷贝了EPC和用户数据, 没有密钥也无法通过认证。这个机制已在药品防伪、奢侈品验证、证件防伪等场景落地。

## 8. 读写器软件接口

### 8.1 LLRP协议

LLRP(Low Level Reader Protocol)是GS1定义的标准读写器控制协议, 基于TCP/IP, 使读写器与上位机之间的通信标准化。

```
上位机/中间件 <--LLRP(TCP)--> 读写器

LLRP消息类型:
  ADD_ROSPEC:       配置盘存操作参数
  ENABLE_ROSPEC:    启动盘存
  RO_ACCESS_REPORT: 读写器上报标签数据
  SET_READER_CONFIG: 设置射频参数(功率/频率等)
```

LLRP使不同厂商的读写器可以被同一套软件控制, 降低了系统集成的复杂度。

### 8.2 与IoT平台的集成

在IoT架构中, RFID读写器通常通过中间件与云端IoT平台集成:

```
RFID读写器 --(LLRP)--> 边缘中间件 --(MQTT)--> IoT平台
                          |
                     过滤/聚合/
                     事件生成
```

中间件负责将原始的"标签被读到"事件转化为有业务含义的"货物入库"、"设备移动"等高级事件。

## 总结

EPC Gen2协议是UHF RFID系统的核心标准, 它定义了从物理层调制编码到应用层安全认证的完整协议栈。理解Gen2协议有助于在实际项目中做出正确的技术选择: 选择FM0还是Miller编码、如何设置Q值和Session、是否需要Gen2v2安全特性。

读写器设计方面, TX-RX隔离和自干扰消除是最核心的挑战。密集读写器部署需要频率规划和编码选择的配合。随着Gen2v2的普及, RFID不仅是识别工具, 更成为了防伪和安全认证的载体。

对于IoT系统集成者来说, 理解这些协议细节并不是要自己设计读写器, 而是能够在系统设计时做出正确的参数配置和产品选型决策。
