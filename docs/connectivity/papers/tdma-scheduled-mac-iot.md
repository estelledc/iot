# TDMA调度式MAC在工业IoT中的确定性通信
> **难度**: 高级 | **领域**: MAC协议 | **阅读时间**: 约 22 分钟

## 引言

想象你在管理一条工厂流水线上的多个工位。如果工人们随时把零件往传送带上放(竞争式), 零件会互相碰撞掉落。更好的办法是: 给每个工位分配固定的时间窗口, 轮到谁谁才放零件, 永远不会撞车。

这就是TDMA(时分多址)的核心思想: 把时间切成一格一格, 每个设备在自己的格子里发送, 保证不碰撞。在工业IoT场景中, 这种确定性尤为重要。工厂里一个阀门的控制命令如果因为碰撞延迟了100ms, 可能导致整条产线停机。

本文将深入探讨TDMA在工业IoT中的设计原理, 重点分析IEEE 802.15.4 TSCH(时隙信道跳频)和WirelessHART的实现方案。

## 1. TDMA基本原理

### 1.1 时间划分

TDMA将时间组织为两层结构:

```
帧(Frame): 固定时长的周期, 循环重复
时隙(Slot): 帧内的最小时间单元

|<============= 帧 (Frame) =============>|
| 时隙0 | 时隙1 | 时隙2 | ... | 时隙N-1 |
| 设备A  | 设备B  | 设备C  | ... | 设备X   |

每个设备只在自己的时隙内发送
不同设备的时隙不重叠, 循环重复
```

### 1.2 时隙内部结构

一个时隙不仅仅是"发送数据", 还包含多个阶段:

```
|<------------ 一个时隙(如10ms) ------------>|
|保护|TX准备|  数据传输  |  等待ACK  |保护|空闲|
|时间|      |           |          |时间|    |

保护时间(Guard Time): 吸收时钟漂移和传播延迟
TX准备: 打开无线电, 频率稳定
数据传输: 实际数据发送
等待ACK: 接收方确认
空闲: 预留余量
```

### 1.3 帧长度选择

帧长度决定了每个设备的通信周期:

```
示例: 100ms帧, 10个时隙
- 每个时隙: 10ms
- 每个设备: 每100ms获得一次发送机会
- 数据包时长: 约2ms(802.15.4, 100字节)
- 每时隙有效利用: 2ms/10ms = 20%
- 其余80%: 保护时间+ACK+空闲

trade-off:
- 更短帧: 更低延迟, 但保护时间占比增大
- 更长帧: 更高效率, 但延迟增大
- 更多时隙: 支持更多设备, 但每设备带宽降低
```

## 2. TDMA的核心优势

### 2.1 确定性保证

与竞争式MAC的对比:

```
CSMA/CA:
- 最好情况: 退避0, 立即发送, 延迟约5ms
- 最坏情况: 最大退避+碰撞重试, 延迟大于100ms
- 无法预测: 取决于网络负载和碰撞
- 不适合: 实时控制

TDMA:
- 固定延迟: 等待自己的时隙, 最大=帧长度
- 可预测: 发送时刻精确已知
- 无碰撞: 不需要重传(除非信道错误)
- 适合: 工业控制, 确定性通信
```

### 2.2 无碰撞通信

TDMA从设计上消除了碰撞: 每个时隙只有一个设备发送, 不存在争抢。不需要CCA也不需要退避, 带宽利用率完全可预测。

能耗收益: CSMA/CA发送一包约0.3mJ(含CCA、退避、碰撞重传), TDMA约0.24mJ(纯数据传输), 节省约20%。

### 2.3 QoS保障

TDMA天然支持服务质量: 带宽保证(分配更多时隙=更大带宽), 延迟保证(时隙位置固定=延迟上界已知), 优先级(关键设备分配更早或更多时隙), 公平性(通过分配策略控制)。

## 3. 时间同步

### 3.1 为什么必须同步

TDMA的正确运行完全依赖时间同步。如果节点时钟漂移导致时隙边界偏移, 相邻时隙的传输会重叠产生碰撞。

```
典型晶振精度: 10-40 ppm(parts per million)
- 20ppm意味着: 每秒偏差20微秒
- 1分钟后偏差: 1.2ms
- 10分钟后偏差: 12ms

对于10ms时隙, 1ms保护时间:
- 保护时间能容纳: 0.5ms单向漂移
- 20ppm晶振: 0.5ms / 20us每秒 = 25秒
- 即: 每25秒必须重新同步一次!
```

### 3.2 信标同步

```
网络协调器周期性广播信标:
|信标|                              |信标|
  ^                                   ^
  所有节点对齐时间                    再次对齐

信标内容: 当前帧号, 时隙号, 精确时间戳, 调度信息
节点收到信标: 计算偏差, 调整时钟, 对齐时隙边界
```

### 3.3 ACK时间戳同步(TSCH方式)

在TSCH中每次通信都携带时间修正: 发送方在固定时隙开头发送, 接收方记录实际收到时刻vs预期时刻, 差值即时钟偏差, 在ACK中携带修正信息。优势是不需要专门信标, 通信本身就在做同步。

## 4. 时隙分配策略

### 4.1 集中式分配

网络协调器统一管理: 新节点向协调器请求时隙, 协调器根据可用时隙和需求分配, 通知所有相关节点。优点全局最优无冲突, 缺点单点故障和灵活性低。

### 4.2 静态分配

部署时预配置, 运行期间不变。节点编号i使用时隙(i-1), 简单映射无需运行时协商。适合节点固定、流量恒定的工业环境。

### 4.3 动态分配

根据流量需求动态调整: 高频传感器分配2个时隙每帧, 低频传感器每2帧分配1个时隙, 突发流量临时增加。6TiSCH的SF(Scheduling Function)实现: SF0最小调度, MSF按需增减, 自定义SF应用层优化。

## 5. IEEE 802.15.4 TSCH

### 5.1 TSCH概述

TSCH(Time-Slotted Channel Hopping)是IEEE 802.15.4-2015标准的一部分, 将TDMA与频率跳变结合:

```
传统TDMA: 每个时隙固定频率
- 如果该频率有干扰: 每次通信都失败, 可靠性差

TSCH: 每个时隙使用不同频率
- 即使某频率有干扰, 下次通信换频率
- 干扰被打散到多个频率上
- 显著提高可靠性
```

### 5.2 Slotframe结构

```
TSCH时隙帧(Slotframe): 二维调度(时间 x 频率)

       信道0  信道1  信道2  信道3
时隙0: [A->B]  ---    ---    ---
时隙1:  ---   [C->D]  ---    ---
时隙2:  ---    ---   [E->F]  ---
时隙3: [B->G]  ---    ---    ---
...

Slotframe长度S: 如101(质数, 提高跳频多样性)
时隙时长: 10ms(WirelessHART) 或 15ms(6TiSCH默认)
信道数: 16个(2.4GHz ISM频段)
```

### 5.3 信道跳变公式

```
channel = (ASN + channelOffset) mod numChannels

ASN: Absolute Slot Number(绝对时隙号, 从网络启动递增)
channelOffset: 调度表中指定的信道偏移
numChannels: 可用信道数(如16)

示例:
- ASN = 1000, channelOffset = 3, numChannels = 16
- channel = (1000 + 3) mod 16 = 1003 mod 16 = 11
- 使用第11号信道

每个时隙ASN递增, 同一链路每次用不同信道
跳频序列伪随机, 干扰影响被打散
```

### 5.4 TSCH时隙内部时序

```
一个10ms TSCH TX时隙:

|<------------------ 10ms ------------------>|
|TxOffset|数据包发送| |等待ACK |ACK接收| |余量|
|  2ms   |  约4ms  |G| 1ms  | 约1ms |G|    |

G = Guard time(约0.1ms)
TxOffset: 时隙开始后等待一段时间再发(让接收方开启无线电)

接收方在TxOffset前开启无线电准备接收
```

## 6. WirelessHART

### 6.1 WirelessHART概述

WirelessHART是第一个工业无线标准(2007年IEC 62591): 基于IEEE 802.15.4物理层(2.4GHz, 250kbps), TDMA加信道跳频(TSCH的先驱), 面向过程工业(化工、炼油、制药), 要求大于99.9%可靠性和小于1s确定性延迟。

### 6.2 网络架构

```
     [网络管理器] -- 集中式调度
          |
     [网关/接入点] -- 连接有线网络
      /    |    \
    [R1]  [R2]  [R3]    -- 路由设备(中继)
    / \    |     / \
  [S1][S2][S3] [S4][S5]  -- 现场设备(传感器/执行器)
```

### 6.3 超帧与可靠性

时隙长度10ms固定, 超帧长度由应用决定(如100时隙=1秒), 15个跳频信道(排除WiFi重叠频段), 支持多个超帧叠加实现不同速率。

WirelessHART实现大于99.9%可靠性的手段: 信道跳频(15信道避免持续干扰), 路径冗余(每个节点有备选路径), 时隙重试(同一数据可在不同时隙/信道重传), 图路由(Graph Routing, 多条路径同时活跃)。

## 7. TDMA vs CSMA/CA对比

| 维度 | TDMA | CSMA/CA |
|------|------|---------|
| 碰撞 | 无(设计保证) | 有(概率降低) |
| 延迟确定性 | 确定(帧长度为上界) | 不确定(负载相关) |
| 时间同步 | 必须(核心依赖) | 不需要 |
| 设备加入 | 复杂(需分配时隙) | 简单(立即竞争) |
| 低负载效率 | 低(时隙空闲浪费) | 高(按需使用) |
| 高负载效率 | 高(无碰撞开销) | 低(碰撞增多) |
| 实现复杂度 | 高 | 中 |
| 适用场景 | 工业控制/确定性 | 监测/尽力而为 |

必须选TDMA: 延迟SLA要求确定性, 大于99.9%可靠性, 高密度频繁通信, 工业合规认证。可以选CSMA: 偶尔丢包可接受, 节点动态增减, 低复杂度优先, 低频通信。

## 8. 混合MAC方案

### 8.1 IEEE 802.15.4 GTS

超帧同时包含竞争和非竞争区域:

```
|<=============== 超帧 ===============>|
|信标|<-- CAP(竞争) -->|<-- CFP(非竞争) -->|非活跃|
|    | CSMA/CA接入     | GTS1 | GTS2 | GTS3 |      |

CAP: CSMA/CA方式, 用于命令和管理消息
CFP: GTS(最多7个), TDMA方式给特定设备
兼得灵活性和确定性
```

### 8.2 6TiSCH架构

IETF 6TiSCH将IPv6引入TSCH网络: CoAP/UDP/IPv6/RPL/6LoWPAN/TSCH/802.15.4。定义调度函数接口, 使TSCH网络能接入互联网, 支持分布式调度(节点间协商时隙)。

### 8.3 工业现场混合部署

```
关键控制回路: TDMA
- 阀门控制命令: 保证小于100ms延迟
- 分配固定时隙, 即使无数据也保留

常规监测: CSMA(共享时隙)
- 温度/压力定期上报, 允许偶尔延迟

告警: 预留优先时隙
- 紧急告警使用预留时隙, 不与常规竞争
```

## 9. 实际案例: 工厂自动化

### 9.1 场景描述

化工厂反应釜监控: 50个传感器(温度、压力、流量、液位), 20个执行器(阀门、泵、加热器)。控制要求: 阀门命令小于100ms延迟, 99.99%可靠。监测要求: 传感器数据每秒一次, 99.9%可靠。环境: 金属结构, 电机干扰, WiFi共存。

### 9.2 TSCH方案设计

```
网络参数:
- 时隙长度: 10ms
- Slotframe: 10个时隙 = 100ms周期
- 信道: 16个(2.4GHz)
- 拓扑: 网状, 最大3跳到网关

时隙分配:
时隙0: 信标/同步(广播)
时隙1-3: 执行器命令(TDMA, 关键)
时隙4-7: 传感器数据(TDMA, 常规)
时隙8-9: 共享时隙(CSMA, 非关键/重传)
```

### 9.3 性能分析

```
延迟: 执行器命令1跳最大100ms, 3跳最大300ms
可靠性计算:
- 单链路含干扰成功率: 90%
- 信道跳频后: 1-(0.1 x 1/16) = 99.4%
- 加重传(2次机会): 1-(0.006)^2 = 99.996%
- 满足99.99%要求

能耗: 传感器占空比约3%, AA电池1-2年
碰撞: TDMA时隙零碰撞

对比纯CSMA/CA同场景:
- 70节点共享, 40%负载
- 碰撞率: 15-25%
- 执行器延迟: 最好5ms, 最坏大于200ms
- 无法保证小于100ms! 不满足工业要求
```

## 10. TDMA挑战与应对

**可扩展性**: 节点增加需更多时隙, 帧更长延迟增大。应对: 空间复用(远距节点复用时隙), 多帧叠加, 信道复用扩展容量。

**动态流量**: 固定调度无法适应突发。应对: 预留共享时隙, 6TiSCH动态调度, 优先级抢占。

**新节点加入**: 需扫描信标、同步、认证、请求时隙, 总加入时间几秒到几十秒。

**鲁棒性**: 协调器故障影响全网。应对: 备份协调器, 分布式调度方案。

## 总结

TDMA调度式MAC通过严格的时间划分实现确定性通信, 是工业IoT的核心使能技术:

- 核心优势: 零碰撞、确定性延迟、可预测性能, 满足工业控制严格要求
- 核心代价: 时间同步依赖、调度管理复杂、灵活性不如竞争式
- TSCH创新: TDMA加信道跳频, 兼得确定性和抗干扰能力
- WirelessHART验证: 在恶劣工业环境中实现大于99.9%可靠性
- 混合趋势: TDMA处理关键流量加CSMA处理尽力而为流量

正是TDMA带来的确定性保证, 让无线网络能够承担以前只有有线才能完成的控制任务。
