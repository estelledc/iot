# 室内外天线传播特性与IoT部署考量
> **难度**：🟡 中级 | **领域**：天线与传播 | **阅读时间**：约 20 分钟

## 引言

天线是 IoT 系统中连接数字世界与物理无线信道的桥梁——将电信号转换为电磁波（发送）或将电磁波转回电信号（接收）。对功率受限的 IoT 设备而言，天线往往是决定可达覆盖范围的**最关键因素**。良好的天线设计弥补低发射功率的不足，糟糕的选择则让再好的射频芯片也发挥不出性能。

本文从 IoT 天线类型出发，分析关键参数，深入讨论室内外不同传播环境的特性差异，并给出实际部署中的工程决策指导。

---

## 1. 天线在 IoT 系统中的角色

天线是换能器（Transducer）：发送时将传输线上的高频电流转换为向空间辐射的电磁波，接收时将空间电磁波感应为传输线上的电流。互易性（Reciprocity）原则使同一天线的收发特性对称。

IoT 设备受到多重约束使天线选择格外重要：功率限制（电池供电典型 0-14 dBm）、尺寸限制（设备小型化）、成本限制（大规模部署要求天线成本极低）、环境多样（从开阔农田到密集工厂）。在这些约束下，天线设计的每一个 dB 增益或损耗都直接影响系统可行性。

---

## 2. IoT 常用天线类型

### 2.1 PCB 走线天线（PCB Trace Antenna）

在 PCB 上用铜箔走线形成天线结构。常见形式包括倒 F 天线（IFA，最流行的 2.4GHz IoT 天线）、PIFA（平面倒 F）、曲折线天线（通过折叠缩小天线长度）。

优势：BOM 成本为零（天线是 PCB 的一部分）、不增加装配工序、与电路集成无需连接器。挑战：性能受地平面尺寸影响显著、周围元器件改变天线特性、外壳材料和形状影响调谐、设计周期长需仿真+调试。

频段尺寸参考：2.4GHz 约 15mm，868MHz 约 40mm，433MHz 约 80mm（可能需折叠适配 PCB）。

### 2.2 芯片天线（Chip Antenna）

陶瓷基板上制作的微型化天线组件，极小尺寸（2-5mm），标准 SMD 封装可自动贴装。需配合特定尺寸地平面。性能略低于 PCB 天线（微型化牺牲效率）。适合空间极度受限的可穿戴设备或微型传感器。

### 2.3 线天线（Wire Antenna）

简单金属导线，长度为工作波长的 1/4（四分之一波长天线）。结构最简单、性能优异（地平面良好时接近理论增益）、成本极低。但体积较大（868MHz 四分之一波长 ≈ 86mm）。适合有空间容纳天线的室外环境传感器。

### 2.4 外置天线（External Antenna）

独立于设备主体的专用天线，通过 SMA/U.FL 等连接器接入。类型包括胶棒天线（路由器常见）、玻璃钢天线（室外高增益全向）、八木/平板天线（定向高增益）。

性能最优（不受设备内部约束），可更换升级，可选择方向性。但增加成本（天线+连接器+馈线损耗）、体积和 IP 防护挑战。

## 3. 关键天线参数

**增益（Gain）**：天线在某方向聚焦能量的能力，单位 dBi。PCB 天线 2-3 dBi、外置全向 5-6 dBi、定向平板 8-12 dBi、高增益八木 15+ dBi。关键：每增加 3 dB ≈ 通信距离增加 40%（自由空间）。

**辐射方向图（Radiation Pattern）**：全向天线水平面各方向均匀辐射，适合方向不确定的传感器节点；定向天线能量集中于特定方向，增益更高但覆盖范围窄，适合点对点链路。

**极化（Polarization）**：电场振动方向。收发极化不匹配导致 3-20 dB 附加损耗。垂直对水平理论无穷大损耗（实际 20-30 dB）。IoT 实践：确保终端与网关天线极化一致。

**阻抗匹配**：应与 50Ω 传输线一致。不匹配导致能量反射、辐射效率下降。要求：回波损耗 < -10 dB（VSWR < 2:1）。

**带宽**：阻抗和增益满足要求的频率范围。IoT 通常需覆盖整个 ISM 频段。

**效率（Efficiency）**：输入功率中实际辐射的比例。PCB 天线典型 50-80%（-3 到 -1 dB），芯片天线 30-60%（-5 到 -2 dB）。损耗来源：介质损耗、导体损耗、地平面不足。

## 4. 室外传播特性

### 4.1 自由空间路径损耗（FSPL）

无障碍理想条件下信号随距离的衰减：

```
FSPL(dB) = 20·log₁₀(d) + 20·log₁₀(f) + 32.45
```
（d 为 km，f 为 MHz）

示例：868MHz 1km = 91.2 dB；2.4GHz 100m = 80.0 dB。频率翻倍 FSPL 增 6 dB——这解释了为什么 Sub-GHz 比 2.4GHz 有更远覆盖距离。

### 4.2 地面反射与双径模型

实际室外有直射+地面反射两条路径，叠加可能增强或抵消。在较远距离（超过 breakpoint distance）衰减加速为距离四次方，比 FSPL 更快。

### 4.3 障碍物绕射

信号被建筑、山丘、树木阻挡时，低频绕射能力更强（868MHz > 2.4GHz）。菲涅尔区（Fresnel Zone）——信号传播需要的"通道"空间——被阻挡 >40% 时产生显著附加损耗。868MHz 在 1km 距离第一菲涅尔区半径 ≈ 9.3m。

### 4.4 植被损耗

树木和植被对信号的额外吸收：2.4GHz 穿越 10m 茂密树林额外 10-20 dB，868MHz 为 5-10 dB。季节性变化：夏季（有叶）比冬季高 3-6 dB。

### 4.5 天气影响

对 Sub-GHz 和 2.4GHz 频段：雨衰 <1 dB/km，雾/雪影响可忽略。结论：对 IoT 常用频段，天气不是显著因素。

## 5. 室内传播特性

### 5.1 与室外的根本差异

室内比室外复杂得多：密集障碍物（墙壁、楼板、家具、人体）、多径效应显著（反射、散射、衍射）、衰减快速变化（人员走动、门开关）、大部分场景 NLOS（非视距）。

### 5.2 建筑材料穿透损耗

| 材料 | 868MHz 损耗 | 2.4GHz 损耗 |
|------|------------|-------------|
| 石膏板墙 | 2-4 dB | 3-5 dB |
| 砖墙 | 5-10 dB | 8-15 dB |
| 混凝土墙 | 10-15 dB | 15-25 dB |
| 钢筋混凝土 | 15-25 dB | 20-35 dB |
| Low-E 玻璃（节能） | 20-30 dB | 25-40 dB |
| 金属门/电梯 | 30-40 dB | 35-50+ dB |

**关键注意**：现代节能建筑大量使用 Low-E 玻璃和金属隔热层，对无线穿透极为不利——这是室内 IoT 部署中最常见的意外障碍。

### 5.3 楼层间传播

普通混凝土楼板每层 15-20 dB，钢筋混凝土每层 20-30 dB。穿越 2-3 层后信号基本不可用。实践含义：通常每层需要独立的网关/中继器。

### 5.4 多径效应

**快衰落**：多条反射路径相位叠加产生驻波，相隔半波长（2.4GHz 约 6cm）信号强度可变化 30-40 dB。固定传感器可能恰好处于深衰落点。

**时延扩展**：不同路径长度导致到达时间不同，室内典型 30-300 ns。对窄带 IoT 信号（LoRa、Zigbee）影响有限。

### 5.5 人体影响

人体含 ~60% 水，对微波有强吸收。单人遮挡 3-8 dB 附加损耗，人群环境 10-20 dB 变化。可穿戴设备天线被人体直接去调谐（改变阻抗和方向图）。

## 6. 频段选择指导

| 考量因素 | Sub-GHz (868/915MHz) | 2.4GHz |
|---------|---------------------|--------|
| 覆盖距离 | 远（低 FSPL + 好绕射） | 近 |
| 穿透能力 | 较好 | 较差 |
| 天线尺寸 | 大（~86mm 四分之一波） | 小（~31mm） |
| 可用带宽 | 窄（kHz 级信道） | 宽（MHz 级信道） |
| 数据速率 | 低 | 高 |
| 频段拥挤 | 相对空闲 | 非常拥挤 |

选型建议：覆盖优先（室外传感器网络）→ Sub-GHz；数据率优先（视频/音频）→ 2.4GHz/5GHz；室内穿墙（智能家居跨房间）→ Sub-GHz 优势明显；极小型化（可穿戴）→ 2.4GHz。

## 7. 部署工程实践

### 7.1 网关天线选择与安装

**室内网关**：外置全向天线（3-5 dBi 胶棒），安装尽量高（天花板下方或高墙面），避免紧贴金属表面（反射影响方向图），避免被大型物体遮挡。

**室外网关**：玻璃钢全向天线（6-8 dBi），安装在建筑物顶部或高杆上，确保第一菲涅尔区无遮挡，必须加装避雷器做雷电防护。

### 7.2 终端设备天线注意事项

PCB 天线设计原则：保留完整地平面（天线性能依赖地平面）、天线区域下不走线不铺铜、天线所在 PCB 边缘不被外壳金属遮挡、产品装壳后必须重新测试（外壳影响极大）。

安装方向：传感器天线与网关天线极化一致。垂直安装设备天线朝上（与网关平行），水平安装注意方向图是否覆盖网关方向。

### 7.3 链路预算计算

```
接收功率 = 发射功率 + TX天线增益 - 路径损耗 + RX天线增益 - 附加裕度
```

**示例：LoRaWAN 室外 5km**
- 发射功率 14 dBm + 终端天线 2 dBi - 路径损耗 112 dB + 网关天线 6 dBi - 裕度 10 dB = -100 dBm
- LoRa SF12 灵敏度 -137 dBm → 裕度 37 dB（充足）

### 7.4 现场测试方法

部署前的实地测试步骤：
1. 在计划的网关位置安装临时天线和网关设备
2. 携带终端设备（或专用测试仪）在覆盖区域内系统性行走
3. 记录各位置的 RSSI（接收信号强度）和 SNR（信噪比）
4. 绘制覆盖热力图，标注信号强度等级
5. 识别覆盖盲区和信号边缘区域
6. 在盲区尝试调整网关位置或规划中继器位置
7. 验证调整后的覆盖改善效果

测试应在实际工作条件下进行（人员活动、设备运行），而非空楼。不同时段的测试结果可能差异显著。

## 8. 特殊环境考量

**地下/地埋**：土壤穿透损耗 10-30 dB/m（含水量越高越大），Sub-GHz 优于 2.4GHz。实践：天线伸出地面或使用地上中继。

**金属环境（工业）**：金属反射造成复杂多径，遮挡造成阴影衰落，天线贴近金属面性能急剧下降。解决：贴片天线（有地平面隔离）、增加安装间距。

**高温/恶劣环境**：>85°C PCB 变形影响天线谐振频率；潮湿/腐蚀环境需要连接器密封和防腐蚀涂层；强振动环境需天线牢固固定防止松动。方案：使用高 Tg（玻璃化转变温度）PCB 材料、IP67+ 密封设计、抗振连接器（如 MMCX 替代 U.FL）。

**水下/液体环境**：水对无线信号衰减极大（2.4GHz 在水中衰减约 20 dB/cm）。Sub-GHz 稍好但仍严重。水下 IoT 通常使用声波或有线方案。水面附近的浮标传感器应将天线伸出水面。

## 9. 天线分集与 MIMO

**天线分集**：利用多根天线对抗衰落。主要形式包括：

- **空间分集**：两根天线物理分隔（至少半波长），两个位置同时处于深衰落的概率极低
- **极化分集**：一根垂直极化 + 一根水平极化，不受传感器安装方向影响
- **选择合并（Selection Combining）**：选择两根天线中信号更强的那个
- **最大比合并（MRC）**：加权合并两路信号，性能最优但复杂度高

分集通常在**网关端**实现（有空间和供电支持），终端设备因尺寸和成本限制一般不采用。LoRaWAN 网关常见双天线设计就是空间分集的实际应用。

**MIMO 在 IoT 中**：WiFi（802.11n/ac/ax）已标配 2×2 或 4×4 MIMO 提升吞吐量和可靠性。但 NB-IoT 不使用（180kHz 带宽太窄无法受益）、LoRaWAN 不使用（协议设计不支持多流）。MIMO 目前主要在 WiFi IoT 网关和高端终端中有实际意义。

## 10. 链路预算实例：智慧农业

### 场景

5km×5km 农田，100 个土壤湿度/温度传感器，LoRaWAN 868MHz，网关在农场建筑屋顶（高 8m）。

### 最远传感器链路计算（3.5km）

- FSPL（3.5km, 868MHz）= 102.1 dB
- 附加损耗（地面反射+植被穿越）：15 dB
- 衰落裕度：10 dB
- 接收功率 = 14 + 1.5 - 127.1 + 6 = -105.6 dBm
- LoRa SF10 灵敏度 -129 dBm → 裕度 23.4 dB（充足）

### 优化决策

23.4 dB 裕度充足 → 可降 SF 到 SF7（灵敏度 -123 dBm，裕度 17.4 dB），空口时间从 370ms 降到 36ms，电池寿命延长 10 倍。或降发射功率到 8 dBm 节省能量（裕度仍有 17.4 dB）。网关天线升级 8 dBi 额外 2 dB 裕度应对极端天气。

## 11. 常见部署错误

**天线被遮挡**：传感器装在金属壳内辐射被阻。→ 塑料外壳或天线区域开窗/外置天线。

**地平面不足**：缩小 PCB 裁剪地平面导致性能急剧下降。→ 最小 30mm×30mm（2.4GHz），Sub-GHz 更大。

**忽略安装环境**：实验室调好到现场性能下降。→ 最终产品形态下调谐，部署前现场测试。

**极化不匹配**：终端水平网关垂直造成 15-20 dB 损耗。→ 统一极化方向，或网关用圆极化天线（牺牲 3 dB 覆盖所有极化）。

**馈线损耗**：长馈线吃掉天线增益。→ 馈线尽量短；RG174 每米 0.5 dB（868MHz），LMR-400 仅 0.07 dB/m。馈线总损耗不应超过天线增益的一半。

## 12. 实践案例：智慧楼宇多楼层覆盖

### 场景

10 层办公楼，每层约 1000m²。Zigbee（2.4GHz）照明控制 + LoRaWAN（868MHz）环境监测。

### 覆盖设计

**Zigbee（照明控制）**：每层 1 个协调器 + 2-3 路由器，Mesh 组网确保单节点失效不影响整层，信道 25（避 WiFi）。单层独立，不依赖跨楼层通信。

**LoRaWAN（环境监测）**：全楼仅 1 个网关安装在 5F 中间层。868MHz 穿楼板能力强，可穿 2-3 层覆盖上下各几层。

### 链路验证（10F 边角最差链路）

楼层穿透 + 水平距离 + NLOS 附加，总路径损耗约 85 dB。接收功率 = 14 + 2 - 85 + 3 = -66 dBm，LoRa SF7 灵敏度 -123 dBm，裕度 57 dB（极其充足）。1 个网关放中间层即可覆盖全楼——体现 Sub-GHz 室内穿透的巨大优势。

### 经验总结

Sub-GHz（LoRaWAN）适合跨楼层低频次数据采集，2.4GHz（Zigbee）适合单层内高频次控制指令。两种技术互补：一个覆盖广速率低，一个速率高覆盖近。每层独立 Zigbee 网络确保照明控制实时性不受跨楼层链路影响。

### 部署后的长期考量

实际运行中还需注意：
- **楼层装修变化**：增加金属隔断、更换玻璃类型都可能影响信号传播
- **设备老化**：天线连接器松动、PCB 氧化会导致性能逐渐下降
- **新增干扰源**：租户引入新的无线系统可能影响现有部署
- **季节性变化**：室内温湿度变化对部分天线材料有微小影响
- **固件升级**：射频参数调整需在升级后重新验证覆盖

建议每半年进行一次覆盖复测，对比初始基线，及时发现并解决性能退化问题。

---

## 总结

天线与传播特性是 IoT 部署中常被低估却影响巨大的因素。核心要点：

1. **天线选型匹配场景**：PCB 适合小型低成本设备，外置适合网关和对距离有要求的场景
2. **频率决定特性**：Sub-GHz 覆盖远穿透好但速率低；2.4GHz 相反
3. **室内比室外复杂得多**：材料穿透、多径效应、人体影响都需额外裕度
4. **链路预算是部署决策基础**：部署前就能预判可行性
5. **现场测试不可替代**：再精确的计算也不能取代实际环境测量

掌握这些基础知识后，IoT 系统架构师可以在项目初期就做出正确的天线和频段选择，避免部署后才发现"信号不够"的尴尬局面。天线看似是硬件细节，实际上是影响整个系统架构决策的关键因素——它决定了需要多少个网关、能覆盖多大范围、电池能用多久，以及最终部署成本的量级。
