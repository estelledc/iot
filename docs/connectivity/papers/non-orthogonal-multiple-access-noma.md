# 非正交多址NOMA在大规模IoT中的应用
> **难度**: 高级 | **领域**: 多址接入 | **阅读时间**: 约 22 分钟

## 引言

想象一个教室里30个学生要同时向老师提问。传统方式是举手点名,一个一个来(正交接入: 同一时间只允许一人说话)。但如果允许多人同时说话呢?只要声音大小不同——前排轻声说、后排大声说——老师可以先听清最大声的,记住答案后"脑中消去",再分辨较小的声音。这就是NOMA的核心思想。

大规模IoT中,成千上万设备需要接入网络,但可用资源块有限。传统正交多址(OMA)每个资源块只服务一个设备,严重限制容量。NOMA允许多设备共享同一资源块,通过接收端的逐次干扰消除(SIC)分离叠加信号,成倍提升连接能力。

## 1. 多址接入演进与瓶颈

### 1.1 正交多址回顾

```
1G FDMA: 每用户独占一个频率通道
2G TDMA: 每用户独占一个时间槽
3G CDMA: 正交码区分用户, 互不干扰
4G OFDMA: 时频二维资源块分配
共同点: 每个资源块只能分配给一个设备
```

### 1.2 OMA的根本限制

```
NB-IoT一个载波 = 12个子载波
每子载波每ms = 1个资源块
=> 同一时刻最多12个设备

大规模IoT: 10000+设备 vs 数十个资源块 => 严重不足

NOMA突破: 1个资源块 --> K个设备 (K > 1)
"过载"(Overloading): 设备数可以超过资源块数
```

## 2. 功率域NOMA

### 2.1 基本原理

功率域NOMA让多设备在同一时频资源以不同功率发送:

```
基站接收到的叠加信号:
y = h_A*x_A + h_B*x_B + h_C*x_C + noise

设备A(近): 信号最强 -60dBm
设备B(中): 信号中等 -75dBm
设备C(远): 信号最弱 -85dBm
三个设备信号叠加在一起
```

### 2.2 下行功率域NOMA

基站向多设备发送叠加信号,远设备分更多功率保证公平。近设备先解码远设备信号(功率大),SIC消除后解码自己的信号。

### 2.3 上行功率域NOMA

上行更自然: 距离不同自动形成功率差,基站执行SIC解码。这对IoT最关键——设备端零复杂度增加,所有SIC处理在基站完成。设备只需正常发送数据,无需知道NOMA的存在。

## 3. 逐次干扰消除(SIC)

### 3.1 工作流程

```
接收到叠加信号: y = s1(强) + s2(中) + s3(弱) + noise

步骤1: 解码s1 (将s2+s3视为噪声, s1最强所以SINR够高)
步骤2: 重建s1信号并从y中减去
步骤3: 解码s2 (将s3视为噪声, s2现在最强)
步骤4: 重建s2并减去
步骤5: 解码s3 (仅剩噪声干扰)
```

### 3.2 SINR计算

```python
def sic_sinr(signal_powers, noise_power):
    """
    SIC各级解码的SINR
    signal_powers: 按强到弱排序的接收功率列表
    """
    sinr = []
    for k in range(len(signal_powers)):
        # 第k个用户: 信号 / (后续用户干扰 + 噪声)
        interference = sum(signal_powers[k+1:]) + noise_power
        sinr.append(signal_powers[k] / interference)
    return sinr

# 示例: [-60dBm, -75dBm, -85dBm], 噪声-100dBm
# 用户1 SINR: 约14.8dB, 用户2: 约9.5dB, 用户3: 15dB
```

### 3.3 误差传播问题

SIC的关键风险: 第1步解码错误导致重建不准,残余包含错误分量,后续所有解码受影响。

缓解措施: 保证先解码用户有足够高SINR(10dB以上)、CRC校验确认解码正确性、错误时回退到传统OMA解码、先进的并行/迭代SIC算法。

## 4. 码域NOMA

### 4.1 SCMA(稀疏码多址)

华为提出的SCMA是典型码域NOMA:

```
6用户共享4个资源(过载150%):
资源  用户1 用户2 用户3 用户4 用户5 用户6
R1:    x     x     0     x     0     0
R2:    x     0     x     0     x     0
R3:    0     x     x     0     0     x
R4:    0     0     0     x     x     x

每用户占2个资源(稀疏), 每资源被3个用户共享
接收端使用消息传递算法(MPA)解码
```

### 4.2 MUSA(多用户共享接入)

中兴提出的MUSA: 每用户分配复数扩频序列,序列之间非正交但低相关。设备自行选择序列发送,接收端SIC解码。适合免授权接入和大量设备场景。

### 4.3 功率域 vs 码域对比

| 特性 | 功率域NOMA | 码域NOMA |
|------|-----------|---------|
| 分离依据 | 功率差异 | 码字差异 |
| 过载率 | 2-3倍 | 1.5-6倍 |
| 接收端 | SIC | MPA/SIC |
| 功率控制 | 关键 | 不太关键 |
| IoT适用性 | 上行自然功率差 | 大量设备过载 |

## 5. NOMA对IoT的核心优势

### 5.1 过载能力

传统OMA: 100个资源块最多100个设备。NOMA(3倍过载): 100个资源块可服务300个设备。对智慧城市(每平方公里百万设备)至关重要。

### 5.2 免授权兼容

NOMA天然适合免授权接入:

```
传统(OMA+调度):
设备 --> [调度请求] --> 基站 --> [授权] --> 设备 --> [数据] --> 基站
10+ms延迟, 高信令开销

NOMA+免授权:
设备 --> [直接发送数据] --> 基站(内部SIC分离)
<1ms延迟, 零信令开销
```

### 5.3 频谱效率

在低速率多设备场景(IoT典型特征),NOMA总吞吐优于OMA。OMA每资源块1设备但速率OK;NOMA每资源块K设备,单设备速率略降,但总吞吐更高。

### 5.4 系统容量对比

```python
import numpy as np

def capacity_comparison(snr_db, num_users, bandwidth):
    """NOMA vs OMA容量对比"""
    snr = 10 ** (snr_db / 10)
    # OMA: 每用户分到 bandwidth/num_users
    cap_oma = num_users * (bandwidth/num_users) * np.log2(1+snr)
    # NOMA: 共享全部带宽, SIC完美
    cap_noma = bandwidth * np.log2(1 + num_users * snr)
    return cap_oma, cap_noma

# 10个IoT设备共享200kHz, SNR=10dB
# NOMA总容量约为OMA的1.4-2倍
```

## 6. 上行NOMA: IoT最相关场景

IoT流量以上行为主(传感器到云端),上行NOMA直接解决主要矛盾:

```
设备A(近): 发射20dBm, 到达基站时-60dBm
设备B(中): 发射20dBm, 到达基站时-75dBm
设备C(远): 发射23dBm, 到达基站时-85dBm

基站SIC:
1. 解码A(最强) -> 减去A -> 解码B -> 减去B -> 解码C
2. 一个资源块服务了3个设备
3. 设备端无任何变化, 只是正常发送
```

### 6.1 功率控制策略

最简单方案: 不做功率控制,直接利用距离差异产生的自然功率差做SIC。最适合超低成本IoT设备——设备固定功率发送即可。

高级方案: 按接收功率将设备分组,组内NOMA(功率差大利于SIC),组间OMA(正交资源避免组间干扰)。

## 7. NOMA与LPWAN结合

### 7.1 LoRaWAN

LoRa天然具有部分NOMA能力: 不同扩频因子(SF7-SF12)之间准正交,本质是码域NOMA。同SF内功率差大于6dB时,强信号可被解码(捕获效应),本质是功率域NOMA。

研究表明: 网关侧加SIC可提升容量2倍,设备端无变化。

### 7.2 NB-IoT

```
当前OMA: 12子载波, 每时隙最多12设备
+NOMA: 每子载波叠加2-3设备, 每时隙24-36设备
容量增益: 2-3倍
3GPP在Release 17+持续研究
```

## 8. 免授权NOMA

免授权加NOMA是大规模IoT的理想组合: 设备无需调度直接发送(免授权),碰撞由SIC在接收端解决(NOMA)。

活跃用户检测利用压缩感知: 10000注册设备中某时刻仅50个活跃,利用活跃模式的稀疏性高效检测并解码。每设备有唯一导频序列,基站检测哪些导频出现,联合完成活跃检测加数据解码。

## 9. NOMA面临的挑战

**SIC误差传播**: 先解码用户出错影响后续所有用户。需保证每级足够SINR余量,使用CRC校验,错误时回退OMA。

**功率控制**: 需一定功率差使SIC工作,但差太大弱用户性能差。IoT可利用自然功率差缓解,分组NOMA平衡性能。

**信道估计**: 多用户同时传输使估计更难,导频有限,低SNR设备估计不准。这是NOMA实用化的关键瓶颈。

**接收端复杂度**: K用户需K次解码加K-1次重建消除。复杂度全在基站侧,基站FPGA/DSP资源可支撑。对IoT设备零影响。

## 10. 3GPP标准化进展

```
Release 14-15: 研究启动, 评估15+种NOMA方案
Release 16 (2020): 结论有增益但复杂度高, 未纳入标准
Release 17+: 继续研究, 聚焦大规模IoT场景

候选方案:
SCMA (华为): 稀疏码, MPA检测
MUSA (中兴): 复数扩频序列
PDMA (大唐): 模式多样性
NOCA (诺基亚): 非正交覆盖码

最可能率先应用: mMTC免授权接入, 可能在6G早期标准
```

## 11. NOMA vs OMA适用场景

| 场景 | OMA更好 | NOMA更好 |
|------|---------|---------|
| 少设备高速率 | 简单可靠 | 增益不明显 |
| 多设备低速率 | 资源不足 | 过载能力强 |
| 延迟敏感 | 调度延迟高 | 免授权低延迟 |
| 设备端复杂度 | 需调度逻辑 | 直接发送即可 |

实际系统可能采用混合方案: 设备少时OMA(简单可靠),拥挤时切换NOMA(过载),高速率设备OMA独占,低速率IoT用NOMA共享。

## 12. 实际应用案例: 智慧城市停车

```
场景: 10000个地磁停车传感器
每传感器每天报告3次, 总消息30000条/天
正常: 约0.35条/秒
突发高峰(上班时段): 100+设备同时报告

NB-IoT + OMA方案:
每时隙12设备, 100设备需约9时隙调度
排队延迟: 可能数百ms, 高峰性能下降

NB-IoT + NOMA方案:
每子载波叠加3-4设备, 每时隙可服务48设备
100设备仅需约2时隙
延迟大幅降低, 高峰仍保持良好性能
频谱效率提升3倍以上

部署优势:
- 传感器端零变化, 仅基站软件/固件升级
- 渐进部署: 先在热点区域启用NOMA
- NOMA和OMA设备可共存
- 通过基站接收机升级即可三倍网络容量
```

### 12.1 SIC在基站端的实现细节

```
接收信号 = 设备A(强) + 设备B(中) + 设备C(弱) + 噪声

步骤1: 解码设备A(最强信号, 将B和C视为噪声)
步骤2: 重建A信号并从接收信号中减去, 残余 = B + C + 噪声
步骤3: 解码设备B(现在最强), 将C视为噪声
步骤4: 减去B, 残余 = C + 噪声
步骤5: 解码设备C, 无其他干扰

关键: 传感器端零变化, 所有复杂度在基站
通过基站接收机升级即可实现网络容量三倍提升
```

### 12.2 部署考量

NOMA的渐进部署策略: 先在设备密集的热点区域(如停车场上班高峰)启用NOMA,基站的SIC接收机处理叠加信号; 非热点区域仍用OMA,保持简单。两种模式可在同一基站上动态切换,根据实时负载决定是否启用NOMA。传感器完全无感知——它只知道"发送成功了",不知道是OMA还是NOMA在服务它。

## 总结

NOMA通过允许多设备在相同时频资源叠加传输,突破OMA的设备数量限制。核心优势: 过载能力使系统服务超过资源块数量的设备、免授权传输消除调度开销、SIC处理集中在基站侧使IoT设备保持简单。

最吸引人的特点是对终端透明——传感器正常发送,多用户分离全由基站完成。可通过升级基站接收算法提升已部署网络容量,无需替换终端。虽未在5G NR正式标准化,但SCMA/MUSA等方案持续研究,未来很可能成为大规模IoT连接的标准方案。
