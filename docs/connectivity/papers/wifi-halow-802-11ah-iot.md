# WiFi HaLow 802.11ah Sub-1GHz IoT长距离通信
> **难度**：🟡 中级 | **领域**：WiFi IoT专用 | **阅读时间**：约 20 分钟

## 引言

想象你在一个很大的农场里，需要监测散布在各处的土壤湿度。你有两种对讲机可以选：一种是普通的步话机(传统 WiFi)，声音清晰、说话速度快，但喊不了多远，隔一座小山就听不到了；另一种是低频大功率对讲机(WiFi HaLow)，声音虽然不够清晰、说话速度慢一些，但低沉的声波能翻过山头传到几公里外。

WiFi HaLow 就是后面这种"低频大功率对讲机"的无线网络版本。它把传统 WiFi 的协议栈从 2.4GHz/5GHz 搬到了 Sub-1GHz 频段(850-950MHz 附近)，利用低频信号天然的长距离传播和强穿透能力，专门为物联网场景设计。最关键的是，它保留了 WiFi 的全部优势——IP 原生通信、标准的安全机制、成熟的协议栈——你不需要额外的网关来翻译协议。

本文将深入解析 WiFi HaLow 的技术特点、与其他 LPWAN 技术的对比，以及它在实际 IoT 部署中的应用场景。

## 1. WiFi HaLow 技术概述

### 1.1 标准化历程

WiFi HaLow 基于 IEEE 802.11ah 标准，是 WiFi 家族中第一个专门为物联网设计的成员：

| 时间 | 事件 |
|------|------|
| 2010年 | IEEE 802.11ah 工作组成立 |
| 2013年 | Draft 1.0 发布 |
| 2016年 | IEEE 802.11ah 标准正式发布 |
| 2021年 | WiFi 联盟启动 HaLow 认证项目 |
| 2023年 | 首批认证产品上市(Morse Micro等) |
| 2024年 | HaLow R2 规范发布 |

"HaLow"的发音是"halo"(光环)，寓意是为 IoT 设备提供一层连接的"光环"。

### 1.2 核心设计目标

WiFi HaLow 的设计围绕四个核心目标展开：长距离覆盖(1km+)、低功耗(年级电池寿命)、大容量(单 AP 支持数千设备)、IP 原生(标准 TCP/IP 协议栈)。

这四个目标在传统 WiFi 中是无法同时实现的。WiFi HaLow 通过切换到 Sub-1GHz 频段并重新设计物理层参数来达成这一平衡。

### 1.3 关键技术参数

| 参数 | WiFi HaLow (802.11ah) | 传统WiFi (802.11n) |
|------|----------------------|-------------------|
| 频段 | Sub-1GHz (850-950MHz) | 2.4GHz / 5GHz |
| 信道带宽 | 1/2/4/8/16MHz | 20/40MHz |
| 最大吞吐 | 约40Mbps (16MHz) | 约300Mbps (40MHz) |
| 最低吞吐 | 约150kbps (1MHz) | 约6.5Mbps (20MHz) |
| 覆盖距离 | 1-3km(室外) | 50-100m(室外) |
| 最大关联数 | 8191 | 约200(实际) |
| 节能模式 | TWT + RAW | PSM |
| 安全 | WPA3 | WPA2/WPA3 |

## 2. Sub-1GHz 传播特性

### 2.1 为什么低频传得远

无线信号的传播距离与频率密切相关。低频信号有两个天然优势：

第一是自由空间路径损耗更小。根据 FSPL 公式，频率每降低一半，路径损耗减少 6dB，等效于覆盖距离翻倍。从 2.4GHz 降到 900MHz，频率降低约 2.7 倍，理论覆盖距离增加约 2.7 倍。

第二是绕射能力更强。900MHz 信号的波长约 33cm(2.4GHz 约 12.5cm)，更长的波长意味着信号可以更好地绕过障碍物(如建筑拐角、树木)，穿透墙壁和地面的衰减也更小。

```
频率与覆盖距离的关系:
  2.4GHz WiFi: 室内 30m, 室外 100m
  900MHz HaLow: 室内 100m+, 室外 1-3km

穿墙衰减对比:
  混凝土墙(15cm):
    2.4GHz: 约15dB衰减
    900MHz: 约8dB衰减
  砖墙(10cm):
    2.4GHz: 约10dB衰减
    900MHz: 约5dB衰减
```

### 2.2 与 LoRa 频段的关系

WiFi HaLow 使用的 Sub-1GHz 频段与 LoRa 非常接近(都在 ISM 频段的 900MHz 附近)，但两者的协议完全不同。LoRa 使用扩频调制(CSS)来换取极远的传输距离(10km+)，而 WiFi HaLow 使用 OFDM 调制来提供更高的吞吐量。可以理解为：同一段高速公路上，LoRa 是长途大巴(慢但能走很远)，WiFi HaLow 是城际快车(速度适中，距离也不错)。

## 3. 信道带宽与吞吐量

### 3.1 灵活的信道带宽

WiFi HaLow 支持 5 种信道带宽，从 1MHz 到 16MHz。这种灵活性是传统 WiFi 所不具备的：

| 信道带宽 | 子载波数 | 数据速率范围 | 典型覆盖 | 适用场景 |
|---------|---------|------------|---------|---------|
| 1MHz | 24+2 | 150kbps-4Mbps | 最远(1km+) | 远距离传感器 |
| 2MHz | 52+2 | 650kbps-7.8Mbps | 远 | 通用IoT |
| 4MHz | 108+6 | 1.35-18Mbps | 中 | 高频数据采集 |
| 8MHz | 234+6 | 2.93-39Mbps | 近 | 视频/大数据 |
| 16MHz | 468+14 | 5.85-78Mbps | 最近 | 高吞吐应用 |

关键取舍：信道越窄，覆盖距离越远但吞吐量越低。对于大多数 IoT 传感器(每次发送几十到几百字节)，1MHz 信道提供的 150kbps 已经绰绰有余。

### 3.2 OFDM 参数对比

WiFi HaLow 的 OFDM 参数是传统 WiFi 的十分之一缩放(down-clocked)版本：

```
传统WiFi (20MHz):
  子载波间距: 312.5kHz
  符号时间: 3.2us + 0.8us GI = 4us
  子载波数: 52 (数据) + 4 (导频)

WiFi HaLow (2MHz):
  子载波间距: 31.25kHz
  符号时间: 32us + 8us GI = 40us
  子载波数: 52 (数据) + 2 (导频)
```

符号时间变为原来的 10 倍(40us vs 4us)，这意味着每个符号更"坚固"——在信号较弱的远距离传输中不容易出错。代价是吞吐量也降为十分之一，但这对 IoT 来说不是问题。

## 4. 节能机制

### 4.1 TWT(Target Wake Time)

WiFi HaLow 是 TWT 技术的发源地——802.11ah 标准最先定义了 TWT，后来被 WiFi 6(802.11ax)采纳。TWT 允许设备和 AP 协商一个精确的唤醒时间表：

```
土壤传感器 TWT 示例:
  唤醒间隔: 15分钟
  唤醒持续: 5ms

  00:00  [醒来] 发送湿度数据(50字节) --> [休眠]
  00:15  [醒来] 发送湿度数据(50字节) --> [休眠]
  00:30  [醒来] 发送湿度数据(50字节) --> [休眠]
  ...

  每天唤醒次数: 96次
  每次活跃时间: 5ms
  每天总活跃时间: 480ms = 不到0.5秒
  其余时间: 深度休眠(约10uA)
```

### 4.2 RAW(Restricted Access Window)

RAW 是 WiFi HaLow 独有的节能和接入管理机制。当一个 AP 关联了数千个设备时，如果所有设备同时醒来竞争信道，碰撞会非常严重。RAW 解决这个问题的方法是将时间划分为多个窗口，每个窗口只允许一组指定的设备访问信道：

```
RAW 时间划分:
|====== RAW 1 ======|====== RAW 2 ======|====== RAW 3 ======|
| 设备组A (传感器)   | 设备组B (电表)     | 设备组C (水表)    |
| 只有A组设备能发送   | 只有B组设备能发送   | 只有C组设备能发送  |

优势:
- 减少碰撞: 每个窗口内设备数量少
- 节省功耗: 不在自己窗口内的设备可以继续休眠
- 公平性: 每组设备都有保证的接入机会
```

### 4.3 电池寿命估算

```
假设: AA电池(2500mAh), 每15分钟上报一次

功耗分析:
  深度休眠电流: 10uA
  唤醒+传输电流: 120mA
  每次唤醒时间: 5ms

平均电流计算:
  休眠: 10uA x (900000-5)ms / 900000ms = 10uA
  传输: 120mA x 5ms / 900000ms = 0.67uA
  总平均: 约 10.67uA

电池寿命:
  2500mAh / 0.01067mA = 234000小时 = 约26年(理论)
  考虑电池自放电(每年约3-5%): 实际约 5-10年
```

## 5. 大容量接入：8191 个设备

### 5.1 AID 扩展

传统 WiFi 的 AID(Association ID)是 11 位的，最多支持 2007 个关联设备。WiFi HaLow 将 AID 扩展到 13 位，最大支持 8191 个设备关联到单个 AP。

### 5.2 分层 AID 结构

WiFi HaLow 的 AID 采用分层结构，方便 AP 对设备进行分组管理：

```
AID 结构 (13位):
+-------+-------+---------+
| Page  | Block | Sub-block|
| (2位) | (5位) | (6位)   |
+-------+-------+---------+

4 个 Page x 32 个 Block x 64 个 Sub-block = 8192 个设备

分组管理示例:
  Page 0: 环境传感器 (温度/湿度/光照)
  Page 1: 能源管理 (电表/水表/气表)
  Page 2: 安防设备 (门锁/烟感/摄像头)
  Page 3: 预留

AP可以按Page或Block为单位进行RAW调度和TWT分组
```

### 5.3 短帧优化

为了减少大量小数据包的开销，WiFi HaLow 引入了 NDP(Null Data Packet)相关的优化。传感器可以使用短帧格式发送数据，减少 MAC 头部开销。对于只有几十字节的传感器数据来说，这种优化可以将空中传输时间减少 30-50%。

## 6. 与其他 LPWAN 技术的对比

### 6.1 WiFi HaLow vs LoRaWAN

| 维度 | WiFi HaLow | LoRaWAN |
|------|-----------|---------|
| 协议层 | IP原生(TCP/IP) | 非IP(需网关转换) |
| 吞吐量 | 150kbps-40Mbps | 0.3-50kbps |
| 覆盖距离 | 1-3km | 5-15km |
| 双向通信 | 完全双向 | 受限(Class A/B/C) |
| 网络拓扑 | 星型(AP-Station) | 星型(GW-Node) |
| 安全性 | WPA3 | AES-128 |
| 部署成本 | 私有网络，无月费 | 私有网络，无月费 |
| 芯片成本 | 约3-5美元 | 约2-3美元 |
| 生态成熟度 | 新兴 | 成熟 |

WiFi HaLow 的核心优势在于 IP 原生和更高的吞吐量。LoRaWAN 设备发送的数据需要经过网关协议转换才能进入 IP 网络，而 WiFi HaLow 设备可以直接与云端通信。但 LoRaWAN 在覆盖距离上仍有明显优势。

### 6.2 WiFi HaLow vs NB-IoT

| 维度 | WiFi HaLow | NB-IoT |
|------|-----------|--------|
| 频谱 | 免授权ISM | 授权蜂窝频段 |
| 运营费用 | 无月费 | 需要数据套餐 |
| 覆盖 | 自建网络1-3km | 运营商覆盖 |
| 部署灵活性 | 即装即用 | 依赖运营商 |
| 吞吐量 | 高 | 中(200kbps) |
| 延迟 | 低(毫秒级) | 较高(秒级) |
| 功耗 | 极低 | 低 |

WiFi HaLow 的最大优势是不需要运营商参与——你自己部署 AP 就能建成完整的网络。NB-IoT 需要依赖运营商的基础设施和数据套餐。

### 6.3 WiFi HaLow vs BLE

| 维度 | WiFi HaLow | BLE 5.0 |
|------|-----------|---------|
| 覆盖距离 | 1-3km | 30-100m |
| 吞吐量 | 高 | 中(2Mbps PHY) |
| 功耗 | 低 | 极低 |
| IP原生 | 是 | 否(需要网关) |
| 节点数/AP | 8191 | 约20(同时连接) |
| 适用场景 | 广域IoT | 近距离IoT |

BLE 在近距离、极低功耗场景中仍是最优选择(如可穿戴设备)。WiFi HaLow 的优势在于远距离和大容量场景。

## 7. 实际应用场景

### 7.1 智慧农业

这是 WiFi HaLow 最典型的应用场景之一：

```
农场部署方案:
  面积: 500m x 1000m = 50公顷
  传感器: 50个土壤湿度传感器, 分布在田间
  AP: 1个WiFi HaLow AP, 安装在农场中心制高点

网络参数:
  信道: 1MHz (最大覆盖)
  数据速率: 150kbps (绰绰有余)
  上报频率: 每15分钟一次
  数据量: 每次约100字节(湿度+温度+GPS+电池电压)

电池寿命:
  使用TWT, 每15分钟唤醒
  AA电池组: 5年以上

总成本:
  1个HaLow AP: 约200美元
  50个传感器节点: 约50美元/个 x 50 = 2500美元
  总计: 约2700美元
  无月费, 无运营商依赖
```

### 7.2 工业园区

在一个中型工业园区中部署环境监测网络：

```
部署场景:
  园区面积: 200m x 300m
  建筑: 5栋厂房
  传感器类型: 温湿度(200个), 振动(50个), 气体(30个)
  AP部署: 2个HaLow AP (主+备)

信道配置:
  AP1: 2MHz信道, 管理温湿度传感器(低频上报)
  AP2: 4MHz信道, 管理振动+气体传感器(较高频上报)
```

### 7.3 智慧城市

停车位检测、路灯控制、环境监测等城市应用：

```
智慧停车部署:
  区域: 市中心5个街区
  停车位传感器: 500个(地磁检测)
  AP: 3个HaLow AP (屋顶安装)
  每个AP覆盖约170个传感器

工作流程:
  车辆驶入 --> 地磁传感器检测变化
  --> 唤醒WiFi HaLow模块
  --> 发送状态更新(占用/空闲, 20字节)
  --> 云端更新停车位地图
  --> 休眠等待下次变化
```

## 8. 硬件生态

### 8.1 主要芯片厂商

WiFi HaLow 的芯片生态在快速发展中：

**Morse Micro MM6108** 是目前最成熟的 HaLow SoC。它集成了 802.11ah 基带和 RF 前端，支持 1/2/4/8MHz 信道带宽，内置 ARM Cortex-M3 处理器，深度睡眠电流低于 10uA。

**Newracom NRC7292** 是另一个主要的 HaLow 芯片方案，提供完整的 802.11ah MAC 和 PHY，支持 SPI/SDIO 主控接口，适合与外部 MCU 配合使用。

### 8.2 开发板资源

目前市面上有多款 HaLow 开发板可用于原型开发。Morse Micro 提供了基于 MM6108 的评估套件，包含 AP 和 Station 各一套。Newracom 也有基于 NRC7292 的开发板，兼容 Arduino 和 Raspberry Pi 生态。

## 9. 监管与频段差异

### 9.1 各地区频段分配

Sub-1GHz ISM 频段在不同地区有不同的分配：

| 地区 | 频段 | 可用带宽 | 功率限制 |
|------|------|---------|---------|
| 美国 | 902-928MHz | 26MHz | 1W EIRP |
| 欧洲 | 863-868MHz | 5MHz | 25mW ERP |
| 中国 | 755-787MHz | 32MHz | 根据型号核准 |
| 日本 | 916.5-927.5MHz | 11MHz | 20mW |
| 澳大利亚 | 915-928MHz | 13MHz | 1W EIRP |
| 韩国 | 917.5-923.5MHz | 6MHz | 10mW |

### 9.2 全球部署挑战

频段的地区差异是 WiFi HaLow 全球部署的最大挑战。一个为美国市场设计的 HaLow 设备(902-928MHz)不能直接在欧洲(863-868MHz)使用。产品设计需要考虑多频段支持或者针对不同市场做不同的硬件版本。

欧洲的 5MHz 可用带宽也显著限制了可用信道数量，使得在欧洲部署大规模 HaLow 网络时需要更精细的频率规划。

## 10. 局限性与发展趋势

### 10.1 当前局限

WiFi HaLow 目前面临的主要挑战包括：生态系统尚不成熟(芯片选择有限、AP 设备少)、吞吐量低于传统 WiFi(1MHz 信道仅 150kbps)、地区频段差异增加了产品开发复杂度、市场认知度不高(很多人不知道 HaLow 的存在)。

### 10.2 发展趋势

随着 IoT 市场的持续增长，WiFi HaLow 正在加速成熟。芯片成本在下降，更多厂商进入市场。WiFi 联盟的 HaLow R2 认证进一步提升了互操作性。与 Matter 协议的潜在集成也将为 HaLow 打开智能家居市场。

## 总结

WiFi HaLow 填补了物联网连接技术谱系中的一个重要空白。它比传统 WiFi 覆盖更远、更省电，比 LoRaWAN 吞吐更高、通信更灵活，比 NB-IoT 部署更自主、无需月费。它的 IP 原生特性意味着传感器可以直接与云端通信，无需协议转换网关，这大大简化了系统架构。

对于需要在几百米到几公里范围内部署大量传感器、且不想依赖运营商的场景(如农场、工业园区、大型建筑群)，WiFi HaLow 是一个值得认真考虑的选项。虽然其生态系统仍在成长中，但随着 Morse Micro 等厂商的芯片量产和 WiFi 联盟认证的推进，商用部署的条件正在快速成熟。
