# Z-Wave与Zigbee智能家居协议对比
> **难度**：🟢 初级 | **领域**：智能家居选型 | **阅读时间**：约 18 分钟

## 引言

你要装修新家的智能系统,面前摆着两套方案。方案A像是一个严格管理的快递公司:员工不多但个个通过考核,走的是人少的小路,包裹必达(Z-Wave)。方案B像是一个开放的众包平台:骑手数量庞大,走的是大马路,偶尔会堵车,但覆盖的区域和商品种类更多(Zigbee)。

这两家"快递公司"都能把"包裹"(控制命令)送到"收件人"(智能设备)手上,但他们的运作方式、覆盖范围和服务承诺截然不同。选哪家,取决于你家的具体情况。

本文将从频段、距离、组网、互操作性、安全性等多个维度对Z-Wave和Zigbee进行系统对比,帮助你做出适合自己的选择。

## 1. 基本概况对比

### 1.1 协议背景

Z-Wave由丹麦Zensys公司在2001年开发,目前由Silicon Labs维护,Z-Wave Alliance管理认证。它从一开始就专注于智能家居场景,所有设计决策都围绕家庭自动化的需求。

Zigbee由ZigBee Alliance(现改名为Connectivity Standards Alliance,简称CSA)基于IEEE 802.15.4标准开发,2004年发布1.0版本。Zigbee的设计目标更广泛,覆盖家居、工业、照明、能源管理等多个领域。这种广泛定位带来了更大的生态系统,但也导致了早期互操作性的问题。

### 1.2 核心参数一览

| 参数 | Z-Wave | Zigbee |
|------|--------|--------|
| 标准组织 | Z-Wave Alliance | CSA (原ZigBee Alliance) |
| 频段 | Sub-GHz (908/868MHz) | 2.4GHz (全球通用) |
| 数据速率 | 9.6/40/100 kbps | 250 kbps |
| 最大节点数 | 232 | 理论65000 |
| 网络拓扑 | Mesh (源路由) | Mesh (AODV路由) |
| 芯片供应商 | 仅Silicon Labs | 多家(SiLabs/NXP/TI/Nordic等) |
| 认证要求 | 强制互操作认证 | 认证可选(Zigbee 3.0后改善) |

从参数表就能看出两种协议的设计哲学差异:Z-Wave追求的是可控性和确定性(单一供应商、强制认证、小规模网络),Zigbee追求的是开放性和规模(多供应商、大网络、高吞吐)。

## 2. 频段差异与实际影响

### 2.1 频率选择的本质区别

Z-Wave使用Sub-GHz频段,具体频率因地区而异:美国908MHz、欧洲868MHz、日本922MHz、澳大利亚921MHz等。这意味着Z-Wave设备不能跨地区使用——美国版设备拿到欧洲无法工作,因为射频硬件不同。

Zigbee使用2.4GHz频段,全球统一,有16个可用信道(信道11-26)。一个Zigbee设备可以在全世界任何地方使用,这对跨国销售的厂商来说是明显的优势,不需要为每个地区设计不同的射频硬件。

### 2.2 穿墙能力对比

频率越低,波长越长,绕过障碍物的能力越强。这是Z-Wave选择Sub-GHz的核心原因。在室内环境中,这个差异非常明显:

| 障碍物材料 | Sub-GHz衰减(Z-Wave) | 2.4GHz衰减(Zigbee) |
|-----------|---------------------|---------------------|
| 木门/干墙 | 2-3 dB | 3-4 dB |
| 砖墙 | 5-8 dB | 6-10 dB |
| 混凝土墙 | 8-15 dB | 10-20 dB |
| 楼板(木) | 5-8 dB | 8-12 dB |
| 楼板(混凝土) | 10-18 dB | 15-25 dB |

在一栋三层楼的混凝土住宅中,Z-Wave信号从一楼到三楼穿过两层楼板会衰减20-36dB;而Zigbee信号穿过同样的楼板会衰减30-50dB。这个差距在实际使用中意味着Z-Wave可能一跳直接到达,而Zigbee需要在中间楼层部署中继设备。

对于住在老房子(厚砖墙/石墙)或多层独立住宅的用户,Z-Wave的穿墙优势尤其明显。而对于开间布局的公寓或新建的轻隔墙住宅,两者差异不大。

### 2.3 干扰环境

2.4GHz频段是最拥挤的无线频段:

```
2.4GHz频段的常见干扰源:
  - WiFi路由器 (最大干扰源,每个路由器占40MHz带宽)
  - 蓝牙设备 (手机/耳机/手表,数量众多)
  - 微波炉 (工作时产生2.4GHz宽带噪声)
  - USB 3.0设备 (已知产生2.4GHz杂散辐射)
  - 无线婴儿监视器/无线键盘鼠标

Sub-GHz频段的常见设备:
  - 少量无线报警系统
  - 车库门遥控器
  - 气象站无线传感器
  - 极少的消费电子设备
```

在公寓楼中,周围可能有几十个WiFi接入点和上百个蓝牙设备,Zigbee信道与WiFi信道存在重叠(Zigbee信道15/20/25/26与WiFi信道1/6/11重叠最少),选错信道可能导致明显的通信问题。

Sub-GHz频段则安静得多,Z-Wave设备很少遇到干扰,通信可靠性更高。

不过需要指出,Zigbee协议设计了信道切换机制来应对干扰。协调器可以检测当前信道的干扰水平,必要时引导整个网络切换到更清洁的信道。在大多数家庭环境中,只要初始信道选择得当,干扰影响并不严重。

## 3. 组网能力对比

### 3.1 路由机制差异

Z-Wave和Zigbee都支持mesh组网,但路由策略截然不同:

```
Z-Wave 源路由 (Source Routing):
  控制器维护完整路由表
  数据包携带完整路径: A -> B -> C -> D
  每个中继节点只需查看下一跳,不做路由决策
  优点: 中继节点实现简单,路由路径可预测
  缺点: 依赖控制器,最多4跳限制

Zigbee AODV路由 (Ad-hoc On-demand Distance Vector):
  每个路由节点维护自己的路由表
  发送前按需发现路由: 广播路由请求(RREQ)
  目标节点回复路由应答(RREP),建立路径
  优点: 分布式决策,自愈能力强,无跳数硬限制
  缺点: 路由发现有延迟,每个路由节点需内存存表
```

源路由就像GPS导航——你出发前就知道完整路线,每个路口只管照着导航走。AODV就像问路——不知道路时现场打听,虽然多花点时间,但即使某条路断了也能快速找到替代路线。

### 3.2 网络规模

Z-Wave单网络最多232个节点(8位Node ID),对普通家庭来说绑绑有余——很少有家庭需要超过100个智能设备。但对于小型商业场景(如酒店、办公楼)可能不够。Z-Wave Long Range将容量扩展到4232个节点(经典232+LR4000)。

Zigbee理论上支持65000个节点(16位地址空间),实际部署中几百到几千个节点是常见规模。大型商业照明系统可能用到数百个Zigbee节点,这是Z-Wave经典模式难以支撑的。

### 3.3 网络自愈能力

两种协议都支持网络自愈(某个中继节点失效时自动寻找替代路径),但实现方式和效果有差异:

Z-Wave的自愈依赖控制器:发送失败后尝试备用路由,如果仍失败则通知控制器重新探索拓扑。这个过程可能需要几秒到几十秒,有时需要用户在App中手动触发"网络修复(Network Heal)"。

Zigbee的自愈是分布式的:路由节点发现链路断开后,自动发起路由发现广播,找到替代路径并更新本地路由表。这个过程通常在几秒内自动完成,无需人工干预。

整体来看,Zigbee的分布式路由在网络动态变化时(如设备临时断电、移动位置)的恢复速度优于Z-Wave。

## 4. 互操作性

### 4.1 Z-Wave的强制认证

Z-Wave Alliance的强制互操作性认证是Z-Wave最独特的优势。每个Z-Wave产品上市前必须通过联盟的测试,测试内容包括:设备是否正确实现了所有声称支持的命令类、是否能与其他Z-Wave设备正常通信、安全机制是否正确实现。

通过测试的设备获得Z-Wave认证标志,消费者看到这个标志就可以放心购买——任何Z-Wave灯泡都能被任何Z-Wave控制器控制,不存在品牌锁定。认证过程需要几周到几个月,费用约数千美元。这提高了设备质量但也增加了小厂商的进入门槛。

### 4.2 Zigbee的互操作历史

Zigbee的互操作性走过了一段弯路。早期Zigbee定义了多个独立的应用Profile:Home Automation(HA)用于智能家居,Light Link(LL)用于照明,Smart Energy(SE)用于能源管理。不同Profile的设备之间往往无法互通,给消费者带来了困扰。

2016年发布的Zigbee 3.0是一个重要转折点,它统一了所有应用Profile为一个标准,大幅改善了互操作性。Zigbee 3.0还引入了Install Code机制提升入网安全性。但由于认证执行不如Z-Wave严格,实际中一些小品牌设备偶尔仍存在兼容性问题。

从消费者角度总结:Z-Wave设备之间的兼容性几乎不会出问题;Zigbee大品牌(Philips Hue、IKEA TRADFRI、Aqara)之间通常没问题,但小品牌偶尔会让人踩坑。

## 5. 芯片供应与成本

### 5.1 供应商格局

Z-Wave的芯片只有Silicon Labs一家供应,当前产品线包括700系列(EFR32ZG14)和800系列(EFR32ZG23)。单一来源意味着价格缺乏竞争,芯片成本相对较高。

Zigbee芯片有多家供应商:Silicon Labs的EFR32MG系列、NXP的JN5189、TI的CC2652系列、Nordic的nRF52840、Telink的TLSR8258等。多来源带来价格竞争和供应保障。

### 5.2 设备终端价格对比

芯片成本和认证成本的差异直接反映在设备价格上:

| 设备类型 | Z-Wave价格范围(美元) | Zigbee价格范围(美元) |
|---------|---------------------|---------------------|
| 门窗传感器 | 25-35 | 10-20 |
| 智能插座 | 30-40 | 12-25 |
| 运动传感器 | 30-45 | 15-25 |
| 温湿度传感器 | 25-40 | 10-20 |
| 智能灯泡 | 25-35 | 10-20 |

Z-Wave设备通常比同类Zigbee设备贵50%-100%。如果你计划部署30-50个设备,总成本差异可能达到几百美元。

### 5.3 供应链风险分析

单一供应商是Z-Wave的双刃剑。优势是Silicon Labs对协议的演进方向有完全控制力,版本兼容性好;风险是整个生态依赖一家公司的战略决策。如果Silicon Labs减少对Z-Wave的投入(它同时也生产Zigbee和Thread芯片),整个Z-Wave生态都会受到影响。

Zigbee的多供应商模式更稳健:某个芯片商退出市场,其他供应商可以填补;芯片商之间的竞争持续推动技术创新和成本下降。

## 6. 数据速率与功耗

### 6.1 速率对比与实际影响

Z-Wave当前主流速率是100kbps(700/800系列),Zigbee在2.4GHz频段提供250kbps。对于智能家居控制命令来说,两种速率都远远超过需求:

```
一个"开灯"命令的数据量和传输时间:
  命令大小: 约20字节
  Z-Wave 100kbps: 1.6ms
  Zigbee 250kbps: 0.6ms
  人的反应时间: 约200ms
  
  结论: 1ms的差异完全无法感知
```

速率差异在OTA(空中固件升级)时比较明显。一个100KB的固件文件,Z-Wave大约需要8秒传输时间,Zigbee大约需要3.2秒(不含协议开销和重传)。不过固件升级频率很低,这个差异对日常体验没有影响。

### 6.2 功耗特性对比

两种协议都支持电池设备的低功耗运行,核心功耗参数:

| 功耗指标 | Z-Wave (800系列) | Zigbee (典型) |
|---------|-----------------|--------------|
| 睡眠电流 | 约1.2uA | 约1.0uA |
| 发射电流 | 约30mA (+14dBm) | 约15mA (+8dBm) |
| 接收电流 | 约10mA | 约8mA |
| 每包能耗 | 较高 | 较低 |
| 典型电池寿命 | 1-3年(CR2032) | 1-3年(CR2032) |

虽然Zigbee每个数据包的能耗更低,但Z-Wave的长距离意味着同等覆盖面积下可能需要更少的中继节点(mesh跳数更少),总体系统功耗取决于具体的网络布局和设备密度。

## 7. 安全机制对比

### 7.1 Z-Wave S2安全

Z-Wave从2017年起强制要求S2安全框架。S2使用ECDH(椭圆曲线Diffie-Hellman)进行密钥交换,AES-128加密数据传输,DSK(设备特定密钥)验证设备身份。S2定义了三个安全等级:Access Control(门锁级,需输入完整DSK)、Authenticated(标准级,需输入部分DSK)和Unauthenticated(基础级,无需DSK但仍加密)。

由于S2对所有新认证设备强制执行,Z-Wave的安全基线是统一且可靠的。

### 7.2 Zigbee 3.0安全

Zigbee 3.0使用AES-128进行网络层和应用层加密,引入Install Code机制改善入网安全。Trust Center(通常是协调器)负责管理和分发网络密钥。

Zigbee安全的历史弱点在于早期设备入网时,网络密钥在空中明文传输(或使用已知的默认Trust Center Link Key),可能被截获。Install Code机制通过设备出厂预置的唯一代码加密入网过程,大大缓解了这个问题。但旧设备如果不支持Install Code,仍然是安全链条中的薄弱环节。

总体来说,两种协议的安全水平都足以满足消费级智能家居需求。Z-Wave S2的优势在于强制执行,安全基线统一;Zigbee 3.0的安全足够好,但旧设备兼容可能引入风险。

## 8. Hub生态与地区差异

### 8.1 Hub/控制器支持

主流智能家居Hub的协议支持情况:

```
同时支持Z-Wave和Zigbee:
  Samsung SmartThings, Hubitat Elevation
  Home Assistant (通过USB适配器)
  Homey, Vera/Ezlo

仅支持Zigbee:
  Amazon Echo (部分型号内置Zigbee协调器)
  IKEA DIRIGERA, Philips Hue Bridge
  Xiaomi/Aqara Hub, Tuya网关

仅支持Z-Wave:
  较少,大部分Z-Wave Hub同时也支持Zigbee
```

Zigbee由于芯片成本低,被更多低价Hub采用。许多亚洲品牌的智能家居Hub(绿米Aqara、小米、涂鸦智能)选择Zigbee作为唯一或主要协议。

### 8.2 地区差异

Z-Wave在北美市场份额更高,许多美国流行的智能家居品牌(Aeotec、Zooz、GE/Jasco)以Z-Wave为主。北美用户对Z-Wave的认知度和接受度较高。

Zigbee在亚洲(尤其是中国)市场占据压倒性优势。中国的智能家居生态几乎全部基于Zigbee或WiFi,Z-Wave设备在中国市场很少见。欧洲市场则是两者并存。

### 8.3 设备品类覆盖

Z-Wave认证的设备约有4000多款,集中在智能家居核心品类:开关、插座、传感器、门锁、恒温器。

Zigbee的设备品类更广泛,除了智能家居,还包括商业照明(Philips Hue商业方案)、工业传感器、能源管理等。仅智能家居品类的Zigbee设备数量就已超过Z-Wave。

## 9. 未来展望与Matter的影响

### 9.1 Matter对两种协议的不同影响

Matter协议的出现对Z-Wave和Zigbee的影响方式截然不同:

Zigbee与Matter共享同一个管理组织(CSA)。Matter可以运行在Thread传输层上,而Thread与Zigbee共享IEEE 802.15.4物理层。许多Zigbee芯片(如Silicon Labs EFR32MG、Nordic nRF52840)可以通过固件更新支持Thread/Matter。Zigbee到Matter的过渡路径相对平滑。

Z-Wave运行在完全不同的物理层(Sub-GHz),无法直接运行Matter协议。Z-Wave设备只能通过Hub桥接到Matter生态——Hub同时充当Z-Wave控制器和Matter桥接器。Z-Wave Alliance继续独立发展Z-Wave Long Range技术,强调Sub-GHz频段的不可替代性。

### 9.2 长期共存判断

两种协议在可预见的未来都将继续存在。Z-Wave凭借Sub-GHz穿墙优势和严格互操作认证,在多层住宅、厚墙建筑等场景中不可替代。Zigbee凭借更大的生态、更低的成本和与Thread/Matter的平滑过渡路径,在主流市场继续保持强劲竞争力。

已安装的数百万Z-Wave和Zigbee设备不会一夜之间被Matter替代。对消费者来说,两种协议都是经过时间验证的成熟选择。

## 10. 选型建议

### 10.1 选Z-Wave的场景

Z-Wave更适合以下情况:你住在多层独立住宅或厚墙老房子(穿墙需求强),你对设备兼容性有很高的确定性要求(不想折腾),设备总数在100个以内,所在地区Z-Wave设备选择丰富(北美或欧洲)。

### 10.2 选Zigbee的场景

Zigbee更适合以下情况:你对预算比较敏感(需要部署大量设备),你需要更丰富的设备品类和品牌选择,你在亚洲/中国市场购买设备,你的网络规模可能很大(几百个节点),你考虑未来平滑过渡到Matter/Thread生态。

### 10.3 混合使用策略

如果你的Hub同时支持Z-Wave和Zigbee(如SmartThings、Hubitat、Home Assistant),最务实的策略是根据具体需求混合使用:

```
按设备类型选择协议的参考建议:

设备类型        推荐协议    理由
门锁           Z-Wave     S2 Access Control安全等级最高
车库门控制器    Z-Wave     穿墙到车库,可靠性关键
地下室传感器    Z-Wave     Sub-GHz穿越楼板能力强
智能灯泡       Zigbee     品类丰富,价格低
窗帘电机       Zigbee     选择多,性价比高
温湿度传感器    两者皆可    按品牌偏好和价格选择
运动传感器      两者皆可    按具体产品评价选择
智能插座       Zigbee     部署量大时成本优势明显
烟雾报警器     Z-Wave     可靠性关键,穿墙到每个房间
```

这种混合策略充分发挥了两种协议各自的优势,是目前智能家居爱好者社区中最受推崇的做法。

## 总结

Z-Wave和Zigbee是智能家居领域两个最成熟的mesh网络协议,它们代表了两种不同的设计哲学。

Z-Wave选择了控制和确定性:单一芯片供应商保证一致性,强制认证保证互操作,Sub-GHz频段保证穿墙性能。代价是更高的价格、更小的生态、以及单一供应商的风险。

Zigbee选择了开放和规模:多芯片供应商带来竞争和低价,大生态提供丰富选择,全球统一频率简化产品设计。代价是历史上的互操作性问题(Zigbee 3.0已大幅改善)和2.4GHz频段的潜在干扰。

对于大多数家庭来说,最稳妥的策略不是二选一,而是选择一个同时支持两种协议的Hub,然后根据每个设备的实际需求灵活选择。
