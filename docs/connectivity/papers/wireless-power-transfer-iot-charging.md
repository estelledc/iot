# 无线电力传输在IoT设备充电中的应用
> **难度**: 中级 | **领域**: 无线供电 | **阅读时间**: 约 20 分钟

## 引言

想象一下你把手机放在桌面上,不需要插任何线缆,它就开始自动充电了——这就是无线电力传输(Wireless Power Transfer, WPT)最直观的体现。现在把这个场景缩小到一颗纽扣大小的温度传感器上:它被封装在旋转的机械轴承内部,根本无法引出电线,却需要持续工作数年。WPT技术让这种"不可能"变成了现实。

对于IoT设备而言,供电一直是最大的瓶颈之一。电池更换成本高、布线受限于物理环境、太阳能受光照条件制约。WPT提供了一种全新的能量获取方式,尤其适合那些部署在恶劣或不可达环境中的传感器节点。

本文将系统介绍WPT的基本原理、主要技术路线、标准化进展,以及在IoT领域的具体应用场景和工程挑战。

## 1. WPT基本原理

### 1.1 什么是无线电力传输

无线电力传输是指不通过物理导线,而是利用电磁场、电磁波或其他介质将电能从发射端传递到接收端的技术。其核心思想可以用一个类比来理解:有线充电像是用水管接水,WPT则像是用喷泉——能量在空间中传播,接收端"接住"其中一部分。

### 1.2 近场与远场

WPT技术根据传输距离和电磁场特性,分为近场(Near-field)和远场(Far-field)两大类:

| 特性 | 近场WPT | 远场WPT |
|------|---------|---------|
| 传输距离 | 厘米级(cm) | 米级(m) |
| 功率范围 | 瓦级(W) | 微瓦-毫瓦级(uW-mW) |
| 效率 | 高(60%-90%+) | 低(1%-30%) |
| 典型技术 | 电磁感应、磁共振 | RF能量收集 |
| 对准要求 | 严格到中等 | 宽松 |
| 适用场景 | 手机充电、植入设备 | 分布式传感器 |

这两种方式各有适用场景。近场适合需要较大功率但距离短的场合,远场适合功率需求极低但需要长距离传输的IoT节点。

## 2. 电磁感应耦合

### 2.1 工作原理

电磁感应是最成熟的WPT技术。其原理基于法拉第电磁感应定律:发射线圈中的交变电流产生交变磁场,该磁场穿过接收线圈时感应出电动势。

```
发射端                    接收端
+--------+              +--------+
| AC电源  |---->[ 发射线圈 ]~~~~磁场~~~~[ 接收线圈 ]---->| 整流 |---->负载
+--------+              +--------+
         |<--- 几mm到几cm --->|
```

关键参数包括:

- 耦合系数k: 表示两线圈间磁通量的耦合程度,k越接近1效率越高
- 工作频率: 通常在100-300kHz范围
- 线圈对准: 横向偏移和角度倾斜都会降低效率

### 2.2 Qi标准

Qi(读作"chee")是无线充电联盟(WPC)制定的主流近场WPT标准:

- 工作频率: 87-205kHz(低功率)和80-300kHz(中功率)
- 传输距离: 典型5-10mm
- 功率范围: 5W(基础)、15W(扩展)、最高可达50W+
- 效率: 对准良好时可达80%以上
- 通信: 发射端与接收端通过负载调制进行通信,实现功率协商

### 2.3 IoT中的应用限制

对于大多数IoT设备,电磁感应耦合存在明显限制:

- 距离太短: 传感器往往部署在难以接近的位置
- 对准要求高: 自动充电场景中难以保证精确对准
- 接收线圈尺寸: IoT设备体积小,线圈面积受限,接收功率下降

## 3. 磁共振耦合

### 3.1 工作原理

磁共振耦合是电磁感应的"升级版"。两个线圈被设计为在同一频率上谐振,当频率匹配时,即使耦合系数较低(松耦合),也能实现高效能量传输。

这就像两个音叉的共振:当一个音叉振动时,即使距离较远,同频率的另一个音叉也会开始振动。

### 3.2 技术特点

```python
# 磁共振系统关键参数计算示例
import math

def resonant_frequency(L, C):
    """计算LC谐振频率"""
    # f = 1 / (2 * pi * sqrt(L * C))
    f = 1 / (2 * math.pi * math.sqrt(L * C))
    return f

def quality_factor(f, L, R):
    """计算品质因数Q"""
    # Q = 2 * pi * f * L / R
    Q = 2 * math.pi * f * L / R
    return Q

def coupling_efficiency(k, Q_tx, Q_rx):
    """估算耦合效率(简化模型)"""
    # eta ≈ (k^2 * Q_tx * Q_rx) / (1 + k^2 * Q_tx * Q_rx)
    kQ_product = k**2 * Q_tx * Q_rx
    eta = kQ_product / (1 + kQ_product)
    return eta

# 示例: 6.78MHz系统
L = 2e-6      # 2uH电感
C = 276e-12   # 276pF电容
R = 0.5       # 0.5欧姆线圈电阻

f_res = resonant_frequency(L, C)
Q = quality_factor(f_res, L, R)
print(f"谐振频率: {f_res/1e6:.2f} MHz")
print(f"品质因数Q: {Q:.1f}")

# 不同距离下的效率
for k in [0.3, 0.1, 0.05, 0.02]:
    eta = coupling_efficiency(k, Q, Q)
    print(f"耦合系数k={k}: 效率={eta*100:.1f}%")
```

### 3.3 与电磁感应的对比

磁共振相比传统电磁感应的优势:

- 传输距离更远: 10-50cm(vs 感应的几mm)
- 对准容忍度高: 可接受较大的横向偏移
- 支持一对多: 一个发射端可同时为多个接收端供电

代价是:

- 系统复杂度更高(需要精确频率调谐)
- 效率随距离下降(50cm时可能只有30-50%)
- 电磁兼容(EMC)问题更突出

## 4. RF能量收集

### 4.1 工作原理

RF能量收集是从环境中的射频信号(如WiFi、蜂窝基站、专用RF发射器)中提取能量并转换为直流电的技术。核心组件是整流天线(Rectenna),由接收天线和整流电路组成。

```
RF信号源          空间传播          接收端
+--------+                    +-----------+    +--------+    +------+
| RF发射  |))))))))))))))))))))| 接收天线   |--->| 整流电路 |--->| 储能  |
+--------+                    +-----------+    +--------+    +------+
                                                              |
                                                         IoT传感器
```

### 4.2 可用能量估算

RF能量收集的功率密度取决于距离和发射功率:

| RF源 | 频率 | 距离 | 可收集功率 |
|------|------|------|-----------|
| 专用2.4GHz发射器(1W EIRP) | 2.4GHz | 1m | ~100uW |
| 专用915MHz发射器(4W EIRP) | 915MHz | 5m | ~40uW |
| WiFi接入点 | 2.4GHz | 5m | ~1-10uW |
| 蜂窝基站 | 900MHz | 100m | ~0.1-1uW |
| 环境RF(城市) | 多频 | - | ~0.01-1uW |

### 4.3 整流天线(Rectenna)设计

整流天线的转换效率是关键指标:

- 天线增益: 高增益天线可以捕获更多能量,但增大体积
- 整流效率: 肖特基二极管在低功率时效率下降(典型20-60%)
- 匹配网络: 天线阻抗与整流电路阻抗的匹配至关重要
- 多频段设计: 同时从多个频段收集能量可提高总功率

### 4.4 IoT中的实际功率需求

好消息是,许多IoT传感器的功率需求极低:

- 温度传感器单次采样: ~1uJ
- LoRa一次发送: ~50mJ
- BLE广播一次: ~0.01mJ
- 深度睡眠维持: ~1uW

如果传感器每10分钟采样一次并发送,平均功耗可能只有几十微瓦——这完全在RF能量收集的能力范围内。

## 5. SWIPT: 同时无线信息与功率传输

### 5.1 概念

SWIPT(Simultaneous Wireless Information and Power Transfer)是一种将数据通信和能量传输整合到同一射频信号中的技术。对于IoT设备,这意味着基站在向设备发送控制命令的同时,也在为设备充电。

### 5.2 接收架构

SWIPT有两种主要的接收架构:

- 时间切换(Time Switching, TS): 在不同时隙分别用于能量收集和信息解码
- 功率分割(Power Splitting, PS): 将接收信号按比例分为能量收集和信息解码两路

```
时间切换架构:
时隙1: [====能量收集====]
时隙2: [====信息解码====]

功率分割架构:
        +---> alpha * P ---> 能量收集
接收信号 -|
        +---> (1-alpha) * P ---> 信息解码
```

### 5.3 与IoT的结合

SWIPT在IoT中的潜在应用包括:

- NB-IoT下行链路中集成能量传输
- WiFi信标帧同时携带能量
- 专用RFID读写器为被动标签供电并通信

目前SWIPT仍主要处于研究阶段,商业化应用有限,但其"一举两得"的特性对于大规模IoT部署具有重要意义。

## 6. WPT标准与生态

### 6.1 主要标准

| 标准/技术 | 组织 | 技术路线 | 频率 | 距离 |
|-----------|------|---------|------|------|
| Qi | WPC | 电磁感应 | 87-205kHz | <10mm |
| Qi2 | WPC | 感应+磁共振 | 扩展频率 | 扩展距离 |
| AirFuel RF | AirFuel | RF远场 | 900MHz/2.4GHz/5.8GHz | 数米 |
| Energous WattUp | Energous | RF远场 | 915MHz | ~5m |
| Cota | Ossia | RF远场 | 2.4GHz/5.8GHz | ~10m |

### 6.2 IoT专用方案

针对IoT场景的WPT方案通常具有以下特点:

- 超低功率优化: 接收端功耗控制在微瓦级
- 多设备支持: 一个发射器同时为数十个传感器供电
- 环境友好: 满足RF安全限值(如FCC/ICNIRP规定)
- 小型化接收: 整流天线集成在PCB上,无需额外组件

## 7. 工程挑战

### 7.1 效率与距离的矛盾

这是WPT最根本的trade-off:

```
效率(%)
100|*
   | *
 80|  **
   |    ***
 60|       ***
   |          ****
 40|              *****
   |                   ******
 20|                         ********
   |                                 ***********
  0+----+----+----+----+----+----+----+----+----> 距离
   0   0.5   1   1.5   2   2.5   3   3.5   4 (倍波长)
```

近场技术在短距离内高效但距离有限;远场技术覆盖范围大但效率很低。没有一种技术能"兼得"。

### 7.2 安全法规

RF能量传输需要遵守严格的人体暴露安全标准:

- FCC(美国): 在2.4GHz频段,最大EIRP通常限制在1W
- ICNIRP(国际): 公众暴露限值在900MHz约为4.5W/m^2
- 中国: GB 8702-2014规定公众电磁辐射暴露限值

这些限制直接约束了远场WPT的可用功率密度。

### 7.3 对准与追踪

对于近场WPT,接收端移动或旋转会导致效率剧烈下降。解决方案包括:

- 多线圈阵列: 发射端使用线圈阵列,自动选择最近的线圈
- 波束成形: 远场WPT中通过相控阵将能量聚焦到接收端
- 信标辅助: 接收端发送信标信号帮助发射端定位

### 7.4 环境影响

金属物体、人体、液体等都会影响WPT性能:

- 金属物体可能产生涡流损耗,导致发热
- 人体组织会吸收RF能量
- 多径反射可能形成盲区

## 8. 实际案例: 旋转机械上的无线供电传感器

### 8.1 场景描述

工业环境中,旋转设备(如电机轴承、涡轮叶片、离心机)的振动和温度监测至关重要,但这些部件持续旋转,无法使用有线供电和有线通信。

### 8.2 系统架构

```
固定端(Stator侧)              旋转端(Rotor侧)
+------------------+          +------------------+
| 工业网关          |          | 温度传感器        |
| + WPT发射电路     |          | + 振动传感器      |
| + 环形发射线圈    |~~~~磁场~~~| + 环形接收线圈    |
| + BLE接收        |<--BLE---| + 超级电容储能    |
+------------------+          | + BLE发送        |
       |                      +------------------+
       | 以太网
       v
  工业SCADA系统
```

### 8.3 设计要点

- WPT技术选择: 旋转耦合器(Rotary Coupler),基于电磁感应
- 发射/接收线圈: 同心环形结构,旋转不影响耦合
- 工作频率: 200kHz
- 传输间隙: 2-5mm(轴承间隙)
- 传输功率: 2W(足够供电传感器+BLE模块)
- 效率: 约70%(考虑气隙和旋转)

### 8.4 功率预算

```
接收功率:              2W * 0.7 = 1.4W
传感器采样(1Hz):       50mW(峰值)
BLE发送(每秒一次):     30mW(峰值)
MCU运行:              5mW(平均)
深度睡眠:             10uW
平均功耗:             约15mW
富余系数:             1.4W / 15mW ≈ 93倍
```

大量的功率富余意味着即使效率下降(如间隙增大),系统仍能正常工作。超级电容提供短时缓冲,应对瞬时功率波动。

### 8.5 实施效果

- 消除了滑环(slip ring)的磨损问题,免维护
- 传感器持续在线,无需更换电池
- 振动数据实现实时监控,提前预警轴承故障
- 系统已在多个工厂的旋转设备上部署超过3年

## 9. 未来趋势

### 9.1 激光无线供电

使用聚焦激光束将能量传输到远处的光伏接收器,可实现数百米甚至公里级的定向功率传输。安全性和对准是主要挑战。

### 9.2 超材料增强

利用超材料(Metamaterial)聚焦或引导电磁场,提升WPT在特定距离上的效率。

### 9.3 大规模IoT无线供电网络

未来可能出现类似WiFi热点的"充电热点"网络,为区域内所有IoT设备持续供电,彻底消除电池限制。

## 总结

无线电力传输为IoT设备的能量供给提供了革命性的解决方案。核心要点如下:

- 近场WPT(感应/共振)适合短距离高功率场景,效率高但距离受限
- 远场WPT(RF收集)适合长距离超低功率场景,覆盖广但功率小
- 许多IoT传感器功耗极低(微瓦级),RF能量收集已足够支撑
- SWIPT将通信和供电合一,是IoT的理想方案但仍在研究阶段
- 工业场景中的旋转耦合器已经是成熟的商业化方案
- 效率与距离的根本矛盾决定了需要根据场景选择合适的技术路线

选择WPT方案时,核心考量是: 设备功耗有多低、距离有多远、是否可以对准。三者的组合决定了最佳技术路线。
