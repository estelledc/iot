# 无线信息与能量同传SWIPT在IoT中的应用
> **难度**: 高级 | **领域**: 能量收集 | **阅读时间**: 约 22 分钟

## 引言

想象你家里的路由器不仅能传输WiFi信号让你上网,还能同时给桌上的遥控器、温度传感器无线充电。你不需要换电池,也不需要插电源线,设备就像植物吸收阳光一样,从空气中的无线电波里获取能量和数据。这就是SWIPT(Simultaneous Wireless Information and Power Transfer,无线信息与能量同传)的核心思想。

在传统的无线通信中,射频(RF)信号承载信息后到达接收端,其中蕴含的能量通常被浪费掉。SWIPT的革命性在于: 同一个RF信号既是信息的载体,又是能量的来源。对于大量部署的低功耗IoT设备来说,这意味着可以摆脱电池更换或有线供电的束缚,实现真正的"部署即遗忘"。

本文将深入探讨SWIPT的理论基础、接收机架构、与MIMO等先进技术的结合,以及在IoT场景中的实际应用前景。

## 1. SWIPT的基本概念与动机

### 1.1 从信息论到能量收集

Shannon的信息论告诉我们,无线信道的容量取决于信号功率和噪声。而从物理角度看,接收到的RF信号本身就是电磁波,携带着可被转换为直流电的能量。SWIPT的理论根基在于:

- 信息解码需要信号的波形特征(相位、幅度、频率)
- 能量收集需要信号的功率(振幅的平方)
- 两者来源于同一个接收信号,但提取方式不同

### 1.2 为什么IoT需要SWIPT

传统IoT设备的能量来源面临诸多挑战:

| 供能方式 | 问题 |
|---------|------|
| 一次性电池 | 需人工更换,大规模部署时成本极高 |
| 可充电电池 | 仍需充电基础设施 |
| 太阳能 | 依赖光照,室内/暗处不可用 |
| 振动能量 | 功率极低,场景受限 |
| 专用无线充电 | 需额外发射设备,不传数据 |

SWIPT的优势在于复用已有的通信基础设施,不需要额外的充电网络,通信和供能合二为一。

### 1.3 基本系统模型

一个典型的SWIPT系统包含:

- 发射端(Base Station / Access Point): 发送包含信息的RF信号
- 无线信道: 信号在传播中衰减
- 接收端(IoT Device): 同时从信号中提取信息和能量

接收信号可以表示为:

```
y(t) = h * x(t) + n(t)

其中:
  h: 信道增益
  x(t): 发射信号
  n(t): 噪声
  接收功率: P_r = |h|^2 * P_t
```

关键约束: 信息解码需要较高的信噪比(SNR),而能量收集需要较高的接收功率。两者对信号的需求方向一致(都希望收到强信号),但处理方式互相冲突。

## 2. 接收机架构

### 2.1 理想接收机(理论基准)

理论上,如果接收机能同时从相同信号中完美提取信息和能量,就得到了Rate-Energy(R-E)区域的外边界。但由于信息解码电路和能量收集电路的物理机制不同,实际中无法同时对同一信号做两件事。

### 2.2 时间切换(Time Switching, TS)

时间切换是最直观的方案: 将时间分成两段,一段专门用于信息解码,另一段专门用于能量收集。

```
时间帧结构:
|<--- alpha*T --->|<--- (1-alpha)*T --->|
|   能量收集阶段   |    信息解码阶段      |

alpha: 时间切换因子 (0 < alpha < 1)
收集能量: E = alpha * T * eta * P_r
信息速率: R = (1-alpha) * B * log2(1 + SNR)

其中:
  eta: 射频到直流转换效率 (通常 10%-80%)
  B: 信道带宽
  T: 总帧时长
```

优点: 接收机结构简单,只需一个RF开关在两种模式间切换。

缺点: 同一时刻只能做一件事,存在时间资源的浪费。

### 2.3 功率分割(Power Splitting, PS)

功率分割方案将接收信号的功率按比例分成两路: 一路送入信息解码电路,另一路送入能量收集电路。

```
接收信号功率分配:
         接收信号 P_r
            |
     -------|-------
     |             |
  rho*P_r    (1-rho)*P_r
     |             |
  能量收集     信息解码

rho: 功率分割因子 (0 < rho < 1)
收集能量: E = eta * rho * P_r * T
信息速率: R = B * log2(1 + (1-rho)*P_r / (sigma^2))
```

优点: 信息传输和能量收集可以同时进行,时间利用率高。

缺点: 需要精确的功率分割器硬件,且分割比例需要根据信道状态动态调整。

### 2.4 两种架构对比

| 特性 | 时间切换(TS) | 功率分割(PS) |
|------|-------------|-------------|
| 硬件复杂度 | 低(仅需RF开关) | 中(需功率分割器) |
| 同时收发 | 不支持 | 支持 |
| 最优性 | 次优 | 接近理论极限 |
| 适用场景 | 突发传输,间歇通信 | 持续数据流 |
| 实现难度 | 简单 | 需精确校准 |

### 2.5 天线切换(Antenna Switching)

在多天线设备上,还可以将部分天线专门用于信息接收,另一部分专门用于能量收集。这种方案在MIMO系统中特别有效。

## 3. 速率-能量(R-E)权衡

### 3.1 R-E区域的含义

SWIPT的核心理论问题是: 给定一个发射功率预算和信道条件,信息速率R和收集能量E之间的最优权衡是什么?

R-E区域是所有可达(R, E)对的集合。其边界代表帕累托最优: 要增加能量收集就必须牺牲信息速率,反之亦然。

### 3.2 数学表达

对于功率分割接收机,R-E权衡可以表示为:

```
最大化: R = B * log2(1 + (1-rho) * gamma)
约束:   E >= E_min = eta * rho * P_r * T
         0 <= rho <= 1

其中 gamma = P_r / sigma^2 为接收SNR

拉格朗日分析得最优分割:
  rho* = E_min / (eta * P_r * T)
  R* = B * log2(1 + (1 - E_min/(eta*P_r*T)) * gamma)
```

### 3.3 关键洞察

- 当距离增大时,R-E区域急剧缩小(路径损耗的双重影响)
- 高频信号衰减更快,但可以承载更多信息
- 实际RF-DC转换效率(eta)是决定性能的关键瓶颈

## 4. SWIPT与MIMO/波束赋形

### 4.1 MIMO-SWIPT的优势

多天线(MIMO)系统为SWIPT带来额外自由度:

- 空间复用: 不同空间方向可以同时服务信息用户和能量用户
- 波束赋形增益: 将能量集中到特定方向,补偿路径损耗
- 干扰管理: 对信息用户的干扰可以成为能量用户的"养分"

### 4.2 波束赋形设计

在多用户MIMO-SWIPT中,基站需要同时设计:

```
信息波束: w_k  (为第k个信息用户)
能量波束: v_j  (为第j个能量用户)

优化问题:
  最大化: sum_k R_k (总信息速率)
  约束1: E_j >= E_min_j, for all j (能量用户最低收集要求)
  约束2: sum ||w_k||^2 + sum ||v_j||^2 <= P_total
```

### 4.3 专用能量波束 vs 信息信号收集

一种策略是发送专门的能量信号(对信息用户是干扰,但能量用户受益)。另一种是让能量用户仅从信息信号中收集能量。实践中通常采用混合策略。

### 4.4 大规模MIMO(Massive MIMO)的优势

当基站天线数量很大(如64或128根)时:

- 极窄波束可以精确瞄准IoT设备,大幅提升接收功率
- 多个IoT设备可以在同一时频资源上被同时服务
- 信道硬化效应使得传输更加可预测和稳定

## 5. 实际功率范围与限制

### 5.1 RF能量收集的现实

理论很美好,但实际的RF能量收集面临严峻挑战:

| 距离(m) | 接收功率(典型值) | 可供能的设备 |
|---------|-----------------|-------------|
| 1 | 约 100 uW | 低功耗传感器 |
| 3 | 约 10 uW | 仅最省电的设备 |
| 5 | 约 1-5 uW | 几乎无法直接驱动 |
| 10 | 约 0.1 uW | 需超长时间蓄能 |

注: 上述数据基于典型的915MHz RFID频段,1W EIRP发射功率。

### 5.2 路径损耗的影响

自由空间路径损耗公式:

```
PL(dB) = 20*log10(d) + 20*log10(f) - 147.55

在900MHz频段:
  1m处: PL = -31.5 dB
  5m处: PL = -45.5 dB (比1m少14dB, 即1/25的功率)
  10m处: PL = -51.5 dB (比1m少20dB, 即1/100的功率)
```

### 5.3 RF-DC转换效率

整流天线(Rectenna)的效率受输入功率影响很大:

- 高输入功率(>0 dBm): 效率可达 60-80%
- 中等功率(-10 to 0 dBm): 效率约 30-50%
- 低功率(-20 dBm以下): 效率急剧下降到 10%以下

这意味着距离越远,不仅接收功率低,转换效率也低,形成双重打击。

## 6. 与其他能量收集技术的对比

### 6.1 综合对比

| 技术 | 功率密度 | 可预测性 | 全天候 | 适用距离 |
|------|---------|---------|--------|---------|
| SWIPT(RF) | 低(uW级) | 高(可控) | 是 | 1-10m |
| 太阳能 | 高(mW/cm2) | 低(天气) | 否 | N/A |
| 热电 | 中(uW-mW) | 中 | 部分 | N/A |
| 振动 | 低(uW) | 低(不可控) | 部分 | N/A |
| 专用无线充电 | 高(mW) | 高 | 是 | 1-5m |

### 6.2 SWIPT的独特优势

尽管SWIPT的功率密度低,但它有一个其他技术无法比拟的优势: 能量和信息来自同一信号。这意味着:

- 不需要额外的充电基础设施
- 通信调度和能量调度可以联合优化
- 通信越频繁的设备,获得的能量也越多

### 6.3 混合能量收集

实际IoT设备往往采用混合策略:

- 主要能量来自太阳能或振动
- SWIPT作为补充,在通信时"顺便"充电
- 超级电容器或薄膜电池作为能量缓冲

## 7. 前沿研究方向

### 7.1 IRS辅助SWIPT

智能反射面(IRS, Intelligent Reflecting Surface)是近年来最热门的研究方向之一。IRS由大量可编程反射单元组成,可以控制无线信号的反射方向。

IRS对SWIPT的帮助:

```
传统SWIPT:
  发射机 ----直射路径(衰减大)----> IoT设备

IRS辅助SWIPT:
  发射机 ---> IRS(增强反射) ---> IoT设备
  等效信道增益 = |h_direct + H_IRS * Phi * g|^2

其中:
  h_direct: 直射信道
  H_IRS: 发射机到IRS的信道
  Phi: IRS相位控制矩阵(可优化)
  g: IRS到IoT设备的信道
```

通过优化IRS的相位,可以使信号在IoT设备处相干叠加,显著提升接收功率。研究表明IRS可以将SWIPT的有效范围扩展2-3倍。

### 7.2 NOMA-SWIPT

非正交多址(NOMA)允许多个用户共享同一时频资源,通过功率域区分。在NOMA-SWIPT中:

- 近端用户(信号强): 解码信息 + 收集多余能量
- 远端用户(信号弱): 主要收集能量,低速率通信
- 利用串行干扰消除(SIC)在近端用户处解码并去除远端用户的信号

### 7.3 全双工SWIPT

全双工技术允许设备在收集能量的同时上传数据:

- 下行: 接收基站信号,收集能量
- 上行: 同时利用收集的能量发送传感器数据
- 挑战: 自干扰消除的能量开销

### 7.4 宽带SWIPT

利用OFDM等宽带信号的特点:

- 不同子载波可以分配不同的信息/能量比例
- 频率选择性信道中,深衰落子载波用于能量收集(反正解码也困难)
- 联合子载波分配和功率控制优化

## 8. 实际案例: 智能零售货架标签

### 8.1 应用场景

大型超市中有数千个电子价格标签(ESL, Electronic Shelf Label),需要显示商品价格并支持动态更新。传统方案使用纽扣电池,每1-2年需要人工更换。

### 8.2 SWIPT方案设计

```
系统架构:
  天花板AP(Access Point)
       |
       | RF信号 (920MHz ISM频段)
       | (同时携带价格数据和能量)
       v
  [ESL标签1] [ESL标签2] ... [ESL标签N]
  (距离AP约2-4米)

AP发射参数:
  频率: 920 MHz
  EIRP: 4W (36 dBm), 符合ISM频段法规
  带宽: 500 kHz
  调制: FSK
  天线: 4x4 MIMO面板天线

ESL标签接收端:
  接收天线: 微带贴片天线
  接收架构: 功率分割 (rho = 0.7用于能量, 0.3用于信息)
  能量存储: 100uF超级电容
  显示: 电子墨水屏(仅刷新时耗电)
```

### 8.3 能量预算分析

```
接收功率计算(3米距离):
  路径损耗: 20*log10(3) + 20*log10(920e6) - 147.55 = 41.3 dB
  接收功率: 36 - 41.3 = -5.3 dBm = 约 295 uW

能量收集:
  分割给能量的部分: 295 * 0.7 = 206 uW
  RF-DC效率(在-5.3dBm时): 约 45%
  实际收集: 206 * 0.45 = 93 uW

ESL功耗:
  待机: 0.5 uW (电子墨水保持显示不耗电)
  刷新一次: 约 15 mJ (持续3秒, 5mW)
  每天刷新次数: 3次(早中晚价格更新)
  日均功耗: 0.5 uW + 15e-3*3/(86400) = 约 1 uW

能量盈余:
  AP每天广播充电时间: 每小时5分钟 = 120分钟/天
  日收集能量: 93e-6 * 120 * 60 = 669.6 mJ
  日消耗: 约 0.13 mJ
  结论: 能量充裕,系统可持续运行
```

### 8.4 通信协议设计

```
帧结构 (每5分钟一个超帧):
|--能量收集(4min)--|--信息传输(1min)--|

信息传输阶段:
  1. AP广播Beacon (同步+设备唤醒)
  2. AP发送价格更新数据包 (广播或单播)
  3. ESL回复ACK (利用积累的能量)

数据帧格式:
  [前导码 8B][帧头 4B][ESL_ID 4B][价格数据 16B][CRC 2B]
  总长: 34 字节
  传输时间: 34*8/500000 = 0.544 ms (远小于1分钟窗口)
```

### 8.5 系统优势总结

与传统电池供电ESL相比:

| 指标 | 传统电池ESL | SWIPT-ESL |
|------|-----------|-----------|
| 电池寿命 | 1-2年需更换 | 无需电池 |
| 维护成本(1000标签) | 约每年5000元 | 接近零 |
| 更新频率 | 受限于电池容量 | 不受限 |
| 环境影响 | 电池废弃物 | 绿色环保 |
| 初始部署成本 | 低 | 中(需AP) |

## 总结

SWIPT代表了无线通信和无线供能的融合方向。其核心价值在于复用已有的通信基础设施为IoT设备供能,消除电池依赖。虽然当前的功率水平(微瓦级)限制了应用范围,但随着IRS、大规模MIMO等技术的发展,以及IoT设备功耗的持续降低,SWIPT有望在智能零售、仓储物流、环境监测等场景中实现规模化应用。

关键要点:

- 时间切换和功率分割是两种基本接收机架构,各有优劣
- R-E权衡是SWIPT的核心理论问题
- MIMO波束赋形可以显著扩展SWIPT的有效范围
- IRS和NOMA是当前最活跃的研究方向
- 实际应用需要极低功耗设备配合(如电子墨水标签)
