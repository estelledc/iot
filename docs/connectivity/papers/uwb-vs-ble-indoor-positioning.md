# UWB与BLE室内定位精度对比
> **难度**：🟡 中级 | **领域**：室内定位技术 | **阅读时间**：约 20 分钟

## 引言

想象你在一个巨大的仓库里找一箱零件。如果有人只告诉你"在东边那一片"，你可能要花十几分钟翻找；但如果有人指着说"第三排货架第二层左边第三个格子"，你几秒钟就能拿到。室内定位技术的精度差异就像这两种指路方式的区别——粗略的方向和精确的坐标带来的效率完全不同。

GPS 在室外定位精度约 3-5 米，足以满足导航需求。但 GPS 信号无法穿透建筑物，在室内基本不可用。室内定位需要依赖建筑内部署的基础设施，通过无线信号来确定设备位置。目前 IoT 领域最主流的两种室内定位技术是 UWB(超宽带)和 BLE(低功耗蓝牙)，它们采用完全不同的物理原理，在精度、成本、功耗等维度各有所长。

本文从技术原理出发，系统对比 UWB 和 BLE 的室内定位方案，帮助你在实际项目中做出合理选择。

## 1. 室内定位的核心挑战

### 1.1 为什么GPS在室内失效

GPS 卫星信号频率为 1.575 GHz，功率极低(到达地面约 -130 dBm)。建筑物的墙壁、楼板、金属结构会将信号衰减 20-40 dB，使得室内接收机无法锁定足够数量的卫星来计算位置。即使在窗户边偶尔收到信号，多径反射也会导致定位误差超过 10 米。

### 1.2 室内定位的基本思路

室内定位的核心思路是：在已知位置部署参考节点(锚点/信标)，让待定位设备与这些参考节点交换无线信号，然后通过信号的某种物理特征来推算距离或方向，最终计算出位置。

```
室内定位基本模型:

  [锚点A]         [锚点B]         [锚点C]
     \               |               /
  测距/测角      测距/测角      测距/测角
       \             |             /
        \            |            /
         +----[待定位设备]----+
                  |
          几何计算得到坐标
```

可用于定位的物理特征包括: 信号飞行时间(ToF)对应距离、信号强度(RSSI)对应距离、信号到达角度(AoA/AoD)对应方向、信号特征指纹对应已知位置。不同技术利用的特征不同，决定了精度上限。

## 2. UWB定位方法

### 2.1 时间飞行测距(ToF/TWR)

UWB 定位的核心优势来自其超大带宽(500 MHz以上)，能够发射纳秒级窄脉冲。信号飞行时间的测量精度直接决定测距精度——光速每纳秒走 30 厘米，UWB 的时间分辨率可达亚纳秒，因此测距精度轻松达到 10 厘米级别。

双向测距(TWR)是最常用的 UWB 测距方式：

```
设备A(Tag)                     设备B(Anchor)
  |                                |
  |--- UWB测距请求 ------------->|  t1: A发送
  |                                |  t2: B接收
  |                                |  (B处理)
  |<-- UWB测距响应 --------------|  t3: B发送
  |  t4: A接收                     |
  |                                |
  距离 = c * (t4-t1-t3+t2) / 2
  c = 光速 ≈ 3*10^8 m/s
```

改进版 DS-TWR(双侧双向测距)让双方各测一次往返时间取几何平均，消除时钟漂移误差，精度从 30cm 提升到 5-10cm。

### 2.2 三边定位

获得到多个锚点的距离后，通过三边定位(trilateration)计算设备坐标：

```python
import numpy as np

def trilaterate_2d(anchors, distances):
    """
    anchors: [(x1,y1), (x2,y2), (x3,y3)] 锚点坐标
    distances: [d1, d2, d3] 到各锚点的测距结果
    返回: (x, y) 待定位设备坐标
    """
    # 以第一个锚点为原点建立方程组
    A = []
    b = []
    x1, y1 = anchors[0]
    for i in range(1, len(anchors)):
        xi, yi = anchors[i]
        A.append([2*(xi - x1), 2*(yi - y1)])
        b.append(distances[0]**2 - distances[i]**2
                 - x1**2 + xi**2 - y1**2 + yi**2)
    A = np.array(A)
    b = np.array(b)
    position = np.linalg.lstsq(A, b, rcond=None)[0]
    return position

# 示例: 3个锚点, 各距离约2米
anchors = [(0, 0), (5, 0), (2.5, 4)]
dists = [2.1, 3.2, 2.8]
pos = trilaterate_2d(anchors, dists)
print(f"设备位置: ({pos[0]:.1f}, {pos[1]:.1f})")
```

2D 定位至少需要 3 个锚点，3D 定位至少需要 4 个。实际部署中通常多于最低数量以提高精度和鲁棒性。

### 2.3 UWB定位的基础设施

UWB 定位需要专用的 UWB 锚点(Anchor)：

```
典型UWB定位部署(仓库):

  天花板高度4-6m
  +--[A1]----------[A2]----------[A3]--+
  |                                     |
  |   间距8-15m                         |
  |                                     |
  +--[A4]----------[A5]----------[A6]--+
  |                                     |
  +--[A7]----------[A8]----------[A9]--+

  每个锚点需要: 电源 + 网络(以太网或WiFi回传)
  TDoA模式还需要: 锚点间纳秒级时钟同步
```

锚点单价 100-300 美元，加上布线、安装和校准，一个 2000 平方米的区域典型投入约 5000-15000 美元。

## 3. BLE定位方法

### 3.1 RSSI指纹定位

BLE 最传统的定位方式是基于接收信号强度(RSSI)。基本原理是：信号随距离衰减，通过测量信号强度可以估算距离。

但 RSSI 受环境影响极大——人体遮挡、金属反射、湿度变化都会导致同一距离的 RSSI 波动 10-20 dB。因此直接用 RSSI 测距精度很差(5-10 米)。

指纹定位是 RSSI 定位的改进方案：

```
指纹定位两阶段:

1. 离线采集(建立指纹库):
   在每个参考点记录收到各信标的RSSI值
   参考点1: [BeaconA=-45dBm, BeaconB=-62dBm, BeaconC=-78dBm]
   参考点2: [BeaconA=-58dBm, BeaconB=-41dBm, BeaconC=-69dBm]
   ...

2. 在线定位(实时匹配):
   测量当前RSSI: [BeaconA=-46dBm, BeaconB=-60dBm, BeaconC=-77dBm]
   与指纹库比对 --> 最接近参考点1 --> 估计位置
```

指纹法典型精度 3-5 米，优点是不需要精确的信号传播模型，缺点是环境变化后需要重新采集指纹。

### 3.2 BLE AoA/AoD测向定位

BLE 5.1 引入了到达角(AoA)和出发角(AoD)测向功能，通过天线阵列测量信号相位差来确定方向：

```
AoA定位:
  [定位器-天线阵列]
       |  测得角度theta
       |  /
       | /
       |/ theta
  [BLE标签]

  多个定位器各测角度 -> 角度交叉 -> 确定位置
  精度: 0.5-1米(良好环境)
```

AoA/AoD 定位精度比 RSSI 提升了一个数量级，但需要专用的天线阵列定位器(4-16根天线)，成本介于纯信标和 UWB 锚点之间。

### 3.3 BLE基础设施

BLE 信标的最大优势是低成本和易部署：

```
BLE信标特点:
- 单价: 10-30美元(普通信标), 200-500美元(AoA定位器)
- 供电: 纽扣电池(CR2032)可运行1-3年
- 安装: 粘贴或磁吸, 无需布线
- 维护: 定期换电池

同样2000平米区域:
- RSSI方案: 20-30个信标, 约500-1000美元
- AoA方案: 8-12个定位器, 约2000-5000美元
```

另外 BLE 的一个天然优势是手机端的普及——几乎所有智能手机都内置 BLE，可以直接作为被定位设备或定位基础设施的一部分。

## 4. 精度对比

### 4.1 各技术路线精度汇总

```
| 技术方案      | 典型精度    | 最佳精度    | 刷新率      |
|--------------|-----------|-----------|------------|
| UWB TWR      | 10-30cm   | 5cm       | 10-100 Hz  |
| UWB TDoA     | 20-50cm   | 10cm      | 100+ Hz    |
| BLE AoA      | 0.5-1m    | 0.3m      | 1-10 Hz    |
| BLE RSSI指纹  | 2-5m      | 1.5m      | 0.5-2 Hz   |
| BLE RSSI测距  | 3-8m      | 2m        | 0.5-2 Hz   |
```

UWB 在精度维度上有压倒性优势，主要原因是物理层的根本差异: UWB 利用纳秒脉冲直接测量信号飞行时间，精度受限于时钟分辨率；BLE 利用窄带信号的强度或相位，精度受限于环境干扰。

### 4.2 NLOS(非视距)性能

室内环境中经常存在遮挡(人体、墙壁、货架)，非视距(NLOS)条件对各技术的影响差异很大：

```
NLOS场景精度退化:

UWB:
- LOS(视距): 10-30cm
- NLOS轻度(一层墙): 30-80cm, 可检测NLOS状态
- NLOS重度(多层遮挡): 50-150cm, 首径检测仍有效

BLE RSSI:
- LOS: 3-5m
- NLOS轻度: 5-10m, 无法判断是否NLOS
- NLOS重度: 10m+, 基本不可用

BLE AoA:
- LOS: 0.5-1m
- NLOS轻度: 1-3m
- NLOS重度: 3-5m, 多径严重干扰角度测量
```

UWB 的窄脉冲能够在时域上分辨直达路径和反射路径(首径检测)，即使在 NLOS 条件下也能保持较好精度。BLE 窄带信号无法区分多径，NLOS 对精度影响严重。

## 5. 成本对比

### 5.1 基础设施成本

| 成本项目 | UWB方案 | BLE RSSI方案 | BLE AoA方案 |
|---------|---------|-------------|-------------|
| 锚点/信标单价 | 100-300美元 | 10-30美元 | 200-500美元 |
| 2000平米所需数量 | 8-12个 | 20-30个 | 8-12个 |
| 硬件总成本 | 800-3600美元 | 200-900美元 | 1600-6000美元 |
| 安装成本 | 高(需布线) | 低(电池供电) | 中(部分需供电) |
| 维护成本 | 低(有线供电) | 中(换电池) | 低-中 |

### 5.2 终端设备成本

```
UWB模块/标签:
- UWB芯片: 1.5-5美元(Qorvo DW3000系列)
- UWB标签成品: 15-50美元
- 手机端: iPhone 11+, Galaxy S21+有UWB

BLE模块/标签:
- BLE芯片: 0.5-2美元(nRF52系列)
- BLE标签成品: 3-15美元
- 手机端: 几乎所有智能手机都支持

终端成本结论: BLE约为UWB的1/3到1/5
```

### 5.3 总拥有成本(TCO)

UWB 方案前期投入高但精度高，BLE 方案前期投入低但精度有限。对于大规模部署(如 10000 平米以上)，需要综合考虑部署面积、所需精度、设备数量、维护周期等因素来计算 TCO。

## 6. 功耗与延迟对比

### 6.1 功耗对比

```
单次操作功耗:
- UWB测距(TWR): 发射30-50mA x 1-2ms = 30-100uAh
- BLE广告(RSSI): 发射5-15mA x 1-3ms = 5-45uAh
- BLE CTE(AoA): 发射8-20mA x 2-5ms = 16-100uAh

以1秒更新间隔计算(CR2032纽扣电池230mAh):
- UWB标签: 约3-6个月
- BLE信标(RSSI): 约1-3年
- BLE标签(AoA): 约6-12个月
```

UWB 功耗高于 BLE，但差距没有想象中那么大。实际应用中可以通过降低更新频率(如从 1Hz 降到 0.1Hz)大幅延长电池寿命。另外 BLE 唤醒加 UWB 精确测距的混合策略也很常见。

### 6.2 延迟对比

```
定位延迟(从触发到获得位置):
- UWB TWR: <10ms (单次测距<1ms, 3次测距+计算)
- UWB TDoA: <5ms (标签只需发射, 无需等响应)
- BLE RSSI: 1-5秒 (依赖扫描间隔, 需多次采样平均)
- BLE AoA: 100ms-1秒 (依赖广播间隔)
```

UWB 的低延迟使其适合实时跟踪场景(如叉车防碰撞预警要求延迟低于 100ms)，BLE 更适合不要求实时的场景(如资产存在性检测)。

## 7. 应用场景映射

### 7.1 UWB最佳场景

UWB 在以下场景中优势明显:

- 仓库叉车追踪: 需要 30cm 精度实时跟踪叉车位置，防碰撞预警要求低延迟，UWB TDoA 是最佳选择
- 制造业机器人定位: 自动导引车(AGV)路径引导需要 10-20cm 精度，只有 UWB 能满足
- 安全门禁与精确接近检测: 需要准确判断人在门的哪一侧，BLE 精度不够，UWB 的厘米级精度可以可靠区分

### 7.2 BLE最佳场景

BLE 在以下场景中更有优势:

- 零售客流分析: 统计顾客在各区域停留时间，3-5 米精度足够，BLE RSSI 方案成本最低
- 博物馆展品导览: 当游客靠近某展品时推送介绍，BLE 信标只需判断接近而非精确坐标
- 大规模资产存在性检测: 上万个资产标签只需知道"在哪个房间/区域"，BLE 的低成本和长电池寿命至关重要

### 7.3 重叠场景

办公空间利用率监测是一个典型的重叠场景：BLE 可以告诉你"3 楼东区有 15 人"，UWB 可以告诉你"A 会议室座位 3 被占用"。选择取决于你需要区域级还是座位级的粒度。

## 8. 混合部署方案

### 8.1 分区精度策略

在同一建筑中根据不同区域的精度需求混合部署 UWB 和 BLE:

```
混合部署示例(智能仓库):

[高精度区 - UWB]          [低精度区 - BLE]
 装卸区/分拣线              一般存储区/办公区
 需要: <30cm               需要: <3m
 UWB锚点密集部署            BLE信标稀疏部署
      |                         |
      +---------+---------------+
                |
         [定位服务器]
         统一坐标系
         融合UWB+BLE定位结果
```

### 8.2 双模标签

支持 UWB 和 BLE 双模的标签是混合部署的关键。标签在 BLE 覆盖区域使用 BLE 定位，进入 UWB 覆盖区域自动切换到 UWB 高精度模式。这种方案在总成本和精度之间取得最优平衡。

```
双模标签工作逻辑:

开机 --> BLE广播(低功耗)
  |
检测到UWB锚点? -- 否 --> 继续BLE定位(省电)
  |
  是
  |
切换UWB测距(高精度) --> 离开UWB区域 --> 回到BLE
```

### 8.3 BLE辅助UWB唤醒

另一种常见模式是用 BLE 做存在性检测和粗定位，只在需要精确位置时唤醒 UWB 模块。这样可以将 UWB 标签的电池寿命从几个月延长到一年以上。

## 9. 选型决策指南

### 9.1 快速决策矩阵

```
需要精度 <50cm?
  |-- 是 --> UWB (无可替代)
  |-- 否 -->
      需要精度 <1m?
        |-- 是 --> BLE AoA (精度与成本平衡)
        |-- 否 -->
            需要精度 <3m?
              |-- 是 --> BLE RSSI指纹 (最低成本)
              |-- 否 --> BLE RSSI测距 (最简部署)

其他考虑因素:
- 预算极有限 --> BLE RSSI
- 需要手机直接定位 --> BLE(今天), UWB(未来)
- 任务关键型(安全/自动化) --> UWB
- 标签数量 >1000 --> BLE(成本)或UWB TDoA(容量)
- 需要实时跟踪 <100ms --> UWB
```

### 9.2 技术演进趋势

UWB 芯片成本持续下降(从 2020 年的 3-5 美元降至 2024 年的 1.5-2.5 美元)，越来越多手机集成 UWB。BLE 6.0 引入的 Channel Sounding 技术将 BLE 测距精度提升到 10-30cm，可能改变竞争格局。未来 UWB 和 BLE 的边界可能进一步模糊，混合方案将成为主流。

## 总结

UWB 和 BLE 室内定位服务于不同的精度和成本需求。UWB 凭借纳秒脉冲和 ToF 测距实现 10-30cm 的最佳精度，适合仓储物流、工业自动化、安全门禁等高要求场景。BLE 依托低成本信标和手机普及性，以 0.5-5 米的精度覆盖零售分析、资产管理等广泛场景。在实际项目中，根据各区域的精度需求混合部署 UWB 和 BLE，配合双模标签实现成本与精度的最优平衡，是最务实的策略。
