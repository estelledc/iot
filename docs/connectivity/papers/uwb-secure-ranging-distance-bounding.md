# UWB安全测距与距离限界协议
> **难度**：🔴 高级 | **领域**：UWB安全 | **阅读时间**：约 22 分钟

## 引言

假设你家的智能门锁通过无线信号感知钥匙是否在门前。正常情况下, 你拿着手机走到门前, 门锁检测到手机在 1 米内, 自动开门。但如果有人在你家门口放一个小盒子, 在你办公室旁也放一个小盒子, 两个盒子通过网络连接, 把你手机的信号远程中继到门锁前, 门锁就会误以为你就在门口, 然后开门。这就是中继攻击, 传统的 BLE 和 NFC 无钥匙系统几乎无法防御。UWB 安全测距和距离限界协议就是为了从物理层面解决这个问题: 用光速这个不可逾越的物理极限, 证明设备确实在声称的距离内。

本文深入分析中继攻击威胁模型、距离限界的密码学基础、UWB 802.15.4z 的安全机制、FiRa/CCC 规范, 以及实际部署考量。

## 1 安全测距的动机

### 1.1 测距用于访问控制

越来越多的安全系统依赖距离判断来做访问决策: 汽车无钥匙进入(钥匙在 1-2m 内才解锁)、智能门锁(手机靠近自动开门)、非接触支付(设备在终端附近才能交易)、办公门禁(手机在 50cm 内才开门)。这些系统的安全前提是设备真的在物理上靠近。

### 1.2 中继攻击的威胁

```
中继攻击原理:

正常: [车钥匙] ----2m---- [汽车] -> 解锁

攻击:
  [车钥匙]               [汽车]
       |                     |
  [攻击设备A]           [攻击设备B]
       |                     |
       +=== 无线/有线连接 ===+

  1. 汽车发送挑战信号
  2. 设备 B 接收, 转发给设备 A
  3. 设备 A 在钥匙旁重放信号
  4. 钥匙正常响应
  5. 设备 A 把响应转发给设备 B
  6. 设备 B 向汽车重放响应
  7. 汽车收到有效响应 -> 误解锁
```

### 1.3 真实世界案例

中继攻击不是理论威胁: 2019 年英国研究发现大多数 BLE/NFC 无钥匙系统可被中继攻击, 攻击设备成本低至 20-50 美元(使用 SDR), 攻击距离可达数百米, 攻击时间几秒钟。

## 2 距离限界概念

### 2.1 什么是距离限界

距离限界(Distance Bounding)是一种密码学协议, 用来证明物理距离的上界:

```
核心思想:
  没有任何信号能比光速更快
  如果我发出挑战, 对方在时间 T 内响应
  那么对方的距离不超过 c * T / 2
  (c = 光速, 除以 2 因为是往返)
```

### 2.2 距离限界的三个阶段

```
阶段 1: 慢阶段(Slow Phase)
  验证者和证明者交换密码学承诺, 不计时

阶段 2: 快阶段(Fast Phase)
  验证者发送随机比特挑战
  证明者必须立即响应, 严格计时
  响应延迟 = 距离证据

阶段 3: 验证阶段
  检查快阶段响应是否正确
  检查响应时间是否在距离阈值内
```

### 2.3 距离限界与简单测距的区别

| 特性 | 简单测距 | 距离限界 |
|------|---------|---------|
| 密码学保护 | 无 | 有(共享密钥) |
| 防中继攻击 | 不能 | 能 |
| 防距离缩减 | 不能 | 能 |
| 需要预共享密钥 | 不需要 | 需要 |

## 3 UWB的距离限界优势

### 3.1 时间分辨率对比

能否检测中继攻击, 取决于能否测量中继引入的额外延迟:

```
中继设备最小处理延迟:
  模拟中继(放大转发): 约 1-10ns
  数字中继(解调再调制): 约 100ns-1us

各技术时间分辨率:
  BLE (2MHz): 500ns -> 150m 距离分辨率 (无法检测)
  NFC:        约 1us -> 300m 距离分辨率 (无法检测)
  UWB (500MHz): 2ns -> 0.6m 距离分辨率 (可检测)

UWB 可以检测到 0.6m 级别的中继延迟
```

### 3.2 UWB脉冲特性

UWB 使用极短脉冲(不到 2ns), 带来独特优势: 首达路径检测能分辨最早到达的脉冲, 窄脉冲难以被中继(需极高带宽中继设备), 宽频谱特征难以被窄带干扰。

## 4 802.15.4z STS安全机制

### 4.1 STS原理

STS(Scrambled Timestamp Sequence)是 802.15.4z 的核心安全特性:

```
STS 生成:
  输入: 共享密钥 K + 计数器 N
  算法: AES-128
  输出: STS = AES-128(K, N) || AES-128(K, N+1) || ...
  STS 长度: 典型 32-128 符号
  每个符号是 UWB 脉冲, 位置和极性由 STS 比特决定
```

### 4.2 STS在帧中的位置

```
802.15.4z 帧结构(Mode 1):
+------+------+-------------+------+
| SHR  | PHR  | PHY Payload | STS  |
+------+------+-------------+------+
  同步/信道估计   MAC数据     安全测距
```

### 4.3 STS验证流程

```python
def verify_sts(received_signal, shared_key, counter):
    """验证接收到的 STS"""
    # 生成期望 STS
    expected_sts = generate_sts(shared_key, counter)
    # 互相关
    correlation = cross_correlate(received_signal, expected_sts)
    peak_value = max(correlation)
    if peak_value > 0.7:  # 阈值
        toa = peak_index_to_time(argmax(correlation))
        return True, toa  # STS 有效
    return False, None     # 可能是攻击
```

## 5 攻击类型与防御

### 5.1 中继攻击

```
攻击方式:
  攻击者在两端各放置中继设备
  将信号实时转发到另一端
  不需要知道密钥或协议细节

STS 防御:
  中继引入额外延迟(至少几纳秒)
  UWB 2ns 分辨率可检测 0.6m 级延迟
  测量距离 > 实际距离 -> 触发警报
  即使中继很快, STS 时间戳验证能发现异常
```

### 5.2 距离缩减攻击

```
攻击方式:
  攻击者试图让测量距离小于实际距离
  可能通过提前发送响应来实现

STS 防御:
  攻击者不知道密钥 K
  无法预测 STS 内容
  因此无法提前构造有效的 STS 响应
  提前发送的伪造 STS 在验证时失败
```

### 5.3 早期检测攻击

攻击者等待收到部分帧后, 猜测剩余内容并提前发送。STS 由 AES 加密生成具有伪随机性, 知道前 N 个比特不能预测第 N+1 个比特, 任何猜测都会导致 STS 验证失败。

### 5.4 Cicada攻击(高级)

攻击者已获得密钥(如通过社会工程), 在目标附近放置合法 UWB 设备在正确时间发送正确 STS。这不是纯物理层能防的攻击, 需要更高层安全机制: 密钥定期更新和轮换、设备认证(证书链)、异常行为检测(如同一密钥在不同位置同时使用)。

## 6 FiRa联盟

### 6.1 FiRa简介

FiRa(Fine Ranging)Consortium 是推动 UWB 商用的行业联盟, 2019 年成立。核心成员包括 Apple, Samsung, NXP, Qorvo, BMW, Google 等。基于 802.15.4z 定义互操作性规范和安全配置文件。

### 6.2 FiRa安全配置

| 级别 | STS 模式 | 密钥管理 | 适用场景 |
|------|---------|---------|---------|
| 基础 | Mode 0 | 无 | 室内定位(无安全需求) |
| 标准 | Mode 1 | 静态密钥 | 资产追踪 |
| 增强 | Mode 3 | 动态密钥 | 门禁系统 |
| 最高 | Mode 2 | 安全元件 | 汽车钥匙/支付 |

## 7 CCC数字车钥匙

### 7.1 CCC简介

CCC(Car Connectivity Consortium)定义了 Digital Key 3.0 标准, 引入 UWB 安全测距。成员包括 Apple, Samsung, BMW, Audi, VW, Honda 等。

### 7.2 安全测距流程

```
CCC 数字车钥匙流程:

1. BLE 发现: 手机和汽车通过 BLE 发现, 建立安全连接
2. 密钥派生: 从 BLE 密钥派生 UWB STS 密钥, 存入安全元件
3. UWB 测距: DS-TWR + STS Mode 2, 车上 4-6 个锚点同时测距
4. 距离判断:
   - 车外 2m 内 -> 解锁车门
   - 车内驾驶位 -> 允许启动
   - 远离 -> 锁车
5. 验证: STS 通过且距离在阈值内才执行动作
```

### 7.3 与传统系统对比

| 特性 | 传统射频钥匙 | BLE 智能钥匙 | UWB CCC 钥匙 |
|------|------------|-------------|--------------|
| 测距精度 | 无 | 3-5m(RSSI) | 10cm(ToF) |
| 防中继 | 不能 | 不能 | 能(STS) |
| 定位能力 | 无 | 粗略 | 精确 3D |
| 用户体验 | 按键操作 | 自动但不精确 | 自动且精确 |

## 8 安全测距协议流程

### 8.1 完整会话

```
阶段 1: 安全通道建立(BLE)
  交换证书, 建立加密通道, 协商 UWB 参数

阶段 2: 密钥派生
  session_key = KDF(master_key, session_nonce)
  sts_key = KDF(session_key, "STS")

阶段 3: UWB 测距(重复)
  每轮: sts = AES(sts_key, counter++)
  执行 DS-TWR with STS, 验证完整性, 报告距离

阶段 4: 决策
  所有 STS 验证通过 AND 距离 < 阈值 -> 允许
  否则 -> 拒绝
```

### 8.2 密钥刷新

每会话通过 BLE 建立新的 session_key, 每帧通过递增 counter 生成不同 STS(防重放), 主密钥定期轮换并安全删除旧密钥。

## 9 实现考量

### 9.1 安全元件

密钥必须存储在防篡改的安全元件(SE)中: Apple Secure Enclave、Android StrongBox/Titan M、汽车专用安全芯片(如 NXP SE050)。功能包括安全密钥存储、AES 加密运算和防侧信道攻击。

### 9.2 抗重放机制

STS counter 单调递增不允许回退, 接收端记录最后有效 counter 值, 收到更小 counter 的帧直接拒绝。允许小范围乱序(处理丢帧)但不允许大幅回退。

## 10 与其他近距离技术对比

### 10.1 BLE RSSI

```
BLE RSSI 测距:
  原理: 信号强度随距离衰减
  精度: 3-5m(受多径、人体遮挡、环境影响)
  安全性: 信号强度可被欺骗(增大发射功率)
  防中继: 不能
  优势: 成本极低, 几乎所有手机都支持
```

### 10.2 NFC

```
NFC:
  原理: 近场电磁耦合
  有效距离: 约 10cm
  安全性: 极短距离本身提供一定安全保障
  防中继: 有一定保护(中继延迟可能超过协议容忍)
  局限: 必须几乎贴近, 用户体验差
  适合: 支付、门禁(需要刻意贴近的场景)
```

### 10.3 综合对比

| 特性 | BLE RSSI | NFC | UWB 802.15.4z |
|------|----------|-----|---------------|
| 测距精度 | 3-5m | 10cm(有效距离) | 5-10cm |
| 有效范围 | 10-30m | 10cm | 10-50m |
| 防中继 | 不能 | 部分 | 能(STS) |
| 密码学保护 | 可选 | 可选 | 内置(STS) |
| 功耗 | 低 | 极低 | 中等 |
| 芯片成本 | 低(1美元以下) | 低 | 中(2-5美元) |
| 手机支持 | 几乎所有 | 大多数 | 旗舰机(增长中) |

UWB 在精度和安全性上远超 BLE 和 NFC, 是安全访问控制的最佳选择。

## 11 实际部署案例

### 11.1 汽车行业

BMW 从 2022 年起在部分车型部署 UWB 数字车钥匙: 车上 4-6 个锚点, 支持 iPhone 和部分 Android 手机, 走近自动解锁, 坐入驾驶位允许启动, 离开自动锁车。

### 11.2 智能手机

Apple 从 iPhone 11(U1)开始集成 UWB, iPhone 15 升级到 U2。应用包括 AirTag 精确寻物、近距离 AirDrop 和数字车钥匙(CarKey)。

### 11.3 门禁系统

企业和住宅智能门禁引入 UWB: 手机在口袋中走近门自动解锁, 比 NFC 方便(不需掏手机), 比 BLE 安全(防中继), 能精确判断人在门的哪一侧(防跟随进入)。

## 总结

UWB 安全测距与距离限界协议解决了一个根本性问题: 如何在无线环境中证明物理距离的真实性。传统的 BLE 和 NFC 系统缺乏足够的时间分辨率和密码学保护来防御中继攻击, 而 802.15.4z 的 STS 机制将密码学验证嵌入物理层, 利用光速不可超越的物理极限, 构建了可靠的距离证明。

FiRa 联盟和 CCC 将这些底层安全能力封装为可互操作的标准, 推动 UWB 安全测距从实验室走向汽车、手机和门禁等大规模消费市场。随着 UWB 芯片成本持续下降和手机支持率提高, 基于密码学距离限界的安全访问控制将成为物联网安全的标准配置。
