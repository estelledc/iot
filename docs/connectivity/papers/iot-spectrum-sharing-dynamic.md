# IoT动态频谱共享与频谱数据库
> **难度**: 高级 | **领域**: 频谱管理 | **阅读时间**: 约 22 分钟

## 引言

想象一座城市里只有几条公路，但每天有数百万辆车想上路。如果每辆车都被永久分配一条固定车道，大部分时间车道是空的(车主可能在睡觉)，但其他车辆却不能使用这些空闲车道。这就是无线频谱面临的困境：大量频段被授权给特定用户，实际利用率却很低，而ISM公共频段(就像不收费的免费公路)已经被挤得水泄不通。

动态频谱共享(Dynamic Spectrum Sharing, DSS)就是解决这个矛盾的方案——让设备像智能导航系统一样，实时查询哪些"车道"是空闲的，动态借用这些资源，在原主人需要时立即让出。对于IoT来说，这意味着数十亿设备可以获得更多可用频谱，而不需要花费天价购买独占频谱许可。

本文将深入解析动态频谱共享的核心机制、频谱数据库架构、CBRS三层模型、SAS系统，以及欧洲LSA框架，帮助读者理解这项技术如何为IoT开辟新的频谱通道。

## 1. 频谱稀缺问题

### 1.1 ISM频段的拥挤现状

ISM(工业、科学与医疗)频段是IoT设备最常使用的频谱资源。2.4GHz频段承载着WiFi、蓝牙、Zigbee、Thread等多种协议，在密集部署区域已经严重饱和。

```
ISM 2.4GHz 频段占用情况示意:
|---WiFi Ch1---|---WiFi Ch6---|---WiFi Ch11--|
|  BLE Adv    |  Zigbee     |  BLE Adv     |
|  微波炉干扰  |  Thread     |  监控摄像头   |
总带宽: 仅 83.5MHz，承载 6+ 种协议
```

常见ISM频段及其拥挤程度:

| 频段 | 带宽 | 主要协议 | 拥挤程度 |
|------|------|----------|----------|
| 433MHz | 1.74MHz | LoRa, 遥控器 | 中等 |
| 868MHz(欧) | 7MHz | LoRa, Sigfox | 中等 |
| 915MHz(美) | 26MHz | LoRa, Sigfox | 较高 |
| 2.4GHz | 83.5MHz | WiFi/BLE/Zigbee | 极高 |
| 5.8GHz | 150MHz | WiFi 5/6 | 较高 |

### 1.2 授权频段的低利用率

与ISM频段的拥挤形成鲜明对比的是，大量授权频段的实际利用率非常低。研究表明，在任意给定时间和地点，授权频段的平均利用率仅为15%-85%不等，大量"频谱空洞"(spectrum holes)存在。

```
频谱利用率的时空变化:
时间维度:
  电视频段: 白天利用率 ~30%，夜间 ~70%
  军用雷达: 特定演习时段 ~90%，平时 ~5%
空间维度:
  城市中心: 蜂窝频段利用率 ~80%
  郊区农村: 蜂窝频段利用率 ~10%
```

这种"有的撑死、有的饿死"的局面催生了动态频谱共享的需求。

### 1.3 IoT规模化的频谱挑战

到2030年预计将有超过250亿IoT设备连接，每个设备都需要频谱资源进行通信。仅靠现有ISM频段无法满足需求：

- 智慧城市传感器网络需要可靠的远程覆盖
- 工业IoT要求低延迟的确定性通信
- 农业IoT需要广域覆盖但数据量小
- 车联网(V2X)需要低延迟高可靠频谱

动态频谱共享为这些场景提供了获取额外频谱的路径。

## 2. 动态频谱接入的核心概念

### 2.1 什么是动态频谱接入

动态频谱接入(Dynamic Spectrum Access, DSA)是一种允许非授权用户在不干扰授权用户的前提下，机会性地使用空闲频谱的技术框架。

核心原则是"用而不占"：

```
传统静态分配模式:
  频段A -> 用户X (独占，无论是否使用)
  频段B -> 用户Y (独占，无论是否使用)
  IoT设备 -> 只能用ISM频段

动态频谱接入模式:
  频段A -> 用户X优先 -> X不用时 -> IoT设备可以临时使用
  频段B -> 用户Y优先 -> Y不用时 -> IoT设备可以临时使用
  保护机制: X或Y需要时，IoT设备必须立即让出
```

### 2.2 三种频谱接入模型

根据技术实现方式，动态频谱接入分为三种主要模型:

第一种是频谱感知模型(Spectrum Sensing)。设备自己监听频谱环境，判断哪些频段空闲。类似于你在停车场自己转一圈看哪里有空位。优点是不依赖外部基础设施，缺点是感知精度有限，存在隐藏终端问题。

第二种是频谱数据库模型(Spectrum Database)。设备向中央数据库查询可用频谱信息，数据库根据地理位置和时间返回可用频道列表。类似于停车场有一个电子显示屏告诉你每层楼还有几个空位。这是目前最成熟的方案。

第三种是混合模型。结合数据库查询和本地感知，先从数据库获取候选频道，再通过本地感知确认实际可用性。提供最高的可靠性，但复杂度也最高。

### 2.3 频谱数据库的工作原理

频谱数据库(Geolocation Database)是动态频谱共享的核心基础设施:

```
频谱数据库查询流程:

IoT设备                    频谱数据库                  监管机构
  |                           |                         |
  |-- 1.报告位置/设备参数 --->|                         |
  |                           |-- 查询授权用户信息 ----->|
  |                           |<-- 返回保护要求 ---------|
  |                           |                         |
  |                           |[计算可用频道/功率限制]    |
  |                           |                         |
  |<-- 2.返回可用频道列表 ----|                         |
  |    (频道/最大功率/有效期)  |                         |
  |                           |                         |
  |-- 3.在指定频道上通信       |                         |
  |                           |                         |
  |-- 4.定期重新查询 -------->|                         |
```

查询参数通常包括设备类型、GPS坐标、天线高度、设备类别和请求带宽。数据库返回可用频道列表，每个频道包含频率、带宽、最大EIRP功率限制和有效期。设备必须在有效期到期前重新查询以获取最新分配。

## 3. CBRS三层频谱共享模型

### 3.1 CBRS概述

CBRS(Citizens Broadband Radio Service)是美国FCC在3550-3700MHz频段(150MHz带宽)建立的创新性三层频谱共享框架，也是目前最成功的动态频谱共享实践。

### 3.2 三层接入架构

CBRS将用户分为三个优先级层次:

```
CBRS 三层模型 (3550-3700 MHz):

第一层: 现有用户 (Incumbent Access)
+-----------------------------------------------+
| 美国海军雷达、卫星地面站等                       |
| 享有最高保护优先级，不受任何干扰                  |
+-----------------------------------------------+
        |  必须保护  |
        v            v
第二层: 优先接入许可 (Priority Access License, PAL)
+-----------------------------------------------+
| 通过拍卖获得，按县级区域授权                      |
| 3年许可期，可续期                                |
| 获得干扰保护，但必须让位于第一层                  |
+-----------------------------------------------+
        |  机会性接入  |
        v              v
第三层: 通用授权接入 (General Authorized Access, GAA)
+-----------------------------------------------+
| 无需许可证，免费使用                              |
| 不享受干扰保护                                   |
| IoT设备主要使用此层                               |
+-----------------------------------------------+
```

### 3.3 功率限制与覆盖

不同层级的功率限制直接影响覆盖范围:

| 参数 | 类别A(室内) | 类别B(室外) |
|------|------------|------------|
| 最大EIRP | 30 dBm (1W) | 47 dBm (50W) |
| 天线高度 | 室内 | <=6m AGL |
| 典型覆盖 | 50-200m | 1-5km |
| IoT适用性 | 室内传感器 | 室外广域 |

### 3.4 SAS频谱接入系统

SAS(Spectrum Access System)是CBRS的核心管控平台，负责整个频谱共享的协调工作:

```
SAS 系统架构:

                    +------------------+
                    |   FCC 数据库      |
                    | (现有用户注册)     |
                    +--------+---------+
                             |
                    +--------v---------+
                    |    SAS 服务器     |
                    | (频谱分配引擎)    |
                    +--+-----+------+--+
                       |     |      |
              +--------+  +--+--+  +--------+
              |           |     |           |
         +----v----+ +----v----+ +----v----+
         | ESC传感器| | PAL用户 | | GAA设备  |
         | (感知   | | (运营商) | | (IoT)   |
         | 雷达)   | |         | |         |
         +---------+ +---------+ +---------+

ESC: Environmental Sensing Capability
用于检测第一层用户(如海军雷达)的活动
```

SAS的核心功能包括接收设备注册和频谱请求、查询现有用户保护区域、计算干扰模型确定可用频谱、分配频道和功率限制、当第一层用户激活时命令下层用户退避，以及记录所有频谱使用活动。

### 3.5 动态保护区(DPA)

当ESC传感器检测到海军雷达等第一层用户活动时，SAS会动态建立保护区。整个响应流程包括ESC检测雷达信号(10秒内)、SAS计算保护区边界并通知设备(60秒内)、设备完成频道切换或降功率(300秒内)，总响应时间控制在6分钟以内。雷达停止活动后，SAS逐步恢复频谱分配。

## 4. 欧洲的LSA框架

### 4.1 LSA概述

Licensed Shared Access(LSA)是欧洲提出的频谱共享框架，主要针对2.3-2.4GHz频段。与CBRS不同，LSA采用双层模型而非三层。

### 4.2 LSA与CBRS的对比

| 特性 | CBRS (美国) | LSA (欧洲) |
|------|------------|-----------|
| 频段 | 3550-3700 MHz | 2300-2400 MHz |
| 层级数 | 三层 | 两层 |
| 管控系统 | SAS | LSA控制器+LSA仓库 |
| 第二层获取 | 拍卖(PAL) | 个别授权 |
| 免许可层 | GAA(有) | 无 |
| IoT友好度 | 高(GAA免费) | 低(需授权) |
| 成熟度 | 已商用 | 试验阶段 |

### 4.3 LSA架构

LSA系统由三个核心组件构成: LSA仓库(Repository)存储现有用户的使用规则和保护需求; LSA控制器(Controller)查询仓库后做出频谱分配决策; LSA被许可方(如运营商)获得分配的频谱资源。对于IoT来说，LSA的实用性不如CBRS，因为缺少免许可的GAA层级，IoT设备需要通过运营商间接获取LSA频谱资源。

## 5. 频谱感知vs数据库驱动

### 5.1 两种方法的技术对比

频谱感知(Spectrum Sensing)方法让设备自行检测频谱占用情况，而数据库驱动(Database-driven)方法依赖集中式查询:

| 比较维度 | 频谱感知 | 数据库驱动 |
|----------|----------|-----------|
| 基础设施需求 | 低(设备端完成) | 高(需部署数据库) |
| 检测精度 | 受限于灵敏度 | 高(基于注册信息) |
| 隐藏终端 | 存在问题 | 无此问题 |
| 能耗 | 高(持续监听) | 低(周期性查询) |
| 实时性 | 高(本地检测) | 中(取决于查询间隔) |
| IoT适用性 | 差(能耗高) | 好(简单查询) |
| 监管接受度 | 低 | 高(可审计) |

### 5.2 为什么数据库方案胜出

对于IoT设备而言，数据库驱动方案有明显优势:

能耗方面，IoT设备大多使用电池供电，持续进行频谱感知的能耗是不可接受的。周期性查询数据库(例如每小时一次)的能耗远低于连续感知。

复杂度方面，频谱感知需要在设备端集成宽频接收机和信号处理算法，这增加了成本和芯片面积。数据库查询只需要基本的网络连接能力。

可靠性方面，感知检测可能因为隐藏终端、多径衰落等问题产生误判。数据库基于注册信息和传播模型计算，结果更可靠。

### 5.3 混合方案的价值

最佳实践往往结合两种方法:

```
混合方案流程:
1. 设备向数据库查询 -> 获得候选频道列表
2. 设备对候选频道进行快速感知 -> 确认实际可用性
3. 选择最佳频道开始通信
4. 通信过程中持续监测信道质量
5. 定期重新查询数据库更新频道列表

优势: 数据库提供宏观指导，本地感知处理微观变化
```

## 6. IoT动态频谱接入的收益与挑战

### 6.1 核心收益

动态频谱共享为IoT带来的直接好处是显而易见的:

频谱可用性大幅提升——以CBRS为例，150MHz的GAA频谱对IoT完全开放。干扰显著减少——通过SAS/数据库的协调，设备使用经过规划的频谱，避免了ISM频段的无序竞争。成本有效降低——GAA接入无需支付频谱许可费用，对大规模IoT部署节省了巨大的运营成本。

### 6.2 面临的挑战

数据库精度与时效性是首要挑战。频谱数据库依赖准确的传播模型和及时的信息更新。如果数据库中的地形数据不精确，或者现有用户的活动信息更新延迟，可能导致对可用频谱的误判。

查询延迟影响是另一个重要考量。IoT设备在发送数据前需要查询数据库，这引入了额外延迟。对于时延敏感的应用(如工业控制)，这可能是不可接受的。

```
查询延迟分解:
  设备唤醒 + 建立网络连接: 100-500ms
  数据库查询往返: 50-200ms
  SAS计算响应: 10-100ms
  频道切换: 10-50ms
  总额外延迟: 170ms - 850ms

对比:
  NB-IoT典型延迟: 1-10s (可接受)
  工业控制延迟要求: <10ms (不可接受)
```

可用性保障也是挑战之一。GAA用户没有干扰保护权利，当第一层或PAL用户激活时必须退避。对于关键任务IoT应用，这种不确定性需要在系统设计中预留冗余。

设备端实现复杂度方面，IoT设备需要支持SAS协议栈、定位能力、多频段切换等功能，这增加了硬件和软件复杂度。

### 6.3 缓解策略

针对上述挑战，业界采取了多种缓解措施:

```
应对查询延迟:
  - 预缓存: 设备缓存最近的频道分配，在缓存有效期内直接使用
  - 批量查询: 一次查询多个备选频道
  - 异步更新: 在通信空闲时提前刷新频道信息

应对可用性不确定:
  - 备选频道: 始终维护2-3个备选频道
  - 快速切换: 设计支持毫秒级频道切换的射频前端
  - ISM回退: 当所有动态频谱不可用时，回退到ISM频段
```

## 7. 实际案例: 智慧城市传感器通过CBRS GAA接入

### 7.1 场景描述

某城市部署了5000个环境监测传感器(空气质量、噪声、温湿度)，分布在50平方公里的城区范围内。传统方案使用LoRaWAN在ISM 915MHz频段通信，但随着部署密度增加，频段拥挤导致数据包丢失率上升到15%以上。

### 7.2 CBRS GAA方案设计

```
系统架构:

[5000个传感器节点]
    |  CBRS 3.5GHz (GAA)
    v
[200个CBRS小基站] -- 回程网络 --> [城市数据中心]
    |                                    |
    +---- SAS协议 ----> [SAS服务器] <----+
                         (Google/Federated Wireless等)
```

传感器节点采用ARM Cortex-M4主控加CBRS射频模组，设备类别为A类(低功率)，最大EIRP 24dBm，太阳能加锂电池供电，每15分钟上报一次数据。SAS交互由小基站代理完成: 小基站向SAS查询可用频道，获取分配后通知关联节点，节点在分配的频道上进行数据上报，小基站每小时刷新频道分配。

### 7.3 部署效果

将方案实施前后的关键指标进行对比:

| 指标 | LoRaWAN 915MHz | CBRS GAA |
|------|---------------|----------|
| 可用带宽 | 500kHz(受限) | 10-20MHz |
| 数据包丢失率 | 15% | 2% |
| 上行速率 | 5.5kbps | 1Mbps |
| 延迟(平均) | 3-5s | 200-500ms |
| 频谱费用 | 免费(ISM) | 免费(GAA) |
| 覆盖半径 | 3-5km | 500m-1km |
| 所需基站 | 10个网关 | 200个小基站 |

### 7.4 关键经验

基站密度与频谱效率的平衡是最重要的经验。虽然CBRS提供了更大带宽，但由于GAA的功率限制(类别A仅1W)，覆盖半径较LoRaWAN小得多，需要部署更多基站。对于城区密集传感器场景，这是可接受的(基站可以复用路灯杆等基础设施)。

退避机制的应对策略同样关键。在该部署区域，约有3%的时间因海军雷达活动需要退避。系统设计中每个传感器节点都配置了备选频道，退避期间的数据会在本地缓存，退避结束后批量上传，实际数据丢失率几乎为零。

## 8. 未来发展方向

未来发展方向包括四个方面: AI驱动的频谱管理——下一代频谱数据库将集成机器学习模型预测频谱使用模式，主动推送最优频道给设备，减少查询次数和设备能耗。6GHz频段开放——FCC已将6GHz频段开放为免许可使用(WiFi 6E/7)，未来可能引入类似CBRS的共享机制。卫星与地面频谱共享——随着LEO卫星星座的部署，ITU正在制定卫星与地面IoT之间的频谱共存框架。区块链频谱交易——利用去中心化平台允许频谱持有者直接将空闲频谱出租给IoT运营商。

## 总结

动态频谱共享是解决IoT频谱需求与频谱稀缺矛盾的关键技术路径。通过频谱数据库的集中式管控，IoT设备可以安全、有序地接入原本闲置的授权频谱资源。

CBRS三层模型是目前最成功的实践，其GAA层级为IoT提供了免费的150MHz频谱资源，SAS系统保障了不同层级用户之间的和谐共存。欧洲的LSA框架虽然在IoT友好度上不及CBRS，但同样验证了频谱共享的可行性。

对于IoT系统设计者来说，需要权衡频谱共享带来的带宽收益与退避不确定性、查询延迟等代价。在非关键任务场景下，CBRS GAA是极具吸引力的频谱选择。随着AI驱动的频谱管理和更多频段的开放，动态频谱共享将在IoT领域发挥越来越重要的作用。
