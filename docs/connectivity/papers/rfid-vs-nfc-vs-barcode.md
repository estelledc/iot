# RFID/NFC/条码在IoT识别中的对比选型
> **难度**：🟢 初级 | **领域**：自动识别技术 | **阅读时间**：约 18 分钟

## 引言

想象你在图书馆管理10万本书。第一种方法是翻开每本书的封面看书号(类似条码扫描, 逐本查看); 第二种是拿着感应器在书架旁走一遍, 所有书自动"报到"(类似RFID批量读取); 第三种是用手机贴一下借书卡, 确认你是谁(类似NFC身份验证)。这三种方法分别对应了自动识别(AutoID)领域的三大技术: 条码/二维码、RFID和NFC。

每种技术都有自己的"甜蜜区"(sweet spot)。选错了技术, 要么成本过高, 要么功能不够。本文将从原理、性能、成本、安全性等维度对三种技术做系统对比, 帮助你在IoT项目中做出合理的选型决策。

## 1. 自动识别技术概述

### 1.1 什么是AutoID

自动识别(Automatic Identification, AutoID)是指不需要人工手动输入就能识别物品身份的技术总称。在IoT系统中, AutoID是物理世界与数字世界的桥梁: 每个物品都需要一个可以被机器读取的"身份标识"。

三大主流AutoID技术:

```
自动识别技术
  |
  +-- 光学识别
  |     +-- 1D条码 (UPC/EAN/Code 128)
  |     +-- 2D码   (QR Code/Data Matrix)
  |
  +-- 射频识别
        +-- NFC  (13.56 MHz, <10cm)
        +-- HF RFID (13.56 MHz, <1m)
        +-- UHF RFID (860-960 MHz, 1-12m)
        +-- LF RFID (125 kHz, <10cm)
```

### 1.2 选型的核心维度

选择AutoID技术时, 需要在以下维度上权衡:

| 维度 | 关键问题 |
|------|---------|
| 读取距离 | 需要多远? 接触式还是远距离? |
| 读取速度 | 一次一个还是批量? |
| 标签成本 | 大规模部署时单价敏感吗? |
| 读写器成本 | 是否需要专用设备? |
| 数据容量 | 需要存多少信息? |
| 安全性 | 是否需要防伪/加密? |
| 环境适应性 | 脏污/潮湿/金属环境? |

## 2. 条码与二维码

### 2.1 一维条码(1D Barcode)

一维条码是最古老、最普及的AutoID技术。超市收银台的"滴"声就来自条码扫描。

工作原理: 激光或图像传感器读取黑白条纹的宽度组合, 解码为数字或字符。

主要类型:

| 类型 | 数据容量 | 应用 |
|------|---------|------|
| UPC-A | 12位数字 | 北美零售商品 |
| EAN-13 | 13位数字 | 国际零售商品 |
| Code 128 | 约20个字符 | 物流/制造 |
| Code 39 | 约30个字符 | 工业/军事 |

一维条码的优势在于极其成熟和廉价。任何打印机都能打印, 任何智能手机都能扫描。但它的信息容量有限, 且只能标识"一类商品"(如"可口可乐330ml"), 不能标识"具体哪一瓶"。

### 2.2 二维码(QR Code)

二维码是一维条码的升级版, 用二维图案编码, 数据容量大幅提升:

```
QR Code容量:
  数字:     最多 7089 个字符
  字母数字:  最多 4296 个字符
  二进制:   最多 2953 字节
  
对比一维条码:
  UPC-A:    12 位数字
  Code 128: 约 20 个字符

QR Code的容量是一维条码的100倍以上
```

二维码支持纠错编码(Reed-Solomon), 即使部分损坏也能正常扫描。有四个纠错等级: L(7%)、M(15%)、Q(25%)、H(30%)。纠错等级越高, 容错越好, 但容量越小。

二维码的最大优势是"免费读写器": 几乎所有智能手机都能扫描QR码, 不需要任何专用设备。

### 2.3 条码/二维码的局限性

尽管普及率高, 光学识别有几个根本性限制:

必须可视(Line-of-Sight): 扫描器必须"看到"条码才能读取。条码被遮挡、包在箱子里或者面朝下时无法扫描。

逐个扫描: 每次只能读取一个条码, 需要人工对准。批量操作效率很低。

容易损坏: 条码印在标签上, 磨损、污渍、褶皱都会导致无法扫描。

只读: 条码一旦印刷就无法修改内容。如果需要更新信息, 必须打印新标签。

## 3. NFC技术特性

### 3.1 NFC基本参数

NFC(Near Field Communication, 近场通信)工作在13.56MHz频率, 读取距离通常不超过10cm。这个极短的距离不是缺陷, 而是设计特性: 它确保了只有"有意识的贴近"才能触发读写, 提供了天然的安全性。

核心参数:

| 参数 | 数值 |
|------|------|
| 工作频率 | 13.56 MHz |
| 读取距离 | < 10 cm |
| 数据格式 | NDEF(NFC Data Exchange Format) |
| 数据容量 | 通常数百字节, 最大数KB |
| 读写能力 | 可读可写 |
| 标签成本 | $0.10-1.00 |
| 读写器 | 智能手机内置 |

### 3.2 NFC的三种工作模式

NFC支持三种工作模式, 覆盖了不同的使用场景:

```
模式1: 读写器模式(Reader/Writer)
  手机 --> NFC标签
  应用: 读取商品信息, 写入配置数据

模式2: 点对点模式(Peer-to-Peer)
  手机 <--> 手机
  应用: 文件传输, 蓝牙配对引导

模式3: 卡模拟模式(Card Emulation)
  手机模拟NFC卡 --> 读卡器
  应用: 移动支付, 门禁
```

### 3.3 NDEF数据格式

NDEF(NFC Data Exchange Format)是NFC标签中存储数据的标准格式。它定义了一种通用的消息封装方式, 使不同设备和应用都能理解标签中的数据。

```
NDEF消息结构:

+------- NDEF Message -------+
| Record 1: URL              |
|   Type: "U"                |
|   Payload: "example.com"   |
+----------------------------+
| Record 2: Text             |
|   Type: "T"                |
|   Payload: "Hello World"   |
+----------------------------+

常见NDEF记录类型:
  URI:    网址, 手机自动打开浏览器
  Text:   纯文本
  MIME:   任意媒体类型
  Smart Poster: URI + 标题 + 图标
```

### 3.4 NFC安全特性

NFC可以提供较强的安全性:

通信加密: NFC-V和NFC-F支持加密通信, 防止窃听。安全元件(Secure Element): 手机中的独立安全芯片, 存储支付凭证和密钥。通信距离短: 10cm以内的距离使远程窃听极其困难(但并非不可能)。认证机制: NFC标签可以支持挑战-响应认证, 防止伪造。

## 4. UHF RFID技术特性

### 4.1 UHF RFID基本参数

UHF RFID工作在860-960MHz频段, 是三种技术中唯一支持远距离批量读取的。

| 参数 | 数值 |
|------|------|
| 工作频率 | 860-960 MHz |
| 读取距离 | 1-12 m(无源标签) |
| 数据格式 | EPC(Electronic Product Code) |
| 数据容量 | 96-bit EPC + 0-512 bit用户区 |
| 读取速度 | 每秒数百个标签 |
| 标签成本 | $0.05-0.50 |
| 读写器成本 | $500-5000 |

### 4.2 UHF RFID的独特优势

UHF RFID有几个其他技术无法替代的能力:

批量读取: 一个读写器可以在数秒内识别数百个标签。这对仓库盘点、物流分拣等场景至关重要。

无需可视: RF信号可以穿过纸箱、塑料袋、布料等非金属材料。货物不需要拆包就能盘点。

无源运行: 标签没有电池, 理论寿命无限。只要标签物理完整, 就可以一直使用。

远距离读取: 最远可达12米, 物品经过读写器覆盖区域时自动被识别, 不需要人工参与。

### 4.3 UHF RFID的局限

金属和液体: UHF信号被金属反射, 被液体吸收。金属表面的标签需要特殊设计(抗金属标签), 成本更高。

读写器成本: UHF RFID读写器价格在$500-5000, 比手机扫描二维码的"零成本"高得多。

隐私问题: 标签可以被任何兼容读写器远距离读取, 存在隐私风险。虽然Gen2v2增加了安全特性, 但旧标签没有保护。

读取区域控制: 有时读写器会读到不该读的标签(如相邻货架的货物), 需要通过天线设计和功率控制来精确限定读取区域。

## 5. 六维对比分析

### 5.1 读取距离

```
距离谱:

0cm        10cm       50cm       1m         5m        12m
 |----------|----------|----------|----------|----------|
 [条码: 0-50cm, 需可视]
 [NFC: 0-10cm, 有意贴近]
                       [HF RFID: 0-30cm]
                                  [UHF RFID: 1-12m, 自动识别]
```

选型建议: 如果需要"自动感知, 人不参与"的远距离识别, 只有UHF RFID能满足。如果需要"用户有意识地触碰"操作, NFC最合适。如果只需要简单的扫一下, 条码/二维码最经济。

### 5.2 读取速度

| 技术 | 速度 | 场景适配 |
|------|------|---------|
| 1D条码 | 1个/秒, 需对准 | 收银台逐件扫描 |
| QR码 | 1个/秒, 自动识别 | 手机扫码, 电子票务 |
| NFC | <1秒/次 | 门禁, 支付, 快速验证 |
| UHF RFID | 200-1000个/秒 | 批量盘点, 流水线识别 |

### 5.3 标签与系统成本

```
成本结构对比(管理10万个物品):

技术      标签单价     10万标签    读写器    总系统成本
-----------------------------------------------------------
条码      约$0.01     $1,000     $200      $1,200
QR码      约$0.01     $1,000     $0(手机)   $1,000
NFC       约$0.30     $30,000    $0(手机)   $30,000
UHF RFID  约$0.08     $8,000     $5,000    $13,000

注意: 这里没有计算人工成本。如果算上人工:
  条码手动盘点10万件: 约200人时 * $15/时 = $3,000/次
  RFID自动盘点10万件: 约10人时 * $15/时 = $150/次
```

当操作频次高、人工成本贵时, RFID的高初始投资会被运营成本节省所弥补。

### 5.4 数据容量

| 技术 | 容量 | 足够存什么 |
|------|------|-----------|
| 1D条码 | 20-30字符 | 商品编号 |
| QR码 | 最多4296字符 | URL + 描述 + 序列号 |
| NFC | 数百字节到数KB | 配置数据, 网址, 证书 |
| UHF RFID EPC | 12字节EPC + 64字节用户区 | 唯一编号 + 少量元数据 |

如果需要在标签中存储丰富的信息(如产品说明、使用手册URL), QR码和NFC是更好的选择。如果只需要一个唯一ID, 然后通过网络查询详细信息, UHF RFID的96-bit EPC足够了。

### 5.5 环境适应性

| 环境因素 | 条码 | NFC | UHF RFID |
|---------|------|-----|---------|
| 脏污/灰尘 | 严重影响 | 不影响 | 不影响 |
| 潮湿/水 | 可能损坏标签 | 不影响 | 信号衰减 |
| 金属表面 | 不影响(可视即可) | 影响(需铁氧体) | 严重影响(需特殊标签) |
| 高温 | 标签可能变色 | 芯片可耐受 | 特殊标签可耐高温 |
| 遮挡/包装内 | 无法读取 | 薄材料可穿过 | 非金属可穿过 |

在工业环境(油污、灰尘、潮湿)中, 射频技术(NFC/RFID)比光学技术(条码)更可靠。但在金属密集环境中, 所有技术都面临挑战。

### 5.6 安全性

```
安全等级:

条码/QR码:  [低]  -- 可以拍照复制, 无加密
NFC:        [高]  -- 支持AES加密, 安全元件, 适合支付
UHF RFID:   [中]  -- Gen2基础版弱, Gen2v2支持加密认证

适合的安全场景:
  防伪验证:  NFC(加密认证) 或 UHF RFID Gen2v2
  支付:      NFC(安全元件 + 加密)
  物流追踪:  UHF RFID(不需要强安全)
  票务:      QR码(动态生成 + 时效性) 或 NFC
```

## 6. 应用场景选型指南

### 6.1 零售与商超

传统零售: 条码仍是主流。成熟的生态系统, 极低的标签成本, 全球标准化。改变的动力不足。

高端零售(服装): UHF RFID正在快速替代条码。服装的高单价可以承受RFID标签成本, 而RFID带来的库存准确性和防损效果显著。

无人商店: QR码(手机扫码支付)和RFID(商品识别)结合使用。

### 6.2 门禁与访问控制

首选NFC。短距离确保有意操作, 加密认证防止伪造, 手机可以作为虚拟门卡。传统方案使用HF RFID卡(如Mifare), NFC完全兼容。

### 6.3 供应链与物流

首选UHF RFID。从制造到零售的全链条追踪, 批量读取满足高吞吐需求, 无源标签降低大规模部署成本。托盘、箱子、单品三个层级都可以贴标签。

### 6.4 产品防伪

NFC是最佳选择。加密认证使标签无法被复制, 消费者用手机即可验证真伪, 不需要任何专用设备。适合药品、奢侈品、电子产品等高价值商品。

### 6.5 图书馆

HF RFID(13.56MHz)是行业标准。每本书贴一个HF标签, 支持自助借还和快速盘点。虽然距离不如UHF, 但图书馆场景下不需要远距离读取, HF的抗金属性能更好(书架通常是金属的)。

### 6.6 快速决策流程

```
选型决策树:

需要批量自动读取?
  |
  +-- 是 --> UHF RFID
  |
  +-- 否 --> 需要安全/加密?
                |
                +-- 是 --> NFC
                |
                +-- 否 --> 需要手机作为读写器?
                             |
                             +-- 是 --> QR码 或 NFC
                             |
                             +-- 否 --> 条码(最低成本)
```

## 7. 多技术融合方案

### 7.1 为什么需要融合

很多实际系统不只使用一种AutoID技术, 而是组合使用:

供应链: 制造端用UHF RFID追踪, 消费端用NFC防伪验证, 包装上印QR码供消费者扫描获取信息。

零售: 商品贴UHF RFID标签用于库存管理, 同时保留条码用于收银(兼容现有POS系统), 高价值商品额外加NFC标签防伪。

### 7.2 融合标签

一些先进的标签产品集成了多种技术:

双频标签: 同一个标签集成UHF RFID和NFC功能, 供应链侧用UHF批量读取, 消费者侧用NFC手机交互。

RFID + 二维码: 标签面上印有QR码, 内部嵌入RFID芯片。既支持手机扫码, 也支持RFID自动读取。

这种融合方案的成本高于单一技术, 但在需要覆盖多个使用场景时, 综合成本往往低于部署多套独立系统。

## 8. 技术发展趋势

### 8.1 条码/二维码的演进

视觉码并没有被射频技术淘汰, 反而在某些方向持续进化。动态QR码(内容可通过服务器端更新)、彩色QR码(更高数据密度)、隐形条码(用UV或IR油墨印刷, 不影响产品外观)都是活跃的研究方向。

### 8.2 NFC的扩展

NFC正在从"读标签"扩展到"充电+通信"一体化(NFC WLC), 以及更深入的IoT设备配置和管理场景。手机作为NFC读写器的普及使NFC成为消费者最容易触达的射频技术。

### 8.3 UHF RFID的降本增效

UHF RFID标签的价格持续下降, 已经接近$0.03的目标价(大批量)。同时, 单芯片面积不断缩小, 灵敏度不断提高, 读取距离和可靠性持续改善。RAIN RFID联盟预测, 到2030年全球将有超过1万亿个物品被RFID标记。

## 总结

条码/二维码、NFC和UHF RFID三种AutoID技术各有所长, 不存在"一种技术替代所有"的情况。条码以近乎零成本的标签和手机即可扫描的便利性占据着最广泛的市场。NFC以安全性和"一贴即用"的交互体验在支付、门禁、防伪领域不可替代。UHF RFID以远距离批量读取的独特能力主导着供应链和资产管理市场。

选型的核心原则是: 从业务需求出发, 而非从技术特性出发。先回答"我需要在什么距离、以什么速度、读取多少物品", 再对照各技术的能力边界, 答案往往自然浮现。在复杂场景中, 多技术融合是值得认真考虑的方案。
