# DALI数字照明控制在IoT智能照明中的应用
> **难度**：🟢 初级 | **领域**：照明控制 | **阅读时间**：约 18 分钟

## 引言

回忆一下老式台灯上的旋钮调光器。你转动旋钮,灯泡亮度随之变化,这是最简单的模拟调光。但想象一下,如果一栋办公楼有 500 盏灯,你想让靠窗的灯暗一些(因为有自然光),中间的亮一些,会议室的灯在无人时自动关掉——用旋钮一个一个调显然不现实。

DALI(数字可寻址照明接口)就是为了解决这个问题而生的。它给每一盏灯分配一个"门牌号",让你可以通过数字指令单独控制任何一盏灯的亮度、色温甚至颜色。就像从"群发短信"升级到了"逐个@某人",精准且高效。

## 1. DALI标准概述

### 1.1 从模拟到数字的演进

在 DALI 出现之前,商业照明调光主要使用 1-10V 模拟接口。控制器输出 1V 到 10V 的电压,灯具根据电压值调节亮度。这种方式简单但有明显缺陷: 没有地址概念,一根控制线上的所有灯只能同升同降; 单向通信,控制器无法知道灯具是否正常工作; 每个控制区域需要独立的控制线。

DALI(IEC 62386标准)彻底改变了这一局面。它是专为照明设计的数字通信协议,实现了对每一盏灯的独立寻址和双向通信。

### 1.2 标准化组织

DALI 由 DiiA(Digital Illumination Interface Alliance,数字照明接口联盟)负责标准的推广和认证。IEC 62386 标准分为多个部分,涵盖了从物理层到应用层的完整定义。通过 DiiA 认证的产品保证了不同制造商之间的互操作性。

## 2. DALI总线基础

### 2.1 两线制总线

DALI 使用一条专用的两线总线,与电力线(强电)分开布设。这两根线没有极性要求,安装时无需区分正负,降低了施工难度。

总线的关键参数如下:

| 参数 | 数值 |
|------|------|
| 最大设备数 | 64个(每条总线) |
| 地址范围 | 0-63(短地址) |
| 通信速率 | 1200 bps |
| 通信方式 | 半双工,主从式 |
| 总线电压 | 约16V(空闲态) |
| 最大总线长度 | 300m(取决于线径) |

### 2.2 设备寻址

每条 DALI 总线上的每个设备都有一个唯一的短地址(0-63),因此最多支持 64 个设备。这个地址在调试阶段分配,可以手动指定,也可以自动分配。

### 2.3 双向通信

DALI 的双向通信能力是相对于 1-10V 单向控制的重大进步。控制器可以向灯具发送命令,灯具也可以回报自己的状态:

```
控制器 ---> 灯具: "请将亮度设为 80%"
灯具 ---> 控制器: "当前亮度 80%, 灯具正常"

控制器 ---> 灯具: "报告灯具状态"
灯具 ---> 控制器: "灯具故障, 需要更换"
```

这使得远程诊断和预测性维护成为可能。

## 3. DALI命令体系

### 3.1 直接亮度控制

最基本的命令是直接设置亮度等级。DALI 使用 0-254 的数值范围:

- 0: 熄灭(OFF)
- 1: 最低亮度(约 0.1%)
- 254: 最大亮度(100%)

DALI 采用对数调光曲线,这意味着数值的每一步变化对应的视觉亮度变化是均匀的,符合人眼对光线的感知特性。

### 3.2 组命令

除了单独控制每个灯具,DALI 还支持组(Group)概念。一个灯具可以属于多个组(最多 16 个组)。向组发送命令时,组内所有灯具同时响应:

```
# 概念示例
发送到组 3: "亮度 = 200"
# 组 3 中的所有灯具同时调至对应亮度
```

这在实际应用中非常实用,比如"窗边组"、"走道组"、"会议桌上方组"等。

### 3.3 场景命令

DALI 支持 16 个预设场景(Scene 0-15)。每个灯具可以为每个场景存储一个亮度值。调用场景时,所有灯具同时切换到各自在该场景中的预设亮度:

| 场景 | 灯具A | 灯具B | 灯具C |
|------|-------|-------|-------|
| 0-工作 | 254 | 254 | 200 |
| 1-演示 | 0 | 0 | 100 |
| 2-休息 | 120 | 120 | 80 |
| 3-清洁 | 254 | 254 | 254 |

### 3.4 查询命令

控制器可以查询灯具的各种状态信息:

- 当前实际亮度等级
- 灯具是否故障
- 灯具工作小时数
- 电源状态
- 调光曲线类型

## 4. DALI-2增强特性

### 4.1 从DALI到DALI-2

DALI-2 是 DALI 标准的重大更新,在保持向下兼容的同时引入了多项增强:

输入设备标准化是 DALI-2 最重要的改进之一。在 DALI-1 中,只有灯具(输出设备)被标准化。按钮、传感器等输入设备的行为由各厂家自定义,导致互操作性问题。DALI-2 将按钮开关、运动传感器、光照传感器等输入设备也纳入标准,它们直接连接在 DALI 总线上。

### 4.2 色彩控制(设备类型8)

DALI-2 引入了 Device Type 8(DT8),支持高级色彩控制:

- Tc(色温控制): 调节暖白到冷白,如 2700K 到 6500K
- RGBWAF: 全彩控制,独立控制红、绿、蓝、白、琥珀等通道
- xy 色度坐标: 精确的色彩指定

这为人因照明(Human-Centric Lighting)提供了技术基础。

### 4.3 互操作性测试

DALI-2 引入了更严格的互操作性测试程序。通过 DiiA 的 DALI-2 认证的产品,在不同制造商之间的兼容性远优于 DALI-1 时代。

## 5. DALI与IoT的集成

### 5.1 DALI网关

DALI 网关是连接 DALI 总线与 IP 网络的桥梁。它将 DALI 命令转换为标准的网络协议(如 BACnet/IP、Modbus TCP、REST API),使得:

- 楼宇管理系统(BMS)可以集中管理照明
- 云平台可以收集照明数据
- 手机 App 可以远程控制

### 5.2 数据上云

通过 DALI 网关连接到 IoT 云平台后,每一盏灯的运行数据都可以被收集和分析:

```
[DALI灯具] --DALI总线--> [DALI网关] --以太网--> [云平台]

上传的数据包括:
- 每盏灯的开关状态和亮度
- 累计工作小时数
- 故障报警
- 占空传感器的触发记录
- 光照传感器的环境光数据
```

### 5.3 智慧照明应用场景

基于 DALI 和 IoT 的融合,可以实现多种智慧应用:

人因照明(Human-Centric Lighting)根据一天中的时间自动调节光的色温和亮度。早晨使用冷白光(5000K以上)提神,下午使用中性白光(4000K),傍晚切换到暖白光(3000K以下)放松。DALI-2 的 DT8 色温控制让这一切可以精确实现。

基于占空感应的控制: 运动传感器检测区域是否有人,无人时自动调暗或关灯,有人进入时恢复到设定亮度。

日光采集(Daylight Harvesting): 光照传感器监测自然光照度,自动调节人工照明,使总照度保持在目标值。靠窗区域白天几乎不需要人工照明。

## 6. DALI与其他照明控制方式的对比

### 6.1 三种主流方式

| 特性 | 1-10V模拟 | DALI | DMX512 |
|------|-----------|------|--------|
| 应用领域 | 简单商业照明 | 商业/建筑照明 | 舞台/娱乐照明 |
| 寻址能力 | 无(按区域) | 64个独立地址 | 512个通道 |
| 通信方向 | 单向 | 双向 | 单向 |
| 调光精度 | 连续模拟 | 254级(对数) | 256级(线性) |
| 状态反馈 | 无 | 有(故障/状态) | 无 |
| 标准 | 无统一标准 | IEC 62386 | ANSI E1.11 |
| 最适合 | 成本敏感/简单项目 | 专业建筑照明 | 演出/动态灯光 |

### 6.2 DALI的独特位置

DALI 在建筑照明领域的独特价值在于: 它既有足够的精细度(逐灯控制),又有适当的复杂度(不像 DMX 那样需要专业灯光师),同时具备双向通信能力,非常适合需要长期运维的建筑照明系统。

## 7. DALI网络拓扑

### 7.1 单总线架构

一条 DALI 总线最多支持 64 个设备。对于小型项目(如一层办公室的部分区域),一条总线可能就够了。总线采用自由拓扑,可以是星形、树形、链形,或者混合拓扑,布线灵活。

### 7.2 多总线架构

大型建筑通常需要多条 DALI 总线。DALI-2 引入了"应用控制器"(Application Controller)的概念,一个控制器可以管理多条总线:

```
DALI 应用控制器
  |--- DALI 总线 1 (东区, 60盏灯)
  |--- DALI 总线 2 (西区, 55盏灯)
  |--- DALI 总线 3 (走廊, 40盏灯)
  |--- DALI 总线 4 (会议室区, 30盏灯)
  |
  |--> 以太网 --> BMS / IoT 云平台
```

### 7.3 典型分区策略

实际项目中,通常按照楼层或功能区域划分 DALI 总线:

- 每层楼或每个大开间一条总线
- 特殊区域(如会议室群、大堂)单独一条总线
- 应急照明独立总线(满足消防法规要求)

## 8. DALI IoT节能效益

### 8.1 节能手段与效果

基于 DALI 和 IoT 的智能照明系统可以通过多种手段实现显著的能源节约:

占空感应控制是最直接的节能手段。通过在 DALI 总线上的运动传感器,系统可以在区域无人时自动将灯光调暗到 10% 甚至完全关闭。典型的开放办公区域通过占空感应可以节省 30% 到 50% 的照明能耗。

日光采集是另一个重要的节能途径。DALI 总线上的光照传感器持续监测自然光照度,控制器据此自动调节人工照明的亮度,使工作面总照度始终维持在目标值(如500 lux)。在日照充足的区域,这可以额外节省 20% 到 40% 的能耗。

时间表控制确保在非工作时间(如夜间、周末)只保留最低限度的安保照明。

综合使用以上手段,相比传统的"全亮或全灭"控制方式,DALI IoT 照明系统通常可以实现 50% 到 70% 的总能耗降低。

### 8.2 逐灯能耗监测

DALI 的双向通信使得系统可以监测每一盏灯的运行状态和能耗。IoT 云平台可以生成详细的能耗报告:

- 按区域/楼层/总线的能耗统计
- 异常高耗设备的识别
- 照明使用模式分析
- 与历史数据的趋势对比

## 9. DALI调试与维护

### 9.1 传统调试方式

早期的 DALI 系统调试需要现场使用专用工具逐一为灯具分配地址。调试人员站在灯下,用手持控制器闪烁某盏灯以确认身份,然后手动指定地址。对于数百盏灯的大型项目,这是一项耗时耗力的工作。

### 9.2 现代自动化调试

现代 DALI-2 系统支持更高效的调试流程。自动地址分配功能可以自动为总线上的所有设备分配不重复的短地址。结合楼层平面图映射工具,调试人员可以在软件中将每个地址与平面图上的灯位一一对应。

### 9.3 IoT远程运维

通过 DALI 网关和 IoT 云连接,许多维护工作可以远程完成:

- 远程修改灯光场景和策略,无需到现场
- 灯具故障自动报警,维护团队提前准备替换件
- 灯具寿命预测: 根据累计工作小时数预判需更换的灯具
- 软件更新可远程推送到 DALI 控制器

## 10. 实际案例: 大型开放办公室

### 10.1 项目规模

一间 500 平方米的开放办公室,采用 DALI-2 智能照明系统:

| 设备类型 | 数量 |
|----------|------|
| DALI-2 LED 灯具 | 120 盏 |
| 占空/日光传感器 | 30 个 |
| DALI 总线 | 2 条 |
| DALI 网关 | 1 台 |
| 应用控制器 | 1 台 |

### 10.2 系统功能

系统实现了以下功能:

自动人因照明: 早上 8 点色温 5000K,中午 4500K,下午 4000K,傍晚 3000K,模拟自然光节律。

智能占空控制: 传感器划分为 15 个控制区域,每个区域独立检测。无人区域 10 分钟后自动调暗至 20%,30 分钟后完全关闭。

日光采集: 靠窗两排灯具根据外部光照自动调节,晴天时亮度降至 30% 以下。

### 10.3 节能与运维成果

与该办公室之前使用的传统荧光灯相比:

- 照明能耗降低 60%
- 灯具维护从被动换灯改为预测性维护,维护效率提升
- IoT 云平台仪表盘实时显示各区域能耗和设备状态
- 维护人员可以远程调整照明策略,减少现场服务次数
- 灯具接近寿命终点时系统自动发出更换提醒

## 总结

DALI 是商业建筑照明控制的事实标准。它的核心价值在于: 逐灯寻址实现精细控制,双向通信支持状态反馈和远程诊断,标准化保证多厂商互操作。

DALI-2 的推出进一步增强了 DALI 的 IoT 能力,将传感器纳入标准化体系,支持色温控制等人因照明需求。通过 DALI 网关接入 IoT 云平台后,DALI 系统从一个本地照明控制总线升级为云端智慧照明管理系统,能够实现远程运维、能耗分析和预测性维护。

对于初学者而言,理解 DALI 的关键是两点: 每盏灯有独立地址(可以精确控制),总线是双向的(灯可以"回话")。这两个特性是 DALI 区别于传统模拟调光的根本所在,也是 DALI 能够融入 IoT 生态的技术基础。

## 附录: DALI常用术语

| 术语 | 含义 |
|------|------|
| Short Address | 短地址,0-63,每个灯具唯一 |
| Group | 组,最多16个,灯具可属于多组 |
| Scene | 场景,最多16个,预存亮度组合 |
| Arc Power | 弧光功率,即灯具亮度等级 |
| Fade Time | 渐变时间,亮度变化的过渡时长 |
| DALI-2 DT8 | 设备类型8,支持色温和全彩控制 |
| Application Controller | 应用控制器,管理多条DALI总线 |
| Commissioning | 调试,为灯具分配地址和配置参数 |
| DiiA | Digital Illumination Interface Alliance |
| BMS | Building Management System,楼宇管理系统 |
