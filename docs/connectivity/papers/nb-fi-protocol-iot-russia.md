# NB-Fi协议在IoT中的技术特点分析
> **难度**: 中级 | **领域**: LPWAN协议 | **阅读时间**: 约 20 分钟

## 引言

想象在俄罗斯西伯利亚的广袤土地上,城市之间隔着上百公里的荒野。在这样的环境中部署IoT传感器网络(比如监测输油管道或远程抄表),面临核心矛盾: 基站数量有限(地广人稀建站贵),但传感器需要可靠地把数据传回来。

NB-Fi(Narrow Band Fidelity)就是在这种背景下诞生的LPWAN协议。由俄罗斯公司WAVIoT开发,通过极窄带调制(每个信道只有100Hz宽),NB-Fi能在30公里甚至更远的距离上可靠传输,用很少的基站覆盖很大的区域。

## 1. NB-Fi技术背景

### 1.1 开发公司

NB-Fi由俄罗斯WAVIoT公司开发,成立于2013年,总部位于莫斯科,专注于大规模IoT连接方案。核心目标是为俄罗斯和独联体(CIS)国家提供适合其地理特点的LPWAN技术--这些地区幅员辽阔、人口稀疏,对通信距离的要求远超欧洲和亚洲典型场景。

### 1.2 在全球LPWAN格局中的定位

| 技术 | 主导地区 | 主推方 | 技术路线 |
|------|---------|--------|---------|
| LoRaWAN | 全球 | Semtech | 扩频(CSS) |
| Sigfox | 欧洲/全球 | Sigfox(已重组) | 超窄带(UNB) |
| NB-IoT | 全球 | 3GPP/运营商 | 蜂窝(授权频段) |
| NB-Fi | 俄罗斯/CIS | WAVIoT | 超窄带(UNB) |

NB-Fi在技术路线上更接近Sigfox(都是超窄带方案),但在市场上是"俄罗斯本土替代方案"。

## 2. 物理层技术特性

### 2.1 超窄带调制

NB-Fi最显著的特征是极窄的信道带宽:

```
信道带宽对比:
  NB-Fi:   ~100Hz/信道   |=|
  Sigfox:  ~100Hz/信道   |=|
  LoRa:    125kHz/信道   |====================================|
  NB-IoT:  180kHz/信道   |===============================================|
```

超窄带的物理学优势在于接收灵敏度。灵敏度与带宽的对数成正比:

```
灵敏度计算(简化):
  热噪声功率 = kTB (k=玻尔兹曼常数, T=温度, B=带宽)
  带宽越窄, 噪声功率越低, 接收灵敏度越高

  100Hz vs 125kHz: 10*log10(125000/100) = 约31dB
  实际增益约12dB(考虑编码增益差异等因素)
  12dB链路增益 -> 距离增加约2-4倍
```

### 2.2 调制方式

NB-Fi使用DBPSK(差分二进制相移键控):

```
DBPSK调制原理:
  数据"0": 相位不变
  数据"1": 相位翻转180度

  发送序列: 1 0 1 1 0
  相位变化: 翻 不变 翻 翻 不变

优点:
  - 解调简单: 只需比较前后符号相位差
  - 对频率偏移容忍度高: 低成本晶振即可
  - 功耗低: 发射电路简单
```

### 2.3 关键物理层参数

| 参数 | 数值 | 说明 |
|------|------|------|
| 工作频段 | 868MHz(俄罗斯/CIS) | Sub-GHz ISM频段 |
| 信道带宽 | 约100Hz | 极窄带 |
| 调制方式 | DBPSK | 差分二进制相移键控 |
| 上行数据速率 | 50-400bps | 根据模式可调 |
| 下行数据速率 | 400-3200bps | 下行可更快 |
| 最大发射功率 | 25mW(14dBm) | 受法规限制 |
| 接收灵敏度 | -148dBm(理论) | 极窄带优势 |
| 通信距离 | 10-30km(典型) | 视地形而定 |

## 3. 协议栈结构

### 3.1 分层架构

```
+---------------------------+
| 应用层 (用户数据, 最大240字节) |
+---------------------------+
| 传输层 (分片/重组, 可靠传输)   |
+---------------------------+
| 网络层 (设备寻址, 路由)       |
+---------------------------+
| MAC层 (信道接入, ALOHA协议)   |
+---------------------------+
| PHY层 (DBPSK调制, 100Hz信道)  |
+---------------------------+
```

### 3.2 MAC层: ALOHA接入

NB-Fi使用ALOHA协议: 设备想发送数据时直接发送,不做信道侦听。

```
为什么用ALOHA而非CSMA?

  CSMA(先听再发): 发送前侦听信道是否忙
  ALOHA(直接发): 想发就发, 不侦听

  NB-Fi选ALOHA的三个理由:
  1. 超窄带信号难以有效侦听(100Hz太窄)
  2. ALOHA更简单, 终端功耗更低
  3. 可用信道多: 600kHz/100Hz = 约6000个信道
     随机选信道后冲突概率本身就低
```

### 3.3 双向通信

与Sigfox最初只支持上行不同,NB-Fi从设计之初就支持双向通信:

```
NB-Fi双向通信模式:

  上行(UP):
    带宽: 100Hz(超窄带)
    速率: 50-400bps
    用途: 传感器数据上报

  下行(DOWN):
    带宽: 数kHz(稍宽)
    速率: 400-3200bps
    用途: 配置更新/固件OTA/确认应答

  时序:
    设备发送上行 -> 打开接收窗口(~2秒)
    -> 基站在窗口内发送下行
    -> 超时则关闭接收器省电
```

### 3.4 数据帧格式

```
NB-Fi上行帧:
+----------+--------+--------+---------+--------+
| 前导码   | 同步字 | 设备ID | 载荷    | CRC    |
| (~500ms) | (4B)   | (4B)   | (0-240B)| (2B)   |
+----------+--------+--------+---------+--------+
典型10字节载荷: 发送时间约2-5秒
```

## 4. 网络架构

### 4.1 星型拓扑

```
[终端设备群] --无线--> [基站] --互联网/卫星--> [云平台]

每个基站覆盖半径: 10-30km
每个基站容量: 数万终端设备
```

### 4.2 基站设计

NB-Fi基站需要宽带接收和并行解调能力:

```
NB-Fi基站接收架构:

  [天线] -> [LNA低噪声放大] -> [宽带ADC采样]
    -> [DSP/FPGA: FFT频谱分析]
    -> [将600kHz分解为~6000个100Hz信道]
    -> [并行解调多个设备信号]
    -> [解码输出到云平台]

  架构与Sigfox基站非常类似
```

## 5. 与Sigfox的详细对比

### 5.1 技术相似性

NB-Fi和Sigfox都属于超窄带(UNB)LPWAN: 信道带宽约100Hz、DBPSK调制、Sub-GHz ISM频段。NB-Fi常被称为"俄罗斯版Sigfox"。

### 5.2 关键差异

| 特性 | NB-Fi | Sigfox |
|------|-------|--------|
| 双向通信 | 原生支持,下行较强 | 早期几乎只有上行 |
| 最大载荷 | 240字节 | 12字节(上行)/8字节(下行) |
| 每日消息限制 | 无硬性限制(受占空比约束) | 140条上行/4条下行 |
| 网络模式 | 可私有部署 | 仅公共网络 |
| 覆盖范围 | 俄罗斯/CIS为主 | 全球72国(巅峰期) |
| 商业模式 | 卖硬件+平台 | 运营网络卖订阅 |
| 当前状态 | 持续运营 | 2022年破产被UnaBiz收购 |

NB-Fi相对Sigfox的核心优势:

```
NB-Fi vs Sigfox 核心差异详解:

1. 数据载荷: 240B vs 12B
   - NB-Fi一次可发送多个读数+元数据
   - Sigfox 12字节只能发一个简单读数

2. 双向通信
   - NB-Fi下行支持远程配置和OTA
   - Sigfox下行极其有限(每天最多4条)

3. 网络模式
   - NB-Fi可私有部署, 不依赖公共网络覆盖
   - Sigfox仅公共网络, 必须有覆盖

4. 消息限制
   - NB-Fi无人工消息数量限制(仅受占空比约束)
   - Sigfox每天最多140条上行
```

## 6. 与LoRaWAN的对比

NB-Fi和LoRa代表了LPWAN的两种不同技术路线: LoRa用扩频把窄带信号扩展到宽带(125kHz)换取灵敏度和抗干扰; NB-Fi用窄带把信号压缩到极窄带(100Hz)换取灵敏度。

| 特性 | NB-Fi | LoRaWAN |
|------|-------|---------|
| 信道带宽 | ~100Hz | 125-500kHz |
| 数据速率 | 50-400bps | 300-50000bps |
| 灵敏度 | -148dBm | -137dBm(SF12) |
| 典型距离 | 10-30km | 2-15km |
| 抗干扰 | 中等(窄带易受窄带干扰) | 强(扩频增益) |
| 双向通信 | 支持 | 支持(Class A/B/C) |
| 私有部署 | 支持 | 支持 |
| 生态成熟度 | 低(区域性) | 高(全球化) |

```
NB-Fi vs LoRa: "用速率换距离"的trade-off:

  NB-Fi比LoRa多出约11dB链路预算
  -> 自由空间中距离增加约3.5倍

  代价:
  - 数据速率远低于LoRa (400bps vs 50kbps峰值)
  - 抗干扰能力不如扩频方案
  - 同频窄带干扰难以规避

  适用场景:
  NB-Fi: 数据量小+距离要求极高 (如俄罗斯远程抄表)
  LoRa: 数据量中等+距离适中 (如城市IoT)
```

## 7. 生态系统现状

### 7.1 部署地区

主要在俄罗斯(核心市场,覆盖50+城市,用于公用事业抄表和管道监测)和独联体国家(哈萨克斯坦、白俄罗斯、乌兹别克斯坦)。全球扩展受地缘政治影响有限。

### 7.2 硬件生态

| 类别 | 产品 | 特点 |
|------|------|------|
| 终端模块 | WAVIoT NB-Fi模组 | SPI/UART接口 |
| 智能水表 | WAVIoT智能水表 | 内置NB-Fi,电池10年 |
| 智能电表 | WAVIoT智能电表 | 双向通信,远程拉合闸 |
| 基站 | WAVIoT网关 | 室内/室外型号 |
| 平台 | WAVIoT Platform | 云端管理 |

### 7.3 生态局限

LoRaWAN有10+芯片供应商、50+模块厂商、数万人开发者社区和500+成员联盟。NB-Fi基本只有WAVIoT自己的芯片/模块/网关,小型俄语圈社区。这种"生态孤岛"是NB-Fi面临的最大非技术挑战。

## 8. 技术优势与局限总结

### 8.1 核心优势

```
NB-Fi核心优势量化:

  超远通信距离: 10-30km
    一个基站覆盖面积可达LoRa的4-9倍
    例: 20km半径基站覆盖~1256km2

  超低功耗:
    极窄带发射简单, 深度睡眠<1uA
    电池寿命10-15年

  大容量:
    6000个100Hz信道 + ALOHA接入
    单基站十万级终端

  双向通信:
    相比Sigfox, 下行能力更强
    支持OTA和远程配置
```

### 8.2 核心局限

```
NB-Fi核心局限:

  超低数据速率: 50-400bps
    10KB固件OTA需要~200秒(不含重传)
    不适合数据量较大的应用

  抗干扰能力有限:
    窄带信号对同频窄带干扰敏感
    不如扩频方案(LoRa)的处理增益

  生态封闭:
    硬件和平台由WAVIoT垄断
    供应商锁定风险高

  地缘政治影响:
    俄罗斯公司产品国际推广受限
```

## 9. 实践案例: 俄罗斯农村地区公用事业抄表

### 9.1 场景

俄罗斯某省对分布在广袤农村的20万只水表电表进行远程抄读,最远村镇距最近城镇超25公里。传统人工抄表每月驾车行驶数千公里。

### 9.2 方案比选

```
LoRaWAN: 需约50个网关(每个5-8km), 农村光纤覆盖差回传困难
NB-IoT:  需运营商基站覆盖(农村4G不全), 20万设备月度订阅费高
NB-Fi:   仅需15个基站(每个15-25km), 回传可用卫星, 自建网络无月租
```

### 9.3 部署方案

覆盖约15000平方公里,15个基站部署在主要城镇高点(水塔/通信铁塔),回传用4G(有覆盖处)+卫星VSAT(无覆盖处)。部分站点太阳能+蓄电池供电(无市电)。

### 9.4 终端集成

```python
# NB-Fi水表终端伪代码
def main_loop():
    while True:
        reading = water_meter_sensor.get_reading()
```

## 10. 未来展望

NB-Fi的未来与俄罗斯IoT市场紧密相关。技术演进方向包括更高数据速率模式、增强定位功能(TDOA)、多基站协作接收和卫星回传深度集成。在俄罗斯国内公用事业和智慧城市领域有望持续增长,但国际化扩展面临挑战。技术上NB-Fi有独特优势(超远距离、私有部署),但生态封闭和地缘政治因素限制了全球影响力。

## 总结

NB-Fi是俄罗斯WAVIoT公司开发的超窄带LPWAN协议,技术路线与Sigfox类似但在双向通信、数据载荷和私有部署能力上有所改进。核心优势在于极窄带(100Hz)DBPSK调制带来的超高接收灵敏度,使单基站覆盖半径可达30公里,特别适合俄罗斯和CIS国家地广人稀的环境。在公用事业抄表领域已取得实际部署成果。然而封闭生态(WAVIoT单一供应商)、有限国际社区和地缘政治因素制约了全球发展。对于俄罗斯/CIS市场的超远距离低成本私有网络需求,NB-Fi是实用选择; 全球化部署仍推荐LoRaWAN或NB-IoT。
