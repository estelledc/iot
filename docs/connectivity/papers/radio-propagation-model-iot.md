# 无线电传播模型在IoT覆盖规划中的应用
> **难度**：🟡 中级 | **领域**：无线传播 | **阅读时间**：约 20 分钟

## 引言

想象你要在一个城市里开一家广播电台。你在城中心架了一个发射塔,希望全城的收音机都能听到你的节目。但实际情况是: 离发射塔近的地方声音清晰,远一点的地方有杂音,到了城市边缘几乎听不到了。更麻烦的是,有些高楼后面的区域完全是死角,而某些开阔地带反而能收到意外远的信号。

你需要一张"信号地图",标出城市每个角落能收到多强的信号。画这张地图的工具,就是无线电传播模型。对于 IoT 网络来说,部署 LoRaWAN 网关、规划 NB-IoT 基站、甚至决定 WiFi 接入点的位置,都需要用传播模型来预测信号覆盖范围。如果不做这个预测,要么网关部署太少导致覆盖盲区,要么部署太多浪费资金。

## 1. 传播模型的基本概念

### 1.1 为什么需要传播模型

部署一个 IoT 无线网络时,最基本的问题是: 需要多少个网关(基站),放在哪里?

直觉告诉我们,信号会随距离变弱。但"弱多少"才是关键。在空旷的草原上,信号衰减比较均匀;在密集的城市中,建筑物会阻挡、反射和散射信号,衰减变得复杂而不可预测。

传播模型就是用数学公式来描述信号强度与距离、环境之间关系的工具。它的输出是: 在距离发射机 d 米(或公里)的地方,信号强度是多少 dBm。

### 1.2 路径损耗

路径损耗(Path Loss, PL)是指无线信号从发射机到接收机过程中损失的功率,单位是 dB。路径损耗越大,接收到的信号越弱。

```
接收信号强度 = 发射功率 + 发射天线增益 - 路径损耗 + 接收天线增益
```

所有传播模型的核心目标都是预测路径损耗。不同的模型用不同的方法和参数来估算路径损耗。

### 1.3 影响传播的关键因素

无线信号的传播受多种因素影响: 距离(信号功率与距离的平方或更高次方成反比)、频率(频率越高损耗越大)、环境(城市/郊区/农村/室内衰减差异巨大)、天线高度(越高越不容易被遮挡)、地形(山丘/山谷/水面影响传播路径)以及气象条件(雨雾对10GHz以上高频有显著衰减,对Sub-GHz影响较小)。

## 2. 自由空间路径损耗(FSPL)

### 2.1 最简单的模型

自由空间路径损耗(Free Space Path Loss, FSPL)是最简单的传播模型。它假设发射机和接收机之间没有任何障碍物,信号在真空或无遮挡的空间中传播。

FSPL 的公式:

```
FSPL(dB) = 20 * log10(d) + 20 * log10(f) + 32.44

其中:
  d = 距离(km)
  f = 频率(MHz)
```

### 2.2 计算示例

以 LoRa 868MHz 为例,计算不同距离的自由空间路径损耗:

| 距离(km) | FSPL(dB) |
|-----------|----------|
| 0.1 | 71.2 |
| 0.5 | 85.2 |
| 1.0 | 91.2 |
| 5.0 | 105.2 |
| 10.0 | 111.2 |

可以看到,距离每增加一倍,路径损耗增加约6dB。频率每增加一倍,路径损耗也增加约6dB。

### 2.3 FSPL的局限性与频率对比

FSPL 假设没有任何障碍物,实际环境中的路径损耗总是大于 FSPL。因此 FSPL 被视为路径损耗的下限,主要用途是作为更复杂模型的基准线。

FSPL 清楚地展示了频率对覆盖范围的影响:

```
同样距离1km:
  433MHz: FSPL = 85.2 dB
  868MHz: FSPL = 91.2 dB
  2400MHz: FSPL = 100.0 dB

868MHz 比 2400MHz 少约 8.8dB 损耗
433MHz 比 2400MHz 少约 14.8dB 损耗
```

这就是 LPWAN 技术选择 Sub-GHz 频段的根本物理原因。

## 3. 对数距离路径损耗模型

### 3.1 模型公式

对数距离路径损耗模型(Log-Distance Path Loss Model)是对 FSPL 的扩展,引入了路径损耗指数来描述不同环境的衰减特性:

```
PL(d) = PL(d0) + 10 * n * log10(d / d0) + X_sigma

其中:
  PL(d0) = 参考距离d0处的路径损耗(通常d0=1m或1km)
  n = 路径损耗指数
  d = 距离
  X_sigma = 阴影衰落(标准差为sigma的正态随机变量)
```

### 3.2 路径损耗指数

路径损耗指数 n 是这个模型的关键参数。不同环境下,n 的典型值:

| 环境 | 路径损耗指数 n | 说明 |
|------|---------------|------|
| 自由空间 | 2.0 | 理想无障碍环境 |
| 开阔平地 | 2.0-2.5 | 农村, 公园 |
| 城区(视距) | 2.5-3.0 | 有少量建筑物遮挡 |
| 城区(非视距) | 3.0-4.0 | 密集建筑物遮挡 |
| 室内(同层) | 3.0-4.0 | 办公室, 商场 |
| 室内(跨楼层) | 4.0-6.0 | 信号穿越楼板 |
| 密集城区 | 4.0-5.0 | 高层建筑密集 |

n=2 意味着距离每增加一倍,路径损耗增加 6dB(与 FSPL 一致)。n=4 意味着距离每增加一倍,路径损耗增加 12dB,衰减快了一倍。

### 3.3 阴影衰落与实际应用

X_sigma 代表阴影衰落(Shadow Fading),描述同一距离上由于建筑物布局差异导致的信号随机波动。例如,距基站500米的两个位置,一个在大楼阴影中(信号弱),另一个在直对基站的街道上(信号强)。IoT 规划中通常取 X_sigma 为 6-10dB,保证95%位置正常通信需预留约 1.65 倍标准差的余量。

对数距离模型因简洁而广泛应用于 IoT 规划: 先确定环境选择 n 值,在参考距离处测量 PL(d0),用公式计算目标距离的中值路径损耗,加上阴影衰落余量即得规划值。如果有现场测量数据,可通过回归分析拟合更精确的参数。

## 4. Okumura-Hata模型

### 4.1 模型背景

Okumura-Hata 模型是最经典的经验传播模型之一,基于 Okumura 在1960年代在东京的大量实测数据,由 Hata 在1980年公式化。适用条件: 频率 150-1500MHz,基站天线高度 30-200m,终端天线高度 1-10m,距离 1-20km。这正好覆盖了 Sub-GHz IoT 的典型部署场景。

### 4.2 公式

Hata 模型的城市环境路径损耗公式:

```
PL_urban = 69.55 + 26.16*log10(f) - 13.82*log10(hb) - a(hm)
           + (44.9 - 6.55*log10(hb)) * log10(d)

其中:
  f = 频率(MHz)
  hb = 基站天线高度(m)
  hm = 终端天线高度(m)
  d = 距离(km)
  a(hm) = 终端天线高度修正因子
```

对于中小城市: a(hm) = (1.1*log10(f) - 0.7)*hm - (1.56*log10(f) - 0.8)

对于郊区环境,在城市公式基础上减去修正项:

```
PL_suburban = PL_urban - 2*(log10(f/28))^2 - 5.4
```

对于开阔地区(农村):

```
PL_open = PL_urban - 4.78*(log10(f))^2 + 18.33*log10(f) - 40.94
```

### 4.3 计算示例

以 LoRaWAN 网关部署为例:

```
参数:
  频率 f = 868 MHz
  网关天线高度 hb = 30 m(安装在建筑物楼顶)
  终端天线高度 hm = 1.5 m(地面传感器)
  距离 d = 5 km

城市环境:
  a(hm) = (1.1*log10(868) - 0.7)*1.5 - (1.56*log10(868) - 0.8)
         = (1.1*2.938 - 0.7)*1.5 - (1.56*2.938 - 0.8)
         = 3.35 - 3.78
         = -0.43

  PL_urban = 69.55 + 26.16*2.938 - 13.82*log10(30) - (-0.43)
             + (44.9 - 6.55*log10(30)) * log10(5)
           = 69.55 + 76.82 - 20.40 + 0.43 + (44.9 - 9.67)*0.699
           = 126.40 + 24.63
           = 151.0 dB

郊区环境:
  PL_suburban = 151.0 - 2*(log10(868/28))^2 - 5.4
              = 151.0 - 2*(1.491)^2 - 5.4
              = 151.0 - 4.45 - 5.4
              = 141.2 dB
```

城市环境下5km处的路径损耗约为151dB,郊区约为141dB,差异接近10dB。

### 4.4 在IoT规划中的应用

Okumura-Hata 模型特别适合 LoRaWAN 和 Sigfox 等 Sub-GHz LPWAN 的覆盖规划。它的输入参数(频率、天线高度、距离、环境类型)容易获取,计算简单,可以快速估算大范围的覆盖情况。

注意事项: 该模型给出的是大范围统计平均值,不考虑具体的建筑物布局。对于精确的站点规划,需要结合地理信息数据和更精细的模型。

## 5. COST-231 Hata模型

### 5.1 Hata模型的频率扩展

COST-231 Hata 模型(也称 Hata-COST231)是 Okumura-Hata 模型的频率扩展版,将适用范围从 1500MHz 扩展到 2000MHz。

适用条件:

- 频率范围: 1500-2000MHz
- 基站天线高度: 30-200m
- 终端天线高度: 1-10m
- 距离范围: 1-20km

### 5.2 公式

```
PL = 46.3 + 33.9*log10(f) - 13.82*log10(hb) - a(hm)
     + (44.9 - 6.55*log10(hb)) * log10(d) + Cm

其中:
  Cm = 0 dB (中小城市和郊区)
  Cm = 3 dB (大城市/密集城区)
  其他参数含义同Hata模型
```

### 5.3 应用场景

COST-231 Hata 模型主要用于:

- NB-IoT 在 1800MHz 频段的覆盖规划
- LTE-M 覆盖范围估算
- WiFi HaLow(如果工作在更高频段)的粗略估算

对于 Sub-GHz IoT(LoRa 868/915MHz),仍然使用标准的 Okumura-Hata 模型。两个模型可以配合使用: 用 Okumura-Hata 估算 Sub-GHz LPWAN 覆盖,用 COST-231 估算蜂窝 IoT 覆盖,从而对比两种方案的覆盖差异。

## 6. 室内传播模型

### 6.1 室内环境的特殊性

室内环境的信号传播与室外有显著不同。信号需要穿越墙壁、门、窗户、楼板等建筑结构,每穿越一个障碍物都会产生额外的衰减。

对于 IoT 来说,室内传播非常重要: 智能家居设备需要在家中各个房间正常工作,智能建筑的传感器分布在不同楼层,地下室的水表或电表需要将数据发送到地面。

### 6.2 ITU-R P.1238模型

ITU-R P.1238 是 ITU 推荐的室内传播模型:

```
PL = 20*log10(f) + N*log10(d) + Lf(n) - 28

其中:
  f = 频率(MHz)
  d = 距离(m)
  N = 距离功率损耗系数
  Lf(n) = 楼板穿透损耗, n为穿越的楼板数
```

### 6.3 建筑材料衰减与IoT启示

不同建筑材料的信号衰减差异巨大: 石膏板墙 3-5dB、砖墙 5-10dB、混凝土墙 10-20dB、钢筋混凝土楼板 15-25dB、普通玻璃 2-4dB、Low-E节能玻璃 10-15dB、金属门/墙 20-30dB。

这对 IoT 部署有直接指导意义: 智能家居中 WiFi 路由器穿过两面砖墙后信号可能衰减 10-20dB,应放在中心位置;地下室计量设备信号需穿越混凝土楼板(15-25dB损耗),这就是 LPWAN 需要极高灵敏度的原因;智能建筑中使用 Zigbee 或 BLE Mesh 通常每1-2层需要一个中继节点。

## 7. Sub-GHz与2.4GHz传播对比

### 7.1 理论对比

从 FSPL 公式可以直接计算出 Sub-GHz 和 2.4GHz 在同等距离下的路径损耗差异:

```
频率比: 2400MHz / 868MHz = 2.76
路径损耗差: 20*log10(2.76) = 8.8 dB

频率比: 2400MHz / 433MHz = 5.54
路径损耗差: 20*log10(5.54) = 14.9 dB
```

8.8dB 的差异意味着什么?如果发射功率和接收灵敏度相同,868MHz 的覆盖范围约为 2.4GHz 的2.75倍。14.9dB 的差异则意味着 433MHz 的覆盖范围约为 2.4GHz 的5.5倍。

### 7.2 实测对比

实际测量数据通常显示比理论更大的差异,因为低频信号在建筑物穿透和绕射方面也有优势:

| 场景 | 868MHz典型损耗 | 2.4GHz典型损耗 | 差异 |
|------|----------------|----------------|------|
| 1km城市 | 100 dB | 115 dB | 15 dB |
| 穿一面砖墙 | 5 dB | 8 dB | 3 dB |
| 穿一层楼板 | 12 dB | 18 dB | 6 dB |
| 进入地下室 | 15 dB | 25 dB | 10 dB |

### 7.3 为什么LPWAN选择Sub-GHz

LPWAN 技术(LoRa, Sigfox, Wireless M-Bus)选择 Sub-GHz 频段的核心原因就是传播优势:

更远的覆盖距离意味着更少的网关,降低基础设施成本。更好的建筑物穿透力意味着更可靠的室内覆盖,特别是对于安装在地下室或管井中的计量设备。更好的绕射能力意味着在非视距(NLOS)场景下仍能维持通信。

这些优势对于大规模、低成本、长续航的 IoT 部署至关重要。

## 8. 覆盖规划流程

### 8.1 规划步骤

一个完整的 IoT 无线网络覆盖规划通常包括以下步骤:

第一步: 定义覆盖需求。明确覆盖区域(面积、地形)、目标覆盖概率(95%或99%)、所需数据速率、设备位置(室内/室外/地下)。

第二步: 确定系统参数。选择工作频率、发射功率、天线类型和高度、接收灵敏度。

第三步: 计算链路预算。将所有增益和损耗加总,得到最大允许路径损耗(MAPL)。这是传播模型的输入。

第四步: 选择传播模型。根据环境类型选择合适的模型: 城市用 Okumura-Hata,郊区用 Hata 郊区修正,室内用 ITU-R P.1238。

第五步: 计算小区半径。将 MAPL 代入传播模型公式,反解出最大通信距离。

第六步: 确定网关数量。用覆盖区域面积除以单个网关的覆盖面积,再考虑重叠因子。

第七步: 现场验证。在实际环境中进行测试,验证模型预测是否准确,根据测试结果调整网关位置和数量。

### 8.2 覆盖面积计算

假设网关在中心位置,覆盖区域近似为圆形:

```
单个网关覆盖面积 = pi * R^2

其中 R 为计算得到的小区半径

所需网关数量 = 总覆盖面积 / (单个网关覆盖面积 * 重叠因子)

重叠因子通常取 0.6-0.7, 考虑覆盖边缘和重叠区域
```

### 8.3 余量预留

实际规划中必须预留充足的余量:

| 余量类型 | 典型值(dB) | 说明 |
|----------|-----------|------|
| 阴影衰落余量 | 8-12 | 保证95%位置覆盖 |
| 建筑穿透损耗 | 10-25 | 室内覆盖需求 |
| 人体损耗 | 3-5 | 设备被人体遮挡 |
| 时间衰落余量 | 3-5 | 季节性植被变化等 |
| 设备老化余量 | 2-3 | 天线和器件老化 |

## 9. 规划工具与仿真

### 9.1 商业与开源工具

商业工具包括: EDX SignalPro(支持多种传播模型,可导入建筑物和地形数据生成覆盖热力图)、Atoll(广泛用于蜂窝和LPWAN网络规划)、ASSET/Aircom(主流蜂窝规划工具)。

开源和免费工具包括: Radio Mobile(使用 SRTM 地形数据和 Longley-Rice 传播模型)、SPLAT!(Linux下的RF传播分析工具)、CloudRF(在线覆盖预测服务)。

### 9.2 众包验证

TTN Mapper 专门用于 LoRaWAN 的众包覆盖地图,用户携带 GPS+LoRa 设备移动,实时记录信号强度生成覆盖地图。传统的 Drive Testing 方法则是在车上安装接收设备沿道路行驶记录信号。最佳实践是将建模预测和现场测量结合: 先用模型做初步规划,再通过现场测试验证和微调。

## 10. 实际案例: 智慧城市LoRaWAN部署

### 10.1 项目背景

某中等城市计划部署 LoRaWAN 网络,覆盖城区50平方公里,支持智能停车、垃圾桶监测和环境传感器。目标是实现95%的室外覆盖率。

### 10.2 系统参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 工作频率 | 868 MHz | 欧洲Sub-GHz频段 |
| 网关发射功率 | 27 dBm | 500mW(869.525MHz子频段) |
| 网关天线增益 | 6 dBi | 全向玻璃钢天线 |
| 网关天线高度 | 30 m | 安装在建筑物楼顶 |
| 终端发射功率 | 14 dBm | 25mW |
| 终端天线增益 | 0 dBi | 内置PCB天线 |
| 终端灵敏度 | -137 dBm | LoRa SF12/125kHz |

### 10.3 覆盖计算

使用 Okumura-Hata 城市模型计算最大覆盖半径:

```
最大允许路径损耗(MAPL):
  上行: 14 + 0 + 137 + 6 = 157 dB
  (终端发射功率 + 终端天线增益 + 网关灵敏度 + 网关天线增益)

将 MAPL = 157 dB 代入 Hata 城市公式, 反解距离 d:
  157 = 69.55 + 76.82 - 20.40 + 0.43 + 35.23*log10(d)
  157 = 126.40 + 35.23*log10(d)
  30.60 = 35.23*log10(d)
  log10(d) = 0.869
  d = 7.4 km

减去阴影衰落余量10dB后:
  147 = 126.40 + 35.23*log10(d)
  20.60 = 35.23*log10(d)
  d = 4.0 km

考虑95%覆盖概率, 有效覆盖半径约 3 km
```

### 10.4 网关数量估算

```
单个网关覆盖面积 = 3.14 * 3^2 = 28.3 km^2
重叠因子 = 0.65
有效覆盖面积 = 28.3 * 0.65 = 18.4 km^2

所需网关数量 = 50 / 18.4 = 2.7, 取整为 3 个

但考虑实际城市形状不规则, 初步规划 8 个网关
```

### 10.5 现场验证与调整

部署8个网关后进行 Drive Test,发现城区约90%区域信号良好(RSSI高于-120dBm),但两个区域因高层建筑群遮挡存在盲区,一个工业区因金属厂房信号衰减严重。在两个盲区各增加1个网关,最终以10个网关实现全城95%以上的室外覆盖。

## 总结

无线电传播模型是 IoT 网络规划的基础工具。从最简单的 FSPL 到经验性的 Okumura-Hata,再到室内专用的 ITU-R P.1238,不同模型适用于不同的场景和精度需求。实际规划中,模型预测只是起点,必须结合链路预算计算和现场测量验证,才能得到可靠的部署方案。对于 IoT 从业者,掌握基本的传播模型知识和规划流程,能够在项目初期快速评估网络可行性,避免盲目部署造成的资源浪费。
