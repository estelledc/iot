## 引言

想象你走进一个大型购物中心,手机上的导航App告诉你"前方10米左转到达星巴克"——但你在室内,GPS信号根本收不到。你抬头看了看天花板上密密麻麻的LED灯,可能不会想到,这些灯本身就能充当"室内GPS卫星"。每一盏灯都在以人眼完全察觉不到的速度闪烁着,像无声的灯塔一样广播着自己的身份编号,而你手机的前置摄像头或专用光传感器就是接收器,通过识别头顶是哪盏灯,就能确定你站在哪里。

这就是可见光通信(Visible Light Communication, VLC)在室内定位中的应用,而LiFi(Light Fidelity)则是VLC的网络化扩展,类似于WiFi对无线局域网的定义。本文将深入介绍VLC和LiFi的工作原理、室内定位技术、IoT应用场景,以及当前的技术挑战和商业进展。

## 1. 可见光通信基础

### 1.1 核心原理

可见光通信的核心思想是让LED灯同时承担照明和通信两个功能。LED是一种半导体器件,可以在纳秒级别的时间尺度上进行开关切换。人眼的视觉暂留效应使得我们无法感知高于约200Hz的闪烁,因此当LED以MHz甚至GHz的频率调制时,人眼只会看到稳定的灯光,而光电探测器可以精确捕捉到这种快速变化并从中恢复数据。

基本通信链路由以下部分组成:

```
发送端:
LED灯 + 驱动电路 + 调制器
    |
    v (可见光信道: 400-800THz)
    |
接收端:
光电二极管(PD) + 跨阻放大器(TIA) + ADC + 解调器
```

### 1.2 调制技术

VLC使用的调制技术根据应用需求分为两大类:

简单调制适用于低速场景。OOK(On-Off Keying,开关键控)是最基本的方式:LED亮表示"1",LED灭表示"0"。实际实现中,LED不会完全熄灭(否则会出现可见闪烁),而是在两个不同的亮度水平之间切换。VPPM(Variable Pulse Position Modulation)则通过脉冲在时隙中的位置来编码数据,同时允许调节LED的调光比例,兼顾照明亮度控制。

高速调制适用于LiFi数据传输。OFDM(Orthogonal Frequency Division Multiplexing,正交频分复用)将可用带宽分成多个窄带子载波,每个子载波独立调制。由于LED的调制带宽有限(白光LED约3-5MHz,磷光体响应慢),OFDM可以在有限带宽内最大化频谱效率。DCO-OFDM(DC-biased Optical OFDM)在信号上叠加直流偏置,使得调制信号始终为正(因为光强不能为负),是VLC中最常用的OFDM变体。

### 1.3 可见光频谱的优势

可见光的频率范围大约在400THz到800THz之间,这是一段巨大的、完全未被管制的频谱资源。与射频频谱(从kHz到几百GHz)相比,可见光频段的带宽大了约1000倍。虽然实际可用带宽受限于LED器件的调制速率和光电探测器的响应速度,但潜在的容量是巨大的。

## 2. 从VLC到LiFi

### 2.1 LiFi的定义

LiFi这个术语由爱丁堡大学的Harald Haas教授在2011年的TED演讲中首次提出。如果说VLC是"用光传数据"的基础技术,那么LiFi就是建立在VLC之上的完整网络系统,类似于WiFi是建立在无线电通信之上的网络标准。

LiFi系统的关键组成:

- 接入点(AP): 天花板上的LED灯具,相当于WiFi路由器
- 终端设备: 配备光电探测器的手机、平板或IoT设备
- 下行链路: LED到终端,使用可见光
- 上行链路: 终端到LED,通常使用红外光(对人眼不可见)
- 网络协议: 支持IP数据包的完整协议栈

### 2.2 LiFi vs WiFi

| 特性 | LiFi | WiFi |
|------|------|------|
| 传输介质 | 可见光/红外光 | 无线电波 |
| 频谱 | 400-800THz(无需授权) | 2.4/5/6GHz(ISM频段) |
| 穿墙能力 | 无(光不穿墙) | 有 |
| 干扰 | 与RF无干扰 | 与其他WiFi/蓝牙干扰 |
| 安全性 | 天然物理隔离(不出房间) | 需要加密(信号可穿墙) |
| 实验室峰值速率 | 1-10Gbps | WiFi 6: 9.6Gbps |
| 商用产品速率 | 10-100Mbps | WiFi 6: 数百Mbps |
| 移动性 | 受限(需在灯光覆盖区) | 良好 |
| 基础设施 | 利用现有LED照明 | 需要额外路由器 |

### 2.3 IEEE 802.15.7标准

IEEE 802.15.7是VLC的国际标准,定义了物理层和MAC层规范:

PHY I: 适用于户外低速场景,使用OOK/VPPM,速率11.67-266.6kbps。PHY II: 室内中速场景,使用OOK/VPPM,速率1.25-96Mbps。PHY III: 使用CSK(Color Shift Keying,颜色偏移键控),利用RGB LED的三个颜色通道进行调制,速率12-96Mbps。

标准还定义了设备发现、关联、认证和数据传输的MAC层流程,以及与室内定位相关的功能。

## 3. VLC室内定位技术

### 3.1 定位原理

VLC室内定位的核心思路非常直观:建筑中的每盏LED灯位置是固定且已知的,如果终端设备能识别出它头顶是哪盏灯,就能确定自己的位置。

具体实现分为几个层次:

最简单的方案是基于灯具ID的区域定位。每盏灯广播一个唯一的ID,终端接收到ID后查表得到灯的位置,精度等于灯具的覆盖范围(通常是几平方米)。这种方案实现简单,对于"你在哪个区域"的定位需求已经足够。

更精确的方案使用信号强度(RSS)。终端同时接收到多盏灯的光信号,根据各灯信号的强弱来三角定位。接收光功率与距离的平方成反比(符合光学逆平方定律),结合已知的灯具位置和发射功率,可以估算终端到每盏灯的距离。

最高精度的方案基于到达角度(AoA)或图像传感器。使用多像素的光传感器或摄像头,通过灯光在传感器上的成像位置来计算入射角度,再结合几何关系确定位置。这种方法可以达到厘米级精度。

### 3.2 信号编码方式

每盏灯需要广播自己的ID,但又不能让人眼看到闪烁。常用的编码方式有以下几种:

频率编码: 每盏灯使用不同的调制频率来标识自己,接收器通过频谱分析区分不同灯具。优点是简单,缺点是可分配的频率数量有限。

时分编码: 所有灯具使用相同的载波频率,但在不同的时间段广播自己的ID。需要灯具之间的时间同步。

CDMA编码: 每盏灯使用不同的扩频码(类似CDMA手机通信),可以同时广播,接收器通过相关解码区分不同信号。这种方式频谱效率最高,但实现复杂度也最高。

### 3.3 与射频定位技术的对比

| 定位技术 | 典型精度 | 基础设施成本 | 穿墙/NLOS | 环境依赖 |
|----------|----------|-------------|-----------|----------|
| VLC | 10-50cm | 低(用现有灯) | 不支持 | 需要灯光 |
| UWB | 10-30cm | 高(专用锚点) | 部分支持 | 多径影响 |
| BLE信标 | 1-5m | 中(蓝牙信标) | 支持 | 多径影响 |
| WiFi指纹 | 2-5m | 低(现有AP) | 支持 | 环境变化 |

VLC定位的最大优势在于精度高且可利用现有LED照明设施,不需要部署额外硬件(从基础设施角度)。最大劣势在于严格要求视线通路,且终端设备需要光传感器。

## 4. IoT应用场景

### 4.1 商场导航与精准营销

在大型购物中心中,天花板LED灯具被升级为VLC发射器,每个灯具广播区域编码。顾客手机通过前置摄像头或专用USB光传感器接收信号,实现精确到柜台级别的室内导航。

更进一步,当系统知道顾客站在某个货架前时,可以推送相关商品的优惠券或推荐信息,实现基于位置的精准营销。这种精度是WiFi或BLE信标定位难以达到的。

### 4.2 博物馆和展馆导览

每件展品上方的射灯可以作为VLC发射器,当参观者走到展品前时,手机自动接收该灯的信号,触发展品的语音解说、文字介绍或AR(增强现实)内容。由于光不会穿过展板,相邻展品之间不会互相干扰,体验非常精确。

### 4.3 医院和飞机中的数据通信

医院中大量使用的医疗设备对射频干扰敏感,LiFi提供了一种安全的替代方案。病房内的LED灯可以作为LiFi接入点,为医疗设备和患者娱乐系统提供网络连接,不会对生命体征监测仪等设备产生电磁干扰。

飞机客舱也是类似的场景。目前已有航空公司测试在阅读灯中集成LiFi模块,为乘客提供上网服务,避免可能影响航电系统的射频干扰(尽管现代飞机已经对WiFi做了充分的屏蔽)。

### 4.4 工业资产追踪

在工厂和仓库中,安装在天花板的LED灯可以为货物搬运车、自动导引车(AGV)和工人提供精确定位。配合VLC接收器的资产标签可以在经过每个灯具下方时更新位置信息,实现区域级的资产追踪。

### 4.5 水下IoT通信

射频波在水中衰减极为严重,WiFi和蓝牙在水下几乎不可用。可见光(尤其是蓝绿光)在清澈的水中的传播距离可达数十米,因此VLC成为水下IoT通信(如水下传感器网络、潜水器通信)的有力候选方案。

## 5. 技术挑战

### 5.1 上行链路问题

VLC的下行链路(灯到终端)比较自然,因为LED灯本来就在那里。但上行链路(终端到灯)是个难题:手机不能向天花板发射强可见光(会晃到人),所以通常使用红外光作为上行通道。这意味着终端设备需要额外集成红外发射器,增加了硬件成本和设计复杂度。

另一种方案是使用RF(如WiFi或BLE)作为上行通道,形成混合系统。下行用光,上行用射频,可以利用各自的优势。

### 5.2 环境光干扰

太阳光中包含全波段的可见光,是VLC系统面临的最大噪声源。在靠近窗户或有天窗的区域,环境光噪声可能淹没通信信号。解决方案包括: 使用窄带光学滤波器(只允许特定波长通过)、采用差分检测(用两个探测器消除共模噪声)、以及选择环境光较弱的波长区域进行通信。

### 5.3 视线遮挡和移动性

与红外通信类似,VLC也要求发送端和接收端之间有直接的视线通路。人体遮挡、手机放在口袋里、以及用户快速移动导致的频繁切换都是实际部署中的挑战。

多灯覆盖和快速切换是解决移动性问题的关键。类似于蜂窝网络的切换(handover),LiFi系统需要在用户从一盏灯的覆盖区移动到另一盏灯时,无缝切换连接。

### 5.4 终端设备适配

目前绝大多数智能手机没有专用的VLC接收器。虽然前置摄像头可以用于低速VLC(如接收灯具ID用于定位),但其帧率(通常30-60fps)严重限制了数据速率。专用的光电探测器模块可以提供MHz级的带宽,但需要额外的硬件。

一些解决方案包括: 使用手机的环境光传感器(带宽有限)、开发USB-C外接VLC模块、或在IoT设备(非手机)中直接集成光电探测器。

## 6. 实现参考

### 6.1 简单VLC发射器

以下是使用微控制器实现基本VLC发射的概念代码:

```c
// 简单OOK调制VLC发射器
// LED连接到GPIO, 通过PWM实现载波调制

#define LED_PIN     GPIO_NUM_2
#define CARRIER_FREQ 1000000  // 1MHz载波

// 灯具ID: 每个灯具唯一的8位标识
uint8_t lamp_id = 0x3A;

void vlc_send_bit(uint8_t bit) {
    if (bit) {
        // 逻辑1: 输出载波(LED高频闪烁)
        pwm_start(LED_PIN, CARRIER_FREQ, 50);  // 50%占空比
    } else {
        // 逻辑0: LED常亮(无载波调制)
        gpio_set_level(LED_PIN, 1);
    }
    delay_us(100);  // 每位持续100us = 10kbps
}

void vlc_broadcast_id(void) {
    // 发送前导码(用于接收端同步)
    for (int i = 0; i < 8; i++) {
        vlc_send_bit(1);
        vlc_send_bit(0);
    }
    // 发送灯具ID(8位)
    for (int i = 7; i >= 0; i--) {
        vlc_send_bit((lamp_id >> i) & 0x01);
    }
}
```

### 6.2 定位计算

已知三盏灯的位置和到终端的距离估算,可以用三边测量法计算位置:

```python
import numpy as np

def vlc_trilateration(lamps, distances):
    """
    lamps: [(x1,y1), (x2,y2), (x3,y3)] 灯具坐标(米)
    distances: [d1, d2, d3] 估算距离(米)
    返回: (x, y) 终端估算坐标
    """
    x1, y1 = lamps[0]
    x2, y2 = lamps[1]
    x3, y3 = lamps[2]
    d1, d2, d3 = distances

    # 构建线性方程组 Ax = b
    A = np.array([
        [2*(x3-x1), 2*(y3-y1)],
        [2*(x3-x2), 2*(y3-y2)]
    ])
    b = np.array([
        d1**2 - d3**2 - x1**2 + x3**2 - y1**2 + y3**2,
        d2**2 - d3**2 - x2**2 + x3**2 - y2**2 + y3**2
    ])

    # 最小二乘解
    pos = np.linalg.lstsq(A, b, rcond=None)[0]
    return pos[0], pos[1]

# 示例: 三盏灯的坐标
lamps = [(0, 0), (5, 0), (2.5, 4)]
# 接收信号强度反推的距离
distances = [2.1, 3.2, 2.8]
x, y = vlc_trilateration(lamps, distances)
print(f"Estimated position: ({x:.2f}, {y:.2f})")
```

## 7. 商业发展现状

### 7.1 主要厂商

pureLiFi是LiFi领域的先驱企业,由Harald Haas教授联合创立。其产品包括LiFi接入点和USB接收加密狗(dongle),主要面向企业和政府客户,在需要高安全性和无RF干扰的环境中部署。

Signify(飞利浦照明)推出了基于VLC的室内定位系统,已经在多个大型零售商(如家乐福、Target)的门店中部署。该系统利用店内LED灯具广播位置编码,顾客手机通过前置摄像头接收信号,实现店内导航和精准促销。

Oledcomm是法国的LiFi公司,开发了面向航空、医疗和教育行业的LiFi解决方案。其产品已在法国航空的航班上进行了测试。

### 7.2 市场前景

VLC/LiFi目前仍是一个相对小众的市场,与WiFi和5G相比,商用规模差距很大。但在以下细分场景中有明确的增长空间:

室内定位市场正在快速增长,VLC定位以其高精度和低额外基础设施成本的优势,有望在零售、物流和智慧建筑领域占据一席之地。RF敏感环境(医院、飞机、矿井)对无RF通信有刚性需求,LiFi是目前最成熟的解决方案之一。安全通信领域,光不穿墙的特性使得VLC在政府和军事场景中受到关注。

## 8. 与其他IoT定位技术的协同

### 8.1 多技术融合定位

在实际部署中,VLC定位很少作为唯一方案使用,更常见的是与其他定位技术融合:

VLC提供在灯光覆盖区域内的高精度定位(10-50cm),BLE信标作为补充,覆盖VLC信号无法到达的区域(如走廊转角、无灯区域),WiFi指纹定位作为粗定位的兜底方案。

这种多技术融合的定位系统可以兼顾精度、覆盖率和成本,是当前室内定位的发展趋势。

### 8.2 融合架构示例

一个典型的商场室内定位融合架构如下:

```
                    定位服务器(云端)
                    /      |      \
                   /       |       \
           VLC网关    BLE网关    WiFi AP
              |         |          |
        LED灯(VLC)  BLE信标    WiFi信号
              |         |          |
        用户手机(前置摄像头 + BLE + WiFi)

定位决策逻辑:
1. 优先使用VLC(精度最高, 10-50cm)
2. VLC不可用时退化到BLE(1-5m)
3. BLE信标也收不到时使用WiFi指纹(2-5m)
```

在这种架构中,手机App同时监听VLC、BLE和WiFi三种信号,并根据信号可用性和质量自动选择最佳定位源。位置数据上传到定位服务器,服务器可以融合多种信号源的结果,输出最终的位置估计。

### 8.3 VLC定位的部署建议

对于考虑部署VLC室内定位的场景,以下几点建议值得注意。首先是LED灯具的密度和布局:灯具间距越小,定位精度越高,但改造成本也越高。通常建议在高价值区域(如促销柜台、展品前方)加密灯具布局,在通道等低优先区域使用稀疏布局。

其次是接收端的选择:如果目标是消费者导航(手机用户),需要考虑手机摄像头的兼容性和App的开发成本;如果目标是资产追踪(工业标签),可以使用专用光电探测器模块获得更好的性能。

最后是与现有照明系统的兼容性:VLC发射功能需要LED灯具的驱动电路支持高频调制。新建建筑可以直接采用VLC兼容灯具,已有建筑则需要评估灯具改造的可行性和成本。

## 总结

可见光通信(VLC)利用LED灯具的高速调制能力,实现照明与数据通信的双重功能。LiFi将VLC扩展为完整的网络系统,具备与WiFi类似的功能,但使用光而非无线电波作为传输介质。

在IoT室内定位领域,VLC的核心优势在于: 利用现有LED照明设施,无需部署额外硬件; 光不穿墙,天然实现房间级隔离,避免射频多径效应; 定位精度可达10-50cm,优于大多数RF定位方案。

但VLC也面临显著挑战:视线遮挡要求、环境光干扰、上行链路的实现复杂性、以及终端设备缺乏专用光传感器。当前VLC/LiFi仍处于商业化早期,在零售室内导航、RF敏感环境和高安全需求场景中已有成功部署,未来有望与BLE、WiFi等技术融合,形成更完善的室内IoT解决方案。
