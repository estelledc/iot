# LTE Cat-M1与NB-IoT技术选型决策
> **难度**：🟡 中级 | **领域**：蜂窝IoT选型 | **阅读时间**：约 20 分钟

## 引言

想象你要从北京寄一个包裹到上海。你有两种选择: 顺丰快递(速度快、服务全、价格稍高)和普通平邮(速度慢、只能寄信件、但便宜到几乎免费)。LTE Cat-M1 和 NB-IoT 就像这两种快递服务------它们都属于 3GPP 标准化的低功耗广域网(LPWAN)技术, 目标都是让物联网设备用极低功耗连接蜂窝网络, 但设计哲学和适用场景截然不同。

运营商在部署时往往面临一个关键抉择: 选 Cat-M1、选 NB-IoT、还是两者都部署? 不同地区给出了不同答案------北美倾向 Cat-M1, 中国力推 NB-IoT, 欧洲则两者兼顾。本文将从技术参数、覆盖能力、功耗成本等维度做系统对比, 帮助你在实际项目中做出合理选型。

## 1 两种标准的起源与定位

### 1.1 3GPP 的 IoT 双轨策略

3GPP 从 Release 13(2016年)开始, 同时引入了两种面向物联网的蜂窝技术:

- **Cat-M1(LTE-M)**: 基于 LTE 的简化版本, 保留了一定的移动性和语音能力
- **NB-IoT**: 全新设计的窄带协议, 追求极致的低功耗和深覆盖

这种"双轨"策略并非重复建设, 而是为了覆盖不同的 IoT 场景谱系。Cat-M1 瞄准中等复杂度的连接设备, NB-IoT 瞄准最简单的传感器节点。

### 1.2 设计哲学差异

Cat-M1 的思路是"在 LTE 基础上做减法": 保留 LTE 的核心架构, 去掉高阶调制、多天线等复杂特性, 降低成本的同时维持合理的传输能力。

NB-IoT 的思路是"从零开始做加法": 以最窄的带宽和最简的协议栈为起点, 只加入 IoT 场景必需的功能, 把复杂度和功耗压到最低。

## 2 Cat-M1 技术概述

### 2.1 核心参数

Cat-M1(也称 LTE Category M1 或 eMTC)的关键技术指标:

| 参数 | Cat-M1 规格 |
|------|-------------|
| 带宽 | 1.4 MHz |
| 下行峰值速率 | 1 Mbps |
| 上行峰值速率 | 1 Mbps |
| 双工模式 | 半双工 FDD |
| 语音支持 | VoLTE |
| 移动性 | 支持连接态切换 |
| MCL(最大耦合损耗) | 155.7 dB |

### 2.2 关键特性

**带宽与速率**: 1.4 MHz 的带宽在 LTE 体系中是最窄的配置, 但对 IoT 来说已经足够支撑固件 OTA 升级、较丰富的传感数据上报等场景。1 Mbps 的峰值速率远超一般传感器的需求, 为更复杂的应用留出了余量。

**移动性支持**: Cat-M1 支持连接态(Connected Mode)下的小区切换(Handover), 这意味着设备在移动过程中不会断连。对于车载追踪器、可穿戴设备等需要在移动中保持连接的场景, 这是关键优势。

**语音能力**: Cat-M1 支持 VoLTE(Voice over LTE), 可以实现语音通话。这对于紧急呼叫按钮、老人看护设备等需要语音功能的 IoT 产品至关重要。

### 2.3 协议栈简化

相比完整的 LTE Cat-1, Cat-M1 做了以下简化:

```
完整 LTE Cat-1          Cat-M1 简化
-----------------      -----------------
20 MHz 带宽      -->   1.4 MHz 带宽
10 Mbps DL       -->   1 Mbps DL
5 Mbps UL        -->   1 Mbps UL
全双工 FDD       -->   半双工 FDD
2 RX 天线        -->   1 RX 天线
```

这些简化直接降低了芯片面积和模组成本, 同时减少了功耗。

## 3 NB-IoT 技术概述

### 3.1 核心参数

| 参数 | NB-IoT 规格 |
|------|-------------|
| 带宽 | 200 kHz(单载波 180 kHz) |
| 下行峰值速率 | 250 kbps(Release 14) |
| 上行峰值速率 | 250 kbps(Release 14) |
| 双工模式 | 半双工 FDD |
| 语音支持 | 不支持 |
| 移动性 | 仅小区重选(Release 13) |
| MCL(最大耦合损耗) | 164 dB |

### 3.2 关键特性

**极窄带宽**: 200 kHz 的带宽意味着 NB-IoT 可以部署在 LTE 频段的保护带(Guard Band)中, 甚至独立部署在 GSM 频段上, 不占用 LTE 主载波资源。

**深度覆盖**: 164 dB 的 MCL 比标准 LTE 高出约 20 dB, 意味着信号可以穿透更多墙壁和楼层, 到达地下停车场、地下管廊等极端位置。每增加 10 dB 相当于覆盖距离翻倍, 所以 20 dB 的增益非常显著。

**超低复杂度**: NB-IoT 的基带处理比 Cat-M1 更简单, 直接反映在模组成本上。协议栈精简到只保留数据传输必需的功能, 去掉了语音、切换等特性。

### 3.3 部署模式

NB-IoT 支持三种灵活的部署方式:

```
独立部署(Standalone):
  使用独立的 200 kHz 频段, 如退役的 GSM 载波
  [----200kHz NB-IoT----]

保护带部署(Guard Band):
  部署在 LTE 载波的保护带中
  [Guard][===LTE 载波===][Guard]
    ^NB                    ^NB

带内部署(In-Band):
  占用 LTE 载波内的一个 PRB
  [===LTE===|NB|===LTE===]
```

## 4 核心维度对比

### 4.1 带宽与数据速率

Cat-M1 的 1.4 MHz 带宽是 NB-IoT 200 kHz 的 7 倍, 峰值速率是 NB-IoT 的 4 倍。这个差距在以下场景中尤为明显:

- **固件 OTA 升级**: 100 KB 的固件包, Cat-M1 约 1 秒传完, NB-IoT 需要约 4 秒。看似差距不大, 但传输期间设备无法进入深度睡眠, 传输时间直接影响电池寿命。
- **突发数据上报**: 当传感器需要上报一段较长的数据(如振动波形), Cat-M1 的高吞吐量优势明显。

### 4.2 覆盖能力

| 指标 | Cat-M1 | NB-IoT |
|------|--------|--------|
| MCL | 155.7 dB | 164 dB |
| 相对 LTE 增益 | +15 dB | +20 dB |
| 典型覆盖场景 | 室内、郊区 | 深度室内、地下 |

NB-IoT 的覆盖优势来自更窄的带宽集中了功率谱密度、支持更多次的重复传输(Repetition)以及更长的传输时间间隔。对于地下室水表、地下管廊传感器等深覆盖场景, NB-IoT 是更可靠的选择。

### 4.3 移动性

这是两者最本质的差异之一:

```
Cat-M1 移动场景:
  基站A ----切换----> 基站B ----切换----> 基站C
  设备持续保持连接, 数据不丢失

NB-IoT 移动场景(Release 13):
  基站A ----断开----> 搜索 ----重连----> 基站B
  设备需要重新建立连接, 有数据中断
```

Cat-M1 支持连接态切换, 设备在基站间移动时数据流不中断。NB-IoT 在 Release 13 中仅支持空闲态的小区重选, 设备移动时需要断开再重连。

### 4.4 语音支持

Cat-M1 支持 VoLTE, 可以实现紧急呼叫(E911)、语音通话功能和双向语音交互。NB-IoT 完全不支持语音, 只能传输数据。对于需要语音功能的设备(如紧急求助按钮、智能手表), Cat-M1 是唯一选择。

### 4.5 功耗对比

两者都支持 3GPP 定义的省电机制:

- **PSM(Power Saving Mode)**: 设备进入深度睡眠, 功耗降至微安级
- **eDRX(extended DRX)**: 延长寻呼监听周期, 减少唤醒次数

NB-IoT 由于更简单的协议栈和更窄的带宽, 在活跃传输阶段的功耗略低于 Cat-M1。但两者在 PSM 状态下的功耗几乎相同(均为 1-3 uA 级别)。实际电池寿命取决于数据传输频率和每次传输的数据量。

### 4.6 模组成本

| 成本项 | Cat-M1 | NB-IoT |
|--------|--------|--------|
| 模组目标价 | $3-5 | $2-3 |
| 芯片复杂度 | 中等 | 低 |
| 相比 LTE Cat-1 | 约 1/3 | 约 1/5 |

NB-IoT 更简单的基带设计带来了明显的成本优势。但随着出货量增加和芯片工艺进步, 两者的价格差距在逐渐缩小。

## 5 全球部署格局

### 5.1 地区差异

不同地区的运营商做出了不同的选择, 背后有频谱政策、产业生态、目标市场等多重因素:

**北美(Cat-M1 主导)**: AT&T、Verizon、T-Mobile 均优先部署 Cat-M1。北美市场对移动资产追踪需求大, Cat-M1 的移动性和语音能力更符合需求。

**中国(NB-IoT 主导)**: 中国移动、中国电信、中国联通大规模部署 NB-IoT。中国的智慧城市建设产生了海量的表计类(水表、气表、电表)需求, 这些场景正好是 NB-IoT 的强项。

**欧洲(混合部署)**: 沃达丰、德国电信等运营商同时部署 Cat-M1 和 NB-IoT, 根据客户需求灵活选择。

### 5.2 运营商部署现状

```
地区        Cat-M1                    NB-IoT
-------     -----------------------   -----------------------
北美        AT&T, Verizon, T-Mobile   有限部署
中国        有限部署                   三大运营商全覆盖
欧洲        多数运营商已部署           多数运营商已部署
日韩        有部署                     有部署
东南亚      逐步扩展                   逐步扩展
```

## 6 典型应用场景映射

### 6.1 Cat-M1 优势场景

- **资产追踪器**: 物流车辆、集装箱追踪, 需要移动中持续上报位置
- **可穿戴设备**: 智能手表、儿童定位手表, 需要移动性加语音
- **POS 终端**: 移动支付设备, 需要中等数据速率和移动性
- **紧急报警设备**: 需要 VoLTE 语音通话能力

### 6.2 NB-IoT 优势场景

- **智能表计**: 水表、气表、电表, 固定安装, 每天上报几十字节
- **停车检测**: 地磁传感器, 埋在地下, 需要深覆盖
- **农业监测**: 土壤湿度、温度传感器, 偏远地区, 需要广覆盖
- **环境监测**: 空气质量、噪声传感器, 固定部署, 小数据量

### 6.3 场景选择速查表

```
需要语音通话?        --> Cat-M1
需要深度地下覆盖?    --> NB-IoT
设备会移动?          --> Cat-M1
追求最低成本?        --> NB-IoT
需要 OTA 升级大固件? --> Cat-M1
每次只发几十字节?    --> NB-IoT
不确定?              --> 双模模组
```

## 7 双模模组: 两全其美

### 7.1 主流双模芯片

对于无法确定部署地区或需要全球漫游的场景, 双模模组提供了灵活性:

- **Qualcomm MDM9206**: 支持 Cat-M1 加 NB-IoT, 广泛应用于追踪器
- **Nordic nRF9160**: Cat-M1/NB-IoT 加 GNSS 集成, 适合可穿戴
- **Sequans Monarch**: 超低功耗双模平台, 主打传感器市场

### 7.2 双模策略

双模模组的工作方式:

```
设备上电
  |
  v
扫描可用网络
  |
  +--> 检测到 Cat-M1 信号? --> 优先连接 Cat-M1
  |
  +--> 仅有 NB-IoT 信号?   --> 回退到 NB-IoT
  |
  +--> 根据应用需求选择    --> 需要移动性用 Cat-M1
                                需要深覆盖用 NB-IoT
```

双模方案的代价是模组成本略高(增加 $1-2), 但换来了全球部署的灵活性。

## 8 选型决策树

针对实际项目, 建议按以下流程做决策:

```
[项目需求分析]
       |
       v
  需要语音功能?
  YES --> Cat-M1
  NO  --> |
          v
     设备是否移动?
     YES --> Cat-M1
     NO  --> |
             v
        需要深度覆盖(地下/室内)?
        YES --> NB-IoT
        NO  --> |
                v
           数据量 > 100KB/次?
           YES --> Cat-M1
           NO  --> |
                   v
              追求最低模组成本?
              YES --> NB-IoT
              NO  --> |
                      v
                 部署地区确定?
                 YES --> 选当地运营商主推技术
                 NO  --> 双模模组
```

## 9 未来演进与融合

### 9.1 持续演进

两种技术在 3GPP 后续版本中都在持续改进:

- **Release 14**: NB-IoT 增加了多载波、定位、多播; Cat-M1 提升了速率
- **Release 15/16**: 与 5G 核心网集成, 两者都可以连接 5G 核心网
- **Release 17**: 进一步功耗优化和覆盖增强

### 9.2 5G RedCap 的影响

5G NR RedCap(Reduced Capability)作为 5G 时代的中等能力 IoT 技术, 定位在 Cat-M1 之上。但 3GPP 已明确: NB-IoT 和 Cat-M1 至少支持到 2030 年, 不会被立即替代。

```
当前                    未来(2025-2030)
NB-IoT     ---------> NB-IoT(持续演进)
Cat-M1     ---------> Cat-M1(持续演进)
                       5G RedCap(新增中间层)
                       eRedCap(进一步简化)
```

### 9.3 选型建议

对于新项目:
- 短期(1-3年): 根据场景选 Cat-M1 或 NB-IoT, 两者生态成熟
- 中期(3-5年): 关注 RedCap 商用进展, 为迁移做技术储备
- 长期(5年以上): NB-IoT/Cat-M1 仍可用, 但新设计可考虑 RedCap

## 总结

Cat-M1 和 NB-IoT 并非竞争关系, 而是互补关系。Cat-M1 是"能力更全面的物联网蜂窝连接", 适合需要移动性、语音、较高数据速率的场景; NB-IoT 是"极致精简的传感器连接", 适合固定部署、小数据量、深覆盖的场景。

选型的核心逻辑很简单: 设备会动吗? 需要说话吗? 埋在地下吗? 回答这三个问题, 答案基本就出来了。如果实在拿不准, 双模模组是安全的兜底选择。
