# 分集技术提高IoT无线链路可靠性
> **难度**：🔴 高级 | **领域**：无线可靠性 | **阅读时间**：约 22 分钟

## 引言

假设你要从家里打电话给一个住在山谷里的朋友。如果只有一条山路上的信号塔,遇到暴风雨信号塔故障,电话就打不通了。但如果山谷周围有三座信号塔,即使一座出了问题,另外两座仍然可以工作——这就是分集(Diversity)的核心思想: 不把所有鸡蛋放在一个篮子里。

在无线通信中,衰落会导致信号随机下降。分集技术通过提供信号的多个独立副本,使得所有副本同时衰落的概率大大降低。这是提高IoT无线链路可靠性的最基本也是最有效的方法之一。

## 1. 为什么需要分集

### 1.1 衰落的本质问题

瑞利衰落中,信号低于均值20dB的概率约为1%。对于每小时发送一次数据的IoT设备,1%的丢包率意味着每4天丢一个包。对于关键应用(如烟雾报警),这是不可接受的。

### 1.2 分集的基本原理

如果一条路径衰落,另一条路径可能不衰落。前提是多条路径的衰落必须独立。提供N个独立副本,所有副本同时深衰落的概率从P降低到P^N。例如单路径20dB衰落概率1%,双独立路径同时衰落概率0.01%。

### 1.3 分集增益

```
典型分集增益:
- 双天线选择分集: 约3-5dB
- 双天线MRC: 约5-7dB
- 三网关宏分集: 约8-10dB
- 四信道频率分集: 约6-8dB
```

## 2. 分集类型概述

| 分集类型 | 独立性来源 | 实现复杂度 | IoT适用性 |
|---|---|---|---|
| 空间分集 | 不同位置 | 中等 | 网关侧适用 |
| 频率分集 | 不同频率 | 低 | 广泛使用 |
| 时间分集 | 不同时间 | 低 | 广泛使用 |
| 极化分集 | 不同极化方向 | 中等 | 网关侧适用 |
| 码分集 | 不同扩频码 | 高 | 特定协议 |

## 3. 空间分集

### 3.1 基本原理

在不同空间位置放置多根天线,由于位置不同,各天线经历的衰落独立。天线间距需大于半个波长: 868MHz时约17cm, 2.4GHz时约6.25cm。

### 3.2 微分集与宏分集

微分集(Micro-Diversity)在同一设备上使用多根天线,适合IoT网关(空间充足)。宏分集(Macro-Diversity)在不同地理位置放置多个接收点,LoRaWAN天然支持此架构。

### 3.3 增益对比

```
选择分集(选最强信号):
- 2天线: 约3dB / 3天线: 约5dB / 4天线: 约6dB

MRC(最大比合并):
- 2天线: 约5dB / 3天线: 约7dB / 4天线: 约8.5dB
```

## 4. 频率分集

### 4.1 基本原理

在不同频率上发送相同数据,如果频率间隔大于相干带宽,各频率上的衰落独立。

### 4.2 LoRaWAN的频率分集

```
LoRaWAN EU868频段:
- 8个上行信道, 间隔200kHz
- 每次上行随机选择一个信道
- 重传时选择不同信道增加独立性

蓝牙经典:
- 79个信道, 1600跳/秒
- 窄带干扰只影响1/79的时间
```

## 5. 时间分集

### 5.1 基本原理

在不同时间发送相同数据,如果时间间隔大于相干时间,各次传输经历独立衰落。

### 5.2 IoT中的时间分集

静态IoT设备相干时间很长,短时间内重传可能遇到相同衰落。解决方案是结合频率分集(不同时间加不同频率重传)。大多数IoT数据容忍延迟,可以充分利用时间分集。

```
时间分集的实现方式:
- 简单重传(ARQ): 丢包后等待一段时间重发
- 交织(Interleaving): 将数据在时间上打散
- HARQ: 结合FEC和ARQ, 合并多次传输

LoRaWAN的时间分集:
- 确认帧: 未收到ACK则在不同时间+不同频率重传
- NbTrans: 可配置重传次数(1-15)
- 每次重传随机选择信道, 同时获得时间+频率分集
```

### 5.3 NB-IoT的重复传输

```
NB-IoT覆盖增强:
- 最多128次重复传输
- 理论增益: 10*log10(128) = 21dB
- 覆盖等级: CE0(无重复) -> CE2(最大重复,如深地下室)
```

## 6. 合并技术

### 6.1 三种合并方式

```
选择合并(Selection Combining):
- 从多个副本中选择信号最强的一个
- 不需要相位信息, 实现最简单
- 2分支增益: 约3dB

最大比合并(MRC):
- 每个分支乘以SNR权重后求和
- 数学上最优线性合并
- 合并后SNR = 各分支SNR之和
- 2分支增益: 约5dB

等增益合并(EGC):
- 各分支等权重相加(需相位对齐)
- 性能接近MRC, 实现更简单
- 2分支增益: 约4.5dB
```

### 6.2 合并技术对比

| 技术 | 复杂度 | 需要信息 | 性能(2分支) |
|---|---|---|---|
| 选择合并 | 低 | 各分支SNR | 约3dB |
| 等增益合并 | 中 | 各分支相位 | 约4.5dB |
| 最大比合并 | 高 | SNR+相位 | 约5dB |

对于IoT网关,MRC是首选因为它提供最优性能,且网关有足够的处理能力。对于低功耗终端设备,选择合并更实际因为它不需要相位信息,硬件最简单。

## 7. LPWAN中的宏分集

### 7.1 LoRaWAN的宏分集架构

LoRaWAN终端广播上行数据,所有覆盖范围内的网关都接收。网络服务器收集所有副本,去重后取最佳。终端无需知道有多少网关在接收。

### 7.2 宏分集的可靠性提升

```
假设单网关丢包率P=5%:
- 1网关: 5%丢包
- 2网关: 0.25%丢包
- 3网关: 0.0125%丢包

从95%可靠性提升到99.99%, 只需3个网关
```

### 7.3 宏分集的额外好处

除了抗衰落,宏分集还提供覆盖冗余(单网关故障不影响服务)、定位能力(TDOA定位)、负载均衡和渐进部署能力。

## 8. LoRaWAN的综合分集策略

LoRaWAN组合了多种分集: 频率分集(8信道随机选择)、时间分集(确认帧重传)、空间/宏分集(多网关部署)、扩频增益(CSS调制SF12约20dB处理增益)。终端完全透明,无需额外功耗。

```
LoRaWAN分集层次:
第一层: CSS扩频增益(SF7-SF12, 21-36dB)
第二层: 频率分集(8信道随机选择)
第三层: 宏分集(多网关独立接收)
第四层: 时间分集(确认帧重传)
第五层: ADR自适应(根据信道质量调整SF)

终端无感知: 只需发射, 网络侧处理所有分集合并
```

## 9. NB-IoT的分集机制

### 9.1 多层次分集

- 频率分集: 窄带但重复传输可在不同子载波
- 时间分集: HARQ重传加最多128次重复
- 空间分集: 基站多天线MIMO(2T2R或2T4R)

### 9.2 覆盖增强模式

```
NB-IoT各信道均支持重复:
- NPRACH(随机接入): 前导重复
- NPUSCH(上行数据): 最多128次重复
- NPDCCH(控制): 重复传输
- NPDSCH(下行数据): 重复传输
```

每次重复相当于一次独立观测,接收端合并后等效于分集增益。128次重复理论上提供10*log10(128)=21dB的增益,代价是数据速率降低128倍。这种以极端的低速换取极高可靠性的策略,使NB-IoT能覆盖地下车库和深层室内场景。

### 9.3 NB-IoT与LoRaWAN分集对比

NB-IoT的分集主要靠重复传输(时间分集),需要大量空中时间;LoRaWAN的分集主要靠多网关(宏分集),不增加单设备的空中时间。NB-IoT的优势在于基站多天线MIMO提供空间分集;LoRaWAN的优势在于频率跳变和CSS扩频增益。两者在覆盖增强策略上有本质区别: NB-IoT用时间换可靠性,LoRaWAN用基础设施冗余换可靠性。

## 10. IoT网关的天线分集

### 10.1 双天线网关设计

```
设计参数:
- 天线间距: > 半波长(868MHz时 > 17cm)
- 合并方式: 选择合并或MRC
- 预期增益: 3-5dB

成本分析:
- 额外一根天线: 约5-20美元
- 覆盖面积增加40-70%
```

### 10.2 与增加网关的对比

```
方案A: 额外部署一个新网关
- 成本: 200-500美元, 增益8-10dB

方案B: 现有网关升级为双天线
- 成本: 15-50美元, 增益3-5dB

结论: 先升级天线(低成本), 覆盖仍不足时再增加网关
```

## 11. 极化分集

### 11.1 基本原理

使用两根正交极化的天线(垂直加水平),衰落通常独立。两根天线可以在同一位置(节省空间),提供约3-4dB分集增益。

### 11.2 与空间分集的组合

在实际网关中可同时使用: 2根天线间距17cm加正交极化,获得空间加极化的联合增益5-7dB,是单网关能获得的最大微分集增益。

### 11.3 极化分集的适用场景

极化分集特别适合IoT网关空间有限但需要分集的场景。室内环境中极化散射严重(墙壁反射会旋转极化方向),正交极化的独立性很好。工业环境中金属结构导致极化旋转明显,极化分集效果显著。城市部署中建筑物反射同样导致极化混合,使得极化分集成为有效手段。

## 12. 实际部署案例

### 12.1 场景描述

```
工业IoT部署在金属结构密集的工厂:
- 面积: 200m x 100m生产车间
- 障碍物: 大量金属设备、管道、货架
- 衰落类型: 瑞利衰落, 深衰落频繁
- 设备: 150个温湿度/振动传感器
```

### 12.2 分集方案

```
部署方案:
1. 宏分集: 3个LoRaWAN网关, 分布在车间不同位置
2. 天线分集: 每个网关配备双天线(间距20cm)
3. 频率分集: 使用全部8个上行信道
4. 时间分集: 确认帧模式, 最多2次重传
```

### 12.3 性能结果

```
无分集基准(单网关单天线单信道): 丢包率约15%
+频率分集(8信道): 丢包率降至8%
+天线分集(双天线MRC): 丢包率降至4%
+宏分集(3网关): 丢包率降至0.06%
+时间分集(2次重传): 丢包率降至0.001%

最终结果: 99.9%数据包送达率
```

### 12.4 经验总结

单一分集技术不够,需要多种分集组合。宏分集效果最显著但成本最高; 频率分集几乎零成本(协议层面支持); 天线分集性价比高(少量硬件成本换显著改善)。

### 12.5 成本效益分析

```
分集方案成本效益排序(性价比从高到低):
1. 频率分集: 零硬件成本, 8信道随机选择, 协议内建
2. 天线分集: 15-50美元/网关, 3-5dB增益
3. 极化分集: 20-60美元/网关, 3-4dB增益(可与天线分集叠加)
4. 时间分集: 零硬件成本, 以延迟换可靠性
5. 宏分集: 200-500美元/网关, 8-10dB增益, 最显著

推荐策略: 先启用协议内建分集(频率+时间),
再升级网关天线(双天线+极化),
最后增加网关(宏分集)
```

## 总结

分集技术是对抗无线衰落、提高IoT链路可靠性的核心方法。其本质是提供信号的多个独立副本,使得所有副本同时衰落的概率大大降低。

在IoT系统中,最实用的分集组合是: 频率分集(协议层面,几乎零成本)、宏分集(多网关部署,LoRaWAN天然支持)、天线分集(网关双天线,低成本高回报)、以及时间分集(重传机制,以延迟换可靠性)。

实际部署表明,通过合理组合多种分集技术,即使在金属结构密集的恶劣射频环境中,也能实现99.9%以上的数据包送达率。
