# 认知无线电频谱感知在IoT中的应用前景
> **难度**：🔴 高级 | **领域**：频谱管理 | **阅读时间**：约 22 分钟

## 引言

想象你住在一栋公寓楼里，楼里有 100 间房，但任何时候只有 20 间房亮着灯。如果物业管理规定每户只能用自己那间房，那 80 间空房就白白浪费了。但如果允许住户在空闲房间里活动，整栋楼的利用率就大幅提升。

无线电频谱就是这栋"公寓楼"。频谱是有限的自然资源，但研究表明，已分配频段的平均利用率不到 20%。大量的授权频段在时间和空间上存在丰富的空闲机会。认知无线电 (Cognitive Radio, CR) 的核心思想就是：让无线设备像灵活的住户一样，感知哪些频段当前空闲，在不干扰授权用户 (primary user) 的前提下临时使用这些频段。

对于面临频谱稀缺困境的 IoT 来说，认知无线电提供了一条打开新频谱大门的路径。本文将系统介绍频谱感知技术、动态频谱接入机制及其在 IoT 中的应用前景。

## 一、频谱稀缺问题

### ISM 频段的拥挤困境

IoT 设备大量部署在免授权的 ISM 频段 (Industrial, Scientific, Medical band)，如 433MHz、868MHz、915MHz、2.4GHz。这些频段无需申请许可，但也意味着所有人都可以自由使用，导致日益严重的拥挤问题。

随着 IoT 设备数量呈指数级增长，ISM 频段上的干扰日益严重。特别是在 2.4GHz 频段，WiFi、蓝牙、Zigbee 以及其他 IoT 协议相互竞争有限的带宽，碰撞概率急剧上升，通信可靠性和能效随之下降。

### 授权频段的低利用率

与 ISM 频段的拥挤形成鲜明对比的是，大量已分配的授权频段 (如电视广播频段、蜂窝频段的部分子带) 在很多时间和地点处于空闲状态。多项测量研究显示，已授权频段的平均频谱利用率低于 20%，某些地区的某些时段甚至低于 5%。

这种"频谱稀缺但频谱浪费"的矛盾，正是认知无线电技术产生的根本动因。

## 二、认知无线电概念

### 定义与核心能力

认知无线电是一种能够感知周围射频 (RF) 环境并自适应调整其工作参数的无线通信系统。其核心能力包括：

频谱感知：检测当前频段是否被授权用户占用，发现可用的"频谱空洞" (spectrum holes / white spaces)。

机会式传输：在检测到空闲频段时，临时使用该频段进行通信，不干扰授权用户。

快速退出：当授权用户重新出现时，迅速腾出频段，切换到其他空闲信道。

智能频谱共享：通过学习历史占用模式，预测未来可用性，优化频谱使用策略。

### 认知无线电循环

```
认知无线电工作循环:

1. 观察 (Observe)
   感知频谱状态 -> 检测可用频段

2. 决策 (Decide)
   选择最佳可用信道 -> 考虑干扰约束和业务需求

3. 行动 (Act)
   在选定信道上传输数据 -> 动态配置射频参数

4. 学习 (Learn)
   记录频谱占用模式 -> 优化未来决策

5. 退出 (Vacate)
   检测到授权用户返回 -> 立即腾出信道
```

## 三、频谱感知技术

### 能量检测

能量检测是最简单的频谱感知方法。其原理是测量接收信号的功率，与预设阈值比较：超过阈值认为频段被占用，低于阈值认为空闲。

```python
# 能量检测伪代码
import numpy as np

def energy_detection(signal, threshold, N):
    """
    signal: 接收信号采样
    threshold: 判决阈值
    N: 采样点数
    """
    energy = np.sum(np.abs(signal[:N])**2) / N
    if energy > threshold:
        return "OCCUPIED"  # 频段被占用
    else:
        return "FREE"       # 频段空闲

# 优点: 不需要信号的先验知识, 计算简单
# 缺点: 低信噪比下性能急剧恶化
#       无法区分授权用户和其他干扰源
```

能量检测的优点是不需要知道授权信号的任何先验信息，计算复杂度低，适合资源受限的 IoT 设备。缺点是在低信噪比下检测性能急剧下降，且无法区分授权用户信号和噪声/干扰。

### 匹配滤波检测

匹配滤波器将接收信号与已知的授权信号模板进行相关运算，在加性高斯白噪声信道下是最优检测器。

匹配滤波检测的缺点是需要知道授权信号的完整先验信息 (如调制方式、帧结构、同步序列等)，并且需要专门的接收电路来处理每种信号类型，增加了硬件复杂度。

### 循环平稳特征检测

调制后的信号通常具有周期性统计特征 (循环平稳性)，而噪声没有这种特性。循环平稳特征检测通过分析信号的循环自相关函数或循环谱来识别授权信号。

这种方法能够在低信噪比下有效工作，且能区分不同类型的信号。但其计算复杂度高，需要较长的观测时间，对 IoT 设备的处理器和能耗提出了挑战。

| 感知方法 | 先验知识需求 | 计算复杂度 | 低SNR性能 | 适用性 |
|----------|-------------|-----------|-----------|--------|
| 能量检测 | 无 | 最低 | 差 | 最广 |
| 匹配滤波 | 完整信号信息 | 中 | 最优 | 受限 |
| 循环平稳 | 部分特征 | 高 | 好 | 中等 |

## 四、TV 白空间与 IoT

### TV 频段的优势

电视广播频段 (470-790MHz, UHF 频段) 具有优异的传播特性：波长较长，绕射能力强，穿透建筑物的损耗小，传播距离远。这些特性使其对 IoT 应用极具吸引力。

由于数字电视的频道划分和地域覆盖，并非所有 TV 频道在每个地点都被使用。这些空闲的 TV 频道被称为 TV 白空间 (TV White Spaces, TVWS)。

### TVWS 标准

IEEE 802.22 是第一个基于认知无线电的无线区域网络 (WRAN) 标准，工作在 TV 频段，覆盖半径可达数十公里，适合农村宽带接入和广域 IoT 应用。

IEEE 802.11af (WiFi HaLow 的前身之一) 将 WiFi 扩展到 TV 白空间，提供更大的覆盖范围和更好的穿透能力。

在监管层面，美国 FCC 和英国 Ofcom 都允许在 TV 频段进行次要使用，但要求设备具备频谱感知能力或使用地理频谱数据库查询可用信道。

## 五、频谱数据库方法

### 工作原理

频谱数据库方法避免了直接进行频谱感知的困难。设备通过网络连接查询在线频谱数据库，获取当前位置可用的信道列表。

数据库中存储了所有授权发射机的信息，包括位置、发射功率、覆盖范围等。当设备查询时，数据库根据设备的位置坐标计算出哪些信道在该位置可用。

```
频谱数据库查询流程:

1. IoT 设备获取自身 GPS 坐标
2. 向频谱数据库发送查询请求 (位置 + 设备参数)
3. 数据库计算:
   - 该位置可用的 TV 信道列表
   - 允许的最大发射功率
   - 可用时间窗口
4. 设备收到可用信道列表
5. 选择合适信道开始传输
6. 定期重新查询 (通常每 60 秒)
```

### 优势与局限

频谱数据库方法比直接感知更可靠，因为它基于授权发射机的已知信息而非噪声中的信号检测。FCC 和 ETSI 已批准其作为 TVWS 的主要合规方案。

局限性在于：需要网络连接来访问数据库 (但 IoT 设备可能正是为了获得网络连接才需要频谱)；查询结果可能不够实时，无法捕捉动态变化的频谱占用；数据库信息需要定期更新维护。

## 六、IoT 动态频谱接入

### 接入流程

IoT 设备的动态频谱接入 (Dynamic Spectrum Access, DSA) 流程如下：

```
IoT 动态频谱接入:

步骤 1: 频谱感知或数据库查询
  -> 获取当前可用信道列表

步骤 2: 信道选择
  -> 根据业务需求选择最佳信道
  -> 考虑因素: 带宽、干扰水平、预期占用时间

步骤 3: 数据传输
  -> 在选定信道上发送数据
  -> 同时持续监测信道状态

步骤 4: 频谱切换
  -> 若检测到授权用户出现
  -> 立即停止传输
  -> 切换到另一个可用信道继续通信

步骤 5: 记录与学习
  -> 记录信道占用模式
  -> 优化未来信道选择策略
```

### 对 IoT 的价值

动态频谱接入使 IoT 设备能够在不申请新频谱许可的情况下获得更多可用带宽。这意味着在 ISM 频段过于拥挤时，IoT 设备可以"借用"空闲的授权频段，大大增加了通信容量。

## 七、协作感知

### 单节点感知的局限

单个 IoT 设备的频谱感知面临几个根本性困难：

隐藏节点问题：IoT 设备可能因为遮挡而无法检测到授权发射机的信号，误判频段空闲，从而对授权接收机造成干扰。

低信噪比：在远距离或深衰落条件下，授权信号到达 IoT 设备时已极度微弱，难以可靠检测。

多径衰落：无线信道的多径效应导致信号强度波动，造成检测不稳定。

### 协作感知机制

协作感知通过多个 IoT 设备共享各自的感知结果来提高整体检测可靠性：

```
协作感知架构:

IoT 设备 1 --\
IoT 设备 2 ----> 融合中心 (Fusion Center)
IoT 设备 3 --/       |
                      |
              综合所有感知结果
                      |
              做出最终频谱占用判决
```

融合中心可以是一个专门的汇聚节点或 IoT 网关。它收集各设备的本地感知结果 (可以是硬判决 0/1 或软判决概率值)，通过一定的融合规则 (如多数投票、OR 准则、似然比检验) 做出最终判决。

协作感知的代价是感知报告本身需要消耗带宽和能量。对于能量受限的 IoT 设备，需要优化协作策略，在检测性能和开销之间取得平衡。

## 八、感知-吞吐量权衡

### 问题的本质

频谱感知和数据显示传输不能同时进行 (在单天线设备上)，因此存在一个根本性的时间分配问题：

```
帧结构:

|<--- 感知 --->|<--- 数据传输 --->|
|    Ts        |       Td          |
|<------------- 帧长 T ----------->|

感知时间 Ts 越长:
  -> 检测精度越高
  -> 但数据传输时间 Td = T - Ts 越短
  -> 吞吐量下降

感知时间 Ts 越短:
  -> 检测精度越低
  -> 误检/漏检概率上升
  -> 但有更多时间传输数据
```

### IoT 的特殊约束

对于 IoT 设备，感知-吞吐量权衡还有一个额外的维度：能量。感知过程本身需要消耗能量 (接收机处于工作状态)，而 IoT 设备的电池容量有限。

因此，IoT 场景下的优化目标是三重的：在满足检测概率要求的前提下，同时最大化吞吐量和最小化能耗。通常需要将感知开销压缩到极低水平，这对感知算法的效率提出了很高要求。

## 九、能量高效的感知策略

### 睡眠-感知-传输调度

IoT 设备不能持续进行频谱感知，需要合理的调度策略：

```
节能调度方案:

周期 1: 睡眠 (低功耗) -> 唤醒 -> 感知 -> 传输 -> 睡眠
周期 2: 睡眠 (低功耗) -> 唤醒 -> 感感 -> 传输 -> 睡眠
...

关键参数:
  - 感知频率: 每隔多久感知一次
  - 感知时长: 每次感知持续多久
  - 传输时长: 感知后传输多长时间

优化目标: 最小化 (感知能耗 + 传输能耗) / 有效数据量
```

### 协作节能

在协作感知中，可以采用轮值策略：不是所有设备都同时感知，而是分组轮流感知，其他设备进入睡眠模式。感知结果在组内共享，所有设备都能获得频谱状态信息，但只有部分设备承担感知能耗。

### 唤醒无线电

一种极端的节能方案是使用独立的超低功耗唤醒无线电 (Wake-up Radio)。主无线电平时处于深度睡眠状态，唤醒无线电以极低功耗持续监听特定信号。当检测到需要通信的事件时 (如授权用户离开、有数据要收)，唤醒无线电触发主无线电开始感知和传输。

## 十、IoT 面临的挑战

### 计算复杂度

频谱感知算法 (特别是循环平稳特征检测) 需要大量的 FFT 运算和统计分析，对 IoT 设备上资源有限的 MCU 来说是沉重负担。

### 能量开销

感知过程需要接收机处于工作状态，持续消耗能量。对于设计目标是数年电池寿命的 IoT 设备，感知开销可能占能耗预算的很大比例。

### 隐藏节点问题

即使采用协作感知，隐藏节点问题也不能完全消除。如果所有协作设备都被遮挡而无法检测到授权发射机，协作感知仍然会产生误判。

### 实时性要求

监管机构要求认知无线电设备在检测到授权用户后必须在很短时间内 (通常数百毫秒) 腾出频段。这对 IoT 设备的处理速度和信道切换速度提出了挑战。

### 标准化和监管

认知无线电的规则和标准仍在演进中，不同国家和地区的监管政策差异较大。实际部署的 IoT 认知无线电系统仍然有限，大规模商用还面临不确定性。

## 十一、实际案例：智慧城市 IoT 利用 TV 白空间

考虑一个部署在中等规模城市的环境监测 IoT 网络：

```
场景设定:
  - 城市面积: 约 200 平方公里
  - 环境传感器数量: 5000 个 (空气质量、噪声、温湿度)
  - 通信需求: 每个传感器每 15 分钟上报一次, 数据量约 200 bytes
  - 现有 ISM 频段: 已被 WiFi、蓝牙、Zigbee 等大量占用, 干扰严重

方案: 利用 TV 白空间

基础设施:
  - 部署城市级频谱数据库, 实时更新 TV 信道占用状态
  - 各区域的 IoT 网关查询数据库获取可用 TVWS 信道

传感器端:
  - 传感器通过本地网关间接获取可用信道信息
  - 使用 470-790MHz TVWS 信道进行上行传输
  - 发射功率: 100mW (符合 FCC TVWS 规定)

效果对比:
  ISM 433MHz:
    - 通信距离: 约 2km (受建筑物遮挡)
    - 丢包率: 15% (干扰严重)
    - 需要 15 个网关覆盖全城

  TVWS 470MHz:
    - 通信距离: 5km+ (低频段传播优势)
    - 丢包率: 3% (干扰少)
    - 仅需 5 个网关覆盖全城
    - 建筑物穿透损耗比 433MHz 低约 3-4dB

  频谱容量:
    - ISM 433MHz 可用带宽: 约 1.7MHz
    - TVWS 可用带宽: 约 20-40MHz (视地点而定)
    - 可用频谱增加 10 倍以上

运营效果:
  - 网关数量减少 67% -> 部署成本大幅降低
  - 丢包率从 15% 降至 3% -> 数据可靠性显著提升
  - 当 TV 广播频率调整时, 传感器自动切换信道
  - 整体网络容量提升数倍, 支持未来扩展
```

这个案例展示了认知无线电技术如何为 IoT 带来更多频谱资源和更好的传播特性，尽管实际部署中仍需解决数据库维护、信道切换延迟等工程问题。

## 总结

认知无线电频谱感知技术为 IoT 打开了一扇通往更多频谱资源的大门。本文系统讨论了以下要点：

频谱稀缺是 IoT 面临的严峻挑战，ISM 频段日益拥挤而授权频段利用率低下，这一矛盾催生了认知无线电技术。

频谱感知是认知无线电的基础能力，能量检测简单但性能有限，匹配滤波最优但需要先验知识，循环平稳检测兼顾性能和适用性但复杂度高。

TV 白空间为 IoT 提供了优质的低频段频谱资源，配合频谱数据库方法可以实现可靠的动态接入，FCC 和 ETSI 已将其纳入监管框架。

协作感知通过多设备信息共享提高了检测可靠性，但带来了额外的通信开销，需要在检测性能和能量消耗之间权衡。

IoT 场景下的核心挑战在于能量约束——感知开销必须极低才能保证电池寿命，睡眠调度、轮值感知和唤醒无线电是主要的节能策略。

实际案例表明，利用 TV 白空间的 IoT 网络在覆盖范围、抗干扰和频谱容量方面显著优于纯 ISM 频段方案，但标准化和大规模商用仍需时间。
