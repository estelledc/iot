# 电视白频谱TVWS在IoT中的大范围连接
> **难度**: 中级 | **领域**: 频谱利用 | **阅读时间**: 约 20 分钟

## 引言

想象一座城市有100条公交车道。早晚高峰时20条繁忙，但其余80条大部分时间空着——没公交车跑。普通车辆被禁止使用公交车道，即使空着也不能开。每天浪费大量道路资源。

如果政策允许: 公交车道空闲时普通车辆可临时使用，一旦公交车来了立刻让出——这就能大幅提升道路利用率。电视白频谱(TV White Spaces, TVWS)就是这个思路: 电视台的专用频率很多时候很多地方没有节目在播出(数字化转型后更多频道空出)，与其让这些频率白白空着，不如允许IoT设备在确认没有电视广播时临时使用。

这对IoT意义重大: 电视频段(54-790MHz)传播特性优越，穿墙能力强、覆盖距离远，一座基站可覆盖10公里以上。而ISM频段(2.4GHz WiFi, 868/915MHz LoRa)在相同功率下覆盖距离短得多。TVWS给IoT打开了一扇通往"长距离、好穿透、免费频谱"的大门。

## 1. 什么是电视白频谱

### 1.1 频谱分配与空闲

```
电视频段(VHF/UHF):
  VHF: 54-216MHz (2-13频道)
  UHF: 470-790MHz (14-51频道)
  每个频道占6-8MHz带宽(各国标准不同)

实际使用:
  某城市分配40个电视频道(广电)
  实际在播: 12个(数字电视)
  保护频道: 4个(防相邻干扰)
  完全空闲: 24个频道 × 6MHz = 144MHz空闲频谱!

白频谱(White Space):
  = 已分配给电视但实际未使用的频率
  = 时间+地理位置相关(这里空那里不空)
```

### 1.2 为什么空出这么多

模拟到数字转型: 原来1个模拟频道(6MHz)→1个节目; 现在1个数字频道→4-8个高清节目。数字化需要的频道数大减。频谱拍卖: 多国将700MHz+重新拍给移动运营商(4G/5G)，剩余频道更宽松。地理复用: 电视台覆盖有限，北京用的频道在天津可能没电视台用。

### 1.3 TVWS的法律框架

```
各国监管框架:

美国(FCC):
  2010年允许TVWS设备使用
  分为固定设备(最大4W EIRP)和便携设备(最大100mW)
  必须查询认证的TVWS数据库获取可用频道
  2020年颁布6GHz新规则更进一步

英国(Ofcom):
  2015年开放TVWS使用(首个欧洲国家)
  要求设备每小时更新可用频道列表
  数据库提供商: 多家认证

中国:
  研究阶段，多地试点(广州、深圳)
  600MHz-700MHz频段讨论中
  政策相对保守，仍在制定中
```

## 2. TVWS的物理传播优势

### 2.1 频率越低传播越远

```
自由空间路径损耗公式:
  FSPL(dB) = 20*log10(d) + 20*log10(f) + 32.45
  d=距离(km), f=频率(MHz)

同样距离(5km)、同样功率:
  600MHz: FSPL = 101.0 dB
  868MHz: FSPL = 104.2 dB
  2.4GHz: FSPL = 113.0 dB

600MHz vs 2.4GHz: 少12dB损耗
12dB意味着: 相同功率覆盖面积扩大约16倍!
```

### 2.2 穿透与绕射

低频信号波长更长(600MHz→50cm, 2.4GHz→12.5cm)。穿墙损耗更小(600MHz穿混凝土墙约损失5dB, 2.4GHz约15dB)。绕射能力更强(可绕过建筑物、树木、地形)。对IoT意义: 适合室内传感器覆盖、农业田野、城市密集环境。

### 2.3 实测对比

| 场景 | TVWS(600MHz) | LoRa(868MHz) | WiFi(2.4GHz) |
|------|-------------|-------------|--------------|
| 视距(flat) | 15km | 10km | 0.3km |
| 城市NLOS | 5km | 2km | 0.05km |
| 室内穿透2墙 | 2km | 0.8km | 0.02km |
| 森林(密林) | 3km | 1km | 0.03km |

TVWS在所有场景均优于高频方案，差距在非视距环境尤为显著。

## 3. TVWS接入机制: 地理位置数据库

### 3.1 核心问题

TVWS频谱是"借来的"——电视台是主要用户(Primary User)，IoT设备是次要用户(Secondary User)。次要用户绝不能干扰主要用户。如何确保IoT设备只用确实空闲的频道?

### 3.2 三种接入方式

```
方式1: 感知检测(Spectrum Sensing)
  设备自己监听，没听到电视信号就认为空闲可用
  问题: 隐藏终端(你听不到不代表那里没电视)
  问题: 灵敏度要求极高(-114dBm)增加设备成本
  状态: 不能单独使用，仅作辅助

方式2: 地理位置数据库(Geolocation Database)
  设备向认证数据库报告自己的GPS位置
  数据库返回此位置此时可用的频道列表和功率限制
  设备只在批准的频道和功率下工作
  状态: FCC/Ofcom要求的主要方式

方式3: 信标(Beacon)
  基站广播可用频道信息
  终端收到后按指示工作
  状态: 大部分标准未采纳
```

### 3.3 数据库工作流程

```
TVWS设备接入流程:
  1. 设备开机 → 获取GPS位置(精度±50m)
  2. 连接TVWS数据库(通过已有网络: 以太网/蜂窝)
     POST /whitespace/available
     { "location": {"lat": 39.9, "lon": 116.4},
       "device_type": "fixed",
       "height_m": 10 }
  3. 数据库查询:
     - 此位置周围的电视发射台(覆盖范围模型)
     - 保护区域(电视台周围安全距离)
     - 其他TVWS设备已注册使用
  4. 返回可用频道列表:
     { "channels": [
         {"ch": 21, "freq_mhz": 474, "max_power_dbm": 36},
         {"ch": 25, "freq_mhz": 506, "max_power_dbm": 30},
         {"ch": 33, "freq_mhz": 570, "max_power_dbm": 36}
       ],
       "valid_until": "2024-01-15T12:00:00Z" }
  5. 设备选择频道开始通信
  6. 每30分钟~24小时重新查询(各国要求不同)
```

### 3.4 数据库提供商

已认证: Google Spectrum Database、Microsoft Research White Spaces、Nominet(英国)、Fairspectrum(芬兰)。共同特点: 维护完整的电视发射台数据库+地形模型+传播模型，可用频道的计算实时完成。

## 4. TVWS标准与协议

### 4.1 IEEE 802.22

第一个基于认知无线电的无线标准。基站覆盖半径最大100km(典型30km)。下行OFDMA上行SC-FDMA(类似LTE)。面向固定终端的宽带接入(类似DSL替代)。数据率: 典型4.5-23Mbps(比IoT需求高得多)。

### 4.2 IEEE 802.11af

"Super WiFi" —— WiFi协议适配到TVWS频段。比标准WiFi覆盖大10倍(因为低频)。适合: 农村宽带接入、校园网扩展。与IoT不直接相关但证明TVWS可用于标准协议。

### 4.3 Weightless-W

专为TVWS IoT设计的协议: 窄带(6kHz每子信道)、极低功耗终端设计、自适应调制(DBPSK到16QAM)、上行FDMA+TDMA。设计目标: 和LoRa类似的IoT使用场景但利用TVWS频段获得更好覆盖。项目后来发展成Weightless-P(转向ISM频段)，但技术方案仍有参考价值。

### 4.4 Microsoft Airband

微软推动的TVWS实际部署项目: 用TVWS为美国农村未连接地区提供宽带(至今超20国部署)。证明了TVWS大规模商用可行、设备成本可控($200-500)，同时回传通道可承载大量IoT数据。

## 5. IoT场景适配

### 5.1 精准农业

```
场景: 大型农场(5000亩) + 分布式传感器
  温湿度(每100亩1个) = 50个
  土壤(每50亩1个) = 100个
  气象站(每500亩1个) = 10个
  合计: 160个传感器分布在5km×7km区域

TVWS方案:
  1座TVWS基站(农场中心，杆高15m)
  覆盖半径: 5-8km(600MHz在农田环境)
  1座基站覆盖全部160个传感器!
  每传感器每15分钟上报100字节
  网络容量: 600MHz频道6MHz宽=Mbps级(绰绰有余)

对比LoRa方案(868MHz):
  覆盖半径: 2-3km(农田)
  需3-4个网关才能覆盖全场
  网关间需回传连接

TVWS优势: 基站少(1 vs 4)、部署成本低、免许可频谱(IoT不付频谱费)
```

### 5.2 智慧城市

城市环境TVWS优势: 穿透力解决密集建筑遮挡; 覆盖广减少基站数; 带宽足够(6MHz每频道)支持较多设备和视频数据。

应用: 地下管网监测(信号穿地面+墙)、路灯/停车位传感器(广覆盖减少网关)、环境监测站(散布各处的PM2.5/噪声传感器)、应急通信(基础设施损坏时TVWS覆盖大)。

### 5.3 野生动物追踪

```
需求: 追踪大型野生动物(非洲草原/中国东北林区)
  GPS项圈每小时上报位置
  覆盖区域: 数百平方公里

TVWS方案:
  基站部署在保护区瞭望塔(已有供电)
  频段: 470-510MHz(UHF低端)
  1座基站覆盖半径约15-20km(平坦草原)
  3-4座覆盖整个保护区(500km²)

项圈设计:
  TVWS终端(简化版): 仅上行突发发送
  固定频道(保护区内电视频道固定可用)
  TX功率: 100mW (20dBm)
  电池寿命: 2年(每小时发送10字节GPS数据)
```

## 6. TVWS与其他LPWAN技术对比

| 维度 | TVWS IoT | LoRaWAN | NB-IoT | Sigfox |
|------|----------|---------|--------|--------|
| 频段 | 470-790MHz(认知) | 868/915MHz(ISM) | 运营商频段(授权) | 868MHz(ISM) |
| 频谱成本 | 免费(需数据库) | 免费 | 付费(SIM卡月租) | 付费(订阅) |
| 覆盖(城市) | 5km | 2km | 3km | 5km |
| 覆盖(乡村) | 15km | 10km | 15km | 30km |
| 带宽 | 6MHz(可分) | 125-500kHz | 180kHz | 100Hz |
| 上行速率 | 可达数Mbps | 0.3-50kbps | 200kbps | 100bps |
| 穿透力 | 优 | 良 | 良 | 良 |
| 设备成本 | $10-30(未规模化) | $3-10 | $5-15 | $5 |
| 生态成熟度 | 发展中 | 成熟 | 成熟 | 成熟 |

TVWS定位: 覆盖和穿透最优，带宽最大，适合ISM频段拥挤或覆盖不足的场景。生态成熟度是主要瓶颈。

## 7. 技术挑战

### 7.1 频谱可用性不确定

TVWS频道在时间和空间上变化: 新电视台开播某频道不可用、临时赛事转播占用频道、移动设备到新位置可用频道变化。IoT设备需要: 快速切换频道(ms级)、维持与数据库的连接(需回传网络)、处理"没有可用频道"的极端情况。

### 7.2 设备接入数据库的鸡蛋问题

```
问题:
  TVWS设备要用TVWS频段通信
  → 先要知道哪些频道可用
  → 要查询地理位置数据库
  → 查询数据库需要网络连接
  → 但设备本身的网络就是TVWS...

解决方案:
  1. 固定基站通过有线/蜂窝连数据库(基站有网)
     终端通过基站获取可用频道信息(间接查询)
  2. 终端内置蜂窝模块仅用于数据库查询
     (成本增加但解耦了鸡蛋问题)
  3. 基站广播信标: 告知终端当前可用频道
     (类似方式3但由基站代为完成)
```

### 7.3 终端功耗与复杂度

TVWS终端比LoRa终端复杂: 需要更灵活的射频前端(支持多个频道切换)、GPS模块(定位用于数据库查询)、更复杂的MAC协议(认知无线电机制)、可能的频谱感知电路(辅助检测)。

这些增加了成本($10-30 vs LoRa $3)和功耗。对电池供电IoT设备是挑战。

### 7.4 监管不确定性

各国政策差异大: 美英已商用、欧洲多国进展不一、中国仍在研究阶段。IoT设备全球部署面临标准碎片化。

## 8. 实际部署案例

### 8.1 非洲农村宽带(Microsoft Airband)

背景: 非洲大量农村无互联网覆盖。TVWS基站部署在水塔/信号塔上，覆盖半径10-15km。为学校、诊所、农户提供宽带接入。同时承载IoT: 灌溉传感器数据上报到云端、太阳能水泵远程监控、气象数据采集。

### 8.2 英国乡村IoT试点

Ofcom支持的TVWS IoT试点: 牛津郡农场环境监测、湖区洪水监测传感器。频段: 470-790MHz UHF频道。结果: 单基站覆盖8km，穿过丘陵地形仍可通信。证明TVWS适合英国乡村地形(丘陵多、建筑分散)。

### 8.3 新加坡密集城市试验

高密度城市环境TVWS测试: 穿透高层建筑测试(20层)、地下室覆盖测试。结论: TVWS在10层以上建筑内仍可接收(WiFi通常3-4层即断)，地下2层停车场可覆盖(WiFi不行)。适合智慧建筑全覆盖场景。

## 9. TVWS IoT网络设计

### 9.1 网络架构

```
典型TVWS IoT网络:

  [TVWS数据库(云)] ← 互联网 → [TVWS基站(有线回传)]
                                     |
                         TVWS无线链路(470-790MHz)
                                     |
              [终端A]    [终端B]    [终端C] ...
              (传感器)   (传感器)   (网关)
                                     |
                              本地传感器网络(BLE/Zigbee)

基站:
  有线/光纤回传到互联网
  通过数据库获取当前可用TVWS频道
  向终端广播频道分配
  集中调度上下行

终端:
  按基站指示使用指定频道
  简化设计(不自行查数据库降低成本和功耗)
  可作为边缘网关转发其他短距协议数据
```

### 9.2 容量规划

一个6MHz TVWS频道可承载多少IoT设备?

```
假设:
  频道带宽: 6MHz
  调制: QPSK 1/2码率(保守，适合远距离)
  净吞吐: ~3Mbps(扣除开销)
  每设备每小时上报: 100字节 = 800bit
  每设备占用(含协议开销): ~2000bit = 2kbit/次
  每小时1次: 2kbit/3600s = 0.56bps/设备

单频道可支持设备数:
  3Mbps / 0.56bps = ~5,000,000 (理论)
  考虑MAC竞争效率50%: ~2,500,000
  考虑突发碰撞余量: 实际支持 ~100,000设备

结论: 1个TVWS频道支撑10万量级低速率IoT设备
```

### 9.3 频道聚合

多个可用频道联合使用: 高优先级设备占用完整频道(视频传感器)、低优先级设备共享窄带(温度/湿度)。动态分配: 空闲时聚合获得高速率，电视开播时退让部分频道。

## 10. 未来展望

### 10.1 与5G/6G融合

3GPP研究TVWS作为NR(新空口)补充频段。5G核心网+TVWS接入(频谱聚合)。动态频谱共享(DSS)技术使5G和TVWS共存更灵活。

### 10.2 AI驱动的频谱管理

AI预测模型: 预测各地各时TVWS频道可用性(基于历史数据)。IoT设备提前规划频道切换(减少中断)。联邦学习: 多设备协作学习频谱环境(不上传原始数据)。

### 10.3 中国市场机会

中国700MHz "黄金频段"已划给中国广电/中国移动(5G)。600MHz以下仍有空间讨论TVWS政策。广电数字化改造释放更多频谱资源。一旦政策明确——中国IoT设备体量全球最大——TVWS IoT规模化潜力巨大。

## 总结

电视白频谱为IoT提供了独特的频谱资源: 低频带来的物理传播优势(覆盖远、穿透强)在现有ISM频段方案中无法获得。通过地理位置数据库的认知接入机制确保不干扰电视广播。

TVWS IoT当前面临设备成本、监管碎片化、生态不成熟三大挑战，但随着频谱政策逐步开放、设备芯片化降低成本、AI优化频谱管理，TVWS有望成为ISM频段LPWAN的重要补充——特别是在广域覆盖、建筑穿透、农业偏远地区等ISM方案力不从心的场景中发挥不可替代的作用。
